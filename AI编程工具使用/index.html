    Gemin 使用

    1. AndroidStudio 和 IDEA 使用（最简单）

    AndroidStudio 和 IDEA 里集成了 Gemin 插件，可直接登录使用 ，深度集成。

    注意：插件升级到最新版，旧版和 IDE有兼容问题，用起来会有各种问题，最新更新应该是修复了

    2. gemini CLI 使用

    1. 需要提前安装 Node.js 和 npm

    2. 安装 CLI ` npm install -g @google/gemini-cli`

    3. 安装完成后在命令行输入 gemini 按照指示选择配置风格及登录方式

    4. 设置代理

      注：gemini CLI 没有单独设置代理的配置文件 需要在系统环境变量里设置全局代理。

    5. 配置完成后即可在命令行使用 命令 使用 gemini，gemini--help 查看命令

    6. 完成后就可以终端里使用命令行操作

    注：有基础免费额度使用，限制每分钟 60次，每天1000次

    codex 使用

    codex 有 chatgpt 官方的 应用，可使用官方，此处介绍 cli 使用方式 以及 插件使用方式

    1. codex CLI 使用

    1. 需要提前安装 Node.js 和 npm

    2. 安装 CLI npm install -g @openai/codex

    3. 安装完成后在命令行输入 codex 按照指示选择配置风格及登录方式

      账户登录可以免费使用基础版

      API 登录需要充值Token

    4. 设置代理,在 ·~/.codex/ 目录下找到 .env 文件，没有则新建一个，写入下面代理设置

      1
2

      HTTP_PROXY=http://127.0.0.1:你的 VPN 端口
HTTPS_PROXY=http://127.0.0.1:你的 VPN 端口

    5. 配置完成后即可在命令行使用 命令 使用 codex，claude --help 查看命令

    6. 完成后就可以终端里使用命令行操作

      注：默认使用的模型是 GPT-5.5, 不支持免费的账户登录模式，切换为其他模型

    2. Androidstudio 配置插件使用

    推荐插件： CC GUI
    支持 claude 和 codex ，后续会支持 gemin 和 opencode
    支持 MCP，skills，prompt导入，自定义模型，自定义agent，等

    1. 插件市场安装插件
    2. 插件设置里 安装 codex sdk
    3. 插件设置里 配置 providers ，使用本地授权 ~/.codex/config.toml 和 auth.json

    注：

    1. codex 有免费订阅，有额度限制
    2. 在账户处可以领取优惠， plus 版可以免费使用 1 个月（但需要海外支付卡绑定，可注册虚拟卡 ，但有开卡费用 ）

    claude 使用

    1. claude CLI 使用

    1. 需要提前安装 Node.js 和 npm
    2. 安装 CLI ` npm install -g @anthropic-ai/claude-code`
    3. 安装完成后在命令行输入 claude 按照指示选择配置风格及登录方式
    4. 设置代理
      1. 找到 ~/.claude/settings.json 文件
      2. 设置环境变量（配置代理，CLAUDE_API_KEY等）
    1
2
3
4

     "env": {
        "HTTP_PROXY": "http://127.0.0.1:你的VPN端口",
        "HTTPS_PROXY": "http://127.0.0.1:你的VPN端口",
    },
    1. 配置完成后即可在命令行使用 命令 使用 claude，claude --help 查看命令
    2. 完成后就可以终端里使用命令行操作

    2. Android Studio里配置插件

    同样使用 cc gui 插件

    1. 插件市场安装插件
    2. 插件设置 sdk 管理器里 安装 claucode sdk
    3. 插件设置供应商管理里 配置 providers ，使用本地settings.json授权 或 终端 CLI 登录授权

    注：无基础免费额度，需付费使用 claude 不支持中国 IP，即使使用代理也可能会封号。

    Skills

    Karpathy-Inspired Claude Code Guidelines

    仓库地址：github仓库

    claude 安装 此 skills : 按照官方仓库教程即可

    1. 启动 claude
    2. 添加市场 /plugin marketplace add forrestchang/andrej-karpathy-skills
    3. 安装插件 /plugin install andrej-karpathy-skills@karpathy-skills
    4. 重启 claude
    5. 运行指令查看结果 /skills 可以看到安装的可用的 skills

    codex 安装此 skills 方式：

    1. 启动 codex
    2. 使用命令 $Skill Installer andrej-karpathy-skills@karpathy-skills 安装
    3. 重启 codex
    4. 运行 /skills ，List Skills 查看安装的 skills

    注：最简单安装方式：下载 SKILL.md 文档解压到对应sklills文件夹中

    模型供应商 Provider

    以 codex 和 claude 为例，geimini等其他编程工具类似 更改供应商或者更改模型等实际是更改用户目录下 .codex 或者 .claude 目录下的配置文件 手动更改配置文件过于繁琐，每次更换都得更改。 所以推荐一个 CLI AI切换工具：cc switch,支持主流的多款 C LI 的配置一键切换。

    前面推荐的 CC GUI 插件也提供了 codex 和 claude 的供应商自定义和切换。

    下面给出 codex 和 claude 的第三方供应商的配置，一般各自供应商官网都有配置教程，可去官网查看。

    codex 供应商配置

    codex 更改的是 ~/.codex 目录下的 config.toml 和 auth.json 文件

    • cc switch：在 CC switch 里添加新的供应商，将下面示例的配置写入
    • cc GUI插件：在插件供应商里添加新的供应商，将下面示例的配置写入
