# Google Stitch 和 Figma 对比

| 功能             | Google Stitch                            | Figma                                            |
| :--------------- | :--------------------------------------- | :----------------------------------------------- |
| **主要输入方式** | 自然语言 / 语音 / 图片上传               | 在画布上手动设计                                 |
| **AI 生成**      | 从文字提示生成完整 UI（Gemini 2.5 Pro）  | Figma AI 辅助布局和内容                          |
| **学习曲线**     | 几分钟——描述你想要的即可                 | 需要数周到数月才能熟练                           |
| **实时协作**     | 有限（以单用户为中心）侧重于人与AI的协作 | 拥有成熟的实时多人协作、评论、版本历史和权限管理 |
| **设计系统**     | 通过 URL 或 DESIGN.md 导入               | 完整的组件库、令牌、变量                         |
| **原型制作**     | AI 生成的交互式原型                      | 手动原型连线，支持高级过渡效果                   |
| **代码输出**     | 可用于生产的 HTML + Tailwind CSS         | Dev Mode，带代码检查和片段                       |
| **语音输入**     | 支持——对画布说话，实时评价               | 不支持                                           |
| **插件生态**     | 无,处于早期阶段，生态尚不成熟            | 2,000+ 社区插件                                  |
| **版本控制**     | 代理管理的项目历史                       | 原生版本历史 + 分支                              |
| **开发交付**     | 直接导出代码                             | Dev Mode、设计令牌、CSS/iOS/Android              |
| **响应式设计**   | AI 生成的响应式变体                      | 手动自动布局和约束                               |
| **无障碍工具**   | 基本的 AI 建议                           | 插件 + 原生对比度检查                            |
| **文件格式**     | 基于 Web，可导出到 Figma/代码            | .fig 文件，支持导入/导出 SVG、PNG、PDF           |
| **团队管理**     | Google 账号                              | 组织、团队、权限、SSO、SCIM                      |
| **离线访问**     | 不支持                                   | 有限支持（桌面应用带同步）                       |
| **价格**         | 免费                                     | $0 - $90/席位/月                                 |

stitch优势：

- 构思速度
- 成本
- 对非设计师的可及性
- 语音交互
- 代码生成

Figma 优势：

- 大规模协作
- 设计系统和一致性
- 像素级精确度
- 插件生态系统
- 开发交付
- 企业功能

>**Stitch 主导从 0 到 1 的阶段，Figma 主导从 1 到 100 的阶段**

# 混合工作流

1. **从 Stitch 开始。** 花 10 分钟用文字描述生成十几个设计概念。大胆探索截然不同的方向。使用语音模式进行交互式头脑风暴。
2. **在 Stitch 中收窄。** 选出 2-3 个最佳方向。通过后续提示进一步完善。通过 DESIGN.md 导入你的设计系统，使其更贴近品牌一致性。
3. **导出到 Figma。** 使用 Stitch-to-Figma 插件将你选定的方向导入 Figma，保留图层和组件。
4. **在 Figma 中精炼。** 应用你的实际设计系统。将间距、排版和颜色调整到像素级精确。构建组件变体。使用高级交互连接原型。
5. **从 Figma 交付。** 使用 Dev Mode 为开发者提供所需的规格、令牌和资源。这个工作流将设计最早期阶段从数天压缩到数分钟，同时保持生产阶段的质量和严谨性。
