# RS-232 串行通信标准学习笔记

最后整理：2026-06-11

RS-232 是经典串行通信电气接口标准，常用于计算机、调试口、老式仪器、工业设备和嵌入式设备之间的点对点通信。它定义的是电气特性、信号线和接口行为，不定义上层业务报文含义。

## 协议定位

RS-232 更准确地说是物理层接口标准。工程中常说的“串口协议”往往混合了三件事：

| 层次 | 示例 | 说明 |
|---|---|---|
| 物理层 | RS-232、RS-485、TTL UART | 电压、电气连接、驱动能力 |
| 数据链路/字符帧 | UART 8N1、奇偶校验 | 起始位、数据位、停止位、波特率 |
| 应用层 | Modbus RTU、自定义帧 | 地址、功能码、命令、校验、业务语义 |

## 解决的问题

- 两个设备之间如何用串行方式传输字符或字节。
- 如何定义逻辑 0/1 对应的电压范围。
- 如何用 RTS/CTS、DTR/DSR 等控制线做硬件握手。

## 电气与连接特点

| 特点 | 说明 |
|---|---|
| 通信方式 | 点对点 |
| 信号类型 | 单端信号，以共同地为参考 |
| 距离 | 典型短距离，工程上常按十几米量级理解 |
| 抗干扰 | 弱于差分信号，工业现场长线不优先选 RS-232 |
| 全双工 | 常见 TXD/RXD 分线，可全双工 |

## 常见信号线

| 信号 | 作用 |
|---|---|
| TXD | 发送数据 |
| RXD | 接收数据 |
| GND | 信号地 |
| RTS/CTS | 硬件流控 |
| DTR/DSR | 数据终端/数据设备就绪 |
| DCD | 载波检测 |
| RI | 振铃指示 |

最小通信通常只需要 TXD、RXD、GND。两个 DTE 设备直连时常需要交叉线，也就是 A 的 TXD 接 B 的 RXD，A 的 RXD 接 B 的 TXD。

## UART 帧格式

RS-232 常承载 UART 异步串行数据。常见配置 `9600 8N1` 表示：

| 部分 | 含义 |
|---|---|
| 9600 | 波特率 |
| 8 | 8 个数据位 |
| N | 无奇偶校验 |
| 1 | 1 个停止位 |

一个字符通常由起始位、数据位、可选校验位、停止位组成。双方波特率、数据位、校验位、停止位必须一致。

## 常见问题

- TX/RX 没有交叉，表现为完全收不到数据。
- 缺少共同地，导致波形参考不一致。
- 波特率或校验配置不一致，表现为乱码或 CRC 错误。
- USB 转串口芯片驱动异常，设备号变化或打不开串口。
- 把 TTL 串口直接接 RS-232，可能损坏设备，因为电平标准不同。

## 排查建议

1. 确认物理接线：TX/RX/GND。
2. 确认电平：RS-232、TTL 3.3V、TTL 5V 不能混用。
3. 确认串口参数：波特率、数据位、校验、停止位。
4. 用串口助手发送固定十六进制数据，观察是否稳定回显。
5. 用示波器或逻辑分析仪看波形和字节边界。
