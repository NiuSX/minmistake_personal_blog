# Wi-Fi 物理层 IEEE 802.11 PHY 学习笔记

最后整理：2026-06-11

Wi-Fi 是 IEEE 802.11 标准族的通称。它既包含物理层，也包含数据链路层的 MAC 机制。物理层部分关注无线频段、信道、调制、编码、空间流和信号质量。

## 解决的问题

- 多台设备如何在共享无线频谱中传输数据。
- 无线信号如何在 2.4 GHz、5 GHz、6 GHz 等频段上传播。
- 如何通过 OFDM、MIMO、信道编码提升速率和抗干扰能力。
- 如何在移动、遮挡、干扰环境下维持连接。

## 核心概念

| 概念 | 说明 |
|---|---|
| 频段 | 2.4 GHz 覆盖强但拥挤，5 GHz 速率高且信道多，6 GHz 更干净但设备要求新 |
| 信道宽度 | 20/40/80/160 MHz 等，越宽理论速率越高，但越容易受干扰 |
| RSSI | 接收信号强度，越接近 0 通常越强，例如 -45 dBm 优于 -75 dBm |
| SNR | 信噪比，信号相对噪声越高越稳定 |
| OFDM | 把高速数据拆到多个子载波上传输，提高频谱利用率 |
| MIMO | 多天线多空间流，提高吞吐和可靠性 |

## 典型流程

1. 客户端扫描信道，发现 AP 广播的 SSID。
2. 客户端选择 AP，进行认证和关联。
3. 双方协商速率、能力、加密方式等。
4. 数据通过无线帧发送，物理层根据环境动态调整调制编码方案。
5. 如果丢包或干扰增大，速率可能下降，重传可能增加。

## 常见问题

- 信号满格不等于速度快：还要看信道拥塞、干扰、终端能力和 AP 负载。
- 2.4 GHz 穿墙能力较好，但蓝牙、微波炉、邻居 AP 都可能干扰。
- 信道宽度不是越大越好，在拥挤环境下 80/160 MHz 可能更不稳定。
- Wi-Fi 是共享介质，同一 AP 下终端越多，争用和等待越明显。

## 排查命令与工具

- Windows：`netsh wlan show interfaces`
- Linux：`iw dev wlan0 link`、`iw dev wlan0 station dump`
- Android/iOS：使用 Wi-Fi 分析工具查看 RSSI、信道、频段。
- Wireshark：需要支持 monitor mode 的无线网卡才能完整抓取 802.11 帧。
