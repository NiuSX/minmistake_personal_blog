# 以太网物理层 Ethernet PHY 学习笔记

最后整理：2026-06-11

Ethernet PHY 是以太网在物理介质上的实现部分。以太网常被整体理解为二层协议，但完整以太网由 MAC 子层和 PHY 组成：MAC 负责帧格式、MAC 地址、介质访问控制；PHY 负责把 MAC 交给它的比特流变成电信号或光信号。

## 解决的问题

- 怎样在铜缆或光纤上传输比特。
- 怎样让链路两端协商共同支持的速率和双工模式。
- 怎样在不同介质上实现不同距离和速率，例如 1000BASE-T、10GBASE-SR。
- 怎样把物理信号错误暴露给上层，例如链路 down、CRC 错误、符号错误。

## 常见命名

| 名称 | 含义 |
|---|---|
| 10BASE-T | 10 Mbps，基带传输，双绞线 |
| 100BASE-TX | 100 Mbps，双绞线快速以太网 |
| 1000BASE-T | 1 Gbps，常见千兆铜缆以太网 |
| 10GBASE-T | 10 Gbps，铜缆以太网，对线缆质量要求高 |
| 1000BASE-SX/LX | 千兆光纤，SX 常用于短距多模，LX 常用于长距单模 |
| 10GBASE-SR/LR | 10G 光纤，SR 短距，LR 长距 |

## 工作机制

1. 设备插入线缆或光模块后，PHY 检测链路信号。
2. 双方通过自协商交换能力，包括速率、双工、流控能力。
3. 协商完成后，PHY 进入 link up 状态。
4. MAC 层交付帧数据，PHY 做编码、扰码、调制、发送。
5. 接收端 PHY 恢复时钟和数据，检测错误，再交给 MAC 层。

## 常见排查指标

| 指标 | 说明 | 可能原因 |
|---|---|---|
| link down | 链路未建立 | 网线断、光模块不兼容、对端端口关闭 |
| speed mismatch | 速率不符合预期 | 线缆质量差、自协商异常、设备能力不一致 |
| duplex mismatch | 双工不匹配 | 强制配置和自协商混用 |
| CRC error | 帧校验错误 | 干扰、线缆质量、光功率异常 |
| symbol error | 物理编码层错误 | 光模块、介质、距离、端口硬件问题 |

## 学习重点

- 不要把“网口亮灯”理解为网络一定正常，亮灯只说明物理层可能建立了链路。
- 物理层问题经常表现为 TCP 重传、HTTP 慢、视频卡顿，但抓包只能看到上层症状。
- 数据中心环境要特别注意光模块型号、单模/多模、波长、距离和交换机兼容性。
- 铜缆环境要注意 Cat5e/Cat6/Cat6A 的差异，尤其是 10GBASE-T。
