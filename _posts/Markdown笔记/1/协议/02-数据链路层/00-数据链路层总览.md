# OSI 第 2 层：数据链路层总览

最后整理：2026-06-14

数据链路层负责同一链路或同一二层网络中的通信。它把网络层交下来的包封装成帧，使用 MAC 地址等链路层地址识别节点，并通过校验字段发现传输错误。

## 学习目标

- 理解帧、MAC 地址、交换机、广播域、冲突域。
- 理解 ARP 为什么是 IPv4 局域网通信的关键。
- 理解 VLAN 如何把一个物理交换网络划分成多个逻辑二层网络。
- 理解 CAN、I2C、SPI 这类总线在嵌入式和工业设备中的链路机制。
- 理解 USB 的主机中心化总线模型、枚举、描述符、端点和传输类型。
- 能通过 MAC 表、ARP 表、抓包判断二层问题。

## 核心概念

| 概念 | 含义 |
|---|---|
| 帧 Frame | 链路层传输单位，包含目标/源 MAC、类型、载荷、校验等 |
| MAC 地址 | 链路层地址，通常 48 位，用于同一二层网络内寻址 |
| 广播 | 发给同一广播域内所有主机，例如 ARP Request |
| 交换机 | 根据 MAC 地址表转发以太网帧 |
| VLAN | 用标签隔离二层广播域 |
| FCS/CRC | 帧校验序列，用于发现链路传输错误 |

## 嵌入式与工业总线

除了 Ethernet 这类局域网链路层，工程中还会遇到很多总线型链路：

| 总线 | 典型场景 | 重点 |
|---|---|---|
| CAN | 汽车、工业控制、机器人 | 消息 ID 仲裁、错误检测、终端电阻 |
| I2C | 板级传感器、EEPROM、RTC | 地址、ACK/NACK、上拉电阻、总线电容 |
| SPI | Flash、显示屏、ADC、无线模块 | 时钟模式、片选、全双工、信号完整性 |
| USB | PC 外设、嵌入式设备、调试器、移动设备 | Host/Device、枚举、描述符、端点、传输类型 |
| Wi-Fi MAC | 无线局域网 | 扫描、认证、关联、CSMA/CA、重传、漫游 |
| UART/串口 | MCU 调试、工业仪表、通信模块 | 波特率、数据位、校验位、停止位、电气层和上层协议 |

串口通信容易把电气层、UART 字符帧和应用层报文混在一起。建议先读 `串口通信协议总览.md`，再分别学习 RS-232、RS-485、Modbus RTU、I2C、SPI、CAN 等专题。

USB 也容易把连接器、供电、物理速率和总线协议混在一起。建议先读 `../01-物理层/USB与Type-C物理层.md`，再读 `USB总线协议.md`，分清 Type-C/PD 与 USB 枚举、端点传输之间的边界。

Wi-Fi 同时包含 PHY 和 MAC。信号、频段、信道、MCS 属于物理层重点；扫描、Beacon、认证、关联、CSMA/CA、ACK、漫游属于数据链路层重点。学习时把 `../01-物理层/Wi-Fi物理层-802.11-PHY.md` 和 `Wi-Fi数据链路层-802.11-MAC.md` 放在一起读。

## 数据链路层与网络层的边界

网络层负责“目标 IP 在哪里、下一跳是谁”，数据链路层负责“如何把这一跳发出去”。一次跨网段通信可能经过很多个二层链路，每一跳的源/目标 MAC 都会变化，但源/目标 IP 通常保持不变。

```mermaid
sequenceDiagram
    participant A as 主机A
    participant S as 交换机
    participant R as 网关路由器
    A->>A: 查路由表，发现目标不在本网段
    A->>A: ARP 获取网关 MAC
    A->>S: Ethernet 帧：目标 MAC=网关
    S->>R: 根据 MAC 表转发
    R->>R: 解二层帧，查看 IP 包并路由到下一跳
```

## 常见问题

- ARP 表错误会导致同网段或访问网关失败。
- VLAN 配置错误会导致“物理连通但逻辑不通”。
- 环路会导致广播风暴，需要 STP/RSTP 等机制控制。
- MAC 地址漂移可能表示环路、虚拟机迁移或链路聚合配置问题。
- USB 设备能充电但不识别，可能是 Type-C CC、线缆、D+/D-、枚举描述符或驱动任一环节出问题。
- Wi-Fi 已连接但无网络，可能是关联成功但 DHCP、VLAN、网关或上游网络异常。
