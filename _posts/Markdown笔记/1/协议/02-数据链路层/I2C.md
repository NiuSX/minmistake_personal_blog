# I2C 总线学习笔记

最后整理：2026-06-11

I2C（Inter-Integrated Circuit）是常见板级串行总线，用两根线连接微控制器和外设芯片，例如传感器、EEPROM、RTC、电源管理芯片。它主要用于同一 PCB 或短距离板间通信，不适合长距离工业现场布线。

## 协议定位

I2C 覆盖物理连接和数据链路层机制：SCL 时钟线、SDA 数据线、开漏输出、上拉电阻、地址、ACK/NACK、起止条件等。具体寄存器含义由外设芯片手册定义。

## 解决的问题

- MCU 需要用少量引脚连接多个低速外设。
- 多个设备共享总线，通过地址区分。
- 主机控制时钟，简化从设备设计。

## 核心信号

| 信号 | 说明 |
|---|---|
| SCL | Serial Clock，时钟线 |
| SDA | Serial Data，数据线 |
| GND | 共同地 |

SCL 和 SDA 通常是开漏/开集电极结构，需要上拉电阻。任何设备都可以把线拉低，但不能主动拉高。

## 基本时序

| 条件 | 说明 |
|---|---|
| START | SCL 高电平时 SDA 从高变低 |
| STOP | SCL 高电平时 SDA 从低变高 |
| ACK | 接收方在第 9 个时钟拉低 SDA |
| NACK | 接收方不拉低 SDA |

## 地址与读写

常见 I2C 使用 7 位地址。传输时地址后跟 1 位读写位：

- 写：主机发送设备地址 + W，然后发送寄存器地址和数据。
- 读：主机通常先写寄存器地址，再 repeated START，发送设备地址 + R，读取数据。

## 常见问题

- 忘记上拉电阻，或者上拉太弱导致边沿慢。
- 地址写错：手册给的是 7 位地址，但代码需要 8 位地址，或反过来。
- 多个设备地址冲突。
- 总线电容过大，线太长，速率上不去。
- 某个设备异常拉低 SDA，导致总线挂死。

## 排查建议

1. 用 I2C scanner 扫描地址。
2. 用逻辑分析仪看 START、地址、ACK。
3. 检查上拉电阻和电压域。
4. 降低时钟频率验证是否为边沿或电容问题。
5. 对照芯片手册确认寄存器地址和读写流程。
