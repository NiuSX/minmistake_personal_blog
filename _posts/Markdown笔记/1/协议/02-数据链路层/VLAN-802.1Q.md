# VLAN 与 IEEE 802.1Q 学习笔记

最后整理：2026-06-11

VLAN（Virtual LAN）用于在同一套交换网络上划分多个逻辑二层网络。它解决的是广播域隔离、网络分区和管理边界问题。IEEE 802.1Q 定义了在以太网帧中插入 VLAN 标签的方式。

## 解决的问题

- 一个物理交换网络中不同部门、业务、租户需要隔离。
- 广播域太大，ARP/DHCP/广播流量影响范围过广。
- 多台交换机之间需要承载多个逻辑网络。

## 802.1Q 标签

802.1Q 会在源 MAC 和 EtherType 之间插入 4 字节标签。

| 字段 | 说明 |
|---|---|
| TPID | 通常为 `0x8100`，表示后面是 VLAN 标签 |
| PCP | 3 位优先级，用于 QoS |
| DEI | 丢弃资格标记 |
| VID | 12 位 VLAN ID，范围通常 1-4094 |

## Access 与 Trunk

| 端口类型 | 作用 |
|---|---|
| Access | 面向终端设备，通常只属于一个 VLAN，进出端口时不带标签 |
| Trunk | 面向交换机、路由器、防火墙、虚拟化主机，可承载多个 VLAN，帧带标签 |

## 跨 VLAN 通信

不同 VLAN 是不同广播域。主机跨 VLAN 通信必须经过三层设备，例如路由器、三层交换机、防火墙，常见方式是：

- Router-on-a-stick：路由器一个物理口划分多个子接口，每个子接口对应一个 VLAN。
- 三层交换机 SVI：交换机为每个 VLAN 创建虚拟三层接口。
- 防火墙网关：不同 VLAN 流量经过防火墙策略控制。

## 常见问题

- Access VLAN 配错：终端拿不到正确网段地址。
- Trunk allowed VLAN 缺失：某 VLAN 无法跨交换机。
- Native VLAN 不一致：可能导致未打标签流量异常，也有安全风险。
- VLAN ID 一样不代表一定互通，中间链路和端口策略也必须放行。
