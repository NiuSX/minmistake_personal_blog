# OSI 第 4 层：传输层总览

最后整理：2026-06-14

传输层负责主机到主机之上的进程到进程通信。网络层把包送到目标主机，传输层通过端口把数据交给正确的应用进程，并根据协议提供可靠性、顺序、流控、拥塞控制或低延迟传输。

## 学习目标

- 理解端口、套接字、连接、无连接传输。
- 区分 TCP、UDP、QUIC 的设计目标。
- 理解可靠传输、重传、流量控制、拥塞控制。
- 理解超时、重试、连接池、保活和应用心跳对系统稳定性的影响。
- 能通过抓包判断三次握手、重传、RST、超时等问题。

## TCP、UDP、QUIC 对比

| 协议 | 连接模型 | 可靠性 | 典型场景 |
|---|---|---|---|
| TCP | 面向连接，字节流 | 协议内保证可靠、有序 | Web、数据库、SSH、邮件 |
| UDP | 无连接，数据报 | 不保证可靠和顺序 | DNS、音视频、游戏、隧道 |
| QUIC | 基于 UDP 的加密传输 | 可靠流、多路复用、内建 TLS | HTTP/3、低延迟 Web |

超时、重试、连接池、Keepalive、心跳这些问题跨 TCP、QUIC、HTTP、MQTT、WebSocket、数据库和工业网关，建议读 `连接超时重试与保活.md`。

## 端口

端口用于标识主机上的应用进程。一个 TCP 连接通常由五元组标识：

```text
源 IP、源端口、目标 IP、目标端口、传输协议
```

常见端口：

| 端口 | 协议 |
|---:|---|
| 53 | DNS |
| 80 | HTTP |
| 443 | HTTPS / HTTP/3 |
| 25 | SMTP |
| 22 | SSH |

## 常见排查点

- 连接失败：看 SYN 是否发出、SYN-ACK 是否返回、防火墙是否丢弃。
- 连接被拒绝：通常看到 RST，表示目标主机可达但端口未监听或策略拒绝。
- 慢：看重传、窗口大小、拥塞窗口、应用响应时间。
- UDP 不通：要同时查应用、ACL、防火墙、NAT 映射和返回流量。
- 连接池复用到已被中间代理关闭的空闲连接，会表现为偶发首个请求失败、重试成功。
- 重试没有幂等保护，可能导致重复写入、重复下单或重复控制设备。

## 专题阅读顺序

1. `TCP.md`：理解可靠字节流、三次握手、重传、关闭。
2. `UDP.md`：理解无连接数据报和应用自建可靠性。
3. `QUIC.md`：理解基于 UDP 的加密、多路复用和连接迁移。
4. `SCTP.md`：理解多流、多宿主等特殊传输能力。
5. `连接超时重试与保活.md`：理解工程系统中连接生命周期和故障恢复。
