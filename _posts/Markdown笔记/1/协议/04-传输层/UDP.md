# UDP 用户数据报协议学习笔记

最后整理：2026-06-11

UDP（User Datagram Protocol）是一个简单的无连接传输层协议。它提供端口和校验和，但不保证可靠、不保证顺序、不重传、不做拥塞控制。它的价值在于简单、低开销、低延迟，适合由应用自己控制传输策略的场景。

## 解决的问题

- 在 IP 之上增加端口，让数据交给正确应用。
- 保留消息边界，每个 UDP 数据报对应一次发送。
- 为实时业务、简单查询、隧道协议提供低开销传输。

## UDP 头部

| 字段 | 长度 | 作用 |
|---|---:|---|
| Source Port | 16 位 | 源端口 |
| Destination Port | 16 位 | 目标端口 |
| Length | 16 位 | UDP 头加数据长度 |
| Checksum | 16 位 | 校验 UDP 伪首部、头部和数据 |

## 常见应用

| 应用 | 为什么使用 UDP |
|---|---|
| DNS | 请求/响应短小，超时后应用可重试 |
| DHCP | 客户端一开始可能还没有 IP，需要广播 |
| VoIP/视频会议 | 宁可丢少量包，也不希望排队重传造成延迟 |
| 游戏 | 实时状态更新比可靠传输更重要 |
| QUIC | 在 UDP 上实现自己的可靠传输和加密 |

## UDP 与 TCP 的关键差异

- UDP 有消息边界，TCP 是字节流。
- UDP 没有连接握手，发送前不确认对端是否准备好。
- UDP 丢包不会由协议自动重传。
- UDP 不做拥塞控制，应用层协议应自行考虑网络公平性。

## 常见问题

- UDP 服务“没响应”时，可能是请求丢、响应丢、NAT 映射过期或防火墙阻断。
- UDP 放大攻击利用小请求触发大响应，需要服务端做限速和验证。
- 大 UDP 包容易触发 IP 分片，分片丢失会导致整个数据报不可用。
