# OSI 第 5 层：会话层总览

最后整理：2026-06-11

会话层负责建立、管理和终止通信会话。现代 TCP/IP 协议栈没有一个严格对应“会话层”的通用协议层，很多会话能力被放进应用协议、TLS、RPC 框架、认证系统或连接池中。

## 会话层关心什么

| 能力 | 说明 |
|---|---|
| 会话建立 | 双方确认身份、能力、上下文 |
| 会话保持 | 心跳、保活、超时、状态续期 |
| 会话恢复 | 断线后恢复上下文或重新协商 |
| 会话同步 | 检查点、事务边界、请求-响应对应 |
| 会话终止 | 正常关闭、异常中断、资源释放 |

## 为什么现代网络中会话层不明显

- TCP 已经提供连接，但 TCP 连接不等于应用会话。
- HTTP 本身是无状态语义，但 Cookie、Session、Token 提供应用会话。
- TLS 提供安全会话和会话恢复，但通常被归到表示层或安全层。
- RPC 框架管理调用上下文、连接复用、超时和重试。

## 本目录选择的代表

- NetBIOS Session Service：经典 OSI 会话层风格的会话服务。
- RPC：远程过程调用模型，强调请求/响应、调用上下文和序列化绑定。
- SIP：用于建立、修改、终止多媒体会话，常见于 VoIP。

## 学习重点

不要把“会话”简单等同于“TCP 连接”。一个用户登录会话可以跨多个 TCP 连接，一个 HTTP/2 或 QUIC 连接中也可以承载多个逻辑请求流。排查系统问题时，要分别看传输连接、TLS 会话、应用登录会话、业务事务上下文。
