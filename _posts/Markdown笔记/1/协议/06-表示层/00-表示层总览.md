# OSI 第 6 层：表示层总览

最后整理：2026-06-14

表示层负责数据如何被表示、编码、转换、压缩和加密。现代互联网中，表示层能力通常散落在 TLS、MIME、JSON、Protocol Buffers、ASN.1、字符编码和压缩算法中。

## 学习目标

- 理解“数据含义”和“字节表示”的区别。
- 理解字符集、序列化、内容类型、压缩和加密在协议中的作用。
- 知道 TLS 在实际协议栈中的位置和作用。
- 理解字段演进、字节序、时间格式、大整数和版本兼容的常见坑。
- 能排查乱码、Content-Type 错误、证书错误、编码不兼容等问题。

## 表示层关心什么

| 能力 | 示例 |
|---|---|
| 字符编码 | UTF-8、ASCII |
| 数据序列化 | JSON、ASN.1、Protocol Buffers |
| 内容描述 | MIME Content-Type |
| 压缩 | gzip、br、zstd |
| 加密与身份 | TLS、证书、密钥交换 |

编码、序列化、字节序、时间格式、压缩和版本兼容问题建议读 `编码序列化与兼容性.md`。

## 常见问题

- 字符集不一致导致乱码。
- Content-Type 错误导致浏览器或客户端错误解析。
- JSON 数字精度、时间格式、空值语义不一致导致兼容问题。
- TLS 证书链不完整、域名不匹配、协议版本太低导致握手失败。
- 大整数、金额、时间戳、浮点数和字节序处理不当，会导致业务数据看似“通信成功但值不对”。

## 专题阅读顺序

1. `编码序列化与兼容性.md`：建立表示层通用问题地图。
2. `JSON.md`：理解最常见文本数据交换格式。
3. `MIME.md`：理解内容类型、邮件和 HTTP 内容表示。
4. `ASN.1-BER-DER.md`：理解证书、电信和二进制 TLV 编码。
5. `TLS.md`：理解加密、身份认证和安全通道。
