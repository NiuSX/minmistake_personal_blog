# OSI 第 7 层：应用层总览

最后整理：2026-06-14

应用层协议直接面向具体业务需求，例如浏览网页、解析域名、分配地址、发送邮件、传输文件。它们通常建立在 TCP、UDP、QUIC、TLS 等下层能力之上。

## 学习目标

- 理解应用协议如何定义请求、响应、状态码、头字段和资源标识。
- 了解 DNS、DHCP、HTTP、SMTP、FTP、MQTT、WebSocket 的基本流程。
- 了解 Modbus、OPC UA、CANopen、PROFIBUS、EtherCAT 等工业通信协议的基本定位。
- 能使用 curl、dig/nslookup、抓包工具排查应用层问题。
- 明白“应用层错误”和“网络层/传输层错误”的区别。

## 常见协议

| 协议 | 默认端口 | 传输 | 作用 |
|---|---:|---|---|
| HTTP | 80/443 | TCP/TLS/QUIC | Web 与 API |
| DNS | 53 | UDP/TCP，也可 DoH/DoT | 域名解析 |
| DHCP | 67/68 | UDP | 自动分配 IP 配置 |
| SMTP | 25/465/587 | TCP/TLS | 邮件投递和提交 |
| FTP | 20/21 | TCP | 文件传输 |
| MQTT | 1883/8883 | TCP/TLS | IoT 发布订阅消息 |
| WebSocket | 80/443 | TCP/TLS | Web 实时双向通信 |
| Modbus TCP | 502 | TCP | 工业设备寄存器读写 |
| Modbus RTU | 无固定 TCP/UDP 端口 | 串口/RS-485 | 工业串行主从通信 |
| OPC UA | 4840 | TCP/TLS 等 | 工业数据建模、安全通信和订阅 |
| CANopen | 无固定 TCP/UDP 端口 | CAN | 工业设备对象字典和过程数据 |
| EtherCAT | 以太网帧 | Ethernet | 实时工业以太网 |

## 排查方法

1. 先确认 DNS 是否解析正确。
2. 再确认目标 IP 路由可达。
3. 再确认 TCP/UDP 端口是否可达。
4. 对 TLS 服务检查证书、SNI、ALPN。
5. 最后查看应用协议状态码、错误码、响应体、消息主题、节点状态或异常码。
