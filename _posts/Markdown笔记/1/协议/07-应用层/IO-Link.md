# IO-Link 学习笔记

最后整理：2026-06-11

IO-Link 是面向工业传感器和执行器的点对点数字通信技术，标准化于 IEC 61131-9。它常用于把普通三线传感器升级为可参数化、可诊断、可识别的智能设备。

## 协议定位

IO-Link 不是现场总线替代品，而是传感器/执行器到 IO-Link Master 之间的点对点通信。IO-Link Master 再通过 PROFINET、EtherCAT、EtherNet/IP、Modbus TCP 等上层网络接入控制系统。

## 解决的问题

- 传统开关量/模拟量传感器只能给有限信号，参数和诊断能力弱。
- 更换传感器后需要人工重新设置参数。
- 控制系统希望读取设备身份、过程数据、诊断事件和参数。

## 系统组成

| 组件 | 说明 |
|---|---|
| IO-Link Device | 传感器或执行器 |
| IO-Link Master | 主站，连接多个 IO-Link 端口 |
| IODD | 设备描述文件，描述参数、过程数据、菜单和文本 |
| Engineering Tool | 工程配置工具 |

## 数据类型

| 数据 | 说明 |
|---|---|
| Process Data | 周期过程数据，例如测量值、开关状态 |
| Parameter Data | 非周期参数，例如量程、阈值 |
| Event Data | 诊断事件，例如短路、污染、超温 |
| Device Data | 厂商、型号、序列号等身份信息 |

## 常见能力

- 自动设备识别。
- 参数服务器：更换同型号设备后自动下发参数。
- 传感器诊断和预测性维护。
- 用数字值替代模拟量，减少量化和干扰问题。

## 常见问题

- IODD 文件缺失或版本不匹配。
- Master 端口模式配置错误，设备只以普通 SIO 模式工作。
- 线缆和供电不符合设备要求。
- 上层 PLC 映射的过程数据长度或字节序理解错误。
