# 工业通信协议总览

最后整理：2026-06-11

工业通信协议和普通互联网协议的关注点不同。互联网协议更关注大规模互联、路由、吞吐、通用应用；工业协议更关注确定性、实时性、抗干扰、设备寄存器、控制周期、现场布线和长期稳定运行。

## 学习目标

- 分清物理层、电气接口、串口参数、应用层报文协议。
- 理解 RS-485 与 Modbus RTU 的关系。
- 知道 CAN、CANopen、PROFIBUS、EtherCAT、HART、IO-Link 的定位差异。
- 能从接线、波特率、地址、帧格式、校验、主从关系几个维度排查问题。

## 常见协议与层次

| 名称 | 更接近的层次 | 典型介质 | 说明 |
|---|---|---|---|
| RS-232 | 物理层 | 串口线 | 点对点单端串行 |
| RS-422 | 物理层 | 差分线 | 一个发送器、多接收器 |
| RS-485 | 物理层 | 双绞线 | 多点差分总线 |
| Modbus RTU | 应用层 | RS-485/RS-232 | 主从寄存器读写 |
| Modbus TCP | 应用层 | Ethernet/TCP | Modbus 报文跑在 TCP 502 |
| CAN | 物理/数据链路层 | CAN 总线 | 消息 ID 仲裁短帧通信 |
| CANopen | 应用层 | CAN | 工业设备对象字典和通信模型 |
| PROFIBUS | 现场总线 | RS-485/MBP 等 | 工业自动化现场总线 |
| EtherCAT | 工业以太网 | Ethernet | 高性能实时以太网 |
| HART | 现场仪表协议 | 4-20 mA 叠加 FSK | 过程仪表配置与诊断 |
| IO-Link | 传感器/执行器通信 | 三线点对点 | 智能传感器参数化和诊断 |

## 学习路线

1. 先读 `../02-数据链路层/串口通信协议总览.md`，把 TTL UART、RS-232、RS-485、UART 帧、Modbus RTU 和自定义协议的层次分清。
2. 再学 RS-232、RS-485、UART，理解串口参数和电气层。
3. 再学 Modbus RTU，理解地址、功能码、寄存器和 CRC。
4. 学 Modbus TCP，理解同一应用语义如何迁移到 TCP/IP。
5. 学 CAN，再学 CANopen，理解“底层总线”和“上层设备模型”的分离。
6. 根据工作场景补 PROFIBUS、EtherCAT、HART、IO-Link。

## 工业通信排查通用清单

| 检查项 | 问题例子 |
|---|---|
| 接线 | A/B 反、TX/RX 反、缺 GND、屏蔽层接地不当 |
| 电气 | 电平不匹配、终端电阻错误、线太长、干扰强 |
| 串口参数 | 波特率、校验位、停止位不一致 |
| 地址 | 从站地址重复、主站请求地址错误 |
| 帧格式 | 功能码、寄存器地址、字节序、CRC 错 |
| 时序 | 主站轮询太快、从站响应超时、驱动释放不及时 |
| 网关 | RTU/TCP 网关映射错误、单元标识符配置错误 |
| 文档 | 厂家寄存器表和标准地址基准不一致 |

## 寄存器、线圈、对象字典

工业协议经常把设备能力抽象成可读写对象：

- Modbus：线圈、离散输入、输入寄存器、保持寄存器。
- CANopen：对象字典，使用索引和子索引访问参数。
- IO-Link：设备参数、过程数据、事件和诊断。
- HART：通用命令、常用命令、设备专用命令。

学习时不要只看协议帧，还要看具体设备手册。现场调试最关键的信息通常在厂家寄存器表、对象字典或 EDS/IODD/GSD 文件中。
