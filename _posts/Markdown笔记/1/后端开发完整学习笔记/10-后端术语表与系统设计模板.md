## 10. 系统设计通用模板

设计任何后端系统时，可以按下面顺序思考。

### 10.1 需求与边界

- 这个系统解决什么问题？
- 核心用户是谁？
- 核心业务流程是什么？
- 哪些功能必须实时？
- 哪些功能可以异步？
- 哪些数据是强一致？
- 哪些数据可以最终一致？
- 预期流量和数据规模是多少？

### 10.2 API 设计

- 有哪些资源？
- 使用 REST、RPC、GraphQL 还是事件？
- 请求字段有哪些？
- 响应字段有哪些？
- 错误码有哪些？
- 是否需要分页？
- 是否需要幂等？
- 是否需要版本？
- 是否需要 Webhook？

### 10.3 数据设计

- 核心实体有哪些？
- 表如何设计？
- 主键是什么？
- 唯一约束是什么？
- 状态字段是什么？
- 需要哪些索引？
- 查询模式是什么？
- 数据增长如何？
- 是否需要归档？

### 10.4 事务与一致性

- 哪些操作必须同事务？
- 哪些操作可以最终一致？
- 是否需要状态机？
- 是否需要乐观锁？
- 是否需要唯一约束？
- 是否需要分布式事务？
- 是否可以用 Saga？

### 10.5 缓存

- 哪些数据适合缓存？
- 缓存 key 如何设计？
- TTL 多久？
- 写入后如何失效？
- 穿透怎么处理？
- 击穿怎么处理？
- 雪崩怎么处理？
- 热点 key 怎么处理？

### 10.6 异步与消息

- 哪些步骤可以异步？
- 消息主题是什么？
- 生产失败怎么办？
- 消费失败怎么办？
- 重复消费怎么办？
- 顺序是否重要？
- 积压怎么办？
- 是否需要死信队列？

### 10.7 安全

- 是否需要登录？
- 权限模型是什么？
- 是否有对象级权限？
- 是否有敏感数据？
- 是否需要审计？
- 是否需要签名？
- 是否需要限流？
- 是否防重放？

### 10.8 稳定性

- 远程调用超时多少？
- 是否需要重试？
- 重试是否幂等？
- 是否需要熔断？
- 是否有降级？
- 资源是否隔离？
- 是否有容量预估？

### 10.9 可观测性

- 记录哪些日志？
- 指标有哪些？
- 是否接入链路追踪？
- 关键告警是什么？
- 如何定位慢请求？
- 如何定位数据不一致？

### 10.10 发布与运维

- 是否需要数据库迁移？
- 是否兼容旧版本？
- 是否能灰度？
- 是否能回滚？
- 配置如何管理？
- 是否需要容灾？
- 备份如何做？

## 11. 场景模板：设计登录系统

需要考虑：

- 用户名密码或第三方登录。
- 密码哈希存储。
- 登录失败限流。
- Session 或 Token。
- Token 过期和刷新。
- 设备管理。
- 退出登录。
- 异地登录提醒。
- 审计日志。
- CSRF 或 Token 存储安全。

## 12. 场景模板：设计订单系统

需要考虑：

- 商品快照。
- 价格计算。
- 库存校验。
- 订单状态机。
- 幂等下单。
- 支付回调。
- 超时取消。
- 消息通知。
- 售后退款。
- 对账。

## 13. 场景模板：设计秒杀系统

需要考虑：

- 页面静态化。
- CDN。
- 网关限流。
- 用户资格校验。
- Redis 预扣库存。
- MQ 削峰。
- 异步创建订单。
- 数据库唯一约束。
- 防刷。
- 排队状态查询。

## 14. 场景模板：设计文件上传

需要考虑：

- 文件大小限制。
- 文件类型校验。
- 直传对象存储。
- 上传凭证有效期。
- 文件元数据。
- 权限控制。
- 病毒扫描。
- CDN 分发。
- 生命周期清理。

## 15. 后端面试表达模板

回答系统设计题时：

1. 先澄清需求和规模。
2. 给出核心 API。
3. 设计数据模型。
4. 说明核心流程。
5. 说明缓存和异步。
6. 说明一致性和幂等。
7. 说明高可用和降级。
8. 说明安全。
9. 说明监控和排查。
10. 说明取舍和演进。

## 16. 后端学习复盘模板

学完一个技术后写：

- 它解决什么问题？
- 它的核心概念是什么？
- 它适合什么场景？
- 它不适合什么场景？
- 它的常见坑是什么？
- 它和类似技术有什么区别？
- 它在系统里如何监控？
- 它故障时如何降级？

## 17. 本章小结

术语表帮助快速建立后端语言体系，设计模板帮助把知识用于真实场景。后端学习最终要落到“能设计、能实现、能上线、能排查、能复盘”。
