# 09. 机器人感知几何

## 学习目标

学完本章，你应该能：

- 理解相机模型、内参、外参和畸变。
- 理解 3D 点如何投影到 2D 图像。
- 理解深度相机、点云、ICP 和点云配准。
- 理解手眼标定为什么影响机械臂抓取。
- 能把感知结果转换到机器人坐标系中。

## 1. 感知不是只有深度学习

机器人感知通常包含：

- 传感器标定。
- 坐标变换。
- 图像处理。
- 目标检测。
- 点云处理。
- 位姿估计。
- 多传感器融合。

深度学习可以识别“是什么”，但机器人还需要知道：

- 在哪里。
- 相对机器人是什么位姿。
- 机器人能不能到达。
- 坐标转换是否准确。

## 2. 针孔相机模型

针孔模型描述 3D 点如何投影到图像平面。

简化公式：

```text
u = fx * X / Z + cx
v = fy * Y / Z + cy
```

其中：

- `(X, Y, Z)` 是相机坐标系下的 3D 点。
- `(u, v)` 是图像像素坐标。
- `fx, fy` 是焦距相关参数。
- `cx, cy` 是主点。

## 3. 相机内参

内参矩阵：

```text
K = [[fx,  0, cx],
     [ 0, fy, cy],
     [ 0,  0,  1]]
```

内参描述相机自身成像特性。

用途：

- 像素点反投影成 3D 射线。
- 计算深度点的 3D 坐标。
- 相机标定和视觉定位。

## 4. 畸变

真实镜头会产生畸变。

常见畸变：

- 径向畸变：图像向外或向内弯曲。
- 切向畸变：镜头和成像平面不完全平行。

如果不校正畸变：

- 直线看起来弯。
- 定位和测量不准。
- 手眼标定误差变大。

## 5. 外参

外参描述相机和另一个坐标系之间的变换。

例子：

```text
T_base_camera
```

表示相机坐标系到机器人底盘坐标系的关系。

外参错误会导致：

- 目标识别位置正确，但机器人抓偏。
- 点云投影到地图中偏移。
- 多相机融合失败。

## 6. 深度相机和 3D 点

如果知道像素 `(u, v)` 和深度 `Z`：

```text
X = (u - cx) * Z / fx
Y = (v - cy) * Z / fy
```

这样可以从图像点得到相机坐标系下的 3D 点。

再通过 TF 转换：

```text
p_base = T_base_camera * p_camera
```

## 7. 点云

点云是大量 3D 点的集合。

常见处理：

- 体素降采样。
- 离群点去除。
- 平面分割。
- 聚类。
- 法向量估计。
- 配准。

## 8. ICP 点云配准

ICP 用于对齐两个点云。

基本思想：

1. 找最近点匹配。
2. 估计刚体变换。
3. 应用变换。
4. 重复直到收敛。

ICP 依赖较好的初值。初值太差时容易收敛到错误结果。

常见失败原因：

- 初始位姿太差。
- 点云重叠区域太少。
- 动态物体太多。
- 几何结构退化。
- 噪声太大。

## 9. 手眼标定

手眼标定解决：

```text
相机坐标系和机械臂末端/底座坐标系的关系
```

常见形式：

- Eye-in-hand：相机装在机械臂末端。
- Eye-to-hand：相机固定在外部。

抓取任务中，如果手眼标定不准：

- 检测框是对的。
- 深度也是对的。
- 但机械臂执行会抓偏。

## 10. 视觉和机器人坐标链

典型抓取链路：

```text
像素点 -> 相机 3D 点 -> 机器人坐标系 -> 机械臂目标位姿 -> MoveIt 2 规划
```

每一步都可能出错：

- 相机内参错误。
- 深度错误。
- 外参错误。
- TF 时间戳错误。
- 目标姿态估计错误。
- 机械臂工具坐标错误。

## 11. 常见工程问题

| 现象 | 可能原因 |
|---|---|
| 检测框准但抓取偏 | 手眼标定或工具坐标错误 |
| 点云有明显倾斜 | 相机外参或 IMU 姿态错误 |
| 深度边缘噪声大 | 深度相机原理限制 |
| ICP 配准跳到错误位置 | 初值差、重叠少、环境退化 |
| 相机标定结果不稳定 | 棋盘格角点质量差、姿态覆盖不足 |

## 12. 练习

1. 用 OpenCV 做棋盘格相机标定。
2. 用深度值把检测框中心转换为 3D 点。
3. 在 ROS 2 中把相机点转换到 `base_link`。
4. 用 Open3D 做点云体素降采样和平面分割。
5. 用 ICP 对齐两帧点云，观察初值影响。
