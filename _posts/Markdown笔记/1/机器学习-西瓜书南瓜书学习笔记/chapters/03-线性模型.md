# 第 3 章 线性模型

## 学习目标

线性模型是机器学习中最重要的基线模型。本章需要掌握线性回归、对数几率回归、线性判别分析、多分类策略和类别不平衡处理。

## 先用人话理解

线性模型的核心想法很朴素：给每个特征一个权重，然后加起来做判断。比如判断房价时，面积可能是正权重，离市中心距离可能是负权重；判断好瓜时，敲声、色泽、根蒂都可以贡献不同分数。

线性模型简单，但它非常重要。原因有三点：

- 它是很多复杂模型的基础部件。
- 它容易解释，每个权重都能说明一个特征的影响方向。
- 它常常是强基线，能帮助判断复杂模型是否真的必要。

## 一个小例子：房价预测

假设只用面积预测房价：

```text
房价 = w * 面积 + b
```

如果 `w = 2`，`b = 30`，可以理解为每多 1 平方米，预测房价增加 2 个单位；即使面积为 0，也有一个基础项 30。训练线性回归，就是根据历史成交数据找到最合适的 `w` 和 `b`，让预测值整体尽量贴近真实价格。

## 线性回归

线性回归假设输出是输入属性的线性组合：

```text
f(x) = w^T x + b
```

给定平方损失：

```text
E(w, b) = sum_i (y_i - w^T x_i - b)^2
```

可将偏置并入参数。令每个样本增广为 `xhat_i = [x_i, 1]`，参数增广为 `what = [w, b]`，则：

```text
E(what) = ||y - X what||^2
```

对 `what` 求导：

```text
E = (y - Xw)^T(y - Xw)
  = y^T y - 2 w^T X^T y + w^T X^T X w

partial E / partial w = -2 X^T y + 2 X^T X w
```

令梯度为零：

```text
X^T X w = X^T y
w* = (X^T X)^(-1) X^T y
```

这就是最小二乘的正规方程。若 `X^T X` 不可逆，说明特征线性相关或样本不足，可用伪逆或岭回归。

## 岭回归与 Lasso

岭回归加入 L2 正则：

```text
min_w ||y - Xw||^2 + lambda ||w||_2^2
```

解析解：

```text
w* = (X^T X + lambda I)^(-1) X^T y
```

推导与普通最小二乘相同，只是梯度多出 `2 lambda w`。

Lasso 加入 L1 正则：

```text
min_w ||y - Xw||^2 + lambda ||w||_1
```

L1 在零点不可导，几何上更容易与损失函数等高线在坐标轴处相切，因此更容易产生稀疏解。

## 对数几率回归

二分类中希望输出类别概率。使用 sigmoid：

```text
p(y=1|x) = sigmoid(z) = 1 / (1 + exp(-z))
z = w^T x + b
```

几率是 `p/(1-p)`，对数几率为：

```text
log(p/(1-p)) = w^T x + b
```

这说明逻辑回归实际建模的是对数几率与输入之间的线性关系。

## 类比：逻辑回归像“分数转概率”

线性部分 `w^T x + b` 可以看成模型给样本打出的原始分数。分数很大时，更倾向于正类；分数很小时，更倾向于负类。但分类任务常希望输出概率，于是 sigmoid 像一个转换器，把任意实数压到 0 到 1 之间。

例如：

- 分数为 0，对应概率 0.5，模型拿不准。
- 分数很大，对应概率接近 1。
- 分数很小，对应概率接近 0。

这样，逻辑回归就既有线性模型的可解释性，又能输出概率。

### 负对数似然推导

对单个样本：

```text
P(y_i|x_i; w) = p_i^{y_i} (1-p_i)^{1-y_i}
```

全体样本似然：

```text
L(w) = Π_i p_i^{y_i} (1-p_i)^{1-y_i}
```

取对数并加负号：

```text
J(w) = - sum_i [ y_i log p_i + (1-y_i) log(1-p_i) ]
```

对 `w` 求梯度。因为 `p_i = sigmoid(w^T x_i)`，且：

```text
d sigmoid(z) / dz = sigmoid(z)(1 - sigmoid(z))
```

最终得到：

```text
grad J(w) = sum_i (p_i - y_i) x_i
```

这个结果很重要：预测概率高于真实标签时，梯度推动参数降低该样本的得分；反之提高得分。

## 线性判别分析

LDA 的思想是寻找投影方向，使同类样本投影后尽量集中，不同类中心尽量分开。

二分类中，设两类均值为 `mu_0, mu_1`，类内散度矩阵为：

```text
S_w = sum_{x in C0} (x-mu_0)(x-mu_0)^T + sum_{x in C1} (x-mu_1)(x-mu_1)^T
```

类间散度可用：

```text
S_b = (mu_0 - mu_1)(mu_0 - mu_1)^T
```

优化目标：

```text
max_w (w^T S_b w) / (w^T S_w w)
```

解的方向满足：

```text
w ∝ S_w^(-1)(mu_0 - mu_1)
```

直觉：均值差越大的方向越好，类内方差越大的方向要被抑制。

## 类比：LDA 像找最佳拍照角度

想象桌上有两堆不同颜色的小球。你要从某个方向拍一张二维照片，让两种颜色在照片上分得最开。某些角度看过去，两堆球重叠严重；另一些角度看过去，两堆球分得很清楚。

LDA 做的就是找这个“拍照角度”：投影后，同类样本尽量挤在一起，不同类中心尽量离得远。

## 多分类策略

- OvO：每两个类别训练一个分类器，共 `N(N-1)/2` 个。
- OvR：每个类别对其余类别训练一个分类器，共 `N` 个。
- MvM：每次把类别集合分成两组，如纠错输出码。

OvO 单个分类器训练数据少，但分类器数量多。OvR 分类器少，但正负样本可能不平衡。

## 类别不平衡

类别不平衡时，默认阈值 0.5 未必合适。可使用：

- 重采样：过采样少数类或欠采样多数类。
- 阈值移动：按业务代价调整分类阈值。
- 类别权重：训练时提高少数类损失权重。
- 使用 PR 曲线、F1、AUC 等更合适指标。

## 易错点

- 逻辑回归是分类模型，不是回归模型；名字来自对数几率。
- 线性模型不等于只能拟合直线；通过特征变换可拟合非线性关系。
- L2 正则倾向于缩小参数，L1 正则倾向于稀疏参数。
- 线性回归的正规方程要求矩阵可逆，实际中经常需要数值稳定处理。

## 练习

1. 手推最小二乘正规方程。
2. 手推逻辑回归负对数似然梯度。
3. 用二维点集手算 LDA 投影方向。
4. 比较 OvO 与 OvR 在类别不平衡时的风险。

## 长讲义扩展：线性模型为什么长盛不衰

线性模型看起来简单，却是机器学习中最重要的模型族之一。很多复杂模型最终也会在某个表示空间里使用线性组合。例如神经网络最后一层通常是线性层，SVM 的核心也是在特征空间中寻找线性分隔面，广义线性模型更是把线性打分与不同输出分布连接起来。

线性模型的强大之处不在于它能表达所有复杂关系，而在于它提供了一个清晰、稳定、可解释的起点。每个特征都有一个权重，权重正负代表影响方向，绝对值大小代表影响强弱。在业务场景中，这种透明性非常宝贵。

## 线性回归的完整故事

线性回归解决的是连续值预测问题。它假设输出可以由输入特征加权求和得到：

```text
预测值 = 基础值 + 特征1贡献 + 特征2贡献 + ... + 特征d贡献
```

以房价为例：

```text
房价 = b + w1 * 面积 + w2 * 距地铁距离 + w3 * 楼龄
```

若 `w1` 为正，说明面积越大房价越高；若 `w2` 为负，说明离地铁越远房价越低；若 `w3` 为负，说明楼龄越老价格越低。训练目标是让所有样本的预测误差平方和最小。

为什么用平方误差？一方面，平方误差可导且有解析解；另一方面，大误差会被更重惩罚。预测错 10 个单位的惩罚不是错 1 个单位的 10 倍，而是 100 倍。这适合我们特别不希望出现离谱预测的场景。

## 正规方程的直觉

最小二乘的正规方程：

```text
X^T X w = X^T y
```

可以从几何角度理解。线性回归要在由特征列张成的空间中，找一个最接近真实标签向量 `y` 的预测向量 `Xw`。最优时，残差 `y - Xw` 应该与这个特征空间正交。若残差还在某个特征方向上有投影，就说明沿那个方向还能继续调整权重来降低误差。

因此正规方程不是凭空出现的代数式，它表达的是“误差已经无法再被任何特征方向解释”。

## 多重共线性与正则化

当特征高度相关时，`X^T X` 可能接近不可逆。例如房屋面积和房间数通常相关，模型可能难以区分到底是面积贡献大，还是房间数贡献大。此时普通最小二乘的参数会变得不稳定：数据轻微变化，权重可能大幅波动。

岭回归通过加入 L2 正则解决这个问题：

```text
||y - Xw||^2 + lambda ||w||^2
```

它不只关心拟合误差，还惩罚过大的参数。直觉上，若两个解释差不多的模型，一个权重很极端，另一个权重更平缓，岭回归更偏好后者。这样通常会牺牲一点训练拟合，换来更好的稳定性。

Lasso 使用 L1 正则，进一步鼓励一些权重变成 0。它适合希望自动选择特征的场景。但如果多个特征高度相关，Lasso 可能随机保留其中一个，而不是保留全部。

## 逻辑回归为什么不是“回归”

逻辑回归名字里有“回归”，但它是分类模型。它先像线性回归一样计算一个分数：

```text
z = w^T x + b
```

然后用 sigmoid 把分数转成概率：

```text
p = 1 / (1 + exp(-z))
```

当 `z=0` 时，概率为 0.5；当 `z` 很大，概率接近 1；当 `z` 很小，概率接近 0。这使模型能输出“属于正类的置信程度”。

逻辑回归真正线性的不是概率本身，而是对数几率：

```text
log(p/(1-p)) = w^T x + b
```

这句话很重要。它说明每个特征不是直接让概率线性变化，而是让“正类几率相对负类几率”的对数线性变化。

## 一个逻辑回归小例子

假设判断用户是否会购买商品，模型为：

```text
z = -3 + 0.8 * 浏览次数 + 1.2 * 是否加入购物车
```

若用户浏览 2 次，未加入购物车：

```text
z = -3 + 0.8*2 + 1.2*0 = -1.4
p ≈ 0.20
```

若浏览 3 次，并加入购物车：

```text
z = -3 + 0.8*3 + 1.2*1 = 0.6
p ≈ 0.65
```

这说明加入购物车和浏览次数都提高购买概率，但它们先影响线性分数，再经过 sigmoid 转换。

## 为什么逻辑回归用交叉熵

二分类中，若真实标签为 1，我们希望预测概率 `p` 越接近 1 越好，损失应为 `-log p`；若真实标签为 0，我们希望 `p` 越接近 0 越好，损失应为 `-log(1-p)`。合起来就是交叉熵：

```text
-[y log p + (1-y) log(1-p)]
```

交叉熵的惩罚很符合直觉：如果真实为 1，模型却只给 0.01 的概率，损失会非常大；如果给 0.9，损失就很小。它特别惩罚“自信但错误”的预测。

## LDA 与逻辑回归的差异

LDA 和逻辑回归都能产生线性分类边界，但思路不同。逻辑回归是判别式模型，直接学习 `P(y|x)` 或分类边界；LDA 更接近生成式思路，它假设每个类别的数据服从具有相同协方差的高斯分布，然后推导出线性边界。

如果 LDA 的分布假设大致成立，少量数据时它可能表现很好；如果分布假设明显错误，逻辑回归往往更灵活。实践中，两者都可以作为强基线。

## 线性模型的失效场景

线性模型难以直接处理复杂非线性。例如 XOR 问题中，两个特征单独看都不能线性决定类别，只有组合关系才有意义。此时可以通过特征工程构造交叉项，也可以使用核方法、决策树、神经网络等非线性模型。

但不要因此轻视线性模型。很多高维稀疏任务，如文本分类，线性模型非常强。原因是高维空间中，线性边界已经能表达大量复杂区分。

## 实践建议

训练线性模型前，通常应检查特征尺度。L1、L2 正则对尺度敏感，如果一个特征数值范围很大，它可能在正则项下占便宜或吃亏。标准化能让权重更可比较，也能改善优化。

解释线性模型时，不要只看权重大小，还要结合特征尺度和相关性。高度相关特征会分摊或争夺权重，单个权重不一定代表真实因果影响。

线性模型是机器学习中的“直尺”。它不一定能画出所有复杂边界，但能给你一个清楚、稳定、可解释的参照。如果复杂模型只比线性模型好一点，却代价更大、更难解释，就需要认真评估是否值得。

## 万字版补充：从一维线性回归手算开始

考虑三个样本：

```text
x: 1, 2, 3
y: 2, 4, 5
```

我们用 `y = wx + b` 拟合。直觉上，`x` 每增加 1，`y` 大约增加 1.5 左右，截距接近 0.67。最小二乘会选择让三个点总平方误差最小的直线。

手算时，可以使用均值形式：

```text
w = sum_i (x_i - xbar)(y_i - ybar) / sum_i (x_i - xbar)^2
b = ybar - w xbar
```

这个公式说明，斜率来自 `x` 和 `y` 的共同变化。如果 `x` 高于均值时 `y` 也高于均值，乘积为正，斜率倾向为正。

## 残差分析

线性回归训练后，不应只看 MSE，还应看残差。残差是：

```text
e_i = y_i - yhat_i
```

如果残差随机分布在 0 附近，线性模型可能合适。如果残差随预测值呈现曲线模式，说明真实关系可能非线性。如果残差方差随 `x` 增大而增大，说明存在异方差。若有少数极大残差，可能存在异常点。

残差分析能告诉你模型哪里不合适，而不是只给一个平均误差。

## 特征变换让线性模型变强

线性模型并不只能拟合直线。只要对特征做变换，它可以拟合非线性关系。例如：

```text
y = w1 x + w2 x^2 + b
```

对参数 `w1, w2, b` 仍然是线性的，只是输入特征变成了 `x` 和 `x^2`。这叫线性于参数，而不是线性于原始输入。

类似地，可以加入交叉特征：

```text
收入 * 年龄
浏览次数 * 是否加购
```

线性模型能否表现好，很大程度取决于特征表示是否合适。

## 逻辑回归的决策边界

逻辑回归输出概率，但分类边界由：

```text
w^T x + b = 0
```

决定。因为 sigmoid 单调，`p=0.5` 对应 `z=0`。若调整分类阈值为 0.7，边界变为：

```text
w^T x + b = log(0.7 / 0.3)
```

这说明阈值移动会平移边界，而不是重新训练模型。业务中可以通过阈值控制精准率和召回率。

## 逻辑回归的权重解释

逻辑回归中，权重 `w_j` 表示特征 `x_j` 增加一个单位时，对数几率增加 `w_j`。取指数后，`exp(w_j)` 表示几率倍率。

例如 `w_j = 0.7`，则 `exp(0.7) ≈ 2.01`，表示该特征增加一个单位，正类几率约变为原来的 2 倍。这个解释在医疗、风控等领域很有用。

但解释权重时要注意特征尺度。如果一个特征单位是“元”，另一个单位是“万元”，权重不能直接比较。标准化后权重更可比。

## 多分类逻辑回归

二分类用 sigmoid，多分类常用 softmax。每个类别都有一组权重：

```text
z_k = w_k^T x + b_k
p_k = exp(z_k) / sum_j exp(z_j)
```

模型为每个类别打分，再归一化成概率。训练目标是让真实类别概率尽量高。softmax 回归是神经网络分类头的基础。

## 类别不平衡下的线性模型

若正类很少，逻辑回归可能倾向预测负类。可以使用类别权重，让正类样本损失更大；也可以调整阈值；还可以重采样。

例如正类只占 1%，默认阈值 0.5 可能几乎不预测正类。若业务希望捕捉更多正类，可以把阈值降到 0.1 或 0.05，再根据验证集选择合适点。

## LDA 的概率模型解释

LDA 假设每个类别服从高斯分布，且共享协方差矩阵。共享协方差意味着不同类别的分布形状相同，只是中心不同。在这个假设下，最大后验分类边界是线性的。

如果各类别协方差不同，边界会变成二次形式，对应 QDA。QDA 更灵活，但参数更多，需要更多数据。

## 线性模型与可解释性边界

线性模型可解释，但不是自动因果。权重表示相关性下的预测贡献，不等于改变该特征一定导致结果改变。

例如模型发现“购买高价手机”和“信用好”相关，不代表给用户换手机能提升信用。因果解释需要实验、工具变量、匹配等额外方法。

## 实践中的建模流程

一个常见线性模型流程：

1. 清洗数据，处理缺失和异常。
2. 对数值特征标准化。
3. 对类别特征做 one-hot 或目标编码。
4. 建立无正则或弱正则基线。
5. 使用 L1、L2 或 Elastic Net 调参。
6. 检查验证指标、残差或错误样本。
7. 解释重要权重，和业务常识核对。

线性模型训练快，适合作为第一版模型和诊断工具。

## 本章自测问答

问：线性回归为什么最小化平方误差？

答：平方误差可导、计算方便、对大误差惩罚更重；在高斯噪声假设下也对应极大似然估计。

问：逻辑回归为什么能输出概率？

答：它用 sigmoid 把线性分数映射到 0 到 1，并通过交叉熵训练概率预测。

问：L1 和 L2 最大区别是什么？

答：L2 倾向缩小权重，L1 倾向产生稀疏权重，把部分特征权重压到 0。

问：线性模型一定欠拟合吗？

答：不一定。高维稀疏任务中特征空间很丰富，线性模型可以非常强；通过特征变换也能表达非线性。

## 继续扩展：线性模型的面试高频问题

问：线性回归和逻辑回归的损失函数为什么不同？

答：线性回归预测连续值，常假设噪声服从高斯分布，平方误差对应高斯极大似然。逻辑回归预测类别概率，使用伯努利分布似然，负对数似然就是交叉熵。

问：逻辑回归为什么常需要正则化？

答：当特征多、样本少或数据近似线性可分时，逻辑回归权重可能变得很大，导致过拟合和概率过度自信。正则化限制参数规模，提高泛化。

问：为什么多重共线性会影响线性回归？

答：高度相关特征使 `X^T X` 接近奇异，参数估计不稳定。预测可能还可以，但单个系数解释会变差。

问：岭回归为什么能解决不可逆？

答：`X^T X + lambda I` 增加了正对角项，使矩阵更容易可逆，也降低参数方差。

## 线性模型的错误案例

假设预测工资，特征包括年龄。工资和年龄可能不是简单线性关系：年轻时工资随年龄增长较快，中年后趋于平稳，退休后下降。若只用 `年龄` 一个线性项，模型会欠拟合。

可以加入：

```text
年龄^2
年龄分段
工作年限
行业
职位
地区
```

这说明线性模型表现差时，不一定要马上换复杂模型，也可以先改进特征表示。

## 线性模型在文本分类中的优势

文本可以表示成词袋或 TF-IDF 向量，维度很高但稀疏。每个词就是一个特征。线性模型可以学习每个词对类别的贡献。

例如垃圾邮件分类中：

```text
中奖、免费、点击、领取 -> 正向垃圾权重
会议、报告、项目、附件 -> 正常邮件权重
```

虽然模型是线性的，但高维词特征已经提供了丰富表达。线性分类器训练快、解释清楚，常是文本任务强基线。

## 实践中的特征缩放细节

对带正则的线性模型，标准化很重要。若一个特征范围是 0 到 1，另一个是 0 到 100000，正则对它们的影响不公平。

常用标准化：

```text
x' = (x - mean) / std
```

注意均值和标准差只能在训练集上计算，再应用到验证集和测试集。

## 本章 checklist

训练线性模型前检查：

- 标签是否适合回归或分类。
- 特征是否需要标准化。
- 类别特征是否正确编码。
- 是否存在强共线性。
- 是否需要 L1、L2 或 Elastic Net。
- 指标是否匹配任务。
- 权重解释是否考虑尺度和相关性。

## 万字级补充：从线性模型看“可解释预测”

线性模型常被用作可解释模型，但解释时要有边界。一个正权重说明在其他特征固定时，该特征增大会提高预测分数；负权重则相反。这里的“其他特征固定”非常关键，因为现实中特征常常一起变化。

例如模型中“年龄”权重为正，不代表让一个人年龄变大就会立刻提高收入；它只是说在数据中，控制其他特征后，年龄与收入预测存在正向关系。若年龄和工作年限高度相关，权重解释还会更复杂。

因此，线性模型适合解释预测逻辑，但不能自动给出因果结论。要讨论因果，需要实验设计、自然实验、工具变量或因果图等额外工具。

## 线性模型的工程优势

线性模型训练快、预测快、内存占用低，适合大规模稀疏数据。广告点击率预估、搜索排序初筛、文本分类、风控基线中，线性模型长期都有价值。

在线系统中，延迟非常重要。一个复杂模型如果只提升很小指标，却显著增加延迟和维护成本，未必值得。线性模型的简单性本身就是工程优势。

## 广义线性模型视角

线性回归和逻辑回归都可放在广义线性模型框架下。核心思想是：线性预测器 `w^T x` 通过连接函数与输出分布的均值联系起来。

线性回归对应高斯分布和恒等连接，逻辑回归对应伯努利分布和 logit 连接，泊松回归对应计数数据和对数连接。这个视角说明，线性思想可以适配不同类型输出。

## 本章扩展练习

1. 用一个小数据集手算一元线性回归参数。
2. 解释逻辑回归中权重和 odds ratio 的关系。
3. 构造一个线性模型无法解决的 XOR 数据集。
4. 对一个线性模型结果，说明哪些解释是相关性解释，哪些不能解释为因果。
5. 比较 L1、L2 和 Elastic Net 适合的场景。

## 讲义级补充：如何真正学懂《第 3 章 线性模型》

> 适用位置：机器学习-西瓜书南瓜书学习笔记\chapters\03-线性模型.md  
> 说明：本补充用于把原始提纲扩展成课堂讲义式学习材料。阅读时建议先看原文，再用本节建立知识框架、例子、实践和自测闭环。

### 1. 这一讲要解决什么问题

这类内容的核心不是公式本身，而是“数据如何变成可学习的表示，模型如何产生预测，误差如何反馈改进”。学习时要把数据、假设、损失、优化、评估和部署放在一起理解。

学习本讲时，可以用三个问题检查自己是否真的理解：

1. 它解决的真实问题是什么？
2. 如果没有它，系统会出现什么具体麻烦？
3. 在真实项目中，应该用什么现象或指标判断它做得好不好？

### 2. 核心知识拆解

可以把本讲拆成几块来学：

- 数据：样本、标签、分布、增强、清洗和划分。
- 模型：假设空间、结构、参数和表达能力。
- 训练：损失函数、优化器、正则化、学习率和收敛。
- 评估：指标、误差分析、泛化、鲁棒性和部署表现。

拆解的好处是防止“整章都懂一点，但哪块都说不清”。复习时可以逐块追问：它的输入是什么、输出是什么、依赖什么、失败时有什么表现。

### 3. 通俗类比

可以把模型类比成一个需要反复练习的学生：数据是练习题，标签是参考答案，损失函数是扣分规则，优化器是改错方法，验证集像模拟考试，测试集才像正式考试。只在训练集上表现好，就像只会背做过的题。

类比不是严格定义，但能帮助初学者先建立直觉。真正使用时，还要回到术语、公式、接口、数据结构、时序图或工程规范上，把“感觉理解”变成“可验证理解”。

### 4. 具体例子

学习《第 3 章 线性模型》时，可以准备一个很小的数据集，先手工看 20 个样本和标签，再训练一个基线模型，最后把预测错误的样本逐张分析。很多模型问题不是公式错了，而是数据、标注或评价方式一开始就没有定义清楚。

讲义级学习不能只停留在“概念解释”。至少要有一个能跑、能算、能画或能检查的例子。例子越小，越容易看清关键机制；等机制清楚后，再逐步扩展到复杂项目。

### 5. 学习路径

- 先明确任务类型：分类、回归、检测、分割、生成、序列建模或决策。
- 再梳理数据、特征、模型、损失函数、优化方法和评价指标。
- 最后通过误差分析定位问题来自数据、标注、模型容量、训练策略还是部署环境。

建议每学完一小节都做一次“复述练习”：不用看笔记，用自己的话讲清楚概念、输入、输出、关键步骤和常见错误。如果讲不清，通常说明还没有真正掌握。

### 6. 课堂讲解框架

可以按下面顺序讲解或复习本主题：

1. 背景：先讲这个知识为什么出现，它试图降低什么成本、解决什么风险或提升什么能力。
2. 基本概念：给出核心名词的准确定义，说明它们之间的关系。
3. 工作流程：按时间顺序描述一次完整过程，必要时画出流程图、状态机或数据流图。
4. 关键细节：解释最容易误解的机制，例如边界条件、异常处理、性能限制、资源生命周期或安全约束。
5. 实战例子：用一个足够小但完整的例子，把概念落到命令、代码、图纸、配置、数据或操作步骤上。
6. 反例与排错：展示错误做法会导致什么现象，再说明如何定位和修复。
7. 总结迁移：最后说明它和相邻知识点的区别、联系以及后续该学什么。

### 7. 最小实践任务

为了避免“看懂了但不会用”，建议为本讲配一个最小实践：

- 选一个可以在 30 到 90 分钟内完成的小任务。
- 明确输入、预期输出和验收标准。
- 记录遇到的第一个错误、定位过程和最终修复方法。
- 完成后写 5 行复盘：我原来以为是什么，实际是什么，下次会如何更快处理。

如果本主题偏理论，实践可以是手算一个小例子、画一张流程图、推导一个简化公式或解释一段真实日志；如果偏工程，实践应该尽量落到可运行命令、可测试代码、可检查配置或可测量硬件现象上。

### 8. 常见误区

- 只追求模型结构新，不检查数据质量和标注一致性。
- 只看平均指标，不看错误样本、长尾类别和业务代价。
- 把训练集效果当成真实效果，忽略验证集、测试集和线上分布变化。

遇到这些问题时，不要急着背更多资料。更有效的办法是回到一个最小例子，把输入、状态变化、输出和验证方式重新走一遍。

### 9. 自测题

1. 用一句话说明本讲主题解决的核心问题。
2. 列出本讲最重要的 3 个概念，并说明它们的关系。
3. 举一个生活类比，再指出这个类比在哪些地方不严谨。
4. 写出一个最小实践任务的验收标准。
5. 如果结果不符合预期，你会优先检查哪 3 个环节？为什么？
6. 本讲和相邻章节的知识边界是什么？哪些问题应该交给其他章节解决？

先问场景，再看输入；先拆结构，再走流程；先做小例，再谈优化；先会排错，再做规模化。
