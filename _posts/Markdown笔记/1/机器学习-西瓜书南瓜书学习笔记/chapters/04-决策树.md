# 第 4 章 决策树

## 学习目标

决策树通过一系列属性测试完成分类或回归。本章重点是划分准则、剪枝、连续值处理、缺失值处理和树模型的归纳偏好。

## 先用人话理解

决策树像一套问答流程：先问最关键的问题，根据答案进入下一步，再继续问问题，直到能给出判断。例如挑西瓜时可能先问“纹理是否清晰”，再问“根蒂是否蜷缩”，最后判断是不是好瓜。

它的优点是像人类规则一样容易读懂；缺点是如果问题问得太细，就可能把训练样本中的偶然现象也记住，导致过拟合。

## 一个小例子

判断一个水果是不是苹果，可以构造这样的树：

```text
是否圆形？
  否 -> 不是苹果
  是 -> 颜色是否红/绿？
       否 -> 不是苹果
       是 -> 直径是否在合理范围？
            是 -> 可能是苹果
            否 -> 不是苹果
```

机器学习中的决策树不是人工手写问题顺序，而是根据数据自动选择“最能把样本分清楚”的问题。

## 基本流程

决策树从根节点开始，递归选择一个属性划分样本。停止条件通常包括：

- 当前节点样本属于同一类别。
- 属性集为空或所有样本在剩余属性上取值相同。
- 节点样本数低于阈值。
- 划分收益低于阈值。

叶节点输出类别，多数情况下取当前节点样本中最多的类别。

## 信息熵

若数据集 `D` 中第 `k` 类比例为 `p_k`，信息熵：

```text
Ent(D) = - sum_k p_k log_2 p_k
```

熵越大，不确定性越高。若所有样本同类，熵为 0；若类别均匀混合，熵较大。

类比：一个盒子里如果全是红球，你摸之前就知道结果，不确定性为 0；如果红球蓝球各一半，你摸之前最没把握，不确定性高。

## 信息增益

属性 `a` 将数据集划分为若干子集 `D_v`，信息增益：

```text
Gain(D, a) = Ent(D) - sum_v (|D_v| / |D|) Ent(D_v)
```

推导直觉：划分前的不确定性减去划分后的加权不确定性。若某属性划分后子节点更纯，信息增益更大。

ID3 使用信息增益选择划分属性。

直觉上，信息增益是在问：“问完这个问题之后，我的不确定性减少了多少？”能让混杂样本变成更纯子集的问题，就是好问题。

## 增益率

信息增益偏好取值多的属性。例如样本编号能把每个样本单独分开，信息增益极高但没有泛化意义。

属性固有值：

```text
IV(a) = - sum_v (|D_v| / |D|) log_2(|D_v| / |D|)
```

增益率：

```text
Gain_ratio(D, a) = Gain(D, a) / IV(a)
```

C4.5 使用增益率，但通常先筛选信息增益高于平均水平的属性，再从中选择增益率最高者，避免偏好取值过少的属性。

## 基尼指数

基尼值：

```text
Gini(D) = 1 - sum_k p_k^2
```

属性划分后的基尼指数：

```text
Gini_index(D, a) = sum_v (|D_v| / |D|) Gini(D_v)
```

CART 选择使基尼指数最小的划分。

## 连续属性处理

对连续属性，通常将候选切分点设为相邻排序值的中点。若属性值排序为 `a_1 <= a_2 <= ... <= a_m`，候选切分点：

```text
T = { (a_i + a_{i+1}) / 2 | 1 <= i < m }
```

对每个切分点，把样本分成 `<= t` 与 `> t` 两组，选择最优切分点。

## 缺失值处理

可从两个问题处理：

1. 如何在属性值缺失时评估划分优劣。
2. 如何在划分后将缺失样本分配到子节点。

常见做法：

- 只用属性值非缺失样本计算划分收益，并按非缺失比例加权。
- 将缺失样本按各分支样本比例分配权重。
- 预测时遇到缺失属性，可沿多个分支传播并加权投票。

## 剪枝

剪枝用于降低过拟合。

预剪枝：在划分前判断划分是否能提升验证集性能，若不能则停止。

后剪枝：先生成完整树，再自底向上考察是否将子树替换为叶节点。

比较：

- 预剪枝效率高，但可能过早停止，带来欠拟合。
- 后剪枝通常泛化更好，但计算成本更高。

## 类比：决策树像写排查手册

如果一本故障排查手册写得太粗，比如“电脑坏了就重启”，它会欠拟合，解决不了复杂问题。如果写得太细，比如“周三下午插着某品牌鼠标时按三次电源键”，它可能只适合某一次事故，泛化很差。

剪枝就是把过细、没有普遍意义的判断删掉，让规则更稳。

## 决策树的特点

优点：

- 可解释性强。
- 可处理离散和连续属性。
- 对特征缩放不敏感。
- 能表达非线性和特征交互。

缺点：

- 容易过拟合。
- 对数据扰动敏感。
- 贪心划分不保证全局最优。
- 轴平行划分对某些斜向边界效率低。

## 易错点

- 信息增益越大不一定泛化越好，可能只是属性取值多。
- 剪枝判断应基于验证集或复杂度惩罚，而不是只看训练集。
- 决策树学习是贪心递归过程，不是全局搜索最优树。
- 连续属性可以在树中多次使用，离散属性在某些实现中用过后不再使用。

## 练习

1. 给定小数据集，手算信息熵和信息增益。
2. 比较信息增益、增益率和基尼指数的选择结果。
3. 用验证集模拟一次后剪枝。
4. 思考为什么随机森林能缓解决策树高方差问题。

## 长讲义扩展：决策树是机器自动写出的判断流程

决策树最吸引人的地方是可解释。很多算法给出的是一组参数或复杂函数，而决策树给出的是一棵问答树。每个内部节点问一个问题，每条边代表一个答案，每个叶节点给出最终判断。这种结构非常接近人类做决策的方式。

例如医生诊断可能会问：是否发烧？是否咳嗽？血常规是否异常？影像是否有阴影？信贷审批可能会问：收入是否稳定？负债率是否过高？是否有逾期记录？这些都是树式决策。

机器学习中的决策树与人工规则的区别在于：问题顺序不是专家手写，而是算法根据数据自动选择。选择标准就是“哪个问题最能降低不确定性”。

## 从一个小数据集看树如何生长

假设有 8 个西瓜样本，其中 4 个好瓜、4 个坏瓜。当前节点中好坏各半，不确定性很高。现在有两个候选属性：

- 色泽：青绿、乌黑、浅白。
- 敲声：浊响、沉闷、清脆。

如果按色泽划分后，每个子节点仍然好坏混杂，说明色泽帮助不大。如果按敲声划分后，“浊响”几乎都是好瓜，“清脆”几乎都是坏瓜，说明敲声更有区分力。信息增益、增益率、基尼指数都是在量化这种“分完以后更纯了吗”。

树的生成过程就是不断重复这个动作：在当前节点上寻找最有帮助的问题，把数据分成更纯的子集。

## 信息熵的直觉再展开

熵衡量的是不确定性。若一个节点里全是好瓜，根本不需要再问问题，熵为 0。若好瓜坏瓜各一半，你最不确定，熵最大。信息增益就是问：问某个属性之后，不确定性平均减少了多少？

这可以类比猜物游戏。你心里想一个动物，别人每次问一个是/否问题。好问题能迅速缩小范围，比如“它会飞吗”；差问题几乎不能缩小范围，比如“它是不是你昨天想到的动物”。决策树选划分属性，就是在自动寻找好问题。

## 为什么信息增益偏好取值多的属性

设想有一个属性叫“样本编号”，每个样本编号都不同。用它划分后，每个子节点只有一个样本，纯度必然最高，信息增益极大。但这个属性对新样本没有泛化意义，因为新样本的编号从未出现过。

这说明只看训练集纯度会鼓励模型记忆样本。增益率通过除以属性固有值惩罚取值过多的属性，试图避免这种偏好。但增益率又可能偏好取值过少的属性，所以 C4.5 通常先筛掉信息增益太低的属性，再比较增益率。

## 基尼指数为什么常用于 CART

基尼指数也衡量不纯度。若节点中类别比例为 `p_k`，随机抽两个样本类别不同的概率与基尼值相关。基尼越小，节点越纯。

相比信息熵，基尼指数不需要计算对数，计算更快；实际效果通常相近。CART 使用二叉划分和基尼指数，既可做分类树，也可做回归树。

## 回归树如何工作

分类树叶节点输出类别，回归树叶节点输出数值。回归树划分时不再看信息熵，而是看划分后平方误差是否下降。

例如预测房价时，一个节点里房价差异很大。如果用“面积 <= 90 平方米”划分后，小户型和大户型分别进入两个子节点，每个子节点内部房价更接近，那么这个划分就有价值。叶节点通常输出该叶中训练样本的平均值。

因此，回归树仍然是问答流程，只是目标从“让类别更纯”变成“让数值更集中”。

## 决策边界的形状

普通决策树每次按一个特征做划分，因此在二维空间中会形成横平竖直的边界。复杂边界可以由许多矩形区域拼接出来。这让决策树能表达非线性，但也带来问题：如果树太深，它可以把训练集切得非常碎，几乎记住每个样本。

这正是决策树容易过拟合的原因。单棵深树方差很高，数据轻微变化，树结构可能完全不同。随机森林通过训练多棵树并投票，能显著缓解这个问题。

## 剪枝的必要性

不剪枝的树会不断追求训练集纯度，直到叶节点很小甚至每个叶只有一个样本。这就像写规则时过度细化：

```text
如果色泽青绿、根蒂蜷缩、敲声浊响、纹理稍糊、密度 0.697，则是好瓜
```

这种规则对训练样本有效，但对新样本没有意义。剪枝就是删除不可靠的细枝末节，保留更稳定的主干。

预剪枝在树生长时提前停止，效率高但可能错过后续有用划分。后剪枝先长出完整树，再用验证集或复杂度惩罚删除子树，通常更稳健。

## 缺失值不是简单删除

现实数据经常缺失。若直接删除含缺失值的样本，可能浪费数据，甚至引入偏差。例如高收入用户更不愿填写收入，删除这些样本会改变数据分布。

决策树可以比较自然地处理缺失：计算划分优劣时只使用非缺失样本，并给划分收益乘上非缺失比例；分裂时把缺失样本按各分支比例分配权重；预测时也可以沿多个分支走并加权投票。

## 决策树的实践建议

使用决策树时，应重点调节：

- 最大深度：控制树的复杂度。
- 叶节点最小样本数：防止过小叶子记忆噪声。
- 最小划分增益：收益太小就不再划分。
- 剪枝强度：在拟合与泛化之间折中。

若目标是解释，浅树很有价值。若目标是预测性能，单棵树通常不是最强选择，可以考虑随机森林、梯度提升树等集成方法。

## 本章学习重点回收

决策树的核心不是某个公式，而是递归划分思想。每一步都问：当前数据最应该用哪个问题来分？公式只是衡量“这个问题好不好”的工具。

学完本章后，你应能从三个角度理解决策树：从人类角度，它是问答规则；从数学角度，它是降低不纯度的贪心搜索；从泛化角度，它是高方差模型，需要剪枝或集成来稳定。

## 万字版补充：手算一次信息增益

假设某节点有 10 个样本，6 个好瓜、4 个坏瓜。划分前熵为：

```text
Ent(D) = -0.6 log2 0.6 - 0.4 log2 0.4 ≈ 0.971
```

现在用“敲声”划分：

```text
浊响: 4 个样本，3 好 1 坏
沉闷: 3 个样本，2 好 1 坏
清脆: 3 个样本，1 好 2 坏
```

各子集熵分别约为：

```text
Ent(浊响) ≈ 0.811
Ent(沉闷) ≈ 0.918
Ent(清脆) ≈ 0.918
```

划分后加权熵：

```text
0.4*0.811 + 0.3*0.918 + 0.3*0.918 ≈ 0.875
```

信息增益：

```text
Gain = 0.971 - 0.875 = 0.096
```

如果另一个属性划分后加权熵更低，它的信息增益就更高。手算一次后就能明白：信息增益不是神秘公式，只是在算“问完问题后少了多少不确定性”。

## 连续值划分示例

假设用密度判断好瓜，样本密度排序后为：

```text
0.32, 0.41, 0.52, 0.61, 0.73
```

候选切分点是相邻值中点：

```text
0.365, 0.465, 0.565, 0.670
```

对每个切分点，把样本分成“小于等于阈值”和“大于阈值”两组，计算信息增益或基尼指数，选择最优阈值。

这说明决策树处理连续特征时，本质上是在学习一组阈值规则。

## 决策树为什么对特征缩放不敏感

线性模型和 SVM 常需要标准化，因为权重和距离受尺度影响。决策树主要关心排序和阈值。例如密度单位从 `kg/m^3` 换成另一种线性单位，样本顺序不变，候选切分本质不变。

这使树模型在表格数据中很方便。不过，如果特征经过非单调变换，排序改变，树的划分可能变化。

## 决策树处理类别特征

类别特征可以按取值多路划分，也可以在 CART 中寻找二分子集。例如颜色有红、绿、蓝，二叉树可以尝试：

```text
{红} vs {绿, 蓝}
{绿} vs {红, 蓝}
{蓝} vs {红, 绿}
```

类别取值很多时，搜索空间会变大，也容易过拟合。因此高基数类别特征要谨慎处理，如用户 ID、商品 ID、邮编等。

## 决策树和规则系统的区别

决策树每个样本从根走到一个叶，路径唯一。规则系统可以有多条规则同时匹配一个样本，也可以设置优先级或投票。

决策树更结构化，容易训练；规则系统更灵活，便于加入人工规则。许多业务系统会把树模型发现的模式转成人工可审核规则。

## 决策树的稳定性问题

决策树使用贪心划分。根节点一旦选了某个属性，后续结构都会受影响。如果两个属性收益接近，训练数据轻微变化可能导致根节点不同，整棵树也不同。

这就是单棵树高方差的来源。它不是小缺陷，而是树模型的重要性质。随机森林通过平均多棵不同树降低这种不稳定性，梯度提升树通过逐步拟合残差提升预测能力。

## 后剪枝示例

假设一棵子树在训练集上把 20 个样本分成多个叶节点，训练错误 1 个；如果把整棵子树替换成一个叶节点，训练错误变成 4 个。只看训练集，保留子树更好。

但在验证集上，子树错误 6 个，替换成叶节点错误 3 个，说明子树过拟合。后剪枝会删除这棵子树。

剪枝的核心思想是：训练集收益不一定代表泛化收益，复杂结构必须证明自己在验证数据上有用。

## 决策树在业务中的解释方式

解释一棵树时，不要只展示完整树。深树很难读。更常见做法是：

- 展示前几层关键划分。
- 提取覆盖样本多、准确率高的路径规则。
- 分析特征重要性。
- 对典型样本展示决策路径。

例如风控模型可以解释为：“该用户被判为高风险，路径是：历史逾期次数 > 2，近 30 天申请次数 > 5，收入稳定性低。”这种解释比单纯给一个分数更容易被业务理解。

## 树模型的实践参数

常见参数及直觉：

- `max_depth`：树最大深度，越大越复杂。
- `min_samples_leaf`：叶节点最少样本数，越大越平滑。
- `min_samples_split`：节点继续划分所需最少样本。
- `max_features`：每次划分考虑的特征数，随机森林中常用。
- `ccp_alpha` 或剪枝参数：控制复杂度惩罚。

调参时可以先限制深度和叶节点最小样本数，观察训练与验证差距。如果训练好验证差，说明树太复杂；如果两者都差，可能树太浅或特征不足。

## 本章自测问答

问：为什么信息增益偏好取值多的属性？

答：取值多的属性更容易把训练样本切得很碎，使子节点纯度高，但这种划分可能没有泛化意义。

问：预剪枝和后剪枝哪个更稳？

答：后剪枝通常更稳，因为它先允许树发现结构，再用验证集删除无效子树；预剪枝可能过早停止。

问：为什么随机森林常比单棵树好？

答：单棵树方差高，随机森林通过样本随机和特征随机训练多棵树，平均后降低方差。

问：决策树能处理非线性吗？

答：能。它通过多次轴平行划分拼接出复杂区域，但边界可能需要很多节点才能逼近斜向或曲线结构。

## 继续扩展：ID3、C4.5、CART 对比

ID3 使用信息增益，主要处理离散属性，容易偏好取值多的属性。

C4.5 使用增益率，支持连续属性和缺失值处理，并引入剪枝。它可以看作 ID3 的改进版本。

CART 使用二叉树结构，分类树常用基尼指数，回归树使用平方误差。CART 的二叉结构便于剪枝和集成，是很多现代树模型的基础。

理解三者时，不要只背名字，而要看它们在回答哪些问题：

```text
用什么标准选划分？
支持什么类型特征？
树是多叉还是二叉？
如何防止过拟合？
能否做回归？
```

## 回归树手算直觉

假设 6 个房屋样本，按面积排序，价格分别为：

```text
面积: 60, 70, 80, 120, 130, 140
价格: 200, 220, 240, 400, 420, 440
```

如果不划分，叶节点预测均值 320，误差很大。若按 `面积 <= 100` 划分：

```text
左叶均值: 220
右叶均值: 420
```

两个叶节点内部误差明显降低。回归树选择划分点，就是寻找让子节点数值更集中的切分。

## 决策树与缺失值的业务处理

有时缺失本身有意义。比如用户没有填写收入，可能和风险有关；商品没有评分，可能是新品。简单填均值会丢失“是否缺失”这个信号。

常用做法：

- 填充默认值并增加“是否缺失”指示特征。
- 树模型中把缺失作为单独分支。
- 使用模型内置缺失方向。

具体选择取决于缺失机制：完全随机缺失、依赖观测变量缺失，还是依赖未观测因素缺失。

## 树模型的解释风险

单棵浅树很容易解释，但深树的路径可能很多。过深的树虽然每条路径看起来像规则，却可能只是训练样本的偶然切分。

解释树模型时应关注稳定规则。可以在不同训练子集上训练树，观察关键划分是否一致。如果规则不稳定，就不应过度解释。

## 从决策树到集成树

单棵树高方差，但它是集成树的基础。随机森林通过多棵树投票降低方差，GBDT 通过多棵树逐步修正错误降低偏差。

因此学好决策树，对理解后面的集成学习非常关键。信息增益、基尼指数、平方误差划分、剪枝这些概念都会在集成树中继续出现。

## 本章 checklist

使用决策树时检查：

- 树是否过深。
- 叶节点样本是否过少。
- 训练和验证误差差距是否过大。
- 高基数类别特征是否导致过拟合。
- 缺失值是否有合理处理。
- 解释规则是否稳定。

## 万字级补充：决策树为什么适合表格数据

表格数据常包含数值特征、类别特征、缺失值、非线性关系和特征交互。决策树天然适合这类数据，因为它不要求特征线性可加，也不强依赖标准化。

例如“收入高”是否意味着低风险，可能取决于“负债率”是否也高；“年龄”影响可能在不同职业中不同。树模型通过分裂路径自然表达这种交互：

```text
如果负债率高，再看收入
如果负债率低，再看历史逾期
```

线性模型需要人工构造交互项，而树可以自动发现一部分交互。

## 树模型的局部性

决策树的每个叶节点只负责一块局部区域。一个样本的预测由它落入的叶节点决定。这样做的好处是能表达局部规律，坏处是叶节点样本少时估计不稳定。

这也是为什么要设置叶节点最小样本数。叶子太小，预测像记忆；叶子适当大，预测更像局部平均规律。

## 决策树的外推能力弱

回归树对训练数据范围外的样本外推能力较弱。因为叶节点输出通常是训练样本均值。若房屋面积超过训练集中最大值，线性回归可能继续按斜率外推，回归树通常只能输出某个叶子的平均价格。

因此，在需要外推的场景中，树模型要谨慎。它擅长插值和分段规则，不擅长基于趋势外推。

## 本章扩展练习

1. 构造一个样本编号属性，说明为什么信息增益会偏好它。
2. 画出二维决策树的轴平行划分边界。
3. 对一个回归问题，手算某个切分点前后的平方误差下降。
4. 说明树深度、叶节点样本数如何影响偏差和方差。
5. 比较单棵树、随机森林和 GBDT 的解释性。

## 讲义级补充：如何真正学懂《第 4 章 决策树》

> 适用位置：机器学习-西瓜书南瓜书学习笔记\chapters\04-决策树.md  
> 说明：本补充用于把原始提纲扩展成课堂讲义式学习材料。阅读时建议先看原文，再用本节建立知识框架、例子、实践和自测闭环。

### 1. 这一讲要解决什么问题

这类内容的核心不是公式本身，而是“数据如何变成可学习的表示，模型如何产生预测，误差如何反馈改进”。学习时要把数据、假设、损失、优化、评估和部署放在一起理解。

学习本讲时，可以用三个问题检查自己是否真的理解：

1. 它解决的真实问题是什么？
2. 如果没有它，系统会出现什么具体麻烦？
3. 在真实项目中，应该用什么现象或指标判断它做得好不好？

### 2. 核心知识拆解

可以把本讲拆成几块来学：

- 数据：样本、标签、分布、增强、清洗和划分。
- 模型：假设空间、结构、参数和表达能力。
- 训练：损失函数、优化器、正则化、学习率和收敛。
- 评估：指标、误差分析、泛化、鲁棒性和部署表现。

拆解的好处是防止“整章都懂一点，但哪块都说不清”。复习时可以逐块追问：它的输入是什么、输出是什么、依赖什么、失败时有什么表现。

### 3. 通俗类比

可以把模型类比成一个需要反复练习的学生：数据是练习题，标签是参考答案，损失函数是扣分规则，优化器是改错方法，验证集像模拟考试，测试集才像正式考试。只在训练集上表现好，就像只会背做过的题。

类比不是严格定义，但能帮助初学者先建立直觉。真正使用时，还要回到术语、公式、接口、数据结构、时序图或工程规范上，把“感觉理解”变成“可验证理解”。

### 4. 具体例子

学习《第 4 章 决策树》时，可以准备一个很小的数据集，先手工看 20 个样本和标签，再训练一个基线模型，最后把预测错误的样本逐张分析。很多模型问题不是公式错了，而是数据、标注或评价方式一开始就没有定义清楚。

讲义级学习不能只停留在“概念解释”。至少要有一个能跑、能算、能画或能检查的例子。例子越小，越容易看清关键机制；等机制清楚后，再逐步扩展到复杂项目。

### 5. 学习路径

- 先明确任务类型：分类、回归、检测、分割、生成、序列建模或决策。
- 再梳理数据、特征、模型、损失函数、优化方法和评价指标。
- 最后通过误差分析定位问题来自数据、标注、模型容量、训练策略还是部署环境。

建议每学完一小节都做一次“复述练习”：不用看笔记，用自己的话讲清楚概念、输入、输出、关键步骤和常见错误。如果讲不清，通常说明还没有真正掌握。

### 6. 课堂讲解框架

可以按下面顺序讲解或复习本主题：

1. 背景：先讲这个知识为什么出现，它试图降低什么成本、解决什么风险或提升什么能力。
2. 基本概念：给出核心名词的准确定义，说明它们之间的关系。
3. 工作流程：按时间顺序描述一次完整过程，必要时画出流程图、状态机或数据流图。
4. 关键细节：解释最容易误解的机制，例如边界条件、异常处理、性能限制、资源生命周期或安全约束。
5. 实战例子：用一个足够小但完整的例子，把概念落到命令、代码、图纸、配置、数据或操作步骤上。
6. 反例与排错：展示错误做法会导致什么现象，再说明如何定位和修复。
7. 总结迁移：最后说明它和相邻知识点的区别、联系以及后续该学什么。

### 7. 最小实践任务

为了避免“看懂了但不会用”，建议为本讲配一个最小实践：

- 选一个可以在 30 到 90 分钟内完成的小任务。
- 明确输入、预期输出和验收标准。
- 记录遇到的第一个错误、定位过程和最终修复方法。
- 完成后写 5 行复盘：我原来以为是什么，实际是什么，下次会如何更快处理。

如果本主题偏理论，实践可以是手算一个小例子、画一张流程图、推导一个简化公式或解释一段真实日志；如果偏工程，实践应该尽量落到可运行命令、可测试代码、可检查配置或可测量硬件现象上。

### 8. 常见误区

- 只追求模型结构新，不检查数据质量和标注一致性。
- 只看平均指标，不看错误样本、长尾类别和业务代价。
- 把训练集效果当成真实效果，忽略验证集、测试集和线上分布变化。

遇到这些问题时，不要急着背更多资料。更有效的办法是回到一个最小例子，把输入、状态变化、输出和验证方式重新走一遍。

### 9. 自测题

1. 用一句话说明本讲主题解决的核心问题。
2. 列出本讲最重要的 3 个概念，并说明它们的关系。
3. 举一个生活类比，再指出这个类比在哪些地方不严谨。
4. 写出一个最小实践任务的验收标准。
5. 如果结果不符合预期，你会优先检查哪 3 个环节？为什么？
6. 本讲和相邻章节的知识边界是什么？哪些问题应该交给其他章节解决？

### 10. 复习口诀

先问场景，再看输入；先拆结构，再走流程；先做小例，再谈优化；先会排错，再做规模化。
