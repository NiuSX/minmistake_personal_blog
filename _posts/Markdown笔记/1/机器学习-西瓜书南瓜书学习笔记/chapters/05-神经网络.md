## 讲义导读：用泛化主线读懂这一章

这一章讲的是 **第 5 章 神经网络**。阅读时不要只背公式和算法名，而要把它放进机器学习的完整链路：现实问题如何变成数据，样本和标签是否可靠，模型假设空间是什么，损失函数如何定义好坏，优化算法如何找到参数，验证集如何选择模型，测试集如何估计泛化，最后错误样本如何反过来改进数据和模型。机器学习真正要解决的不是训练集上得分高，而是在未来数据上稳定可用。

### 一句话先懂

神经网络用多层非线性组合学习复杂函数，关键不是神秘智能，而是表示能力、反向传播和泛化控制。

### 通俗类比

可以把机器学习想成从错题本里总结解题方法：训练集是做过的题，模型是假设的解题套路，损失函数是扣分规则，优化是反复订正，验证集像模拟考，测试集像正式考试，泛化能力就是换一套题还能不能做对。复杂模型不是天生更聪明，它只是可选择的解题套路更多，也更容易把错题本里的偶然细节背下来。

类比只能帮助入门。回到技术上，要把每个概念对应到精确定义：样本、特征、标签、假设空间、目标函数、训练误差、泛化误差、评价指标、正则化和数据分布。只有这些对象清楚，公式才不是符号堆，实验结果才有解释意义。

### 本章学习主线

1. **问题定义**：先确定任务是分类、回归、聚类、降维、序列决策还是概率推断。
2. **数据假设**：检查样本来源、标签噪声、特征含义、训练测试分布和数据泄漏。
3. **模型偏好**：说清算法偏好什么样的规律，例如线性、大间隔、短规则、平滑函数、稀疏特征或局部相似。
4. **损失与优化**：理解模型到底在最小化什么，优化失败、局部最优或过拟合会怎样表现。
5. **评估与泛化**：用合适指标、交叉验证、学习曲线、误差分析和消融实验判断模型是否可靠。
6. **落地边界**：关注可解释性、推理成本、鲁棒性、公平性、分布漂移和维护成本。

## 学习目标

本章重点是理解神经元模型、多层前馈网络、误差反向传播、正则化与深度学习基本思想。关键不是记住网络结构名称，而是掌握链式法则如何高效计算梯度。

## 先用人话理解

神经网络可以看作一组可调节的“加工流水线”。输入数据进入第一层，被加工成一些中间特征；再进入下一层，组合成更复杂的特征；最后输出预测结果。

以识别图片为例，底层可能识别边缘、颜色和纹理，中间层组合成眼睛、轮廓、局部形状，高层再组合成“猫”“狗”等概念。虽然真实神经网络内部不一定完全按人类概念组织，但“逐层组合简单模式形成复杂模式”是理解深度网络的好入口。

## 一个小例子：判断是否好瓜

输入特征可能是色泽、根蒂、敲声、纹理。第一层神经元可能学到一些局部判断：

- 纹理清晰且根蒂蜷缩，给出较高分。
- 敲声沉闷且色泽异常，给出较低分。

第二层再组合这些局部判断，最后输出“好瓜概率”。训练的过程就是不断调整每条连接的权重，让最终预测越来越接近标签。

## 神经元模型

一个神经元接收输入 `x`，计算加权和：

```text
z = w^T x + b
```

再通过激活函数：

```text
a = sigma(z)
```

激活函数提供非线性。没有非线性时，多层线性变换仍等价于单层线性变换。

常见激活：

- sigmoid：输出在 `(0,1)`，易饱和。
- tanh：输出在 `(-1,1)`，零中心。
- ReLU：`max(0,z)`，计算简单，缓解梯度消失，但可能死亡。
- Leaky ReLU/GELU：改进 ReLU 的局限。

## 多层前馈网络

第 `l` 层：

```text
z^l = W^l a^{l-1} + b^l
a^l = sigma(z^l)
```

输入层为 `a^0 = x`，输出层根据任务选择 sigmoid、softmax 或线性输出。

## 反向传播推导

设损失为 `L`。定义第 `l` 层误差信号：

```text
delta^l = partial L / partial z^l
```

对输出层，`delta^L` 由损失函数和输出激活决定。

对隐藏层，利用链式法则：

```text
delta^l = ((W^{l+1})^T delta^{l+1}) ⊙ sigma'(z^l)
```

参数梯度：

```text
partial L / partial W^l = delta^l (a^{l-1})^T
partial L / partial b^l = delta^l
```

推导直觉：

1. `W^{l+1}` 描述第 `l` 层激活如何影响后一层。
2. 后一层误差 `delta^{l+1}` 反传回来，乘以连接权重。
3. 还要乘本层激活函数导数，因为 `a^l` 来自 `z^l`。
4. 权重梯度等于“本层误差”乘“前一层输入”。

## 类比：反向传播像追责链

如果最终预测错了，反向传播要回答：每个参数对这个错误负多少责任？

比如一家工厂生产出的产品不合格，不能只怪最后一道工序。要沿着流水线往回查：最后一步受哪一步影响，上一道工序又受更早的哪些设置影响。链式法则就是这套“责任分摊机制”。参数责任越大，更新幅度越大。

## 输出层与损失函数

二分类常用 sigmoid + 交叉熵：

```text
L = -[y log p + (1-y) log(1-p)]
```

多分类常用 softmax：

```text
p_k = exp(z_k) / sum_j exp(z_j)
```

配合交叉熵：

```text
L = - sum_k y_k log p_k
```

softmax + 交叉熵的梯度简化为：

```text
partial L / partial z_k = p_k - y_k
```

这是深度学习中最常用、也最重要的简化之一。

## 梯度消失与梯度爆炸

反向传播中梯度会连续乘多个权重矩阵和激活导数。如果这些因子范数长期小于 1，梯度会消失；长期大于 1，梯度会爆炸。

缓解方法：

- 使用 ReLU 系列激活。
- 合理初始化，如 Xavier、He 初始化。
- 批归一化或层归一化。
- 残差连接。
- 梯度裁剪。

## 正则化

常用方法：

- L2 权重衰减：限制参数过大。
- Dropout：训练时随机丢弃部分神经元，降低共适应。
- 早停：验证集性能不再提升时停止训练。
- 数据增强：增加训练分布覆盖。

Dropout 可理解为训练许多共享权重的子网络，预测时使用完整网络并缩放激活。

## 训练时常见现象

- 训练损失下降、验证损失也下降：模型正在学到有效规律。
- 训练损失下降、验证损失上升：可能开始过拟合。
- 两者都很高：模型太弱、特征不足、学习率不合适或训练不够。
- 损失剧烈震荡：学习率可能过大。
- 几乎不学习：学习率可能过小、初始化差或梯度消失。

## 深度学习基本思想

深层网络通过多层非线性组合学习层级表示。低层通常学习局部简单模式，高层组合成更抽象的表示。相比手工特征，深度学习倾向于端到端地共同学习特征和预测器。

## 易错点

- 多层线性网络仍然只是线性模型。
- BP 不是一种特殊模型，而是高效求梯度的算法。
- 训练误差下降不代表泛化性能上升。
- sigmoid 在深层网络中容易饱和，导致梯度变小。

## 长讲义扩展：神经网络到底在学什么

神经网络常被讲得很神秘，但从函数近似角度看，它就是一个由许多简单函数组合出来的复杂函数。每个神经元做一件简单事：加权求和，再经过非线性变换。大量神经元分层组合后，就能表达非常复杂的映射。

如果线性模型像一把直尺，神经网络就像许多可以弯折的拼接片。单个神经元能力有限，但多个神经元可以拼出复杂边界；浅层网络可以表达非线性，深层网络可以通过层级组合更高效地表达复杂结构。

## 为什么必须有激活函数

假设没有激活函数，每一层只是线性变换：

```text
a^1 = W^1 x
a^2 = W^2 a^1 = W^2 W^1 x
```

多个线性变换相乘仍然是一个线性变换。因此没有非线性时，网络再深也等价于一个线性模型。激活函数的作用就是打破线性限制，让网络能表达曲线、区域组合和复杂交互。

sigmoid 和 tanh 在早期很常用，但它们在输入很大或很小时导数接近 0，容易导致梯度消失。ReLU 在正区间导数为 1，计算简单，训练深层网络更容易，因此成为深度学习中的基础激活。

## 前向传播：从输入到预测

前向传播就是数据一层层往前走。以二分类网络为例：

```text
输入 x
第一层: z1 = W1 x + b1, a1 = ReLU(z1)
第二层: z2 = W2 a1 + b2, a2 = ReLU(z2)
输出层: z3 = W3 a2 + b3, p = sigmoid(z3)
```

每一层都把上一层的表示重新组合。底层更接近原始特征，高层更接近任务相关特征。训练前，权重随机初始化，网络输出几乎没有意义；训练后，权重被调整到能产生有效预测。

## 反向传播：高效计算所有参数的责任

神经网络参数很多，不能对每个参数都单独扰动试错。反向传播利用链式法则，从输出层开始，把误差信号一层层传回去。

如果最终预测偏大，输出层参数要调整；但输出层的输入来自上一层，上一层也要负责；上一层又来自更早层。反向传播就是沿着计算图，把“最终损失对每个中间量的影响”逐步算出来。

对某层权重 `W^l`，梯度形式：

```text
partial L / partial W^l = delta^l (a^{l-1})^T
```

这句话很有直觉：一个权重该怎么改，取决于两件事。第一，本层误差信号有多大；第二，前一层输入激活有多大。如果某条连接输入几乎为 0，它对输出影响小，梯度也小。

## softmax 加交叉熵为什么常见

多分类任务需要把多个类别分数转成概率。softmax 做的是指数归一化：

```text
p_k = exp(z_k) / sum_j exp(z_j)
```

它有两个效果：所有概率为正，并且总和为 1。若某个类别分数比其他类别高很多，它的概率会接近 1。

配合交叉熵后，梯度简化为：

```text
p_k - y_k
```

这个结果非常漂亮。若真实类别概率应为 1，但模型只预测 0.2，梯度为 -0.8，会强烈推动该类别分数上升；若某错误类别预测 0.6，梯度为 0.6，会推动它下降。

## 过拟合在神经网络中为什么常见

神经网络参数量大，表达能力强，能够拟合复杂规律，也能拟合噪声。尤其在数据少、网络大、训练时间长时，模型可能逐渐记住训练样本细节。

常见信号是：训练损失继续下降，但验证损失开始上升。这说明模型在训练集上变得更熟练，却对新样本更差。

正则化方法各有直觉：

- L2 权重衰减：限制权重过大，让函数更平滑。
- Dropout：训练时随机屏蔽部分神经元，防止神经元之间过度依赖。
- 早停：在验证集表现开始变差时停止，避免继续记忆噪声。
- 数据增强：让模型见到更多变化，提高对扰动的鲁棒性。

## 初始化和学习率为什么重要

神经网络训练是非凸优化，对初始化和学习率敏感。如果权重初始化太大，激活和梯度可能爆炸；太小，信号可能逐层衰减。Xavier 和 He 初始化就是为了让每层前向信号和反向梯度保持合适尺度。

学习率决定每次参数更新步子多大。学习率太大，可能在最优点附近来回跳甚至发散；学习率太小，训练非常慢，还可能停在不理想区域。实际训练常使用学习率衰减、Adam、动量等优化技巧。

## 神经网络和传统机器学习的关系

传统机器学习常依赖人工特征工程：人先设计特征，再用模型分类。神经网络倾向于端到端学习：从原始输入到最终输出，中间表示也由数据学习得到。

这不是说特征工程不重要，而是特征工程的一部分被网络内部表示学习替代了。在小数据、表格数据、强业务约束场景中，传统模型仍然很强；在图像、语音、自然语言等高维复杂数据中，深度学习优势明显。

## 学习本章的正确姿势

不要试图一开始就背各种网络结构。先掌握三件事：前向传播如何计算预测，损失函数如何衡量错误，反向传播如何计算梯度。只要这三件事清楚，卷积网络、循环网络、Transformer 都可以看作在计算图中加入特定结构。

神经网络不是魔法，它只是把可微分模块组合起来，然后用梯度下降调整参数。复杂之处在于规模、结构和训练技巧，而基础逻辑始终是“前向计算、反向求导、迭代更新”。

## 万字版补充：一个两层网络的计算过程

假设输入有两个特征：

```text
x = [x1, x2]
```

隐藏层有 3 个神经元，输出层 1 个神经元。前向传播可以写成：

```text
z1 = W1 x + b1
a1 = ReLU(z1)
z2 = W2 a1 + b2
p = sigmoid(z2)
```

第一层的每个神经元都像一个小探测器。它可能检测“特征1高且特征2低”这样的局部模式。输出层再把这些局部模式组合成最终概率。

训练时，如果真实标签为 1 而预测 `p=0.2`，损失很大。反向传播会先告诉输出层：“你给正类的分数太低了”；再告诉隐藏层：“哪些隐藏模式导致输出偏低”；最后更新所有相关权重。

## 为什么深度有用

理论上，一个足够宽的浅层网络也能近似复杂函数。但深层网络往往能更高效地表达层级结构。图像中，边缘组成纹理，纹理组成部件，部件组成物体；语言中，字符组成词，词组成短语，短语组成句义。

深度的作用是复用中间表示。底层学到的边缘可以服务于许多不同形状，高层再组合这些共享部件。若没有层级复用，模型可能需要更多参数才能表达同样复杂的函数。

## 激活函数的选择

sigmoid 输出范围固定，适合作为二分类输出层，但隐藏层中容易饱和。tanh 零中心，比 sigmoid 更适合隐藏层，但仍有饱和问题。ReLU 简单高效，但负半轴梯度为 0，可能出现死亡 ReLU。Leaky ReLU 给负半轴一个小斜率，缓解这个问题。GELU 和 Swish 在现代网络中也常见。

选择激活函数时，要考虑梯度传播、计算成本和任务经验。默认情况下，隐藏层用 ReLU 或其变体通常是稳妥起点。

## 损失函数要匹配任务

回归任务常用 MSE 或 MAE。MSE 对大误差敏感，适合希望强烈惩罚离谱预测的场景；MAE 更稳健，不容易被异常值主导。

二分类常用 sigmoid + binary cross entropy。多分类常用 softmax + cross entropy。多标签分类不是 softmax，而是每个标签一个 sigmoid，因为多个标签可以同时为真。

损失函数定义了模型努力优化的方向。损失选错，模型可能很认真地优化一个不符合任务目标的东西。

## 优化器的直觉

普通梯度下降每次沿负梯度方向更新参数。SGD 使用小批量样本估计梯度，计算更快，也带来一定随机性，有时有助于跳出差的区域。

动量方法像给更新方向加惯性。如果连续几步方向相似，就加速前进；如果方向震荡，就相互抵消。Adam 进一步为不同参数自适应调整学习率，实践中常作为默认优化器。

但 Adam 不是永远最好。有些任务中 SGD + momentum 泛化更好。优化器选择应结合验证集表现。

## Batch Normalization 的直觉

批归一化把中间激活调整到较稳定的分布，再学习缩放和平移参数。它能缓解训练过程中各层输入分布不断变化的问题，使网络更容易训练。

它还有轻微正则化效果，因为每个样本的归一化依赖同一批次其他样本。小 batch 时 BN 可能不稳定，此时 LayerNorm、GroupNorm 等方法更合适。

## 残差连接为什么重要

深层网络难训练的一个原因是梯度要穿过很多层。残差连接让网络学习：

```text
输出 = 输入 + 变化量
```

如果某些层暂时学不到有用变换，至少可以近似恒等映射，让信息直接通过。这大幅缓解深层网络退化问题，是 ResNet 成功的关键。

## 训练过程如何排查问题

如果训练损失不下降，先检查数据和标签是否对应、学习率是否合理、损失函数是否正确、输出层是否匹配任务。

如果训练损失下降但验证损失不降，可能过拟合。可以增加数据增强、正则化、Dropout、早停或减小模型。

如果训练不稳定，可能学习率过大、梯度爆炸、初始化不合适。可以降低学习率、使用梯度裁剪、检查输入标准化。

如果模型预测全是一个类别，可能类别不平衡、阈值不合适、损失实现错误或学习率异常。

## 神经网络的解释问题

神经网络可解释性较弱，但并非完全不可分析。常见方法包括：

- 查看特征重要性或梯度敏感性。
- 图像任务中使用 saliency map、Grad-CAM。
- 文本任务中分析 attention 或输入扰动。
- 用简单模型拟合局部行为。

这些方法能提供线索，但不应过度解读。深度模型的内部表示通常不是简单的人类概念。

## 本章自测问答

问：没有激活函数的多层网络为什么仍是线性模型？

答：多个线性变换的复合仍是线性变换，可以合并成一个矩阵。

问：反向传播是不是一种模型？

答：不是。反向传播是计算梯度的算法，可用于训练各种可微计算图。

问：Dropout 为什么有正则化效果？

答：它让网络不能过度依赖某些神经元，迫使表示更分散、更鲁棒。

问：学习率太大会怎样？

答：参数更新步子过大，可能越过最优区域，导致损失震荡或发散。

## 继续扩展：反向传播的局部计算视角

反向传播最重要的思想是局部梯度复用。一个复杂网络可以拆成许多小运算：加法、乘法、矩阵乘法、激活函数、损失函数。每个小运算只需要知道自己的输入、输出和上游梯度，就能计算下游梯度。

例如：

```text
z = wx + b
a = sigmoid(z)
L = loss(a, y)
```

反向传播从 `partial L / partial a` 开始，乘以 `partial a / partial z`，再乘以 `partial z / partial w`，得到 `partial L / partial w`。深度学习框架的自动求导就是把这种链式计算组织成计算图。

## 过参数化与泛化

现代神经网络常有远多于样本数量的参数，却仍能泛化。这和传统“参数越多越过拟合”的直觉有张力。原因可能包括：优化算法偏好某些简单解，网络结构带有归纳偏好，数据本身有强结构，正则化和数据增强限制了有效复杂度。

这提醒我们，参数数量不是唯一复杂度指标。卷积网络的局部连接和权重共享就是强归纳偏好；Transformer 的注意力结构也假设 token 间关系可以通过相似性动态建模。

## 常见网络结构概览

多层感知机适合一般向量输入，是最基础结构。

卷积神经网络适合图像和局部空间结构。卷积核在不同位置共享权重，体现“同一种局部模式可能出现在不同位置”的假设。

循环神经网络适合序列数据，但长距离依赖训练困难。LSTM、GRU 用门控缓解梯度问题。

Transformer 使用自注意力，让序列中任意位置直接交互，已成为自然语言处理和多模态任务的核心结构。

这些结构不同，但训练基础仍是前向传播、损失函数和反向传播。

## 数据增强的例子

图像任务中，可以随机裁剪、翻转、颜色扰动、旋转。文本任务中，可以同义改写、随机遮盖、回译。语音任务中，可以加噪声、变速、频谱遮挡。

数据增强的目标是告诉模型：这些变化不应改变标签。它把人类先验注入训练过程，是一种非常重要的正则化。

### 精讲扩展 5：神经网络 的优化算法、泛化能力 与工程化理解

学习 $topic 时，不能只把它当成一个孤立知识点来背诵，而要把它放到 $category 的完整问题链条里理解。一个知识点通常同时包含概念定义、适用边界、输入输出、运行过程、常见异常和工程取舍。真正掌握它，意味着看到一个具体场景时，能够判断它解决什么问题、依赖哪些前提、失败时会出现什么现象，以及应该用什么手段验证自己的判断。

从 $a 的角度看，最重要的是先建立清晰的对象模型。也就是明确系统里有哪些参与者、它们之间如何连接、数据或控制信号如何流动、哪些环节是同步的、哪些环节是异步的、哪些状态是临时状态、哪些状态需要长期保存。很多初学问题并不是公式不会、API 不熟，而是对象边界不清：把配置当成状态，把结果当成过程，把局部现象当成全局规律。写笔记时建议始终追问：这个概念的主体是谁，输入是什么，输出是什么，中间约束是什么，错误会在哪里暴露。

从 $b 的角度看，流程比单点知识更关键。一个成熟方案通常不是单个技巧，而是一组步骤：先确定目标，再拆分约束，然后选择工具，最后通过测试和复盘确认效果。比如在实际项目中，不能只问“怎么实现”，还要问“为什么要这样实现”“有没有更简单的替代方案”“边界条件是什么”“数据量、并发量、实时性、可靠性变化后还能不能工作”。这种流程意识能够避免把学习停留在教程层面，也能让后续排错有明确路线。

$topic 的 $c 往往决定它在真实项目中的稳定性。理论上可行的方案，到了工程环境中会受到数据质量、硬件条件、依赖版本、网络环境、团队协作、部署方式和维护成本影响。写代码或做设计时，应该把正常路径和异常路径同时考虑：正常情况下如何运行，输入为空怎么办，超时怎么办，重复执行怎么办，部分成功怎么办，版本升级后兼容性怎么办，日志和指标如何证明系统确实按预期工作。

进一步看 $d，它通常对应性能、可靠性或可维护性的核心矛盾。很多技术选择并没有绝对正确答案，只有是否适合当前约束。例如追求极致性能可能牺牲可读性，追求高度抽象可能增加调试成本，追求快速交付可能留下技术债，追求完全通用可能让简单场景变复杂。高质量笔记应该把这些取舍写出来，而不是只给一个“推荐方案”。推荐方案背后的条件越清楚，迁移到新场景时越不容易误用。

最后从 $e 的角度进行复盘，可以把知识从“看懂”推进到“会用”。建议为 $topic 建立三个层次的检查：第一层是概念检查，确认术语、流程和边界没有混淆；第二层是实践检查，确认能够独立完成一个最小案例；第三层是工程检查，确认这个案例在异常、规模、性能和维护方面经得起追问。每次学习完一个章节，都可以用这三层检查反向补齐笔记。
