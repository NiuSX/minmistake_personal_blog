## 学习目标

SVM 的核心是最大间隔、对偶优化和核方法。本章要理解：为什么最大间隔有助于泛化，为什么只由支持向量决定分类面，以及核函数如何实现非线性分类。

## 先用人话理解

支持向量机想做一件很直观的事：在两类样本之间画一条分界线，而且这条线要尽量“离两边都远”。如果分界线紧贴某一类样本，稍微有点噪声就可能分错；如果分界线在中间留出最大安全距离，模型通常更稳。

这个“安全距离”就是间隔。SVM 的核心信念是：不仅要分对训练样本，还要分得有余量。

## 一个小例子

想象地上有红球和蓝球，你要画一条直线把它们分开。可能有很多条线都能分开，但其中有些线贴着红球，有些线贴着蓝球。SVM 会选择离最近红球和最近蓝球都尽量远的那条线。

离边界最近、真正决定这条线位置的点，就是支持向量。其他离得很远的点，就算稍微移动，分界线也可能不变。

## 函数间隔与几何间隔

线性分类器：

```text
f(x) = w^T x + b
```

分类正确要求：

```text
y_i (w^T x_i + b) > 0
```

函数间隔：

```text
gamma_hat_i = y_i(w^T x_i + b)
```

几何间隔：

```text
gamma_i = y_i(w^T x_i + b) / ||w||
```

函数间隔会随 `w,b` 同比例缩放而变化，几何间隔才是真实距离。

## 硬间隔 SVM

可分数据下，希望最大化最小几何间隔。通过缩放令支持向量满足：

```text
y_i(w^T x_i + b) >= 1
```

此时间隔宽度为 `2/||w||`，最大化间隔等价于最小化 `||w||^2`：

```text
min_{w,b} 1/2 ||w||^2
s.t. y_i(w^T x_i + b) >= 1
```

类比：硬间隔就像要求路中间画隔离带，而且所有车都必须完全在自己车道内，不能压线。这在路况干净时可行，但如果有异常车辆或标注错误，就会过于苛刻。

## 拉格朗日对偶推导

构造拉格朗日函数：

```text
L(w,b,alpha) = 1/2 ||w||^2 + sum_i alpha_i [1 - y_i(w^T x_i + b)]
```

其中 `alpha_i >= 0`。对 `w,b` 求偏导并令零：

```text
partial L / partial w = w - sum_i alpha_i y_i x_i = 0
=> w = sum_i alpha_i y_i x_i

partial L / partial b = - sum_i alpha_i y_i = 0
=> sum_i alpha_i y_i = 0
```

代回得到对偶问题：

```text
max_alpha sum_i alpha_i - 1/2 sum_i sum_j alpha_i alpha_j y_i y_j x_i^T x_j
s.t. alpha_i >= 0, sum_i alpha_i y_i = 0
```

KKT 条件中的互补松弛：

```text
alpha_i [ y_i(w^T x_i + b) - 1 ] = 0
```

因此只有位于间隔边界上的样本可能有 `alpha_i > 0`，这些样本就是支持向量。

## 软间隔 SVM

现实数据常不可完全线性可分，引入松弛变量：

```text
min 1/2 ||w||^2 + C sum_i xi_i
s.t. y_i(w^T x_i + b) >= 1 - xi_i
xi_i >= 0
```

`C` 控制间隔和训练错误之间的权衡：

- `C` 大：更重视训练集正确，间隔可能变窄。
- `C` 小：容忍更多错误，间隔更宽。

软间隔等价于使用合页损失：

```text
max(0, 1 - y_i f(x_i))
```

类比：软间隔允许少数车临时压线，但会罚款。`C` 就是罚款力度。罚得太重，系统会为了照顾个别异常点而牺牲整体道路宽度；罚得太轻，又可能分类太随意。

## 核方法

若原空间线性不可分，可映射到高维特征空间：

```text
phi(x)
```

对偶问题中样本只以内积形式出现，因此可用核函数：

```text
K(x_i, x_j) = phi(x_i)^T phi(x_j)
```

常见核：

- 线性核：`K(x,z)=x^T z`
- 多项式核：`K(x,z)=(x^T z + c)^d`
- 高斯核/RBF：`K(x,z)=exp(-||x-z||^2/(2 sigma^2))`

核函数本质是合法的相似度函数，对应某个隐式特征空间中的内积。

## 类比：核函数像换观察角度

有些数据在平面上无法用直线分开，比如一圈点包围另一圈点。但如果把每个点加上“离中心多远”这个新特征，问题可能立刻变得线性可分。

核函数做的事就是：不用真的把所有高维特征算出来，也能得到高维空间里的内积效果。它像是偷偷换了一个更适合分类的观察角度。

## SMO 思想

SVM 对偶问题有约束 `sum_i alpha_i y_i = 0`。SMO 每次选择两个变量优化，其余固定，这样可以保持等式约束并得到解析更新。

## 易错点

- 支持向量不是所有分类正确的点，而是决定间隔边界或违反间隔的关键点。
- 核函数不是显式升维后再计算，而是直接计算隐式空间内积。
- `C` 越大不一定越好，过大可能过拟合。
- 硬间隔要求数据线性可分，现实中通常使用软间隔。

## 长讲义扩展：SVM 的核心是一种“稳”的思想

支持向量机不像一些模型那样只追求把训练样本分对。它还要求分类边界离两类样本都尽可能远。这个思想非常朴素：如果边界贴着样本，稍微有噪声就会分错；如果边界有宽阔间隔，新样本即使有轻微扰动也更可能仍在正确一侧。

因此，SVM 是一种把“正确”与“有把握地正确”区分开的模型。训练样本被分对只是最低要求，分得有余量才是目标。

## 从二维直线到高维超平面

二维中分类边界是一条直线：

```text
w1 x1 + w2 x2 + b = 0
```

三维中是一个平面，高维中称为超平面：

```text
w^T x + b = 0
```

`w` 决定边界方向，`b` 决定边界位置。样本代入 `w^T x + b` 后，符号表示它在边界哪一侧，绝对值与离边界远近相关。

SVM 通过约束：

```text
y_i(w^T x_i + b) >= 1
```

要求所有样本不仅分类正确，而且至少达到单位函数间隔。随后最小化 `||w||^2`，等价于最大化几何间隔。

## 为什么只有支持向量重要

训练完成后，很多样本离边界很远。它们对边界位置没有直接影响，因为边界稍微移动，它们仍然分类正确且间隔充足。真正卡住边界的是离边界最近的样本，也就是支持向量。

这像在两排柱子中间放一块木板。离木板很远的柱子碰不到它，只有最近的几根柱子决定木板最多能放在哪里。SVM 的稀疏性就来自这里：决策函数只依赖支持向量。

## 对偶问题的意义不只是推导

很多人学习 SVM 时会觉得对偶推导很抽象。它至少有三个实际意义。

第一，对偶问题让约束优化更易处理。原问题中有 `w` 和 `b`，对偶问题转为优化样本对应的 `alpha_i`。

第二，对偶形式揭示了支持向量。KKT 条件告诉我们，只有 `alpha_i > 0` 的样本会影响最终模型，而这些样本正是支持向量。

第三，对偶形式中样本只以内积 `x_i^T x_j` 出现。这为核函数打开了入口：只要把内积替换成核函数，就能在隐式高维空间分类。

## 软间隔解决现实数据

硬间隔要求训练数据完全线性可分，但现实数据常有噪声、异常点和重叠类别。若坚持硬间隔，模型可能被少数异常样本拖着走，边界变得很差。

软间隔引入松弛变量，允许少数样本违反间隔甚至被分错，但要付出代价。参数 `C` 控制代价大小。

如果 `C` 很大，模型会尽力让训练样本都满足要求，可能过拟合异常点。如果 `C` 很小，模型更重视大间隔，允许更多训练错误，可能欠拟合。调节 `C` 本质上是在训练误差和间隔大小之间折中。

## 合页损失的直觉

合页损失：

```text
max(0, 1 - y f(x))
```

它不只是惩罚分错样本，也惩罚“虽然分对但不够有把握”的样本。若 `y f(x) >= 1`，说明分类正确且间隔足够，损失为 0；若 `0 < y f(x) < 1`，说明分类正确但离边界太近，仍有损失；若 `y f(x) < 0`，说明分类错误，损失更大。

这体现了 SVM 的风格：不仅要对，还要留余量。

## 核函数的直觉与风险

核函数允许 SVM 处理非线性问题。RBF 核尤其灵活，它相当于把样本映射到无限维特征空间。灵活性强带来好处，也带来过拟合风险。

RBF 核中的 `sigma` 或 `gamma` 控制相似度衰减速度。若影响范围太小，每个样本只影响很近区域，边界会非常曲折，容易过拟合；若影响范围太大，模型又接近过于平滑，可能欠拟合。

因此核 SVM 的关键超参数通常包括 `C` 和核参数，需要通过验证集或交叉验证选择。

## SVM 与逻辑回归的比较

二者都可以学习线性分类边界，但损失函数不同。逻辑回归使用交叉熵，关注概率校准；SVM 使用合页损失，关注间隔。逻辑回归通常能输出较自然的概率，SVM 原始输出是距离边界的分数，不是概率。

在高维稀疏数据上，线性 SVM 和逻辑回归都很强。若任务更重视排序和边界，SVM 可能表现好；若需要概率解释和后续决策，逻辑回归更方便。

## 实践建议

使用 SVM 前应标准化特征，因为间隔和距离对尺度敏感。一个数值范围很大的特征会主导内积和距离。

样本量很大时，核 SVM 训练可能较慢，因为核矩阵规模随样本数平方增长。此时可优先考虑线性 SVM、随机特征近似或其他可扩展模型。

SVM 的学习重点不是记住复杂推导，而是掌握主线：最大间隔带来稳健性，对偶形式带来支持向量和核方法，软间隔处理噪声数据。

## 万字版补充：从间隔几何看 SVM

点到超平面 `w^T x + b = 0` 的距离是：

```text
|w^T x + b| / ||w||
```

如果样本标签为 `y_i ∈ {-1, +1}`，分类正确且距离越远，`y_i(w^T x_i + b)/||w||` 越大。SVM 最大化最小几何间隔，就是让离边界最近的样本也尽量远。

通过缩放 `w,b`，可以令最近样本满足：

```text
y_i(w^T x_i + b) = 1
```

两侧支持向量之间的距离为 `2/||w||`。最大化距离等价于最小化 `||w||`，为了求导方便写成最小化 `1/2 ||w||^2`。

## KKT 条件的直觉

KKT 条件中最关键的是互补松弛：

```text
alpha_i [y_i(w^T x_i + b) - 1] = 0
```

它表示两种情况：

- 如果样本不在间隔边界上，即 `y_i(w^T x_i+b)>1`，那么 `alpha_i=0`。
- 如果 `alpha_i>0`，样本必须在间隔边界上。

这就是支持向量的数学来源。非支持向量的 `alpha_i=0`，不会出现在最终 `w = sum_i alpha_i y_i x_i` 中。

## 软间隔中的三类样本

软间隔 SVM 中，样本可分为几类：

- `y f(x) > 1`：分类正确且间隔足够，无损失。
- `y f(x) = 1`：刚好在间隔边界上，是支持向量。
- `0 < y f(x) < 1`：分类正确但间隔不足，有损失。
- `y f(x) < 0`：分类错误，损失更大。

这个分类有助于理解合页损失。SVM 关心的不只是错分点，也关心那些太靠近边界的点。

## 核函数合法性

不是任意相似度函数都能作为核函数。合法核函数应对应某个特征空间中的内积，通常要求核矩阵半正定。Mercer 条件提供了理论基础。

实践中常用成熟核函数即可：线性核、多项式核、RBF 核、sigmoid 核等。自定义核时要谨慎，否则优化问题可能失去凸性或数值不稳定。

## RBF 核参数的影响

RBF 核：

```text
K(x,z)=exp(-gamma ||x-z||^2)
```

`gamma` 大时，相似度随距离快速衰减，每个样本影响范围小，边界可能非常复杂。`gamma` 小时，相似度衰减慢，模型更平滑。

调参时常在 `C` 和 `gamma` 的网格上交叉验证。高 `C` + 高 `gamma` 是过拟合高风险组合；低 `C` + 低 `gamma` 可能欠拟合。

## 多分类 SVM

SVM 原生是二分类模型。多分类常用 OvR 或 OvO。OvR 为每个类别训练一个“该类 vs 其他类”分类器，预测时选得分最高。OvO 为每对类别训练分类器，预测时投票。

OvO 分类器数量多，但每个分类器训练数据更少，常用于 SVM。类别很多时，计算成本要重点考虑。

## SVM 的优缺点

优点：

- 最大间隔思想清晰。
- 凸优化，线性和核 SVM 有较成熟理论。
- 高维小样本场景表现常较好。
- 通过核函数可处理非线性。

缺点：

- 核 SVM 大样本训练成本高。
- 概率输出不自然，需要额外校准。
- 对参数和特征尺度敏感。
- 核选择需要经验和验证。

## SVM 和神经网络的差异

SVM 的优化目标通常是凸的，解更稳定；神经网络非凸，表达能力更强，但训练更复杂。SVM 依赖核函数预设相似性，神经网络通过数据学习表示。

在深度学习兴起前，SVM 是许多高维任务的强方法。现在它仍适合中小规模、高维、需要稳健边界的任务。

## 本章自测问答

问：为什么最大间隔可能提升泛化？

答：间隔大意味着边界对样本扰动更不敏感，分类更有余量。

问：支持向量是什么？

答：决定分类边界的关键样本，通常在间隔边界上或违反间隔。

问：核函数解决什么问题？

答：在不显式计算高维映射的情况下，计算高维特征空间中的内积，从而实现非线性分类。

问：`C` 越大越好吗？

答：不是。`C` 太大可能过度拟合训练样本和异常点，间隔变窄。

## 继续扩展：SVM 调参案例

使用 RBF 核 SVM 时，通常调两个参数：`C` 和 `gamma`。

如果训练准确率和验证准确率都低，模型可能欠拟合。可以增大 `C`，或增大 `gamma`，让边界更灵活。

如果训练准确率很高但验证准确率低，模型可能过拟合。可以减小 `C`，或减小 `gamma`，让边界更平滑。

调参可以用网格搜索：

```text
C: 0.01, 0.1, 1, 10, 100
gamma: 0.001, 0.01, 0.1, 1, 10
```

配合交叉验证选择组合。特征标准化应放在交叉验证管道内部。

## SVM 输出分数如何理解

SVM 的决策函数 `f(x)` 表示样本到边界的有符号距离相关分数。分数为正预测正类，负预测负类，绝对值越大离边界越远。

但它不是概率。如果需要概率，可以使用 Platt scaling 等方法在验证集上校准。

## 异常点对 SVM 的影响

硬间隔 SVM 对异常点非常敏感，因为一个错标样本可能让数据不可分。软间隔通过松弛变量允许违反约束，但若 `C` 太大，异常点仍会强烈影响边界。

因此数据清洗和合理设置 `C` 很重要。面对噪声多的数据，过强追求训练正确率通常不是好策略。

## SVM 的面试高频问题

问：为什么 SVM 使用合页损失而不是 0/1 损失？

答：0/1 损失不可导且难优化，合页损失是凸替代，并且体现间隔思想。

问：为什么要转对偶？

答：对偶形式便于处理约束，揭示支持向量，并允许使用核函数。

问：支持向量数量多说明什么？

答：可能边界复杂、数据重叠多或参数设置导致许多样本参与决策，模型预测成本也会增加。

## 本章 checklist

使用 SVM 时检查：

- 特征是否标准化。
- 样本量是否适合核 SVM。
- 是否需要概率校准。
- `C` 和核参数是否通过验证选择。
- 支持向量数量是否过多。
- 类别不平衡时是否设置 class weight。

SVM 适合作为中小规模数据的强基线。若数据是高维稀疏文本，优先尝试线性 SVM；若样本量中等且边界非线性，可以尝试 RBF 核。若样本量非常大，核 SVM 可能计算成本过高。

## 继续补充：线性 SVM 在文本分类中的应用

文本向量通常维度极高，但非常稀疏。线性 SVM 在这种场景下表现很好，因为高维空间中类别往往更容易被线性分开。每个词或 n-gram 都是一个维度，分类边界可以组合大量词证据。

例如情感分类中，“优秀、喜欢、推荐”可能推动正类分数，“糟糕、失望、差评”推动负类分数。线性 SVM 不输出概率，但能给出稳定的间隔分类。

## 支持向量数量和预测成本

核 SVM 的预测函数依赖支持向量：

```text
f(x)=sum_i alpha_i y_i K(x_i,x)+b
```

如果支持向量很多，预测每个新样本都要和大量支持向量计算核函数，成本会高。支持向量数量也是观察模型复杂度的一个信号。

线性 SVM 可以把权重合并成 `w`，预测只需计算 `w^T x`，因此更适合大规模部署。

## 类别不平衡下的 SVM

类别不平衡时，可以为不同类别设置不同惩罚系数。少数类错分代价更高，模型会更重视少数类。

但提高少数类权重会影响边界位置，通常提高召回率、可能降低精准率。仍需通过验证集选择权重和阈值。

第二个误区是只关注工具，不关注模型。工具会变化，模型更稳定。无论工具名称如何变化，底层仍然要解决输入建模、状态管理、资源调度、错误恢复、性能约束和质量验证这些问题。学习 $topic 时，应该把工具用法和底层模型分开记录：工具命令解决“怎么做”，底层模型解释“为什么这样做”。

第三个误区是没有验证意识。很多笔记写得很完整，但没有说明如何确认自己做对了。对于 $category 相关主题，验证至少应包含最小样例、边界样例、异常样例和性能样例。最小样例证明流程跑通，边界样例证明理解完整，异常样例证明系统可恢复，性能样例证明方案在目标规模下仍然可用。

第四个误区是忽略可维护性。短期学习时，能跑通就容易产生掌握的错觉；长期使用时，命名、分层、注释、测试、日志、版本管理和文档才会决定知识能否转化为稳定能力。扩充 $topic 笔记时，应把“如何写得清楚、如何排查、如何交接、如何复盘”也纳入内容。

#### 学习与实践建议

建议围绕 $topic 做一个小型闭环练习：先用自己的话解释概念，再画出流程，再实现一个最小案例，然后主动制造一个错误并排查，最后写下复盘。这个过程看起来比直接读资料慢，但能显著提高迁移能力。很多人学完后不会用，根本原因是缺少“从概念到问题再到验证”的闭环。

### 精讲扩展 5：支持向量机 的优化算法、泛化能力 与工程化理解

学习 $topic 时，不能只把它当成一个孤立知识点来背诵，而要把它放到 $category 的完整问题链条里理解。一个知识点通常同时包含概念定义、适用边界、输入输出、运行过程、常见异常和工程取舍。真正掌握它，意味着看到一个具体场景时，能够判断它解决什么问题、依赖哪些前提、失败时会出现什么现象，以及应该用什么手段验证自己的判断。

从 $a 的角度看，最重要的是先建立清晰的对象模型。也就是明确系统里有哪些参与者、它们之间如何连接、数据或控制信号如何流动、哪些环节是同步的、哪些环节是异步的、哪些状态是临时状态、哪些状态需要长期保存。很多初学问题并不是公式不会、API 不熟，而是对象边界不清：把配置当成状态，把结果当成过程，把局部现象当成全局规律。写笔记时建议始终追问：这个概念的主体是谁，输入是什么，输出是什么，中间约束是什么，错误会在哪里暴露。

从 $b 的角度看，流程比单点知识更关键。一个成熟方案通常不是单个技巧，而是一组步骤：先确定目标，再拆分约束，然后选择工具，最后通过测试和复盘确认效果。比如在实际项目中，不能只问“怎么实现”，还要问“为什么要这样实现”“有没有更简单的替代方案”“边界条件是什么”“数据量、并发量、实时性、可靠性变化后还能不能工作”。这种流程意识能够避免把学习停留在教程层面，也能让后续排错有明确路线。

$topic 的 $c 往往决定它在真实项目中的稳定性。理论上可行的方案，到了工程环境中会受到数据质量、硬件条件、依赖版本、网络环境、团队协作、部署方式和维护成本影响。写代码或做设计时，应该把正常路径和异常路径同时考虑：正常情况下如何运行，输入为空怎么办，超时怎么办，重复执行怎么办，部分成功怎么办，版本升级后兼容性怎么办，日志和指标如何证明系统确实按预期工作。

进一步看 $d，它通常对应性能、可靠性或可维护性的核心矛盾。很多技术选择并没有绝对正确答案，只有是否适合当前约束。例如追求极致性能可能牺牲可读性，追求高度抽象可能增加调试成本，追求快速交付可能留下技术债，追求完全通用可能让简单场景变复杂。高质量笔记应该把这些取舍写出来，而不是只给一个“推荐方案”。推荐方案背后的条件越清楚，迁移到新场景时越不容易误用。

最后从 $e 的角度进行复盘，可以把知识从“看懂”推进到“会用”。建议为 $topic 建立三个层次的检查：第一层是概念检查，确认术语、流程和边界没有混淆；第二层是实践检查，确认能够独立完成一个最小案例；第三层是工程检查，确认这个案例在异常、规模、性能和维护方面经得起追问。每次学习完一个章节，都可以用这三层检查反向补齐笔记。

#### 典型场景拆解

在真实场景中，$topic 通常会经历“需求出现、方案选择、实现落地、问题暴露、持续优化”几个阶段。需求出现时，要先判断这个需求属于基础能力、性能优化、体验改进、可靠性建设还是长期架构演进。不同类型的需求对方案的评价标准不同：基础能力看正确性，性能优化看指标，体验改进看路径是否顺滑，可靠性建设看故障时能否降级和恢复，架构演进看未来变化是否容易吸收。

方案选择阶段，最容易犯的错误是直接套用熟悉工具。更稳妥的方式是列出约束：数据规模、时延要求、资源预算、团队熟悉度、运维能力、安全要求、可测试性和长期维护成本。只有把约束列清楚，才能解释为什么选择当前方案。否则方案看似高级，实际可能只是增加了复杂度。

实现落地阶段，要把 $a 和 $b 拆成可验证的小步骤。每一步都应该有明确的输入、输出和检查方式。对学习笔记而言，这意味着不能只有大段概念，还应该补充流程图式的文字描述、伪代码、命令示例、参数解释、错误现象和排查路径。这样以后复习时，笔记不仅能帮助理解，也能直接指导实践。

问题暴露阶段，要优先区分“理解错误、实现错误、环境错误、数据错误、依赖错误、边界条件错误”。很多复杂问题之所以难排，是因为一开始就把问题归因到错误层级。例如把配置问题当成算法问题，把权限问题当成代码问题，把数据分布变化当成模型失效，把硬件噪声当成软件逻辑错误。好的排查顺序应该从可观测事实开始，而不是从猜测开始。

持续优化阶段，不应只追求把当前问题压下去，还要沉淀成规则。比如记录触发条件、影响范围、定位方法、最终修复、预防措施和可监控指标。这样下一次出现类似问题时，团队可以复用经验，而不是重新从零排查。
