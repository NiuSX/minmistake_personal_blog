# 第 7 章 贝叶斯分类器

## 学习目标

本章从概率角度理解分类。重点是贝叶斯决策论、极大似然估计、朴素贝叶斯、半朴素贝叶斯和贝叶斯网络的基本思想。

## 先用人话理解

贝叶斯分类器的想法是：看到证据以后，重新判断每个类别的可能性。它不像线性模型那样直接画边界，而是问：“如果这是某一类，它出现这些特征的概率有多大？”

例如医生诊断疾病时，会结合先验概率和症状证据。某种病本来很罕见，即使某个症状有点像，也不能轻易判断；某种病很常见，多个症状又吻合，后验概率就会升高。

## 一个小例子：垃圾邮件

判断邮件是否垃圾邮件，可以看几个词：

- “中奖”
- “免费”
- “发票”
- “会议”

朴素贝叶斯会估计：

```text
P(垃圾邮件) * P(中奖|垃圾邮件) * P(免费|垃圾邮件) * ...
P(正常邮件) * P(中奖|正常邮件) * P(免费|正常邮件) * ...
```

哪个结果更大，就预测哪个类别。虽然词之间并不真的独立，但这个近似在文本分类中经常很好用。

## 贝叶斯决策论

给定样本 `x`，若预测为类别 `c_i` 的损失为 `lambda_{ij}`，表示真实类别为 `c_j` 时预测为 `c_i` 的代价，则条件风险：

```text
R(c_i | x) = sum_j lambda_{ij} P(c_j | x)
```

最优决策选择条件风险最小的类别：

```text
h*(x) = argmin_i R(c_i | x)
```

若 0/1 损失下，正确分类损失为 0，错误为 1，则最优规则退化为：

```text
h*(x) = argmax_c P(c | x)
```

## 贝叶斯公式

```text
P(c|x) = P(c) P(x|c) / P(x)
```

对分类而言，`P(x)` 对所有类别相同，所以：

```text
h(x) = argmax_c P(c) P(x|c)
```

生成式分类器通常估计先验 `P(c)` 和类条件概率 `P(x|c)`。

## 类比：先验和证据

贝叶斯公式可以理解为：

```text
最终相信程度 = 原本相信程度 * 证据支持程度 / 证据常见程度
```

如果一个证据在所有类别中都很常见，它区分能力就弱；如果某个证据在某类中特别常见、在其他类中罕见，它对分类就很有帮助。

## 极大似然估计

假设样本独立同分布，参数为 `theta`，似然：

```text
L(theta) = Π_i P(x_i | theta)
```

取对数：

```text
ell(theta) = sum_i log P(x_i | theta)
```

极大似然选择使观测数据出现概率最大的参数。

离散类别先验的 MLE：

```text
P(c) = |D_c| / |D|
```

若连续属性服从高斯分布，则估计每类下该属性的均值和方差。

## 朴素贝叶斯

朴素贝叶斯假设给定类别后属性条件独立：

```text
P(x|c) = Π_j P(x_j | c)
```

分类规则：

```text
h(x) = argmax_c P(c) Π_j P(x_j | c)
```

为了避免概率连乘下溢，实际常用对数：

```text
h(x) = argmax_c [ log P(c) + sum_j log P(x_j | c) ]
```

## 为什么“朴素”还能好用

朴素贝叶斯的独立性假设通常不严格成立。例如“免费”和“中奖”在垃圾邮件中显然相关。但分类不一定需要精确估计真实概率，只要不同类别的相对得分排序大致正确，就能做出不错的判断。

这也是机器学习中的常见现象：模型假设不必完全真实，只要偏差适合任务，仍可能有效。

## 拉普拉斯修正

若某类别下某属性值从未出现，估计概率为 0，会导致整个连乘为 0。拉普拉斯平滑：

```text
P(c) = (|D_c| + 1) / (|D| + N)
P(x_j=a | c) = (count(x_j=a, c) + 1) / (|D_c| + N_j)
```

其中 `N` 为类别数，`N_j` 为属性 `j` 的可能取值数。

## 半朴素贝叶斯

朴素贝叶斯的条件独立假设过强。半朴素贝叶斯允许每个属性依赖类别和另一个属性：

```text
P(x|c) = Π_j P(x_j | c, parent(x_j))
```

常见方法：

- SPODE：所有属性依赖同一个超父属性。
- TAN：在类别条件下，属性之间形成树结构。

## 贝叶斯网络

贝叶斯网络用有向无环图表达变量依赖。联合分布可分解为：

```text
P(x_1, ..., x_n) = Π_i P(x_i | Parents(x_i))
```

图结构编码条件独立关系，使高维联合分布可由局部条件概率组合表示。

## 易错点

- 朴素贝叶斯的属性独立假设通常不真实，但分类效果仍可能好。
- 极大似然只考虑数据似然，最大后验还会加入先验。
- 拉普拉斯修正不是提高训练拟合，而是避免零概率并改善泛化。
- 生成式模型能生成或评估样本概率，判别式模型通常直接学习边界或后验。

## 练习

1. 从 0/1 损失推导最小风险分类等价于最大后验分类。
2. 手算朴素贝叶斯分类。
3. 推导离散分布参数的极大似然估计。
4. 解释为什么对数概率不会改变分类结果。

## 长讲义扩展：贝叶斯方法是一种“证据更新”框架

贝叶斯思想的核心是：我们原本对某件事有一个相信程度，看到新证据后，应当更新这个相信程度。机器学习中的贝叶斯分类器把这个思想用于分类：先有类别先验，再根据样本特征计算每个类别的后验概率。

例如看到一封邮件前，你可能认为它是垃圾邮件的概率是 20%。如果邮件中出现“中奖”“免费领取”“点击链接”等词，垃圾邮件的后验概率会上升。如果出现“会议纪要”“项目排期”等词，正常邮件的后验概率可能上升。

## 生成式分类的思路

贝叶斯分类器通常是生成式的。它试图回答：如果样本属于某一类，那么生成这些特征的概率有多大？

分类时比较：

```text
P(c) P(x|c)
```

其中 `P(c)` 是类别本身有多常见，`P(x|c)` 是该类别下出现当前特征的可能性。一个类别即使先验高，如果特征非常不像它，后验也可能低；一个类别即使先验低，如果证据强烈支持它，后验也可能高。

## 朴素贝叶斯的手算示例

假设训练集中有 10 封邮件，6 封正常、4 封垃圾。统计两个词：

```text
P(正常) = 0.6
P(垃圾) = 0.4
P(免费|正常) = 0.1
P(中奖|正常) = 0.05
P(免费|垃圾) = 0.8
P(中奖|垃圾) = 0.7
```

现在一封邮件包含“免费”和“中奖”。朴素贝叶斯比较：

```text
正常得分 = 0.6 * 0.1 * 0.05 = 0.003
垃圾得分 = 0.4 * 0.8 * 0.7 = 0.224
```

垃圾得分更高，因此预测垃圾邮件。

这个例子忽略了词之间的相关性，但计算简单、可解释。在高维文本分类中，这种简单性反而是优势。

## 条件独立假设为什么大胆

朴素贝叶斯假设：给定类别后，各个特征相互独立。这个假设在现实中经常不成立。例如垃圾邮件中的“免费”和“中奖”显然相关，医学症状之间也常互相影响。

但分类任务不一定要求概率估计完全准确。只要正确类别的相对得分通常高于其他类别，分类就能成功。朴素贝叶斯牺牲了部分真实性，换来极低的估计难度。若没有独立假设，高维联合概率需要估计的参数会爆炸。

## 拉普拉斯平滑的必要性

若某个词在训练集中从未出现在垃圾邮件里，直接估计会得到：

```text
P(词|垃圾) = 0
```

那么只要测试邮件包含这个词，垃圾类别总得分就变成 0。这个结论过于武断，因为训练集没见过不代表真实概率为 0。拉普拉斯平滑相当于给每种可能取值一个很小的默认计数，避免零概率。

平滑体现了一个朴素但重要的思想：有限样本统计不应过度自信。

## 极大似然与最大后验

极大似然估计只看数据：

```text
选择让观测数据最可能出现的参数
```

最大后验估计还看先验：

```text
选择在观测数据之后最可信的参数
```

如果数据很多，先验影响通常变小；如果数据很少，先验能提供稳定性。例如抛硬币只抛 2 次都正面，极大似然会估计正面概率为 1，但加入合理先验后，估计会更保守。

## 贝叶斯决策论和代价敏感分类

最大后验分类默认错误代价相同。但现实中错误代价可能不同。医学筛查中，把病人判为健康的代价可能远高于把健康人判为疑似病人。此时应该最小化条件风险，而不是简单选最大后验类别。

条件风险：

```text
R(c_i|x) = sum_j lambda_ij P(c_j|x)
```

这提醒我们：概率只是中间结果，最终决策还要结合代价。

## 贝叶斯方法的优缺点

优点：

- 概率解释自然。
- 可处理不确定性。
- 训练和预测通常高效。
- 小数据时可结合先验。

缺点：

- 分布假设可能不准确。
- 朴素独立假设过强。
- 连续特征分布选择会影响效果。
- 概率校准不一定可靠。

## 学习本章的主线

贝叶斯分类器不是一组孤立公式，而是一条清晰链路：先验表示类别原本有多可能，似然表示证据对类别的支持，后验表示看到证据后的相信程度，决策规则再根据后验和损失选择行动。

只要抓住“先验 + 证据 -> 后验 -> 决策”这条线，本章大多数公式都能自然理解。

## 万字版补充：从医学诊断理解贝叶斯

假设某种疾病在人群中发病率为 1%，某检测对患者阳性率为 99%，对健康人误报率为 5%。如果一个人检测阳性，他真的患病概率是多少？

直觉上很多人会回答 99%，但贝叶斯公式告诉我们不是。设 10000 人中约 100 人患病，9900 人健康。检测阳性人数：

```text
患者阳性: 100 * 0.99 = 99
健康误阳性: 9900 * 0.05 = 495
```

阳性总人数 594，其中真正患病 99，后验概率约为 16.7%。这说明先验非常重要。罕见病即使检测准确率高，阳性结果也可能包含大量误报。

这个例子能帮助理解：后验不是只看证据强度，还要结合事件本身有多常见。

## 连续特征的朴素贝叶斯

离散特征可以计数估计概率。连续特征常假设在每个类别下服从高斯分布：

```text
P(x_j|c) = N(x_j | mu_cj, sigma_cj^2)
```

训练时估计每个类别下每个特征的均值和方差。预测时，把测试样本每个特征代入高斯密度函数，得到类条件概率。

这种方法简单，但高斯假设不一定成立。若连续特征分布明显偏斜，可以考虑变换、分箱或非参数估计。

## 文本分类中的多项式朴素贝叶斯

文本分类常用词袋表示。多项式朴素贝叶斯关注词出现次数：

```text
P(文档|类别) ∝ Π_词 P(词|类别)^{词频}
```

如果“退款”“发票”“合同”等词在商务邮件中更常见，模型会把它们作为证据。如果“中奖”“免费”“链接”等词在垃圾邮件中更常见，垃圾邮件得分会上升。

文本特征维度高、稀疏，而朴素贝叶斯参数估计简单，所以它曾长期作为文本分类强基线。

## 对数空间计算

朴素贝叶斯会连乘很多概率，每个概率都小于 1，乘积可能非常接近 0，导致数值下溢。因此实际计算用对数：

```text
log P(c) + sum_j log P(x_j|c)
```

对数函数单调递增，不改变大小比较。乘法变加法，也更稳定。

## 平滑参数不一定只能加 1

拉普拉斯平滑是加 1，也可以使用加 `alpha` 的平滑：

```text
(count + alpha) / (total + alpha * K)
```

`alpha` 越大，估计越接近均匀分布；`alpha` 越小，越相信训练计数。这个参数体现了先验强度。实际可通过验证集选择。

## 朴素贝叶斯与逻辑回归比较

朴素贝叶斯是生成式模型，估计 `P(x|y)` 和 `P(y)`；逻辑回归是判别式模型，直接估计 `P(y|x)`。

生成式模型在小数据时可能更快达到不错效果，因为它引入了较强假设。判别式模型在数据充足时通常能学到更灵活的边界。

文本分类中，朴素贝叶斯训练快、解释直观；逻辑回归通常精度更高、概率可通过正则和校准改善。

## 半朴素贝叶斯的意义

完全独立假设太强，完全联合建模又太难。半朴素贝叶斯位于中间：允许每个属性依赖一个或少数其他属性。

TAN 用树结构表示属性依赖，既增加表达能力，又避免参数爆炸。它体现了建模中的常见折中：比朴素模型更真实，比完整模型更可估计。

## 贝叶斯网络与朴素贝叶斯

朴素贝叶斯可以看成一个特殊贝叶斯网络：类别节点指向所有特征节点，特征之间没有边。

```text
类别 -> 特征1
类别 -> 特征2
类别 -> 特征3
```

如果加入特征之间的依赖边，就得到更一般的贝叶斯网络或半朴素结构。

## 本章自测问答

问：朴素贝叶斯的“朴素”指什么？

答：指给定类别后，各特征条件独立。

问：为什么需要平滑？

答：避免未出现事件概率被估计为 0，从而让整个类别得分变成 0。

问：贝叶斯分类器是否一定概率准确？

答：不一定。若模型假设错误，概率估计可能不准，但分类排序仍可能有效。

问：先验什么时候影响大？

答：数据少、证据弱或类别极不平衡时，先验影响更大。

## 继续扩展：朴素贝叶斯的特征工程

在文本任务中，是否使用停用词、词频还是二值出现、是否使用 n-gram，都会影响朴素贝叶斯。

例如情感分类中，“不 好”两个词分开看可能误导模型，使用 bigram “不好”能捕捉更准确语义。垃圾邮件中，某些短语比单个词更有辨别力。

朴素贝叶斯简单，但不代表不需要特征工程。它对特征表示非常敏感。

## 类别先验的调整

训练集类别比例不一定等于真实部署比例。例如为了训练方便，可能对少数类过采样，使训练集中正负类各半。但真实线上正类只有 1%。此时直接使用训练先验会导致概率偏移。

可以根据真实业务先验调整 `P(c)`，或做概率校准。分类模型的先验应与部署环境一致。

## 贝叶斯方法和不确定性

贝叶斯思想强调不确定性表达。参数不是固定未知常数，而可以看作随机变量。完整贝叶斯学习会计算参数后验分布，再对预测进行积分。

虽然朴素贝叶斯通常使用点估计，但理解完整贝叶斯观点有助于理解先验、后验、平滑和不确定性。

## 常见错误

错误一：把 `P(x|c)` 和 `P(c|x)` 混淆。前者是类别给定时特征出现概率，后者是看到特征后类别概率。

错误二：认为朴素贝叶斯假设特征完全独立。更准确地说，是给定类别后条件独立。

错误三：忘记使用对数概率，导致数值下溢。

错误四：不做平滑，导致零概率。

## 本章 checklist

使用朴素贝叶斯时检查：

- 特征是离散、计数还是连续。
- 是否选择合适的事件模型。
- 是否做平滑。
- 是否用对数空间计算。
- 类别先验是否符合部署分布。
- 特征之间强相关是否会造成重复计证据。

## 深入补充：朴素贝叶斯为什么适合高维稀疏数据

高维稀疏数据中，每个样本只激活少量特征。文本就是典型例子：词表可能有几十万词，但一篇文章只包含几百个词。若要估计完整联合分布，参数数量不可接受；朴素贝叶斯把联合概率拆成单词条件概率，训练就变得可行。

这种拆分牺牲了词序和词间依赖，但换来了极高的数据效率。很多词只要在某类中出现明显更多，就能成为有用证据。比如科技文章中的“芯片、算法、模型、训练”，财经文章中的“利率、股票、利润、财报”，体育文章中的“比赛、进球、球队、教练”。

朴素贝叶斯不会真正理解文章，但它能快速积累统计证据。这种“简单但有效”的特点，是很多传统机器学习方法的共同价值。

## 伯努利、多项式和高斯朴素贝叶斯

伯努利朴素贝叶斯关注特征是否出现，适合二值特征。例如某个词出现或不出现。

多项式朴素贝叶斯关注出现次数，适合词频、计数数据。文本分类最常用这一类。

高斯朴素贝叶斯假设连续特征在每个类别下服从高斯分布，适合数值型表格数据的快速基线。

选择哪种模型，取决于特征生成方式。不要把所有特征都硬塞进同一种事件模型。

## 概率校准与朴素贝叶斯

朴素贝叶斯由于独立性假设，常常会输出过于极端的概率。多个相关特征会被当成独立证据重复计算，导致后验过度自信。

例如“中奖”和“领取奖金”高度相关，模型却可能把它们当成两条独立强证据，使垃圾邮件概率接近 1。分类结果可能没问题，但概率数值不一定可信。

若后续要根据概率做精细决策，如风险定价、医学分层，就应做概率校准或选择更适合概率估计的模型。

## MAP 决策与拒识机制

最大后验分类总会选一个类别，即使模型很不确定。但实际系统可以设置拒识机制：若最高后验概率低于阈值，就交给人工或输出“不确定”。

这在医疗、内容审核、金融风控等高风险任务中很重要。模型知道自己不确定，比强行给出错误答案更安全。

拒识机制本质上把分类问题扩展为决策问题：不仅要考虑哪个类别概率最大，还要考虑自动决策和人工复核的成本。

## 万字级补充：本章扩展练习

1. 用垃圾邮件小数据集手算朴素贝叶斯分类。
2. 构造一个零概率问题，并用拉普拉斯平滑修正。
3. 对医学检测例子，改变疾病先验，观察后验如何变化。
4. 比较朴素贝叶斯和逻辑回归在文本分类中的优缺点。
5. 说明为什么相关特征会让朴素贝叶斯概率过度自信。

## 万字级补充：应用建议

朴素贝叶斯适合作为快速基线，尤其适合文本、离散特征和小数据场景。若需要高精度或良好概率校准，可以与逻辑回归、SVM、树模型比较。若只是需要一个可解释、训练快、部署简单的模型，它仍然很有价值。

## 继续补充：贝叶斯分类器中的先验设计

先验可以来自历史频率，也可以来自领域知识。垃圾邮件比例、疾病发病率、欺诈交易比例都可以作为类别先验。但训练集往往经过采样，类别比例未必等于真实先验。

例如欺诈检测中，为了训练方便，可能把欺诈样本过采样到 50%。如果直接把这个比例当作先验，模型会严重高估欺诈概率。部署前应根据真实业务比例调整先验或校准概率。

## 朴素贝叶斯和特征重复计数

如果两个特征表达同一信息，朴素贝叶斯会把它们当成两条独立证据。例如“含有链接”和“URL 数量大于 0”几乎重复。若二者都强烈指向垃圾邮件，模型会重复计算证据，使后验过度偏向垃圾类别。

因此使用朴素贝叶斯时，应避免大量重复特征，或接受它主要用于分类而非精确概率估计。

## 贝叶斯思维在机器学习中的延伸

贝叶斯思想不只用于朴素贝叶斯分类器。贝叶斯线性回归、贝叶斯神经网络、高斯过程、主题模型、概率图模型都建立在类似框架上：用先验表达已有相信程度，用似然表达数据证据，用后验表达更新后的不确定性。

这种思维特别适合小数据、高风险和需要不确定性估计的场景。

## 本章总结

贝叶斯分类器的核心是概率推理。先验告诉我们类别原本多常见，似然告诉我们证据多支持该类别，后验告诉我们看到证据后该如何更新判断。朴素贝叶斯用强独立假设换取简单高效，是理解概率机器学习的很好入口。

## 讲义级补充：如何真正学懂《第 7 章 贝叶斯分类器》

### 1. 这一讲要解决什么问题

这类内容的核心不是公式本身，而是“数据如何变成可学习的表示，模型如何产生预测，误差如何反馈改进”。学习时要把数据、假设、损失、优化、评估和部署放在一起理解。

学习本讲时，可以用三个问题检查自己是否真的理解：

1. 它解决的真实问题是什么？
2. 如果没有它，系统会出现什么具体麻烦？
3. 在真实项目中，应该用什么现象或指标判断它做得好不好？

### 2. 核心知识拆解

可以把本讲拆成几块来学：

- 数据：样本、标签、分布、增强、清洗和划分。
- 模型：假设空间、结构、参数和表达能力。
- 训练：损失函数、优化器、正则化、学习率和收敛。
- 评估：指标、误差分析、泛化、鲁棒性和部署表现。

拆解的好处是防止“整章都懂一点，但哪块都说不清”。复习时可以逐块追问：它的输入是什么、输出是什么、依赖什么、失败时有什么表现。

### 3. 通俗类比

可以把模型类比成一个需要反复练习的学生：数据是练习题，标签是参考答案，损失函数是扣分规则，优化器是改错方法，验证集像模拟考试，测试集才像正式考试。只在训练集上表现好，就像只会背做过的题。

类比不是严格定义，但能帮助初学者先建立直觉。真正使用时，还要回到术语、公式、接口、数据结构、时序图或工程规范上，把“感觉理解”变成“可验证理解”。

### 4. 具体例子

学习《第 7 章 贝叶斯分类器》时，可以准备一个很小的数据集，先手工看 20 个样本和标签，再训练一个基线模型，最后把预测错误的样本逐张分析。很多模型问题不是公式错了，而是数据、标注或评价方式一开始就没有定义清楚。

讲义级学习不能只停留在“概念解释”。至少要有一个能跑、能算、能画或能检查的例子。例子越小，越容易看清关键机制；等机制清楚后，再逐步扩展到复杂项目。

### 5. 学习路径

- 先明确任务类型：分类、回归、检测、分割、生成、序列建模或决策。
- 再梳理数据、特征、模型、损失函数、优化方法和评价指标。
- 最后通过误差分析定位问题来自数据、标注、模型容量、训练策略还是部署环境。

建议每学完一小节都做一次“复述练习”：不用看笔记，用自己的话讲清楚概念、输入、输出、关键步骤和常见错误。如果讲不清，通常说明还没有真正掌握。

### 6. 课堂讲解框架

可以按下面顺序讲解或复习本主题：

1. 背景：先讲这个知识为什么出现，它试图降低什么成本、解决什么风险或提升什么能力。
2. 基本概念：给出核心名词的准确定义，说明它们之间的关系。
3. 工作流程：按时间顺序描述一次完整过程，必要时画出流程图、状态机或数据流图。
4. 关键细节：解释最容易误解的机制，例如边界条件、异常处理、性能限制、资源生命周期或安全约束。
5. 实战例子：用一个足够小但完整的例子，把概念落到命令、代码、图纸、配置、数据或操作步骤上。

### 8. 常见误区

- 只追求模型结构新，不检查数据质量和标注一致性。
- 只看平均指标，不看错误样本、长尾类别和业务代价。

遇到这些问题时，不要急着背更多资料。更有效的办法是回到一个最小例子，把输入、状态变化、输出和验证方式重新走一遍。
