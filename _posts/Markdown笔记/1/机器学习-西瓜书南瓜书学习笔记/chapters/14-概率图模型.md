### 一句话先懂

概率图模型用图表示随机变量依赖关系，把复杂联合分布拆成可理解、可推断的局部结构。

### 通俗类比

可以把机器学习想成从错题本里总结解题方法：训练集是做过的题，模型是假设的解题套路，损失函数是扣分规则，优化是反复订正，验证集像模拟考，测试集像正式考试，泛化能力就是换一套题还能不能做对。复杂模型不是天生更聪明，它只是可选择的解题套路更多，也更容易把错题本里的偶然细节背下来。

类比只能帮助入门。回到技术上，要把每个概念对应到精确定义：样本、特征、标签、假设空间、目标函数、训练误差、泛化误差、评价指标、正则化和数据分布。只有这些对象清楚，公式才不是符号堆，实验结果才有解释意义。

1. **问题定义**：先确定任务是分类、回归、聚类、降维、序列决策还是概率推断。
2. **数据假设**：检查样本来源、标签噪声、特征含义、训练测试分布和数据泄漏。

## 学习目标

概率图模型用图结构表达变量之间的依赖关系。重点是贝叶斯网、马尔可夫网、条件随机场、推断和学习。

## 先用人话理解

概率图模型解决的是“很多变量互相影响时，如何清楚地表达关系”。如果变量少，可以直接写联合概率；变量一多，完整联合分布会爆炸。图模型用节点表示变量，用边表示依赖，把复杂分布拆成局部关系。

它像一张关系图：不用每个人和每个人都列一遍关系，只要画出关键连接，就能推断很多间接影响。

## 一个小例子

考虑下雨、地面湿、堵车三个变量：

- 下雨会影响地面是否湿。
- 下雨也可能影响是否堵车。
- 地面湿本身未必直接导致堵车。

贝叶斯网可以画成：

```text
下雨 -> 地面湿
下雨 -> 堵车
```

这样联合分布可写为：

```text
P(下雨, 地面湿, 堵车) = P(下雨) P(地面湿|下雨) P(堵车|下雨)
```

这比枚举所有组合更清晰。

## 贝叶斯网

贝叶斯网是有向无环图。每个节点表示随机变量，边表示直接依赖。联合分布分解为：

```text
P(x_1, ..., x_n) = Π_i P(x_i | Pa_i)
```

其中 `Pa_i` 是节点 `x_i` 的父节点集合。

优点：用局部条件概率表替代完整联合分布，大幅减少参数。

## 条件独立与 d 分离

贝叶斯网编码条件独立关系。典型结构：

- 链式：`A -> B -> C`，给定 B 后 A 与 C 独立。
- 分叉：`A <- B -> C`，给定 B 后 A 与 C 独立。
- 碰撞：`A -> B <- C`，未给定 B 时 A 与 C 独立；给定 B 或其后代后 A 与 C 可能相关。

碰撞结构最容易出错，因为“解释竞争”会引入相关性。

## 类比：解释竞争

如果“草地湿”可能由“下雨”或“洒水器”造成，那么在不知道草地湿之前，下雨和洒水器可能无关。但一旦知道草地湿了，如果又知道洒水器开了，那么“下雨”的可能性反而会下降。

这就是碰撞结构中给定结果后，两个原因之间会产生相关性的直觉。

## 马尔可夫网

马尔可夫网是无向图，适合表达对称关系。联合分布可由团势函数表示：

```text
P(x) = (1/Z) Π_C psi_C(x_C)
```

其中 `C` 为团，`Z` 是归一化常数：

```text
Z = sum_x Π_C psi_C(x_C)
```

势函数不一定是概率，只需非负；归一化后得到概率分布。

## 贝叶斯网和马尔可夫网怎么选

如果关系有明确方向，比如疾病导致症状、主题生成词语，贝叶斯网更自然。

如果关系更像相互约束，比如图像中相邻像素倾向相似、社交网络中朋友兴趣相近，马尔可夫网更自然。

## 条件随机场

CRF 是判别式概率图模型，直接建模：

```text
P(y|x)
```

在线性链 CRF 中：

```text
P(y|x) = (1/Z(x)) exp(sum_t sum_k lambda_k f_k(y_{t-1}, y_t, x, t))
```

常用于序列标注，如分词、词性标注、命名实体识别。

## 推断

推断任务包括：

- 边际推断：求某变量或变量子集的边际概率。
- 最大后验推断：求最可能的变量赋值。

精确推断方法：

- 变量消去。
- 信念传播。
- Junction tree。

近似推断方法：

- MCMC 采样。
- 变分推断。
- Loopy belief propagation。

## 学习

参数学习：

- 完全观测时，可用极大似然估计。
- 存在隐变量时，常用 EM。

结构学习：

- 搜索评分方法：定义结构评分并搜索。
- 约束方法：通过条件独立检验确定结构。

结构学习通常困难，因为可能图结构数量巨大。

## HMM

隐马尔可夫模型包含隐状态序列 `z_t` 和观测序列 `x_t`。

假设：

- 当前状态只依赖上一状态。
- 当前观测只依赖当前状态。

三个基本问题：

- 概率计算：前向算法。
- 解码：Viterbi 算法。
- 参数学习：Baum-Welch，即 EM。

## 易错点

- 贝叶斯网的边不一定代表因果，除非有额外因果假设。
- 马尔可夫网势函数不是条件概率。
- CRF 是判别式模型，HMM 是生成式模型。
- 精确推断在一般图上可能计算困难。

## 练习

1. 根据一个贝叶斯网写出联合分布分解。
2. 判断链式、分叉、碰撞结构中的条件独立。
3. 比较 HMM 与线性链 CRF。
4. 解释变量消去为什么受消去顺序影响。

## 长讲义扩展：图模型是概率世界的结构化语言

当变量很多时，直接建模联合分布几乎不可行。假设有 20 个二值变量，完整联合分布需要描述 `2^20` 种组合。图模型的作用是利用条件独立，把大问题拆成局部小问题。

图模型不是单一算法，而是一种表达语言。贝叶斯网用有向图表达依赖生成过程，马尔可夫网用无向图表达相互约束，CRF 直接建模条件分布，HMM 则是序列隐变量模型的经典例子。

## 贝叶斯网：用局部条件概率拼联合分布

贝叶斯网的每个节点只依赖父节点。若图结构正确，联合分布可以分解为：

```text
P(x_1, ..., x_n) = Π_i P(x_i | Pa_i)
```

这大幅减少参数。比如疾病会导致多个症状，建模时只需写 `P(症状|疾病)`，而不必枚举所有症状组合的完整联合概率。

贝叶斯网也便于解释。边可以表示直接依赖，但不一定自动表示因果。若要解释为因果关系，需要额外假设和实验设计。

## 三种基本结构

链式结构：

```text
A -> B -> C
```

给定 B 后，A 与 C 独立。因为 B 已经截断了从 A 到 C 的信息通路。

分叉结构：

```text
A <- B -> C
```

B 是共同原因。给定 B 后，A 与 C 独立。

碰撞结构：

```text
A -> B <- C
```

未给定 B 时，A 与 C 独立；给定 B 后，A 与 C 可能相关。这个结构最反直觉，但非常重要。

## 马尔可夫网：相互约束的关系

马尔可夫网没有方向，适合表达对称关系。例如图像中相邻像素倾向于颜色相近，社交网络中相连用户兴趣相似。这种关系不一定有明确父子方向。

马尔可夫网用势函数表示局部相容性：

```text
P(x) = (1/Z) Π_C psi_C(x_C)
```

势函数越大，表示该局部配置越偏好。归一化常数 `Z` 保证整体成为概率分布。难点是 `Z` 往往需要对所有变量配置求和，计算昂贵。

## HMM：序列中的隐状态

HMM 假设有一串看不见的状态生成一串看得见的观测。例如词性标注中，词性是隐状态，单词是观测；语音识别中，音素或状态是隐变量，声学信号是观测。

两个核心假设：

- 当前隐状态只依赖前一隐状态。
- 当前观测只依赖当前隐状态。

这两个假设让序列推断变得可计算。前向算法计算观测序列概率，Viterbi 算法寻找最可能隐状态序列，Baum-Welch 用 EM 学参数。

## CRF：直接建模条件概率

HMM 是生成式模型，需要建模 `P(x,y)`。CRF 是判别式模型，直接建模 `P(y|x)`。这让 CRF 能使用丰富、重叠、非独立的输入特征，而不必为观测建生成分布。

在线性链 CRF 中，标签之间有转移依赖，输入特征影响每个位置的标签。它曾广泛用于分词、词性标注、命名实体识别等序列标注任务。

## 推断为什么困难

图模型的推断通常是在问：已知一些变量，其他变量的概率是多少？若图结构简单，可以精确推断；若图中环很多、变量多，精确推断可能不可行。

变量消去的复杂度高度依赖消去顺序。某些顺序会产生很大的中间因子，计算爆炸。Junction tree、信念传播、采样和变分推断都是为处理推断复杂性而发展的方法。

## 学习：参数和结构

若图结构已知，参数学习相对直接。完全观测时可用极大似然；有隐变量时常用 EM。

结构学习更困难，因为可能图结构数量巨大。评分搜索方法给每个结构打分并搜索，约束方法通过条件独立检验推断结构。实际中常结合领域知识限制搜索空间。

## 图模型与深度学习

深度学习兴起后，许多传统图模型在感知任务中使用减少，但图模型思想仍然重要。结构化预测、因果推断、知识图谱、时序建模、概率编程都大量使用图模型思想。

深度模型擅长表示学习，图模型擅长表达不确定性和结构依赖。二者并不是互斥关系。

## 学习本章的主线

本章公式很多，但核心是一个问题：如何用图把复杂联合分布拆成可理解、可计算的局部关系？

掌握贝叶斯网的分解、马尔可夫网的势函数、HMM 的三大问题、CRF 的判别式序列建模，就抓住了主要框架。

## 万字版补充：变量消去小例子

假设联合分布分解为：

```text
P(A,B,C) = P(A) P(B|A) P(C|B)
```

现在要求 `P(C)`：

```text
P(C) = sum_A sum_B P(A) P(B|A) P(C|B)
```

可以先对 A 求和：

```text
g(B) = sum_A P(A) P(B|A)
P(C) = sum_B g(B) P(C|B)
```

这就是变量消去的思想：逐步把不关心的变量求和掉，并缓存中间因子。消去顺序不同，中间因子的大小可能差很多，影响计算复杂度。

## 信念传播的直觉

在树形图中，信念传播可以看作节点之间传消息。每个节点把自己从邻居那里收到的信息，结合本地势函数，再传给其他邻居。经过有限轮后，可以得到精确边际概率。

在有环图上，Loopy BP 不保证精确，但实践中常有不错效果。它像在关系网络中反复交换局部看法，最终形成近似全局一致的信念。

## EM 在图模型中的角色

有隐变量时，最大似然估计困难，因为看不到完整数据。EM 的思路是：

- E 步：根据当前参数推断隐变量分布。
- M 步：把隐变量的期望当作软信息，更新参数。

HMM 的 Baum-Welch 算法就是 EM。它先估计每个时刻处于各隐状态的概率，再用这些概率更新转移和发射参数。

## HMM 三个问题的例子

以天气和活动为例。隐状态是天气：晴、雨；观测是活动：散步、购物、打扫。

概率计算问题：给定活动序列“散步、购物、打扫”，它出现的概率是多少？用前向算法。

解码问题：这串活动最可能对应什么天气序列？用 Viterbi。

学习问题：只有大量活动序列，不知道天气，如何估计天气转移和活动发射概率？用 Baum-Welch。

## CRF 为什么适合序列标注

序列标注中，标签之间有依赖。例如命名实体识别中，`I-PER` 通常不能出现在没有 `B-PER` 之前的位置。CRF 可以学习标签转移约束，同时利用输入特征。

与逐位置独立分类相比，CRF 能输出全局更一致的标签序列。

## 概率图模型和因果图

贝叶斯网形式上像因果图，但普通贝叶斯网不自动等于因果模型。因果解释需要干预语义，例如 `do(X=x)`。仅凭观测相关性，通常无法确定因果方向。

不过概率图模型为因果推断提供了语言基础。混杂、碰撞、条件独立、d 分离等概念在因果图中非常重要。

## 图模型的工程挑战

图模型表达能力强，但工程上常遇到：

- 图结构难确定。
- 精确推断计算昂贵。
- 参数学习依赖大量数据或强先验。
- 连续变量和复杂特征处理不如深度模型方便。

因此实际应用常结合近似推断、采样、神经网络表示或领域知识。

## 本章自测问答

问：贝叶斯网如何分解联合分布？

答：每个节点条件依赖父节点，联合分布是所有局部条件概率的乘积。

问：马尔可夫网中的势函数是概率吗？

答：不一定。势函数只需非负，归一化后整体才是概率分布。

问：HMM 和 CRF 最大区别是什么？

答：HMM 是生成式模型，建模联合分布；CRF 是判别式模型，直接建模标签条件分布。

问：为什么推断困难？

答：变量组合空间巨大，图中环和高树宽会导致中间因子爆炸。

## 继续扩展：图模型 checklist

建模前检查：

- 变量之间是否有明确依赖结构。
- 关系是有向生成过程还是无向相互约束。
- 是否存在隐变量。
- 需要边际推断还是最大后验推断。
- 精确推断是否可行。
- 是否有足够数据估计局部概率。

## 常见面试题

问：碰撞结构为什么特殊？

答：两个原因共同指向一个结果。未观察结果时原因独立，观察结果后原因会相关。

问：CRF 为什么不需要建模 `P(x)`？

答：CRF 是判别式模型，直接建模 `P(y|x)`，因此可以使用丰富输入特征而不定义观测生成过程。

问：EM 解决什么问题？

答：处理含隐变量的参数估计，通过估计隐变量期望和更新参数交替优化。

## 深入补充：图模型中的板块表示

当有重复结构时，图模型常用板块表示。例如有 N 个样本，每个样本都有隐类别 `z_i` 和观测 `x_i`。不用画 N 遍节点，而是用一个 plate 表示重复。

板块表示让模型结构更简洁，常见于主题模型、混合模型、贝叶斯层次模型。

## 主题模型 LDA 的直觉

潜在狄利克雷分配 LDA 是概率图模型经典应用。它假设每篇文档是多个主题的混合，每个主题是词的分布。生成过程大致为：

1. 为文档抽取主题比例。
2. 对每个词，先抽一个主题。
3. 再从该主题的词分布中抽词。

LDA 可以在没有人工标签的情况下发现主题结构。它体现了隐变量图模型的思想：可观测词背后有不可观测主题。

## 近似推断方法

MCMC 通过采样近似目标分布，理论上灵活，但可能收敛慢。

变分推断把复杂后验近似成较简单分布，通过优化让近似分布接近真实后验。它通常更快，适合大规模数据，但有近似偏差。

采样和变分是概率机器学习中两条重要路线。

## 图模型的可解释性

图模型结构清楚，变量关系明确，因此可解释性较好。你可以指出某个变量依赖哪些父节点，某个观测如何通过隐变量影响预测。

但结构可解释不代表结果一定准确。若图结构假设错了，推断仍会错。图模型同样需要数据验证。

## 图模型应用场景

常见应用包括：

- 医学诊断中的疾病-症状网络。
- 自然语言处理中的序列标注。
- 机器人定位中的状态空间模型。
- 推荐系统中的隐变量模型。
- 因果推断中的结构图。
- 知识图谱中的关系推理。

图模型适合变量结构明确、不确定性表达重要的任务。

## 继续补充：从垃圾短信识别看图模型

假设短信是否垃圾由几个隐含因素影响：营销意图、诈骗意图、正常通知。短信中的词语、链接数量、发送频率、号码历史都与这些隐含因素相关。

可以构建一个贝叶斯网：

```text
发送者风险 -> 链接数量
发送者风险 -> 发送频率
短信意图 -> 关键词
短信意图 -> 是否垃圾
发送者风险 -> 是否垃圾
```

这个结构比单纯特征加权更清楚地表达变量关系。若发送者风险高且短信包含高风险关键词，垃圾概率上升。若只是链接数量多但发送者可信，判断可能不同。

## 图模型中的独立性带来参数节省

若有 10 个二值变量，完整联合分布需要 `2^10 - 1 = 1023` 个自由参数。若用图结构分解，每个变量只依赖少数父节点，参数数量会大幅减少。

这就是图模型的核心价值：用条件独立换取可估计性。没有结构假设，高维概率建模几乎不可行。

## HMM 前向算法直觉

前向算法维护：

```text
alpha_t(i) = 到第 t 个观测为止，且当前隐状态为 i 的概率
```

每一步用上一时刻所有状态的概率，经转移概率到当前状态，再乘当前观测概率。它把指数级的所有路径求和，压缩成动态规划。

Viterbi 类似，但把求和换成取最大，因此得到最可能路径，而不是总概率。

## CRF 的特征优势

CRF 可以使用任意设计的特征函数，例如：

```text
当前词是否大写
当前词是否出现在地名词典
前一个标签是否为 B-PER
当前词后缀是否为 ing
```

HMM 若要使用这些复杂特征，需要建模观测生成概率，会很困难。CRF 直接建模条件概率，因此更灵活。

## 练习答案提示

疾病-症状贝叶斯网中，若疾病 D 指向症状 S1、S2，则联合分布可写为：

```text
P(D,S1,S2)=P(D)P(S1|D)P(S2|D)
```

前向算法回答“观测序列概率是多少”，Viterbi 回答“最可能隐状态序列是什么”。二者目标不同。

## 继续补充：马尔可夫毯

在贝叶斯网中，一个节点的马尔可夫毯包括它的父节点、子节点以及子节点的其他父节点。给定马尔可夫毯后，该节点与网络中其他节点条件独立。

这个概念很有用，因为它告诉我们预测某个变量时，哪些局部变量已经足够。特征选择、因果推断和概率推断中都会用到类似思想。

## 图模型与缺失数据

图模型可以自然处理缺失变量。若某些变量未观测，就在推断时对它们求和或积分。比如医学诊断中，有些检查未做，模型仍可根据已知症状和检查结果估计疾病概率。

这比简单删除缺失样本更优雅。但代价是推断可能更复杂。

## 动态贝叶斯网

HMM 是动态贝叶斯网的特例。动态贝叶斯网用时间片表示状态随时间演化，每个时间片包含多个变量。

例如自动驾驶中，车辆位置、速度、周围障碍物都随时间变化。动态贝叶斯网可以表达这些变量的时间依赖和观测关系。

## 概率图模型章节总结

概率图模型的主线是：用图结构表达依赖，用局部分布拼出整体分布，用推断算法回答未知变量问题。它的优势是结构清楚、概率解释自然；挑战是结构学习和推断计算复杂。

### 精讲扩展 5：概率图模型 的优化算法、泛化能力 与工程化理解

从 $a 的角度看，最重要的是先建立清晰的对象模型。也就是明确系统里有哪些参与者、它们之间如何连接、数据或控制信号如何流动、哪些环节是同步的、哪些环节是异步的、哪些状态是临时状态、哪些状态需要长期保存。很多初学问题并不是公式不会、API 不熟，而是对象边界不清：把配置当成状态，把结果当成过程，把局部现象当成全局规律。写笔记时建议始终追问：这个概念的主体是谁，输入是什么，输出是什么，中间约束是什么，错误会在哪里暴露。

从 $b 的角度看，流程比单点知识更关键。一个成熟方案通常不是单个技巧，而是一组步骤：先确定目标，再拆分约束，然后选择工具，最后通过测试和复盘确认效果。比如在实际项目中，不能只问“怎么实现”，还要问“为什么要这样实现”“有没有更简单的替代方案”“边界条件是什么”“数据量、并发量、实时性、可靠性变化后还能不能工作”。这种流程意识能够避免把学习停留在教程层面，也能让后续排错有明确路线。

$topic 的 $c 往往决定它在真实项目中的稳定性。理论上可行的方案，到了工程环境中会受到数据质量、硬件条件、依赖版本、网络环境、团队协作、部署方式和维护成本影响。写代码或做设计时，应该把正常路径和异常路径同时考虑：正常情况下如何运行，输入为空怎么办，超时怎么办，重复执行怎么办，部分成功怎么办，版本升级后兼容性怎么办，日志和指标如何证明系统确实按预期工作。
