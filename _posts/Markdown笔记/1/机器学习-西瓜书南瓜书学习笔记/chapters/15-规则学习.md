# 第 15 章 规则学习

## 讲义导读：用泛化主线读懂这一章

这一章讲的是 **第 15 章 规则学习**。阅读时不要只背公式和算法名，而要把它放进机器学习的完整链路：现实问题如何变成数据，样本和标签是否可靠，模型假设空间是什么，损失函数如何定义好坏，优化算法如何找到参数，验证集如何选择模型，测试集如何估计泛化，最后错误样本如何反过来改进数据和模型。机器学习真正要解决的不是训练集上得分高，而是在未来数据上稳定可用。

### 一句话先懂

规则学习用可读规则表达分类或决策逻辑，优势是解释清楚，风险是规则过窄或互相冲突。

### 通俗类比

可以把机器学习想成从错题本里总结解题方法：训练集是做过的题，模型是假设的解题套路，损失函数是扣分规则，优化是反复订正，验证集像模拟考，测试集像正式考试，泛化能力就是换一套题还能不能做对。复杂模型不是天生更聪明，它只是可选择的解题套路更多，也更容易把错题本里的偶然细节背下来。

类比只能帮助入门。回到技术上，要把每个概念对应到精确定义：样本、特征、标签、假设空间、目标函数、训练误差、泛化误差、评价指标、正则化和数据分布。只有这些对象清楚，公式才不是符号堆，实验结果才有解释意义。

### 本章学习主线

1. **问题定义**：先确定任务是分类、回归、聚类、降维、序列决策还是概率推断。
2. **数据假设**：检查样本来源、标签噪声、特征含义、训练测试分布和数据泄漏。
3. **模型偏好**：说清算法偏好什么样的规律，例如线性、大间隔、短规则、平滑函数、稀疏特征或局部相似。
4. **损失与优化**：理解模型到底在最小化什么，优化失败、局部最优或过拟合会怎样表现。
5. **评估与泛化**：用合适指标、交叉验证、学习曲线、误差分析和消融实验判断模型是否可靠。
6. **落地边界**：关注可解释性、推理成本、鲁棒性、公平性、分布漂移和维护成本。

### 概念怎么学才不容易忘

遇到新算法时，建议按 白话任务 -> 模型形式 -> 损失函数 -> 优化方法 -> 泛化控制 -> 失败模式 六步理解。比如学习决策树，不只记信息增益，还要知道树为什么会过拟合、剪枝如何改变泛化；学习 SVM，不只写对偶问题，还要理解最大间隔为什么是一种归纳偏好；学习聚类，不只跑 K-means，还要说明距离度量和簇形状假设。

### 最小实践任务

把一棵小决策树的路径改写成规则集，比较覆盖率、准确率和剪枝前后变化。

实践时要保留失败案例：训练分数高但测试差、指标选择不匹配、类别不平衡、数据泄漏、特征尺度错误、验证集被反复调参污染、聚类结果不可解释。机器学习能力很大一部分来自能解释这些失败。

### 读完本章应该能做到

- 用自己的话解释本章算法或概念解决什么学习问题。
- 写出数据、模型、损失、优化、评估和泛化控制六个要素。
- 给出一个适合场景和一个不适合场景，并说明背后的假设。
- 设计一个最小实验，能观察关键参数变化对结果的影响。
- 能解释规则覆盖、序贯覆盖、FOIL、剪枝、一阶规则和规则冲突处理。

## 学习目标

规则学习用人类可读的“若条件则结论”形式表达模型。本章重点是规则表示、序贯覆盖、规则剪枝、一阶规则和归纳逻辑程序设计。

## 先用人话理解

规则学习追求的是把模型写成清楚的判断语句。它不像神经网络那样把知识藏在大量权重里，而是直接产生类似专家系统的规则：

```text
如果纹理清晰，并且根蒂蜷缩，那么更可能是好瓜。
```

这种模型特别适合需要解释的场景，例如医疗、风控、审批和故障诊断。

## 一个小例子

判断客户是否有较高流失风险，可以学习规则：

```text
IF 近30天登录次数很少 AND 最近一次投诉未解决 THEN 高流失风险
IF 套餐费用上升 AND 使用时长下降 THEN 高流失风险
```

这些规则不一定覆盖所有客户，但能给运营人员明确的行动线索。

## 规则表示

典型规则：

```text
IF 条件1 AND 条件2 AND ... THEN 类别
```

规则的覆盖：满足规则前件的样本集合。

规则的准确率：

```text
accuracy = 覆盖且分类正确的样本数 / 覆盖样本数
```

规则的覆盖率：

```text
coverage = 覆盖样本数 / 总样本数
```

高准确率规则可能覆盖很少样本，高覆盖率规则可能不够精确。

## 序贯覆盖

序贯覆盖是一种逐条学习规则的方法：

1. 从训练集中学习一条规则。
2. 删除该规则覆盖的正例。
3. 在剩余样本上继续学习下一条规则。
4. 直到正例基本被覆盖。

它把复杂规则集学习分解为多次单规则学习。

## 类比：序贯覆盖像逐条写办案规则

先找一条能抓住一批典型正例的规则，把这批样本处理掉；再看剩下没覆盖的正例，继续写下一条规则。每条规则解决一部分情况，最后组合成规则集。

它的风险是：前面的规则会影响后面的学习。如果前面规则太宽，可能带走很多不该带走的样本。

## 学习单条规则

常见方式是从空规则开始逐渐添加条件，使规则越来越具体。

每次添加条件时可使用评价函数，如信息增益、准确率提升、覆盖正例数量等。

规则越具体，覆盖样本越少，准确率可能提高，但泛化可能下降。

## 规则的宽与窄

规则太宽：

```text
IF 访问次数下降 THEN 流失风险高
```

覆盖很多人，但误报可能高。

规则太窄：

```text
IF 访问次数下降 AND 周二投诉 AND 使用安卓手机 AND 套餐A THEN 流失风险高
```

训练集上可能很准，但覆盖太少，也可能只是偶然现象。

## 剪枝

规则容易过拟合，需要剪枝。常见做法：

- 预剪枝：若添加条件收益不足则停止。
- 后剪枝：先学习较复杂规则，再删除贡献不大的条件。

剪枝评价应基于验证集或复杂度惩罚。

## 命题规则与一阶规则

命题规则只处理固定属性，例如：

```text
IF 色泽=青绿 AND 根蒂=蜷缩 THEN 好瓜
```

一阶规则能表达对象和关系，例如：

```text
grandparent(X, Y) :- parent(X, Z), parent(Z, Y)
```

一阶表达能力更强，但搜索空间也更大。

## 归纳逻辑程序设计

ILP 结合机器学习和逻辑程序设计，从样例和背景知识中学习逻辑规则。

基本元素：

- 正例：应被规则推出的事实。
- 反例：不应被规则推出的事实。
- 背景知识：已有逻辑事实和规则。
- 假设：待学习的逻辑规则。

目标是在背景知识基础上找到能覆盖正例、排除反例的规则集。

## 规则学习的特点

优点：

- 可解释性强。
- 便于加入领域知识。
- 能表达离散逻辑关系。

缺点：

- 搜索空间大。
- 对噪声敏感。
- 连续属性需要离散化或特殊处理。
- 规则之间可能冲突。

## 易错点

- 规则准确率高不代表整体模型好，还要看覆盖率。
- 序贯覆盖删除正例后，后续规则会受前面规则影响。
- 可解释模型不等于一定简单；规则太多时仍难理解。
- 一阶规则表达能力强，但学习难度显著上升。

## 练习

1. 给定小数据集，手工学习两条分类规则。
2. 比较决策树路径与规则集的关系。
3. 解释规则剪枝如何缓解过拟合。
4. 写出一个一阶规则并说明变量含义。

## 长讲义扩展：规则学习的价值在可解释性

规则学习的核心目标不是追求最复杂的函数，而是学习人能读懂、能检查、能执行的规则。它特别适合需要解释和审计的领域，比如医疗诊断、金融风控、法律合规、工业故障排查。

一条规则通常形如：

```text
IF 条件1 AND 条件2 THEN 结论
```

这类表达很接近专家经验。相比黑盒模型，规则模型更容易回答“为什么做出这个判断”。

## 规则和决策树的关系

决策树从根到叶的每条路径都可以转成一条规则。例如：

```text
IF 纹理=清晰 AND 根蒂=蜷缩 THEN 好瓜
```

但规则集不一定来自一棵树。规则学习可以逐条学习规则，每条规则覆盖一部分样本。规则之间可能有顺序，也可能通过投票或优先级解决冲突。

决策树强调层级划分，规则集强调局部模式。某些场景中，少量高质量规则比一棵复杂树更容易解释。

## 覆盖率与准确率的平衡

规则评价不能只看准确率。一个规则如果只覆盖 1 个样本且判断正确，准确率 100%，但价值有限。另一个规则覆盖 1000 个样本，准确率 90%，可能更有用。

因此要同时看：

- 准确率：覆盖样本中判断对多少。
- 覆盖率：规则能作用于多少样本。
- 简洁性：条件是否过多。
- 稳定性：换一批数据是否仍成立。

规则越具体，通常准确率上升、覆盖率下降；规则越宽泛，覆盖率上升、准确率可能下降。

## 序贯覆盖的完整过程

序贯覆盖先学习一条规则，覆盖一批正例，然后从数据中移除这些正例，再学习下一条规则。这个过程持续到正例基本被覆盖。

它像写一本故障处理手册：先写最典型、最常见的一类故障规则；再看剩下没解释的案例，继续写第二条、第三条。

学习单条规则时，常从空前件开始逐渐添加条件。每添加一个条件，规则变得更具体，覆盖样本变少，准确率可能提高。算法要决定什么时候停止添加条件。

## 规则剪枝为什么必要

规则也会过拟合。如果一条规则包含太多条件，它可能只是在描述训练集中某几个样本的偶然组合。

例如：

```text
IF 年龄=37 AND 城市=某地 AND 周三登录 AND 使用某型号手机 THEN 流失
```

这种规则可能训练集上准确，但没有泛化意义。剪枝会尝试删除某些条件，若验证性能不下降甚至上升，就说明这些条件不必要。

## 一阶规则的表达能力

命题规则处理固定属性，一阶规则可以表达对象之间的关系。例如：

```text
grandparent(X, Y) :- parent(X, Z), parent(Z, Y)
```

它不是在说某个固定表格特征，而是在表达变量和关系。这让一阶规则能处理知识图谱、逻辑推理、关系数据库等场景。

表达能力增强后，搜索空间也急剧增大。学习一阶规则需要背景知识、语言偏置和有效搜索策略。

## ILP 的基本思想

归纳逻辑程序设计从正例、反例和背景知识中学习逻辑规则。它结合了机器学习的归纳能力和逻辑程序的表达能力。

例如已知许多家庭关系事实：

```text
parent(alice, bob)
parent(bob, carol)
```

以及一些 `grandparent` 的正反例，ILP 可以学习出祖父母关系规则。

这类方法在关系结构强、背景知识丰富、可解释性要求高的任务中有价值。

## 规则学习的局限

规则模型对连续变量不天然友好，通常需要离散化或学习阈值。规则过多时，可解释性也会下降。噪声数据会导致规则不稳定，规则之间还可能冲突。

此外，贪心学习规则可能错过全局最优。早期学到的一条规则若覆盖了不该覆盖的样本，后续规则会受影响。

## 实践建议

若目标是解释，规则应尽量短、稳定、覆盖足够样本。可以限制最大条件数、设置最小覆盖数、使用验证集剪枝。

在业务场景中，规则学习常和人工专家结合。模型生成候选规则，专家审核规则含义，再部署到系统中。这样既利用数据发现模式，又保留可控性。

学习本章时，重点不是记住某个规则搜索算法，而是理解规则模型的三角权衡：准确率、覆盖率和简洁性。

## 万字版补充：手工学习一条规则

假设有客户流失数据，正例是已流失客户。我们从空规则开始：

```text
IF  THEN 流失
```

这条规则覆盖所有人，准确率等于总体流失率，通常很低。现在尝试添加条件：

```text
近30天登录次数 < 2
投诉次数 > 0
套餐价格上涨
客服评分低
```

分别计算每个条件加入后的覆盖正例数、覆盖反例数和准确率。若“近30天登录次数 < 2”覆盖 200 人，其中 120 人流失，准确率 60%；“投诉次数 > 0”覆盖 80 人，其中 60 人流失，准确率 75%。选择哪个取决于评价函数：更重准确率还是覆盖率。

接着继续添加第二个条件：

```text
IF 近30天登录次数 < 2 AND 投诉次数 > 0 THEN 流失
```

规则更具体，准确率可能提高，覆盖率下降。学习单条规则就是不断在这个权衡中前进。

## FOIL 增益的直觉

FOIL 是一类学习一阶规则的算法。它选择能提高规则区分能力的文字。直觉上，一个新条件好不好，要看它是否保留了较多正例，同时排除了较多反例。

这和决策树的信息增益类似，都是在问：加入这个判断后，样本是否更纯？

## 规则冲突如何处理

多个规则可能同时匹配同一样本，并给出不同结论。解决方式包括：

- 按规则顺序，先匹配先输出。
- 按置信度或准确率排序。
- 多条规则投票。
- 使用默认规则处理未覆盖样本。

规则顺序会影响结果，因此规则集不仅是规则的集合，也可能是一套决策列表。

## 默认规则

很多样本可能不满足任何特定规则。此时需要默认规则，例如预测多数类。默认规则看似简单，但很重要，因为它决定规则系统在未知情况中的行为。

如果大量样本都落入默认规则，说明规则覆盖不足；如果几乎所有样本都被复杂规则覆盖，可能过拟合。

## 规则质量的业务评估

业务中评价规则不仅看准确率，还看可行动性。例如：

```text
IF 用户近7天未登录 THEN 可能流失
```

这条规则可行动，可以推送召回。但：

```text
IF 用户已提交注销申请 THEN 可能流失
```

虽然准确，但可能太晚，干预价值低。

规则学习要考虑规则是否提前、是否可解释、是否能触发行动。

## 规则模型和合规

在风控、保险、医疗等场景，规则模型常用于满足审计要求。每个决策都能追溯到具体条件，便于人工复核。

但规则也可能引入偏见。例如使用地区、设备、职业等特征时，可能间接影响特定群体。可解释不等于公平，规则仍需合规审查。

## 与关联规则的区别

关联规则如：

```text
买了面包 -> 也买牛奶
```

它主要发现项集之间的共现关系，常用支持度、置信度、提升度评价。分类规则则以预测某个标签为目标。二者形式相似，但任务不同。

## 本章自测问答

问：规则准确率高就一定好吗？

答：不一定。还要看覆盖率、稳定性、简洁性和业务可行动性。

问：序贯覆盖为什么可能受早期规则影响？

答：前面规则覆盖并移除部分正例，后续规则只能在剩余样本上学习。

问：一阶规则比命题规则强在哪里？

答：能表达对象之间的关系和变量绑定，而不只是固定属性判断。

问：规则剪枝的目的是什么？

答：删除偶然条件，降低过拟合，提高泛化和可解释性。

## 继续扩展：规则学习 checklist

学习规则前检查：

- 是否真的需要可解释规则。
- 特征是否适合写成条件。
- 连续变量是否需要分箱或阈值。
- 规则最小覆盖样本数是多少。
- 如何处理规则冲突。
- 是否需要默认规则。

规则上线前检查：

- 规则是否稳定。
- 是否存在歧视性或合规风险。
- 业务人员是否能理解和执行。
- 规则触发后是否有明确动作。
- 是否监控规则命中率和效果。

## 常见面试题

问：规则学习和决策树哪个更可解释？

答：浅决策树和短规则集都可解释；深树和大量规则都会变难解释。规则集更灵活，决策树结构更统一。

问：规则太多有什么问题？

答：维护困难、冲突增多、整体逻辑难理解，也可能过拟合。

问：覆盖率低的高准确规则有没有用？

答：可能有用，尤其用于高价值或高风险小群体；但不能单独代表整体模型效果。

## 深入补充：规则的生命周期

规则不是学出来就结束。真实系统中的规则有生命周期：

1. 发现：由数据挖掘、模型解释或专家提出。
2. 验证：在历史数据上评估准确率、覆盖率、收益和风险。
3. 审核：由业务、法务、风控或医学专家确认合理性。
4. 上线：加入规则引擎或决策系统。
5. 监控：观察命中率、误判率、业务收益。
6. 下线或更新：当环境变化、规则失效或带来副作用时调整。

这说明规则学习既是算法问题，也是知识管理问题。

## 规则和模型结合

很多工业系统不是纯规则或纯模型，而是二者结合。例如：

- 硬规则先拦截明显违规样本。
- 机器学习模型处理复杂灰区。
- 模型高风险但规则未命中的样本进入人工审核。
- 规则用于解释或约束模型输出。

规则的优点是确定、可控、易审计；模型的优点是能处理复杂统计模式。结合使用往往更实用。

## 规则漂移

规则会过时。欺诈检测中，对手会根据规则改变行为；用户运营中，活动策略变化会改变用户反应；医疗诊断中，新设备和新流程会改变数据分布。

因此规则需要监控命中率和效果。如果一条规则命中率突然升高或降低，都可能说明环境变了。

## 规则学习和可解释 AI

可解释 AI 中常用规则近似黑盒模型。例如在某个局部区域，用规则解释神经网络或 GBDT 的预测。这样可以帮助人理解模型行为，但要注意近似规则不一定代表全局模型逻辑。

规则解释应标明适用范围。局部解释不能随意推广到所有样本。

## 更多自测

问：规则系统为什么需要默认规则？

答：因为不是所有样本都会匹配已有规则，默认规则决定未覆盖样本的处理方式。

问：规则越短越好吗？

答：不一定。短规则易解释但可能太宽泛；长规则更精确但可能过拟合。

问：规则学习适合连续特征吗？

答：可以，但通常要学习阈值或先离散化。

## 继续补充：规则学习的完整案例

假设一个互联网平台要识别异常账号。机器学习模型可以输出风险分，但风控团队还需要可解释规则，用于快速拦截明显异常行为。

候选规则可能包括：

```text
IF 注册时间 < 1 天 AND 登录 IP 国家变化次数 > 3 THEN 高风险
IF 10 分钟内发送私信数 > 100 AND 被举报次数 > 5 THEN 高风险
IF 设备指纹关联账号数 > 20 THEN 高风险
```

每条规则都要评估覆盖率、准确率和误伤成本。第一条可能覆盖新型攻击，第二条可能拦截垃圾消息，第三条可能发现批量注册设备。

上线前，业务人员需要确认规则是否合理，是否可能误伤正常用户。例如海外旅行用户也可能 IP 变化，不能只凭单一条件封禁账号。

## 规则的优先级设计

规则系统中常有不同级别：

- 强拦截规则：命中后直接拒绝。
- 弱风险规则：命中后提高风险分。
- 审核规则：命中后进入人工审核。
- 提示规则：只记录，不影响决策。

不是所有规则都应该直接决定结果。高准确但覆盖小的规则适合强拦截；准确率一般但召回高的规则适合辅助模型或人工审核。

## 规则和阈值

很多规则包含阈值，如“近 10 分钟发送私信数 > 100”。阈值不能随便拍脑袋。可以通过历史数据画出不同阈值下的命中率、准确率和误伤率，再结合业务承受能力选择。

阈值还可能随时间变化。活动期间用户行为更活跃，固定阈值可能误伤。规则系统需要支持动态阈值或场景化阈值。

冲突规则可以用优先级、置信度排序、投票或人工审核解决。

## 继续补充：规则学习中的离散化

连续特征要进入规则，通常需要阈值。例如年龄、收入、登录次数都可以转为条件：

```text
年龄 > 60
收入 < 3000
近7天登录次数 = 0
```

阈值可以由专家给出，也可以由数据学习。用数据学习阈值时，要注意过拟合。若每个连续特征都允许大量候选阈值，规则搜索空间会变大。

## 规则集的压缩

学出的规则可能很多，其中一些规则高度相似。可以做规则合并和压缩。例如：

```text
IF A AND B AND C THEN 高风险
IF A AND B AND D THEN 高风险
```

若 C 和 D 都只是弱条件，可以考虑合并为：

```text
IF A AND B THEN 高风险
```

压缩后规则更简单，但可能降低准确率。需要在可解释性和性能之间权衡。

## 规则和人类知识

规则学习特别适合结合专家知识。专家可以限制搜索空间，如只允许医学上合理的条件组合；也可以否决数据中偶然但不合理的规则。

纯数据驱动规则可能发现伪相关，专家知识能提高规则可信度。反过来，数据也能发现专家未注意到的新模式。

## 规则学习章节总结

规则学习的核心价值是可读、可审计、可行动。它不一定追求最高预测分数，而是追求把规律表达成人能理解和维护的形式。评价规则时必须同时看准确率、覆盖率、简洁性、稳定性和业务后果。
