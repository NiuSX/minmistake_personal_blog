# 学习路线

## 讲义导读：用泛化主线读懂这一章

这一章讲的是 **学习路线**。阅读时不要只背公式和算法名，而要把它放进机器学习的完整链路：现实问题如何变成数据，样本和标签是否可靠，模型假设空间是什么，损失函数如何定义好坏，优化算法如何找到参数，验证集如何选择模型，测试集如何估计泛化，最后错误样本如何反过来改进数据和模型。机器学习真正要解决的不是训练集上得分高，而是在未来数据上稳定可用。

### 一句话先懂

总览类笔记的任务是把西瓜书/南瓜书的章节串成学习路线：先理解泛化和评估，再学模型族，最后看无监督、概率图、强化学习和理论边界。

### 通俗类比

可以把机器学习想成从错题本里总结解题方法：训练集是做过的题，模型是假设的解题套路，损失函数是扣分规则，优化是反复订正，验证集像模拟考，测试集像正式考试，泛化能力就是换一套题还能不能做对。复杂模型不是天生更聪明，它只是可选择的解题套路更多，也更容易把错题本里的偶然细节背下来。

类比只能帮助入门。回到技术上，要把每个概念对应到精确定义：样本、特征、标签、假设空间、目标函数、训练误差、泛化误差、评价指标、正则化和数据分布。只有这些对象清楚，公式才不是符号堆，实验结果才有解释意义。

### 本章学习主线

1. **问题定义**：先确定任务是分类、回归、聚类、降维、序列决策还是概率推断。
2. **数据假设**：检查样本来源、标签噪声、特征含义、训练测试分布和数据泄漏。
3. **模型偏好**：说清算法偏好什么样的规律，例如线性、大间隔、短规则、平滑函数、稀疏特征或局部相似。
4. **损失与优化**：理解模型到底在最小化什么，优化失败、局部最优或过拟合会怎样表现。
5. **评估与泛化**：用合适指标、交叉验证、学习曲线、误差分析和消融实验判断模型是否可靠。
6. **落地边界**：关注可解释性、推理成本、鲁棒性、公平性、分布漂移和维护成本。

### 概念怎么学才不容易忘

遇到新算法时，建议按 白话任务 -> 模型形式 -> 损失函数 -> 优化方法 -> 泛化控制 -> 失败模式 六步理解。比如学习决策树，不只记信息增益，还要知道树为什么会过拟合、剪枝如何改变泛化；学习 SVM，不只写对偶问题，还要理解最大间隔为什么是一种归纳偏好；学习聚类，不只跑 K-means，还要说明距离度量和簇形状假设。

### 最小实践任务

按自己的基础列出四周学习计划，每周选一个章节、一个手算例子和一个 scikit-learn 小实验。

实践时要保留失败案例：训练分数高但测试差、指标选择不匹配、类别不平衡、数据泄漏、特征尺度错误、验证集被反复调参污染、聚类结果不可解释。机器学习能力很大一部分来自能解释这些失败。

### 读完本章应该能做到

- 用自己的话解释本章算法或概念解决什么学习问题。
- 写出数据、模型、损失、优化、评估和泛化控制六个要素。
- 给出一个适合场景和一个不适合场景，并说明背后的假设。
- 设计一个最小实验，能观察关键参数变化对结果的影响。
- 能说明每章在全书中的位置，以及它和前后章节的依赖关系。

## 总体方法

学习机器学习时，不建议只按算法名称记忆。更稳定的组织方式是围绕以下问题：

1. 数据是什么：样本、属性、标签、分布、噪声。
2. 模型是什么：假设空间、参数、结构、表达能力。
3. 目标是什么：经验风险、结构风险、似然、间隔、后验概率。
4. 如何求解：解析解、梯度下降、凸优化、贪心搜索、动态规划、采样。
5. 如何评价：训练误差、测试误差、交叉验证、泛化能力、统计显著性。
6. 何时有效：模型假设、数据规模、特征类型、噪声水平、计算成本。

每学一个算法，都建议按下面四步走：

1. 先讲人话：这个算法想解决什么问题？不用公式能不能说清？
2. 看一个例子：给它一个小数据集时，它会怎么一步步做？
3. 再看公式：公式中的每一项分别对应算法直觉里的哪部分？
4. 最后做对比：它和相邻算法有什么不同，什么时候会失败？

例如学习 SVM，不要先背对偶问题。先理解“在两类样本中间留出最大安全距离”，再理解为什么要最小化 `||w||^2`，最后再看拉格朗日乘子和核函数。

## 阶段一：核心概念

目标：能解释机器学习在做什么，并能正确评估一个模型。

重点章节：

- 绪论
- 模型评估与选择
- 线性模型
- 决策树
- 贝叶斯分类器

应完成的任务：

- 区分训练误差、验证误差、测试误差、泛化误差。
- 手写混淆矩阵、准确率、查准率、查全率、F1。
- 推导最小二乘线性回归的正规方程。
- 推导逻辑回归的对数似然与梯度。
- 手算信息增益、基尼指数、朴素贝叶斯分类。

通俗理解主线：

- 机器学习像从练习题中总结解题套路。
- 评估模型像设计公平考试，不能让模型提前见到试卷。
- 线性模型像给每个证据打分再求和。
- 决策树像一套自动生成的问答排查流程。
- 贝叶斯分类器像看到证据后不断更新相信程度。

## 阶段二：优化与模型能力

目标：理解为什么同一个经验误差下，不同模型泛化能力可能不同。

重点章节：

- 神经网络
- 支持向量机
- 集成学习

应完成的任务：

- 能写出多层感知机的前向传播和反向传播。
- 理解 SVM 中间隔最大化、拉格朗日乘子、KKT 条件的作用。
- 能说明 AdaBoost 为什么会提高错分样本的权重。
- 能比较 Bagging、Boosting 和随机森林的差异。

通俗理解主线：

- 神经网络像多层加工流水线，反向传播像沿流水线分摊错误责任。
- SVM 像在两类样本之间画一条安全距离最大的分界线。
- 集成学习像让多个视角不同的人投票或接力纠错。

## 阶段三：表示、结构与理论

目标：从“会用算法”转向“知道算法为何适用、为何失效”。

重点章节：

- 聚类
- 降维与度量学习
- 特征选择与稀疏学习
- 计算学习理论

应完成的任务：

- 实现 K-means，并解释其目标函数为何单调下降。
- 推导 PCA 最大方差形式与最小重构误差形式的等价关系。
- 理解 L1 正则为什么更容易得到稀疏解。
- 能解释 VC 维、PAC 学习和泛化界的直觉。

通俗理解主线：

- 聚类像整理杂物，不同整理标准会得到不同分组。
- 降维像给复杂数据写摘要，保留主线、压缩细节。
- 特征选择像抓关键证据，稀疏学习像限制模型少说废话。
- 学习理论研究“多少数据才足以让我们放心”。

## 阶段四：复杂学习范式

目标：面对标签不足、结构化变量、序列决策时，能选择合适建模方式。

重点章节：

- 半监督学习
- 概率图模型
- 规则学习
- 强化学习

应完成的任务：

- 区分生成式半监督和判别式半监督。
- 能画出贝叶斯网、马尔可夫网并写出联合分布分解。
- 理解决策列表、序贯覆盖和规则剪枝。
- 写出 Bellman 方程，区分价值函数、动作价值函数和策略。

通俗理解主线：

- 半监督学习像少量带答案题目带着大量无答案题目一起学习。
- 概率图模型像把变量之间的依赖画成关系图。
- 规则学习像整理专家经验：“如果满足这些条件，那么作出这个判断”。
- 强化学习像通过试错拿奖励，学习长期收益最高的行动策略。

## 2026 机器学习资料与实践核对补充

机器学习的经典理论稳定，但工具库、默认参数、数据集版本和评估规范会变化。复现实验前建议记录 Python、NumPy、SciPy、scikit-learn、PyTorch 或其他库版本，固定随机种子、数据切分、预处理步骤、特征工程、评价指标和硬件环境。尤其要防止数据泄漏、验证集污染和只报告最好一次结果。

学习西瓜书和南瓜书时，建议把书中推导、南瓜书公式解释、scikit-learn 实验和自己的手算例子结合起来。官方/经典资料负责正确性，代码实验负责直觉，误差分析负责落地判断。
