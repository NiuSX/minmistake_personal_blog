## 1. 强化学习是什么

强化学习是一类研究智能体如何通过与环境交互、根据奖励信号进行试错学习，从而学会最优决策策略的机器学习方法。

强化学习的核心过程是：

```text
智能体观察状态 -> 选择动作 -> 环境反馈奖励和新状态 -> 智能体更新策略
```

英文中强化学习称为 Reinforcement Learning，简称 RL。

### 1.1 一个直观例子

假设训练一个智能体玩迷宫游戏。

智能体每一步可以选择：

```text
上、下、左、右
```

环境反馈：

- 撞墙：奖励 -1
- 走普通格子：奖励 0
- 到达终点：奖励 +10

智能体一开始不知道怎么走，只能尝试。经过多次试错后，它会学到：

```text
哪些状态下走哪个方向更可能到达终点。
```

这就是强化学习。

### 1.2 强化学习的本质

强化学习的本质是序列决策。

一个动作不仅影响当前奖励，还会影响未来状态和未来奖励。

例如：

```text
当前看起来收益最高的动作，可能导致未来陷入困境。
当前看起来收益较低的动作，可能通向更高长期回报。
```

因此强化学习关注的是长期累计奖励，而不是单步奖励。

### 1.3 强化学习适合什么问题

强化学习适合：

- 决策会影响未来状态。
- 没有直接监督标签。
- 可以通过奖励评价行为好坏。
- 需要在探索和利用之间权衡。

典型场景：

- 游戏 AI。
- 机器人控制。
- 自动驾驶决策。
- 推荐系统策略优化。
- 广告竞价。
- 资源调度。
- 对话系统。
- 大模型对齐中的 RLHF。

### 1.4 强化学习为什么难

强化学习难点包括：

- 奖励可能延迟。
- 数据不是独立同分布。
- 探索可能代价高。
- 训练不稳定。
- 样本效率低。
- 奖励设计困难。
- 评估成本高。
- 环境可能非平稳。

## 2. 强化学习与监督学习、无监督学习的区别

### 2.1 监督学习

监督学习有明确标签：

```text
输入 x -> 真实标签 y
```

模型学习从输入到标签的映射。

例如：

```text
图片 -> 猫/狗
用户特征 -> 是否流失
```

### 2.2 无监督学习

无监督学习没有标签，目标是发现数据结构。

例如：

- 聚类。
- 降维。
- 密度估计。

### 2.3 强化学习

强化学习没有每一步的正确动作标签，只有行为后的奖励反馈。

例如：

```text
状态 s 下没有人告诉智能体正确动作是 a。
智能体执行动作后，只知道获得了多少奖励。
```

强化学习学习的是策略：

```text
在什么状态下采取什么动作。
```

### 2.4 三者对比

| 对比项 | 监督学习 | 无监督学习 | 强化学习 |
| --- | --- | --- | --- |
| 数据 | 有标签样本 | 无标签样本 | 交互轨迹 |
| 反馈 | 正确答案 | 无显式答案 | 奖励 |
| 目标 | 预测准确 | 发现结构 | 最大化长期回报 |
| 数据分布 | 通常固定 | 通常固定 | 由策略影响 |
| 难点 | 泛化 | 表示和结构 | 探索、延迟奖励、稳定性 |

### 2.5 强化学习的数据不是独立同分布

监督学习通常假设样本独立同分布。

强化学习中，数据来自智能体与环境的交互：

```text
s0, a0, r1, s1, a1, r2, s2, ...
```

当前状态取决于之前动作，样本之间高度相关。

这也是深度强化学习需要经验回放等技巧的重要原因。

## 3. 强化学习基本要素

强化学习由几个核心要素组成。

### 3.1 Agent 智能体

智能体是做决策的主体。

例如：

- 游戏中的玩家。
- 机器人控制器。
- 推荐系统中的策略模型。
- 自动驾驶决策模块。

### 3.2 Environment 环境

环境是智能体交互的对象。

智能体执行动作后，环境返回：

- 新状态。
- 奖励。
- 是否终止。

### 3.3 State 状态

状态 `s` 描述环境当前情况。

例如：

- 游戏画面。
- 机器人关节角度和速度。
- 用户当前上下文。
- 棋盘局面。

状态应尽可能包含决策所需信息。

### 3.4 Action 动作

动作 `a` 是智能体可以选择的行为。

动作空间可以是：

- 离散动作空间。
- 连续动作空间。

离散动作例子：

```text
上、下、左、右
```

连续动作例子：

```text
机器人电机扭矩：[-1.0, 1.0]
```

### 3.5 Reward 奖励

奖励 `r` 是环境对动作结果的反馈。

奖励设计决定智能体优化方向。

例如：

- 到达目标：+100
- 撞墙：-10
- 每走一步：-1

### 3.6 Policy 策略

策略 `π` 定义智能体如何根据状态选择动作。

确定性策略：

```text
a = π(s)
```

随机策略：

```text
π(a | s) = P(a given s)
```

### 3.7 Value 价值

价值函数衡量某个状态或动作的长期价值。

它不是只看当前奖励，而是看未来累计回报。

### 3.8 Trajectory 轨迹

轨迹是一段交互序列：

```text
τ = (s0, a0, r1, s1, a1, r2, ..., sT)
```

强化学习算法通常从大量轨迹中学习。

## 4. 马尔可夫决策过程 MDP

强化学习通常用马尔可夫决策过程建模。

MDP 是 Markov Decision Process。

### 4.1 MDP 五元组

一个 MDP 通常定义为：

```text
MDP = (S, A, P, R, gamma)
```

其中：

- `S`：状态空间。
- `A`：动作空间。
- `P`：状态转移概率。
- `R`：奖励函数。
- `gamma`：折扣因子。

### 4.2 状态转移概率

状态转移概率：

```text
P(s' | s, a)
```

表示在状态 `s` 执行动作 `a` 后，转移到状态 `s'` 的概率。

如果环境是确定性的，某个 `(s, a)` 只会到达一个确定的 `s'`。

如果环境是随机的，同一个动作可能导致不同结果。

### 4.3 奖励函数

奖励函数可以写为：

```text
R(s, a)
```

或：

```text
R(s, a, s')
```

表示在状态 `s` 执行动作 `a` 后获得的期望奖励。

### 4.4 马尔可夫性

马尔可夫性指：

> 未来只依赖当前状态和动作，而不依赖更早历史。

公式：

```text
P(s_{t+1} | s_t, a_t, s_{t-1}, a_{t-1}, ...)
= P(s_{t+1} | s_t, a_t)
```

如果状态包含足够信息，就可以认为满足马尔可夫性。

### 4.5 完全可观测与部分可观测

如果智能体能观察到完整状态，就是 MDP。

如果只能观察到部分信息，就是 POMDP：

```text
Partially Observable Markov Decision Process
```

例如扑克游戏中，玩家看不到对手手牌，就是部分可观测。

在部分可观测环境中，常用历史信息或 RNN 来弥补状态不足。

## 5. 回报、折扣因子与目标函数

### 5.1 回报 Return

从时间步 `t` 开始的累计回报定义为：

```text
G_t = R_{t+1} + gamma R_{t+2} + gamma^2 R_{t+3} + ...
```

也可以写成：

```text
G_t = sum(k=0..∞) gamma^k R_{t+k+1}
```

### 5.2 折扣因子 gamma

`gamma` 是折扣因子，范围通常为：

```text
0 <= gamma <= 1
```

含义：

- `gamma = 0`：只关注即时奖励。
- `gamma` 接近 1：更重视长期奖励。

### 5.3 为什么需要折扣

折扣因子有几个作用：

- 让无限时域回报收敛。
- 表示未来奖励不确定性更高。
- 控制短期和长期收益权衡。

### 5.4 强化学习目标

强化学习目标是找到最优策略 `π*`，最大化期望回报：

```text
π* = argmax_π E_π[G_0]
```

对于持续任务，也可能最大化平均奖励。

### 5.5 Episodic 和 Continuing

Episodic 任务有终止状态。

例如：

- 一局游戏结束。
- 机器人到达目标。
- 一次对话结束。

Continuing 任务没有自然终止。

例如：

- 数据中心资源调度。
- 长期推荐系统。
- 交通信号控制。

## 6. 策略、价值函数与优势函数

### 6.1 策略函数

策略表示行为方式。

随机策略：

```text
π(a | s)
```

表示在状态 `s` 下选择动作 `a` 的概率。

确定性策略：

```text
a = μ(s)
```

### 6.2 状态价值函数

状态价值函数 `Vπ(s)` 表示从状态 `s` 开始，按照策略 `π` 行动，能获得的期望回报：

```text
Vπ(s) = Eπ[G_t | S_t = s]
```

### 6.3 动作价值函数

动作价值函数 `Qπ(s, a)` 表示在状态 `s` 采取动作 `a`，之后按照策略 `π` 行动的期望回报：

```text
Qπ(s, a) = Eπ[G_t | S_t = s, A_t = a]
```

### 6.4 最优价值函数

最优状态价值：

```text
V*(s) = max_π Vπ(s)
```

最优动作价值：

```text
Q*(s, a) = max_π Qπ(s, a)
```

如果知道 `Q*(s, a)`，最优策略就是：

```text
π*(s) = argmax_a Q*(s, a)
```

### 6.5 优势函数

优势函数衡量某动作比当前状态平均水平好多少：

```text
Aπ(s, a) = Qπ(s, a) - Vπ(s)
```

如果 `Aπ(s, a) > 0`，说明动作 `a` 比平均动作更好。

优势函数常用于策略梯度和 PPO。

## 7. 贝尔曼方程

贝尔曼方程是强化学习的核心。

### 7.1 状态价值贝尔曼方程

对于策略 `π`：

```text
Vπ(s) = sum_a π(a|s) sum_{s'} P(s'|s,a)
        [R(s,a,s') + gamma Vπ(s')]
```

含义：

```text
当前状态价值 = 当前期望奖励 + 折扣后的下一状态价值
```

### 7.2 动作价值贝尔曼方程

```text
Qπ(s,a) = sum_{s'} P(s'|s,a)
          [R(s,a,s') + gamma sum_{a'} π(a'|s') Qπ(s',a')]
```

### 7.3 贝尔曼最优方程

最优状态价值：

```text
V*(s) = max_a sum_{s'} P(s'|s,a)
        [R(s,a,s') + gamma V*(s')]
```

最优动作价值：

```text
Q*(s,a) = sum_{s'} P(s'|s,a)
          [R(s,a,s') + gamma max_{a'} Q*(s',a')]
```

### 7.4 贝尔曼方程的重要性

贝尔曼方程把长期回报分解为：

- 当前奖励。
- 未来价值。

这使得强化学习可以用递推方式学习价值函数。

动态规划、TD、Q-learning、DQN 都依赖贝尔曼思想。

## 8. 动态规划方法

动态规划适用于已知环境模型的情况。

已知模型表示知道：

```text
P(s' | s, a)
R(s, a, s')
```

### 8.1 策略评估

策略评估是计算给定策略 `π` 的价值函数。

迭代更新：

```text
V_{k+1}(s) = sum_a π(a|s) sum_{s'} P(s'|s,a)
             [R(s,a,s') + gamma V_k(s')]
```

重复直到收敛。

### 8.2 策略改进

根据当前价值函数改进策略：

```text
π'(s) = argmax_a sum_{s'} P(s'|s,a)
        [R(s,a,s') + gamma Vπ(s')]
```

### 8.3 策略迭代

策略迭代交替进行：

```text
1. 策略评估。
2. 策略改进。
3. 重复直到策略稳定。
```

### 8.4 价值迭代

价值迭代直接使用贝尔曼最优方程更新：

```text
V_{k+1}(s) = max_a sum_{s'} P(s'|s,a)
             [R(s,a,s') + gamma V_k(s')]
```

收敛后再提取策略。

### 8.5 动态规划的局限

局限：

- 需要已知环境模型。
- 状态空间大时不可行。
- 动作空间大时计算成本高。

现实问题中通常不知道转移概率，因此需要无模型方法。

## 9. 蒙特卡洛方法

蒙特卡洛方法不需要环境模型，通过完整 episode 的实际回报估计价值。

### 9.1 基本思想

运行一条轨迹：

```text
s0, a0, r1, s1, ..., sT
```

episode 结束后，计算每个状态的真实回报 `G_t`，用它更新价值估计。

### 9.2 状态价值更新

```text
V(s) <- V(s) + alpha [G_t - V(s)]
```

其中：

- `G_t` 是实际采样回报。
- `V(s)` 是当前估计。
- `alpha` 是学习率。

### 9.3 First-Visit 和 Every-Visit

First-Visit MC：

- 一个 episode 中某状态第一次出现时更新。

Every-Visit MC：

- 一个 episode 中某状态每次出现都更新。

两者都可以收敛到真实价值。

### 9.4 蒙特卡洛优点

优点：

- 不需要环境模型。
- 直接用真实回报。
- 无 bootstrap 偏差。

### 9.5 蒙特卡洛缺点

缺点：

- 必须等 episode 结束才能更新。
- 不适合无终止任务。
- 方差高。
- 样本效率较低。

## 10. 时序差分学习 TD

TD 是 Temporal Difference Learning，结合了动态规划和蒙特卡洛思想。

### 10.1 TD 的核心思想

TD 不需要等 episode 结束，而是用一步奖励和下一状态价值来更新当前价值。

TD(0) 更新：

```text
V(S_t) <- V(S_t) + alpha [R_{t+1} + gamma V(S_{t+1}) - V(S_t)]
```

### 10.2 TD 误差

TD 误差定义为：

```text
δ_t = R_{t+1} + gamma V(S_{t+1}) - V(S_t)
```

它表示当前估计和 bootstrap 目标之间的差异。

### 10.3 TD 与 MC 的区别

| 对比项 | MC | TD |
| --- | --- | --- |
| 更新时机 | episode 结束后 | 每一步 |
| 是否 bootstrap | 否 | 是 |
| 方差 | 高 | 较低 |
| 偏差 | 较低 | 有 bootstrap 偏差 |
| 适用持续任务 | 不方便 | 适合 |

### 10.4 TD 的优点

优点：

- 在线更新。
- 样本效率较高。
- 适合持续任务。
- 是 Q-learning、SARSA 等算法基础。

## 11. SARSA 与 Q-learning

SARSA 和 Q-learning 是经典表格型强化学习算法。

### 11.1 SARSA

SARSA 名字来自：

```text
S_t, A_t, R_{t+1}, S_{t+1}, A_{t+1}
```

更新公式：

```text
Q(S_t,A_t) <- Q(S_t,A_t)
    + alpha [R_{t+1} + gamma Q(S_{t+1},A_{t+1}) - Q(S_t,A_t)]
```

SARSA 是 on-policy 算法，因为它用当前策略实际选择的下一个动作更新。

### 11.2 Q-learning

Q-learning 更新公式：

```text
Q(S_t,A_t) <- Q(S_t,A_t)
    + alpha [R_{t+1} + gamma max_a Q(S_{t+1},a) - Q(S_t,A_t)]
```

它使用下一状态最大 Q 值作为目标。

Q-learning 是 off-policy 算法，因为它学习的是贪心最优策略，但行为策略可以是 epsilon-greedy。

### 11.3 On-policy 与 Off-policy

On-policy：

```text
学习的策略 = 采样数据的策略
```

Off-policy：

```text
学习的策略 != 采样数据的策略
```

Q-learning 可以一边用探索策略采样，一边学习最优贪心策略。

### 11.4 SARSA 和 Q-learning 的区别

SARSA 更保守，因为它考虑实际探索动作的风险。

Q-learning 更激进，因为它假设下一步会选择最优动作。

经典 cliff walking 中：

- SARSA 会学到远离悬崖的安全路径。
- Q-learning 会学到贴近悬崖的最短路径。

### 11.5 表格方法局限

表格方法需要为每个状态-动作对存储 Q 值。

如果状态空间巨大或连续，就不可行。

因此需要函数逼近。

## 12. 探索与利用

探索与利用是强化学习的核心矛盾。

### 12.1 利用

利用是选择当前认为最好的动作。

例如：

```text
a = argmax_a Q(s,a)
```

优点：

- 当前收益高。

缺点：

- 可能错过更好的未知动作。

### 12.2 探索

探索是尝试不确定或未知动作。

优点：

- 发现更优策略。

缺点：

- 可能短期收益低。
- 现实任务中可能有风险。

### 12.3 Epsilon-Greedy

epsilon-greedy 策略：

```text
以 1 - epsilon 概率选择当前最优动作。
以 epsilon 概率随机选择动作。
```

常用做法是 epsilon 随训练递减：

```text
初期多探索，后期多利用。
```

### 12.4 Softmax 探索

根据动作价值生成概率：

```text
π(a|s) = exp(Q(s,a)/τ) / sum_b exp(Q(s,b)/τ)
```

温度 `τ` 越高，动作越随机；`τ` 越低，越偏向高价值动作。

### 12.5 UCB

UCB 是 Upper Confidence Bound，上置信界。

思想：

```text
选择价值高且不确定性高的动作。
```

常用于多臂老虎机问题。

### 12.6 熵正则

在策略优化中加入熵奖励：

```text
Loss = 原目标 - beta * entropy(policy)
```

熵越大，策略越随机。

熵正则鼓励探索，防止策略过早坍缩。

## 13. 函数逼近与深度强化学习

### 13.1 为什么需要函数逼近

真实问题状态空间很大：

- 图像像素状态。
- 机器人连续状态。
- 推荐系统用户上下文。

无法为每个状态动作对建表。

因此用函数逼近：

```text
Q(s,a; θ)
V(s; θ)
π(a|s; θ)
```

### 13.2 线性函数逼近

用特征向量表示状态动作：

```text
Q(s,a; w) = w^T φ(s,a)
```

优点：

- 简单。
- 理论较清晰。

缺点：

- 表达能力有限。

### 13.3 神经网络函数逼近

深度强化学习用神经网络表示价值函数或策略。

例如 DQN：

```text
输入状态 s
输出每个动作的 Q 值
```

### 13.4 深度强化学习不稳定原因

原因包括：

- 数据强相关。
- 目标值不断变化。
- bootstrap 误差传播。
- 非线性函数逼近不稳定。
- 探索导致数据分布变化。

DQN 使用经验回放和目标网络缓解这些问题。

## 14. DQN 及其改进

DQN 是 Deep Q-Network，是深度强化学习的经典算法。

### 14.1 DQN 核心思想

DQN 使用神经网络近似 Q 函数：

```text
Q(s, a; θ)
```

对于离散动作，网络通常输入状态，输出所有动作的 Q 值。

### 14.2 DQN 目标

TD 目标：

```text
y = r + gamma max_{a'} Q(s', a'; θ^-)
```

损失：

```text
L(θ) = (y - Q(s, a; θ))^2
```

其中 `θ^-` 是目标网络参数。

### 14.3 经验回放 Experience Replay

智能体把交互数据存入 replay buffer：

```text
(s, a, r, s', done)
```

训练时随机采样 mini-batch。

作用：

- 打破样本相关性。
- 提高样本利用率。
- 稳定训练。

### 14.4 目标网络 Target Network

DQN 使用两个网络：

- Online network：当前训练网络。
- Target network：计算 TD 目标。

目标网络每隔一段时间从 online network 同步参数。

作用：

- 减少目标值快速变化。
- 稳定训练。

### 14.5 Double DQN

DQN 中的 `max` 操作容易高估 Q 值。

Double DQN 将动作选择和动作评估分开：

```text
a* = argmax_a Q_online(s', a)
y = r + gamma Q_target(s', a*)
```

这样能缓解过估计。

### 14.6 Dueling DQN

Dueling DQN 将 Q 值分解为：

```text
Q(s,a) = V(s) + A(s,a)
```

网络分成两条分支：

- 状态价值分支。
- 优势函数分支。

适合很多动作在某些状态下差异不大的场景。

### 14.7 Prioritized Experience Replay

优先经验回放根据 TD 误差采样：

```text
TD error 大的样本更常被采样。
```

优点：

- 更关注学习价值大的样本。

需要重要性采样权重修正偏差。

### 14.8 Rainbow DQN

Rainbow 将多种 DQN 改进结合：

- Double DQN。
- Dueling Network。
- Prioritized Replay。
- Multi-step Learning。
- Distributional RL。
- Noisy Networks。

## 15. 策略梯度方法

价值方法先学 Q 或 V，再间接得到策略。策略梯度直接优化策略参数。

### 15.1 策略参数化

随机策略：

```text
πθ(a | s)
```

目标：

```text
J(θ) = E_{τ~πθ}[G(τ)]
```

### 15.2 Policy Gradient Theorem

策略梯度基本形式：

```text
∇θ J(θ) = E[∇θ log πθ(a|s) Qπ(s,a)]
```

直观理解：

- 如果某动作带来高回报，就提高它的概率。
- 如果某动作带来低回报，就降低它的概率。

### 15.3 REINFORCE

REINFORCE 是经典蒙特卡洛策略梯度算法。

更新方向：

```text
∇θ log πθ(a_t|s_t) G_t
```

### 15.4 Baseline

为了降低方差，可以减去 baseline：

```text
∇θ log πθ(a_t|s_t) [G_t - b(s_t)]
```

常用 baseline 是状态价值函数 `V(s)`。

此时：

```text
G_t - V(s_t)
```

近似优势函数。

### 15.5 策略梯度优点

优点：

- 可处理连续动作。
- 可学习随机策略。
- 直接优化目标。

### 15.6 策略梯度缺点

缺点：

- 方差高。
- 样本效率低。
- 训练不稳定。

Actor-Critic 用价值函数降低方差。

## 16. Actor-Critic 方法

Actor-Critic 结合策略学习和价值学习。

### 16.1 Actor 和 Critic

Actor：

```text
策略网络，负责选择动作。
```

Critic：

```text
价值网络，负责评估动作好坏。
```

### 16.2 基本更新

Critic 学习价值函数：

```text
V(s) 或 Q(s,a)
```

Actor 根据 Critic 提供的优势信号更新策略：

```text
∇θ log πθ(a|s) A(s,a)
```

### 16.3 Advantage Actor-Critic

优势函数：

```text
A(s,a) = Q(s,a) - V(s)
```

实践中常用 TD 误差近似：

```text
δ = r + gamma V(s') - V(s)
```

### 16.4 A2C

A2C 是 Advantage Actor-Critic 的同步版本。

多个环境并行采样，然后同步更新模型。

### 16.5 A3C

A3C 是 Asynchronous Advantage Actor-Critic。

多个 worker 异步与环境交互并更新全局模型。

异步采样能增加数据多样性。

### 16.6 Actor-Critic 优缺点

优点：

- 比纯策略梯度方差低。
- 可处理连续动作。
- 适合深度强化学习。

缺点：

- Actor 和 Critic 互相影响，训练可能不稳定。
- Critic 估计偏差会影响 Actor。
- 参数较多。

---

## 17. PPO、DDPG、TD3、SAC

### 17.1 PPO

PPO 是 Proximal Policy Optimization，当前最常用的策略优化算法之一。

PPO 的目标是：

```text
每次策略更新不要离旧策略太远。
```

### 17.2 PPO Clipped Objective

定义概率比：

```text
r_t(θ) = πθ(a_t|s_t) / πθ_old(a_t|s_t)
```

PPO clipped 目标：

```text
L = E[min(
    r_t(θ) A_t,
    clip(r_t(θ), 1-epsilon, 1+epsilon) A_t
)]
```

clip 限制策略更新幅度，提升稳定性。

### 17.3 PPO 优点

优点：

- 稳定。
- 实现相对简单。
- 适用范围广。
- 对超参数不算特别敏感。

缺点：

- 样本效率不如一些 off-policy 算法。
- 训练仍然需要大量交互。

### 17.4 DDPG

DDPG 是 Deep Deterministic Policy Gradient。

用于连续动作空间。

特点：

- Actor 输出确定性动作。
- Critic 学习 Q 函数。
- 使用经验回放。
- 使用目标网络。

### 17.5 TD3

TD3 是 Twin Delayed DDPG，改进 DDPG。

核心改进：

- 双 Critic，取较小 Q 值，缓解过估计。
- 延迟策略更新。
- 目标动作加平滑噪声。

TD3 通常比 DDPG 更稳定。

### 17.6 SAC

SAC 是 Soft Actor-Critic。

它最大化：

```text
期望回报 + 熵
```

目标：

```text
不仅要奖励高，还要策略有足够随机性。
```

### 17.7 SAC 优点

优点：

- 探索能力强。
- 稳定性好。
- 样本效率较高。
- 连续控制任务中表现强。

### 17.8 算法选择建议

离散动作：

- DQN 系列。
- PPO。

连续动作：

- PPO。
- SAC。
- TD3。

如果追求稳定和通用，PPO 是常见选择。

如果连续控制且重视样本效率，SAC 常是强基线。

## 18. 奖励设计与稀疏奖励

### 18.1 奖励设计的重要性

奖励函数决定智能体学习目标。

如果奖励设计不好，智能体会学到错误行为。

### 18.2 奖励稀疏

稀疏奖励指大多数时间没有奖励，只有完成目标时才有奖励。

例如：

```text
到达终点 +1，其余所有步骤 0。
```

问题：

- 智能体很难知道哪些动作有帮助。
- 探索成本高。
- 学习很慢。

### 18.3 奖励塑形 Reward Shaping

奖励塑形通过添加中间奖励帮助学习。

例如迷宫：

- 靠近目标给小正奖励。
- 远离目标给小负奖励。

注意：奖励塑形可能改变最优策略，需要谨慎。

### 18.4 Potential-based Reward Shaping

一种较安全的奖励塑形：

```text
F(s,a,s') = gamma Φ(s') - Φ(s)
```

这种形式在理论上可以保持最优策略不变。

### 18.5 奖励黑客 Reward Hacking

智能体可能利用奖励漏洞，获得高奖励但不符合真实目标。

例如：

- 清洁机器人反复制造垃圾再清理。
- 游戏智能体卡 bug 刷分。
- 推荐系统只追求点击，导致低质量内容泛滥。

奖励设计必须结合业务指标和约束。

## 19. 模型无关与模型有关强化学习

### 19.1 Model-Free

模型无关方法不显式学习环境转移模型。

它直接学习：

- 价值函数。
- 策略。

代表：

- Q-learning。
- DQN。
- PPO。
- SAC。

优点：

- 不需要建模复杂环境。
- 实现相对直接。

缺点：

- 样本效率低。

### 19.2 Model-Based

模型有关方法学习或使用环境模型：

```text
P(s'|s,a), R(s,a)
```

然后用模型规划。

优点：

- 样本效率高。
- 可以做规划和模拟。

缺点：

- 模型误差会影响策略。
- 高维复杂环境建模困难。

### 19.3 Planning

规划使用模型预测未来结果，然后选择动作。

例如：

- 动态规划。
- MCTS。
- MPC。

AlphaGo 使用蒙特卡洛树搜索结合神经网络，就是 model-based planning 的代表之一。

### 19.4 Dyna

Dyna 框架结合真实经验和模型生成经验：

```text
真实交互更新模型和价值。
用学习到的模型模拟经验继续训练。
```

## 20. 离线强化学习与离策略学习

### 20.1 Off-policy

Off-policy 指学习目标策略和采样行为策略不同。

例如 Q-learning：

```text
用 epsilon-greedy 采样。
学习 greedy 最优策略。
```

### 20.2 Offline RL

离线强化学习只使用已有数据集训练策略，不能和环境继续交互。

数据集：

```text
D = {(s, a, r, s')}
```

来自历史策略、人工操作或日志。

### 20.3 离线 RL 的难点

最大问题是分布外动作。

如果模型学到某个动作很好，但历史数据中很少出现这个动作，Q 值估计可能不可靠。

策略一旦选择训练数据外动作，可能出现严重错误。

### 20.4 常见思路

离线 RL 常用保守策略：

- 限制新策略不要偏离行为策略太远。
- 惩罚分布外动作。
- 学习保守 Q 值。
- 使用行为克隆作为约束。

代表算法：

- BCQ。
- CQL。
- IQL。

### 20.5 与监督学习的关系

离线 RL 比监督学习难，因为动作会影响未来状态和奖励。

但在某些场景，可以先用行为克隆：

```text
学习历史专家动作。
```

作为强基线。

## 22. 强化学习中的稳定性与评估

### 22.1 随机种子很重要

强化学习结果对随机种子非常敏感。

评估时应使用多个随机种子：

```text
mean ± std
```

不要只报告一次训练结果。

### 22.2 训练曲线

应监控：

- episode return。
- episode length。
- loss。
- entropy。
- value estimate。
- Q value scale。
- success rate。

### 22.3 评估策略

训练时可能有探索噪声，评估时通常关闭探索或使用确定策略。

例如：

- DQN 评估时 epsilon 设置很小或为 0。
- PPO 评估时使用均值动作或最大概率动作。

### 22.4 样本效率

强化学习除了最终性能，还要看样本效率：

```text
达到某个分数需要多少环境交互步。
```

现实场景中，交互成本可能非常高。

### 22.5 安全性

强化学习智能体探索时可能做出危险动作。

真实系统中需要：

- 仿真训练。
- 安全约束。
- 人工审核。
- 离线评估。
- 灰度上线。

## 23. Python 实战

### 23.1 一个简单 GridWorld 环境

```python
import numpy as np


class GridWorld:
    def __init__(self, size=4):
        self.size = size
        self.start = (0, 0)
        self.goal = (size - 1, size - 1)
        self.state = self.start

    def reset(self):
        self.state = self.start
        return self._state_to_index(self.state)

    def step(self, action):
        row, col = self.state

        if action == 0:
            row -= 1
        elif action == 1:
            row += 1
        elif action == 2:
            col -= 1
        elif action == 3:
            col += 1

        row = min(max(row, 0), self.size - 1)
        col = min(max(col, 0), self.size - 1)

        self.state = (row, col)

        done = self.state == self.goal
        reward = 10 if done else -1

        return self._state_to_index(self.state), reward, done

    def _state_to_index(self, state):
        return state[0] * self.size + state[1]
```

动作定义：

```text
0: 上
1: 下
2: 左
3: 右
```

### 23.2 Q-learning 实现

```python
import numpy as np


def q_learning(
    env,
    n_states,
    n_actions,
    episodes=1000,
    alpha=0.1,
    gamma=0.99,
    epsilon=0.1,
):
    q_table = np.zeros((n_states, n_actions))

    for _ in range(episodes):
        state = env.reset()
        done = False

        while not done:
            if np.random.random() < epsilon:
                action = np.random.randint(n_actions)
            else:
                action = np.argmax(q_table[state])

            next_state, reward, done = env.step(action)

            td_target = reward + gamma * np.max(q_table[next_state]) * (not done)
            td_error = td_target - q_table[state, action]
            q_table[state, action] += alpha * td_error

            state = next_state

    return q_table
```

### 23.3 训练和查看策略

```python
env = GridWorld(size=4)
n_states = 16
n_actions = 4

q_table = q_learning(
    env,
    n_states=n_states,
    n_actions=n_actions,
    episodes=2000,
    alpha=0.1,
    gamma=0.99,
    epsilon=0.1,
)

policy = np.argmax(q_table, axis=1).reshape(4, 4)
print(policy)
```

### 23.4 SARSA 实现

```python
def epsilon_greedy(q_table, state, epsilon):
    if np.random.random() < epsilon:
        return np.random.randint(q_table.shape[1])
    return np.argmax(q_table[state])


def sarsa(
    env,
    n_states,
    n_actions,
    episodes=1000,
    alpha=0.1,
    gamma=0.99,
    epsilon=0.1,
):
    q_table = np.zeros((n_states, n_actions))

    for _ in range(episodes):
        state = env.reset()
        action = epsilon_greedy(q_table, state, epsilon)
        done = False

        while not done:
            next_state, reward, done = env.step(action)
            next_action = epsilon_greedy(q_table, next_state, epsilon)

            td_target = reward + gamma * q_table[next_state, next_action] * (not done)
            td_error = td_target - q_table[state, action]
            q_table[state, action] += alpha * td_error

            state = next_state
            action = next_action

    return q_table
```

### 23.6 DQN 训练伪代码

```python
for step in range(total_steps):
    action = epsilon_greedy(q_network, state)
    next_state, reward, done = env.step(action)

    replay_buffer.add(state, action, reward, next_state, done)
    state = next_state

    if replay_buffer.size() > warmup_steps:
        batch = replay_buffer.sample(batch_size)

        with torch.no_grad():
            next_q = target_network(batch.next_state).max(dim=1).values
            target = batch.reward + gamma * next_q * (1 - batch.done)

        q = q_network(batch.state).gather(1, batch.action)
        loss = mse_loss(q, target)

        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

    if step % target_update_interval == 0:
        target_network.load_state_dict(q_network.state_dict())
```

## 25. 常见误区与排查清单

### 25.1 常见误区

#### 误区一：强化学习就是深度学习加奖励

强化学习核心是序列决策、长期回报、探索和环境交互，不只是换一个损失函数。

#### 误区二：有奖励就适合强化学习

如果问题可以直接监督学习解决，通常先用监督学习。强化学习成本高、稳定性差，不应滥用。

#### 误区三：训练曲线上升就代表策略好

训练奖励可能被奖励漏洞利用。必须在独立评估环境和真实业务指标上验证。

#### 误区四：仿真成功就能直接上线

仿真和现实可能存在差距。机器人和自动驾驶中尤其需要 sim-to-real 处理。

#### 误区五：只跑一个随机种子就报告结果

强化学习方差很大，必须多种子评估。

### 25.2 排查清单

训练不稳定时检查：

- 奖励尺度是否过大？
- 学习率是否过高？
- gamma 是否合理？
- 探索是否足够？
- replay buffer 是否正常？
- target network 是否更新过快？
- Q 值是否爆炸？
- advantage 是否标准化？
- 随机种子结果是否一致？
- 环境终止逻辑是否正确？

效果不好时检查：

- 奖励设计是否对齐目标？
- 是否存在奖励黑客？
- 观察状态是否包含足够信息？
- 动作空间是否设计合理？
- 是否需要更强探索？
- 是否需要课程学习？
- 是否纯监督或模仿学习更合适？

## 26. 常见面试题

### 26.1 什么是强化学习？

强化学习是智能体通过与环境交互，根据奖励反馈学习决策策略，以最大化长期累计回报的机器学习方法。

### 26.2 强化学习和监督学习有什么区别？

监督学习有明确标签，学习输入到标签的映射；强化学习没有每一步正确动作标签，只有奖励反馈，并且动作会影响未来状态和长期收益。

### 26.3 什么是 MDP？

MDP 是马尔可夫决策过程，通常由状态空间、动作空间、转移概率、奖励函数和折扣因子组成，用于建模强化学习问题。

### 26.4 什么是马尔可夫性？

马尔可夫性指未来状态只依赖当前状态和动作，而不依赖更早历史。当前状态应包含决策所需的全部信息。

### 26.5 什么是价值函数？

价值函数衡量从某个状态或状态动作对出发，在某个策略下能获得的期望累计回报。包括状态价值 `V(s)` 和动作价值 `Q(s,a)`。

### 26.6 贝尔曼方程是什么？

贝尔曼方程把当前价值分解为当前奖励和下一状态价值，是强化学习价值递推的基础。

### 26.7 Q-learning 和 SARSA 有什么区别？

SARSA 是 on-policy，用实际选择的下一个动作更新；Q-learning 是 off-policy，用下一状态最大 Q 值更新，学习贪心最优策略。

### 26.8 On-policy 和 Off-policy 有什么区别？

On-policy 学习的策略和采样数据的策略相同；Off-policy 学习目标策略和行为采样策略可以不同。

### 26.9 DQN 为什么需要经验回放？

经验回放可以打破样本相关性，提高样本利用率，使训练更稳定。

### 26.10 DQN 为什么需要目标网络？

目标网络用于计算相对稳定的 TD 目标，避免在线网络参数快速变化导致目标值不稳定。

### 26.11 Double DQN 解决什么问题？

Double DQN 将动作选择和动作评估分离，缓解 DQN 中 `max` 操作导致的 Q 值过估计。

### 26.12 策略梯度的核心思想是什么？

策略梯度直接优化策略参数，使高回报动作概率增加，低回报动作概率降低。

### 26.13 Actor-Critic 是什么？

Actor-Critic 包含 Actor 和 Critic。Actor 学习策略，Critic 学习价值函数并评价 Actor 的动作，用于降低策略梯度方差。

### 26.14 PPO 为什么稳定？

PPO 通过 clipped objective 限制新旧策略概率比，避免策略单次更新过大，从而提高训练稳定性。

### 26.15 DDPG、TD3、SAC 适合什么动作空间？

它们主要用于连续动作空间。DDPG 是确定性策略梯度，TD3 改进 DDPG 稳定性，SAC 加入最大熵目标增强探索。

### 26.16 什么是探索与利用？

利用是选择当前认为最优的动作，探索是尝试未知动作以发现潜在更优策略。强化学习需要在二者之间权衡。

### 26.17 奖励塑形有什么风险？

奖励塑形可能改变智能体优化目标，使其学到不符合真实目标的行为，甚至产生奖励黑客。

### 26.18 Model-free 和 Model-based 有什么区别？

Model-free 不显式学习环境模型，直接学习价值或策略；Model-based 学习或使用环境转移模型进行规划，样本效率通常更高但受模型误差影响。

### 26.19 离线强化学习为什么难？

离线强化学习不能与环境交互，只能用历史数据。策略可能选择数据分布外动作，而这些动作的价值估计不可靠，容易导致严重过估计。

### 26.20 强化学习落地最重要的挑战是什么？

主要挑战包括奖励设计、样本效率、安全探索、训练稳定性、离线评估和仿真到现实差距。

#### 典型场景拆解

在真实场景中，$topic 通常会经历“需求出现、方案选择、实现落地、问题暴露、持续优化”几个阶段。需求出现时，要先判断这个需求属于基础能力、性能优化、体验改进、可靠性建设还是长期架构演进。不同类型的需求对方案的评价标准不同：基础能力看正确性，性能优化看指标，体验改进看路径是否顺滑，可靠性建设看故障时能否降级和恢复，架构演进看未来变化是否容易吸收。

方案选择阶段，最容易犯的错误是直接套用熟悉工具。更稳妥的方式是列出约束：数据规模、时延要求、资源预算、团队熟悉度、运维能力、安全要求、可测试性和长期维护成本。只有把约束列清楚，才能解释为什么选择当前方案。否则方案看似高级，实际可能只是增加了复杂度。

实现落地阶段，要把 $a 和 $b 拆成可验证的小步骤。每一步都应该有明确的输入、输出和检查方式。对学习笔记而言，这意味着不能只有大段概念，还应该补充流程图式的文字描述、伪代码、命令示例、参数解释、错误现象和排查路径。这样以后复习时，笔记不仅能帮助理解，也能直接指导实践。

问题暴露阶段，要优先区分“理解错误、实现错误、环境错误、数据错误、依赖错误、边界条件错误”。很多复杂问题之所以难排，是因为一开始就把问题归因到错误层级。例如把配置问题当成算法问题，把权限问题当成代码问题，把数据分布变化当成模型失效，把硬件噪声当成软件逻辑错误。好的排查顺序应该从可观测事实开始，而不是从猜测开始。

持续优化阶段，不应只追求把当前问题压下去，还要沉淀成规则。比如记录触发条件、影响范围、定位方法、最终修复、预防措施和可监控指标。这样下一次出现类似问题时，团队可以复用经验，而不是重新从零排查。

#### 常见误区与纠偏

第一个误区是只记结论，不记前提。$topic 中很多结论都是有条件的：适用于小规模，不一定适用于大规模；适用于离线处理，不一定适用于实时系统；适用于单机环境，不一定适用于分布式环境；适用于教学案例，不一定适用于生产项目。纠偏方法是在每个重要结论后面补一句“适用条件”和“不适用情况”。

第二个误区是只关注工具，不关注模型。工具会变化，模型更稳定。无论工具名称如何变化，底层仍然要解决输入建模、状态管理、资源调度、错误恢复、性能约束和质量验证这些问题。学习 $topic 时，应该把工具用法和底层模型分开记录：工具命令解决“怎么做”，底层模型解释“为什么这样做”。

第三个误区是没有验证意识。很多笔记写得很完整，但没有说明如何确认自己做对了。对于 $category 相关主题，验证至少应包含最小样例、边界样例、异常样例和性能样例。最小样例证明流程跑通，边界样例证明理解完整，异常样例证明系统可恢复，性能样例证明方案在目标规模下仍然可用。

第四个误区是忽略可维护性。短期学习时，能跑通就容易产生掌握的错觉；长期使用时，命名、分层、注释、测试、日志、版本管理和文档才会决定知识能否转化为稳定能力。扩充 $topic 笔记时，应把“如何写得清楚、如何排查、如何交接、如何复盘”也纳入内容。

#### 学习与实践建议
