# 机器学习、深度学习与数据处理专业术语学习笔记

## 0. 阅读说明

这份笔记用于系统梳理机器学习、深度学习与数据处理相关的常见专业术语。每个术语尽量给出中文名称、英文名称、英文简称和解释。实际学习时不要只背缩写，更要知道它属于哪个环节、解决什么问题、和相近概念有什么区别。

可以把一个完整 AI 项目理解为下面的链路：

```text
业务问题 -> 数据采集 -> 数据清洗 -> 特征工程 -> 模型训练 -> 模型评估 -> 模型部署 -> 监控迭代
```

机器学习术语通常围绕“数据、特征、模型、损失、优化、评估、部署”展开；深度学习术语更多围绕“神经网络结构、训练技巧、表示学习、硬件加速、大模型”展开；数据处理术语则围绕“数据类型、存储、清洗、转换、统计、质量、管道”展开。

## 2. 机器学习基本任务术语

| 中文术语 | 英文全称 | 简称 | 解释 |
|---|---|---|---|
| 监督学习 | Supervised Learning | SL | 使用带标签数据训练模型，学习输入到输出的映射。 |
| 无监督学习 | Unsupervised Learning | UL | 使用无标签数据发现结构、聚类、分布或低维表示。 |
| 半监督学习 | Semi-Supervised Learning | SSL | 同时使用少量有标签数据和大量无标签数据训练模型。 |
| 自监督学习 | Self-Supervised Learning | SSL | 从数据本身构造监督信号，例如遮盖预测、对比学习。 |
| 强化学习 | Reinforcement Learning | RL | 基于状态、动作、奖励和策略进行序列决策学习。 |
| 分类 | Classification | - | 预测离散类别，例如是否违约、图片类别。 |
| 二分类 | Binary Classification | - | 分类结果只有两个类别，例如正类和负类。 |
| 多分类 | Multi-Class Classification | - | 样本属于多个类别中的一个。 |
| 多标签分类 | Multi-Label Classification | - | 一个样本可以同时属于多个标签。 |
| 回归 | Regression | - | 预测连续数值，例如房价、温度、销量。 |
| 聚类 | Clustering | - | 将无标签样本按相似性分组。 |
| 降维 | Dimensionality Reduction | DR | 将高维数据映射到低维空间，保留主要信息。 |
| 密度估计 | Density Estimation | - | 估计数据的概率分布。 |
| 异常检测 | Anomaly Detection | AD | 识别与大多数样本明显不同的异常样本。 |
| 离群点检测 | Outlier Detection | OD | 检测偏离正常分布的数据点，常与异常检测接近。 |
| 排序学习 | Learning to Rank | LTR | 学习对候选项进行排序，常用于搜索和推荐。 |
| 序列建模 | Sequence Modeling | - | 处理具有顺序依赖的数据，如文本、语音、时间序列。 |
| 时间序列预测 | Time Series Forecasting | TSF | 根据历史序列预测未来数值。 |
| 生成建模 | Generative Modeling | - | 学习数据分布并生成新样本。 |
| 表征学习 | Representation Learning | - | 学习更适合下游任务的特征表示。 |
| 迁移学习 | Transfer Learning | TL | 将一个任务中学到的知识迁移到另一个任务。 |
| 多任务学习 | Multi-Task Learning | MTL | 一个模型同时学习多个相关任务。 |
| 在线学习 | Online Learning | - | 模型随数据流持续更新。 |
| 主动学习 | Active Learning | AL | 模型主动选择最有价值的样本请求标注。 |
| 联邦学习 | Federated Learning | FL | 数据不离开本地，多方协作训练模型。 |
| 元学习 | Meta Learning | - | 学习如何学习，使模型能快速适应新任务。 |
| 小样本学习 | Few-Shot Learning | FSL | 使用极少样本完成新任务。 |
| 零样本学习 | Zero-Shot Learning | ZSL | 不使用目标类别训练样本也能进行预测。 |
| 单样本学习 | One-Shot Learning | OSL | 每个类别只用一个样本进行学习或识别。 |

## 3. 数据集与样本术语

| 中文术语 | 英文全称 | 简称 | 解释 |
|---|---|---|---|
| 数据集 | Dataset | - | 用于训练、验证、测试或分析的数据集合。 |
| 样本 | Sample / Instance / Example | - | 数据集中的一条记录或一个观测对象。 |
| 特征 | Feature | - | 描述样本的输入变量。 |
| 标签 | Label | - | 监督学习中的目标输出或真实类别。 |
| 目标变量 | Target Variable | Target | 模型需要预测的变量。 |
| 因变量 | Dependent Variable | - | 统计建模中被解释或预测的变量。 |
| 自变量 | Independent Variable | - | 用来解释或预测目标的变量。 |
| 观测值 | Observation | - | 一次数据观测或一行样本。 |
| 维度 | Dimension | Dim | 特征空间中的特征数量或向量长度。 |
| 特征空间 | Feature Space | - | 由所有特征维度组成的空间。 |
| 数据点 | Data Point | - | 特征空间中的一个样本。 |
| 训练集 | Training Set | Train Set | 用于拟合模型参数的数据。 |
| 验证集 | Validation Set | Val Set | 用于选择模型、调参和早停的数据。 |
| 测试集 | Test Set | - | 用于最终评估模型泛化能力的数据。 |
| 留出法 | Hold-Out Method | - | 将数据划分为训练集、验证集和测试集的评估方法。 |
| 交叉验证 | Cross-Validation | CV | 多次划分数据训练和验证，降低评估偶然性。 |
| K 折交叉验证 | K-Fold Cross-Validation | K-Fold CV | 将数据分成 K 份，轮流用一份验证、其余训练。 |
| 分层抽样 | Stratified Sampling | - | 按类别比例抽样，保持不同数据集的类别分布一致。 |
| 数据泄漏 | Data Leakage | - | 训练过程中意外使用了测试集或未来信息，导致评估虚高。 |
| 类别不平衡 | Class Imbalance | - | 不同类别样本数量差异很大。 |
| 少数类 | Minority Class | - | 类别不平衡中样本较少的类别。 |
| 多数类 | Majority Class | - | 类别不平衡中样本较多的类别。 |
| 噪声样本 | Noisy Sample | - | 标签错误、测量误差或质量较差的样本。 |
| 离群点 | Outlier | - | 明显偏离整体分布的数据点。 |
| 缺失值 | Missing Value | NA / Null | 未采集到、不可用或为空的数据值。 |
| 标注 | Annotation / Labeling | - | 人工或自动为数据添加标签。 |
| 标注一致性 | Annotation Consistency | - | 多个标注者或多次标注结果的一致程度。 |
| 金标准数据 | Gold Standard Dataset | - | 高质量、可信赖、常用于评估的数据集。 |
| 基准数据集 | Benchmark Dataset | - | 用于比较不同方法性能的标准数据集。 |
| 开放数据集 | Open Dataset | - | 公开可用的数据集。 |
| 合成数据 | Synthetic Data | - | 通过模拟、生成模型或规则构造的数据。 |
| 数据增强 | Data Augmentation | DA | 通过变换或生成增加训练样本多样性。 |
| 数据蒸馏 | Data Distillation | DD | 将大规模训练数据中的关键信息压缩到更小、更高价值的数据表示或样本集合中，用较少数据达到接近原数据训练效果。 |
| 数据集蒸馏 | Dataset Distillation | DD | 学习一个小型合成数据集，使模型在该小数据集上训练后能接近在完整数据集上训练的效果。 |
| 数据凝缩 | Dataset Condensation | DC | 与数据集蒸馏接近，强调把大数据集凝缩为少量代表性样本或合成样本。 |
| 样本选择 | Sample Selection | - | 从原始数据中挑选最有代表性、最有训练价值或最符合目标分布的样本。 |
| 核心集选择 | Core-Set Selection | Coreset | 从大数据集中选择一个小子集，使其尽量代表完整数据集。 |
| 软标签数据 | Soft-Labeled Data | - | 标签不是单一类别，而是概率分布或模型输出分数的数据，常用于蒸馏和弱监督。 |
| 伪标签 | Pseudo Label | - | 用模型为无标签数据自动生成的标签，常用于半监督学习和数据扩充。 |

## 6. 数据清洗与预处理术语

| 中文术语 | 英文全称 | 简称 | 解释 |
|---|---|---|---|
| 数据清洗 | Data Cleaning | - | 修复缺失、异常、重复、错误和不一致数据。 |
| 数据预处理 | Data Preprocessing | - | 建模前对数据进行清洗、转换、编码、缩放等处理。 |
| 缺失值处理 | Missing Value Handling | - | 对空值进行删除、填充、插值或建模处理。 |
| 均值填充 | Mean Imputation | - | 用均值替换缺失值。 |
| 中位数填充 | Median Imputation | - | 用中位数替换缺失值，对异常值更稳健。 |
| 众数填充 | Mode Imputation | - | 用出现频率最高的值替换缺失值。 |
| 插值 | Interpolation | - | 根据相邻数据估计缺失值，常用于时间序列。 |
| 删除缺失值 | Drop Missing Values | - | 删除含缺失值的样本或字段。 |
| 异常值处理 | Outlier Handling | - | 检测、删除、截断或修正离群值。 |
| 缩尾处理 | Winsorization | - | 将极端值限制在指定分位数范围内。 |
| 截断 | Clipping | - | 将数值限制在上下界之间。 |
| 去重 | Deduplication | - | 删除重复记录。 |
| 数据标准化 | Standardization | Z-Score | 将数据变换为均值 0、标准差 1。 |
| 数据归一化 | Normalization | - | 将数据缩放到固定范围，如 0 到 1。 |

### 6.1 数据清洗概念详解

数据清洗，英文是 Data Cleaning，也常被放在 Data Preprocessing 数据预处理的大范围内。它的核心目标是把原始数据中“不可靠、不一致、不可直接建模”的部分处理掉，使数据达到可以分析、训练和评估的质量。

原始数据通常不会天然干净。真实业务数据可能来自用户填写、系统日志、传感器、爬虫、第三方接口、数据库同步或人工标注，其中经常出现缺失值、重复记录、格式不统一、异常值、错误标签、单位混乱、时间错位、字段含义变化等问题。如果这些问题不处理，模型学到的可能不是业务规律，而是数据采集和处理过程中的噪声。

数据清洗通常解决的问题：

- 缺失值：例如年龄为空、价格缺失、传感器断点。
- 重复值：例如同一订单被重复写入，同一用户多条重复记录。
- 异常值：例如年龄为 999、价格为负数、温度超出合理范围。
- 格式不一致：例如日期同时存在 `2026-06-21`、`2026/6/21`、`21-06-2026`。
- 单位不一致：例如长度同时使用米、厘米、毫米。
- 类别不一致：例如“北京”“北京市”“beijing”表示同一地区。
- 标签错误：例如垃圾邮件被标成正常邮件。
- 数据类型错误：例如数值字段被读成字符串。
- 时间穿越：例如训练数据中出现预测时不可能知道的未来信息。

常见数据清洗方法：

| 清洗对象 | 常见方法 | 注意事项 |
|---|---|---|
| 缺失值 | 删除、均值填充、中位数填充、众数填充、插值、模型预测填充 | 删除可能损失样本；填充可能引入偏差。 |
| 重复值 | 按主键去重、按多字段匹配去重、保留最新记录 | 要先定义什么叫“重复”，否则可能误删有效样本。 |
| 异常值 | 业务规则过滤、分位数截断、IQR 检测、Z-score 检测 | 异常值不一定是错误，也可能是重要信号。 |
| 格式问题 | 日期解析、字符串清理、大小写统一、空格清除 | 要保留原始字段，方便回溯。 |
| 类别问题 | 同义类别合并、枚举值标准化、低频类别归并 | 类别合并要结合业务含义。 |
| 标签问题 | 人工复核、多标注者一致性检查、噪声标签检测 | 标签质量往往比模型结构更影响最终效果。 |
| 数据泄漏 | 按时间切分、只在训练集拟合预处理器、排除未来字段 | 数据泄漏会让离线指标虚高。 |

数据清洗的基本流程：

```text
理解字段含义 -> 做数据概览 -> 检查缺失和重复 -> 检查异常和分布
-> 统一格式和单位 -> 处理标签质量 -> 记录清洗规则 -> 验证清洗后数据
```

数据清洗不是简单地“删脏数据”。更准确地说，它是在业务规则、统计分布和建模目标之间做判断。比如风控任务中的极端交易金额可能是异常输入，也可能正是欺诈信号；设备故障检测中的异常波形不能随便删除，因为它可能是模型最需要学习的样本。

实践建议：

- 清洗前保留原始数据，不要直接覆盖。
- 每条清洗规则都要能解释，最好记录原因和影响样本量。
- 训练集、验证集、测试集要避免互相污染。
- 标准化、归一化、目标编码等需要学习统计量的步骤，只能在训练集上 `fit`。
- 清洗后要重新检查类别分布、缺失率、数值范围和样本数量变化。

### 6.2 数据增强概念详解

数据增强，英文是 Data Augmentation，简称 DA。它的核心目标是基于已有数据生成更多合理变体，让模型见到更丰富的数据形态，从而提升泛化能力、缓解过拟合、降低对小规模训练集的依赖。

数据增强不是随意制造数据，而是在不改变样本语义标签的前提下，引入模型在真实环境中可能遇到的变化。例如一张猫的图片水平翻转后仍然是猫；一段文本做轻微同义改写后语义仍然相同；一段语音加入轻微背景噪声后说话内容仍然不变。

数据增强的目的：

- 增加训练样本多样性。
- 提升模型对旋转、裁剪、噪声、光照、表达方式等变化的鲁棒性。
- 缓解训练数据不足。
- 减少过拟合。
- 改善类别不平衡。
- 模拟真实部署环境中的扰动。

不同数据类型的常见增强方法：

| 数据类型 | 常见增强方法 | 示例 |
|---|---|---|
| 图像 | 翻转、旋转、裁剪、缩放、颜色扰动、加噪、模糊、Cutout、MixUp、CutMix | 训练图像分类、检测、分割模型。 |
| 文本 | 同义词替换、随机删除、随机交换、回译、模板改写、LLM 改写 | 训练文本分类、意图识别、问答模型。 |
| 音频 | 加噪、变速、变调、时间平移、SpecAugment | 训练语音识别、声纹识别模型。 |
| 时间序列 | 抖动、缩放、时间扭曲、窗口裁剪、随机遮盖 | 训练传感器、金融、设备预测模型。 |
| 表格数据 | SMOTE、过采样、特征扰动、生成模型合成样本 | 类别不平衡分类。 |
| 多模态数据 | 图文改写、图像扰动、文本 paraphrase、跨模态生成 | 训练图文检索、VQA、多模态模型。 |

经典增强方法：

| 方法 | 英文名称 | 解释 |
|---|---|---|
| MixUp | MixUp | 将两个样本和标签按比例线性混合，使模型边界更平滑。 |
| CutMix | CutMix | 将一张图像的一块区域替换为另一张图像区域，并按面积混合标签。 |
| Cutout | Cutout | 随机遮挡图像部分区域，提升模型对局部缺失的鲁棒性。 |
| RandAugment | RandAugment | 随机选择增强操作和强度，减少人工调策略成本。 |
| AutoAugment | AutoAugment | 自动搜索最优图像增强策略。 |
| SpecAugment | SpecAugment | 对语音频谱做时间遮盖、频率遮盖等增强。 |
| Back Translation | Back Translation | 文本先翻译成另一种语言再翻译回来，得到语义近似表达。 |
| SMOTE | Synthetic Minority Over-sampling Technique | 对少数类样本插值生成合成样本。 |

数据增强的关键原则：

- 标签语义不能被破坏。比如数字 6 旋转 180 度可能变成 9，这种增强就不适合。
- 增强分布要接近真实场景。部署环境不会出现的强扰动可能伤害模型。
- 训练集可以增强，验证集和测试集通常不要做随机增强。
- 增强不是越强越好，过强增强会引入标签噪声。
- 对文本任务要特别小心，同义替换可能改变语气、实体、否定关系和事实含义。

数据增强和过采样的关系：

过采样通常是处理类别不平衡的一类方法，可以简单复制少数类，也可以生成少数类样本。数据增强范围更广，不只处理类别不平衡，也用于提升泛化能力。SMOTE 可以看作表格数据中的一种合成式过采样，也可以被归入广义数据增强。

### 6.3 数据蒸馏概念详解

数据蒸馏，英文常见说法是 Data Distillation。相关概念还有 Dataset Distillation 数据集蒸馏、Dataset Condensation 数据凝缩、Coreset Selection 核心集选择。它的核心目标是把大规模数据集中的关键信息压缩到更小的数据集合、合成样本或高价值表示中，让模型用更少的数据也能获得接近完整数据集训练的效果。

数据蒸馏可以理解为：

```text
大规模原始数据 -> 少量高信息密度数据 -> 用更低训练成本达到接近效果
```

它和知识蒸馏不同。知识蒸馏压缩的是模型能力，通常是“大模型教小模型”；数据蒸馏压缩的是训练数据，通常是“大数据集变成小数据集”。它和数据增强也不同，数据增强通常让数据变多，数据蒸馏通常让数据变少但更有代表性。

数据蒸馏的主要目标：

- 降低训练成本。
- 减少数据存储和传输成本。
- 提高数据使用效率。
- 在有限算力下快速训练或快速验证模型。
- 为小模型、边缘设备、快速实验提供精简训练数据。
- 在隐私或版权受限场景中，用合成数据替代部分原始数据。

常见数据蒸馏思路：

| 方法 | 英文名称 | 核心思想 |
|---|---|---|
| 核心集选择 | Core-Set Selection | 从原始数据中选择最有代表性的真实样本子集。 |
| 数据集蒸馏 | Dataset Distillation | 直接学习少量合成样本，使模型在这些样本上训练后效果接近完整数据集。 |
| 数据凝缩 | Dataset Condensation | 将大数据集压缩为小规模代表性数据，常与数据集蒸馏接近。 |
| 梯度匹配 | Gradient Matching | 让小数据集产生的训练梯度接近完整数据集产生的梯度。 |
| 分布匹配 | Distribution Matching | 让蒸馏数据在特征空间中的分布接近原始数据。 |
| 轨迹匹配 | Training Trajectory Matching | 让模型在蒸馏数据上的训练轨迹接近完整数据训练轨迹。 |
| 软标签蒸馏 | Soft-Label Data Distillation | 为样本附加教师模型输出的概率分布，保留类别间相似性信息。 |
| 合成样本优化 | Synthetic Sample Optimization | 把少量样本本身作为可学习变量进行优化。 |

数据蒸馏适合的场景：

- 原始数据量巨大，完整训练成本太高。
- 需要快速做模型结构搜索、超参数搜索或算法验证。
- 边缘设备或低算力环境只能保存少量训练数据。
- 数据共享受限制，希望用合成数据表达原始数据主要信息。
- 教学、调试、基准测试中需要一个小而有代表性的数据集。

数据蒸馏的风险和限制：

- 蒸馏数据可能只适合某类模型结构，对其他模型泛化不好。
- 合成样本可能难以解释，尤其是图像或高维数据。
- 如果原始数据有偏差，蒸馏后可能保留甚至放大偏差。
- 蒸馏过程本身可能计算成本较高。
- 在隐私场景下，合成样本仍可能泄露原始数据的信息，需要额外隐私评估。

一个简单例子：

```text
原始数据：60000 张手写数字图片
数据蒸馏目标：为每个数字类别学习 10 张高信息密度合成图片
训练方式：模型只用这 100 张合成图片训练
期望结果：模型测试效果尽量接近用 60000 张原图训练的效果
```

可以把数据清洗、数据增强和数据蒸馏放在一条数据处理链路中理解：

```text
数据清洗：让数据更正确、更一致、更可靠
数据增强：让数据更多样、更接近真实扰动
数据蒸馏：让数据更精简、更高信息密度
```

## 31. 易混概念辨析

### 31.1 AI、机器学习与深度学习

AI 是最宽泛的概念，目标是让机器表现出智能。机器学习是 AI 的一种实现路径，通过数据学习规律。深度学习是机器学习中的一种方法，核心是使用多层神经网络自动学习表示。

关系可以理解为：

```text
AI > 机器学习 > 深度学习
```

### 31.2 参数与超参数

参数是模型从数据中自动学习的变量，例如线性回归权重、神经网络权重。超参数是训练前设置的配置，例如学习率、批大小、树深度、正则化强度。参数通过训练得到，超参数通常通过验证集或交叉验证选择。

### 31.3 训练集、验证集与测试集

训练集用于学习参数，验证集用于调参和模型选择，测试集只用于最终评估。测试集不能参与特征选择、标准化拟合、调参或早停，否则会产生数据泄漏。

### 31.4 归一化与标准化

归一化通常把数据缩放到固定范围，例如 0 到 1。标准化通常把数据变为均值 0、标准差 1。神经网络、SVM、KNN、线性模型通常对特征尺度敏感，树模型一般不太敏感。

### 31.5 过拟合与欠拟合

过拟合是模型在训练集上好、测试集上差，说明模型过度记住训练数据。欠拟合是训练集和测试集上都差，说明模型表达能力不足或特征不足。

### 31.6 Precision 与 Recall

Precision 关心“预测为正的里面有多少是真的”，Recall 关心“真实为正的里面有多少被找出来”。垃圾内容审核、疾病筛查、欺诈检测等任务通常要根据业务代价选择侧重点。

### 31.7 生成模型与判别模型

判别模型学习 `P(y|x)` 或决策边界，关注如何分类。生成模型学习数据分布 `P(x)` 或联合分布 `P(x,y)`，关注如何生成数据或解释数据产生过程。

### 31.8 离线评估与在线评估

离线评估使用历史数据计算指标，速度快、成本低。在线评估在真实用户流量中验证效果，更接近业务结果，但成本和风险更高。推荐、广告、搜索场景中，离线指标好不一定线上指标好。

### 31.9 知识蒸馏、数据蒸馏与数据增强

知识蒸馏的核心是“模型教模型”：通常用大模型、教师模型的 logits、软标签、中间特征或行为输出，指导小模型、学生模型学习。

数据蒸馏的核心是“数据压缩数据”：目标是从大规模数据中提取少量高价值样本、合成样本或压缩表示，使模型用更少数据也能学到接近完整数据集的信息。

数据增强的核心是“扩充数据”：通过翻转、裁剪、加噪、同义改写、混合样本等方式增加训练数据多样性。它通常让数据变多；数据蒸馏通常让数据变少但信息密度更高。

简单区分：

```text
知识蒸馏：大模型 -> 小模型
数据蒸馏：大数据集 -> 小数据集或高价值数据表示
数据增强：原始数据 -> 更多变体数据
```

## 32. 学习建议

学习这些术语时可以按项目流程记忆，而不是按字母表死背：

1. 先掌握数据相关术语：样本、特征、标签、训练集、验证集、测试集、数据泄漏。
2. 再掌握传统机器学习：分类、回归、聚类、SVM、决策树、随机森林、GBDT。
3. 然后掌握训练过程：损失函数、优化器、正则化、过拟合、欠拟合、交叉验证。
4. 接着学习深度学习：神经元、层、激活函数、反向传播、CNN、RNN、Transformer。
5. 最后学习工程化：特征库、模型部署、监控、数据漂移、MLOps。

真正掌握一个术语，至少要能回答四个问题：

- 它的英文全称和简称是什么？
- 它解决什么问题？
- 它出现在机器学习流程的哪个环节？
- 它和相近术语有什么区别？

## 33. 总结

机器学习、深度学习和数据处理的专业术语很多，但底层逻辑并不乱。数据处理负责把原始数据变成可靠输入，机器学习负责从数据中学习规律，深度学习负责用神经网络自动学习复杂表示，评估指标负责判断模型是否有效，MLOps 负责让模型稳定进入生产环境。

如果把所有术语放到同一张图里，可以概括为：

```text
数据层：采集、清洗、编码、特征、质量、治理
建模层：任务、算法、损失、优化、正则、调参
评估层：指标、验证、测试、误差分析、可解释性
深度学习层：网络结构、训练技巧、预训练、微调、多模态
```
