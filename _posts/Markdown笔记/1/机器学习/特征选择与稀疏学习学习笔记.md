## 1. 特征选择是什么

特征选择是从原始特征集合中选出一部分对建模最有用的特征，去掉无关、冗余、噪声或可能导致泄漏的特征。

假设原始数据有 `p` 个特征：

```text
X = [x1, x2, ..., xp]
```

特征选择的目标是找到一个子集：

```text
S ⊆ {x1, x2, ..., xp}
```

使得模型在这个子集上具有更好的泛化能力、解释性或工程效率。

### 1.1 特征选择不是简单删列

特征选择不是随意删除字段，而是一个有目标、有验证、有约束的过程。

它要回答：

- 哪些特征对目标变量有预测价值？
- 哪些特征只是噪声？
- 哪些特征高度冗余？
- 哪些特征会造成数据泄漏？
- 哪些特征增加了计算成本但收益很小？
- 哪些特征保留后更便于解释和部署？

### 1.2 稀疏学习是什么

稀疏学习是通过模型或优化方法，使模型参数、特征表示或编码结果中大部分元素为零，只保留少量关键元素。

例如线性模型：

```text
y = w1*x1 + w2*x2 + ... + wp*xp
```

如果很多 `wi = 0`，说明这些特征没有被模型使用，模型就是稀疏的。

稀疏学习的目标通常包括：

- 自动选择特征。
- 提高解释性。
- 降低模型复杂度。
- 改善高维场景下泛化能力。
- 获得简洁表示。

### 1.3 特征选择和稀疏学习的关系

特征选择关注“选哪些特征”。

稀疏学习关注“如何让模型自动只使用少量特征或基表示”。

二者关系密切：

- Lasso 通过 L1 正则让部分系数变成 0，从而实现特征选择。
- 稀疏逻辑回归可以在分类任务中自动选特征。
- 稀疏表示通过少量字典原子表示样本。

可以说，稀疏学习是实现特征选择的重要思想和工具之一。

## 2. 为什么需要特征选择

### 2.1 降低过拟合风险

特征越多，模型越容易在训练集上学到偶然噪声，尤其在样本数少、特征数多时。

例如：

```text
样本数 n = 100
特征数 p = 10000
```

这种场景下，即使特征和标签完全随机，也可能找到一些在训练集上看起来相关的特征。直接使用所有特征很容易过拟合。

### 2.2 提高泛化能力

去掉无关特征和噪声特征后，模型更容易学习真实规律。

好的特征选择可以：

- 降低方差。
- 减少噪声干扰。
- 提升验证集和测试集表现。

### 2.3 提高可解释性

在医疗、金融、风控、工业等场景中，模型解释非常重要。

如果模型使用 5000 个特征，很难解释；如果只使用 20 个关键特征，就容易分析每个特征的业务含义。

### 2.4 降低计算成本

特征越多：

- 训练更慢。
- 预测更慢。
- 存储更多。
- 特征生产成本更高。
- 部署链路更复杂。

特征选择可以减少训练和线上推理成本。

### 2.5 缓解维度灾难

在高维空间中，样本变得稀疏，距离度量退化，模型学习难度增加。

特征选择可以降低维度，缓解维度灾难。

### 2.6 删除泄漏特征

有些特征在训练集上效果极好，但包含了未来信息或标签信息。

例如预测用户是否流失时，如果使用“流失后客服回访次数”，就是明显泄漏。特征选择过程必须识别并删除这类特征。

## 3. 特征选择、特征提取与降维的区别

这三个概念相关但不同。

### 3.1 特征选择

特征选择从原始特征中挑选一部分。

例如：

```text
[年龄, 收入, 性别, 城市, 点击次数, 购买次数]
```

选择：

```text
[收入, 点击次数, 购买次数]
```

原始特征语义保留，解释性好。

### 3.2 特征提取

特征提取是从原始数据生成新的特征。

例如：

- 从文本中提取 TF-IDF。
- 从图像中提取 CNN embedding。
- 从时间序列中提取均值、方差、峰值。
- 从用户行为中提取近 7 天活跃次数。

特征提取关注“构造新表示”。

### 3.3 降维

降维把高维特征映射到低维空间。

例如 PCA：

```text
原始 100 维 -> 主成分 10 维
```

低维特征通常是原特征的组合，解释性可能下降。

### 3.4 三者对比

| 方法 | 是否保留原特征 | 解释性 | 常见方法 |
| --- | --- | --- | --- |
| 特征选择 | 是 | 强 | 相关性、RFE、Lasso、树重要性 |
| 特征提取 | 不一定 | 中等或弱 | 文本向量、图像 embedding、统计特征 |
| 降维 | 通常否 | 较弱 | PCA、LDA、t-SNE、UMAP、自编码器 |

### 3.5 什么时候选择哪一种

如果需要解释，优先特征选择。

如果原始数据不能直接建模，例如文本、图像、音频，需要特征提取。

如果特征很多且高度相关，可考虑降维。

实际项目中常组合使用：

```text
特征提取 -> 特征选择 -> 模型训练
```

或：

```text
特征清洗 -> 降维 -> 聚类/分类
```

## 4. 特征选择的基本原则

### 4.1 以目标为导向

不同任务需要不同特征。

例如用户分群、流失预测、购买预测即使用同一批用户数据，最优特征也可能不同。

不要脱离任务做特征选择。

### 4.2 只在训练集上做选择

特征选择必须放在交叉验证或训练流程内部。

错误做法：

```text
先在全量数据上选择特征，再划分训练集和测试集。
```

这会造成数据泄漏，因为测试集信息参与了特征选择。

正确做法：

```text
训练集上选择特征。
验证集/测试集只应用已选特征。
```

在 scikit-learn 中，应使用 `Pipeline` 把特征选择和模型训练封装起来。

### 4.3 结合业务理解

纯算法选择可能保留泄漏特征或难以部署的特征。

业务检查包括：

- 特征是否上线可得？
- 特征是否包含未来信息？
- 特征是否稳定？
- 特征是否有明确含义？
- 特征计算成本是否可接受？

### 4.4 关注稳定性

如果换一个随机种子或换一批样本，选出的特征完全不同，说明特征选择不稳定。

稳定性差可能来自：

- 样本量不足。
- 特征高度相关。
- 标签噪声。
- 模型过拟合。
- 选择阈值过于敏感。

### 4.5 不追求特征越少越好

特征选择不是越少越好，而是在性能、解释性和成本之间权衡。

如果删除特征导致性能明显下降，就不应为了稀疏而稀疏。

## 5. 过滤法 Filter

过滤法不依赖具体模型，而是根据特征本身与标签或特征之间的统计关系进行筛选。

### 5.1 过滤法特点

优点：

- 速度快。
- 实现简单。
- 模型无关。
- 适合预筛选高维特征。

缺点：

- 通常只考虑单个特征与标签的关系。
- 难以捕捉特征交互。
- 可能删除单独弱但组合强的特征。

### 5.2 方差过滤

方差很低的特征通常信息量小。

例如：

```text
某特征 99.9% 样本取值相同
```

这种特征对区分样本帮助有限。

方差过滤适合：

- 删除常量特征。
- 删除近似常量特征。

注意：方差低不一定完全无用。例如极少数异常值可能很重要。因此要结合业务判断。

### 5.3 缺失率过滤

缺失率过高的特征可能不可用。

例如：

```text
某特征缺失率 98%
```

保留它可能带来噪声和复杂处理成本。

常见策略：

- 缺失率超过 90% 删除。
- 缺失本身有业务意义时保留缺失指示变量。
- 高价值特征即使缺失率高也可尝试保留。

### 5.4 单值率过滤

单值率指某个取值占比。

例如：

```text
特征 A 中 99.5% 都是 0
```

如果单值率过高，说明该特征变化很少。

### 5.5 相关性过滤

如果两个特征高度相关，可能存在冗余。

例如：

```text
年收入
月收入
```

二者可能高度线性相关，只保留一个即可。

常用相关系数：

- Pearson 相关系数。
- Spearman 秩相关。
- Kendall 秩相关。

### 5.6 与标签相关性筛选

可以计算每个特征与目标变量的相关性，保留相关性较高的特征。

适合：

- 连续特征和连续目标。
- 初步筛选。

局限：

- 只能捕捉线性或单调关系。
- 对非线性关系不敏感。
- 对异常值敏感。

### 5.7 卡方检验

卡方检验常用于分类任务中的非负特征。

它检验特征和类别标签是否独立。

适合：

- 文本分类中的词频特征。
- 类别计数特征。
- 非负离散特征。

如果某个词在不同类别中出现频率差异很大，则卡方统计量较高，说明该词对分类可能有用。

### 5.8 ANOVA F 检验

ANOVA F 检验用于衡量数值特征在不同类别之间均值是否显著不同。

适合：

- 分类任务。
- 数值特征。
- 标签为离散类别。

如果某特征在不同类别中的均值差异明显，则该特征可能有区分能力。

### 5.9 过滤法适用场景

适合：

- 特征维度很高。
- 需要快速预筛选。
- 文本分类。
- 基因数据。
- 建模前数据清洗。

不适合单独作为最终方案的场景：

- 特征交互很重要。
- 非线性关系复杂。
- 冗余特征很多但单特征都有效。

## 6. 包裹法 Wrapper

包裹法把模型性能作为特征子集好坏的评价标准。

### 6.1 包裹法基本思想

包裹法会尝试不同特征子集，训练模型并评估性能，然后选择表现最好的子集。

它直接面向目标模型和评价指标。

### 6.2 前向选择

前向选择从空特征集开始，每次加入一个能最大提升性能的特征。

流程：

```text
1. 初始特征集为空。
2. 对每个候选特征，尝试加入当前特征集。
3. 训练模型并验证。
4. 选择提升最大的特征加入。
5. 重复直到性能不再提升或达到特征数量上限。
```

优点：

- 相对简单。
- 适合特征数不是特别大的场景。

缺点：

- 贪心算法，不保证全局最优。
- 计算成本较高。

### 6.3 后向消除

后向消除从全部特征开始，每次删除一个影响最小的特征。

流程：

```text
1. 初始使用全部特征。
2. 逐个尝试删除某个特征。
3. 训练模型并验证。
4. 删除对性能影响最小或能提升性能的特征。
5. 重复直到满足停止条件。
```

优点：

- 初始考虑全部特征。
- 能保留有组合价值的特征。

缺点：

- 特征很多时成本非常高。
- 初始全特征训练可能已经很慢。

### 6.4 双向选择

双向选择结合前向加入和后向删除。

它可以：

- 加入新特征。
- 删除加入后变得不重要的旧特征。

比单纯前向或后向更灵活，但实现更复杂。

### 6.5 穷举搜索

如果有 `p` 个特征，总共有：

```text
2^p
```

种特征子集。

穷举搜索只适用于特征数非常少的场景。实际高维数据中不可行。

### 6.6 包裹法优缺点

优点：

- 直接优化模型性能。
- 能考虑特征交互。
- 与具体模型和指标一致。

缺点：

- 计算成本高。
- 容易对验证集过拟合。
- 特征多时不可扩展。
- 结果依赖模型选择。

## 7. 嵌入法 Embedded

嵌入法在模型训练过程中完成特征选择。

### 7.1 嵌入法特点

嵌入法介于过滤法和包裹法之间。

优点：

- 比包裹法高效。
- 能结合模型目标。
- 可捕捉部分特征交互。

缺点：

- 依赖具体模型。
- 不同模型选出的特征可能不同。

### 7.2 L1 正则

L1 正则可以使部分参数变为 0，从而自动选择特征。

典型模型：

- Lasso 回归。
- L1 逻辑回归。
- LinearSVC with L1 penalty。

### 7.3 树模型特征重要性

树模型可以根据分裂增益或不纯度下降计算特征重要性。

典型模型：

- 决策树。
- 随机森林。
- Extra Trees。
- GBDT。
- XGBoost。
- LightGBM。

### 7.4 正则化树模型

XGBoost、LightGBM、CatBoost 通过分裂增益、叶子正则、最小叶子样本数等机制控制复杂度。虽然它们不直接输出稀疏线性系数，但可以通过特征重要性辅助筛选特征。

### 7.5 嵌入法适用场景

适合：

- 希望选择过程和模型训练结合。
- 特征维度较高。
- 希望兼顾效率和效果。
- 使用线性模型或树模型。

## 8. 统计检验与相关性分析

统计检验是过滤法的重要基础。

### 8.1 Pearson 相关系数

Pearson 相关系数衡量两个变量之间的线性相关性：

```text
r = cov(X, Y) / (std(X) * std(Y))
```

取值范围：

```text
[-1, 1]
```

含义：

- 接近 1：强正线性相关。
- 接近 -1：强负线性相关。
- 接近 0：无线性相关。

注意：Pearson 只能衡量线性关系。

### 8.2 Spearman 秩相关

Spearman 相关衡量单调关系。它先把变量转换为排名，再计算 Pearson。

适合：

- 非线性但单调关系。
- 对异常值相对稳健。

### 8.3 Kendall 秩相关

Kendall 也用于衡量秩相关，基于样本对的一致性。

它通常比 Spearman 更稳健，但计算成本更高。

### 8.4 卡方检验

卡方检验用于判断两个离散变量是否独立。

原假设：

```text
特征与标签独立。
```

如果 p 值很小，则拒绝原假设，说明特征和标签可能有关。

### 8.5 F 检验

F 检验用于比较不同类别下数值特征均值是否有显著差异。

在 scikit-learn 中：

- `f_classif` 用于分类。
- `f_regression` 用于回归。

### 8.6 p 值的注意事项

p 值不是特征重要性的绝对证明。

注意：

- 样本很大时，微小差异也可能显著。
- 多重检验会增加假阳性。
- p 值不表示效果大小。
- 统计显著不等于业务显著。

### 8.7 多重检验问题

如果同时检验 10000 个特征，每个检验显著性水平为 0.05，即使所有特征都无效，也可能有约 500 个特征偶然显著。

常见修正：

- Bonferroni 校正。
- Benjamini-Hochberg FDR 控制。

## 9. 互信息与非线性关系

### 9.1 互信息是什么

互信息衡量两个变量之间共享的信息量。

```text
I(X; Y) = sum_x sum_y p(x, y) * log( p(x, y) / (p(x)p(y)) )
```

如果 `X` 和 `Y` 独立：

```text
I(X; Y) = 0
```

互信息越大，说明二者关系越强。

### 9.2 互信息的优点

互信息可以捕捉非线性关系。

例如：

```text
Y = X^2
```

如果 `X` 对称分布，Pearson 相关可能接近 0，但 `X` 和 `Y` 显然有关。互信息可以发现这种关系。

### 9.3 互信息的缺点

缺点：

- 估计困难。
- 对连续变量需要近似。
- 对样本量敏感。
- 不容易直接解释大小。

### 9.4 互信息适用场景

适合：

- 非线性关系可能较强。
- 过滤式特征选择。
- 分类或回归任务初筛。

在 scikit-learn 中常用：

- `mutual_info_classif`
- `mutual_info_regression`

## 10. 递归特征消除 RFE

RFE 是 Recursive Feature Elimination，递归特征消除。

### 10.1 RFE 的基本思想

RFE 使用一个可评估特征重要性的模型，反复训练并删除最不重要的特征。

流程：

```text
1. 使用全部特征训练模型。
2. 根据模型系数或特征重要性排序。
3. 删除最不重要的一部分特征。
4. 在剩余特征上重新训练模型。
5. 重复直到达到指定特征数量。
```

### 10.2 RFECV

RFECV 是带交叉验证的 RFE。

它会在不同特征数量下进行交叉验证，选择验证性能最好的特征数量。

优点：

- 自动选择特征数量。
- 比固定选择数量更灵活。

缺点：

- 计算成本高。

### 10.3 RFE 适用模型

适合具有 `coef_` 或 `feature_importances_` 的模型：

- 线性回归。
- 逻辑回归。
- SVM 线性核。
- 随机森林。
- GBDT。

### 10.4 RFE 的注意点

RFE 结果依赖模型。

如果基础模型不适合数据，选出的特征也可能不可靠。

RFE 成本较高，特征维度很大时通常先用过滤法预筛选。

## 11. 正则化基础

正则化是在损失函数中加入模型复杂度惩罚，用来防止过拟合。

### 11.1 正则化目标

普通经验风险最小化：

```text
min_w Loss(w)
```

加入正则化：

```text
min_w Loss(w) + lambda * Penalty(w)
```

其中：

- `Loss(w)` 衡量模型拟合训练数据的误差。
- `Penalty(w)` 惩罚模型复杂度。
- `lambda` 控制正则化强度。

### 11.2 为什么正则化能防止过拟合

过拟合模型往往参数很大或结构复杂。正则化通过限制参数大小或数量，让模型更简单。

正则化可以：

- 降低方差。
- 提高泛化。
- 缓解多重共线性。
- 实现特征选择。

### 11.3 常见正则项

L1 正则：

```text
||w||_1 = sum_j |w_j|
```

L2 正则：

```text
||w||_2^2 = sum_j w_j^2
```

Elastic Net：

```text
alpha * ||w||_1 + (1 - alpha) * ||w||_2^2
```

### 11.4 正则化和约束的关系

正则化问题：

```text
min Loss(w) + lambda * ||w||_1
```

等价于某种约束问题：

```text
min Loss(w)
subject to ||w||_1 <= t
```

`lambda` 越大，约束越强，参数越小。

## 12. L1 正则与 Lasso

### 12.1 Lasso 是什么

Lasso 是 Least Absolute Shrinkage and Selection Operator。

在线性回归中，Lasso 的目标函数为：

```text
min_w (1 / 2n) * ||y - Xw||_2^2 + alpha * ||w||_1
```

其中 `alpha` 控制 L1 正则强度。

### 12.2 L1 为什么能产生稀疏解

L1 正则能让部分参数精确变为 0。

几何直观：

- L1 约束区域在二维中是菱形。
- 损失函数等高线与菱形约束区域相切时，更容易落在坐标轴上。
- 落在坐标轴上表示某个参数为 0。

因此 L1 正则天然具有特征选择能力。

### 12.3 Lasso 的效果

Lasso 会：

- 压缩系数。
- 将部分系数压到 0。
- 自动选择特征。

当 `alpha` 越大：

- 正则越强。
- 非零特征越少。
- 模型越稀疏。
- 过大可能欠拟合。

### 12.4 Lasso 的优点

优点：

- 自动特征选择。
- 模型解释性强。
- 适合高维数据。
- 可以产生稀疏模型。

### 12.5 Lasso 的缺点

缺点：

- 对高度相关特征不稳定。
- 相关特征中可能随机选一个。
- 当特征数远大于样本数时，选择结果可能不稳定。
- 对特征尺度敏感，必须标准化。

### 12.6 Lasso 和多重共线性

当多个特征高度相关时，Lasso 可能只选择其中一个，忽略其他。

这在解释上可能有问题：

```text
特征 A 和 B 都有业务意义，但 Lasso 只保留 A。
```

如果希望相关特征成组保留，可考虑 Elastic Net 或 Group Lasso。

## 13. L2 正则与 Ridge

### 13.1 Ridge 是什么

Ridge 回归使用 L2 正则：

```text
min_w (1 / 2n) * ||y - Xw||_2^2 + alpha * ||w||_2^2
```

### 13.2 L2 的作用

L2 正则会压缩系数，但通常不会把系数压到精确 0。

它让模型：

- 参数更小。
- 对噪声更不敏感。
- 多重共线性下更稳定。

### 13.3 Ridge 与 Lasso 的区别

| 对比项 | Lasso | Ridge |
| --- | --- | --- |
| 正则 | L1 | L2 |
| 是否稀疏 | 是 | 通常否 |
| 特征选择 | 能 | 不能直接 |
| 相关特征 | 可能只选一个 | 倾向一起保留并分摊权重 |
| 稳定性 | 相关特征下较不稳定 | 更稳定 |

### 13.4 Ridge 适用场景

适合：

- 特征高度相关。
- 希望保留所有特征。
- 预测性能优先于特征选择。
- 线性模型过拟合。

如果目标是稀疏特征选择，Ridge 不是首选。

## 14. Elastic Net

Elastic Net 结合 L1 和 L2 正则。

### 14.1 目标函数

Elastic Net 目标函数：

```text
min_w (1 / 2n) * ||y - Xw||_2^2
      + alpha * l1_ratio * ||w||_1
      + alpha * (1 - l1_ratio) / 2 * ||w||_2^2
```

其中：

- `alpha` 控制整体正则强度。
- `l1_ratio` 控制 L1 和 L2 的比例。

### 14.2 Elastic Net 的作用

它兼具：

- Lasso 的稀疏性。
- Ridge 的稳定性。

在高度相关特征中，Elastic Net 更倾向于成组选择相关特征。

### 14.3 适用场景

适合：

- 高维特征。
- 特征之间强相关。
- 希望特征选择但 Lasso 不稳定。
- 基因数据、文本数据、金融因子等。

### 14.4 参数理解

`l1_ratio = 1`：

```text
等价于 Lasso。
```

`l1_ratio = 0`：

```text
等价于 Ridge。
```

实际常尝试：

```text
0.1, 0.3, 0.5, 0.7, 0.9
```

## 15. 稀疏逻辑回归与线性分类

### 15.1 逻辑回归加 L1 正则

逻辑回归用于分类任务。

加入 L1 正则后：

```text
min_w LogisticLoss(w) + lambda * ||w||_1
```

模型会自动将部分特征系数变为 0。

### 15.2 稀疏逻辑回归适用场景

适合：

- 二分类或多分类。
- 高维稀疏特征。
- 文本分类。
- 风控评分卡初筛。
- 需要解释特征方向。

### 15.3 系数解释

逻辑回归中，系数 `wj` 表示特征 `xj` 对 log-odds 的影响。

```text
log(p / (1 - p)) = w0 + w1*x1 + ... + wp*xp
```

如果 `wj > 0`，特征增大倾向于提高正类概率。

如果 `wj < 0`，特征增大倾向于降低正类概率。

如果 `wj = 0`，该特征未被模型使用。

### 15.4 线性 SVM 与 L1

线性 SVM 也可以使用 L1 正则实现稀疏特征选择。

在 scikit-learn 中，常使用：

```python
LinearSVC(penalty="l1", dual=False)
```

适合高维文本分类等场景。

## 16. 树模型与特征选择

### 16.1 树模型如何评估特征

树模型每次分裂会选择一个特征和切分点，使损失下降或不纯度下降最大。

特征重要性可以来自：

- 分裂次数。
- 不纯度下降。
- 增益。
- 覆盖样本数。

### 16.2 随机森林特征重要性

随机森林通常基于平均不纯度下降计算重要性。

优点：

- 能捕捉非线性。
- 能捕捉特征交互。
- 不需要特征标准化。

缺点：

- 偏向连续特征。
- 偏向高基数类别特征。
- 强相关特征会分摊重要性。

### 16.3 GBDT 特征重要性

GBDT、XGBoost、LightGBM 通常提供：

- split importance：分裂次数。
- gain importance：分裂增益。
- cover importance：覆盖样本。

通常 gain 更有参考价值，但仍需谨慎解释。

### 16.4 Permutation Importance

排列重要性更接近泛化影响。

流程：

```text
1. 在验证集上计算原始性能。
2. 打乱某个特征。
3. 再计算性能。
4. 性能下降越多，特征越重要。
```

优点：

- 模型无关。
- 基于验证集性能。

缺点：

- 计算成本高。
- 特征相关时可能低估重要性。

### 16.5 SHAP 用于特征筛选

SHAP 可以衡量每个特征对预测的贡献。

常见做法：

- 计算每个特征的平均绝对 SHAP 值。
- 按平均绝对贡献排序。
- 选择前 N 个特征。

注意：SHAP 更适合解释，不应机械地替代验证集评估。

## 17. 稳定性选择 Stability Selection

稳定性选择关注特征在不同子样本和随机扰动下被选中的稳定程度。

### 17.1 基本思想

流程：

```text
1. 多次从训练集中抽样。
2. 每次使用特征选择方法，例如 Lasso。
3. 记录每个特征被选中的频率。
4. 选择频率高于阈值的特征。
```

例如某特征在 100 次抽样中被选中 92 次，则稳定性很高。

### 17.2 优点

优点：

- 提高选择稳定性。
- 减少偶然特征。
- 适合高维小样本。
- 可以和 Lasso、Elastic Net 等结合。

### 17.3 缺点

缺点：

- 计算成本较高。
- 参数较多。
- 高度相关特征组中选择频率可能分散。

### 17.4 实践建议

稳定性选择适合：

- 需要解释的科研建模。
- 生物信息学。
- 金融因子筛选。
- 样本少而特征多的任务。

## 18. 稀疏学习的数学视角

### 18.1 稀疏性

向量 `w` 是稀疏的，表示其中大多数元素为 0。

例如：

```text
w = [0, 0, 1.2, 0, -0.7, 0, 0]
```

这个向量只有两个非零元素。

### 18.2 L0 范数

L0 范数表示非零元素个数：

```text
||w||_0 = number of non-zero elements in w
```

如果直接做特征选择，可以写成：

```text
min Loss(w) + lambda * ||w||_0
```

问题是 L0 优化通常是组合优化，计算困难。

### 18.3 L1 作为 L0 的凸松弛

L1 正则常被视为 L0 的凸松弛。

L0 更直接控制非零个数，但不可解或难解；L1 是凸的，更容易优化，并且能产生稀疏解。

### 18.4 稀疏性和泛化

稀疏模型的有效自由度较低，通常更不容易过拟合。

但过度稀疏会导致欠拟合。

因此需要通过交叉验证选择正则强度。

### 18.5 稀疏性和可解释性

稀疏模型只依赖少数特征，更容易解释。

但需要注意：

- 稀疏不等于正确。
- 被选中特征不一定有因果意义。
- 高相关特征下，选择结果可能不唯一。

---

## 19. 稀疏表示与字典学习

稀疏学习不仅用于特征选择，也用于表示学习。

### 19.1 稀疏表示

稀疏表示假设一个样本可以由少量基向量线性组合表示。

给定字典 `D`，希望找到稀疏编码 `a`：

```text
x ≈ D a
```

其中 `a` 大部分元素为 0。

优化形式：

```text
min_a ||x - D a||_2^2 + lambda * ||a||_1
```

### 19.2 字典学习

字典学习同时学习字典 `D` 和稀疏编码 `A`：

```text
min_{D, A} ||X - D A||_F^2 + lambda * ||A||_1
```

直观理解：

- 字典 `D` 是一组基础模式。
- 每个样本只用少量基础模式组合表示。

### 19.3 应用场景

稀疏表示和字典学习用于：

- 图像去噪。
- 信号恢复。
- 压缩表示。
- 异常检测。
- 特征学习。

### 19.4 与 PCA 的区别

PCA 使用正交主成分，表示通常是稠密的。

字典学习的字典可以过完备，表示是稀疏的。

| 方法 | 基向量 | 表示 | 目标 |
| --- | --- | --- | --- |
| PCA | 正交主成分 | 通常稠密 | 最大方差 |
| 字典学习 | 可过完备字典 | 稀疏 | 重构误差加稀疏约束 |

## 20. 压缩感知基础

压缩感知是稀疏学习的重要理论方向。

### 20.1 核心思想

如果信号在某个基下是稀疏的，就可以用远少于传统采样定理要求的观测恢复信号。

观测模型：

```text
y = A x
```

其中：

- `x` 是稀疏信号。
- `A` 是测量矩阵。
- `y` 是低维观测。

目标是从 `y` 恢复 `x`。

### 20.2 L0 恢复

理想问题：

```text
min ||x||_0
subject to y = A x
```

但 L0 优化困难。

### 20.3 L1 恢复

常用凸松弛：

```text
min ||x||_1
subject to y = A x
```

在一定条件下，L1 解可以恢复真实稀疏解。

### 20.4 应用

压缩感知用于：

- MRI 成像。
- 信号采样。
- 图像压缩。
- 雷达。
- 传感器网络。

### 20.5 和特征选择的关系

压缩感知关注稀疏信号恢复；特征选择关注从特征中选择有用子集。二者都依赖稀疏性思想和 L1 优化。

## 21. 高维小样本问题

高维小样本是特征选择和稀疏学习最重要的应用场景之一。

### 21.1 什么是高维小样本

高维小样本指：

```text
p >> n
```

即特征数远大于样本数。

例如：

- 基因表达数据：几万个基因，几百个样本。
- 文本分类：几十万词特征，几千样本。
- 金融因子：大量候选因子，样本期数有限。

### 21.2 主要问题

问题包括：

- 过拟合严重。
- 多重共线性。
- 噪声特征多。
- 特征选择不稳定。
- 训练成本高。
- 多重检验假阳性。

### 21.3 常用策略

常用策略：

- 先过滤明显无用特征。
- 标准化。
- 使用 L1 或 Elastic Net。
- 使用稳定性选择。
- 严格交叉验证。
- 控制多重检验。
- 使用领域知识约束特征集合。

### 21.4 文本高维稀疏特征

文本数据常见表示：

- Bag of Words
- TF-IDF
- n-gram

特征高维且稀疏。

常用模型：

- L1/L2 Logistic Regression。
- Linear SVM。
- Naive Bayes。

树模型在超高维稀疏文本上不一定优于线性模型。

## 22. 特征选择的工程实践流程

### 22.1 完整流程

推荐流程：

```text
1. 明确任务目标和评价指标。
2. 划分训练集、验证集、测试集。
3. 检查泄漏特征和不可上线特征。
4. 删除常量、近常量、高缺失率特征。
5. 处理缺失值、异常值和类别编码。
6. 过滤高度冗余特征。
7. 使用过滤法做初筛。
8. 使用嵌入法或包裹法精筛。
9. 用交叉验证评估性能。
10. 检查选择稳定性。
11. 解释特征并做业务确认。
12. 固化特征列表和处理流水线。
```

### 22.2 防止数据泄漏

所有会“学习数据统计量”的步骤都应只在训练集上 fit：

- 缺失值填充。
- 标准化。
- 编码。
- 特征选择。
- PCA。

然后在验证集或测试集上 transform。

推荐使用 Pipeline。

### 22.3 特征选择和模型选择一起做

不同模型对特征需求不同。

例如：

- 线性模型需要减少共线性和无关特征。
- 树模型可以处理部分非线性和特征交互。
- KNN、KMeans 对特征尺度和噪声非常敏感。

因此，特征选择不能脱离最终模型。

### 22.4 记录实验

特征选择实验应记录：

- 特征版本。
- 选择方法。
- 阈值或参数。
- 训练验证切分。
- 模型参数。
- 评价指标。
- 最终特征列表。

否则很难复现。

## 24. 常见误区与排查清单

### 24.1 常见误区

#### 误区一：先全量特征选择再划分数据

这是数据泄漏。特征选择必须只在训练集或交叉验证训练折内完成。

#### 误区二：相关性低就一定无用

相关性低可能只是无线性关系。特征可能和目标存在非线性关系或交互关系。

#### 误区三：Lasso 选出的特征一定最真实

Lasso 选出的特征依赖样本、正则强度和特征相关性，不代表因果真实。

#### 误区四：树重要性最高的特征一定最重要

树重要性可能偏向连续特征和高基数特征，也会受相关特征影响。

#### 误区五：特征越少越好

过度删除特征会欠拟合。特征选择应服务于泛化、解释和成本，而不是盲目追求数量少。

#### 误区六：测试集可用于选择特征

测试集只能用于最终评估，不能参与特征选择、调参或阈值确定。

### 24.2 排查清单

如果特征选择效果不好，检查：

- 是否存在数据泄漏？
- 是否在 Pipeline 内做选择？
- 是否只在训练集 fit？
- 特征是否标准化？
- 选择方法是否适合模型？
- 是否忽略了非线性关系？
- 是否有高度相关特征导致不稳定？
- 是否有标签噪声？
- 选择结果在不同折中是否稳定？
- 删除特征后线上计算是否更简单？

## 25. 常见面试题

### 25.1 什么是特征选择？

特征选择是从原始特征集合中选择一部分有用特征，删除无关、冗余、噪声或泄漏特征，以提升泛化能力、解释性和计算效率。

### 25.2 特征选择有哪些方法？

主要有三类：过滤法、包裹法和嵌入法。过滤法基于统计指标筛选；包裹法基于模型性能搜索特征子集；嵌入法在模型训练过程中完成选择，如 L1 正则和树模型重要性。

### 25.3 Filter、Wrapper、Embedded 有什么区别？

Filter 不依赖模型，速度快但不考虑特征交互；Wrapper 用模型性能评估特征子集，效果直接但成本高；Embedded 在训练过程中选择特征，兼顾效率和模型相关性。

### 25.4 为什么特征选择可能导致数据泄漏？

如果在全量数据上选择特征，再划分训练集和测试集，测试集信息已经影响了特征集合，评估结果会虚高。正确做法是在训练集或交叉验证内部进行特征选择。

### 25.5 L1 正则为什么能做特征选择？

L1 正则的约束区域有尖角，优化解容易落在坐标轴上，使部分系数精确为 0，因此能产生稀疏模型并实现特征选择。

### 25.6 Lasso 和 Ridge 有什么区别？

Lasso 使用 L1 正则，能产生稀疏系数并选择特征；Ridge 使用 L2 正则，会缩小系数但通常不会让系数变成 0，更适合处理多重共线性和提升稳定性。

### 25.7 Elastic Net 解决什么问题？

Elastic Net 结合 L1 和 L2，既能产生稀疏性，又比 Lasso 在高度相关特征下更稳定，倾向于成组选择相关特征。

### 25.8 为什么 Lasso 前需要标准化？

L1 正则对系数大小进行惩罚，如果特征尺度不同，系数大小不可比，正则化会偏向某些尺度特征。因此使用 Lasso 前通常需要标准化。

### 25.9 相关性低的特征一定没用吗？

不一定。相关性通常只能捕捉线性或单调关系，特征可能通过非线性关系或与其他特征交互产生预测价值。

### 25.10 树模型特征重要性有什么问题？

基于不纯度或增益的重要性可能偏向连续特征和高基数特征；强相关特征会分摊重要性；重要性只说明模型使用情况，不代表因果关系。

### 25.11 什么是递归特征消除？

RFE 是一种包裹式方法，反复训练模型，根据系数或重要性删除最不重要的特征，直到达到指定特征数量。

### 25.12 什么是稳定性选择？

稳定性选择通过多次抽样和重复特征选择，统计每个特征被选中的频率，选择稳定出现的特征，以提高高维场景下选择结果的可靠性。

### 25.13 什么是稀疏学习？

稀疏学习通过约束或正则化让模型参数、编码或表示中大部分元素为 0，只保留少量关键元素，从而实现简洁表示、特征选择或信号恢复。

### 25.14 L0 和 L1 有什么关系？

L0 范数直接统计非零元素个数，最符合稀疏性目标，但优化困难。L1 范数是 L0 的常用凸松弛，更容易优化，并且能产生稀疏解。

### 25.15 什么是字典学习？

字典学习希望学习一组基础原子，使每个样本可以由少量原子线性组合表示。它同时优化字典和稀疏编码，常用于图像、信号和表示学习。

### 25.16 什么是压缩感知？

压缩感知认为如果信号在某个基下是稀疏的，就可以用较少观测恢复原信号。常用 L1 优化从低维观测中恢复稀疏信号。

### 25.17 高维小样本为什么需要特征选择？

高维小样本中噪声特征多、过拟合严重、多重共线性明显，直接建模泛化差。特征选择和稀疏正则可以降低有效维度，提高稳定性。

### 25.18 特征选择会不会降低模型性能？

会。如果删除了有用特征，模型可能欠拟合。特征选择应通过交叉验证评估，不能只按特征数量判断。

### 25.19 如何评估特征选择是否有效？

应比较选择前后在验证集或交叉验证中的性能、模型复杂度、稳定性、解释性和部署成本。最终以独立测试集表现为准。

### 25.20 特征选择和降维有什么区别？

特征选择保留原始特征子集，解释性强；降维通常生成原特征的组合或新表示，维度更低但解释性可能下降。

## 26. 学习路线与总结

### 26.1 学习路线

建议按以下顺序学习：

1. 理解特征选择的目的和数据泄漏风险。
2. 学习过滤法：方差、缺失率、相关性、卡方、F 检验。
3. 学习互信息，理解非线性特征筛选。
4. 学习包裹法：前向选择、后向消除、RFE。
5. 学习正则化基础。
6. 深入 L1、L2、Lasso、Ridge。
7. 学习 Elastic Net 和稀疏逻辑回归。
8. 学习树模型重要性、Permutation Importance、SHAP。
9. 学习稳定性选择。
10. 扩展到稀疏表示、字典学习和压缩感知。
11. 在真实项目中练习 Pipeline 防泄漏流程。

### 26.2 核心知识点回顾

特征选择方法主要分为：

- Filter：统计指标筛选。
- Wrapper：模型性能驱动搜索。
- Embedded：模型训练中选择。

稀疏学习核心方法：

- L1 正则。
- Lasso。
- 稀疏逻辑回归。
- Elastic Net。
- 稀疏表示。
- 字典学习。
- 压缩感知。

实践中最重要的是：

- 防止数据泄漏。
- 使用 Pipeline。
- 结合业务理解。
- 检查稳定性。
- 通过验证集评估。
- 不把相关性或重要性当作因果结论。

### 26.3 一句话总结

特征选择与稀疏学习的本质，是在保留有效信息的同时降低模型复杂度；它既是提升泛化能力的技术手段，也是让模型更可解释、更稳定、更容易落地的重要工程环节。
