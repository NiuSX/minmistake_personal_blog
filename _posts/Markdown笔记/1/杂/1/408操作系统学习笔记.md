1. 进程与线程：CPU 如何在多个执行流之间分配。
2. 内存管理：程序如何获得连续、隔离、可扩展的地址空间。
3. 文件管理：外存如何抽象为文件和目录。
4. I/O 管理：设备如何被统一、缓冲、调度和保护。

408 高频题型：

- 进程状态转换、PV 操作、调度算法、死锁判断。
- 地址转换、页表/TLB、页面置换、缺页次数。
- 文件分配方式、索引结点、目录、磁盘调度。
- 中断、DMA、缓冲、SPOOLing、设备分配。

## 第 1 章 计算机系统概述

### 1.1 操作系统概念

操作系统是控制和管理计算机硬件与软件资源、合理组织计算机工作流程、为用户和应用程序提供接口的软件系统。

目标：

- 方便性；
- 有效性；
- 可扩充性；
- 开放性。

功能：

- 处理机管理；
- 存储器管理；
- 文件管理；
- 设备管理；
- 用户接口。

### 1.2 操作系统特征

四大特征：

- 并发：多个程序在同一时间段内交替推进。
- 共享：系统资源可被多个并发执行程序共同使用。
- 虚拟：把物理实体变成多个逻辑对应物，如虚拟内存、虚拟处理器。
- 异步：程序执行以不可预知速度推进。

并发和共享是最基本特征，二者互为存在条件。

### 1.3 操作系统发展

- 手工操作：无操作系统。
- 单道批处理：自动成批执行，资源利用率仍低。
- 多道批处理：多个作业驻留内存，提高 CPU 和 I/O 利用率。
- 分时系统：多个用户通过终端交互，强调响应时间。
- 实时系统：强调截止时间和可靠性。
- 网络/分布式系统：资源跨机器共享。
- 个人计算机、移动与云操作系统。

### 1.4 运行环境

CPU 运行模式：

- 用户态：普通程序运行，不能执行特权指令。
- 核心态：操作系统内核运行，可执行特权指令。

从用户态进入核心态的途径：

- 中断；
- 异常；
- 系统调用。

系统调用是用户程序请求 OS 服务的接口，如文件读写、进程创建、内存申请。

### 1.5 中断和异常

中断：来自 CPU 外部，如 I/O 完成、时钟中断。  
异常：来自 CPU 内部，如除零、缺页、非法指令。

中断处理流程：

1. 关中断或保存中断允许状态。
2. 保存断点和现场。
3. 转入中断服务程序。
4. 执行处理。
5. 恢复现场。
6. 中断返回。

### 1.6 操作系统结构

常见结构：

- 单体内核：效率高，模块耦合强。
- 分层结构：层次清晰，但层次设计困难。
- 微内核：只保留最基本机制，服务放用户态，可靠性好但通信开销大。
- 模块化内核：现代系统常用，兼顾效率和扩展性。

## 第 2 章 进程与线程

### 2.1 进程概念

进程是程序的一次执行过程，是系统进行资源分配和调度的基本单位。

进程组成：

- 程序段；
- 数据段；
- PCB。

PCB 是进程存在的唯一标志，包含：

- 进程标识；
- 处理机状态；
- 进程调度信息；
- 资源清单；
- 队列指针等。

### 2.2 进程状态

基本三态：

- 就绪：具备运行条件，等待 CPU。
- 运行：正在 CPU 上执行。
- 阻塞：等待某事件，如 I/O 完成。

扩展状态：

- 创建态；
- 终止态；
- 就绪挂起；
- 阻塞挂起。

状态转换：

- 就绪 -> 运行：调度。
- 运行 -> 就绪：时间片到或被抢占。
- 运行 -> 阻塞：等待事件。
- 阻塞 -> 就绪：等待事件发生。

阻塞不能直接变运行，必须先变就绪。

### 2.3 进程控制

原语：不可被中断的内核操作。

常见进程控制：

- 创建；
- 终止；
- 阻塞；
- 唤醒；
- 切换。

进程切换需要保存当前进程上下文并恢复新进程上下文，有时间开销。

### 2.4 进程通信

低级通信：

- PV 操作、信号量，只传递少量控制信息。

高级通信：

- 共享存储：速度快，需要同步互斥。
- 消息传递：直接/间接通信。
- 管道通信：半双工，先进先出。

### 2.5 线程

线程是 CPU 调度的基本单位。进程是资源分配的基本单位。

线程共享进程资源：

- 地址空间；
- 打开文件；
- 全局变量。

线程私有：

- 线程 ID；
- 程序计数器；
- 寄存器；
- 栈。

多线程模型：

- 多对一：多个用户线程映射到一个内核线程。
- 一对一：每个用户线程对应一个内核线程。
- 多对多：多个用户线程映射到多个内核线程。

### 2.6 CPU 调度

调度层次：

- 高级调度：作业调度，从外存后备队列选作业进内存。
- 中级调度：内存与外存之间挂起/激活。
- 低级调度：进程调度，选就绪进程上 CPU。

调度指标：

- CPU 利用率；
- 系统吞吐量；
- 周转时间；
- 带权周转时间；
- 等待时间；
- 响应时间。

周转时间：

\[
T=完成时间-提交时间
\]

带权周转时间：

\[
W=\frac{周转时间}{实际运行时间}
\]

### 2.7 调度算法

#### FCFS

先来先服务。  
非抢占。  
对长作业有利，对短作业不利。

#### SJF/SPF

短作业优先。  
平均等待时间短，但可能导致长作业饥饿。

#### HRRN

高响应比优先：

\[
响应比=\frac{等待时间+要求服务时间}{要求服务时间}
\]

兼顾等待时间和服务时间，非抢占。

#### 时间片轮转 RR

适合分时系统。时间片过大退化为 FCFS，过小则切换开销大。

#### 优先级调度

可抢占或非抢占。可能饥饿，可用动态优先级或老化解决。

#### 多级反馈队列

兼顾短作业响应和长作业推进。新进程先进入高优先级短时间片队列，用完时间片后降级。

### 2.8 同步与互斥

临界资源：一次只允许一个进程使用的资源。  
临界区：访问临界资源的代码段。

同步：进程间因合作产生的执行顺序约束。  
互斥：多个进程不能同时进入临界区。

临界区准则：

- 空闲让进；
- 忙则等待；
- 有限等待；
- 让权等待。

### 2.9 信号量与 PV

信号量 \(S\) 是整型变量，只能通过 P/V 原语访问。

P 操作：

```text
S = S - 1
if S < 0: block
```

V 操作：

```text
S = S + 1
if S <= 0: wakeup
```

互斥信号量初值通常为 1。  
资源信号量初值为资源数量。  
同步信号量初值常为 0。

### 2.10 经典同步问题

#### 生产者-消费者

信号量：

- `mutex=1`
- `empty=n`
- `full=0`

生产者：P(empty), P(mutex), 放入, V(mutex), V(full)。  
消费者：P(full), P(mutex), 取出, V(mutex), V(empty)。

注意：同步 P 操作通常在互斥 P 操作之前，避免死锁。

#### 读者-写者

允许多个读者同时读；写者必须互斥访问。可设计读者优先、写者优先或公平策略。

#### 哲学家进餐

解决死锁方法：

- 最多允许 4 个哲学家同时拿筷子；
- 奇偶哲学家拿筷子顺序不同；
- 同时申请两只筷子；
- 引入服务员。

### 2.11 管程

管程把共享变量和对其操作封装在一起，保证任一时刻只有一个进程在管程内执行。

条件变量用于进程等待和唤醒：

- `wait`
- `signal`

管程比信号量更结构化，减少 PV 误用。

### 2.12 死锁

死锁必要条件：

1. 互斥。
2. 不剥夺。
3. 请求并保持。
4. 循环等待。

处理策略：

- 死锁预防：破坏必要条件。
- 死锁避免：动态判断是否安全，如银行家算法。
- 死锁检测和解除。
- 鸵鸟策略：忽略。

安全状态：系统能按某种顺序满足所有进程最大需求，使它们依次完成。安全一定无死锁；不安全不一定已经死锁。

银行家算法核心：

- `Available`
- `Max`
- `Allocation`
- `Need = Max - Allocation`

判断是否存在安全序列。

## 第 3 章 内存管理

### 3.1 内存管理功能

- 内存空间分配与回收；
- 地址转换；
- 内存保护；
- 内存共享；
- 内存扩充。

地址：

- 逻辑地址：程序生成的地址。
- 物理地址：内存实际地址。

地址重定位：

- 静态重定位：装入时完成。
- 动态重定位：运行时由硬件完成，需重定位寄存器。

### 3.2 连续分配

#### 单一连续分配

内存分系统区和用户区，用户区只装一个程序。简单但利用率低。

#### 固定分区

内存划分为固定大小分区。  
有内部碎片。

#### 动态分区

按进程大小动态分配连续空间。  
有外部碎片。

分配算法：

- 首次适应 FF：从头找第一个足够大的空闲区。
- 邻近适应 NF：从上次位置继续找。
- 最佳适应 BF：找最小足够空闲区，易产生小碎片。
- 最坏适应 WF：找最大空闲区。

### 3.3 分页管理

把逻辑地址空间分为页，物理内存分为页框，大小相同。

逻辑地址：

\[
页号P + 页内偏移W
\]

物理地址：

\[
页框号F + 页内偏移W
\]

页大小为 \(2^k\)，则低 \(k\) 位为页内偏移。

地址转换：

1. 由页号查页表。
2. 得到页框号。
3. 拼接页内偏移得到物理地址。

分页无外部碎片，但有内部碎片。

### 3.4 TLB

快表 TLB 缓存页表项。

有效访问时间：

\[
EAT=\alpha(t_{TLB}+t_M)+(1-\alpha)(t_{TLB}+2t_M)
\]

其中 \(\alpha\) 为 TLB 命中率，\(t_M\) 为一次内存访问时间。若题目给定是否并行查找，公式要调整。

### 3.5 分段管理

按程序逻辑模块划分段，如代码段、数据段、栈段。

逻辑地址：

\[
段号S + 段内偏移W
\]

段表项包含段长和基址。地址转换需检查段内偏移是否越界。

分段便于共享和保护，但有外部碎片。

### 3.6 段页式管理

先分段，段内再分页。  
逻辑地址：段号、页号、页内偏移。  
地址转换需查段表和页表，通常配合 TLB。

### 3.7 虚拟内存

基础：

- 请求调入；
- 置换；
- 离散分配；
- 多次性、对换性、虚拟性。

请求分页页表项通常包含：

- 页框号；
- 状态位/存在位；
- 访问字段；
- 修改位；
- 外存地址。

缺页处理：

1. 访问页表发现页不在内存。
2. 产生缺页异常。
3. OS 找空闲页框或选择淘汰页。
4. 若淘汰页被修改，写回外存。
5. 调入目标页。
6. 修改页表。
7. 重新执行被中断指令。

### 3.8 页面置换算法

#### OPT

淘汰未来最长时间不再访问的页。理论最优，实际不可实现。

#### FIFO

淘汰最早进入内存的页。可能出现 Belady 异常。

#### LRU

淘汰最长时间未访问的页。性能好，实现成本高。

#### CLOCK

用访问位近似 LRU。指针扫描，访问位为 1 则置 0 并跳过，为 0 则淘汰。

改进 CLOCK 同时考虑访问位和修改位。

### 3.9 抖动与工作集

抖动：频繁缺页导致系统大量时间用于页面换入换出。

原因：

- 分配页框过少；
- 多道程序度过高；
- 局部性被破坏。

工作集：某段时间窗口内进程实际访问的页面集合。给进程分配不小于工作集大小的页框可减少抖动。

## 第 4 章 文件管理

### 4.1 文件概念

文件是以名字标识的一组相关信息集合。

文件属性：

- 文件名；
- 标识符；
- 类型；
- 位置；
- 大小；
- 保护信息；
- 时间、日期和用户标识。

文件操作：

- 创建；
- 删除；
- 打开；
- 关闭；
- 读；
- 写；
- 定位。

### 4.2 文件逻辑结构

- 无结构文件：字节流。
- 有结构文件：记录式文件。

记录式文件：

- 顺序文件；
- 索引文件；
- 索引顺序文件；
- 直接文件/散列文件。

### 4.3 文件物理结构

#### 连续分配

文件占连续磁盘块。  
优点：顺序和随机访问快。  
缺点：外部碎片，文件扩展困难。

#### 链接分配

文件块通过指针链接。  
优点：无外部碎片，扩展方便。  
缺点：随机访问差，指针可靠性问题。

FAT 把链接指针集中放在文件分配表中。

#### 索引分配

为每个文件建立索引块，记录所有数据块地址。  
支持随机访问，无外部碎片。  
大文件可用多级索引、混合索引。

### 4.4 目录

目录项包含文件名和文件控制信息，或文件名与索引结点指针。

目录结构：

- 单级目录：简单，重名问题。
- 两级目录：按用户分目录。
- 多级树形目录：现代系统常用。
- 无环图目录：支持共享。

路径：

- 绝对路径：从根目录开始。
- 相对路径：从当前目录开始。

### 4.5 文件共享与保护

共享方式：

- 硬链接：多个目录项指向同一 inode。
- 符号链接：保存目标路径名的特殊文件。

硬链接不能轻易跨文件系统，删除一个目录项不一定删除文件数据；当链接计数为 0 且无进程打开时才释放。

保护方式：

- 访问控制表 ACL；
- 口令；
- 权限位，如读/写/执行。

### 4.6 文件系统布局

磁盘上通常包括：

- 引导块；
- 超级块；
- 空闲空间管理结构；
- inode 区；
- 数据区。

空闲空间管理：

- 位示图；
- 空闲链表；
- 成组链接法。

## 第 5 章 I/O 管理

### 5.1 I/O 设备

按传输速率：

- 低速设备：键盘、鼠标。
- 中速设备：打印机。
- 高速设备：磁盘、网络。

按信息交换单位：

- 字符设备；
- 块设备。

### 5.2 I/O 控制方式

程序直接控制：CPU 忙等。  
中断驱动：设备完成后中断 CPU。  
DMA：设备和内存直接传输。  
通道：专用 I/O 处理机，进一步减轻 CPU 负担。

### 5.3 I/O 软件层次

- 用户层 I/O 软件；
- 设备独立性软件；
- 设备驱动程序；
- 中断处理程序；
- 硬件。

设备独立性好处：

- 用户使用统一接口；
- 设备更换不影响应用；
- 便于缓冲、分配、保护。

### 5.4 缓冲

缓冲目的：

- 缓和 CPU 与 I/O 速度不匹配；
- 减少中断次数；
- 提高并行性；
- 解决数据粒度不匹配。

类型：

- 单缓冲；
- 双缓冲；
- 循环缓冲；
- 缓冲池。

### 5.5 设备分配

设备类型：

- 独占设备；
- 共享设备；
- 虚拟设备。

SPOOLing 技术用磁盘模拟独占设备，把独占设备改造成可共享的虚拟设备，如打印机假脱机。

### 5.6 磁盘调度

常见算法：

- FCFS：公平但性能一般。
- SSTF：选最近磁道，可能饥饿。
- SCAN：电梯算法，来回扫描。
- C-SCAN：单向扫描，等待时间更均匀。
- LOOK/C-LOOK：只扫到最远请求处。

题目常要求计算磁头移动总道数。

## 高频公式速查

\[
周转时间=完成时间-提交时间
\]

\[
带权周转时间=\frac{周转时间}{运行时间}
\]

\[
响应比=\frac{等待时间+服务时间}{服务时间}
\]

\[
页号=\left\lfloor \frac{逻辑地址}{页大小}\right\rfloor,\quad 页内偏移=逻辑地址 \bmod 页大小
\]

\[
EAT=\alpha(t_{TLB}+t_M)+(1-\alpha)(t_{TLB}+2t_M)
\]

## 高频易错点

1. 进程是资源分配单位，线程是 CPU 调度单位。
2. 阻塞态不能直接到运行态。
3. PV 中同步信号量和互斥信号量顺序不能乱。
4. 死锁四条件是必要条件，不是充分条件。
5. 安全状态一定无死锁，不安全状态不一定死锁。
6. 分页有内部碎片，分段有外部碎片。
7. TLB 未命中不等于缺页。
8. FIFO 页面置换可能有 Belady 异常。
9. 硬链接和符号链接删除语义不同。
10. DMA 传输仍需 CPU 初始化和结束处理。
