## 第 1 章 绪论

### 1.1 基本概念

数据结构包括三方面：

- **逻辑结构**：数据元素之间的逻辑关系，如集合、线性结构、树形结构、图结构。
- **存储结构**：逻辑关系在计算机中的表示，如顺序存储、链式存储、索引存储、散列存储。
- **数据运算**：在结构上定义的操作，如插入、删除、查找、遍历、排序。

常见术语：

- 数据：信息的载体。
- 数据元素：数据的基本单位。
- 数据项：构成数据元素的不可分割最小单位。
- 数据对象：性质相同的数据元素集合。
- 抽象数据类型 ADT：数据对象、数据关系和基本操作的抽象封装。

### 1.2 逻辑结构

- 集合：元素之间除“同属一个集合”外无其他关系。
- 线性结构：除首尾外，每个元素有唯一前驱和唯一后继。
- 树形结构：一对多层次关系。
- 图结构：多对多关系。

注意：逻辑结构与存储结构不是一一对应。线性表可顺序存储，也可链式存储；树可链式存储，也可用数组顺序存储。

### 1.3 算法特性

算法五个重要特性：

- 有穷性：有限步后结束。
- 确定性：每条指令含义明确。
- 可行性：可由基本操作实现。
- 输入：0 个或多个输入。
- 输出：1 个或多个输出。

好算法标准：

- 正确性；
- 可读性；
- 健壮性；
- 高效率与低存储量。

### 1.4 复杂度

时间复杂度只保留最高阶，并忽略常数：

- \(O(1)\)：常数阶。
- \(O(\log n)\)：对半缩小，如折半查找。
- \(O(n)\)：单层循环。
- \(O(n\log n)\)：分治排序常见。
- \(O(n^2)\)：双层循环。
- \(O(2^n)\)、\(O(n!)\)：指数/阶乘爆炸。

常见求和：

\[
1+2+\cdots+n=\frac{n(n+1)}{2}=O(n^2)
\]

\[
1+2+4+\cdots+2^k=2^{k+1}-1
\]

若循环变量每次乘/除常数，通常是 \(O(\log n)\)。

## 第 2 章 线性表

### 2.1 线性表定义

线性表是具有相同数据类型的 \(n\) 个数据元素的有限序列：

\[
L=(a_1,a_2,\dots,a_n)
\]

特点：

- 有唯一表头和表尾；
- 除首元素外，每个元素有唯一直接前驱；
- 除尾元素外，每个元素有唯一直接后继。

### 2.2 顺序表

顺序表用一组地址连续的存储单元依次存储线性表元素。若第一个元素地址为 \(LOC(A)\)，每个元素占 \(l\) 个存储单元，则：

\[
LOC(a_i)=LOC(A)+(i-1)l
\]

优点：

- 支持随机访问，按位查找 \(O(1)\)。
- 存储密度高，无指针开销。

缺点：

- 插入、删除需要移动大量元素，平均 \(O(n)\)。
- 需要预分配连续空间，扩容代价高。

顺序表插入第 \(i\) 个位置：

1. 判断 \(i\) 是否合法。
2. 判断表是否已满。
3. 从后向前移动第 \(i\) 到第 \(n\) 个元素。
4. 放入新元素。
5. 表长加 1。

顺序表删除第 \(i\) 个位置：

1. 判断 \(i\) 是否合法。
2. 保存被删元素。
3. 从前向后移动第 \(i+1\) 到第 \(n\) 个元素。
4. 表长减 1。

### 2.3 链表

链表通过指针表示元素之间的逻辑关系。

#### 单链表

结点结构：

```c
typedef struct LNode {
    ElemType data;
    struct LNode *next;
} LNode, *LinkList;
```

带头结点的好处：

- 统一空表和非空表操作；
- 插入/删除首元结点不需要特殊处理；
- 头指针始终指向头结点。

按位查找：从头扫描，\(O(n)\)。  
按值查找：顺序比较，\(O(n)\)。  
已知前驱结点插入/删除：\(O(1)\)。  
若只知道被删结点，一般不能直接删除尾结点；非尾结点可用后继覆盖法。

#### 双链表

结点有前驱和后继指针：

```c
typedef struct DNode {
    ElemType data;
    struct DNode *prior, *next;
} DNode, *DLinkList;
```

优点：可双向遍历，删除结点更方便。  
缺点：指针域更多，空间开销大，修改指针时更容易出错。

#### 循环链表

尾结点指针指向头结点或首元结点。适合循环调度、约瑟夫问题等。循环单链表若只设尾指针，可 \(O(1)\) 找到头结点和尾结点。

#### 静态链表

用数组模拟链表，指针用数组下标表示。适合不支持指针的语言或固定内存池。

### 2.4 顺序表与链表比较

| 比较项 | 顺序表 | 链表 |
|---|---|---|
| 存储空间 | 连续 | 可离散 |
| 随机访问 | \(O(1)\) | \(O(n)\) |
| 插入删除 | 平均 \(O(n)\) | 已知位置可 \(O(1)\) |
| 空间开销 | 无指针 | 有指针域 |
| 扩容 | 可能搬移 | 动态申请 |
| 适合场景 | 查找多、长度稳定 | 插删多、长度变化大 |

## 第 3 章 栈、队列和数组

### 3.1 栈

栈是只允许在一端插入和删除的线性表。插入和删除端称为栈顶，另一端称为栈底。

特点：后进先出 LIFO。

常见应用：

- 括号匹配；
- 表达式求值；
- 递归调用；
- 深度优先搜索；
- 函数调用栈；
- 进制转换。

顺序栈：

- 栈空：`top == -1` 或 `top == 0`，取决于定义。
- 栈满：`top == MaxSize - 1`。

共享栈：两个栈共享同一数组，从两端向中间增长。

### 3.2 队列

队列只允许一端入队，另一端出队。  
特点：先进先出 FIFO。

顺序循环队列常见约定：

- 队空：`front == rear`
- 队满：`(rear + 1) % MaxSize == front`
- 队长：`(rear - front + MaxSize) % MaxSize`

为了区分队空和队满，常牺牲一个存储单元。也可增加 `size` 或 `tag` 标志。

链队列通常带头结点，入队改尾指针，出队改头结点后继。删除最后一个元素时，要把尾指针重新指向头结点。

### 3.3 栈和队列的应用

#### 括号匹配

扫描字符串：

- 左括号入栈；
- 右括号与栈顶匹配；
- 不匹配或空栈则失败；
- 扫描结束栈空则成功。

#### 表达式求值

中缀表达式转后缀表达式：

- 操作数直接输出；
- 左括号入栈；
- 右括号弹出直到左括号；
- 运算符按优先级和结合性处理。

后缀表达式求值：

- 操作数入栈；
- 遇运算符弹出操作数计算，再把结果入栈。

#### 递归

递归三要素：

- 递归定义；
- 递归边界；
- 递归推进方向。

递归本质依赖运行时栈。递归可读性好，但可能有栈溢出和重复计算问题。

### 3.4 数组和特殊矩阵

二维数组按行优先存储：

\[
LOC(a_{ij})=LOC(a_{00})+(i\times n+j)L
\]

按列优先：

\[
LOC(a_{ij})=LOC(a_{00})+(j\times m+i)L
\]

特殊矩阵压缩：

- 对称矩阵：只存上三角或下三角。
- 三角矩阵：只存非零三角区和一个常量。
- 对角矩阵：按带状存储。
- 稀疏矩阵：三元组表或十字链表。

## 第 4 章 串

串是由零个或多个字符组成的有限序列。

### 4.1 存储结构

- 定长顺序存储：数组长度固定。
- 堆分配存储：动态分配。
- 块链存储：链表每个结点存多个字符。

### 4.2 模式匹配

朴素匹配最坏时间复杂度 \(O(nm)\)。

KMP 核心是利用模式串自身信息，匹配失败时不回退主串指针。

`next[j]` 的含义：模式串 `P[1..j-1]` 中最长相等前后缀长度加一，或按教材定义略有差异。复习时必须统一下标体系。

KMP 复杂度：

- 求 next：\(O(m)\)
- 匹配：\(O(n)\)

易错点：

- `next` 与 `nextval` 定义不同。
- 下标从 0 还是 1 开始会改变数组值。
- 手算时先写出每一位之前的子串，避免凭感觉跳。

## 第 5 章 树与二叉树

### 5.1 树的基本概念

树是 \(n\) 个结点的有限集合。若 \(n=0\)，为空树；否则有唯一根结点，其余结点分成若干互不相交的子树。

常见术语：

- 结点的度：子树个数。
- 树的度：各结点度的最大值。
- 叶结点：度为 0 的结点。
- 分支结点：度不为 0 的结点。
- 层次：根为第 1 层。
- 高度/深度：最大层数。
- 路径长度：路径上边的条数。

树中结点数与边数关系：

\[
边数 = 结点数 - 1
\]

### 5.2 二叉树性质

二叉树每个结点最多有两棵子树，且左右子树有序。

重要性质：

- 第 \(i\) 层最多有 \(2^{i-1}\) 个结点。
- 高度为 \(h\) 的二叉树最多有 \(2^h-1\) 个结点。
- 对任意非空二叉树，若叶结点数为 \(n_0\)，度为 2 的结点数为 \(n_2\)，则：

\[
n_0=n_2+1
\]

满二叉树：高度为 \(h\)，结点数为 \(2^h-1\)。  
完全二叉树：除最后一层外均满，最后一层从左到右连续。

完全二叉树顺序编号性质：

- 父结点：\(\lfloor i/2 \rfloor\)
- 左孩子：\(2i\)
- 右孩子：\(2i+1\)

### 5.3 二叉树遍历

- 先序：根、左、右。
- 中序：左、根、右。
- 后序：左、右、根。
- 层次：从上到下、从左到右。

由先序+中序可唯一确定二叉树。  
由后序+中序可唯一确定二叉树。  
由先序+后序一般不能唯一确定二叉树，除非每个内部结点都有两个孩子等附加条件。

非递归遍历：

- 先序：栈保存待访问右/左子树。
- 中序：沿左链入栈，弹出访问后转右子树。
- 后序：可用双栈或记录上次访问结点。

### 5.4 线索二叉树

线索化利用空指针域存放前驱/后继信息。

- 中序线索二叉树最常考。
- `ltag=0` 表示 `lchild` 指向左孩子，`ltag=1` 表示指向前驱。
- `rtag=0` 表示 `rchild` 指向右孩子，`rtag=1` 表示指向后继。

线索二叉树适合频繁按某种遍历序列找前驱/后继。

### 5.5 树、森林与二叉树转换

树转二叉树规则：左孩子右兄弟。

- 树的第一个孩子作为二叉树左孩子；
- 兄弟结点作为右孩子链。

森林转二叉树：

- 每棵树先转二叉树；
- 各树根作为右兄弟连接。

遍历对应：

- 树的先根遍历对应二叉树先序遍历。
- 树的后根遍历对应二叉树中序遍历。
- 森林的先序对应二叉树先序。
- 森林的中序对应二叉树中序。

### 5.6 哈夫曼树

带权路径长度：

\[
WPL=\sum_{i=1}^{n} w_i l_i
\]

哈夫曼树构造：

1. 每次选取权值最小的两棵树合并。
2. 新树权值为两者之和。
3. 重复直到只剩一棵树。

哈夫曼编码特点：

- 前缀编码：任一编码不是另一个编码的前缀。
- 权值大的字符编码短，权值小的字符编码长。
- 用于最优二叉前缀编码。

### 5.7 并查集

并查集支持：

- `Find(x)`：查找元素所属集合代表。
- `Union(x,y)`：合并两个集合。

优化：

- 路径压缩；
- 按秩/按大小合并。

应用：

- 连通分量；
- Kruskal 最小生成树；
- 等价类合并。

## 第 6 章 图

### 6.1 基本概念

图 \(G=(V,E)\)，其中 \(V\) 是顶点集，\(E\) 是边集。

分类：

- 无向图/有向图；
- 带权图/无权图；
- 简单图/多重图；
- 连通图/非连通图；
- 强连通图。

无向图所有顶点度数之和：

\[
\sum Degree(v)=2|E|
\]

有向图：

\[
\sum 入度=\sum 出度=|E|
\]

### 6.2 图的存储

#### 邻接矩阵

适合稠密图。空间 \(O(n^2)\)。判断两点是否有边 \(O(1)\)。

无向图邻接矩阵对称。  
顶点 \(i\) 的度为第 \(i\) 行或列非零元素个数。

#### 邻接表

适合稀疏图。空间 \(O(n+e)\)。查找某条边最坏需扫描链表。

无向图每条边在邻接表中出现两次；有向图可用出边表或逆邻接表。

### 6.3 BFS 与 DFS

BFS：

- 用队列；
- 按层扩展；
- 可求无权图最短路径；
- 时间 \(O(n+e)\)。

DFS：

- 用递归或栈；
- 一条路走到底再回溯；
- 可用于连通分量、拓扑排序、环检测；
- 时间 \(O(n+e)\)。

邻接矩阵遍历时间 \(O(n^2)\)，邻接表为 \(O(n+e)\)。

### 6.4 最小生成树

生成树：包含图中全部顶点的极小连通子图。\(n\) 个顶点的生成树有 \(n-1\) 条边。

#### Prim

从一个顶点集合开始，每次选连接集合内外的最小边。

适合稠密图。邻接矩阵实现 \(O(n^2)\)。

#### Kruskal

按边权从小到大排序，依次选不形成回路的边。用并查集判环。

适合稀疏图。复杂度主要由排序决定，约 \(O(e\log e)\)。

### 6.5 最短路径

#### Dijkstra

求单源最短路径，要求边权非负。

步骤：

1. 初始化源点距离为 0，其余为无穷。
2. 每次选择未确定顶点中距离最小者。
3. 用该顶点松弛邻边。

不能处理负权边。

#### Floyd

求所有顶点对最短路径。

\[
A^{(k)}[i][j]=\min(A^{(k-1)}[i][j], A^{(k-1)}[i][k]+A^{(k-1)}[k][j])
\]

时间 \(O(n^3)\)，空间 \(O(n^2)\)。可处理负权边，但不能有负权回路。

### 6.6 拓扑排序

有向无环图 DAG 可进行拓扑排序。

算法：

1. 选择入度为 0 的顶点输出。
2. 删除该顶点及其出边。
3. 重复。

若过程中找不到入度为 0 的顶点但仍有顶点未输出，则图中有环。

### 6.7 关键路径

AOE 网中：

- 顶点表示事件；
- 边表示活动；
- 边权表示活动持续时间。

关键路径是从源点到汇点的最长路径。关键活动延误会导致整个工程延误。

计算：

- 事件最早发生时间 \(ve\)；
- 事件最迟发生时间 \(vl\)；
- 活动最早开始时间 \(e\)；
- 活动最迟开始时间 \(l\)；
- 若 \(e=l\)，该活动为关键活动。

## 第 7 章 查找

### 7.1 顺序查找

适用于顺序表、链表、有序或无序表。平均查找长度：

- 成功等概率：\((n+1)/2\)
- 不成功：通常为 \(n+1\) 或 \(n\)，看是否设置哨兵和判定方式。

哨兵可减少边界判断。

### 7.2 折半查找

要求顺序存储且有序。

时间复杂度 \(O(\log n)\)。  
判定树高度：

\[
h=\lceil \log_2(n+1)\rceil
\]

折半查找不适合链表。

### 7.3 分块查找

索引表有序，块内可无序。查找过程：

1. 在索引表中确定块。
2. 在块内顺序查找。

兼顾动态插删和查找效率。

### 7.4 二叉排序树 BST

性质：

- 左子树所有结点关键字小于根；
- 右子树所有结点关键字大于根；
- 左右子树也为 BST。

中序遍历得到递增序列。

查找、插入、删除平均 \(O(\log n)\)，最坏退化为 \(O(n)\)。

删除：

- 叶结点：直接删。
- 只有一个子树：用子树替代。
- 有两个子树：用中序前驱或后继替代，再删除前驱/后继结点。

### 7.5 平衡二叉树 AVL

平衡因子：

\[
BF=左子树高度-右子树高度
\]

AVL 要求每个结点 BF 只能为 -1、0、1。

插入后失衡类型：

- LL：右旋。
- RR：左旋。
- LR：先左旋再右旋。
- RL：先右旋再左旋。

### 7.6 B 树与 B+ 树

B 树适合外存索引。m 阶 B 树性质：

- 每个结点最多 m 棵子树。
- 除根外，非叶结点至少 \(\lceil m/2\rceil\) 棵子树。
- 所有叶结点在同一层。
- 结点关键字有序。

B+ 树：

- 非叶结点只起索引作用；
- 数据记录都在叶结点；
- 叶结点按关键字顺序链接；
- 更适合范围查询和数据库索引。

### 7.7 散列表

散列函数把关键字映射到地址：

\[
addr=H(key)
\]

冲突不可避免，需处理冲突。

常见散列函数：

- 直接定址法；
- 除留余数法；
- 数字分析法；
- 平方取中法；
- 折叠法。

冲突处理：

- 开放定址法：线性探测、平方探测、伪随机探测。
- 拉链法：同义词链表。

装填因子：

\[
\alpha=\frac{表中记录数}{散列表长度}
\]

装填因子越大，冲突概率越高。

## 第 8 章 排序

### 8.1 排序稳定性

若关键字相等的元素排序后相对次序不变，则排序算法稳定。

稳定性在“按多个关键字排序”时重要。

### 8.2 插入排序

#### 直接插入排序

每次将一个元素插入前面有序序列。

- 最好 \(O(n)\)：原序列有序。
- 平均/最坏 \(O(n^2)\)。
- 稳定。

#### 折半插入排序

用折半查找确定插入位置，减少比较次数，但移动次数不变，仍为 \(O(n^2)\)。

#### 希尔排序

按增量分组插入排序，逐步缩小增量到 1。  
不稳定。复杂度与增量序列有关。

### 8.3 交换排序

#### 冒泡排序

相邻元素比较交换。若一趟无交换，可提前结束。

- 最好 \(O(n)\)。
- 平均/最坏 \(O(n^2)\)。
- 稳定。

#### 快速排序

选枢轴，一趟划分使左边小于枢轴、右边大于枢轴，再递归。

- 平均 \(O(n\log n)\)。
- 最坏 \(O(n^2)\)：每次划分极不平衡。
- 不稳定。
- 递归栈平均 \(O(\log n)\)，最坏 \(O(n)\)。

改进：随机选枢轴、三数取中、小区间插入排序。

### 8.4 选择排序

#### 简单选择排序

每趟选最小元素放到前面。

- 总比较次数固定 \(O(n^2)\)。
- 移动次数少。
- 不稳定。

#### 堆排序

堆是完全二叉树。

大根堆：每个结点大于等于左右孩子。  
小根堆：每个结点小于等于左右孩子。

堆排序流程：

1. 建初堆。
2. 堆顶与末尾元素交换。
3. 调整剩余元素为堆。
4. 重复。

复杂度：

- 建堆 \(O(n)\)。
- 排序 \(O(n\log n)\)。
- 不稳定。
- 空间 \(O(1)\)。

### 8.5 归并排序

将序列分成两半分别排序，再合并。

- 时间 \(O(n\log n)\)。
- 空间 \(O(n)\)。
- 稳定。

常用于外部排序和链表排序。

### 8.6 基数排序

按关键字各位进行分配和收集。适合关键字可分解且范围有限的情况。

- LSD：从低位到高位。
- MSD：从高位到低位。

稳定性依赖每一趟分配收集是否稳定。

### 8.7 排序算法总表

| 算法 | 平均时间 | 最坏时间 | 空间 | 稳定 |
|---|---:|---:|---:|---|
| 直接插入 | \(O(n^2)\) | \(O(n^2)\) | \(O(1)\) | 是 |
| 折半插入 | \(O(n^2)\) | \(O(n^2)\) | \(O(1)\) | 是 |
| 希尔 | 依增量 | 依增量 | \(O(1)\) | 否 |
| 冒泡 | \(O(n^2)\) | \(O(n^2)\) | \(O(1)\) | 是 |
| 快排 | \(O(n\log n)\) | \(O(n^2)\) | \(O(\log n)\) | 否 |
| 简单选择 | \(O(n^2)\) | \(O(n^2)\) | \(O(1)\) | 否 |
| 堆排序 | \(O(n\log n)\) | \(O(n\log n)\) | \(O(1)\) | 否 |
| 归并 | \(O(n\log n)\) | \(O(n\log n)\) | \(O(n)\) | 是 |
| 基数 | \(O(d(n+r))\) | \(O(d(n+r))\) | \(O(r)\) | 是 |

## 高频易错点

1. 顺序表插入删除要注意移动方向：插入从后往前，删除从前往后。
2. 单链表删除结点时要找前驱；双链表要同时改四个方向相关指针。
3. 循环队列队满条件通常牺牲一个单元。
4. KMP 的 `next` 值因下标定义不同而不同，答题要跟题目定义一致。
5. 二叉树“度为 2 的结点数 + 1 = 叶结点数”只适用于非空二叉树。
6. 先序+后序不能唯一确定普通二叉树。
7. 邻接矩阵和邻接表的遍历复杂度不同。
8. Dijkstra 不能处理负权边。
9. 拓扑排序只适用于 DAG。
10. 快排平均快，但最坏可能 \(O(n^2)\)，且不稳定。
