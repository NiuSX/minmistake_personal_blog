# 408 计算机组成原理学习笔记

> 目标：把“数据表示、存储、指令、CPU、总线、I/O”串成一条机器执行程序的主线。

## 总体框架

计算机组成原理研究一台计算机如何由硬件部件协同执行程序。核心链路：

程序和数据以二进制形式存入存储器，CPU 按 PC 取指、译码、执行，通过 ALU 运算、寄存器暂存、Cache/主存交换、总线传输和 I/O 系统完成输入输出。

408 常考模块：

1. 计算机系统概述：层次结构、性能指标、程序执行过程。
2. 数据表示与运算：进制、补码、定点/浮点、溢出、ALU。
3. 存储系统：SRAM/DRAM、主存扩展、Cache、虚拟存储器。
4. 指令系统：指令格式、寻址方式、CISC/RISC、机器级代码。
5. CPU：数据通路、控制器、指令周期、流水线、冒险。
6. 总线：总线结构、仲裁、定时。
7. I/O：I/O 接口、中断、DMA、外设。

## 第 1 章 计算机系统概述

### 1.1 计算机系统层次

计算机系统由硬件和软件组成。典型层次：

- 微体系结构层：寄存器、ALU、数据通路、控制信号。
- 指令集体系结构 ISA：机器指令、寄存器、寻址方式、异常模型。
- 操作系统层：进程、内存、文件、I/O 抽象。
- 汇编语言层。
- 高级语言层。
- 应用层。

组成原理主要关注硬件如何支持 ISA 和程序执行。

### 1.2 冯·诺依曼结构

基本思想：

- 程序和数据以二进制形式存储。
- 程序按地址顺序执行，必要时由转移指令改变顺序。
- 计算机由运算器、控制器、存储器、输入设备、输出设备组成。

现代计算机仍以存储程序思想为基础，但引入 Cache、流水线、多核、虚拟存储等机制。

### 1.3 性能指标

常见指标：

- 机器字长：CPU 一次能处理的二进制位数。
- 数据通路宽度：数据总线一次传送位数。
- 主存容量：可存储信息总量。
- 主频：时钟频率。
- CPI：每条指令平均时钟周期数。
- MIPS：每秒百万条指令。
- FLOPS：每秒浮点运算次数。

CPU 执行时间：

\[
CPU时间 = 指令条数 \times CPI \times 时钟周期
\]

\[
CPU时间 = \frac{指令条数 \times CPI}{主频}
\]

MIPS：

\[
MIPS=\frac{主频}{CPI\times 10^6}
\]

注意：MIPS 不能跨不同 ISA 简单比较，因为“指令条数”和“指令能力”不同。

## 第 2 章 数据的表示和运算

### 2.1 进位计数制

任意 \(r\) 进制数：

\[
(K_nK_{n-1}\dots K_0.K_{-1}\dots K_{-m})_r
=\sum K_i r^i
\]

常见转换：

- 二进制转八进制：每 3 位一组。
- 二进制转十六进制：每 4 位一组。
- 十进制整数转二进制：除 2 取余，逆序。
- 十进制小数转二进制：乘 2 取整，顺序。

### 2.2 BCD 与字符编码

BCD 用 4 位二进制表示 1 位十进制数。8421 BCD 最常见。

ASCII：

- 数字字符 `'0'` 到 `'9'` 编码连续。
- 大写字母和小写字母编码差 32。

### 2.3 定点数编码

#### 原码

最高位为符号位，数值位表示绝对值。  
优点直观；缺点加减运算复杂，有 +0 和 -0。

#### 反码

正数反码等于原码；负数符号位不变，数值位取反。  
也有 +0 和 -0。

#### 补码

正数补码等于原码；负数补码为反码加 1。  
n 位补码表示范围：

\[
-2^{n-1} \sim 2^{n-1}-1
\]

补码优点：

- 0 唯一；
- 加减法统一；
- 符号位可参与运算。

#### 移码

常用于浮点阶码：

\[
移码 = 真值 + 偏置值
\]

8 位移码偏置常为 128，IEEE 754 单精度阶码偏置为 127。

### 2.4 溢出判断

补码加法溢出：

- 两个正数相加结果为负，正溢出。
- 两个负数相加结果为正，负溢出。

也可用符号位进位判断：

\[
OF=C_{符号位进位}\oplus C_{最高数值位进位}
\]

无符号数关注进位 CF；有符号补码关注溢出 OF。

### 2.5 定点运算

移位：

- 逻辑左移/右移：空位补 0。
- 算术右移：符号位保持不变。
- 算术左移：低位补 0，但可能溢出。

补码乘除法常考 Booth 算法思想、移位加减过程和符号处理。复习重点是理解补码算术统一性，而不是死背每一步。

### 2.6 浮点数

浮点数一般表示为：

\[
N=(-1)^S \times M \times R^E
\]

IEEE 754 单精度：

- 1 位符号位；
- 8 位阶码，偏置 127；
- 23 位尾数小数部分，规格化数隐含最高位 1。

双精度：

- 1 位符号；
- 11 位阶码，偏置 1023；
- 52 位尾数小数部分。

特殊值：

- 阶码全 0、尾数全 0：零。
- 阶码全 0、尾数非 0：非规格化数。
- 阶码全 1、尾数全 0：无穷大。
- 阶码全 1、尾数非 0：NaN。

浮点加减步骤：

1. 对阶。
2. 尾数加减。
3. 规格化。
4. 舍入。
5. 判断溢出。

易错点：小阶向大阶对齐，尾数右移会损失精度。

### 2.7 大端与小端

- 大端：高字节存低地址。
- 小端：低字节存低地址。

例如 32 位数 `0x12345678`：

- 大端：12 34 56 78
- 小端：78 56 34 12

## 第 3 章 存储系统

### 3.1 存储层次

从快到慢：

寄存器 -> Cache -> 主存 -> 外存。

局部性原理：

- 时间局部性：最近访问过的信息可能很快再次访问。
- 空间局部性：访问某地址后，邻近地址可能很快被访问。

Cache 和虚拟存储都基于局部性原理。

### 3.2 SRAM 与 DRAM

SRAM：

- 用触发器保存信息；
- 速度快；
- 成本高、容量小；
- 常用于 Cache。

DRAM：

- 用电容保存信息；
- 需要刷新；
- 容量大、成本低；
- 常用于主存。

DRAM 刷新：

- 集中刷新；
- 分散刷新；
- 异步刷新。

### 3.3 主存容量扩展

扩展方式：

- 位扩展：增加字长。
- 字扩展：增加存储单元数量。
- 字位同时扩展。

芯片数量计算：

\[
芯片数=\frac{目标总容量}{单片容量}
\]

片选信号由高位地址译码产生，片内地址由低位地址提供。

### 3.4 磁盘

磁盘访问时间：

\[
T_a=T_s+\frac{1}{2r}+T_t
\]

其中：

- \(T_s\)：寻道时间；
- \(1/(2r)\)：平均旋转延迟；
- \(T_t\)：传输时间；
- \(r\)：转速。

磁盘调度算法：

- FCFS；
- SSTF；
- SCAN；
- C-SCAN；
- LOOK；
- C-LOOK。

### 3.5 Cache

Cache 命中率：

\[
H=\frac{命中次数}{访问总次数}
\]

平均访问时间：

\[
T_a=H T_c+(1-H)(T_c+T_m)
\]

也常写为：

\[
AMAT=命中时间+缺失率\times 缺失代价
\]

#### 映射方式

直接映射：

- 每个主存块只能映射到一个 Cache 行。
- 硬件简单，冲突多。

全相联映射：

- 主存块可放任意 Cache 行。
- 冲突少，比较器多。

组相联映射：

- 先定位组，组内全相联。
- 折中方案。

地址划分：

- 块内地址：由块大小决定。
- 行号/组号：定位 Cache 行或组。
- 标记 Tag：判断是否为目标主存块。

#### 替换算法

- 随机；
- FIFO；
- LRU；
- LFU。

#### 写策略

写命中：

- 写直达：同时写 Cache 和主存。
- 写回：只写 Cache，替换时写回主存。

写不命中：

- 写分配：调入 Cache 后写。
- 非写分配：直接写主存。

### 3.6 虚拟存储器

虚拟存储器使程序看到比实际主存更大的地址空间。

页式虚拟存储：

- 虚拟地址 = 虚页号 + 页内偏移。
- 物理地址 = 页框号 + 页内偏移。
- 页表完成虚页到页框映射。

TLB 是页表项的高速缓存。地址转换流程：

1. 由虚页号查 TLB。
2. 命中则得到页框号。
3. 未命中查页表。
4. 若页表项无效，发生缺页异常。

Cache 与虚拟存储比较：

| 项目 | Cache | 虚拟存储 |
|---|---|---|
| 解决问题 | CPU-主存速度差 | 主存容量不足 |
| 调度单位 | 块 | 页/段 |
| 管理者 | 硬件为主 | 操作系统+硬件 |
| 未命中代价 | 较小 | 很大，需访问外存 |

## 第 4 章 指令系统

### 4.1 ISA

指令集体系结构规定：

- 指令格式；
- 操作码；
- 寻址方式；
- 寄存器集合；
- 数据类型；
- 异常和中断模型；
- 存储访问规则。

### 4.2 指令格式

指令通常包含：

- 操作码字段；
- 地址码字段；
- 寻址方式字段。

按地址数：

- 零地址：栈机器。
- 一地址：累加器机器。
- 二地址：常见，结果覆盖一个操作数。
- 三地址：表达清晰，指令长度较长。

定长操作码：译码简单，但编码空间利用不灵活。  
扩展操作码：短指令多，长指令少，提高编码利用率。

### 4.3 寻址方式

常见数据寻址：

- 立即寻址：操作数在指令中。
- 直接寻址：地址字段给出有效地址。
- 间接寻址：地址字段指向的单元中存有效地址。
- 寄存器寻址：操作数在寄存器。
- 寄存器间接：寄存器中存有效地址。
- 基址寻址：基址寄存器 + 偏移。
- 变址寻址：变址寄存器 + 偏移。
- 相对寻址：PC + 偏移。

### 4.4 CISC 与 RISC

CISC：

- 指令复杂、长度可变；
- 寻址方式多；
- 微程序控制常见；
- 代表：x86。

RISC：

- 指令简单、长度规整；
- load/store 架构；
- 寄存器多；
- 便于流水线；
- 代表：MIPS、RISC-V、ARM 的很多设计思想。

## 第 5 章 CPU

### 5.1 CPU 基本结构

CPU 包括：

- 运算器：ALU、通用寄存器、标志寄存器、移位器。
- 控制器：PC、IR、指令译码器、时序系统、控制信号发生器。

常见寄存器：

- PC：程序计数器，存下一条指令地址。
- IR：指令寄存器。
- MAR：存储器地址寄存器。
- MDR：存储器数据寄存器。
- ACC：累加器。
- PSW：程序状态字。

### 5.2 指令周期

指令周期一般包括：

1. 取指周期。
2. 间址周期。
3. 执行周期。
4. 中断周期。

取指基本微操作：

```text
MAR <- PC
MDR <- M[MAR]
IR  <- MDR
PC  <- PC + 指令长度
```

### 5.3 数据通路

数据通路是数据在 CPU 内部流动的路径。设计关注：

- 哪些寄存器输出到总线；
- ALU 输入来自哪里；
- 运算结果写回哪里；
- 控制信号如何协调同一时刻的数据传送。

单总线结构硬件简单但并行度低；多总线结构可提高并行传送能力。

### 5.4 控制器

硬布线控制：

- 由组合逻辑产生控制信号；
- 速度快；
- 修改困难。

微程序控制：

- 控制信号编码为微指令；
- 微指令组成微程序；
- 设计规整，易修改；
- 速度相对较慢。

### 5.5 指令流水线

经典五级流水：

1. IF：取指。
2. ID：译码/读寄存器。
3. EX：执行/地址计算。
4. MEM：访存。
5. WB：写回。

理想加速比接近流水段数，但实际受冒险影响。

#### 结构冒险

多个指令同时竞争同一硬件资源。解决：

- 增加资源；
- 流水线暂停。

#### 数据冒险

后续指令依赖前面尚未写回的结果。类型：

- RAW：写后读，最常见。
- WAR：读后写。
- WAW：写后写。

解决：

- 转发/旁路；
- 暂停；
- 编译器调度。

#### 控制冒险

分支导致下一条指令地址不确定。解决：

- 分支预测；
- 延迟槽；
- 提前计算分支目标；
- 清空错误路径指令。

### 5.6 多处理器基础

常见概念：

- SISD、SIMD、MISD、MIMD；
- 多核处理器；
- 共享内存与消息传递；
- Cache 一致性。

408 通常只考基本概念，不深入多核协议细节。

## 第 6 章 总线

### 6.1 总线分类

按功能：

- 数据总线；
- 地址总线；
- 控制总线。

按位置：

- 片内总线；
- 系统总线；
- 通信总线。

总线宽度影响一次传输数据位数；总线频率影响传输速度。

### 6.2 总线性能

总线带宽：

\[
带宽=总线宽度 \times 总线频率
\]

若一次传输多个时钟或采用突发传输，需按题目给定节拍计算。

### 6.3 总线仲裁

集中仲裁：

- 链式查询；
- 计数器定时查询；
- 独立请求。

分布仲裁：各模块共同决定总线使用权。

### 6.4 总线定时

- 同步定时：统一时钟，速度快，适合距离短、设备速度接近。
- 异步定时：握手信号，无统一时钟，适合速度差异大设备。
- 半同步/分离式：折中方案。

## 第 7 章 I/O 系统

### 7.1 I/O 接口

I/O 接口功能：

- 地址译码；
- 数据缓冲；
- 格式转换；
- 状态记录；
- 控制命令；
- 中断请求。

I/O 编址：

- 独立编址：I/O 地址空间与内存分离，用专门 I/O 指令。
- 统一编址：I/O 端口映射到内存地址空间，用访存指令访问。

### 7.2 程序查询方式

CPU 反复查询设备状态。  
优点：简单。  
缺点：浪费 CPU 时间。

### 7.3 中断方式

设备准备好后向 CPU 发中断请求。

中断处理流程：

1. 中断请求。
2. 中断判优。
3. 中断响应。
4. 保存现场。
5. 执行中断服务程序。
6. 恢复现场并返回。

中断隐指令通常完成：

- 关中断；
- 保存断点；
- 转入中断服务程序入口。

### 7.4 DMA

DMA 由 DMA 控制器直接在外设和主存之间传送数据，CPU 只在开始和结束时参与。

DMA 传送方式：

- 停止 CPU 访存；
- 周期窃取；
- DMA 与 CPU 交替访存。

适合高速、大批量数据传输。

### 7.5 外部设备

常见设备：

- 输入设备：键盘、鼠标、扫描仪。
- 输出设备：显示器、打印机。
- 外存：磁盘、SSD。

SSD 与磁盘不同，没有机械寻道和旋转延迟，但有擦写次数、写放大、块擦除等特点。

## 高频公式速查

\[
CPU时间 = 指令条数 \times CPI \times 时钟周期
\]

\[
MIPS=\frac{主频}{CPI\times 10^6}
\]

\[
n位补码范围=-2^{n-1}\sim 2^{n-1}-1
\]

\[
AMAT=命中时间+缺失率\times 缺失代价
\]

\[
磁盘平均访问时间=寻道时间+平均旋转延迟+传输时间
\]

\[
总线带宽=总线宽度\times 总线频率
\]

## 高频易错点

1. 补码范围负数比正数多一个。
2. 无符号溢出看进位，有符号补码溢出看符号或 OF。
3. 浮点对阶是小阶向大阶对齐。
4. Cache 地址划分要先确定块大小、行数/组数。
5. 直接映射冲突多，全相联比较器多。
6. TLB 缺失不一定缺页，缺页一定要访问外存。
7. PC 通常在取指后自动加指令长度，转移指令再改 PC。
8. 流水线吞吐率提高，但单条指令执行时间不一定缩短。
9. 中断适合随机小批量 I/O，DMA 适合大批量高速 I/O。
10. 总线带宽题要注意单位：bit/s 与 Byte/s。
