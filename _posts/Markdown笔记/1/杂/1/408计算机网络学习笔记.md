## 第 1 章 计算机网络体系结构

### 1.1 计算机网络概念

计算机网络是互连的、自治的计算机集合。  
互连表示能通信，自治表示没有主从控制关系。

组成：

- 边缘部分：主机、服务器、终端。
- 核心部分：路由器、交换机、通信链路。

功能：

- 数据通信；
- 资源共享；
- 分布式处理；
- 提高可靠性；
- 负载均衡。

### 1.2 交换方式

#### 电路交换

通信前建立专用通路。  
阶段：建立连接、通信、释放连接。  
适合实时连续通信，如传统电话。  
缺点：资源独占，突发数据效率低。

#### 报文交换

整个报文作为单位存储转发。  
无需建立连接，但报文过大导致时延和缓冲需求高。

#### 分组交换

把报文拆成分组，存储转发。  
优点：线路利用率高，适合突发数据。  
缺点：有排队时延、分组头开销、可能乱序/丢失。

### 1.3 网络性能指标

速率：数据传送速率，单位 bit/s。  
带宽：链路最高数据率。  
吞吐量：单位时间实际通过网络的数据量。

时延：

\[
总时延=发送时延+传播时延+处理时延+排队时延
\]

发送时延：

\[
d_{trans}=\frac{分组长度}{发送速率}
\]

传播时延：

\[
d_{prop}=\frac{链路长度}{传播速率}
\]

时延带宽积：

\[
时延带宽积=传播时延\times 带宽
\]

往返时间 RTT：从发送方发送数据开始，到收到确认所经历的时间。

信道利用率：

\[
U=\frac{有效发送时间}{总时间}
\]

### 1.4 分层结构

分层优点：

- 降低复杂度；
- 层间接口清晰；
- 易于标准化；
- 便于独立演进。

基本概念：

- 协议：同层实体通信规则。
- 服务：下层向上层提供的功能。
- 接口：相邻层交换信息的边界。

OSI 七层：

物理层、数据链路层、网络层、运输层、会话层、表示层、应用层。

TCP/IP 四层：

网络接口层、网际层、运输层、应用层。

408 常采用五层模型：

物理层、数据链路层、网络层、传输层、应用层。

## 第 2 章 物理层

### 2.1 基本概念

物理层解决如何在传输媒体上传输比特流。

接口特性：

- 机械特性；
- 电气特性；
- 功能特性；
- 过程特性。

通信方式：

- 单工；
- 半双工；
- 全双工。

串行传输：逐位发送，适合远距离。  
并行传输：多位同时发送，适合短距离。

### 2.2 奈奎斯特准则

理想低通信道最高码元传输速率：

\[
B_{max}=2W
\]

若每个码元有 \(V\) 种离散电平，最大数据率：

\[
R=2W\log_2 V
\]

### 2.3 香农公式

有噪声信道极限信息传输速率：

\[
C=W\log_2(1+S/N)
\]

信噪比常用 dB：

\[
SNR_{dB}=10\log_{10}(S/N)
\]

### 2.4 编码与调制

数字数据编码为数字信号：

- 非归零 NRZ；
- 曼彻斯特编码；
- 差分曼彻斯特编码。

曼彻斯特编码每比特中间跳变，便于同步，但带宽需求较高。

数字数据调制为模拟信号：

- ASK；
- FSK；
- PSK；
- QAM。

### 2.5 传输介质

导向型：

- 双绞线；
- 同轴电缆；
- 光纤。

非导向型：

- 无线电；
- 微波；
- 红外；
- 卫星通信。

物理层设备：

- 中继器：再生信号。
- 集线器 Hub：多端口中继器，工作在物理层，共享冲突域。

## 第 3 章 数据链路层

### 3.1 功能

数据链路层在相邻结点之间传输帧。

功能：

- 封装成帧；
- 透明传输；
- 差错检测；
- 流量控制；
- 可靠传输；
- 介质访问控制。

### 3.2 成帧

常见方法：

- 字符计数法；
- 字节填充法；
- 零比特填充法；
- 违规编码法。

PPP/HDLC 中常见零比特填充：发送端遇到连续 5 个 1 后插入 0，接收端删除该 0。

### 3.3 差错控制

#### 奇偶校验

只能检测奇数位错误，不能纠错。

#### CRC

发送方用生成多项式 \(G(x)\) 进行模 2 除法，余数作为 FCS。

接收方用收到比特串除以同一生成多项式，若余数为 0 则认为无差错。

模 2 运算无进位无借位，本质是异或。

#### 海明码

可纠正一位错误。若数据位为 \(m\)，校验位为 \(r\)，需满足：

\[
2^r \ge m+r+1
\]

校验位放在 \(1,2,4,8,\dots\) 位置。

### 3.4 流量控制与可靠传输

#### 停止-等待协议

发送一个帧后等待确认。  
简单但信道利用率低。

信道利用率近似：

\[
U=\frac{T_D}{T_D+RTT+T_A}
\]

#### 后退 N 帧 GBN

发送窗口大小最大：

\[
W_T \le 2^n-1
\]

接收窗口为 1。出错后从错误帧开始重传后续所有帧。

#### 选择重传 SR

只重传出错帧。发送窗口和接收窗口均大于 1。

为了避免新旧帧序号混淆：

\[
W_T+W_R \le 2^n
\]

常取：

\[
W_T=W_R\le 2^{n-1}
\]

### 3.5 介质访问控制

#### 信道划分

- FDM：频分复用。
- TDM：时分复用。
- WDM：波分复用。
- CDM/CDMA：码分复用。

#### 随机访问

ALOHA、CSMA、CSMA/CD、CSMA/CA。

CSMA/CD 用于传统共享式以太网：

- 先听后发；
- 边发边听；
- 冲突停止；
- 随机退避。

最短帧长要求：

\[
发送时间 \ge 2\tau
\]

即发送方在发送完前能检测到最远冲突。

CSMA/CA 用于无线局域网，避免碰撞而不是检测碰撞。

### 3.6 局域网与以太网

以太网帧包括：

- 目的 MAC；
- 源 MAC；
- 类型；
- 数据；
- FCS。

MAC 地址 48 位。

交换机工作在数据链路层，根据 MAC 地址表转发帧。  
交换机每个端口是独立冲突域，但默认同属一个广播域。

VLAN 可划分广播域。

### 3.7 PPP

PPP 面向点对点链路，提供：

- 成帧；
- 链路控制 LCP；
- 网络控制 NCP；
- 差错检测。

PPP 不提供可靠传输，不进行流量控制。

## 第 4 章 网络层

### 4.1 网络层功能

网络层实现主机到主机的分组传输。

功能：

- 异构网络互连；
- 路由选择；
- 分组转发；
- 拥塞控制；
- 网络层地址。

网络层服务：

- 虚电路服务；
- 数据报服务。

Internet 采用无连接数据报服务。

### 4.2 IPv4

IPv4 地址 32 位，点分十进制表示。

IPv4 首部常见字段：

- 版本；
- 首部长度；
- 总长度；
- 标识、标志、片偏移；
- TTL；
- 协议；
- 首部校验和；
- 源地址、目的地址。

分片：

- 片偏移单位为 8 字节。
- MF=1 表示后面还有分片。
- DF=1 表示禁止分片。

### 4.3 子网划分与 CIDR

CIDR 表示：

```text
IP地址/前缀长度
```

主机号位数为 \(h\)，可用主机数通常：

\[
2^h-2
\]

全 0 主机号表示网络地址，全 1 表示广播地址。

路由聚合：多个连续网络合并成更短前缀。  
转发时采用最长前缀匹配。

### 4.4 NAT

NAT 在私有地址和公网地址之间转换。  
NAPT 使用端口号区分多个内部主机连接。

优点：缓解 IPv4 地址不足。  
缺点：破坏端到端原则，对某些应用协议不友好。

### 4.5 ARP

ARP 用于由 IPv4 地址解析 MAC 地址。

流程：

1. 主机查 ARP 缓存。
2. 若没有，广播 ARP 请求。
3. 目标主机单播 ARP 响应。
4. 请求方更新 ARP 缓存。

ARP 只在同一局域网内工作。跨网段通信时解析的是默认网关 MAC。

### 4.6 DHCP

DHCP 动态分配 IP 配置。流程 DORA：

1. Discover。
2. Offer。
3. Request。
4. ACK。

DHCP 基于 UDP。

### 4.7 ICMP

ICMP 用于差错报告和网络探测。

常见：

- 目的不可达；
- 超时；
- 参数问题；
- 回送请求/回答。

`ping` 使用 ICMP Echo。  
`traceroute` 利用 TTL 逐跳超时。

### 4.8 IPv6

IPv6 地址 128 位。特点：

- 地址空间大；
- 首部固定 40 字节；
- 扩展首部；
- 不在路由器处分片；
- 支持自动配置。

IPv4 到 IPv6 过渡：

- 双协议栈；
- 隧道；
- 翻译。

### 4.9 路由算法与协议

#### 距离向量

基于 Bellman-Ford。每个路由器与邻居交换距离向量。  
代表：RIP。  
问题：坏消息传播慢，可能计数到无穷。

#### 链路状态

每个路由器掌握全网拓扑，用 Dijkstra 算最短路径。  
代表：OSPF。

#### BGP

自治系统之间的路径向量协议，关注策略而不只是最短路径。

## 第 5 章 传输层

### 5.1 传输层作用

传输层提供进程到进程的通信。  
网络层是主机到主机。

端口号标识应用进程。

套接字：

\[
Socket=(IP地址,端口号)
\]

TCP 连接由四元组标识：

\[
(源IP,源端口,目的IP,目的端口)
\]

### 5.2 UDP

UDP 特点：

- 无连接；
- 尽最大努力交付；
- 面向报文；
- 无拥塞控制；
- 首部开销小。

适合 DNS、DHCP、实时音视频等。

UDP 首部 8 字节：

- 源端口；
- 目的端口；
- 长度；
- 校验和。

### 5.3 TCP

TCP 特点：

- 面向连接；
- 可靠传输；
- 面向字节流；
- 全双工；
- 有流量控制和拥塞控制。

TCP 首部重要字段：

- 源/目的端口；
- 序号 seq；
- 确认号 ack；
- 数据偏移；
- 标志位 SYN、ACK、FIN、RST、PSH、URG；
- 窗口；
- 校验和。

确认号含义：期望收到的下一个字节序号。

### 5.4 TCP 连接建立

三次握手：

1. 客户端发送 SYN，seq=x。
2. 服务器发送 SYN+ACK，seq=y，ack=x+1。
3. 客户端发送 ACK，ack=y+1。

为什么不是两次：防止已失效的连接请求报文突然到达服务器导致错误建立连接。

### 5.5 TCP 连接释放

四次挥手：

1. 主动方发送 FIN。
2. 被动方 ACK。
3. 被动方数据发完后发送 FIN。
4. 主动方 ACK，并进入 TIME_WAIT。

TIME_WAIT 作用：

- 保证最后一个 ACK 可重传；
- 让旧连接报文在网络中消失。

### 5.6 TCP 可靠传输

机制：

- 序号；
- 确认；
- 超时重传；
- 快速重传；
- 校验和；
- 滑动窗口。

RTT 估计和超时重传时间 RTO 设置影响性能。

### 5.7 TCP 流量控制

接收方通过接收窗口 `rwnd` 告知发送方可接收空间，避免发送过快淹没接收缓冲。

发送窗口受两者限制：

\[
发送窗口=\min(rwnd,cwnd)
\]

其中 `cwnd` 是拥塞窗口。

### 5.8 TCP 拥塞控制

拥塞控制防止网络过载。

核心变量：

- `cwnd` 拥塞窗口；
- `ssthresh` 慢开始门限。

阶段：

#### 慢开始

`cwnd` 从小开始指数增长，每 RTT 近似翻倍。

#### 拥塞避免

达到 `ssthresh` 后线性增长，每 RTT 大约加 1 MSS。

#### 快速重传

收到 3 个重复 ACK，立即重传丢失报文段。

#### 快速恢复

乘法减小窗口后避免重新慢开始，视具体 TCP 版本而定。

超时通常表示更严重拥塞：`ssthresh=cwnd/2`，`cwnd` 重置为 1 MSS。

## 第 6 章 应用层

### 6.1 DNS

DNS 将域名解析为 IP 地址。

层次：

- 根域名服务器；
- 顶级域名服务器；
- 权威域名服务器；
- 本地域名服务器。

查询：

- 递归查询；
- 迭代查询。

DNS 通常使用 UDP 53；区域传送等场景可用 TCP。

### 6.2 FTP

FTP 使用两个 TCP 连接：

- 控制连接：端口 21，持续存在。
- 数据连接：传输文件时建立。

控制和数据分离是 FTP 特点。

### 6.3 电子邮件

组成：

- 用户代理 UA；
- 邮件服务器；
- 邮件协议。

协议：

- SMTP：发送邮件，推协议。
- POP3：读取邮件，下载式。
- IMAP：读取邮件，服务器保留并管理邮件状态。

MIME 用于支持非 ASCII 内容和附件。

### 6.4 WWW 与 HTTP

URL 组成：

```text
协议://主机:端口/路径
```

HTTP 特点：

- 应用层协议；
- 基于请求-响应；
- 无状态；
- 通常运行在 TCP 上，HTTP/3 运行在 QUIC/UDP 上。

HTTP 报文：

- 请求行/状态行；
- 首部字段；
- 空行；
- 实体体。

常见状态码：

- 200 OK；
- 301/302 重定向；
- 400 Bad Request；
- 403 Forbidden；
- 404 Not Found；
- 500 Internal Server Error。

持久连接减少 TCP 建连开销。  
Cookie 用于在无状态 HTTP 上维持用户状态。

### 6.5 HTTPS

HTTPS = HTTP + TLS。  
提供：

- 机密性；
- 完整性；
- 身份认证。

408 通常只需理解基本作用，不深入 TLS 握手细节。

## 高频易错点

1. 带宽是最高速率，吞吐量是实际速率。
2. 发送时延和传播时延不是一回事。
3. 集线器工作在物理层，交换机工作在数据链路层，路由器工作在网络层。
4. ARP 解析同一局域网内 IP 到 MAC；跨网段解析网关 MAC。
5. IP 提供不可靠无连接服务，TCP 才提供可靠传输。
6. GBN 接收窗口为 1，SR 接收窗口大于 1。
7. 路由转发采用最长前缀匹配。
8. TCP 确认号是期望收到的下一个字节序号。
9. TCP 流量控制看接收方，拥塞控制看网络状态。
10. DNS 通常 UDP，HTTP 通常 TCP，HTTP/3 基于 QUIC/UDP。

## 复习建议

1. 每层记住 PDU 名称：比特、帧、分组/数据报、报文段、报文。
2. 子网划分每天练几题，重点是网络地址、广播地址、可用主机范围。
3. TCP 题画时间线，标 seq、ack、SYN、FIN。
4. 滑动窗口题写出序号空间和窗口边界，避免新旧帧混淆。
5. 应用层协议记“用途、传输层协议、端口、关键流程”。
