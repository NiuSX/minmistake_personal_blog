# Vim 学习笔记

Vim 的核心不是“记命令”，而是理解它的工作方式：

1. 以**模式**驱动操作。
2. 以**文本对象**和**动作**组合编辑。
3. 以**低手离开键盘**为目标，尽量少按键完成复杂任务。

---

## 1. Vim 是什么

Vim 是一款高度可定制的文本编辑器，常用于：

- 写代码
- 改配置
- 快速批量编辑文本
- 在终端里完成高频修改

它最重要的特点是：

- 快：大量操作都能用键盘完成
- 稳：命令可组合、可重复、可撤销
- 强：支持宏、寄存器、脚本、插件

---

## 2. 三大基本模式

### 2.1 普通模式

默认模式。用于移动、删除、复制、搜索、替换、执行命令。

常见操作几乎都从普通模式开始。

### 2.2 插入模式

用于输入文本。

进入插入模式的常见键：

- `i`：在光标前插入
- `a`：在光标后插入
- `I`：行首插入
- `A`：行尾插入
- `o`：下一行新建一行并插入
- `O`：上一行新建一行并插入

### 2.3 命令行模式

用于执行保存、退出、搜索替换等命令。

进入方式：

- `:` 进入命令行模式
- `/` 进入正向搜索
- `?` 进入反向搜索

### 2.4 可视模式

用于选择文本后再操作。

常见进入方式：

- `v`：按字符选择
- `V`：按行选择
- `Ctrl-v`：块选择

## 3. 基础生存操作

### 3.1 退出

- `:q`：退出
- `:q!`：强制退出
- `:wq`：保存并退出
- `:x`：保存并退出
- `ZZ`：保存并退出

### 3.2 保存

- `:w`：保存
- `:w filename`：另存为

### 3.3 进入和离开插入模式

- `Esc`：返回普通模式
- `Ctrl-[`：很多终端里也可替代 `Esc`

建议养成习惯：**输入完立即按 `Esc`**，避免自己忘记当前模式。

---

## 4. 光标移动

### 4.1 基础方向键

- `h`：左
- `j`：下
- `k`：上
- `l`：右

### 4.2 行内移动

- `0`：行首
- `^`：第一个非空白字符
- `$`：行尾
- `g_`：行内最后一个非空白字符
- `|`：跳到指定列，如 `10|`

### 4.3 单词移动

- `w`：到下一个单词开头
- `W`：到下一个“空格分隔的大词”开头
- `b`：到上一个单词开头
- `B`：到上一个大词开头
- `e`：到当前或下一个单词结尾
- `E`：到当前或下一个大词结尾

### 4.4 屏幕移动

- `H`：屏幕顶部
- `M`：屏幕中间
- `L`：屏幕底部
- `Ctrl-u`：向上翻半屏
- `Ctrl-d`：向下翻半屏
- `Ctrl-b`：向上翻整屏
- `Ctrl-f`：向下翻整屏

### 4.5 文件内跳转

- `gg`：文件开头
- `G`：文件末尾
- `nG`：跳到第 `n` 行
- `:n`：跳到第 `n` 行

### 4.6 括号和代码块

- `%`：在匹配的括号、花括号、方括号之间跳转

对编程特别有用。

## 5. 编辑文本

Vim 最强的地方在于“动作 + 操作符”的组合。

### 5.1 删除

- `x`：删除光标下字符
- `X`：删除光标前字符
- `dw`：删除到下一个单词开头
- `de`：删除到单词结尾
- `dd`：删除整行
- `D`：删除到行尾
- `d$`：删除到行尾
- `d0`：删除到行首

### 5.2 修改

`c` 表示 change，本质上是“删除后进入插入模式”。

- `cw`：修改一个单词
- `ce`：修改到单词结尾
- `cc`：修改整行
- `C`：修改到行尾
- `c$`：修改到行尾

### 5.3 替换

- `r<char>`：用一个字符替换当前字符
- `R`：进入覆盖模式
- `~`：切换大小写

### 5.4 插入补充

- `s`：删除当前字符并插入
- `S`：删除整行并插入
- `J`：合并下一行到当前行

### 5.5 撤销和重做

- `u`：撤销
- `Ctrl-r`：重做

### 5.6 重复上一次操作

- `.`：重复上一次编辑命令

这是 Vim 提速的关键键之一。

## 6. 复制、粘贴、剪切

### 6.1 复制

- `yy`：复制整行
- `yw`：复制一个单词
- `y$`：复制到行尾
- `ygg`：从当前位置复制到文件开头
- `yG`：从当前位置复制到文件末尾

### 6.2 粘贴

- `p`：在后面粘贴
- `P`：在前面粘贴

### 6.3 剪切

在 Vim 里，删除命令通常也会进入剪切缓冲区。

- `dd`：剪切整行
- `dw`：剪切一个单词

### 6.4 行与块的配合

先选中，再 `y`、`d`、`p`，是最稳的编辑方式之一。

## 7. 可视模式

### 7.1 常用选择

- `v`：字符级选择
- `V`：行级选择
- `Ctrl-v`：矩形块选择

### 7.2 选择后的操作

选择后可以直接：

- `y`：复制
- `d`：删除
- `c`：修改
- `>`：缩进
- `<`：反缩进
- `:`：把选择范围作为命令范围

### 7.3 常见用法

- 批量注释
- 对齐多行文本
- 修改代码块某一列

## 8. 文本对象

文本对象让 Vim 从“按字符编辑”变成“按语义编辑”。

### 8.1 常见文本对象

- `iw`：inner word，单词内部
- `aw`：a word，包含单词及周边空格
- `i"` / `a"`：双引号内部 / 包含双引号
- `i'` / `a'`：单引号内部 / 包含单引号
- `i(` / `a(`：括号内部 / 包含括号
- `i[` / `a[`：方括号内部 / 包含方括号
- `i{` / `a{`：花括号内部 / 包含花括号
- `ip` / `ap`：段落内部 / 包含段落

### 8.2 常见组合

- `ciw`：修改一个单词
- `di"`：删除双引号中的内容
- `ya(`：复制括号内容及括号
- `vi{`：选择花括号内部

这类组合是 Vim 最值得练的内容。

## 9. 搜索

### 9.1 基本搜索

- `/pattern`：向下搜索
- `?pattern`：向上搜索
- `n`：跳到下一个匹配
- `N`：跳到上一个匹配

### 9.2 搜索技巧

- `*`：搜索当前光标下单词的下一个匹配
- `#`：搜索当前光标下单词的上一个匹配

### 9.3 高亮

- `:set hlsearch`：高亮搜索结果
- `:set nohlsearch`：取消高亮
- `:noh`：临时清除高亮

### 9.4 更实用的搜索设置

- `:set incsearch`：边输入边搜索
- `:set ignorecase`：忽略大小写
- `:set smartcase`：有大写时区分大小写

推荐组合：

```vim
:set incsearch
:set ignorecase
:set smartcase
```

## 10. 替换

### 10.1 单行替换

```vim
:s/old/new/
```

### 10.2 全局替换

```vim
:%s/old/new/g
```

### 10.3 带确认替换

```vim
:%s/old/new/gc
```

其中：

- `%`：整篇文件
- `g`：一行内全部替换
- `c`：每次替换前确认

### 10.4 常见例子

```vim
:%s/\s\+$//g
```

删除行尾空白。

```vim
:%s/\t/    /g
```

把 tab 替换成 4 个空格。

## 11. 缩进与格式

- `>>`：当前行缩进
- `<<`：当前行反缩进
- `=`：自动格式化缩进
- `gg=G`：全文自动缩进

常见场景：

- 对一段代码统一缩进
- 修复粘贴后格式错乱

---

## 12. 窗口、标签页、缓冲区

这三个概念经常混。

### 12.1 Buffer

缓冲区，表示打开的文件内容。

### 12.2 Window

窗口，表示当前屏幕上分割出来的一个视图。

### 12.3 Tab

标签页，通常是窗口布局的一个集合。

### 12.4 常用命令

- `:ls`：查看缓冲区
- `:b n`：切换到第 `n` 个缓冲区
- `:bn`：下一个缓冲区
- `:bp`：上一个缓冲区
- `:bd`：关闭当前缓冲区

### 12.5 分屏

- `:sp`：横向分屏
- `:vsp`：纵向分屏
- `Ctrl-w s`：横向分屏
- `Ctrl-w v`：纵向分屏
- `Ctrl-w w`：在窗口间切换
- `Ctrl-w h/j/k/l`：切换到对应方向窗口
- `Ctrl-w =`：让所有窗口等宽等高

### 12.6 标签页

- `:tabnew`：新建标签页
- `gt`：下一个标签页
- `gT`：上一个标签页

---

## 13. 寄存器

寄存器用来保存复制、删除、命令输出等内容。

### 13.1 基本理解

可以把寄存器理解为“多个剪贴板”。

### 13.2 常见操作

- `"`：在命令前指定寄存器
- `"ayw`：把一个单词复制到 `a` 寄存器
- `"ap`：从 `a` 寄存器粘贴

### 13.3 常见寄存器

- `"`：默认寄存器
- `0`：最近一次复制的内容
- `1` 到 `9`：最近删除的历史
- `+`：系统剪贴板
- `*`：系统选择缓冲区

### 13.4 实际用法

```vim
"+y
"+p
```

和系统剪贴板交互。

---

## 14. 宏

宏适合批量重复结构化操作。

### 14.1 录制宏

- `q{register}`：开始录制
- `q`：停止录制

例如录制到 `a`：

```vim
qa
...
q
```

### 14.2 播放宏

- `@a`：执行 `a` 宏
- `@@`：重复上一次宏
- `10@a`：执行 10 次

### 14.3 适用场景

- 批量改格式
- 批量插入前缀或后缀
- 结构重复但手工操作繁琐的文本

---

## 15. 命令行模式常用命令

- `:set number`：显示行号
- `:set relativenumber`：显示相对行号
- `:set cursorline`：高亮当前行
- `:set showmode`：显示当前模式
- `:set clipboard=unnamedplus`：使用系统剪贴板

### 15.1 文件相关

- `:e filename`：打开文件
- `:r filename`：插入文件内容
- `:w`：保存
- `:w filename`：另存为

### 15.2 Shell 相关

- `:!ls`：执行外部命令
- `:r !date`：把命令输出插入当前文件

## 16. 常用配置

下面是一个实用的起步配置。

```vim
set nocompatible
set number
set relativenumber
set cursorline
set showcmd
set showmode
set hidden
set wrap
set linebreak
set incsearch
set hlsearch
set ignorecase
set smartcase
set tabstop=4
set shiftwidth=4
set expandtab
set autoindent
set smartindent
set clipboard=unnamedplus
set backspace=indent,eol,start
syntax on
filetype plugin indent on
```

### 16.1 说明

- `number` 和 `relativenumber`：方便跳行和配合操作
- `hidden`：允许切换缓冲区时不必强制保存
- `expandtab`：把 tab 转成空格
- `tabstop` / `shiftwidth`：控制缩进宽度
- `clipboard=unnamedplus`：更方便复制到系统剪贴板

## 19. 常见坑

- 忘记当前在插入模式，导致普通键被输入成文本
- 分不清 `d`、`c`、`y`
- 不会用 `.` 重复操作
- 只会方向键，不会 `w/b/e`
- 替换时不加 `g`，结果只改了一处
- 不了解 buffer、window、tab 的区别

## 21. 练习建议

每天只练一个小目标：

1. 只用 `h j k l` 和 `w b e` 移动。
2. 只用 `ciw`、`dd`、`yy`、`p` 改文本。
3. 用 `:%s` 做一次批量替换。
4. 用一次宏改一批相似内容。
5. 把常用配置写进自己的 `vimrc`。

持续一周，比一次性背很多命令有效得多。
