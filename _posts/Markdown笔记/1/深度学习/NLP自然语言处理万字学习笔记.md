## 一、NLP 是什么

### 1. 定义

NLP 是 Natural Language Processing，即自然语言处理。它研究如何让计算机理解、分析、生成和使用人类自然语言。

自然语言包括：

- 中文。
- 英文。
- 日文。
- 语音转写文本。
- 对话文本。
- 文档。
- 代码注释。
- 社交媒体文本。
- 法律、医疗、金融等专业文本。

NLP 的核心目标可以概括为两类：

1. 理解语言：让机器读懂文本。
2. 生成语言：让机器写出文本。

理解类任务包括分类、匹配、抽取、问答、检索、情感分析等。生成类任务包括翻译、摘要、对话、改写、续写、代码生成等。

### 2. NLP 在人工智能中的位置

自然语言是人类表达知识、意图、情绪和逻辑的主要媒介。因此 NLP 是人工智能中最重要的方向之一。很多智能系统最终都需要通过语言和人交互。

常见应用包括：

- 搜索引擎。
- 智能客服。
- 机器翻译。
- 文本审核。
- 舆情分析。
- 智能写作。
- 语音助手。
- 法律文书分析。
- 医疗病历处理。
- 金融研报分析。
- 大语言模型对话系统。

### 3. NLP 的发展阶段

NLP 大致经历了几个阶段：

#### 规则系统阶段

早期 NLP 依赖人工编写规则，例如词典、语法规则、模板匹配。优点是可解释，缺点是维护成本高，泛化能力弱。

#### 统计机器学习阶段

使用概率模型和特征工程，例如 n-gram 语言模型、HMM、CRF、SVM、朴素贝叶斯等。这个阶段强调数据和统计规律。

#### 深度学习阶段

使用神经网络自动学习表示，例如 Word2Vec、CNN、RNN、LSTM、Seq2Seq、Attention 等。减少了人工特征工程依赖。

#### 预训练语言模型阶段

BERT、GPT、T5 等模型通过大规模语料预训练，再迁移到下游任务，大幅提升效果。

#### 大语言模型阶段

更大规模的 Transformer 模型通过指令微调、强化学习、人类反馈、多模态扩展等方式，具备更强的生成、推理和工具调用能力。

## 二、NLP 为什么难

### 1. 自然语言高度歧义

同一句话可能有多种含义。

例如：

```text
我看见了拿望远镜的人。
```

可能表示：

- 我用望远镜看见了一个人。
- 我看见了一个拿着望远镜的人。

机器必须根据上下文判断含义。

### 2. 一词多义

同一个词在不同上下文中含义不同。

```text
苹果发布了新手机。
我吃了一个苹果。
```

第一个“苹果”是公司，第二个“苹果”是水果。

### 3. 同义表达

不同表达可能含义相似。

```text
这部电影很好看。
这片子挺精彩。
这电影值得一看。
```

词面不同，但情感倾向相似。

### 4. 长距离依赖

一个词的含义可能依赖很远的上下文。

```text
虽然这家餐厅环境一般，服务也不算热情，但菜品味道确实不错，所以我还是愿意再来。
```

判断整体情感不能只看“环境一般”“不热情”，还要理解转折后的重点。

### 5. 常识与世界知识

语言理解经常需要常识。

```text
小明把冰淇淋放在太阳下，过了一会儿它化了。
```

机器要知道“冰淇淋在太阳下会融化”。

### 6. 语境和意图

一句话表面意思和真实意图可能不同。

```text
你可真准时啊。
```

如果对方迟到了，这句话可能是讽刺。

### 7. 中文 NLP 的特殊挑战

中文没有天然空格分词，存在：

- 分词歧义。
- 简繁体。
- 口语化表达。
- 同音词。
- 成语俗语。
- 新词热词。
- 字词边界不清。

例如：

```text
南京市长江大桥
```

可以切成：

```text
南京市 / 长江大桥
南京 / 市长 / 江大桥
```

正确分词需要语义判断。

## 三、NLP 基本流程

### 1. 传统 NLP 流程

传统 NLP 项目通常流程为：

```text
文本收集 -> 清洗 -> 分词 -> 特征提取 -> 模型训练 -> 评估 -> 部署
```

例如做情感分类：

1. 收集评论数据。
2. 清洗 HTML、表情、乱码。
3. 中文分词。
4. 提取 TF-IDF 特征。
5. 训练 SVM 或逻辑回归。
6. 用准确率、F1 评估。
7. 部署为接口。

### 2. 深度学习 NLP 流程

深度学习方法通常流程为：

```text
文本 -> tokenization -> token ids -> embedding -> neural network -> output
```

模型自动学习特征，不需要大量手工特征工程。

### 3. 预训练模型流程

使用 BERT、GPT 等预训练模型时，流程通常是：

```text
文本 -> tokenizer -> 预训练模型 -> 任务头 -> 微调/推理
```

例如文本分类：

```text
输入文本 -> BERT -> [CLS] 表示 -> 分类层 -> 类别
```

### 4. 大语言模型应用流程

基于大语言模型的 NLP 应用通常是：

```text
用户输入 -> prompt 构造 -> LLM 推理 -> 后处理 -> 返回结果
```

如果结合知识库：

```text
用户问题 -> 检索相关文档 -> 拼接上下文 -> LLM 生成答案
```

这就是常见的 RAG。

## 四、文本预处理

### 1. 文本清洗

文本清洗是 NLP 项目中非常重要但容易被低估的环节。真实数据通常很脏，包含：

- HTML 标签。
- URL。
- 表情符号。
- 特殊字符。
- 多余空格。
- 编码错误。
- 重复文本。
- 广告内容。
- 敏感信息。

清洗目标不是把文本变得“干净好看”，而是保留对任务有用的信息，删除干扰信息。

### 2. 大小写转换

英文任务中常把文本转小写：

```text
Apple -> apple
```

但这会损失实体信息。比如 Apple 可能是公司，apple 是水果。

是否转小写取决于任务：

- 情感分类：通常可以转小写。
- 命名实体识别：不建议随意转小写。
- 使用大小写敏感预训练模型：保持原文。

### 3. 去停用词

停用词是高频但信息量较低的词，如：

```text
的、了、是、the、is、a
```

传统模型中去停用词常有帮助，但深度学习和预训练模型中不一定需要。因为模型可能利用这些词的语法信息。

原则：

- 传统 TF-IDF 模型可考虑去停用词。
- BERT/GPT 等预训练模型通常不要随便去停用词。
- 生成任务不要随意删除词。

### 4. 标点处理

标点对语义有影响。

```text
好！ 
好？
好……
```

情绪完全不同。文本分类、情感分析、对话任务中标点可能很重要。

### 5. 数字处理

数字可以：

- 保留原样。
- 统一替换为 `<NUM>`。
- 归一化成区间。
- 转换为特定格式。

例如金融文本中数字非常重要，不能简单删除。

### 6. 去重

数据重复会导致：

- 训练集和测试集泄漏。
- 模型过拟合重复样本。
- 评估结果虚高。

去重可以按：

- 完全相同文本。
- 归一化后相同文本。
- 相似文本。

### 7. 数据泄漏

数据泄漏是 NLP 项目常见严重问题。例如同一篇文章的轻微改写同时出现在训练集和测试集，模型看似效果很好，但真实泛化差。

划分数据集时要注意：

- 同一用户数据是否跨集合。
- 同一文档切片是否跨集合。
- 同一时间段数据是否泄漏未来信息。
- 重复或近重复样本是否跨集合。

## 五、分词与子词建模

### 1. 为什么需要分词

模型不能直接处理原始字符串，需要切分成基本单元。这个基本单元可以是：

- 字符。
- 单词。
- 子词。
- 字节。

分词影响词表大小、未知词比例、语义粒度和模型效率。

### 2. 中文分词

中文没有空格，分词是重要问题。

常见方法：

- 基于词典的最大匹配。
- HMM。
- CRF。
- BiLSTM-CRF。
- Transformer 标注模型。
- 子词或字级建模。

传统中文 NLP 常依赖分词。但在 BERT 类模型中，中文常使用字粒度或 WordPiece，减少显式分词依赖。

### 3. 英文分词

英文有空格，但仍有复杂情况：

```text
don't -> do n't
New York -> 一个实体
state-of-the-art -> 复合词
```

现代 tokenizer 会处理标点、缩写、子词等问题。

### 4. OOV 问题

OOV 是 Out Of Vocabulary，即词表外词。传统词级模型遇到新词只能映射为 `<UNK>`，导致信息丢失。

例如：

```text
ChatGPT
量子计算机
某个新品牌名
```

如果词表没有，就无法准确表示。

### 5. BPE

BPE 是 Byte Pair Encoding。它从字符开始，逐步合并高频相邻片段，形成子词词表。

例如：

```text
l o w
l o w e r
n e w e s t
```

高频组合会被合并成子词。

BPE 优点：

- 降低 OOV。
- 词表可控。
- 高频词可作为整体。
- 低频词可拆成子词。

### 6. WordPiece

WordPiece 与 BPE 类似，也是一种子词建模方法。BERT 使用 WordPiece。

示例：

```text
unaffable -> un ##aff ##able
```

`##` 表示该子词接在前一个子词后面。

### 7. SentencePiece

SentencePiece 把文本当作原始字符流，不依赖预分词，适合多语言场景。它常用于现代大模型 tokenizer。

### 8. Tokenization 的影响

Tokenizer 会影响：

- 序列长度。
- 训练效率。
- 多语言效果。
- 罕见词表示。
- 代码和特殊符号处理。
- 中文、英文、混合文本表现。

同一句话在不同 tokenizer 下 token 数可能差异很大，这会直接影响上下文长度和推理成本。

## 六、词表示方法

### 1. One-hot 表示

One-hot 用一个词表长度的向量表示词。

例如词表：

```text
[我, 爱, 自然, 语言, 处理]
```

“爱”表示为：

```text
[0, 1, 0, 0, 0]
```

缺点：

- 维度高。
- 稀疏。
- 无法表达词义相似性。
- 任意两个不同词距离相同。

### 2. Bag of Words

词袋模型只统计词出现次数，不考虑顺序。

```text
我 爱 NLP
NLP 爱 我
```

词袋表示可能相同，但语义可以不同。

### 3. TF-IDF

TF-IDF 衡量一个词对文档的重要性。

```text
TF-IDF = TF * IDF
```

TF 表示词在当前文档中出现频率。IDF 表示词在整个语料中稀有程度。

直觉：

- 一个词在当前文档经常出现，重要性高。
- 一个词在所有文档都出现，区分度低。

TF-IDF 在传统文本分类、检索、关键词提取中仍然有价值。

### 4. Word2Vec

Word2Vec 是经典词向量方法，包括 CBOW 和 Skip-gram。

CBOW：

```text
用上下文预测中心词
```

Skip-gram：

```text
用中心词预测上下文
```

Word2Vec 的核心思想是分布式假设：

> 词的意义由它的上下文决定。

相似上下文中的词会学到相似向量。

### 5. GloVe

GloVe 利用全局词共现统计学习词向量。它结合了矩阵分解和上下文共现思想。

### 6. FastText

FastText 把词拆成字符 n-gram，可以更好处理形态变化和 OOV。

例如：

```text
where -> <wh, whe, her, ere, re>
```

对英文、拼写变化和低频词有帮助。

### 7. 静态词向量的局限

Word2Vec、GloVe、FastText 属于静态词向量。同一个词在不同上下文中表示相同。

例如：

```text
苹果发布新手机。
我吃了一个苹果。
```

“苹果”的向量相同，无法区分公司和水果。

上下文相关表示解决了这个问题。

### 8. 上下文表示

BERT、GPT 等模型会根据上下文生成 token 表示。同一个词在不同句子中表示不同。

这是预训练语言模型相对传统词向量的重要提升。

## 七、语言模型

### 1. 什么是语言模型

语言模型用于计算一段文本出现的概率：

```text
P(w_1, w_2, ..., w_n)
```

根据链式法则：

```text
P(w_1, ..., w_n) = Π P(w_i | w_1, ..., w_{i-1})
```

语言模型的核心任务是预测下一个词或补全缺失词。

### 2. N-gram 语言模型

N-gram 假设当前词只依赖前 N-1 个词。

Bigram：

```text
P(w_i | w_1, ..., w_{i-1}) ≈ P(w_i | w_{i-1})
```

Trigram：

```text
P(w_i | w_1, ..., w_{i-1}) ≈ P(w_i | w_{i-2}, w_{i-1})
```

优点：

- 简单。
- 可解释。
- 训练快。

缺点：

- 稀疏问题严重。
- 长距离依赖弱。
- 泛化能力有限。

### 3. 神经语言模型

神经语言模型用神经网络预测下一个词。输入词向量，输出词表概率。

相比 N-gram，它可以：

- 学习分布式表示。
- 缓解稀疏问题。
- 捕捉更长上下文。

### 4. 自回归语言模型

自回归语言模型从左到右预测：

```text
P(x_t | x_1, ..., x_{t-1})
```

GPT 类模型就是典型自回归语言模型。

适合生成任务，但每个位置不能看未来。

### 5. 掩码语言模型

BERT 使用 Masked Language Modeling。随机遮住部分 token，让模型根据双向上下文预测被遮住的词。

```text
我 喜欢 [MASK] 语言 处理
```

模型可以同时看左右上下文。适合理解任务。

### 6. Perplexity 困惑度

困惑度是语言模型常用指标，越低表示模型越能预测文本。

直觉：

```text
困惑度越低，模型越不困惑。
```

但困惑度不一定完全代表生成质量。一个模型困惑度低，不代表回答一定真实、有帮助或安全。

## 八、传统机器学习 NLP

### 1. 传统文本分类流程

典型流程：

```text
文本清洗 -> 分词 -> 特征提取 -> 分类模型 -> 评估
```

常用特征：

- Bag of Words。
- TF-IDF。
- n-gram。
- 关键词特征。
- 词典特征。

常用模型：

- 朴素贝叶斯。
- 逻辑回归。
- SVM。
- 决策树。
- 随机森林。
- XGBoost。

### 2. 朴素贝叶斯

朴素贝叶斯基于条件独立假设：

```text
P(class | words) ∝ P(class) Π P(word_i | class)
```

优点：

- 简单快速。
- 小数据下表现不错。
- 适合垃圾邮件分类等任务。

缺点：

- 独立性假设过强。
- 难处理复杂语义。

### 3. SVM

SVM 在 TF-IDF 特征上曾经是强基线。适合中小规模文本分类。

优点：

- 对高维稀疏特征友好。
- 泛化能力较好。

缺点：

- 对大规模数据训练较慢。
- 不擅长语义理解。

### 4. CRF

CRF 常用于序列标注任务，如中文分词、词性标注、命名实体识别。

它能建模标签之间的依赖关系。例如实体标签要满足合法转移：

```text
B-PER 后面可以是 I-PER
I-PER 后面不应突然接 I-LOC
```

深度学习时代常见 BiLSTM-CRF、BERT-CRF。

### 5. 传统方法仍有价值吗

有。传统方法在以下场景仍然有用：

- 数据量小。
- 资源有限。
- 需要可解释。
- 任务简单。
- 延迟要求极高。
- 作为强基线。

不要因为大模型流行就完全忽视传统方法。

## 九、深度学习 NLP

### 1. Embedding 层

深度学习 NLP 通常先把 token id 转成 embedding：

```text
token id -> dense vector
```

Embedding 可以随机初始化，也可以使用预训练词向量。

### 2. CNN for NLP

CNN 可以用于文本分类。它通过不同窗口大小捕捉 n-gram 特征。

例如：

- 2-gram：很好。
- 3-gram：非常好。
- 4-gram：真的非常好。

TextCNN 曾是强大的文本分类模型。

优点：

- 并行。
- 训练快。
- 对局部模式敏感。

缺点：

- 长距离依赖较弱。

### 3. RNN

RNN 按顺序处理文本：

```text
h_t = f(x_t, h_{t-1})
```

它天然适合序列，但训练慢，长距离依赖困难。

### 4. LSTM

LSTM 通过门控机制缓解梯度消失：

- 输入门。
- 遗忘门。
- 输出门。
- 记忆单元。

它曾广泛用于机器翻译、文本分类、序列标注等任务。

### 5. GRU

GRU 是 LSTM 的简化版本，参数更少，训练更快。

### 6. BiLSTM

BiLSTM 同时从左到右和从右到左读取序列，适合理解类任务。

例如命名实体识别中，一个词的标签可能依赖左右上下文。

### 7. Seq2Seq

Seq2Seq 用编码器把输入序列编码成上下文，再由解码器生成输出序列。

应用：

- 机器翻译。
- 摘要。
- 对话。

早期 Seq2Seq 的瓶颈是固定长度上下文向量，后来通过注意力机制改进。

## 十、注意力机制与 Transformer

### 1. 注意力机制

注意力机制的核心公式：

```text
Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) V
```

它让模型根据当前 Query 动态关注相关 Key，并加权读取 Value。

### 2. Self-Attention

Self-Attention 中 Q、K、V 都来自同一个序列。它让序列中每个 token 都能关注其他 token。

优点：

- 并行计算。
- 长距离依赖路径短。
- 表达能力强。

缺点：

- 标准复杂度是 `O(n^2)`。

### 3. Multi-Head Attention

多头注意力让模型在多个子空间中学习不同关系。

```text
MultiHead(Q,K,V) = Concat(head_1,...,head_h)W_O
```

### 4. Transformer

Transformer 由以下组件构成：

- Embedding。
- 位置编码。
- Multi-Head Attention。
- Feed Forward Network。
- 残差连接。
- LayerNorm。
- Mask。

它是现代 NLP 的核心架构。

### 5. Encoder-only、Decoder-only、Encoder-Decoder

Encoder-only：

- 代表：BERT。
- 适合理解任务。

Decoder-only：

- 代表：GPT 类模型。
- 适合生成任务。

Encoder-Decoder：

- 代表：T5、BART。
- 适合条件生成，如翻译、摘要。

## 十一、预训练语言模型

### 1. 为什么需要预训练

标注数据昂贵，但无标注文本很丰富。预训练语言模型先在海量无标注文本上学习语言知识，再迁移到下游任务。

这种范式叫：

```text
Pre-train -> Fine-tune
```

### 2. BERT

BERT 是双向 Transformer Encoder。核心训练目标是 Masked Language Modeling。

输入：

```text
[CLS] 句子 A [SEP] 句子 B [SEP]
```

`[CLS]` 表示常用于分类任务。

BERT 适合：

- 文本分类。
- 命名实体识别。
- 句子匹配。
- 抽取式问答。

### 3. GPT

GPT 是自回归 Decoder-only 模型，训练目标是预测下一个 token。

它适合：

- 文本生成。
- 对话。
- 代码生成。
- 指令跟随。

### 4. T5

T5 使用 Encoder-Decoder，把所有 NLP 任务统一成 text-to-text。

例如：

```text
translate English to German: ...
summarize: ...
classify sentiment: ...
```

### 5. Fine-tuning

微调是在预训练模型基础上，用任务数据继续训练。

常见方式：

- 全参数微调。
- 冻结部分层。
- Adapter。
- LoRA。
- Prompt tuning。

### 6. Prompt-based Learning

Prompt 学习把任务转化为语言模型熟悉的形式。

例如情感分类：

```text
这条评论：“电影很好看。” 情感是：正面
```

大语言模型时代，Prompt 设计成为重要工程能力。

## 十二、大语言模型与生成式 NLP

### 1. 大语言模型是什么

大语言模型通常指参数规模大、训练数据大、基于 Transformer 的语言模型。它通过预测下一个 token 学习语言、知识和模式。

它能完成：

- 对话。
- 写作。
- 摘要。
- 翻译。
- 问答。
- 代码。
- 推理。
- 信息抽取。
- 工具调用。

### 2. In-context Learning

大模型可以通过上下文示例学习任务，而不一定需要更新参数。

示例：

```text
文本：这部电影太棒了
情感：正面

文本：剧情很无聊
情感：负面

文本：演员表现不错
情感：
```

模型可以根据示例输出“正面”。

### 3. Instruction Tuning

指令微调使用大量指令-回答数据，让模型更好遵循人类指令。

### 4. RLHF

RLHF 是 Reinforcement Learning from Human Feedback，通过人类偏好训练奖励模型，再优化语言模型输出，使其更符合人类偏好。

### 5. RAG

RAG 是 Retrieval-Augmented Generation，检索增强生成。

流程：

```text
用户问题 -> 向量检索/关键词检索 -> 相关文档 -> 拼接 Prompt -> LLM 生成
```

适合：

- 企业知识库问答。
- 私有文档问答。
- 需要引用来源的问答。
- 降低幻觉。

### 6. Agent

Agent 是能够规划、调用工具、观察结果并继续行动的模型系统。

典型组成：

- LLM。
- Prompt。
- 工具。
- 记忆。
- 规划。
- 执行环境。

Agent 是 NLP 与软件系统结合的重要方向。

### 7. 大模型的局限

大模型仍有问题：

- 幻觉。
- 事实错误。
- 上下文长度限制。
- 推理不稳定。
- 对提示敏感。
- 安全风险。
- 成本较高。

因此实际应用需要检索、校验、约束输出、评估和监控。

## 十三、常见 NLP 任务精讲

### 1. 文本分类

目标：给文本分配类别。

应用：

- 情感分析。
- 垃圾邮件识别。
- 新闻分类。
- 意图识别。
- 风险文本识别。

常见方法：

- TF-IDF + 逻辑回归。
- TextCNN。
- BERT 微调。
- 大模型提示分类。

### 2. 情感分析

判断文本情感倾向：

- 正面。
- 负面。
- 中性。

难点：

- 讽刺。
- 否定。
- 转折。
- 多方面情感。

例如：

```text
屏幕不错，但电池太差。
```

整体情感可能中性，但具体方面有不同评价。

### 3. 命名实体识别

NER 识别文本中的实体：

- 人名。
- 地名。
- 组织名。
- 时间。
- 金额。
- 产品。
- 疾病。

常见标注格式 BIO：

```text
B-PER I-PER O B-LOC
```

方法：

- CRF。
- BiLSTM-CRF。
- BERT-CRF。
- 指令式大模型抽取。

### 4. 关系抽取

识别实体之间的关系。

例如：

```text
乔布斯创立了苹果公司。
```

实体：

```text
乔布斯 -> 苹果公司
关系：创始人
```

### 5. 信息抽取

从文本中抽取结构化信息。

例如从合同中抽取：

- 甲方。
- 乙方。
- 金额。
- 日期。
- 违约条款。

### 6. 文本匹配

判断两个文本是否语义相似或相关。

应用：

- 搜索排序。
- 问答匹配。
- 重复问题检测。
- 语义检索。

方法：

- Siamese 网络。
- Cross-Encoder。
- Bi-Encoder。
- 向量检索。

### 7. 机器翻译

把一种语言翻译成另一种语言。

核心难点：

- 词序差异。
- 多义词。
- 文化表达。
- 长句结构。

Transformer 最初就是在机器翻译中取得重要突破。

### 8. 文本摘要

摘要分为：

- 抽取式摘要：从原文选句子。
- 生成式摘要：重新生成摘要。

难点：

- 保持事实一致。
- 覆盖重点。
- 避免幻觉。

### 9. 问答系统

类型：

- 检索式问答。
- 抽取式问答。
- 生成式问答。
- 开放域问答。
- 知识库问答。

现代系统常结合 RAG。

### 10. 对话系统

对话系统包括：

- 任务型对话。
- 开放域聊天。
- 客服机器人。
- 智能助手。

关键问题：

- 意图识别。
- 状态跟踪。
- 回复生成。
- 安全控制。
- 长期记忆。

### 11. 文本生成

生成任务包括：

- 续写。
- 改写。
- 扩写。
- 风格转换。
- 文案生成。
- 代码生成。

评估生成质量很难，需要结合自动指标和人工评估。

### 12. 语义检索

语义检索把文本编码成向量，通过向量相似度检索相关内容。

常用在：

- 知识库问答。
- 推荐。
- 搜索增强。
- RAG。

## 十四、评估指标

### 1. 准确率

```text
Accuracy = 正确样本数 / 总样本数
```

适合类别均衡任务。

### 2. 精确率、召回率、F1

精确率：

```text
Precision = TP / (TP + FP)
```

召回率：

```text
Recall = TP / (TP + FN)
```

F1：

```text
F1 = 2PR / (P + R)
```

类别不均衡时 F1 更有参考价值。

### 3. BLEU

BLEU 用于机器翻译，衡量生成结果和参考答案的 n-gram 重合度。

缺点是不能很好衡量语义等价。

### 4. ROUGE

ROUGE 常用于摘要任务，关注生成摘要与参考摘要的重合。

### 5. Perplexity

用于语言模型，衡量模型预测文本的困惑程度。

### 6. EM 和 F1

问答任务中常用：

- EM：完全匹配。
- F1：答案 token 级重合。

### 7. 人工评估

生成式任务通常需要人工评估：

- 相关性。
- 流畅性。
- 事实一致性。
- 完整性。
- 安全性。
- 有用性。

### 8. 大模型评估

大模型评估更复杂，需要：

- 基准测试。
- 真实业务测试。
- 对抗测试。
- 长尾案例。
- 人类偏好。
- 安全红队测试。

不要只看单一榜单分数。

## 十五、数据集构建与标注

### 1. 数据质量比模型更重要

在实际 NLP 项目中，数据质量往往比模型结构更影响效果。脏数据、错标、分布偏差会严重限制模型表现。

### 2. 标注规范

标注前必须定义清晰规范：

- 类别定义。
- 正反例。
- 边界规则。
- 冲突处理。
- 特殊情况。

没有规范的标注会导致模型学到混乱信号。

### 3. 一致性检查

多个标注员之间需要计算一致性，例如 Cohen's Kappa。低一致性说明任务定义不清或标注难度过高。

### 4. 数据划分

常见划分：

```text
训练集 / 验证集 / 测试集
```

注意：

- 保持分布一致。
- 防止重复样本泄漏。
- 时间序列任务按时间划分。
- 用户级数据按用户划分。

### 5. 类别不均衡

类别不均衡常见于风险识别、欺诈检测、医疗文本等任务。

处理方法：

- 重采样。
- 类别权重。
- focal loss。
- 收集更多少数类样本。
- 调整阈值。

### 6. 数据增强

文本数据增强方法：

- 同义词替换。
- 回译。
- 随机删除。
- 随机交换。
- 模板生成。
- 大模型生成样本。

数据增强要谨慎，不能改变标签语义。

## 十六、NLP 工程实践

### 1. 建立强基线

不要一开始就上复杂模型。建议先建立基线：

- TF-IDF + Logistic Regression。
- FastText。
- 小型 BERT。

基线能帮助判断任务难度和数据质量。

### 2. 错误分析

错误分析比盲目调参重要。要查看模型错在哪里：

- 某类样本特别差？
- 长文本效果差？
- 口语化文本差？
- 新词识别差？
- 否定句判断错？
- 训练集是否缺少类似样本？

错误分析后再决定改数据、改模型还是改规则。

### 3. 规则与模型结合

实际业务中，不必迷信纯模型。规则在高确定性场景很好用，例如：

- 敏感词拦截。
- 格式校验。
- 黑名单。
- 金额日期抽取。

常见组合：

```text
规则兜底 + 模型泛化 + 人工审核
```

### 4. 阈值选择

分类模型输出概率后，需要选择阈值。默认 0.5 不一定最优。

例如风险识别中，宁愿召回更多可疑样本，再人工审核。此时应提高召回率，降低阈值。

### 5. 在线监控

模型上线后要监控：

- 输入分布变化。
- 预测类别分布。
- 置信度变化。
- 延迟。
- 错误率。
- 用户反馈。

NLP 数据分布变化很快，尤其是舆情、社媒、业务客服场景。

### 6. 版本管理

需要管理：

- 数据版本。
- 代码版本。
- 模型版本。
- tokenizer 版本。
- 配置文件。
- 评估报告。

Tokenizer 和模型必须匹配，否则结果会异常。

## 十七、模型训练、调参与优化

### 1. 常见超参数

- 学习率。
- batch size。
- epoch。
- max sequence length。
- dropout。
- weight decay。
- warmup steps。
- scheduler。
- gradient clipping。

### 2. 学习率

学习率过大，训练不稳定；学习率过小，收敛慢。微调预训练模型时，学习率通常比从头训练小。

### 3. Batch Size

batch size 影响训练稳定性和显存占用。大 batch 可能需要调整学习率。

### 4. 序列长度

序列越长，注意力开销越大。要根据任务选择 max length。

文本分类中，不一定保留全文。可以截断开头、结尾或关键信息段。

### 5. 过拟合

表现：

- 训练集效果高。
- 验证集效果低。

处理方法：

- 增加数据。
- 数据增强。
- Dropout。
- Weight decay。
- Early stopping。
- 减小模型。

### 6. 欠拟合

表现：

- 训练集和验证集都差。

处理方法：

- 增大模型。
- 训练更久。
- 调大学习率或改善优化。
- 检查标签质量。
- 增加有效特征。

### 7. 混合精度训练

使用 FP16/BF16 可以减少显存、加快训练，但需要注意数值稳定。

### 8. LoRA 微调

LoRA 通过低秩矩阵适配大模型，只训练少量参数。

优点：

- 显存需求低。
- 训练成本低。
- 便于保存多个任务适配器。

适合大模型领域适配和指令微调。

## 十八、部署与推理优化

### 1. 推理延迟

NLP 模型部署需要关注：

- 单请求延迟。
- 吞吐量。
- 显存。
- 并发。
- 冷启动。
- 最大输入长度。

### 2. 模型压缩

常见方法：

- 剪枝。
- 蒸馏。
- 量化。
- 小模型替代。

### 3. 知识蒸馏

用大模型作为 teacher，训练小模型 student。小模型更快，更适合线上部署。

### 4. 量化

把权重从 FP32/FP16 转成 INT8、INT4 等低精度，减少显存和计算成本。

量化可能带来精度下降，需要评估。

### 5. 缓存

对重复请求可以缓存结果。RAG 系统中也可以缓存：

- 查询向量。
- 检索结果。
- LLM 输出。

### 6. 批处理

把多个请求组成 batch 可以提高吞吐，但可能增加单请求等待时间。需要在延迟和吞吐之间平衡。

### 7. 安全与合规

NLP 系统可能处理敏感数据，需要关注：

- 隐私保护。
- 日志脱敏。
- 权限控制。
- 数据保留策略。
- 输出安全。

## 十九、常见误区与面试问题

### 1. 常见误区

#### 误区一：NLP 就是调大模型

大模型很强，但 NLP 还包括数据、标注、检索、评估、规则、部署和业务理解。

#### 误区二：预处理越多越好

过度清洗可能删除有用信息。使用预训练模型时，更要保持输入与预训练分布一致。

#### 误区三：准确率高就够了

类别不均衡任务中，准确率可能误导。需要看 Precision、Recall、F1、业务成本。

#### 误区四：模型上线后就结束

NLP 数据会变化，需要监控、反馈和持续迭代。

#### 误区五：大模型不会犯低级错误

大模型可能幻觉、编造、误解指令。关键业务必须加验证和约束。

### 2. 面试问题：NLP 常见流程是什么？

回答：

```text
数据收集 -> 清洗预处理 -> tokenization -> 特征/表示学习 -> 模型训练 -> 评估 -> 错误分析 -> 部署监控
```

使用大模型时，还会包括 prompt 构造、检索增强、输出解析、安全控制。

### 3. 面试问题：Word2Vec 和 BERT 表示有什么区别？

Word2Vec 是静态词向量，同一个词在不同上下文中向量相同。BERT 是上下文相关表示，同一个词会根据上下文生成不同表示。

### 4. 面试问题：BERT 和 GPT 有什么区别？

BERT 是 Encoder-only，双向注意力，适合理解任务；GPT 是 Decoder-only，因果注意力，适合生成任务。

### 5. 面试问题：TF-IDF 的思想是什么？

一个词在当前文档出现越多越重要，但如果它在所有文档都频繁出现，则区分度低。TF-IDF 同时考虑词频和逆文档频率。

### 6. 面试问题：什么是 OOV？

OOV 是词表外词。传统词级模型遇到 OOV 会映射为 `<UNK>`。子词建模如 BPE、WordPiece 可以缓解 OOV。

### 7. 面试问题：NER 常用什么标注格式？

常用 BIO 或 BIOES。BIO 中：

- B 表示实体开始。
- I 表示实体内部。
- O 表示非实体。

### 8. 面试问题：如何处理类别不均衡？

可以使用重采样、类别权重、focal loss、调整阈值、收集更多少数类样本，并使用 F1、PR-AUC 等指标评估。

### 9. 面试问题：RAG 是什么？

RAG 是检索增强生成。它先从知识库检索相关文档，再把文档作为上下文交给大模型生成答案，以提升事实性和可追溯性。

### 10. 面试问题：如何做 NLP 错误分析？

按任务、类别、文本长度、数据来源、语言风格、实体类型、置信度区间等维度切分错误样本，找出主要错误模式，再决定补数据、改标注、调模型还是加规则。

### 精讲扩展 3：NLP自然语言处理万字学习笔记 的模型假设、损失函数 与工程化理解

学习 $topic 时，不能只把它当成一个孤立知识点来背诵，而要把它放到 $category 的完整问题链条里理解。一个知识点通常同时包含概念定义、适用边界、输入输出、运行过程、常见异常和工程取舍。真正掌握它，意味着看到一个具体场景时，能够判断它解决什么问题、依赖哪些前提、失败时会出现什么现象，以及应该用什么手段验证自己的判断。

从 $a 的角度看，最重要的是先建立清晰的对象模型。也就是明确系统里有哪些参与者、它们之间如何连接、数据或控制信号如何流动、哪些环节是同步的、哪些环节是异步的、哪些状态是临时状态、哪些状态需要长期保存。很多初学问题并不是公式不会、API 不熟，而是对象边界不清：把配置当成状态，把结果当成过程，把局部现象当成全局规律。写笔记时建议始终追问：这个概念的主体是谁，输入是什么，输出是什么，中间约束是什么，错误会在哪里暴露。

从 $b 的角度看，流程比单点知识更关键。一个成熟方案通常不是单个技巧，而是一组步骤：先确定目标，再拆分约束，然后选择工具，最后通过测试和复盘确认效果。比如在实际项目中，不能只问“怎么实现”，还要问“为什么要这样实现”“有没有更简单的替代方案”“边界条件是什么”“数据量、并发量、实时性、可靠性变化后还能不能工作”。这种流程意识能够避免把学习停留在教程层面，也能让后续排错有明确路线。

$topic 的 $c 往往决定它在真实项目中的稳定性。理论上可行的方案，到了工程环境中会受到数据质量、硬件条件、依赖版本、网络环境、团队协作、部署方式和维护成本影响。写代码或做设计时，应该把正常路径和异常路径同时考虑：正常情况下如何运行，输入为空怎么办，超时怎么办，重复执行怎么办，部分成功怎么办，版本升级后兼容性怎么办，日志和指标如何证明系统确实按预期工作。

进一步看 $d，它通常对应性能、可靠性或可维护性的核心矛盾。很多技术选择并没有绝对正确答案，只有是否适合当前约束。例如追求极致性能可能牺牲可读性，追求高度抽象可能增加调试成本，追求快速交付可能留下技术债，追求完全通用可能让简单场景变复杂。高质量笔记应该把这些取舍写出来，而不是只给一个“推荐方案”。推荐方案背后的条件越清楚，迁移到新场景时越不容易误用。

最后从 $e 的角度进行复盘，可以把知识从“看懂”推进到“会用”。建议为 $topic 建立三个层次的检查：第一层是概念检查，确认术语、流程和边界没有混淆；第二层是实践检查，确认能够独立完成一个最小案例；第三层是工程检查，确认这个案例在异常、规模、性能和维护方面经得起追问。每次学习完一个章节，都可以用这三层检查反向补齐笔记。

#### 典型场景拆解

在真实场景中，$topic 通常会经历“需求出现、方案选择、实现落地、问题暴露、持续优化”几个阶段。需求出现时，要先判断这个需求属于基础能力、性能优化、体验改进、可靠性建设还是长期架构演进。不同类型的需求对方案的评价标准不同：基础能力看正确性，性能优化看指标，体验改进看路径是否顺滑，可靠性建设看故障时能否降级和恢复，架构演进看未来变化是否容易吸收。

方案选择阶段，最容易犯的错误是直接套用熟悉工具。更稳妥的方式是列出约束：数据规模、时延要求、资源预算、团队熟悉度、运维能力、安全要求、可测试性和长期维护成本。只有把约束列清楚，才能解释为什么选择当前方案。否则方案看似高级，实际可能只是增加了复杂度。

实现落地阶段，要把 $a 和 $b 拆成可验证的小步骤。每一步都应该有明确的输入、输出和检查方式。对学习笔记而言，这意味着不能只有大段概念，还应该补充流程图式的文字描述、伪代码、命令示例、参数解释、错误现象和排查路径。这样以后复习时，笔记不仅能帮助理解，也能直接指导实践。

问题暴露阶段，要优先区分“理解错误、实现错误、环境错误、数据错误、依赖错误、边界条件错误”。很多复杂问题之所以难排，是因为一开始就把问题归因到错误层级。例如把配置问题当成算法问题，把权限问题当成代码问题，把数据分布变化当成模型失效，把硬件噪声当成软件逻辑错误。好的排查顺序应该从可观测事实开始，而不是从猜测开始。

持续优化阶段，不应只追求把当前问题压下去，还要沉淀成规则。比如记录触发条件、影响范围、定位方法、最终修复、预防措施和可监控指标。这样下一次出现类似问题时，团队可以复用经验，而不是重新从零排查。

#### 常见误区与纠偏

第一个误区是只记结论，不记前提。$topic 中很多结论都是有条件的：适用于小规模，不一定适用于大规模；适用于离线处理，不一定适用于实时系统；适用于单机环境，不一定适用于分布式环境；适用于教学案例，不一定适用于生产项目。纠偏方法是在每个重要结论后面补一句“适用条件”和“不适用情况”。

第二个误区是只关注工具，不关注模型。工具会变化，模型更稳定。无论工具名称如何变化，底层仍然要解决输入建模、状态管理、资源调度、错误恢复、性能约束和质量验证这些问题。学习 $topic 时，应该把工具用法和底层模型分开记录：工具命令解决“怎么做”，底层模型解释“为什么这样做”。

第三个误区是没有验证意识。很多笔记写得很完整，但没有说明如何确认自己做对了。对于 $category 相关主题，验证至少应包含最小样例、边界样例、异常样例和性能样例。最小样例证明流程跑通，边界样例证明理解完整，异常样例证明系统可恢复，性能样例证明方案在目标规模下仍然可用。

第四个误区是忽略可维护性。短期学习时，能跑通就容易产生掌握的错觉；长期使用时，命名、分层、注释、测试、日志、版本管理和文档才会决定知识能否转化为稳定能力。扩充 $topic 笔记时，应把“如何写得清楚、如何排查、如何交接、如何复盘”也纳入内容。
