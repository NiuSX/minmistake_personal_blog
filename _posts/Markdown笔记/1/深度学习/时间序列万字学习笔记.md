## 一、时间序列是什么

### 1. 基本定义

时间序列是按照时间顺序记录的一组数据点。每个数据点都对应一个时间戳或时间位置。

形式上可以表示为：

```text
y_1, y_2, ..., y_t, ..., y_T
```

其中 `t` 表示时间，`y_t` 表示第 `t` 个时间点的观测值。

常见时间序列包括：

- 每日销售额。
- 每小时网站访问量。
- 每分钟服务器 CPU 使用率。
- 每秒股票价格。
- 每月用电量。
- 每季度 GDP。
- 每日气温。
- 传感器数据。

时间序列的关键特点是：顺序重要。不能像普通表格数据那样随意打乱。

### 2. 时间序列与普通数据的区别

普通监督学习数据通常假设样本独立同分布，而时间序列数据往往存在时间依赖。

例如今天的销量可能与昨天、上周同一天、节假日、促销活动、季节等因素有关。

时间序列中的核心问题是：

```text
过去如何影响现在，现在如何预测未来。
```

### 3. 时间序列分析与预测

时间序列分析侧重理解数据结构：

- 是否有趋势。
- 是否有季节性。
- 是否平稳。
- 是否存在异常。
- 是否有周期。
- 不同变量是否存在领先滞后关系。

时间序列预测侧重预测未来：

```text
给定历史数据，预测未来一个或多个时间点。
```

例如根据过去 3 年月销量预测未来 6 个月销量。

## 二、时间序列任务类型

### 1. 单步预测

预测下一个时间点：

```text
y_{t+1}
```

例如根据过去 24 小时流量预测下一小时流量。

### 2. 多步预测

预测未来多个时间点：

```text
y_{t+1}, y_{t+2}, ..., y_{t+h}
```

其中 `h` 是预测步长。

多步预测更难，因为误差可能逐步累积。

### 3. 单变量预测

只使用目标变量自身历史值：

```text
过去销量 -> 未来销量
```

### 4. 多变量预测

使用多个变量：

```text
销量、价格、广告、节假日、天气 -> 未来销量
```

多变量模型可以利用外生信息，但也更容易引入噪声和数据泄漏。

### 5. 时间序列分类

输入一段序列，输出类别。

例如：

- 心电图分类。
- 设备状态分类。
- 用户行为序列分类。

### 6. 时间序列异常检测

识别异常时间点或异常片段。

例如：

- 服务器流量突增。
- 传感器故障。
- 金融交易异常。
- 工业设备异常震动。

### 7. 缺失值填补

根据上下文补全缺失时间点。

例如传感器某些时间点丢失，需要估计。

## 三、时间序列的组成部分

### 1. 趋势 Trend

趋势表示长期方向性变化。

例如：

- 用户数逐年增长。
- 设备性能逐渐衰减。
- 城市用电量长期上升。

趋势可以是线性的，也可以是非线性的。

### 2. 季节性 Seasonality

季节性是固定周期内重复出现的模式。

例如：

- 电商销量每周末更高。
- 餐厅午餐和晚餐时间订单更多。
- 用电量夏季和冬季更高。
- 网站访问量工作日和周末不同。

季节性周期可以是：

- 小时内。
- 天内。
- 周内。
- 月内。
- 年内。

### 3. 周期性 Cyclicity

周期性也表现为起伏，但周期长度不一定固定。

例如经济周期、行业周期。它不像季节性那样有稳定固定频率。

### 4. 噪声 Noise

噪声是无法解释的随机波动。任何真实时间序列都包含噪声。

建模目标不是完全消除噪声，而是提取可预测结构。

### 5. 异常 Outlier

异常是明显偏离正常模式的数据点。

异常可能来自：

- 真实事件。
- 数据错误。
- 系统故障。
- 突发活动。
- 极端天气。

处理异常时不能简单删除，需要判断它对业务是否有意义。

### 6. 加法模型与乘法模型

加法分解：

```text
y_t = trend_t + seasonality_t + residual_t
```

适合季节波动幅度基本稳定的序列。

乘法分解：

```text
y_t = trend_t * seasonality_t * residual_t
```

适合季节波动随趋势水平增大的序列。

例如销量越高，季节性波动幅度也越大。

## 四、数据预处理与探索分析

### 1. 时间索引处理

时间序列必须先确保时间字段正确：

- 时间格式是否一致。
- 时区是否正确。
- 是否有重复时间戳。
- 是否有缺失时间点。
- 采样频率是否稳定。

时间字段错误会导致后续分析全部不可靠。

### 2. 重采样

重采样是改变时间粒度。

例如：

- 分钟数据转小时数据。
- 日数据转月数据。
- 月数据转季度数据。

常见聚合方式：

- sum：销售额、访问量。
- mean：温度、CPU 使用率。
- max：峰值流量。
- last：收盘价。

选择聚合方式必须符合业务含义。

### 3. 缺失值处理

缺失值常见原因：

- 采集失败。
- 系统中断。
- 节假日无数据。
- 数据延迟。

处理方法：

- 删除。
- 前向填充。
- 后向填充。
- 线性插值。
- 移动平均。
- 模型填补。

时间序列中不能随便用未来信息填过去，否则会造成数据泄漏。

### 4. 异常值处理

异常处理方式：

- 保留。
- 替换为缺失再填补。
- 截尾。
- 单独加异常标记特征。
- 按业务规则修正。

如果异常代表真实事件，例如促销爆发，不应简单删除；如果是采集错误，可以修正或剔除。

### 5. 可视化

时间序列分析第一步通常是画图。

需要观察：

- 趋势。
- 周期。
- 季节性。
- 异常点。
- 方差是否变化。
- 是否有结构突变。

常用图：

- 折线图。
- 移动平均图。
- 分组箱线图。
- 季节图。
- 自相关图。
- 分解图。

### 6. 移动平均

移动平均用于平滑短期波动：

```text
MA_t = (y_t + y_{t-1} + ... + y_{t-k+1}) / k
```

窗口越大，曲线越平滑，但响应越滞后。

## 五、平稳性、差分与白噪声

### 1. 什么是平稳性

平稳时间序列的统计性质不随时间变化。弱平稳通常要求：

- 均值恒定。
- 方差恒定。
- 自协方差只与时间间隔有关，不与具体时间有关。

很多统计模型，如 ARMA，需要序列平稳。

### 2. 非平稳序列

非平稳常见表现：

- 明显趋势。
- 方差随时间变化。
- 季节性变化。
- 结构突变。

例如长期增长的销售额通常非平稳。

### 3. 为什么平稳重要

如果序列统计规律一直变化，模型很难从过去推断未来。平稳性让历史规律有机会延续到未来。

### 4. 差分

差分用于去除趋势。

一阶差分：

```text
z_t = y_t - y_{t-1}
```

如果原序列是线性趋势，一阶差分可能变平稳。

二阶差分：

```text
w_t = z_t - z_{t-1}
```

差分不要过度。过度差分会引入额外噪声。

### 5. 季节差分

如果存在周期 `s` 的季节性：

```text
z_t = y_t - y_{t-s}
```

例如日数据有周季节性，`s=7`。

### 6. ADF 检验

ADF 检验用于判断序列是否存在单位根。

常见理解：

- p 值小于显著性水平，拒绝单位根假设，认为序列较可能平稳。
- p 值较大，不能拒绝单位根假设，序列可能非平稳。

不要只依赖检验，要结合图形和业务。

### 7. 白噪声

白噪声是无可预测结构的随机序列。

如果残差是白噪声，说明模型已经提取了可预测信息；如果残差仍有自相关，说明模型还有改进空间。

## 六、自相关与偏自相关

### 1. 自相关 ACF

自相关衡量序列与自身滞后版本的相关性。

例如 lag=1：

```text
corr(y_t, y_{t-1})
```

lag=7：

```text
corr(y_t, y_{t-7})
```

ACF 可以帮助发现季节性和 AR/MA 阶数。

### 2. 偏自相关 PACF

偏自相关衡量在控制中间滞后项后，`y_t` 与 `y_{t-k}` 的直接相关性。

PACF 常用于识别 AR 阶数。

### 3. ACF/PACF 判断模型

经验规则：

- AR(p)：PACF p 阶截尾，ACF 拖尾。
- MA(q)：ACF q 阶截尾，PACF 拖尾。
- ARMA：ACF 和 PACF 都拖尾。

这只是经验，不是绝对规则。

### 4. 季节性识别

如果 ACF 在 lag=7、14、21 等位置有明显峰值，可能存在周季节性。

如果月数据在 lag=12、24 明显相关，可能存在年季节性。

## 七、经典统计模型

### 1. AR 模型

AR 是 AutoRegressive，自回归模型。当前值由过去若干值线性组合得到。

AR(p)：

```text
y_t = c + phi_1 y_{t-1} + ... + phi_p y_{t-p} + epsilon_t
```

适合当前值受过去值影响明显的序列。

### 2. MA 模型

MA 是 Moving Average，移动平均模型。当前值由过去误差项线性组合得到。

MA(q)：

```text
y_t = c + epsilon_t + theta_1 epsilon_{t-1} + ... + theta_q epsilon_{t-q}
```

注意这里的移动平均不是简单滑动平均，而是误差项模型。

### 3. ARMA 模型

ARMA 结合 AR 和 MA：

```text
y_t = c + AR 部分 + MA 部分 + epsilon_t
```

适合平稳序列。

### 4. ARIMA 模型

ARIMA 是 AutoRegressive Integrated Moving Average。

ARIMA(p,d,q) 中：

- p：AR 阶数。
- d：差分阶数。
- q：MA 阶数。

流程：

1. 对非平稳序列做 d 阶差分。
2. 对差分后的平稳序列拟合 ARMA(p,q)。

### 5. SARIMA

SARIMA 是带季节项的 ARIMA。

记作：

```text
SARIMA(p,d,q)(P,D,Q,s)
```

其中：

- `P,D,Q` 是季节性 AR、差分、MA 阶数。
- `s` 是季节周期。

适合有明显季节性的序列。

### 6. 模型选择

选择 ARIMA 阶数可参考：

- ACF/PACF。
- AIC。
- BIC。
- 验证集误差。
- 残差诊断。

不要只追求训练集拟合好，预测效果才是重点。

### 7. 残差诊断

拟合模型后检查残差：

- 残差均值是否接近 0。
- 残差是否无自相关。
- 残差方差是否稳定。
- 是否存在异常结构。

如果残差仍有明显自相关，说明模型没捕捉完整结构。

## 八、指数平滑与 ETS 模型

### 1. 简单指数平滑

简单指数平滑适合无明显趋势和季节性的序列。

公式：

```text
l_t = alpha y_t + (1 - alpha) l_{t-1}
```

`alpha` 越大，模型越重视近期观测。

### 2. Holt 线性趋势法

适合有趋势但无季节性的序列。

包含：

- level。
- trend。

### 3. Holt-Winters 方法

适合有趋势和季节性的序列。

分为：

- 加法季节性。
- 乘法季节性。

### 4. ETS 模型

ETS 表示：

- Error。
- Trend。
- Seasonality。

它是指数平滑模型族的统一表达。

### 5. 指数平滑优点

- 简单。
- 解释性强。
- 对短期预测有效。
- 对业务时间序列常有强基线价值。

### 6. 指数平滑局限

- 难处理复杂外生变量。
- 对结构突变敏感。
- 表达能力有限。

## 九、季节性模型与 Prophet

### 1. 复杂季节性

业务数据常有多个季节性：

- 小时季节性。
- 周季节性。
- 年季节性。
- 节假日效应。

传统 ARIMA 处理复杂多季节性可能较麻烦。

### 2. Prophet 的基本思想

Prophet 将时间序列分解为：

```text
y(t) = g(t) + s(t) + h(t) + epsilon_t
```

其中：

- `g(t)` 是趋势。
- `s(t)` 是季节性。
- `h(t)` 是节假日效应。
- `epsilon_t` 是误差。

### 3. Prophet 适合场景

适合：

- 有明显季节性。
- 有节假日效应。
- 缺失值较多。
- 异常点存在。
- 需要快速业务基线。

不适合：

- 高频复杂金融序列。
- 强多变量依赖。
- 极高精度要求。

### 4. 节假日特征

节假日对销量、流量、出行等影响很大。建模时需要加入：

- 春节。
- 国庆。
- 双十一。
- 周末。
- 黑五。
- 特殊促销。

### 5. 业务事件建模

除了法定节假日，还要考虑业务事件：

- 促销活动。
- 版本发布。
- 广告投放。
- 价格调整。
- 供应链中断。

这些事件往往比模型结构更重要。

## 十、时间序列特征工程

### 1. 滞后特征

滞后特征是过去值：

```text
lag_1 = y_{t-1}
lag_7 = y_{t-7}
lag_30 = y_{t-30}
```

它是机器学习时间序列预测最常用特征。

### 2. 滚动窗口特征

例如：

```text
过去 7 天均值
过去 7 天最大值
过去 7 天标准差
过去 30 天中位数
```

这些特征描述近期趋势和波动。

### 3. 时间特征

从时间戳提取：

- 小时。
- 星期几。
- 是否周末。
- 月份。
- 季度。
- 是否月初。
- 是否月末。
- 是否节假日。

### 4. 周期编码

小时、星期、月份是周期变量。可以用 sin/cos 编码：

```text
sin(2π * hour / 24)
cos(2π * hour / 24)
```

这样可以表达 23 点和 0 点相邻。

### 5. 外生变量

外生变量包括：

- 价格。
- 广告。
- 天气。
- 节假日。
- 促销。
- 库存。
- 竞争对手活动。

外生变量必须确保预测未来时可获得。否则会造成数据泄漏。

### 6. 目标编码与分组特征

多商品、多门店预测中，可以构造：

- 商品历史均值。
- 门店历史均值。
- 类目历史趋势。
- 区域销量特征。

构造时必须使用历史窗口，不能使用未来。

## 十一、机器学习时间序列预测

### 1. 将时间序列转成监督学习

把历史窗口作为特征，未来值作为标签。

例如：

```text
特征: y_{t-1}, y_{t-2}, y_{t-7}, rolling_mean_7
标签: y_t
```

这样就可以使用传统机器学习模型。

### 2. 常用模型

- 线性回归。
- Ridge/Lasso。
- 随机森林。
- XGBoost。
- LightGBM。
- CatBoost。

树模型在业务预测中非常常用，尤其是有大量外生变量和类别特征时。

### 3. 优点

- 可处理非线性。
- 可加入多种特征。
- 对缺失和异常相对鲁棒。
- 工程效果好。
- 训练和推理较快。

### 4. 缺点

- 需要手工构造特征。
- 多步预测需要特殊处理。
- 不天然建模序列结构。

### 5. 多步预测策略

#### 递归预测

先预测 `t+1`，再把预测值作为输入预测 `t+2`。

优点简单，缺点误差累积。

#### 直接预测

为每个预测步长训练一个模型：

```text
model_1 -> y_{t+1}
model_2 -> y_{t+2}
```

优点避免误差累积，缺点模型多。

#### 多输出预测

一个模型直接输出未来多个时间点。

### 6. 全局模型与局部模型

局部模型：

```text
每个商品/门店单独训练模型
```

全局模型：

```text
所有商品/门店共用一个模型，并加入 ID 特征
```

全局模型能共享信息，适合大量相关序列。

## 十二、深度学习时间序列模型

### 1. RNN/LSTM/GRU

RNN 类模型按时间步处理序列，适合序列依赖。

LSTM 和 GRU 通过门控机制缓解长距离依赖问题。

输入形状通常为：

```text
[batch, seq_len, features]
```

输出可以是：

- 单步预测。
- 多步预测。
- 序列分类。

### 2. 1D CNN

一维卷积可以在时间轴上提取局部模式。

适合：

- 传感器数据。
- 音频特征。
- 高频序列。
- 局部模式明显的序列。

优点是并行快，缺点是长距离依赖需要多层或大感受野。

### 3. TCN

TCN 是 Temporal Convolutional Network，常使用：

- 因果卷积。
- 空洞卷积。
- 残差连接。

它可以高效建模长序列，并保持时间因果性。

### 4. Transformer

Transformer 通过自注意力建模时间点之间关系。

优点：

- 长距离依赖强。
- 并行训练。
- 适合多变量长序列。

缺点：

- 标准注意力复杂度高。
- 对数据量和调参要求较高。

### 5. Informer、Autoformer 等

时间序列 Transformer 变体常试图解决：

- 长序列计算复杂度。
- 趋势季节性分解。
- 多步预测稳定性。
- 注意力稀疏化。

### 6. 深度模型适用条件

深度学习不是总是更好。它通常需要：

- 足够数据量。
- 多条相关序列。
- 复杂非线性。
- 多变量输入。
- 合理算力。

小数据单变量预测中，ARIMA、ETS、LightGBM 可能更可靠。

## 十三、多变量时间序列

### 1. 多变量与外生变量

多变量时间序列包含多个随时间变化的变量。

例如预测销量：

```text
y: 销量
x1: 价格
x2: 广告
x3: 库存
x4: 天气
```

### 2. VAR 模型

VAR 是 Vector AutoRegression，多变量自回归模型。每个变量都由所有变量的滞后项解释。

适合研究变量间动态关系。

### 3. Granger 因果

如果变量 X 的过去值能显著改善对 Y 的预测，可以说 X Granger 导致 Y。

注意：Granger 因果不是严格因果，只是预测意义上的领先关系。

### 4. 多变量建模难点

- 变量间相关性复杂。
- 外生变量未来值可能未知。
- 缺失值更多。
- 数据泄漏风险更高。
- 维度高，容易过拟合。

### 5. 未来可知变量与未来未知变量

未来可知：

- 日历。
- 节假日。
- 计划促销。
- 预定价格。

未来未知：

- 真实天气。
- 竞争对手行为。
- 未来需求。

建模时必须区分，否则线上无法使用。

## 十四、异常检测

### 1. 异常类型

#### 点异常

单个时间点异常。

#### 上下文异常

在特定上下文中异常。例如冬天 30 度异常，夏天 30 度正常。

#### 集体异常

一段时间整体模式异常。

### 2. 统计方法

常见：

- Z-score。
- IQR。
- 移动均值残差。
- 控制图。

### 3. 基于预测的方法

先用模型预测正常值，再看实际值与预测值偏差：

```text
residual = y_t - y_hat_t
```

残差过大则异常。

### 4. 基于分解的方法

分解出趋势和季节性，对残差做异常检测。

### 5. 机器学习方法

- Isolation Forest。
- One-Class SVM。
- AutoEncoder。
- LSTM AutoEncoder。

### 6. 异常检测难点

- 异常样本少。
- 标签缺失。
- 正常模式随时间变化。
- 误报和漏报成本不同。
- 业务解释很重要。

## 十五、评估指标与验证方法

### 1. MAE

平均绝对误差：

```text
MAE = mean(|y - y_hat|)
```

直观，和原始单位一致。

### 2. MSE 与 RMSE

MSE：

```text
MSE = mean((y - y_hat)^2)
```

RMSE：

```text
RMSE = sqrt(MSE)
```

对大误差更敏感。

### 3. MAPE

平均绝对百分比误差：

```text
MAPE = mean(|(y - y_hat) / y|)
```

缺点：

- 当 y 接近 0 时不稳定。
- 对低销量序列不友好。

### 4. SMAPE

对称 MAPE：

```text
SMAPE = mean(2|y - y_hat| / (|y| + |y_hat|))
```

一定程度缓解 MAPE 问题。

### 5. WAPE

加权绝对百分比误差：

```text
WAPE = sum(|y - y_hat|) / sum(|y|)
```

业务预测中常用，尤其适合聚合评估。

### 6. 时间序列验证

不能随机打乱划分训练测试。应按时间划分：

```text
训练: 过去
验证: 之后
测试: 最后
```

### 7. 滚动验证

滚动窗口验证更接近真实预测：

```text
训练到 t，预测 t+1 到 t+h
训练到 t+h，预测下一段
```

可以评估模型在多个时间段的稳定性。

### 8. Baseline 很重要

常见 baseline：

- 预测等于上一期。
- 预测等于上周同日。
- 移动平均。
- 季节平均。

复杂模型必须明显超过简单 baseline 才有意义。

## 十六、预测区间与不确定性

### 1. 点预测与区间预测

点预测给出一个值：

```text
明天销量 = 100
```

区间预测给出范围：

```text
明天销量 80 到 120，置信水平 90%
```

业务决策常常需要区间而非单点。

### 2. 为什么需要不确定性

库存、排班、能源调度、金融风险都需要考虑不确定性。

只给一个点预测可能导致决策过于自信。

### 3. 分位数预测

模型直接预测不同分位数：

```text
P10, P50, P90
```

损失函数可用 pinball loss。

### 4. 残差法

根据历史残差分布构造预测区间。

简单但假设未来误差分布与历史类似。

### 5. Bootstrap

通过重采样残差或样本估计不确定性。

## 十七、工程实践流程

### 1. 明确业务目标

先问：

- 预测什么？
- 预测多远？
- 多久更新一次？
- 误差成本是什么？
- 未来可用哪些变量？
- 结果如何用于决策？

没有业务定义，模型指标没有意义。

### 2. 数据检查

检查：

- 时间范围。
- 频率。
- 缺失。
- 异常。
- 重复。
- 分布变化。
- 数据延迟。

### 3. 建立 baseline

先做简单模型：

- naive。
- seasonal naive。
- moving average。
- ETS。

### 4. 特征工程

构造：

- lag。
- rolling。
- calendar。
- holiday。
- external features。

确保所有特征在预测时可用。

### 5. 模型迭代

从简单到复杂：

```text
Baseline -> ARIMA/ETS -> LightGBM -> LSTM/Transformer
```

不要一开始就使用最复杂模型。

### 6. 错误分析

按维度切分误差：

- 时间段。
- 商品。
- 门店。
- 区域。
- 节假日。
- 高峰/低谷。
- 异常事件。

找出主要误差来源。

### 7. 上线监控

监控：

- 预测误差。
- 输入数据延迟。
- 特征分布漂移。
- 目标分布漂移。
- 异常比例。
- 模型版本。

### 8. 重训练策略

常见：

- 固定周期重训。
- 数据漂移触发重训。
- 误差超阈值触发重训。
- 滚动窗口更新。

## 十八、常见误区

### 1. 随机划分训练测试

时间序列不能随机打乱，否则未来信息可能泄漏到训练集。

### 2. 使用未来不可得变量

例如预测未来销量时使用未来真实库存、未来真实天气，会导致离线效果虚高。

### 3. 只看一个指标

不同业务对误差敏感不同。可能需要同时看 MAE、RMSE、WAPE、分组误差。

### 4. 复杂模型一定更好

很多业务序列用简单季节 naive 或 ETS 就很强。复杂模型需要证明价值。

### 5. 忽视异常和业务事件

促销、节假日、断货、系统故障会极大影响序列。纯模型不一定能自动理解。

### 6. 过度差分

差分可以去趋势，但过度差分会损失信息并增加噪声。

### 7. 忽略预测周期

预测未来 1 天和未来 90 天是不同问题，模型和特征都可能不同。

## 十九、面试高频问题

### 1. 什么是时间序列？

时间序列是按照时间顺序排列的一组观测值。其核心特点是样本之间通常存在时间依赖，不能随意打乱顺序。

### 2. 时间序列有哪些组成部分？

通常包括趋势、季节性、周期性、噪声和异常。

### 3. 什么是平稳性？

平稳序列的均值、方差和自协方差结构不随时间变化。很多统计模型要求序列平稳。

### 4. 差分的作用是什么？

差分用于去除趋势或季节性，使序列更接近平稳。

### 5. ARIMA 的 p、d、q 分别是什么？

- p：自回归阶数。
- d：差分阶数。
- q：移动平均阶数。

### 6. ACF 和 PACF 有什么区别？

ACF 衡量序列与滞后项的总体相关性；PACF 控制中间滞后后衡量直接相关性。

### 7. 如何做时间序列交叉验证？

应按时间顺序滚动验证，用过去训练、未来验证，不能随机划分。

### 8. MAPE 有什么问题？

当真实值接近 0 时，MAPE 会非常大或不稳定；对低值序列不友好。

### 9. 如何避免时间序列数据泄漏？

所有特征必须只使用预测时间点之前可获得的信息。数据划分按时间进行，滚动特征不能包含未来值。

### 10. 机器学习如何做时间序列预测？

把时间序列转换为监督学习问题，构造滞后特征、滚动统计、时间特征和外生变量，再使用 LightGBM、XGBoost、随机森林等模型。

### 11. 单步预测和多步预测有什么区别？

单步预测只预测下一个时间点。多步预测预测未来多个时间点，可能面临误差累积和不同预测步长模式差异。

### 12. 什么是预测区间？

预测区间给出未来值可能落入的范围，用于表达不确定性，比单点预测更适合风险决策。

## 二十、学习路线与总结

### 1. 第一阶段：基础概念

掌握：

- 趋势。
- 季节性。
- 周期性。
- 噪声。
- 异常。
- 重采样。
- 缺失处理。

### 2. 第二阶段：统计模型

学习：

- 平稳性。
- ACF/PACF。
- AR。
- MA。
- ARMA。
- ARIMA。
- SARIMA。
- ETS。

### 3. 第三阶段：机器学习方法

学习：

- 滞后特征。
- 滚动特征。
- 时间特征。
- LightGBM/XGBoost。
- 多步预测策略。
- 时间序列交叉验证。

### 4. 第四阶段：深度学习

学习：

- LSTM/GRU。
- 1D CNN。
- TCN。
- Transformer。
- 多变量序列预测。

### 5. 第五阶段：工程应用

学习：

- 数据泄漏防控。
- 业务事件建模。
- 异常检测。
- 预测区间。
- 监控和重训练。
- 误差分析。

### 6. 一句话总结

时间序列分析的核心是理解时间依赖结构，时间序列预测的核心是利用历史规律和未来可知信息，对未来做出尽可能可靠的估计。

### 7. 最重要的实践原则

做时间序列项目时，最重要的不是立刻选择复杂模型，而是：

1. 画图理解数据。
2. 建立简单 baseline。
3. 严格避免未来信息泄漏。
4. 根据业务构造特征。
5. 用时间顺序验证。
6. 做分组错误分析。
7. 持续监控线上表现。

复杂模型可以提升效果，但可靠的时间序列系统首先来自正确的数据处理、验证方式和业务理解。
