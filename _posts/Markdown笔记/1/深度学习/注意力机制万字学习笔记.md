## 一、注意力机制解决什么问题

### 1. 问题背景

注意力机制最初在深度学习中流行起来，主要是为了解决序列到序列任务中的信息瓶颈问题。典型任务包括机器翻译、文本摘要、语音识别、图像描述生成等。

在早期 Seq2Seq 模型中，编码器把整个输入序列压缩成一个固定长度向量，解码器再根据这个向量生成输出序列。例如把中文句子翻译成英文：

```text
输入：我 今天 很 开心
输出：I am very happy today
```

传统 Encoder-Decoder 模型会把整个中文句子编码成一个上下文向量，然后解码器依赖这个向量生成英文句子。问题是：如果输入句子很长，一个固定长度向量很难完整保存所有信息。

这就像让一个人读完一本书后，只允许用一句话记住全部内容，再要求他逐字复述。短文本可能还行，长文本就会严重丢失信息。

### 2. 固定向量瓶颈

传统 Seq2Seq 的核心瓶颈是：

```text
任意长度输入序列 -> 固定长度上下文向量 -> 任意长度输出序列
```

输入越长，信息压缩越严重。模型必须把所有词的信息塞进一个向量里，导致：

- 长句翻译效果下降。
- 远距离依赖难以捕捉。
- 解码时无法灵活关注输入不同部分。
- 输出每个词时都使用同一个上下文表示。

但人类翻译不是这样。人在翻译某个词时，会重点看原句中与当前词相关的部分，而不是把整句话压成一个固定记忆。

### 3. 注意力的核心思想

注意力机制的核心思想是：

> 在生成或处理某个位置的信息时，不是平均使用全部输入，而是动态选择最相关的部分，并给予更高权重。

换句话说，注意力让模型具备“根据当前需要查找相关信息”的能力。

例如翻译 “I am very happy today” 中的 “happy” 时，模型应该更多关注中文输入中的“开心”；翻译 “today” 时，应该更多关注“今天”。

注意力机制把原来的一次性压缩，变成了动态读取：

```text
旧方式：输入序列 -> 一个固定上下文向量
新方式：输入序列 -> 多个隐藏状态 -> 解码时按需加权读取
```

### 4. 注意力带来的变化

注意力机制带来了几个重要改变：

1. 解码器每一步可以使用不同上下文。
2. 长序列信息不必全部压缩进一个向量。
3. 模型可以学习输入与输出之间的软对齐关系。
4. 注意力权重提供一定可视化解释能力。
5. 自注意力进一步替代 RNN，成为 Transformer 的核心。

从历史角度看，注意力机制先是在 RNN Seq2Seq 中作为增强模块出现，后来在 Transformer 中成为主干结构，最终成为大语言模型、视觉 Transformer、多模态模型的基础组件。

## 二、从人类注意力到机器注意力

### 1. 人类注意力的直觉

人类处理信息时不会对所有输入一视同仁。比如你在嘈杂咖啡馆里和朋友聊天，虽然耳朵接收到很多声音，但大脑会重点关注朋友的声音。你看一张图片时，也不会同时处理每个像素，而是先关注主体、边缘、颜色突出的区域。

这种选择性处理有几个特点：

- 资源有限：不能同时深入处理所有信息。
- 目标驱动：当前任务决定关注内容。
- 动态变化：注意点会随时间和上下文改变。
- 权重分配：不同信息重要性不同。

机器学习中的注意力机制借鉴了这种思想，但它不是生物学意义上的注意力，而是一种可微分的信息加权机制。

### 2. 机器注意力的抽象

机器注意力可以抽象为三个步骤：

1. 计算当前查询与各个候选信息的相关性。
2. 把相关性转换成权重分布。
3. 用权重对候选信息加权求和。

用公式表示：

```text
Attention(Query, Source) = WeightedSum(Source)
```

更标准的表达是：

```text
Attention(Q, K, V) = softmax(score(Q, K)) V
```

其中：

- Q 是 Query，表示“我现在想找什么”。
- K 是 Key，表示“每个信息项可被匹配的特征”。
- V 是 Value，表示“真正要被读取的信息内容”。

### 3. 注意力不是硬选择

注意力通常是软选择。它不是只选一个词，而是给所有词分配一个概率权重。

例如：

```text
输入词：我 今天 很 开心
权重：0.05 0.10 0.15 0.70
```

这表示当前输出更关注“开心”，但其他词仍可能贡献少量信息。

软注意力的好处是可微分，可以通过反向传播训练。如果是硬选择，例如只选一个词，训练会更困难，通常需要强化学习或采样技巧。

## 三、Seq2Seq 的瓶颈与注意力的引入

### 1. 传统 Seq2Seq

传统 Seq2Seq 模型通常由编码器和解码器组成：

- 编码器读取输入序列，生成隐藏状态。
- 解码器根据编码器最终状态生成输出序列。

如果用 RNN 表示：

```text
h_t = EncoderRNN(x_t, h_{t-1})
s_t = DecoderRNN(y_{t-1}, s_{t-1}, c)
```

其中 `c` 是上下文向量，通常来自编码器最后一个隐藏状态。

问题在于 `c` 对所有解码步骤都相同。输出第一个词和最后一个词都依赖同一个压缩向量。

### 2. 加入注意力后的 Seq2Seq

注意力模型保留编码器每个时间步的隐藏状态：

```text
H = [h_1, h_2, ..., h_n]
```

解码器在生成第 `t` 个输出时，会根据当前解码状态 `s_t` 对所有编码器隐藏状态计算注意力权重：

```text
alpha_{t,i} = softmax(score(s_t, h_i))
```

再加权求和得到当前上下文：

```text
c_t = sum_i alpha_{t,i} h_i
```

于是解码器每一步都有自己的上下文 `c_t`。

```text
生成 y_1 时关注输入某些部分
生成 y_2 时关注输入另一些部分
生成 y_3 时再次调整关注重点
```

这就是早期注意力机制在机器翻译中的核心用法。

### 3. 对齐关系

在机器翻译中，注意力权重可以被理解为源语言和目标语言之间的软对齐。

例如：

```text
中文：我   今天   很   开心
英文：I   am    very happy today
```

当生成 `happy` 时，对“开心”的注意力权重应该较高；生成 `today` 时，对“今天”的权重应该较高。

这种对齐不是人工标注的，而是模型训练时自动学习出来的。

## 四、注意力机制的基本框架

### 1. 通用公式

注意力机制可以写成：

```text
Attention(Q, K, V) = softmax(score(Q, K)) V
```

这条公式是理解后续所有注意力机制的主线。

拆开看：

1. 用 Query 和 Key 计算相似度分数。
2. 对分数做 softmax，得到权重。
3. 用权重对 Value 加权求和。

如果有一个 query 和多个 key-value 对：

```text
q: 当前查询
k_i: 第 i 个键
v_i: 第 i 个值
```

则：

```text
score_i = score(q, k_i)
alpha_i = exp(score_i) / sum_j exp(score_j)
output = sum_i alpha_i v_i
```

### 2. Score 函数

常见 score 函数有：

#### 点积

```text
score(q, k) = q^T k
```

简单高效，适合矩阵并行计算。

#### 缩放点积

```text
score(q, k) = q^T k / sqrt(d_k)
```

Transformer 采用该形式，防止维度较大时点积值过大。

#### 加性注意力

```text
score(q, k) = v^T tanh(W_q q + W_k k)
```

也叫 Bahdanau Attention，通过小型神经网络计算匹配分数。

#### 双线性注意力

```text
score(q, k) = q^T W k
```

比普通点积多一个可学习矩阵。

### 3. Softmax 的作用

Softmax 把任意实数分数转换成概率分布：

```text
alpha_i = exp(score_i) / sum_j exp(score_j)
```

它有三个作用：

- 让权重非负。
- 让所有权重和为 1。
- 放大高分项，压低低分项。

注意力权重就是由 softmax 得到的。

### 4. 加权求和

最终输出是 Value 的加权和：

```text
output = alpha_1 v_1 + alpha_2 v_2 + ... + alpha_n v_n
```

如果某个 value 权重大，它对输出影响就大；权重小，影响就小。

## 五、加性注意力与乘性注意力

### 1. Bahdanau Attention

Bahdanau Attention 是早期神经机器翻译中提出的注意力机制，也称加性注意力。

它的打分函数为：

```text
e_{t,i} = v_a^T tanh(W_s s_{t-1} + W_h h_i)
```

其中：

- `s_{t-1}` 是解码器上一步隐藏状态。
- `h_i` 是编码器第 i 个隐藏状态。
- `W_s`、`W_h`、`v_a` 是可学习参数。

然后：

```text
alpha_{t,i} = softmax(e_{t,i})
c_t = sum_i alpha_{t,i} h_i
```

加性注意力的特点：

- 使用前馈网络计算相关性。
- 对 query 和 key 维度不一致时更灵活。
- 参数较多，计算相对慢。

### 2. Luong Attention

Luong Attention 常见于后续 Seq2Seq 工作中，属于乘性注意力的一类。常见形式有：

点积：

```text
score(s_t, h_i) = s_t^T h_i
```

一般形式：

```text
score(s_t, h_i) = s_t^T W h_i
```

拼接形式：

```text
score(s_t, h_i) = v^T tanh(W [s_t; h_i])
```

乘性注意力通常更快，因为可以利用矩阵乘法高效计算。

### 3. 加性与乘性的比较

| 类型 | 打分方式 | 参数量 | 计算效率 | 特点 |
| --- | --- | --- | --- | --- |
| 加性注意力 | MLP | 较多 | 较慢 | 灵活，适合维度不同 |
| 点积注意力 | 向量内积 | 少 | 快 | 简单高效 |
| 缩放点积注意力 | 点积后缩放 | 少 | 快 | Transformer 标准形式 |
| 双线性注意力 | q^T W k | 中等 | 中等 | 表达能力比点积强 |

在 Transformer 中，主流选择是缩放点积注意力，因为它非常适合 GPU 并行计算。

## 六、Query、Key、Value 的核心理解

### 1. QKV 的直觉类比

可以把注意力机制类比成在数据库中查询信息：

- Query：查询条件。
- Key：每条记录的索引字段。
- Value：每条记录真正包含的内容。

查询时，系统用 Query 和每条记录的 Key 计算匹配程度，然后根据匹配程度读取对应 Value。

### 2. 为什么要分 Key 和 Value

很多初学者会问：为什么要有 Key 和 Value？为什么不直接用输入向量？

原因是匹配信息和输出信息可以不是同一套表示。

Key 用于回答：

```text
这个位置和当前 query 有多相关？
```

Value 用于回答：

```text
如果它相关，我应该从这个位置读取什么内容？
```

就像一本书的目录和正文。目录用于检索，正文用于获取内容。目录不是正文，但它帮助你找到正文。

### 3. QKV 在 Self-Attention 中的来源

在自注意力中，Q、K、V 都来自同一个输入序列 X，但通过不同线性变换得到：

```text
Q = X W_Q
K = X W_K
V = X W_V
```

其中 `W_Q`、`W_K`、`W_V` 是可学习参数。

这意味着每个 token 会被投影成三种角色：

- 作为 Query：它想从其他 token 获取什么。
- 作为 Key：它能被其他 token 如何匹配。
- 作为 Value：它被关注后贡献什么信息。

### 4. 一个词如何关注其他词

句子：

```text
The animal didn't cross the street because it was tired.
```

模型处理 `it` 时，需要判断 `it` 指代什么。Self-Attention 允许 `it` 这个位置把自己的 Query 与其他位置的 Key 比较，可能发现 `animal` 的相关性高，于是从 `animal` 的 Value 中读取信息。

这就是注意力处理长距离依赖的优势。

## 七、Self-Attention 自注意力机制

### 1. 什么是自注意力

Self-Attention 指的是 Query、Key、Value 都来自同一个序列。它让序列中的每个位置都能关注同一序列中的其他位置。

给定输入：

```text
X = [x_1, x_2, ..., x_n]
```

每个位置都会生成自己的输出：

```text
z_i = Attention(q_i, K, V)
```

也就是说，第 i 个 token 的新表示，是它对整个序列所有 token 加权聚合后的结果。

### 2. 自注意力与 RNN 的区别

RNN 按时间顺序处理序列：

```text
x_1 -> x_2 -> x_3 -> ... -> x_n
```

它的缺点是：

- 难以并行。
- 长距离依赖容易衰减。
- 训练速度较慢。

Self-Attention 一次性计算所有 token 两两关系：

```text
每个 token 都可以直接看见其他 token
```

优势：

- 并行计算友好。
- 任意两个位置路径长度为 1。
- 更容易捕捉长距离依赖。
- 可通过多头注意力学习不同关系。

### 3. 自注意力的矩阵形式

输入矩阵：

```text
X: [n, d_model]
```

线性投影：

```text
Q = X W_Q
K = X W_K
V = X W_V
```

形状：

```text
Q: [n, d_k]
K: [n, d_k]
V: [n, d_v]
```

计算注意力分数：

```text
Scores = Q K^T
```

形状：

```text
[n, d_k] @ [d_k, n] = [n, n]
```

`Scores[i, j]` 表示第 i 个 token 对第 j 个 token 的关注分数。

归一化：

```text
A = softmax(Scores / sqrt(d_k))
```

输出：

```text
Z = A V
```

形状：

```text
[n, n] @ [n, d_v] = [n, d_v]
```

### 4. 自注意力的本质

Self-Attention 本质上是在为每个 token 构造一个上下文相关的新表示。原始 token 表示只包含自身信息，经过自注意力后，每个 token 表示都融合了全局上下文。

例如：

```text
bank
```

在不同句子中含义不同：

```text
I deposited money in the bank.
I sat on the bank of the river.
```

Self-Attention 可以让 `bank` 关注 `money` 或 `river`，从而形成不同语义表示。

## 八、Scaled Dot-Product Attention 精讲

### 1. 标准公式

Transformer 中最核心的注意力形式是缩放点积注意力：

```text
Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) V
```

其中：

- `Q` 是 Query 矩阵。
- `K` 是 Key 矩阵。
- `V` 是 Value 矩阵。
- `d_k` 是 Key 的维度。

### 2. 为什么要除以 sqrt(d_k)

如果 `q` 和 `k` 的每个维度都是均值 0、方差 1 的随机变量，那么点积：

```text
q^T k = sum_{i=1}^{d_k} q_i k_i
```

其方差会随 `d_k` 增大而增大。维度越高，点积值越容易变得很大或很小。

如果直接把很大的分数送进 softmax，会导致 softmax 输出非常尖锐：

```text
[10, 1, 0] -> softmax 后第一个接近 1
[100, 1, 0] -> softmax 后几乎完全是第一个
```

过于尖锐会导致梯度变小，训练不稳定。

因此用 `sqrt(d_k)` 缩放，让分数保持在更合理范围。

### 3. Softmax 的维度

在注意力中，softmax 通常沿 Key 的序列维度做归一化。

如果分数矩阵形状是：

```text
[batch, heads, query_len, key_len]
```

则 softmax 作用在 `key_len` 维度：

```text
每个 query 对所有 key 的权重和为 1
```

这是理解代码实现时非常重要的细节。

### 4. 输出含义

注意力输出的每一行对应一个 query 的上下文表示。它是所有 value 的加权和。

例如：

```text
output_i = sum_j attention_weight_{i,j} value_j
```

这表示第 i 个位置根据自己和其他位置的相关性，聚合出新的表示。

## 九、Multi-Head Attention 多头注意力

### 1. 为什么需要多头

单个注意力头只能在一个表示空间中计算关系。语言中的关系很多：

- 主谓关系。
- 指代关系。
- 修饰关系。
- 位置关系。
- 语义相似。
- 句法依赖。

多头注意力允许模型在多个子空间中并行学习不同关系。

### 2. 多头注意力公式

多头注意力定义为：

```text
MultiHead(Q, K, V) = Concat(head_1, ..., head_h) W_O
```

其中：

```text
head_i = Attention(Q W_i^Q, K W_i^K, V W_i^V)
```

每个头都有自己的投影矩阵。

### 3. 维度变化

假设：

```text
d_model = 512
heads = 8
d_k = d_v = 64
```

每个头在 64 维空间中计算注意力。8 个头拼接后：

```text
8 * 64 = 512
```

再经过输出矩阵 `W_O` 映射回 `d_model`。

### 4. 多头不是简单重复

虽然每个头结构相同，但参数不同，因此可以学到不同模式。一个头可能关注前一个词，一个头可能关注句子主语，一个头可能关注标点或特殊 token，一个头可能关注长距离语义。

实际模型中并不是每个头都同样重要。有研究发现某些注意力头可以被剪枝而性能影响很小，但多头整体仍然是 Transformer 表达能力的重要来源。

### 5. 多头注意力的优势

- 提升表达能力。
- 允许模型关注不同关系。
- 增强训练稳定性。
- 保持并行计算能力。
- 与残差连接、层归一化配合效果好。

## 十、Mask 机制：Padding Mask 与 Causal Mask

### 1. 为什么需要 Mask

注意力会对所有 key 计算权重，但有些位置不应该被关注。例如：

- Padding token 是补齐长度用的，不含真实语义。
- 自回归生成时，当前位置不能看到未来 token。

Mask 的作用就是在 softmax 前把不允许关注的位置屏蔽掉。

### 2. Padding Mask

批处理时，不同句子长度不同。为了组成矩阵，需要补齐：

```text
句子 A: I love NLP
句子 B: I love deep learning very much
```

补齐后：

```text
I love NLP [PAD] [PAD] [PAD]
I love deep learning very much
```

`[PAD]` 不应该参与注意力。因此需要 Padding Mask。

做法是在注意力分数中把 PAD 对应位置加上一个极小值：

```text
score = score.masked_fill(mask == 0, -inf)
```

softmax 后这些位置权重接近 0。

### 3. Causal Mask

在语言模型生成中，模型预测第 t 个 token 时，只能看到前面的 token，不能看到未来答案。

例如训练句子：

```text
我 喜欢 学习 深度 学习
```

预测“学习”时不能看到后面的“深度 学习”。

Causal Mask 通常是一个上三角矩阵：

```text
允许关注：
位置1: 1 0 0 0
位置2: 1 1 0 0
位置3: 1 1 1 0
位置4: 1 1 1 1
```

这样每个位置只能关注自己和之前的位置。

### 4. Encoder 与 Decoder 的 Mask 差异

Transformer Encoder：

- 通常使用 Padding Mask。
- 每个 token 可以看见整个输入句子。

Transformer Decoder：

- Self-Attention 使用 Causal Mask。
- 如果有 padding，也要结合 Padding Mask。
- Cross-Attention 对 Encoder 输出使用 Padding Mask。

大语言模型通常是 Decoder-only 架构，核心就是带 Causal Mask 的自注意力。

## 十一、位置编码：为什么注意力需要位置信息

### 1. 自注意力本身不包含顺序

Self-Attention 计算的是 token 两两相似度。如果没有位置信息，模型无法区分：

```text
我 爱 你
你 爱 我
```

这两个句子包含同样的词，但顺序不同，意思完全不同。

RNN 天然按顺序处理，因此有顺序信息；CNN 通过卷积窗口也有局部位置信息；Self-Attention 并行处理所有 token，本身不关心顺序。

因此 Transformer 必须引入位置编码。

### 2. 正弦余弦位置编码

原始 Transformer 使用固定正弦余弦位置编码：

```text
PE(pos, 2i) = sin(pos / 10000^{2i / d_model})
PE(pos, 2i+1) = cos(pos / 10000^{2i / d_model})
```

其中：

- `pos` 是位置。
- `i` 是维度索引。
- `d_model` 是模型维度。

这种编码有几个特点：

- 不需要训练。
- 不同位置有不同向量。
- 可以表达相对位移信息。
- 理论上可外推到训练长度之外。

### 3. 可学习位置编码

许多模型使用可学习位置编码。做法是给每个位置一个可训练向量：

```text
x_i = token_embedding_i + position_embedding_i
```

优点：

- 简单。
- 适合固定最大长度。
- 表达能力强。

缺点：

- 对超过训练长度的位置外推较弱。

### 4. 相对位置编码

绝对位置编码告诉模型“这个 token 在第几个位置”。相对位置编码告诉模型“两个 token 相距多远”。

相对位置在很多语言任务中更自然，因为语法关系往往依赖相对距离。例如一个形容词通常修饰附近名词。

现代大模型常使用更复杂的位置方法，如 RoPE、ALiBi 等。它们主要目标是更好地处理长上下文和相对位置关系。

### 5. RoPE 简要理解

RoPE 是旋转位置编码，常用于现代大语言模型。它通过对 Q 和 K 向量施加与位置相关的旋转，使点积注意力中自然包含相对位置信息。

直觉上，RoPE 不是简单把位置向量加到 token embedding 上，而是在注意力计算前把位置信息注入 Q/K 的几何关系中。

它的优势包括：

- 更适合自回归语言模型。
- 对相对位置建模友好。
- 在长上下文扩展中常被使用。

## 十二、Transformer 中的注意力结构

### 1. Transformer Encoder

Encoder 层通常包括：

1. Multi-Head Self-Attention。
2. Add & Norm。
3. Feed Forward Network。
4. Add & Norm。

输入序列经过多层 Encoder 后，每个 token 的表示都融合了全局上下文。

Encoder 常用于：

- 文本分类。
- 序列标注。
- 语义匹配。
- 表示学习。

BERT 是典型 Encoder-only 模型。

### 2. Transformer Decoder

原始 Transformer Decoder 层包括：

1. Masked Multi-Head Self-Attention。
2. Add & Norm。
3. Cross-Attention。
4. Add & Norm。
5. Feed Forward Network。
6. Add & Norm。

其中：

- Masked Self-Attention 关注已生成 token。
- Cross-Attention 关注 Encoder 输出。
- FFN 对每个位置做非线性变换。

机器翻译中的 Decoder 会一边看已生成目标词，一边通过 Cross-Attention 看源语言表示。

### 3. Decoder-only 架构

大语言模型常用 Decoder-only 架构。它没有 Encoder，也没有 Encoder-Decoder Cross-Attention，主要由多层带 Causal Mask 的 Self-Attention 和 FFN 组成。

训练目标通常是预测下一个 token：

```text
P(x_t | x_1, x_2, ..., x_{t-1})
```

这类模型适合文本生成、对话、代码生成和多任务指令跟随。

### 4. Cross-Attention

Cross-Attention 指 Query 来自一个序列，Key 和 Value 来自另一个序列。

在 Encoder-Decoder Transformer 中：

```text
Q = Decoder hidden states
K = Encoder hidden states
V = Encoder hidden states
```

解码器用当前目标端状态作为 Query，去源端表示中查找相关信息。

Cross-Attention 常用于：

- 机器翻译。
- 图像描述生成。
- 文本到图像生成。
- 多模态模型。

## 十三、注意力机制的计算复杂度

### 1. 标准自注意力复杂度

对于长度为 `n` 的序列，Self-Attention 需要计算 `QK^T`：

```text
[n, d] @ [d, n] = [n, n]
```

时间复杂度大约是：

```text
O(n^2 d)
```

空间复杂度主要来自注意力矩阵：

```text
O(n^2)
```

这意味着序列长度翻倍，注意力矩阵大小会变成四倍。

### 2. 为什么长上下文困难

如果序列长度是：

```text
n = 2048
```

注意力矩阵有：

```text
2048 * 2048 = 4,194,304
```

如果 `n = 32768`：

```text
32768 * 32768 ≈ 1,073,741,824
```

仅注意力权重就非常庞大。因此长上下文模型需要优化注意力计算。

### 3. 常见优化方向

#### 稀疏注意力

只关注部分位置，例如局部窗口、固定间隔、全局 token。

#### 线性注意力

通过核技巧或公式变换，把复杂度从二次降低到近似线性。

#### FlashAttention

通过优化 GPU 内存访问和分块计算，在数学结果等价或近似等价的情况下显著提升效率，减少显存占用。

#### 滑动窗口注意力

每个 token 只关注附近窗口内的 token，适合长文本局部依赖。

#### 分组查询注意力

减少 KV 头数量，降低推理时 KV Cache 的显存占用。

### 4. KV Cache

自回归生成时，如果每生成一个 token 都重新计算所有历史 token 的 K 和 V，会非常浪费。

KV Cache 的做法是缓存历史 token 的 Key 和 Value。生成新 token 时，只计算新 token 的 Q/K/V，然后用新 Query 去关注缓存中的历史 K/V。

优点：

- 大幅提升推理速度。
- 避免重复计算历史上下文。

代价：

- 需要额外显存保存 KV。
- 上下文越长，缓存越大。

## 十四、PyTorch 代码实现

### 1. 单头缩放点积注意力

```python
import math
import torch
import torch.nn.functional as F


def scaled_dot_product_attention(q, k, v, mask=None):
    """
    q: [batch, query_len, d_k]
    k: [batch, key_len, d_k]
    v: [batch, key_len, d_v]
    mask: 可广播到 [batch, query_len, key_len]，1 表示可见，0 表示屏蔽
    """
    d_k = q.size(-1)
    scores = torch.matmul(q, k.transpose(-2, -1)) / math.sqrt(d_k)

    if mask is not None:
        scores = scores.masked_fill(mask == 0, float("-inf"))

    attn = F.softmax(scores, dim=-1)
    output = torch.matmul(attn, v)
    return output, attn
```

关键点：

- `k.transpose(-2, -1)` 用于得到 `K^T`。
- 除以 `sqrt(d_k)` 是缩放。
- mask 要在 softmax 之前加。
- softmax 的维度是最后一维，即 key 维度。

### 2. 多头注意力简化实现

```python
import math
import torch
import torch.nn as nn
import torch.nn.functional as F


class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, num_heads, dropout=0.0):
        super().__init__()
        assert d_model % num_heads == 0

        self.d_model = d_model
        self.num_heads = num_heads
        self.d_head = d_model // num_heads

        self.w_q = nn.Linear(d_model, d_model)
        self.w_k = nn.Linear(d_model, d_model)
        self.w_v = nn.Linear(d_model, d_model)
        self.w_o = nn.Linear(d_model, d_model)
        self.dropout = nn.Dropout(dropout)

    def split_heads(self, x):
        # x: [batch, seq_len, d_model]
        batch_size, seq_len, _ = x.size()
        x = x.view(batch_size, seq_len, self.num_heads, self.d_head)
        return x.transpose(1, 2)
        # [batch, num_heads, seq_len, d_head]

    def combine_heads(self, x):
        # x: [batch, num_heads, seq_len, d_head]
        batch_size, _, seq_len, _ = x.size()
        x = x.transpose(1, 2).contiguous()
        return x.view(batch_size, seq_len, self.d_model)

    def forward(self, query, key, value, mask=None):
        q = self.split_heads(self.w_q(query))
        k = self.split_heads(self.w_k(key))
        v = self.split_heads(self.w_v(value))

        scores = torch.matmul(q, k.transpose(-2, -1)) / math.sqrt(self.d_head)

        if mask is not None:
            # mask 需要能广播到 [batch, heads, query_len, key_len]
            scores = scores.masked_fill(mask == 0, float("-inf"))

        attn = F.softmax(scores, dim=-1)
        attn = self.dropout(attn)

        context = torch.matmul(attn, v)
        context = self.combine_heads(context)
        output = self.w_o(context)

        return output, attn
```

### 3. Causal Mask 实现

```python
def causal_mask(seq_len, device=None):
    mask = torch.tril(torch.ones(seq_len, seq_len, device=device))
    return mask.bool()


mask = causal_mask(4)
print(mask)
```

输出形状类似：

```text
1 0 0 0
1 1 0 0
1 1 1 0
1 1 1 1
```

实际使用时通常 reshape：

```python
mask = mask.unsqueeze(0).unsqueeze(0)
# [1, 1, seq_len, seq_len]
```

### 4. Padding Mask 实现

```python
def padding_mask(input_ids, pad_id=0):
    # input_ids: [batch, seq_len]
    mask = (input_ids != pad_id)
    return mask.unsqueeze(1).unsqueeze(2)
    # [batch, 1, 1, seq_len]
```

它可以广播到：

```text
[batch, heads, query_len, key_len]
```

### 5. 实现时最容易错的地方

- softmax 维度写错。
- mask 形状无法广播。
- mask 放在 softmax 之后。
- 忘记除以 `sqrt(d_k)`。
- 多头拆分后没有 `contiguous()` 就 view。
- 把 query_len 和 key_len 混淆。
- causal mask 方向写反。

## 十五、注意力权重如何解释

### 1. 注意力权重可以提供线索

注意力权重能告诉我们：对某个 query，模型更关注哪些 key。它可以用于观察模型是否学到了合理关联。

例如翻译任务中，可以画出源词和目标词的注意力热力图，观察 “today” 是否关注“今天”。

### 2. 注意力不等于完整解释

需要注意：注意力权重不一定等价于模型决策的真实因果解释。原因包括：

- 后续层会继续变换表示。
- 多头之间关系复杂。
- 残差连接会绕过注意力。
- FFN 层也贡献大量计算。
- 高注意力权重不一定代表最终输出最依赖它。

因此，注意力可视化可以辅助理解，但不能简单说“模型看这里，所以一定是因为这里才做出判断”。

### 3. 多层多头可视化

Transformer 有多层多头。低层注意力可能偏向局部、位置、标点；高层可能偏向语义、任务相关信息。不同头的可解释性差异很大。

观察注意力时最好结合：

- 层数。
- 头编号。
- 输入 token。
- 任务类型。
- 梯度或消融实验。

## 十六、常见注意力变体

### 1. Cross-Attention

Query 来自目标序列，Key/Value 来自源序列。常用于 Encoder-Decoder 和多模态任务。

### 2. Local Attention

每个位置只关注附近窗口。适合长文本和图像局部建模。

优点是降低复杂度，缺点是远距离依赖需要多层累积或特殊全局机制。

### 3. Sparse Attention

通过稀疏模式减少注意力连接，例如只关注局部、跨步或少量全局 token。

适合长序列任务。

### 4. Linear Attention

通过核函数或矩阵结合律把注意力复杂度近似降为线性。它通常牺牲部分表达或改变 softmax 注意力形式。

### 5. Multi-Query Attention

多个 Query 头共享同一组 Key 和 Value 头。主要用于减少推理时 KV Cache。

### 6. Grouped-Query Attention

介于 Multi-Head Attention 和 Multi-Query Attention 之间。多个 Query 头分组共享 Key/Value。它在效果和推理效率之间折中。

### 7. FlashAttention

FlashAttention 更像高效实现方法，而不是新的注意力定义。它通过分块计算和 IO 优化减少显存读写，提升训练和推理效率。

### 8. Additive Attention

通过小型神经网络计算相关性，常见于早期 Seq2Seq。

### 9. Channel Attention 与 Spatial Attention

在计算机视觉中，注意力也可以作用在通道或空间维度：

- Channel Attention：关注哪些特征通道重要。
- Spatial Attention：关注图像哪些空间位置重要。

这类注意力和 Transformer 注意力形式不完全一样，但思想相似。

## 十七、注意力机制在不同任务中的应用

### 1. 机器翻译

注意力最早大放异彩的任务之一是机器翻译。它帮助模型学习源语言和目标语言之间的对齐关系，改善长句翻译。

### 2. 文本摘要

生成摘要时，模型需要从原文中选择关键信息。注意力机制可以让解码器在生成每个摘要词时关注原文不同部分。

### 3. 问答系统

问题作为 Query，文档或上下文作为 Key/Value，模型通过注意力找到与问题相关的证据。

### 4. 文本分类

虽然分类任务不一定必须使用注意力，但注意力可以帮助模型关注对分类最有用的词或句子。

### 5. 语音识别

注意力可以对齐声学帧和输出文字，尤其适用于端到端语音识别。

### 6. 图像任务

Vision Transformer 把图像切成 patch，把 patch 当作 token，通过自注意力建模图像区域之间的关系。

### 7. 多模态任务

文本、图像、音频之间可以通过 Cross-Attention 交互。例如文本到图像生成中，图像特征会关注文本提示词。

### 8. 推荐系统

注意力可以建模用户历史行为中哪些项目对当前推荐更重要。

## 十八、常见误区

### 1. 误区：注意力机制就是 Transformer

注意力机制早于 Transformer。Transformer 是一种大量使用注意力机制的网络架构，但注意力本身可以用于 RNN、CNN、多模态模型等。

### 2. 误区：注意力权重就是解释

注意力权重有参考价值，但不能完全等价于因果解释。

### 3. 误区：Self-Attention 不需要位置编码

Self-Attention 本身不包含顺序信息，因此必须显式或隐式加入位置机制。

### 4. 误区：多头越多越好

头数增加会改变每个头的维度和计算结构。过多头不一定提升性能，甚至可能造成冗余。

### 5. 误区：Mask 可以放在 softmax 后

Mask 应该在 softmax 前作用于分数。如果 softmax 后再置零，权重和不再为 1，语义会变。

### 6. 误区：Q、K、V 必须来自不同输入

Self-Attention 中 Q、K、V 来自同一输入，只是经过不同线性投影。Cross-Attention 中它们才通常来自不同序列。

### 7. 误区：注意力一定比 RNN 更适合所有任务

注意力擅长并行和长距离依赖，但标准注意力复杂度高。在某些低资源、短序列、在线流式场景中，RNN、CNN 或混合架构仍有价值。

## 十九、面试高频问题

### 1. 什么是注意力机制？

注意力机制是一种根据 Query 与 Key 的相关性，对 Value 进行加权聚合的方法。它让模型在处理当前位置时动态关注输入中更相关的部分。

标准公式：

```text
Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) V
```

### 2. 为什么 Transformer 要除以 sqrt(d_k)？

因为高维向量点积的方差会随维度增大而增大，导致 softmax 输入过大，输出过于尖锐，梯度变小。除以 `sqrt(d_k)` 可以稳定分数范围，改善训练。

### 3. Self-Attention 和 Cross-Attention 有什么区别？

Self-Attention 中 Q、K、V 来自同一个序列，用于序列内部信息交互。Cross-Attention 中 Q 来自一个序列，K/V 来自另一个序列，用于两个序列之间交互。

### 4. Multi-Head Attention 为什么有效？

多头注意力让模型在多个子空间中并行学习不同关系，比如句法、语义、位置、指代等。多个头的结果拼接后再线性变换，增强表达能力。

### 5. Padding Mask 和 Causal Mask 的区别？

Padding Mask 屏蔽补齐位置，防止模型关注无意义的 PAD token。Causal Mask 屏蔽未来位置，保证自回归生成时当前位置只能看到过去和当前。

### 6. 为什么 Self-Attention 需要位置编码？

Self-Attention 对输入 token 集合做两两关系计算，本身不感知顺序。如果不加入位置编码，模型无法区分词序不同但词集合相同的句子。

### 7. 注意力机制复杂度是多少？

标准 Self-Attention 时间复杂度约为 `O(n^2 d)`，空间复杂度约为 `O(n^2)`，其中 `n` 是序列长度，`d` 是隐藏维度。

### 8. KV Cache 是什么？

KV Cache 是自回归生成时缓存历史 token 的 Key 和 Value，避免每一步重复计算历史部分。它能提升推理速度，但会增加显存占用。

### 9. 注意力和全连接层有什么区别？

全连接层通常对固定维度输入做固定参数变换；注意力的权重由输入动态计算，同一模型在不同上下文中会产生不同的信息聚合方式。

### 10. 注意力机制为什么适合并行？

Self-Attention 可以同时计算所有 token 的 Q、K、V 和注意力矩阵，不需要像 RNN 那样严格按时间步递归处理，因此更适合 GPU 并行。

### 2. 一句话总结注意力机制

注意力机制是一种让模型根据当前需求动态检索上下文信息的机制，其核心是用 Query 和 Key 计算相关性，再用相关性权重对 Value 加权求和。

### 3. 最核心公式

```text
Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) V
```

理解这条公式，就抓住了现代 Transformer、大语言模型和大量多模态模型的关键基础。

### 4. 最重要的直觉

注意力不是让模型“记住一切”，而是让模型“在需要时找到相关信息”。在自然语言中，一个词的含义依赖上下文；在图像中，一个区域的意义依赖其他区域；在多模态任务中，文本和图像需要互相对齐。注意力机制提供了统一的可学习信息交互方式。

### 5. 最容易被忽略的细节

学习注意力机制时，不要只背公式。真正需要掌握的是：

- Q、K、V 的来源。
- 注意力矩阵的形状。
- softmax 的维度。
- mask 的方向和位置。
- 多头拆分与拼接。
- 位置编码的必要性。
- 自回归生成中的 KV Cache。
- 标准注意力的复杂度瓶颈。

这些细节决定你是否真正理解注意力机制，也决定你能否读懂 Transformer 源码、调试模型维度错误、解释大模型推理过程。

## 附录：最小 Transformer Block 参考代码

```python
import torch
import torch.nn as nn


class TransformerBlock(nn.Module):
    def __init__(self, d_model, num_heads, d_ff, dropout=0.1):
        super().__init__()
        self.attn = MultiHeadAttention(d_model, num_heads, dropout)
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        self.ffn = nn.Sequential(
            nn.Linear(d_model, d_ff),
            nn.GELU(),
            nn.Dropout(dropout),
            nn.Linear(d_ff, d_model),
        )
        self.dropout = nn.Dropout(dropout)

    def forward(self, x, mask=None):
        attn_out, attn_weights = self.attn(x, x, x, mask)
        x = self.norm1(x + self.dropout(attn_out))

        ffn_out = self.ffn(x)
        x = self.norm2(x + self.dropout(ffn_out))

        return x, attn_weights
```

这段代码展示了 Transformer Block 的最小结构：

- 自注意力负责 token 间信息交互。
- 残差连接保留原始信息并改善梯度传播。
- LayerNorm 稳定训练。
- FFN 对每个位置进行非线性变换。

真实工业模型会更复杂，例如使用 Pre-LN、RoPE、RMSNorm、SwiGLU、FlashAttention、KV Cache、张量并行等。但无论如何演化，注意力的核心思想仍然是：

```text
根据相关性分配权重，根据权重聚合信息。
```

掌握这个核心，再理解现代大模型中的各种工程优化，就会容易很多。

第二个误区是只关注工具，不关注模型。工具会变化，模型更稳定。无论工具名称如何变化，底层仍然要解决输入建模、状态管理、资源调度、错误恢复、性能约束和质量验证这些问题。学习 $topic 时，应该把工具用法和底层模型分开记录：工具命令解决“怎么做”，底层模型解释“为什么这样做”。

第三个误区是没有验证意识。很多笔记写得很完整，但没有说明如何确认自己做对了。对于 $category 相关主题，验证至少应包含最小样例、边界样例、异常样例和性能样例。最小样例证明流程跑通，边界样例证明理解完整，异常样例证明系统可恢复，性能样例证明方案在目标规模下仍然可用。

第四个误区是忽略可维护性。短期学习时，能跑通就容易产生掌握的错觉；长期使用时，命名、分层、注释、测试、日志、版本管理和文档才会决定知识能否转化为稳定能力。扩充 $topic 笔记时，应把“如何写得清楚、如何排查、如何交接、如何复盘”也纳入内容。

#### 学习与实践建议

建议围绕 $topic 做一个小型闭环练习：先用自己的话解释概念，再画出流程，再实现一个最小案例，然后主动制造一个错误并排查，最后写下复盘。这个过程看起来比直接读资料慢，但能显著提高迁移能力。很多人学完后不会用，根本原因是缺少“从概念到问题再到验证”的闭环。

### 精讲扩展 3：注意力机制万字学习笔记 的模型假设、损失函数 与工程化理解

学习 $topic 时，不能只把它当成一个孤立知识点来背诵，而要把它放到 $category 的完整问题链条里理解。一个知识点通常同时包含概念定义、适用边界、输入输出、运行过程、常见异常和工程取舍。真正掌握它，意味着看到一个具体场景时，能够判断它解决什么问题、依赖哪些前提、失败时会出现什么现象，以及应该用什么手段验证自己的判断。

从 $a 的角度看，最重要的是先建立清晰的对象模型。也就是明确系统里有哪些参与者、它们之间如何连接、数据或控制信号如何流动、哪些环节是同步的、哪些环节是异步的、哪些状态是临时状态、哪些状态需要长期保存。很多初学问题并不是公式不会、API 不熟，而是对象边界不清：把配置当成状态，把结果当成过程，把局部现象当成全局规律。写笔记时建议始终追问：这个概念的主体是谁，输入是什么，输出是什么，中间约束是什么，错误会在哪里暴露。

从 $b 的角度看，流程比单点知识更关键。一个成熟方案通常不是单个技巧，而是一组步骤：先确定目标，再拆分约束，然后选择工具，最后通过测试和复盘确认效果。比如在实际项目中，不能只问“怎么实现”，还要问“为什么要这样实现”“有没有更简单的替代方案”“边界条件是什么”“数据量、并发量、实时性、可靠性变化后还能不能工作”。这种流程意识能够避免把学习停留在教程层面，也能让后续排错有明确路线。

$topic 的 $c 往往决定它在真实项目中的稳定性。理论上可行的方案，到了工程环境中会受到数据质量、硬件条件、依赖版本、网络环境、团队协作、部署方式和维护成本影响。写代码或做设计时，应该把正常路径和异常路径同时考虑：正常情况下如何运行，输入为空怎么办，超时怎么办，重复执行怎么办，部分成功怎么办，版本升级后兼容性怎么办，日志和指标如何证明系统确实按预期工作。

进一步看 $d，它通常对应性能、可靠性或可维护性的核心矛盾。很多技术选择并没有绝对正确答案，只有是否适合当前约束。例如追求极致性能可能牺牲可读性，追求高度抽象可能增加调试成本，追求快速交付可能留下技术债，追求完全通用可能让简单场景变复杂。高质量笔记应该把这些取舍写出来，而不是只给一个“推荐方案”。推荐方案背后的条件越清楚，迁移到新场景时越不容易误用。

最后从 $e 的角度进行复盘，可以把知识从“看懂”推进到“会用”。建议为 $topic 建立三个层次的检查：第一层是概念检查，确认术语、流程和边界没有混淆；第二层是实践检查，确认能够独立完成一个最小案例；第三层是工程检查，确认这个案例在异常、规模、性能和维护方面经得起追问。每次学习完一个章节，都可以用这三层检查反向补齐笔记。

#### 典型场景拆解

在真实场景中，$topic 通常会经历“需求出现、方案选择、实现落地、问题暴露、持续优化”几个阶段。需求出现时，要先判断这个需求属于基础能力、性能优化、体验改进、可靠性建设还是长期架构演进。不同类型的需求对方案的评价标准不同：基础能力看正确性，性能优化看指标，体验改进看路径是否顺滑，可靠性建设看故障时能否降级和恢复，架构演进看未来变化是否容易吸收。

方案选择阶段，最容易犯的错误是直接套用熟悉工具。更稳妥的方式是列出约束：数据规模、时延要求、资源预算、团队熟悉度、运维能力、安全要求、可测试性和长期维护成本。只有把约束列清楚，才能解释为什么选择当前方案。否则方案看似高级，实际可能只是增加了复杂度。

实现落地阶段，要把 $a 和 $b 拆成可验证的小步骤。每一步都应该有明确的输入、输出和检查方式。对学习笔记而言，这意味着不能只有大段概念，还应该补充流程图式的文字描述、伪代码、命令示例、参数解释、错误现象和排查路径。这样以后复习时，笔记不仅能帮助理解，也能直接指导实践。

问题暴露阶段，要优先区分“理解错误、实现错误、环境错误、数据错误、依赖错误、边界条件错误”。很多复杂问题之所以难排，是因为一开始就把问题归因到错误层级。例如把配置问题当成算法问题，把权限问题当成代码问题，把数据分布变化当成模型失效，把硬件噪声当成软件逻辑错误。好的排查顺序应该从可观测事实开始，而不是从猜测开始。

持续优化阶段，不应只追求把当前问题压下去，还要沉淀成规则。比如记录触发条件、影响范围、定位方法、最终修复、预防措施和可监控指标。这样下一次出现类似问题时，团队可以复用经验，而不是重新从零排查。
