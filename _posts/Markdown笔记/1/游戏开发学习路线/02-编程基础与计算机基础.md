# 02-编程基础与计算机基础

## 1. 为什么游戏开发必须学编程

游戏是大量规则和状态变化组成的交互系统。编程负责把规则变成可运行的行为：

- 玩家输入。
- 角色移动。
- 碰撞检测。
- 敌人 AI。
- 技能和伤害。
- UI 状态。
- 存档读写。
- 关卡事件。
- 网络通信。

即使使用可视化脚本，也需要理解编程思维。

## 2. 推荐语言

| 语言 | 常见用途 |
| --- | --- |
| C# | Unity 主力语言 |
| C++ | Unreal、引擎底层、高性能模块 |
| GDScript | Godot 常用脚本 |
| Lua | 热更新、配置脚本、部分商业项目 |
| Python | 工具脚本、资源处理、自动化 |
| JavaScript/TypeScript | Web 游戏、工具、小游戏 |

初学者建议：

- 学 Unity：先学 C#。
- 学 Unreal：先蓝图，再逐步学 C++。
- 学 Godot：先学 GDScript。
- 做 Web 小游戏：学 JavaScript 或 TypeScript。

## 3. 编程基础

必须掌握：

- 变量。
- 条件判断。
- 循环。
- 函数。
- 数组和列表。
- 字典/映射。
- 类和对象。
- 继承和组合。
- 接口或抽象。
- 事件和回调。
- 调试。

游戏例子：

```csharp
public class Player
{
    public int hp = 100;

    public void TakeDamage(int damage)
    {
        hp -= damage;
        if (hp <= 0)
        {
            Die();
        }
    }

    private void Die()
    {
        // 玩家死亡逻辑
    }
}
```

## 4. 数据结构

常用数据结构：

- **数组/List**：存放敌人、道具、关卡列表。
- **Dictionary/Map**：通过 ID 查找配置、物品、角色。
- **Queue**：任务队列、消息队列。
- **Stack**：撤销、状态回退。
- **Set**：去重，如已收集物品。
- **Tree**：行为树、技能树、UI 层级。
- **Graph**：地图寻路、节点关系。

## 5. 算法基础

游戏中常见算法：

- 排序。
- 搜索。
- 路径查找 A*。
- 随机权重。
- 碰撞检测。
- 插值。
- 状态机。
- 行为树。
- 简单概率和期望。

不需要一开始学成算法竞赛水平，但要能理解常见问题如何建模。

## 6. 数学基础

游戏开发常用数学：

- 向量。
- 坐标系。
- 三角函数。
- 矩阵。
- 四元数。
- 插值。
- 概率。
- 曲线。

2D 游戏重点：

- 坐标。
- 向量方向。
- 距离。
- 速度。
- 碰撞区域。

3D 游戏还要理解：

- 摄像机。
- 法线。
- 光照。
- 旋转。
- 空间变换。

## 7. 调试能力

必须学会：

- 看报错。
- 打断点。
- 单步执行。
- 查看变量。
- 打日志。
- 复现 Bug。
- 缩小问题范围。
- 读调用栈。

调试习惯：

- 不要盲改。
- 先复现，再定位。
- 一次只改一个变量。
- 记录原因和修复方式。

## 8. 版本控制

游戏项目必须学习 Git。

常用操作：

- `git status`
- `git add`
- `git commit`
- `git branch`
- `git checkout`
- `git merge`
- `git pull`
- `git push`

注意：

- Unity/Unreal 项目要正确配置 `.gitignore`。
- 大型美术资源可能需要 Git LFS。
- 提交信息要说明修改内容。
- 不要把临时缓存和构建产物提交进去。
