# 04-游戏程序核心：玩法、物理、AI、存档和工具

## 1. 玩家输入

输入是玩家和游戏交互的起点。

需要处理：

- 键盘。
- 鼠标。
- 手柄。
- 触摸。
- 虚拟摇杆。
- 按键映射。
- 长按、短按、双击。

输入设计要考虑：

- 响应速度。
- 可重映射。
- 不同平台适配。
- UI 输入和游戏输入的冲突。

## 2. 角色控制

常见角色控制：

- 2D 平台跳跃。
- 俯视角移动。
- 第一人称控制。
- 第三人称控制。
- 点击移动。
- 回合制行动。

角色控制要关注：

- 加速度和减速度。
- 转向速度。
- 跳跃手感。
- 碰撞体。
- 地面检测。
- 动画同步。
- 镜头跟随。

## 3. 物理和碰撞

需要理解：

- Rigidbody。
- Collider。
- Trigger。
- Raycast。
- 碰撞层。
- 物理材质。
- 固定时间步。

常见问题：

- 碰撞穿透。
- 角色卡墙。
- 物理抖动。
- Trigger 没触发。
- 物理和动画互相打架。

## 4. 战斗系统

基础模块：

- 属性。
- 伤害计算。
- 技能。
- Buff/Debuff。
- 命中检测。
- 冷却。
- 受击反馈。
- 死亡流程。

简单结构：

```text
输入 -> 释放技能 -> 检测命中 -> 计算伤害 -> 播放反馈 -> 更新状态
```

## 5. 敌人 AI

常见 AI 方式：

- 状态机。
- 行为树。
- Utility AI。
- 导航网格。
- 脚本事件。

简单敌人状态：

```text
巡逻 -> 发现玩家 -> 追击 -> 攻击 -> 失去目标 -> 返回巡逻
```

AI 设计重点：

- 行为可读。
- 反应有节奏。
- 不要全知全能。
- 给玩家反制空间。

## 6. 存档系统

存档内容：

- 玩家位置。
- 等级和经验。
- 背包。
- 任务状态。
- 已解锁内容。
- 设置。
- 统计数据。

常见格式：

- JSON。
- 二进制。
- SQLite。
- 平台云存档。

注意：

- 版本兼容。
- 防止存档损坏。
- 自动存档提示。
- 关键节点备份。

## 7. 配置系统

配置表适合存放：

- 敌人属性。
- 道具数据。
- 关卡配置。
- 技能参数。
- 掉落表。
- 文案。

好处：

- 不用改代码就能调数值。
- 方便策划维护。
- 支持批量生成内容。

## 8. 工具开发

游戏项目常需要自定义工具：

- 关卡编辑器。
- 对话编辑器。
- 技能编辑器。
- 配表检查工具。
- 打包工具。
- 资源导入工具。
- 自动化测试工具。

个人开发也应重视工具。重复劳动超过三次，就可以考虑做工具。

## 9. 架构建议

- 不要把所有逻辑写在角色脚本里。
- 输入、移动、战斗、动画、UI 分开。
- 用事件降低模块耦合。
- 数据和表现分离。
- 核心规则尽量可测试。

## 10. 程序检查

- 功能是否能复现测试。
- 是否有异常状态处理。
- 是否有日志。
- 是否有配置而不是硬编码。
- 是否会因为帧率不同表现不同。
- 是否支持暂停、重启和场景切换。
