# 05-游戏策划与设计基础

## 1. 游戏策划要做什么

游戏策划负责把体验目标变成规则和内容。

主要工作：

- 定义核心体验。
- 设计核心玩法。
- 设计系统和数值。
- 设计关卡和任务。
- 编写设计文档。
- 制作配置表。
- 跟进实现和测试。
- 根据反馈迭代。

## 2. 核心循环

核心循环描述玩家反复进行的行为。

示例：

```text
战斗 -> 获得资源 -> 升级装备 -> 挑战更难关卡 -> 获得更多资源
```

一个好的核心循环要有：

- 明确行动。
- 及时反馈。
- 有价值奖励。
- 下一步目标。

## 3. 机制设计

机制是玩家可操作的规则单元。

例如：

- 跳跃。
- 射击。
- 闪避。
- 建造。
- 合成。
- 抽卡。
- 潜行。
- 谈判。

机制设计要写清：

- 触发方式。
- 生效条件。
- 消耗。
- 冷却。
- 成功结果。
- 失败结果。
- 反馈。

## 4. 系统设计

常见系统：

- 角色成长。
- 装备。
- 技能。
- 任务。
- 商店。
- 背包。
- 成就。
- 掉落。
- 社交。
- 活动。

系统之间要互相驱动，不要孤立存在。

## 5. 数值设计

数值设计关注：

- 成长速度。
- 难度曲线。
- 资源产出和消耗。
- 角色强度。
- 装备收益。
- 付费和免费差距。

常见工具：

- Excel。
- Google Sheets。
- Python 模拟。
- 游戏内调试面板。

## 6. 文档能力

设计文档要让别人能执行。

好的文档：

- 目标明确。
- 规则完整。
- 边界条件清楚。
- 有流程图或表格。
- 有验收标准。
- 版本记录清晰。

差的文档：

- 只有概念。
- 大量形容词。
- 没有数值。
- 没有异常情况。
- 程序无法据此实现。

## 7. 游戏拆解

拆解一款游戏时看：

- 目标玩家。
- 核心循环。
- 主要机制。
- 成长系统。
- 经济系统。
- 关卡节奏。
- 反馈和奖励。
- 付费点。
- 流失点。

## 8. 策划练习

- 写一个 1 页游戏概念文档。
- 拆解一款小游戏的核心循环。
- 设计 10 个技能并写清规则。
- 做一张敌人数值表。
- 设计一个新手关卡。
- 写一个完整 GDD 初稿。
