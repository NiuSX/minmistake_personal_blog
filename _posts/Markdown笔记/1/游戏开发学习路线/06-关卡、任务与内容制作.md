# 06-关卡、任务与内容制作

## 1. 内容制作是什么

内容是玩家实际游玩的材料，包括：

- 关卡。
- 地图。
- 敌人配置。
- 任务。
- 剧情事件。
- 道具。
- 谜题。
- 活动。
- Boss。

系统决定游戏怎么运转，内容决定玩家每次经历什么。

## 2. 关卡设计基础

关卡要有明确目标：

- 教学。
- 练习。
- 测试。
- 探索。
- 战斗。
- 剧情。
- Boss。

关卡结构：

```text
进入 -> 引导 -> 小挑战 -> 变化 -> 高潮 -> 奖励 -> 退出
```

## 3. 地图设计

地图设计关注：

- 玩家路线。
- 视线引导。
- 安全区。
- 危险区。
- 分支路线。
- 捷径。
- 资源分布。
- 敌人位置。
- 复活点。

常见地图问题：

- 玩家迷路。
- 目标不清。
- 空间太空。
- 战斗区域太窄。
- 奖励藏得没有线索。

## 4. 任务设计

任务结构：

```text
接取 -> 目标 -> 行动 -> 变化 -> 完成 -> 奖励
```

任务类型：

- 收集。
- 击败。
- 护送。
- 探索。
- 解谜。
- 对话。
- 防守。
- 选择。

任务要避免：

- 纯跑腿。
- 文案很多但玩法单一。
- 目标位置不清。
- 奖励不匹配。

## 5. 敌人和遭遇设计

遭遇设计要考虑：

- 敌人组合。
- 出现时机。
- 战斗空间。
- 玩家资源状态。
- 逃跑或恢复机会。
- 奖励。

敌人职责：

- 近战压迫。
- 远程骚扰。
- 坦克吸收伤害。
- 控制限制移动。
- 召唤增加压力。
- 治疗保护队友。

## 6. Boss 设计

Boss 通常是机制考试和情绪高潮。

需要设计：

- 阶段。
- 技能。
- 弱点。
- 预警。
- 反制方式。
- 场地变化。
- 奖励。

Boss 技能要可读，玩家失败后要知道如何改进。

## 7. 内容生产管线

内容生产需要规范：

- 命名规则。
- 资源路径。
- 配置表字段。
- 关卡模板。
- 测试流程。
- 版本记录。

批量内容一定要能检查和复用，否则后期维护成本很高。

## 8. 内容检查

- 内容是否服务核心玩法。
- 玩家是否知道目标。
- 难度是否递进。
- 奖励是否合理。
- 是否有重复感。
- 是否有卡死风险。
- 是否能被测试复现。
