# 07-美术基础与游戏资产制作

## 1. 为什么要懂美术

即使不是美术岗位，游戏开发者也需要理解美术资产如何影响体验、性能和制作成本。

美术决定：

- 第一印象。
- 可读性。
- 氛围。
- 角色识别。
- 操作反馈。
- 品质感。
- 性能压力。

## 2. 美术基础

需要了解：

- 构图。
- 色彩。
- 明暗。
- 透视。
- 形状语言。
- 材质。
- 风格统一。
- 可读性。

游戏美术不只是好看，还要让玩家看懂。

## 3. 2D 美术

常见 2D 资产：

- 角色立绘。
- 角色序列帧。
- 地图瓦片。
- 背景。
- 图标。
- UI 面板。
- 特效贴图。

常用工具：

- Photoshop。
- Krita。
- Aseprite。
- Spine。
- TexturePacker。

## 4. 像素画

像素画适合独立游戏，但不代表制作成本低。

需要学习：

- 像素比例。
- 调色板。
- 轮廓。
- 动画帧。
- Tilemap。
- 像素完美显示。

## 5. 3D 美术

常见 3D 资产：

- 角色模型。
- 场景模型。
- 道具。
- 骨骼。
- 贴图。
- 材质。
- 动画。

常用工具：

- Blender。
- Maya。
- 3ds Max。
- ZBrush。
- Substance Painter。

## 6. 3D 基础流程

常见流程：

```text
概念设计 -> 建模 -> UV -> 贴图 -> 绑定 -> 动画 -> 导入引擎 -> 材质和光照调整
```

需要关注：

- 面数。
- 贴图尺寸。
- 骨骼数量。
- LOD。
- 碰撞体。
- 命名。
- 资源单位。

## 7. UI 美术

UI 美术包括：

- 图标。
- 按钮。
- 面板。
- 边框。
- 字体。
- 动效。
- 状态表现。

UI 要兼顾风格和可读性。按钮必须像按钮，危险操作必须明显。

## 8. 资源规范

建议规范：

- 文件命名统一。
- 源文件和导出文件分开。
- 贴图尺寸使用合理规格。
- 透明图注意边缘。
- 资源导入设置固定。
- 删除未使用资源。

## 9. 独立开发素材来源

可使用：

- 自制素材。
- 商店购买素材。
- 免费授权素材。
- 开源素材。
- 外包。

注意授权：

- 是否可商用。
- 是否需要署名。
- 是否允许修改。
- 是否允许再发布。
