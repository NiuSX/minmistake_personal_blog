# 08-动画、特效、音频与手感

## 1. 手感是什么

手感是玩家输入后，游戏在视觉、听觉、触觉和规则上给出的综合响应。

手感来自：

- 输入延迟。
- 角色速度。
- 动画衔接。
- 碰撞判定。
- 镜头反馈。
- 命中暂停。
- 屏幕震动。
- 音效。
- 特效。

## 2. 动画基础

常见动画：

- 待机。
- 移动。
- 跳跃。
- 攻击。
- 受击。
- 死亡。
- 交互。
- UI 动效。

动画需要关注：

- 前摇。
- 生效帧。
- 后摇。
- 取消。
- 过渡。
- 循环。

## 3. 动画状态机

状态机用于管理角色动画。

示例：

```text
Idle -> Run -> Jump -> Fall -> Land
Idle -> Attack -> Idle
AnyState -> Hurt
```

注意：

- 状态不要过度复杂。
- 过渡条件要清楚。
- 动画和逻辑要同步。

## 4. 特效

游戏特效包括：

- 攻击命中特效。
- 技能特效。
- 爆炸。
- 烟雾。
- 光效。
- 拾取效果。
- UI 特效。

特效作用：

- 强化反馈。
- 提示范围。
- 表达属性。
- 增强爽感。

特效不能遮挡关键信息。

## 5. 屏幕震动和命中暂停

动作游戏常用：

- **屏幕震动**：强化冲击。
- **命中暂停**：击中瞬间短暂停顿，增强打击感。
- **慢动作**：突出关键时刻。

注意：

- 不能过度使用。
- 需要提供降低震动选项。
- 不同强度攻击反馈要有层级。

## 6. 音效

常见音效：

- 点击。
- 跳跃。
- 攻击。
- 命中。
- 受伤。
- 爆炸。
- 拾取。
- 升级。
- 警告。
- 环境声。

音效要和动作时间对齐。

## 7. 音乐

音乐用于塑造情绪。

常见音乐：

- 主菜单。
- 探索。
- 战斗。
- Boss。
- 胜利。
- 失败。
- 剧情。

动态音乐可以根据战斗阶段和危险程度变化。

## 8. 手感调试

要调的参数：

- 移动速度。
- 加速度。
- 跳跃高度。
- 重力。
- 攻击前摇。
- 攻击范围。
- 镜头跟随速度。
- 震动强度。
- 音效时间点。

手感必须通过反复试玩调整，不能只靠文档。
