# 09-UI、UX 与玩家引导

## 1. UI 和 UX

- UI 是界面本身。
- UX 是玩家完成目标的整体体验。

好的 UX 会让玩家知道：

- 我在哪里。
- 我现在要做什么。
- 我能做什么。
- 我做完后发生了什么。
- 下一步去哪。

## 2. 常见界面

- 主菜单。
- 设置。
- 暂停。
- HUD。
- 背包。
- 角色。
- 技能。
- 地图。
- 任务。
- 商店。
- 结算。
- 存档。

## 3. 信息层级

界面信息要分层：

- 当前必须知道的信息。
- 当前决策需要的信息。
- 可以点开查看的信息。
- 装饰信息。

不要把所有信息都放在主界面。

## 4. HUD

HUD 常显示：

- 生命。
- 能量。
- 弹药。
- 技能冷却。
- 小地图。
- 任务目标。
- 交互提示。

HUD 要避免遮挡游戏主体。

## 5. 交互流程

设计流程时要画出来：

```text
主菜单 -> 开始游戏 -> 选择关卡 -> 游戏中 -> 结算 -> 下一关/返回
```

每个界面要清楚：

- 入口。
- 出口。
- 返回逻辑。
- 错误状态。
- 加载状态。

## 6. 新手引导

引导原则：

- 一次只教一个点。
- 教完马上实践。
- 不提前讲复杂系统。
- 失败时给提示。
- 允许跳过或回看。

引导方式：

- 任务目标。
- 高亮。
- 箭头。
- 限制操作。
- 关卡教学。
- NPC 提示。

## 7. 文案

UI 文案要短、准、可执行。

差：

```text
发生了一些错误。
```

好：

```text
存档失败，请检查磁盘空间后重试。
```

## 8. 可访问性

需要考虑：

- 字体大小。
- 色盲模式。
- 字幕。
- 按键重映射。
- 降低镜头晃动。
- 难度选项。

## 9. UI/UX 检查

- 玩家是否知道目标。
- 常用功能是否容易找到。
- 重要信息是否明显。
- 错误提示是否清楚。
- 返回逻辑是否一致。
- UI 是否适配分辨率。
- 引导是否打断过多。
