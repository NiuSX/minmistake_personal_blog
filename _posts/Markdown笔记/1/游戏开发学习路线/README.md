# 游戏开发学习路线

这套笔记回答一个核心问题：**要独立或参与做出一款游戏，需要学习哪些内容**。

游戏开发不是只学一门编程语言，也不是只会一个引擎。完整游戏需要策划、程序、美术、音频、交互、关卡、工程管理、测试、发布和运营共同配合。个人开发者不一定每个方向都精通，但必须理解每个方向的基本概念、产出物和协作方式。

## 学习建议

- 先做小游戏，不要一开始做开放世界、MMO、多人竞技或大型 RPG。
- 每学一个模块，都用一个小项目验证。
- 不要只看教程，要自己改规则、加功能、修 Bug。
- 学会拆解已有游戏：玩法循环、输入反馈、关卡节奏、资源流、UI 流程。
- 项目要能打包给别人玩，能发布的作品比半成品更能锻炼能力。
