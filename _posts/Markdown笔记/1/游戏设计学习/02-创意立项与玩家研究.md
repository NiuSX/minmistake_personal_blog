# 02-创意立项与玩家研究

## 1. 创意不是立项

一个点子只是起点，不等于项目可以成立。立项需要回答：

- 目标玩家是谁。
- 玩家为什么会想玩。
- 核心体验是什么。
- 玩法是否能做出差异。
- 团队是否有能力完成。
- 开发周期和成本是否可控。
- 商业模式是否成立。
- 是否有足够内容支撑。

例如“做一款开放世界修仙游戏”只是想法。真正的立项要明确平台、规模、核心循环、操作方式、美术规格、内容范围、团队配置和风险。

## 2. 玩家画像

玩家画像用于帮助团队统一目标受众。

常见维度：

- 年龄段。
- 游戏平台。
- 每天游戏时间。
- 付费习惯。
- 喜欢的游戏类型。
- 讨厌的体验。
- 社交需求。
- 操作能力。
- 对复杂系统的接受度。
- 对剧情、竞技、养成、探索的偏好。

示例：

```text
目标玩家：20-35 岁移动端休闲策略玩家
游戏时间：每天碎片化 10-30 分钟
核心需求：轻度策略、持续成长、低压力竞争
不能接受：强制长时间在线、复杂操作、过重付费压迫
```

## 3. 市场与竞品分析

竞品分析不是复制，而是理解玩家预期和市场空位。

需要分析：

- 游戏类型和平台。
- 核心玩法。
- 首日体验。
- 新手引导。
- 成长系统。
- 付费结构。
- 内容更新方式。
- 社区评价。
- 差评集中点。
- 留存和长期目标。

## 4. 核心体验陈述

核心体验要用一句话表达清楚。

格式：

```text
让 [目标玩家] 在 [场景/平台] 中，通过 [核心行为] 获得 [核心感受]。
```

示例：

- 让休闲玩家通过简单合成和订单规划获得轻松成长感。
- 让硬核动作玩家通过精准闪避和连招获得高压战斗成就感。
- 让合作玩家通过分工和沟通解决混乱厨房任务获得欢乐感。

## 5. 立项评估

### 5.1 玩法风险

- 核心机制是否真的有趣。
- 玩家是否容易理解。
- 操作是否适合目标平台。
- 是否有足够变化空间。
- 是否能支撑长期内容。

### 5.2 技术风险

- 引擎是否适合。
- 网络同步是否复杂。
- AI、物理、渲染、地图生成是否有难点。
- 性能是否可控。
- 是否依赖团队没有经验的技术。

### 5.3 内容风险

- 是否需要大量关卡。
- 是否需要大量剧情文本。
- 是否需要大量美术资产。
- 内容生产管线是否成熟。
- 是否能持续更新。

### 5.4 商业风险

- 玩家是否愿意付费。
- 付费点是否影响公平和体验。
- 是否适合买断、广告、内购、订阅或 DLC。
- 获客成本是否可能过高。

## 6. 立项输出物

立项阶段常见输出：

- 项目一句话介绍。
- 目标玩家画像。
- 竞品分析。
- 核心体验说明。
- 核心玩法草案。
- 美术风格参考。
- 平台和商业模式。
- 项目范围。
- 风险列表。
- 原型验证计划。
