# 03-核心玩法与机制设计

## 1. 核心玩法是什么

核心玩法是玩家在游戏中最频繁、最重要、最能代表游戏体验的行为组合。

例如：

- 射击游戏：瞄准、移动、射击、换弹、躲避、击败敌人。
- 平台跳跃：奔跑、跳跃、躲避陷阱、收集物品、到达终点。
- 卡牌游戏：抽牌、出牌、构筑、资源管理、击败对手。
- 模拟经营：生产、收集、建造、升级、扩大规模。
- Roguelike：探索、战斗、选择奖励、构筑流派、挑战随机关卡。

核心玩法必须尽早用原型验证。

## 2. 核心循环

核心循环是玩家反复执行的行为闭环。

示例：

```text
进入关卡 -> 战斗 -> 获得资源 -> 强化角色 -> 解锁新关卡 -> 再次挑战
```

核心循环要回答：

- 玩家做什么。
- 系统给什么反馈。
- 玩家获得什么奖励。
- 奖励如何推动下一轮行为。

## 3. 机制设计

机制是游戏规则中的基本动作和反应。

常见机制：

- 移动。
- 攻击。
- 防御。
- 闪避。
- 跳跃。
- 收集。
- 合成。
- 建造。
- 交易。
- 探索。
- 解谜。
- 升级。
- 抽卡。
- 匹配。

每个机制需要定义：

- 输入方式。
- 生效条件。
- 消耗资源。
- 成功结果。
- 失败结果。
- 冷却或限制。
- 反馈表现。
- 和其他机制的关系。

## 4. 目标与动机

玩家需要短期、中期、长期目标。

- **短期目标**：几秒到几分钟内完成，如击败敌人、完成一次合成。
- **中期目标**：几十分钟到几小时，如通关章节、解锁角色。
- **长期目标**：几天到几个月，如收集全图鉴、达到高排名。

目标设计要避免：

- 目标过多，玩家不知道做什么。
- 目标太远，玩家没有动力。
- 目标太简单，缺少成就感。
- 目标和核心玩法脱节。

## 5. 挑战设计

挑战来自限制和压力。

常见挑战来源：

- 时间限制。
- 资源限制。
- 敌人压力。
- 空间限制。
- 信息不完全。
- 操作精度。
- 策略选择。
- 风险收益权衡。

好的挑战应该让玩家觉得“我能做到，但需要更好地理解或操作”。

## 6. 奖励设计

奖励用于强化行为。

常见奖励：

- 数值奖励：金币、经验、材料。
- 能力奖励：技能、装备、属性。
- 内容奖励：关卡、剧情、角色。
- 表现奖励：皮肤、称号、特效。
- 社交奖励：排名、点赞、合作成果。
- 情绪奖励：爽感、惊喜、成就。

奖励要和核心玩法绑定。玩家通过主要行为获得主要成长，游戏才会形成正向循环。

## 7. 反馈设计

反馈让玩家知道自己做了什么、系统发生了什么。

反馈类型：

- 视觉：动画、闪光、颜色、UI 变化。
- 听觉：音效、音乐变化。
- 触觉：震动。
- 数值：伤害数字、经验条、分数。
- 规则反馈：敌人后退、门打开、技能冷却。

反馈不足会让游戏显得迟钝；反馈过度会造成噪音和疲劳。

## 8. 机制组合

有深度的玩法通常来自机制之间的组合。

示例：

- 火焰机制 + 草地机制 = 蔓延燃烧。
- 冰冻机制 + 水面机制 = 冻结路径。
- 潜行机制 + 声音机制 = 引诱敌人。
- 卡牌机制 + 能量机制 = 出牌顺序策略。

机制组合要注意：

- 玩家是否能理解。
- 组合是否有可预期结果。
- 是否产生过强套路。
- 是否让系统复杂度失控。

## 9. 核心玩法检查

- 玩家一分钟内主要在做什么。
- 这个行为是否有趣。
- 操作是否清晰。
- 成功和失败是否明确。
- 玩家是否知道下一步目标。
- 奖励是否驱动下一轮行为。
- 是否有成长空间。
- 是否能扩展出更多内容。
