# 05-关卡设计与内容节奏

## 1. 关卡设计是什么

关卡设计是把游戏规则、空间、敌人、资源、任务、谜题、剧情和节奏组织成玩家实际体验的过程。

关卡不是地图本身，而是玩家在空间和规则中经历的一段体验。

## 2. 关卡目标

每个关卡都要有明确目标：

- 教会一个新机制。
- 测试一个已学机制。
- 提供一次强战斗。
- 展示剧情事件。
- 提供探索奖励。
- 制造节奏变化。
- 作为 Boss 前铺垫。

如果关卡没有设计目标，很容易变成无意义填充。

## 3. 关卡基本结构

常见结构：

```text
进入 -> 学习/观察 -> 小挑战 -> 节奏提升 -> 高潮 -> 奖励 -> 退出
```

另一种结构：

```text
安全区 -> 教学区 -> 练习区 -> 组合挑战区 -> 终局挑战
```

## 4. 难度递进

经典设计方法：

1. 引入机制。
2. 让玩家安全练习。
3. 提高压力。
4. 与旧机制组合。
5. 用变化方式测试掌握程度。

示例：

- 先教玩家跳过小坑。
- 再加入移动平台。
- 再加入敌人干扰。
- 最后要求玩家在限时内连续跳跃。

## 5. 空间设计

空间影响玩家行为。

常见空间类型：

- 开阔空间：适合移动、躲避、群战。
- 狭窄通道：适合紧张感、伏击、限制视野。
- 高低差：创造路线选择和战斗优势。
- 分叉路线：提供探索和选择。
- 循环路线：减少回头路疲劳。
- 安全区：让玩家休息、整理和理解目标。

## 6. 引导设计

引导不一定是文字提示，也可以通过空间和视觉完成。

常见方法：

- 光线引导。
- 道路朝向。
- 颜色对比。
- 运动物体吸引注意。
- 远处地标。
- 敌人或奖励摆放。
- 摄像机角度。
- 声音来源。

好的引导让玩家觉得是自己发现的。

## 7. 战斗关卡

战斗关卡要关注：

- 敌人种类。
- 敌人数量。
- 出怪顺序。
- 战斗空间。
- 掩体和障碍。
- 补给位置。
- 逃生路线。
- 高潮时刻。

敌人配置原则：

- 不同敌人承担不同职责。
- 远程、近战、控制、坦克、召唤等角色要形成组合。
- 不要同时引入过多新敌人。
- 失败原因要清晰。

## 8. 解谜关卡

解谜关卡要关注：

- 玩家是否知道目标。
- 线索是否足够。
- 规则是否一致。
- 反馈是否明确。
- 是否有误导但不欺骗。
- 是否有卡死恢复方案。

解谜难度来自理解和推理，不应该来自界面不清或操作别扭。

## 9. 内容节奏

玩家不能一直高压，也不能一直平淡。

常见节奏：

- 教学。
- 探索。
- 小挑战。
- 奖励。
- 强挑战。
- 休息。
- 剧情。
- Boss。

节奏安排示例：

```text
轻松探索 -> 小战斗 -> 新机制教学 -> 组合挑战 -> 补给休息 -> Boss 战 -> 大奖励
```

## 10. 关卡文档需要包含

- 关卡名称。
- 设计目标。
- 目标玩家状态。
- 地图草图。
- 玩家路线。
- 敌人和资源配置。
- 触发器和事件。
- 剧情对白。
- 奖励。
- 失败条件。
- 复活点。
- 特殊规则。
- 测试重点。

## 11. 关卡检查

- 玩家是否知道去哪里。
- 玩家是否知道为什么失败。
- 新机制是否有安全练习空间。
- 难度是否逐步提升。
- 是否有节奏变化。
- 奖励是否和挑战匹配。
- 地图是否存在迷路或卡死。
- 关卡是否服务整体体验。
