# 06-叙事世界观与角色设计

## 1. 游戏叙事的特点

游戏叙事不是只写故事，而是通过玩家行为、规则、场景、任务、角色、道具和反馈共同讲述体验。

游戏叙事可以来自：

- 主线剧情。
- 支线任务。
- 环境细节。
- 角色对白。
- 道具描述。
- 系统规则。
- 玩家选择。
- 关卡事件。
- 音乐和美术氛围。

## 2. 世界观设计

世界观回答“这个世界如何运转”。

需要定义：

- 世界背景。
- 历史事件。
- 地理区域。
- 势力组织。
- 科技或魔法规则。
- 社会结构。
- 资源和冲突来源。
- 普通人的生活状态。

世界观不是越大越好。它应该服务玩法和情绪。

## 3. 角色设计

角色设计需要包含：

- 名字。
- 身份。
- 外形特征。
- 目标。
- 动机。
- 性格。
- 能力。
- 弱点。
- 和主角关系。
- 角色弧光。

角色要能推动玩家行为，而不是只存在于设定文档。

## 4. 主角设计

主角常见类型：

- 明确人格主角。
- 玩家自定义角色。
- 沉默主角。
- 群像主角。

设计主角要考虑：

- 玩家能否代入。
- 主角能力是否符合玩法。
- 主角目标是否和玩家目标一致。
- 主角成长是否能在系统中体现。

## 5. 任务叙事

任务不是让玩家跑腿，而是给玩家明确动机和行为目标。

任务结构：

```text
触发原因 -> 目标说明 -> 行动过程 -> 阻碍变化 -> 结果反馈 -> 奖励或后果
```

常见任务类型：

- 收集。
- 击败。
- 护送。
- 探索。
- 解谜。
- 选择。
- 防守。
- 潜入。

任务设计要避免：

- 目标重复。
- 文案很长但行为单一。
- 玩家不知道为什么做。
- 奖励和任务难度不匹配。

## 6. 环境叙事

环境叙事通过场景讲故事。

示例：

- 破损的门说明这里曾发生战斗。
- 桌上的信件说明人物关系。
- 被封锁的道路暗示灾难范围。
- 墙上的涂鸦表达阵营立场。

环境叙事优点：

- 不打断玩家控制。
- 增强沉浸。
- 让玩家主动发现。
- 可以服务关卡引导。

## 7. 分支和选择

选择要有意义，需要带来：

- 不同结果。
- 不同代价。
- 不同信息。
- 不同角色关系。
- 不同资源收益。
- 不同后续关卡或任务。

伪选择会降低信任。如果资源有限，可以做小范围差异，但要让玩家感受到选择被承认。

## 8. 文案设计

游戏文案要简洁、准确、符合语气。

不同文本要求：

- UI 文案：短、清楚、无歧义。
- 任务文本：说明目标和动机。
- 道具描述：提供功能和世界观信息。
- 角色对白：体现身份和性格。
- 系统提示：告诉玩家发生了什么和下一步怎么做。

## 9. 叙事和玩法结合

好的结合方式：

- 剧情目标转化为关卡目标。
- 角色能力来自故事设定。
- 世界规则影响玩法规则。
- 玩家选择影响系统和内容。
- Boss 机制体现角色性格。

差的结合方式：

- 玩法和剧情互不相关。
- 大段剧情打断核心体验。
- 文案解释规则，但玩家实际体验不到。
- 世界观设定复杂但对游戏没有影响。

## 10. 叙事检查

- 玩家是否知道自己为什么行动。
- 角色目标是否清晰。
- 世界规则是否一致。
- 文案是否过长。
- 剧情是否服务玩法。
- 关键情绪是否通过玩法表达。
- 选择是否有反馈。
- 任务是否避免重复跑腿感。
