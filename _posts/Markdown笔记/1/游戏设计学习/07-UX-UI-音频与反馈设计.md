# 07-UX、UI、音频与反馈设计

## 1. UX 和 UI 的区别

UX 关注玩家完成目标的整体体验，UI 是具体界面呈现和交互控件。

UX 问题：

- 玩家是否知道现在能做什么。
- 玩家是否知道下一步去哪里。
- 操作路径是否顺畅。
- 信息是否足够但不过载。
- 错误是否能恢复。

UI 问题：

- 按钮放哪里。
- 文本怎么写。
- 信息如何排版。
- 图标是否清楚。
- 状态如何显示。

## 2. 信息层级

界面中信息要有优先级。

常见层级：

- 必须立即看到：生命、弹药、关键倒计时。
- 当前决策需要：任务目标、资源数量、技能冷却。
- 可延后查看：背包详情、说明文本、历史记录。
- 装饰信息：氛围、背景、非必要统计。

不要把所有信息都同等强调。

## 3. HUD 设计

HUD 是游戏中持续显示的信息界面。

常见元素：

- 血条。
- 蓝条/能量条。
- 弹药。
- 小地图。
- 技能栏。
- 任务目标。
- 交互提示。
- 状态效果。

HUD 设计原则：

- 不遮挡关键游戏区域。
- 信息稳定，减少跳动。
- 危险信息要明显。
- 非关键状态可以弱化。
- 支持不同分辨率和平台。

## 4. 菜单和流程

常见界面流程：

- 主菜单。
- 设置。
- 存档。
- 角色。
- 背包。
- 商店。
- 任务。
- 地图。
- 结算。

流程设计要关注：

- 常用功能路径短。
- 返回逻辑一致。
- 危险操作有确认。
- 加载和等待有反馈。
- 错误提示可理解。

## 5. 新手引导

引导要让玩家学会游戏，而不是读完说明。

常见方式：

- 分步教学。
- 情境教学。
- 任务引导。
- 高亮提示。
- 限制操作范围。
- 示例演示。
- 失败后提示。

引导原则：

- 一次只教一个核心点。
- 教完马上让玩家实践。
- 不要在玩家还没需要时提前讲复杂系统。
- 支持跳过或回看。

## 6. 操作手感

手感来自输入、响应、动画、音效、镜头和规则共同作用。

影响因素：

- 输入延迟。
- 动画前摇和后摇。
- 移动加速度。
- 碰撞盒大小。
- 镜头跟随。
- 命中暂停。
- 屏幕震动。
- 音效力度。
- 特效清晰度。

动作游戏尤其要重视“按下按钮到角色响应”的时间。

## 7. 反馈设计

反馈要告诉玩家：

- 操作已发生。
- 是否成功。
- 影响了什么。
- 下一步可以做什么。

示例：

- 攻击命中：敌人受击动画、音效、伤害数字、血条变化。
- 技能不可用：按钮变灰、冷却数字、提示音。
- 获得奖励：飞行动画、音效、数量变化。
- 任务完成：弹窗、音效、地图标记变化。

## 8. 音频设计

音频包括：

- 背景音乐。
- 环境音。
- UI 音效。
- 战斗音效。
- 角色语音。
- 提示音。

音频作用：

- 强化情绪。
- 提供反馈。
- 提示危险。
- 区分状态。
- 增强空间感。

注意：

- 重要提示音要容易辨认。
- 高频重复音效不能刺耳。
- 音量层级要合理。
- 音乐变化可以提示阶段变化。

## 9. 无障碍设计

常见考虑：

- 字体大小可调。
- 色盲模式。
- 关键提示不只依赖颜色。
- 按键可重映射。
- 字幕。
- 降低镜头晃动选项。
- 难度选项。

无障碍不是额外装饰，它能扩大玩家范围并减少挫败。

## 10. UX 检查

- 玩家是否知道当前目标。
- 操作是否符合平台习惯。
- 常用功能是否容易找到。
- 重要信息是否明显。
- 提示是否能解决问题。
- 失败后是否知道原因。
- UI 是否遮挡关键内容。
- 音效是否强化而不是干扰体验。
