# 09-游戏生产流程与团队协作

## 1. 游戏项目阶段

常见阶段：

1. 概念阶段。
2. 原型阶段。
3. 预生产阶段。
4. 正式生产阶段。
5. Alpha。
6. Beta。
7. 上线准备。
8. 上线运营。

## 2. 概念阶段

目标：

- 明确项目方向。
- 定义目标玩家。
- 验证市场机会。
- 提出核心体验。

输出：

- 一句话介绍。
- 玩家画像。
- 竞品分析。
- 核心玩法草案。
- 风格参考。
- 风险列表。

## 3. 原型阶段

目标：

- 验证核心玩法是否成立。
- 验证关键技术是否可行。
- 验证操作和节奏。

输出：

- 可玩原型。
- 测试记录。
- 迭代结论。
- 是否继续开发的判断。

## 4. 预生产阶段

目标：

- 建立完整设计框架。
- 跑通技术和内容管线。
- 制作垂直切片。
- 明确项目范围和排期。

输出：

- GDD。
- 技术方案。
- 美术风格规范。
- 关卡样板。
- UI 样板。
- 数值框架。
- 制作规范。
- 项目计划。

## 5. 正式生产阶段

目标：

- 批量制作内容。
- 完成系统功能。
- 持续集成和测试。

设计师任务：

- 维护设计文档。
- 输出配置表。
- 跟进实现质量。
- 验收功能。
- 调整数值。
- 处理反馈。
- 参与版本规划。

## 6. Alpha 阶段

Alpha 通常表示主要功能和内容已接近完整，但仍有大量问题。

重点：

- 功能完整性。
- 主流程跑通。
- 重大缺陷修复。
- 内容缺口确认。
- 性能初步优化。

## 7. Beta 阶段

Beta 更接近上线质量。

重点：

- 平衡性。
- 稳定性。
- 新手体验。
- 兼容性。
- 本地化。
- 运营活动。
- 商业化验证。

## 8. 上线准备

需要确认：

- 版本号。
- 打包流程。
- 平台审核材料。
- 隐私协议。
- 服务器容量。
- 日志和监控。
- 客服工具。
- 新手活动。
- 支付测试。
- 崩溃收集。
- 热更新方案。

## 9. 团队协作

### 9.1 策划和程序

策划要提供：

- 清晰规则。
- 状态流转。
- 边界条件。
- 配置字段。
- 异常处理。
- 验收标准。

程序需要反馈：

- 实现成本。
- 技术限制。
- 性能风险。
- 更合理的数据结构。

### 9.2 策划和美术

策划要提供：

- 功能需求。
- 角色定位。
- 场景用途。
- 交互状态。
- 资源规格。

美术需要反馈：

- 制作周期。
- 表现限制。
- 风格一致性。
- 资源复用方案。

### 9.3 策划和测试

策划要提供：

- 功能说明。
- 测试重点。
- 正常流程。
- 异常流程。
- 配置表说明。

测试需要反馈：

- Bug。
- 体验问题。
- 复现步骤。
- 风险范围。

## 10. 版本管理

设计文档和配置表必须有版本管理。

需要记录：

- 修改日期。
- 修改人。
- 修改内容。
- 影响范围。
- 是否已同步程序、美术、测试。

## 11. 需求变更

需求变更要问：

- 为什么改。
- 影响哪些系统。
- 是否影响排期。
- 是否影响已有资源。
- 是否需要重新测试。
- 是否有替代方案。

不要口头变更后不更新文档。
