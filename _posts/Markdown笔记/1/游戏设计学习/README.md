学习方式建议：

- 先理解完整流程，再针对自己负责的模块深入。
- 每学一章都尝试拆解一款已有游戏。
- 不要只写设定，要能落到规则、界面、数据、关卡和测试验证。
- 游戏设计的核心不是“想点子”，而是把体验目标变成可执行、可测试、可迭代的系统。
