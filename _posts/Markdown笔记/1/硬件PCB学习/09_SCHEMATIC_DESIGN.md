# 09 原理图设计基础

## 学习目标

学完本章，你应该能：

- 理解原理图的作用。
- 掌握符号、网络、位号、电源符号、ERC 等基础概念。
- 能按模块画出清楚可读的原理图。
- 知道原理图设计中常见错误。

原理图是硬件设计的逻辑表达。它应该让人一眼看出电路模块、信号流向、电源关系和关键设计意图。

## 1. 原理图的作用

原理图用于表达：

- 元件之间如何电气连接。
- 每个模块的功能。
- 电源从哪里来，到哪里去。
- 信号从哪里来，到哪里去。
- 芯片外围电路如何配置。
- 接口引脚定义。

原理图不是 PCB 物理位置图。元件在原理图上靠近，不代表 PCB 上必须靠近；但原理图应按逻辑组织。

## 2. 原理图基本元素

| 元素 | 含义 |
| :--- | :--- |
| Symbol | 元器件符号 |
| Reference | 位号，如 R1、C2、U3 |
| Value | 参数，如 10k、100nF |
| Net | 网络，表示电气连接 |
| Net Label | 网络标签 |
| Power Symbol | 电源符号，如 3V3、5V、GND |
| Junction | 连接点 |
| No Connect | 明确不连接 |
| ERC | 电气规则检查 |

## 3. 位号命名

常见位号前缀：

| 前缀 | 元件 |
| :--- | :--- |
| R | 电阻 |
| C | 电容 |
| L | 电感 |
| D | 二极管 / LED |
| Q | 三极管 / MOSFET |
| U | IC |
| J | 连接器 |
| SW | 开关 |
| Y / X | 晶振 |
| F | 保险丝 |
| TP | 测试点 |

位号要唯一。不要出现两个 R1。

## 4. 网络命名

好的网络名能提高可读性。

推荐：

```text
3V3
5V_USB
GND
I2C_SCL
I2C_SDA
UART_TX
UART_RX
SPI_MOSI
SPI_MISO
SPI_SCK
MCU_RESET
LED_STATUS
```

不推荐：

```text
NET1
DATA
LINE
ABC
```

网络名应表达功能和方向。

## 5. 电源网络

电源要明确命名。

例如：

```text
USB_5V
5V_SYS
3V3
3V3_A
VBAT
GND
AGND
```

注意：

- 不同电源不要混用名称。
- 模拟电源和数字电源如果分开，要说明连接点。
- 电源入口、电源转换、电源输出要有清楚关系。

## 6. 模块化画图

推荐把原理图分为：

- 电源输入
- 电源转换
- 主控 MCU
- 时钟
- 复位
- 下载调试接口
- 通信接口
- 传感器
- 输出驱动
- 指示灯和按键
- 保护电路

每个模块附近添加简短注释。

## 7. 信号流向

一般原则：

- 电源从左到右或从上到下。
- 输入在左，输出在右。
- MCU 居中，外设围绕。
- 接口放在边缘逻辑位置。
- GND 符号方向统一。

清楚的原理图能减少沟通成本，也方便自己半年后再看。

## 8. 去耦电容画法

每个 IC 的电源脚通常需要去耦电容。

画法：

```text
3V3 -> 芯片 VCC
      |
     100nF
      |
     GND
```

注意：

- 原理图上可以放在芯片电源脚附近。
- PCB 上必须靠近实际电源脚。
- 多电源脚芯片可能每个电源脚都需要电容。

## 9. 未用引脚处理

未用引脚不能随便悬空。

处理方式看数据手册：

- No Connect
- 上拉
- 下拉
- 接地
- 配置为输出
- 通过电阻处理

原理图中用 No Connect 标记明确“故意不接”，避免 ERC 报错和误解。

## 10. 接口定义

连接器要标清：

- 引脚编号
- 信号名
- 电压
- 方向
- GND
- 极性

例如 UART 接口：

```text
1: GND
2: 3V3
3: UART_TX
4: UART_RX
```

注意：

TX/RX 命名要说明相对于谁。很多错误来自 TX 接 TX。

## 11. ERC 检查

ERC 可以发现：

- 未连接引脚
- 电源未驱动
- 输出短接
- 引脚类型冲突
- 未处理的 No Connect

但 ERC 不能发现：

- 参数选错。
- 芯片应用错误。
- 电平不兼容。
- 去耦电容太远。
- PCB 布局不合理。

## 12. 原理图评审方法

按模块检查：

1. 电源输入是否安全。
2. 电源转换是否符合数据手册。
3. 每个 IC 是否有去耦。
4. MCU 启动、复位、时钟是否完整。
5. 下载接口是否可用。
6. 通信接口电平是否匹配。
7. 外部接口是否有保护。
8. 未用引脚是否处理。
9. ERC 是否通过。

## 实操练习

画一个 MCU LED 按键小板原理图：

- USB 5V 输入。
- LDO 输出 3.3V。
- MCU 供电和去耦。
- 复位按键。
- 下载接口。
- 2 个按键输入。
- 3 个 LED 输出。

要求：

- 网络命名清楚。
- 模块分区清楚。
- 所有元件有位号和值。
- ERC 通过。

## 检查清单

- 原理图是否按模块组织？
- 电源网络是否命名清楚？
- 每个 IC 是否有去耦？
- 复位、时钟、Boot 是否处理？
- 接口引脚是否标清？
- 未用引脚是否处理？
- ERC 是否通过？

## 常见误区

- 误区：原理图越紧凑越好。
  纠正：可读性更重要。

- 误区：用线连很远的网络。
  纠正：远距离连接用网络标签更清楚。

- 误区：No Connect 不重要。
  纠正：明确未连接能避免误判。

## 本章总结

原理图是硬件设计的第一份核心文档。好的原理图不仅电气正确，还要结构清楚、命名明确、便于检查。先把原理图画清楚，再进入 PCB。
