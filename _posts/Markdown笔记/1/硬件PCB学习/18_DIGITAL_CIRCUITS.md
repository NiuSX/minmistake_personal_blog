# 18 数字电路设计基础

## 学习目标

学完本章，你应该能：

- 理解数字电路中的高低电平。
- 掌握上拉、下拉、开漏、推挽、复位、时钟等基础概念。
- 知道不同电压域之间不能随意连接。
- 能设计基础 MCU 数字接口。

数字电路看起来只有 0 和 1，但真实世界里仍然是电压、边沿、噪声和时序。

## 1. 高低电平

数字信号用电压范围表示 0 和 1。

例如 3.3V 系统：

- 低电平接近 0V。
- 高电平接近 3.3V。

但芯片判断高低不是简单等于 0 和 3.3，而是有阈值：

- VIH：输入高电平最小值。
- VIL：输入低电平最大值。

必须看数据手册。

## 2. 电平兼容

常见电压域：

- 1.8V
- 3.3V
- 5V

注意：

- 3.3V 输入不一定能承受 5V。
- 1.8V 高电平不一定能被 3.3V 芯片识别为高。
- 不同电压域通信可能需要电平转换。

## 3. 推挽输出

Push-Pull 输出可以主动拉高和拉低。

特点：

- 速度快。
- 驱动能力较强。
- 不能多个输出直接并联。

普通 GPIO 输出常是推挽。

## 4. 开漏输出

Open-Drain 输出只能主动拉低，不能主动拉高，需要上拉电阻。

特点：

- 多个器件可以共享总线。
- 常用于 I2C、中断线。
- 上升沿速度受上拉电阻和总线电容影响。

I2C 的 SDA、SCL 通常是开漏。

## 5. 上拉和下拉

上拉：

```text
信号 -> 电阻 -> VCC
```

默认高电平。

下拉：

```text
信号 -> 电阻 -> GND
```

默认低电平。

用途：

- 防止输入悬空。
- 设置启动模式。
- 按键输入。
- 总线空闲状态。

常见阻值：

- 4.7k
- 10k
- 100k

## 6. 输入悬空

数字输入如果悬空，电平不确定，可能：

- 随机跳变。
- 增加功耗。
- 引发误触发。

解决：

- 上拉。
- 下拉。
- 软件配置内部上下拉。
- 未用引脚按数据手册处理。

## 7. 复位电路

复位让系统从已知状态启动。

常见方式：

- RC 复位。
- 复位芯片。
- 手动复位按键。
- MCU 内部复位。
- 看门狗复位。

设计注意：

- 复位电平方向。
- 上拉或下拉。
- 复位脉冲宽度。
- 下载器是否需要控制复位。

## 8. 时钟

时钟提供数字系统节拍。

来源：

- 内部 RC。
- 外部晶振。
- 有源晶振。
- PLL。

选择考虑：

- 精度。
- 频率。
- 启动时间。
- 功耗。
- USB 或通信是否要求精确时钟。

## 9. 去抖

机械按键会抖动。

表现：

- 按一次，电平快速跳多次。

处理：

- 软件延时消抖。
- RC 滤波。
- 施密特触发。

新手通常用软件消抖即可。

## 10. 数字接口保护

外部数字接口可能需要：

- 串联电阻。
- 上拉 / 下拉。
- ESD 保护。
- 电平转换。
- 端接电阻。

例如外部按键线较长时，可能需要滤波和 ESD。

## 11. 时序

数字通信要满足时序。

常见：

- 建立时间。
- 保持时间。
- 时钟频率。
- 脉宽。
- 传播延迟。

低速 GPIO 通常不敏感，高速 SPI、并口、存储器则需要关注。

## 12. 数字电路布局

原则：

- 时钟线短。
- 高速线有地参考。
- 不跨地缝。
- 数字噪声远离模拟输入。
- 去耦靠近芯片。
- 电平转换器靠近跨域接口。

## 实操练习

1. 用上拉电阻设计一个按键输入。
2. 用万用表测按键按下和松开的电平。
3. 查 MCU 数据手册，找某 GPIO 的 VIH、VIL。
4. 设计一个 5V 传感器到 3.3V MCU 的电平转换方案。
5. 用逻辑分析仪观察按键抖动。

## 检查清单

- 输入引脚是否避免悬空？
- 上拉电阻接到正确电压了吗？
- 3.3V 和 5V 电平是否兼容？
- 复位电路是否符合手册？
- 时钟是否满足接口要求？
- 外部接口是否有保护？

## 常见误区

- 误区：数字信号只有 0 和 1，不用管模拟问题。
  纠正：数字信号本质仍是模拟电压波形。

- 误区：所有 GPIO 都能承受 5V。
  纠正：必须看是否 5V tolerant。

- 误区：输入脚悬空没关系。
  纠正：悬空会导致不确定状态和功耗增加。

## 本章总结

数字电路基础包括电平、上下拉、输出类型、复位、时钟和时序。低速数字电路容易入门，但仍要尊重电压、电流、边沿和数据手册。
