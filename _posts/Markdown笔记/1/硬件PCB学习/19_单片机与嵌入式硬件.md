# 19 单片机与嵌入式硬件

## 学习目标

学完本章，你应该能：

- 理解 MCU 最小系统包含哪些硬件。
- 知道电源、复位、时钟、下载接口、Boot 配置的重要性。
- 能设计基础 MCU 开发板。
- 知道 MCU 硬件设计中常见错误。

单片机是很多硬件项目的核心。学习 PCB 时，MCU 最小系统板是非常适合的入门项目。

## 1. MCU 是什么

MCU 是 Microcontroller Unit，微控制器。

它通常包含：

- CPU
- Flash
- SRAM
- GPIO
- Timer
- UART
- I2C
- SPI
- ADC
- PWM
- Watchdog

MCU 用于控制外设、读取传感器、通信、执行逻辑。

## 2. MCU 最小系统

最小系统通常包括：

- MCU 芯片。
- 电源。
- 去耦电容。
- 复位电路。
- 时钟。
- 下载调试接口。
- Boot 配置。
- 必要上拉 / 下拉。

缺一项都可能导致 MCU 无法启动或无法下载。

## 3. 电源

检查：

- MCU 供电电压。
- 每个 VDD 是否连接。
- 每个 VSS 是否接地。
- VDDA / VREF 等模拟电源是否处理。
- 去耦电容是否靠近。

常见去耦：

- 每个电源脚 100nF。
- 电源入口或芯片附近 1uF / 4.7uF。

具体看数据手册。

## 4. 复位

复位引脚可能需要：

- 上拉电阻。
- 电容延时。
- 复位按键。
- 下载器连接。

常见错误：

- 复位脚悬空。
- 复位电平方向搞错。
- 复位按键接法错误。
- 复位脚被外部电路强拉。

## 5. 时钟

MCU 可使用：

- 内部 RC。
- 外部晶振。
- 有源晶振。

外部晶振设计：

- 晶振靠近 MCU。
- 负载电容按公式或参考设计。
- 走线短。
- 避免噪声。

有些应用必须高精度时钟：

- USB
- 精确定时
- 某些通信

## 6. 下载和调试接口

常见：

- SWD
- JTAG
- UART Bootloader
- USB DFU

SWD 常见引脚：

- SWDIO
- SWCLK
- RESET
- GND
- VTref / 3V3

注意：

- 下载器需要共地。
- VTref 用于识别目标板电压。
- 不要让其他外设强占调试引脚。
- 量产前也要保留测试或下载方式。

## 7. Boot 配置

Boot 引脚决定 MCU 从哪里启动。

例如：

- 用户 Flash
- 系统 Bootloader
- SRAM

设计要：

- 按手册上拉或下拉。
- 如需下载模式，留按键或跳帽。
- 不要让 Boot 脚悬空。

## 8. GPIO 规划

设计前列 IO 表：

| 引脚 | 功能 | 方向 | 默认状态 | 备注 |
| :--- | :--- | :--- | :--- | :--- |
| PA0 | Button | Input | Pull-up | 按下为低 |
| PA1 | LED | Output | Low | 高亮 |
| PA2 | UART_TX | Output | High idle | 调试串口 |
| PA3 | UART_RX | Input | Pull-up | 调试串口 |

IO 规划要考虑：

- 复用功能。
- 启动状态。
- 是否耐压。
- 是否有 ADC。
- 是否被下载接口占用。
- 上电默认状态是否会误动作。

## 9. 模拟电源和参考

很多 MCU 有：

- VDDA
- VSSA
- VREF+
- VREF-

如果使用 ADC，需要认真处理：

- 模拟电源滤波。
- 参考电压稳定。
- 去耦靠近。
- 模拟输入布局。

即使不用 ADC，也要按手册处理这些引脚。

## 10. 外设连接

常见外设：

- LED
- 按键
- 蜂鸣器
- 传感器
- 显示屏
- 电机驱动
- 存储器

注意：

- GPIO 驱动能力有限。
- 感性负载需要驱动和保护。
- 外设电平要兼容。
- 通信接口要有上拉或终端。

## 11. MCU 板调试顺序

1. 测 3.3V 电源。
2. 测复位脚电平。
3. 测晶振是否工作。
4. 连接下载器。
5. 尝试识别芯片。
6. 下载最简单程序。
7. 点亮 LED。
8. 测串口输出。
9. 再测试外设。

## 实操练习

设计一个 MCU 最小系统：

- 3.3V 电源。
- MCU。
- 每个 VDD 去耦。
- 复位按键。
- 外部晶振。
- SWD 接口。
- 一个 LED。
- 一个按键。
- UART 调试口。

输出：

- 原理图。
- IO 分配表。
- PCB 初步布局。

## 检查清单

- 所有电源脚是否连接？
- 去耦电容是否完整？
- 复位是否正确？
- Boot 是否有默认状态？
- 下载接口是否完整？
- 调试引脚是否被占用？
- 外部晶振电容是否正确？
- IO 电平是否兼容？

## 常见误区

- 误区：MCU 只接一个 VDD 就行。
  纠正：所有电源脚都要按手册连接。

- 误区：下载接口只接两根线就行。
  纠正：还需要 GND，很多情况下需要 VTref 和 Reset。

- 误区：Boot 脚悬空也能启动。
  纠正：悬空可能随机启动失败。

## 本章总结

MCU 最小系统是硬件入门的重要项目。把电源、去耦、复位、时钟、下载、Boot 和 IO 规划做好，就能建立设计嵌入式硬件的基础能力。
