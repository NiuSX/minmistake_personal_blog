# 20 常见通信接口

## 学习目标

学完本章，你应该能：

- 区分 UART、I2C、SPI、USB、CAN、RS-485 的特点。
- 知道每种接口的基本连线和硬件注意事项。
- 能用逻辑分析仪或示波器进行基础调试。
- 避免常见接口接线错误。

通信接口是 MCU 和外部芯片、模块、电脑、传感器之间交换数据的方式。

## 1. UART

UART 是异步串口。

基本信号：

- TX
- RX
- GND

可选：

- RTS
- CTS

特点：

- 简单。
- 常用于调试。
- 点对点通信。

注意：

- A 的 TX 接 B 的 RX。
- A 的 RX 接 B 的 TX。
- 必须共地。
- 波特率一致。
- 电平兼容。

常见错误：

- TX 接 TX。
- 忘记 GND。
- 5V 串口接 3.3V MCU。
- 波特率设置错误。

## 2. I2C

I2C 是双线总线。

信号：

- SCL：时钟
- SDA：数据

特点：

- 多设备共享。
- 每个设备有地址。
- 开漏，需要上拉。

硬件要点：

- SDA、SCL 上拉到正确电压。
- 总线电容不能太大。
- 地址不能冲突。
- 走线不要太长。

常见上拉：

- 2.2k
- 4.7k
- 10k

上拉值取决于速度、总线电容和功耗。

## 3. SPI

SPI 是同步高速接口。

信号：

- SCLK
- MOSI
- MISO
- CS / NSS

特点：

- 速度快。
- 全双工。
- 每个从设备通常一个 CS。

硬件注意：

- SCLK 线尽量短。
- 多从设备时 CS 分开。
- 高速时关注串扰和边沿。
- 必要时串联小电阻改善边沿。

常见错误：

- CPOL / CPHA 软件配置错。
- CS 极性错。
- MISO 多设备冲突。
- 线太长导致波形差。

## 4. USB

USB 常见信号：

- VBUS
- D+
- D-
- GND
- Type-C 还有 CC1 / CC2

特点：

- 差分通信。
- 对布局有要求。
- 外部接口需要 ESD 保护。

硬件要点：

- D+ / D- 成对走线。
- 控制阻抗。
- ESD 靠近接口。
- 注意连接器引脚。
- Type-C 设备端需要 CC 电阻。
- VBUS 保护和限流。

USB 比 UART/I2C/SPI 更依赖 PCB 质量。

## 5. CAN

CAN 常用于汽车和工业。

需要：

- MCU CAN 控制器或外部控制器。
- CAN 收发器。
- CANH / CANL。
- 终端电阻。

特点：

- 差分。
- 抗干扰强。
- 多节点。

硬件注意：

- 总线两端 120Ω 终端。
- ESD 保护。
- 共模电感视需求。
- 收发器供电和逻辑电平匹配。

## 6. RS-485

RS-485 是工业常用差分总线。

信号：

- A
- B
- GND 或参考地

特点：

- 长距离。
- 抗干扰。
- 多节点。
- 半双工常见。

硬件注意：

- 终端电阻。
- 偏置电阻。
- TVS 防护。
- 共地或隔离。
- 方向控制 DE / RE。

## 7. 1-Wire

特点：

- 一根数据线加 GND。
- 常用于温度传感器。
- 需要上拉。

注意：

- 线长和寄生电容影响通信。
- 时序较严格。

## 8. 电平转换

不同电压域通信需要转换。

常见方法：

- 电阻分压：简单单向，速度有限。
- MOSFET 双向转换：常用于 I2C。
- 专用电平转换芯片。
- 收发器芯片。

不要默认 5V 和 3.3V 可以直接连。

## 9. 接口保护

外部接口建议考虑：

- ESD 二极管或 TVS。
- 串联电阻。
- 共模电感。
- 保险丝。
- 防反接。
- 端接电阻。

保护器件应靠近接口。

## 10. 接口调试工具

UART：

- USB 转串口。
- 逻辑分析仪。
- 示波器。

I2C：

- 逻辑分析仪协议解码。
- 示波器看上升沿。

SPI：

- 逻辑分析仪看 SCLK、MOSI、MISO、CS。

USB：

- 示波器、USB 分析仪。
- 新手更多依赖参考设计和软件枚举信息。

CAN / RS-485：

- 总线分析仪。
- 示波器看差分波形。

## 实操练习

1. 用 USB 转串口连接 MCU，输出 Hello。
2. 用逻辑分析仪抓 UART，确认波特率。
3. 接一个 I2C 温湿度传感器，抓 SCL/SDA。
4. 接一个 SPI OLED 或 Flash，抓四根线。
5. 画一个 USB Type-C 供电和 USB 2.0 数据接口原理图。

## 检查清单

- UART TX/RX 是否交叉？
- 通信双方是否共地？
- I2C 是否有上拉？
- I2C 地址是否冲突？
- SPI CS 是否独立？
- USB D+/D- 是否按差分线处理？
- 外部接口是否有 ESD？
- 电平是否兼容？

## 常见误区

- 误区：UART 不需要 GND。
  纠正：没有共同参考地，电平无法可靠识别。

- 误区：I2C 不需要上拉。
  纠正：I2C 开漏结构必须有上拉。

- 误区：USB 随便走两根线就行。
  纠正：USB 是差分高速接口，要按布局规则。

## 本章总结

通信接口各有适用场景。UART 简单适合调试，I2C 适合低速多设备，SPI 适合高速短距离，USB、CAN、RS-485 对硬件设计要求更高。每种接口都要关注电平、地、保护和布局。
