# 35 PCB 设计检查清单

## 学习目标

学完本章，你应该能：

- 在原理图、PCB、生产、焊接和上电前逐项检查。
- 建立自己的设计检查流程。
- 降低低级错误导致打样失败的概率。

检查清单是硬件工程中非常实用的工具。很多错误不是因为不会，而是因为忘了检查。

## 1. 需求检查

- 供电方式是否明确？
- 输入电压范围是否明确？
- 输出接口是否明确？
- 板子尺寸是否明确？
- 是否有外壳和安装孔要求？
- 工作环境是否明确？
- 最大电流是否估算？
- 是否有安全风险？

## 2. 原理图检查

电源：

- 输入保护是否需要？
- 防反接是否需要？
- 保险丝是否需要？
- LDO / DC-DC 选型是否满足电流？
- 去耦电容是否完整？
- 电源网络命名是否清楚？

MCU：

- 所有 VDD / VSS 是否连接？
- VDDA / VREF 是否处理？
- 复位是否正确？
- Boot 是否有默认状态？
- 下载接口是否完整？
- 晶振是否正确？

接口：

- TX/RX 是否正确？
- I2C 是否有上拉？
- SPI CS 是否独立？
- USB 是否有 ESD？
- 外部接口电平是否兼容？

保护：

- 感性负载是否有续流？
- 外部接口是否有 ESD？
- 大电流是否有过流保护？

## 3. 数据手册检查

- 供电范围是否符合推荐工作条件？
- 引脚功能是否确认？
- 未用引脚如何处理？
- 典型应用电路是否参考？
- 推荐电容是否使用？
- Layout Guidelines 是否阅读？
- 封装尺寸是否核对？
- 订购型号是否明确？

## 4. 封装检查

- 封装是否和实物一致？
- 引脚编号是否一致？
- Pin 1 是否正确？
- 极性方向是否正确？
- 通孔孔径是否适合？
- 焊盘大小是否适合焊接？
- 连接器方向是否正确？
- 3D 模型是否大致匹配？

## 5. PCB 布局检查

- 固定器件位置是否正确？
- 安装孔是否留空间？
- 接口是否靠板边？
- 去耦是否靠近芯片？
- 晶振是否靠近 MCU？
- 电源芯片外围是否紧凑？
- 大电流路径是否短？
- 模拟区域是否远离噪声？
- TVS 是否靠近接口？
- 测试点是否可接触？

## 6. PCB 布线检查

- 所有网络是否连接？
- 电源线是否足够宽？
- GND 是否连续？
- 去耦到地路径是否短？
- 高速线是否有参考平面？
- 差分线是否成对？
- 大电流回流是否合理？
- 模拟信号是否避开噪声？
- 铺铜是否有孤岛？
- DRC 是否通过？

## 7. 丝印检查

- 项目名是否有？
- 版本号是否有？
- 电源极性是否标清？
- 接口引脚是否标清？
- LED 方向是否标清？
- 二极管方向是否标清？
- IC Pin 1 是否标清？
- 测试点名称是否标清？
- 丝印是否压焊盘？

## 8. 生产文件检查

- Gerber 是否完整？
- Drill 是否导出？
- 板框是否正确？
- 层数是否正确？
- 阻焊层是否正确？
- 丝印层是否正确？
- BOM 是否完整？
- CPL 是否正确？
- 原理图 PDF 是否导出？
- 文件名是否带版本？

## 9. 焊接前检查

- 元件是否齐全？
- 封装是否匹配？
- 极性器件是否确认方向？
- 烙铁和工具是否准备？
- PCB 是否有明显制造缺陷？
- 是否规划焊接顺序？

## 10. 上电前检查

- 电源和 GND 是否短路？
- 电源极性是否正确？
- 限流电源是否设置？
- 电解电容方向是否正确？
- 芯片方向是否正确？
- LDO / DC-DC 输出是否先空载测试？
- 负载是否暂时断开？

## 11. 调试检查

- 输入电压是否正确？
- 输出电压是否正确？
- 上电电流是否合理？
- 是否有异常发热？
- 复位是否正常？
- 时钟是否正常？
- 下载是否成功？
- UART / I2C / SPI 是否正常？
- 传感器数据是否合理？
- 执行器是否动作正常？

## 12. 改版检查

- 问题是否记录？
- 根因是否确认？
- 修改是否只解决症状还是根因？
- 原理图是否更新？
- PCB 是否更新？
- BOM 是否更新？
- 版本号是否更新？
- CHANGELOG 是否更新？

## 13. 工程规则检查

- 是否按项目类型建立 Net Class？
- 普通信号、电源、大电流、时钟、差分、模拟网络是否使用不同规则？
- 线宽、线距、过孔孔径是否参考了板厂能力表？
- 本项目采用值是否比板厂最小值留有余量？
- 大电流走线是否估算了电流、压降和温升？
- 高压或外部接口是否评估了间隙、爬电距离和保护？
- 差分线是否设置了线宽、线距、间距和长度约束？
- 是否填写了下单参数表？

## 14. 评审与记录检查

- 需求评审是否记录输入电压、负载、接口、尺寸和风险？
- 原理图评审是否记录电源、MCU、接口、保护和测试点？
- 布局评审是否在大量布线前完成？
- 首次上电是否记录限流值、测量点、实测电压和电流？
- 调试问题是否有复现步骤和测量条件？
- 改版记录是否区分了根因、临时修复和正式修改？
- Gerber、BOM、CPL、原理图 PDF 是否来自同一版本？

## 实操练习

把本章清单复制成你的项目检查模板，并在每次打样前逐项填写：

```text
[ ] 已检查原理图
[ ] 已检查封装
[ ] 已检查 PCB 布局
[ ] 已检查 DRC
[ ] 已检查 Gerber
[ ] 已检查 BOM
[ ] 已检查工程规则
[ ] 已填写上电记录
[ ] 已填写改版复盘
```

## 常见误区

- 误区：检查清单太麻烦。
  纠正：打样失败和返工更麻烦。

- 误区：小项目不用检查。
  纠正：小项目也会有连接器反、封装错、电源短路。

- 误区：只靠 DRC。
  纠正：DRC 不能检查设计意图和器件参数。

## 本章总结

硬件检查清单是降低错误率的工程工具。每次打样前都应该走一遍原理图、封装、布局、布线、生产文件和上电安全检查。
