# 04. 计算机组成与体系结构

## 讲义导读：把程序运行拆成系统链路

这一章讲的是 **04. 计算机组成与体系结构**，属于 **计算机组成与体系结构**。读计算机系统时，不要把它当成名词表，而要把它当成一条从源代码到真实机器行为的链路：代码写下去，编译器怎么翻译，链接器怎么装配，加载器怎么启动，进程怎么占用地址空间，CPU 怎么执行指令，内核怎么处理系统调用，文件和网络怎么把数据送出去。

### 一句话先懂

体系结构关心的是 CPU、缓存、内存和 I/O 如何协作，以及为什么同样的算法在机器上可能快慢差很多。

初学时可以先盯住三个问题：第一，程序此刻在哪一层运行；第二，这一层把什么输入转换成了什么输出；第三，如果结果不对，应该用哪个工具观察证据。

### 通俗类比

CPU 很快但离仓库远，所以需要手边抽屉一样的缓存；如果工人总是来回跑仓库，再聪明的工人也会被等待拖慢。

类比只是入门扶手。真正考试、调试或做项目时，要回到准确术语：地址、字节、指令、寄存器、符号、页表、文件描述符、socket、锁、调度、缓存、协议字段和错误码。能把类比翻译回这些具体对象，才算真的懂。

### 本章学习主线

1. **先定位层级**：这是语言问题、编译链接问题、运行时问题、内核问题、硬件问题，还是网络/存储问题？
2. **再追踪路径**：一次完整过程从哪里开始，经过哪些对象，最后在哪里产生可见结果？
3. **然后看状态**：关键状态包括寄存器、栈、堆、页表、文件偏移、进程状态、锁状态、TCP 状态和缓存状态。
4. **接着找边界**：位宽、对齐、权限、资源限制、并发时序、平台差异和标准未定义行为，都会让直觉失效。
5. **最后做验证**：用小程序、命令、日志、反汇编、抓包、性能计数或 sanitizer 证明解释是对的。

### 本章重点抓手

指令流水线、乱序执行、缓存层次、局部性、分支预测、总线、I/O 设备和存储层次。

### 最小实践任务

写矩阵遍历或数组访问小基准，改变访问顺序和数据规模，观察缓存命中率、运行时间和 perf 计数。

建议把每次实验记录成固定格式：目标、环境、最小代码、运行命令、观察结果、底层解释、容易误判的点、下一步问题。这样以后遇到段错误、性能抖动、死锁、网络超时或数据丢失时，不会只凭感觉猜。

### 常见误区

- 只看算法复杂度，不看常数、缓存和分支。
- 把硬件行为想成完全顺序执行。
- 用不稳定的小基准得出过度结论。

### 推荐观察工具

gcc/clang、objdump/readelf、gdb、strace、perf、pmap、ss、tcpdump、/proc、日志和小基准测试。

### 读完本章应该能做到

- 用自己的话解释本章概念，并能指出它处在系统链路的哪一层。
- 画出一个最小路径图，说明数据或控制流从哪里来、到哪里去。
- 用至少一个命令或实验观察到真实现象，而不是只复述结论。
- 说清一个常见故障的表现、可能原因、验证方式和修复方向。

## 1. 计算机组成关注什么

计算机组成与体系结构研究硬件如何执行程序。

核心问题：

- CPU 如何执行指令？
- 寄存器、ALU、控制器如何协作？
- 流水线为什么能提高吞吐？
- 缓存为什么重要？
- 内存为什么慢？
- I/O 设备如何与 CPU 通信？
- 多核系统为什么会有一致性问题？

## 2. 冯·诺依曼模型

经典模型：

```text
输入设备 -> 存储器 -> 运算器 / 控制器 -> 输出设备
```

现代计算机仍然保留基本思想：

- 程序和数据都存储在内存中。
- CPU 从内存取指令。
- CPU 执行指令并读写数据。

但现代系统加入了：

- 多级缓存。
- 流水线。
- 分支预测。
- 乱序执行。
- SIMD。
- 多核。
- 虚拟内存。
- DMA。

## 3. CPU 基本组成

CPU 包括：

| 部件 | 作用 |
|---|---|
| PC / IP | 指向下一条指令 |
| 寄存器堆 | 保存临时数据 |
| ALU | 算术和逻辑运算 |
| 控制单元 | 解码指令并产生控制信号 |
| Cache | 缓存指令和数据 |
| MMU | 地址转换 |
| FPU | 浮点运算 |
| SIMD 单元 | 向量运算 |

## 4. 指令执行周期

简化周期：

```text
Fetch -> Decode -> Execute -> Memory -> Writeback
```

| 阶段 | 含义 |
|---|---|
| Fetch | 取指令 |
| Decode | 解码指令 |
| Execute | 执行 ALU 或地址计算 |
| Memory | 访问内存 |
| Writeback | 写回寄存器 |

## 5. 流水线

流水线把不同指令的不同阶段重叠执行。

不使用流水线：

```text
指令1: F D E M W
指令2:           F D E M W
```

使用流水线：

```text
指令1: F D E M W
指令2:   F D E M W
指令3:     F D E M W
```

流水线提升吞吐，但不一定降低单条指令延迟。

## 6. 流水线冒险

### 6.1 数据冒险

后一条指令依赖前一条结果：

```asm
add r1, r2, r3
sub r4, r1, r5
```

解决：

- 转发。
- 暂停。
- 编译器重排。

### 6.2 控制冒险

分支跳转导致下一条指令不确定。

解决：

- 分支预测。
- 延迟槽。
- 推测执行。

### 6.3 结构冒险

多个指令同时争用硬件资源。

解决：

- 增加硬件资源。
- 分离指令缓存和数据缓存。
- 调度。

## 7. 缓存

CPU 很快，内存相对很慢。缓存用于利用局部性。

### 7.1 局部性

时间局部性：

```text
刚访问过的数据，短时间内可能再次访问。
```

空间局部性：

```text
访问某个地址后，附近地址也可能被访问。
```

数组顺序访问比链表随机访问更快，原因之一就是缓存局部性。

### 7.2 多级缓存

常见层次：

```text
寄存器 -> L1 Cache -> L2 Cache -> L3 Cache -> 内存 -> SSD/HDD
```

越靠近 CPU：

- 越快。
- 越小。
- 越贵。

### 7.3 Cache line

缓存不是按字节搬运，而是按 cache line 搬运。常见 cache line 大小为 64 bytes。

如果访问数组：

```c
for (int i = 0; i < n; i++) {
    sum += a[i];
}
```

读取 `a[i]` 时，附近元素也会被加载到缓存。

### 7.4 缓存不友好例子

二维数组按行存储：

```c
int a[1024][1024];
```

按行访问：

```c
for (int i = 0; i < 1024; i++)
    for (int j = 0; j < 1024; j++)
        sum += a[i][j];
```

按列访问：

```c
for (int j = 0; j < 1024; j++)
    for (int i = 0; i < 1024; i++)
        sum += a[i][j];
```

按行通常更快，因为空间局部性更好。

## 8. 虚拟内存与 MMU

CPU 访问的是虚拟地址。MMU 把虚拟地址转换为物理地址。

```text
虚拟地址 -> MMU / 页表 / TLB -> 物理地址
```

虚拟内存的好处：

- 每个进程有独立地址空间。
- 提供隔离。
- 简化内存管理。
- 支持 mmap。
- 支持按需分页。

## 9. 多核与缓存一致性

多核 CPU 每个核可能有自己的缓存。

问题：

```text
核心 A 修改变量 x
核心 B 何时看到新值？
```

这涉及：

- 缓存一致性协议。
- 内存屏障。
- 原子操作。
- 编译器重排。
- CPU 重排。

并发程序不能只从单核顺序执行角度理解。

## 10. I/O 设备

I/O 设备包括：

- 磁盘。
- 网卡。
- 显卡。
- 键盘鼠标。
- USB 设备。
- 传感器。

CPU 与设备交互方式：

- 端口 I/O。
- 内存映射 I/O。
- 中断。
- DMA。

### 10.1 中断

设备完成任务后通过中断通知 CPU。

例如：

- 网卡收到包。
- 磁盘读完成。
- 键盘按下。

中断避免 CPU 一直轮询设备状态。

### 10.2 DMA

DMA 允许设备直接读写内存，减少 CPU 参与数据搬运。

例如磁盘读数据：

```text
磁盘控制器 -> DMA -> 内存
CPU 只负责发起请求和处理完成事件
```

## 11. 性能直觉

常见性能差异：

```text
寄存器访问     极快
L1 cache       很快
L2/L3 cache    较快
内存访问       慢很多
SSD            更慢
网络           更慢且不稳定
人工交互       极慢
```

程序优化时要关注：

- 内存访问模式。
- 缓存命中率。
- 分支预测。
- 数据结构布局。
- 锁竞争。
- I/O 次数。
- 系统调用次数。

## 12. 常见误区

| 误区 | 正确认识 |
|---|---|
| CPU 只按代码顺序简单执行 | 现代 CPU 有流水线、乱序、预测 |
| 内存访问都一样快 | 缓存命中和未命中差距很大 |
| 多核只是多个 CPU 简单并行 | 涉及缓存一致性和同步 |
| SSD 很快所以不用考虑 I/O | 相比内存仍慢很多 |
| 优化就是换更快算法 | 数据布局和局部性也非常关键 |

这一组笔记建议按“教材主线 + 标准文档 + 本机实验”三层学习，不要只看二手总结。
