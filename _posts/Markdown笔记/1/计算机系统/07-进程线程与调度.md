## 讲义导读：把程序运行拆成系统链路

这一章讲的是 **07. 进程、线程与调度**，属于 **进程、线程与调度**。读计算机系统时，不要把它当成名词表，而要把它当成一条从源代码到真实机器行为的链路：代码写下去，编译器怎么翻译，链接器怎么装配，加载器怎么启动，进程怎么占用地址空间，CPU 怎么执行指令，内核怎么处理系统调用，文件和网络怎么把数据送出去。

### 一句话先懂

进程线程与调度关心的是操作系统如何同时管理多个执行流，以及上下文切换、阻塞、信号和优先级如何影响程序。

初学时可以先盯住三个问题：第一，程序此刻在哪一层运行；第二，这一层把什么输入转换成了什么输出；第三，如果结果不对，应该用哪个工具观察证据。

### 通俗类比

进程像独立办公室，线程像同一办公室里的多人协作；调度器像排班员，决定谁先用 CPU 这台关键设备。

类比只是入门扶手。真正考试、调试或做项目时，要回到准确术语：地址、字节、指令、寄存器、符号、页表、文件描述符、socket、锁、调度、缓存、协议字段和错误码。能把类比翻译回这些具体对象，才算真的懂。

### 本章学习主线

1. **先定位层级**：这是语言问题、编译链接问题、运行时问题、内核问题、硬件问题，还是网络/存储问题？
2. **再追踪路径**：一次完整过程从哪里开始，经过哪些对象，最后在哪里产生可见结果？
3. **然后看状态**：关键状态包括寄存器、栈、堆、页表、文件偏移、进程状态、锁状态、TCP 状态和缓存状态。
4. **接着找边界**：位宽、对齐、权限、资源限制、并发时序、平台差异和标准未定义行为，都会让直觉失效。
5. **最后做验证**：用小程序、命令、日志、反汇编、抓包、性能计数或 sanitizer 证明解释是对的。

## 1. 进程是什么

进程是运行中的程序实例。它包含：

- 独立虚拟地址空间。
- 代码和数据。
- 打开的文件描述符。
- 寄存器上下文。
- 栈。
- 堆。
- 环境变量。
- 当前工作目录。
- 权限和资源限制。

程序是磁盘上的文件，进程是程序运行后的动态实体。

## 2. 进程的核心抽象

操作系统给进程提供两个重要假象：

1. 独占 CPU：通过调度实现。
2. 独占内存：通过虚拟内存实现。

实际上：

- CPU 在多个进程之间快速切换。
- 物理内存由内核统一管理。
- 进程之间被隔离。

## 3. 创建进程

Unix/Linux 常见方式：

```text
fork -> exec
```

`fork()` 创建当前进程的副本。

`exec()` 用新程序替换当前进程映像。

二者边界非常重要：

| 调用 | 做了什么 | 没做什么 |
|---|---|---|
| `fork` | 创建子进程，复制父进程的进程上下文 | 不加载新程序 |
| `exec` | 用新程序替换当前进程的代码、数据、堆栈等用户态映像 | 不创建新进程，PID 通常不变 |
| `wait` / `waitpid` | 回收子进程退出状态 | 不负责终止子进程 |

示例：

```c
pid_t pid = fork();
if (pid == 0) {
    execlp("ls", "ls", "-l", NULL);
} else {
    wait(NULL);
}
```

shell 执行命令大致就是这个模型。

`fork` 后父子进程最初共享许多物理页，依靠 Copy-on-Write 延迟复制。只有某一方写入共享页时，内核才复制物理页。这解释了为什么 `fork` 大进程不一定立即复制全部内存，但如果 `fork` 后父子都大量写内存，成本仍然很高。

常见坑：

- `fork` 后父子都会从 `fork` 返回处继续执行，要根据返回值区分父子。
- 多线程程序中调用 `fork` 后，子进程只保留调用 `fork` 的那个线程，其他线程持有的锁状态可能遗留，容易死锁。
- `exec` 失败时会返回，成功时不会返回；因此子进程中 `exec` 失败后应显式 `_exit` 或处理错误。

## 4. 进程状态

常见状态：

| 状态 | 含义 |
|---|---|
| running | 正在 CPU 上运行 |
| ready | 可运行，等待调度 |
| blocked | 等待 I/O 或事件 |
| zombie | 已退出但父进程未回收 |
| stopped | 被暂停 |

查看：

```bash
ps aux
top
```

## 5. 上下文切换

上下文切换是 CPU 从一个任务切到另一个任务。

需要保存和恢复：

- 寄存器。
- 程序计数器。
- 栈指针。
- 内核调度信息。
- 地址空间相关状态。

上下文切换有成本：

- 保存恢复寄存器。
- 可能破坏缓存局部性。
- 可能造成 TLB 影响。

线程过多会导致调度开销增加。

## 6. 线程是什么

线程是进程内的执行流。

同一进程中的线程共享：

- 地址空间。
- 全局变量。
- 堆。
- 文件描述符。
- 进程权限。

每个线程独有：

- 栈。
- 寄存器上下文。
- 线程局部存储。
- 调度状态。

线程适合：

- I/O 并发。
- 后台任务。
- 多核并行计算。
- 需要共享内存的任务。

风险：

- 数据竞争。
- 死锁。
- 调试复杂。
- 生命周期难管理。

线程共享文件描述符表意味着：一个线程 `close(fd)` 会影响同一进程内其他线程；一个线程修改文件偏移量，也可能影响其他线程对同一 open file description 的读写位置。因此多线程文件 I/O 要么使用独立打开的 fd，要么使用 `pread/pwrite` 这类带偏移的接口，要么在应用层同步。

## 7. 进程和线程对比

| 对比 | 进程 | 线程 |
|---|---|---|
| 地址空间 | 独立 | 共享进程地址空间 |
| 创建成本 | 较高 | 较低 |
| 通信 | IPC | 共享内存更直接 |
| 隔离性 | 好 | 差 |
| 崩溃影响 | 通常影响本进程 | 一个线程崩溃会导致整个进程崩溃 |
| 调试难度 | 相对低 | 并发问题更复杂 |

## 8. 调度

调度器决定哪个任务运行。

目标可能包括：

- 公平性。
- 响应时间。
- 吞吐量。
- 低延迟。
- 实时性。
- 避免饥饿。

常见调度思想：

- FIFO。
- Round Robin。
- Priority。
- Multi-level feedback queue。
- Completely Fair Scheduler。

调度不只发生在进程之间，也发生在线程之间。Linux 中线程本质上也是调度实体，只是多个线程共享同一进程的地址空间和资源。用户态看到的“进程”和“线程”是编程模型，内核调度时关心的是可运行任务。

影响调度的因素：

- 任务是否可运行。
- 优先级和 nice 值。
- 是否等待 I/O。
- CPU 亲和性。
- cgroup 限制。
- 实时调度策略。
- 负载均衡和 NUMA 拓扑。

## 9. CPU 时间

常见指标：

| 指标 | 含义 |
|---|---|
| user time | 用户态执行时间 |
| system time | 内核态执行时间 |
| wall time | 真实流逝时间 |
| CPU utilization | CPU 利用率 |
| load average | 可运行和不可中断任务队列压力 |

一个程序 wall time 长，不一定 CPU 忙，可能在等 I/O。

`load average` 不是 CPU 使用率。Linux load average 统计可运行任务和不可中断睡眠任务数量。机器 CPU 利用率不高但 load 很高，可能是大量任务堵在磁盘 I/O、网络文件系统或内核不可中断等待上。

## 10. 进程间通信 IPC

常见方式：

- 管道。
- 命名管道。
- 信号。
- 共享内存。
- 消息队列。
- Unix domain socket。
- TCP/UDP socket。
- 文件。

选择原则：

| 需求 | 推荐 |
|---|---|
| 父子进程简单流式通信 | pipe |
| 本机服务通信 | Unix domain socket |
| 大数据共享 | shared memory |
| 跨机器通信 | TCP/UDP |
| 简单通知 | signal/eventfd |

## 11. 信号

信号是异步通知机制。

常见信号：

| 信号 | 含义 |
|---|---|
| `SIGINT` | Ctrl+C |
| `SIGTERM` | 请求终止 |
| `SIGKILL` | 强制终止，不能捕获 |
| `SIGSEGV` | 段错误 |
| `SIGCHLD` | 子进程状态变化 |
| `SIGPIPE` | 管道或 socket 对端关闭后写入 |

信号处理要谨慎，因为它可能在任意时刻打断程序。

信号处理函数中只能安全调用 async-signal-safe 的函数。不要在信号处理函数里随意调用 `printf`、`malloc`、加锁、写复杂日志，因为它们可能不是可重入的，可能破坏内部状态或导致死锁。

更稳妥的做法：

- 信号处理函数只设置 `volatile sig_atomic_t` 标志。
- 主循环检查标志后执行清理逻辑。
- 服务端程序可使用 self-pipe、`signalfd` 或事件循环统一处理信号。

`SIGKILL` 和 `SIGSTOP` 不能被捕获、阻塞或忽略。优雅退出通常使用 `SIGTERM`，给程序释放资源和刷日志的机会。

## 12. 守护进程

守护进程是后台长期运行的服务。

特点：

- 脱离终端。
- 后台运行。
- 记录日志。
- 处理信号。
- 由 systemd 等管理。

现代 Linux 常用 systemd 管理服务。

## 13. 常用命令

```bash
ps aux
pstree
top
htop
pidstat 1
kill -TERM <pid>
kill -KILL <pid>
wait
jobs
fg
bg
```

查看进程资源：

```bash
cat /proc/<pid>/status
cat /proc/<pid>/cmdline
ls /proc/<pid>/fd
cat /proc/<pid>/maps
```

跟踪系统调用：

```bash
strace -f ./app
```

定位“卡住”的进程：

```bash
ps -o pid,ppid,stat,wchan,cmd -p <pid>
cat /proc/<pid>/stack
strace -f -tt -p <pid>
gdb -p <pid>
```

定位线程：

```bash
ps -L -p <pid> -o pid,tid,psr,pcpu,stat,wchan,comm
top -H -p <pid>
```

如果某个线程 CPU 高，可以先用 `top -H` 找到线程 TID，再把十进制 TID 转成十六进制，在 `gdb` 或日志中匹配线程 ID。

## 14. 常见问题

| 问题 | 可能原因 |
|---|---|
| 僵尸进程 | 父进程没有 wait |
| CPU 100% | 忙等、死循环、锁竞争 |
| 线程数暴涨 | 线程泄漏、线程池配置错误 |
| 程序卡住 | 等锁、等 I/O、死锁、阻塞系统调用 |
| 子进程没退出 | 父进程未发送终止或未回收 |
| load 高但 CPU 不高 | 大量任务等待 I/O 或不可中断睡眠 |
| `fork` 后子进程卡死 | 多线程进程 fork 后继承了锁状态 |
| 服务无法优雅退出 | 信号处理逻辑缺失或阻塞在不可中断调用 |
