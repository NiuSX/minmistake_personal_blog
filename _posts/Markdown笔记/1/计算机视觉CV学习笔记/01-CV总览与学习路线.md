# 01-CV 总览与学习路线

## 1. 什么是计算机视觉

计算机视觉 Computer Vision，简称 CV，是让计算机从图像、视频、深度图、点云、多光谱图像等视觉数据中理解世界的技术。它的目标不是简单“看见图片”，而是从视觉数据中提取结构化信息，用于识别、检测、定位、测量、跟踪、分割、重建、理解和决策。

常见任务：

- 图像分类：判断图里是什么。
- 目标检测：找出目标位置和类别。
- 图像分割：把图像按像素划分区域。
- 实例分割：区分同类目标的不同实例。
- 关键点检测：检测人体、手、脸、物体关键点。
- OCR：检测和识别图像中的文字。
- 目标跟踪：在视频中持续追踪目标。
- 行为识别：识别视频中的动作或事件。
- 三维重建：从图像恢复三维结构。
- 缺陷检测：发现工业产品表面异常。
- 视觉测量：测量尺寸、角度、位置、面积。
- 机器人视觉：识别物体、定位、抓取、导航。

## 2. CV 学习的知识地图

完整 CV 学习可分为七层：

### 2.1 图像基础层

需要理解：

- 像素、分辨率、通道。
- RGB、BGR、HSV、灰度图。
- 图像坐标系。
- 图像直方图。
- 卷积和滤波。
- 频域和空域。
- 噪声和模糊。

### 2.2 OpenCV 工具层

需要掌握：

- 图像读取、显示、保存。
- 色彩空间转换。
- 缩放、裁剪、旋转、仿射变换。
- 阈值、边缘、轮廓、形态学。
- 模板匹配、霍夫变换。
- 相机标定、特征匹配。

### 2.3 传统算法层

需要掌握：

- 滤波降噪。
- 阈值分割。
- 边缘检测。
- 轮廓分析。
- 连通域分析。
- 形态学处理。
- 角点检测。
- 特征描述子。
- 图像配准。

传统算法优势是可解释、速度快、数据需求低；缺点是对光照、背景、姿态、噪声敏感。

### 2.4 机器学习层

需要理解：

- 特征工程。
- SVM、KNN、随机森林。
- 聚类。
- PCA 降维。
- HOG + SVM。
- Haar + AdaBoost。

这部分是传统 CV 和深度学习之间的重要过渡。

### 2.5 深度学习视觉层

需要掌握：

- CNN。
- 卷积、池化、归一化、激活函数。
- ResNet、MobileNet、EfficientNet。
- Vision Transformer。
- 迁移学习。
- 数据增强。
- 损失函数。
- 优化器。

### 2.6 任务模型层

需要掌握：

- 分类模型。
- 检测模型：YOLO、Faster R-CNN、SSD、RetinaNet。
- 分割模型：FCN、U-Net、DeepLab、Mask R-CNN、SAM。
- 关键点模型：OpenPose、HRNet、MediaPipe。
- OCR 模型：CRNN、DBNet、PaddleOCR。
- 跟踪模型：SORT、DeepSORT、ByteTrack。

### 2.7 工程落地层

需要掌握：

- 数据采集。
- 标注规范。
- 模型训练。
- 评估指标。
- 推理加速。
- ONNX、TensorRT、OpenVINO。
- 边缘部署。
- 摄像头标定。
- 线上监控。
- 误报漏报分析。

## 3. CV 常见应用场景

### 3.1 工业视觉

任务：

- 缺陷检测。
- 尺寸测量。
- OCR 读码。
- 定位引导。
- 装配检测。
- 外观检测。

特点：

- 场景可控。
- 光源和相机很重要。
- 精度和稳定性要求高。
- 误报漏报成本明确。
- 传统算法和深度学习经常混合使用。

### 3.2 自动驾驶和机器人

任务：

- 车道线检测。
- 车辆行人检测。
- 语义分割。
- 深度估计。
- SLAM。
- 目标跟踪。
- 视觉导航。

特点：

- 环境复杂。
- 实时性要求高。
- 需要多传感器融合。
- 安全性要求极高。

### 3.3 安防和视频分析

任务：

- 人脸识别。
- 行人检测。
- 越界检测。
- 人群计数。
- 行为识别。
- 目标跟踪。

特点：

- 摄像头多。
- 光照变化大。
- 遮挡多。
- 隐私和合规重要。

### 3.4 医学影像

任务：

- 病灶检测。
- 器官分割。
- 影像分类。
- 三维重建。

特点：

- 数据标注昂贵。
- 误判影响大。
- 可解释性重要。
- 需要医生参与评估。

## 4. 学习路线

### 阶段 1：图像和 OpenCV

目标：

- 能读写图像。
- 能进行预处理。
- 能用阈值、边缘、轮廓解决简单问题。

练习：

- 提取图片中的红色区域。
- 统计目标面积。
- 检测圆形零件。
- 识别二维码区域。

### 阶段 2：传统算法

目标：

- 理解滤波、边缘、轮廓、特征。
- 能用传统方法做定位和测量。

练习：

- 用 Canny 提取边缘。
- 用霍夫变换检测直线和圆。
- 用轮廓计算物体中心和角度。
- 用 ORB 做图像匹配。

### 阶段 3：深度学习

目标：

- 能训练分类和检测模型。
- 理解数据增强、损失函数、指标。

练习：

- 训练猫狗分类。
- 训练 YOLO 检测自定义目标。
- 训练 U-Net 做分割。

### 阶段 4：工程部署

目标：

- 模型能在真实环境稳定运行。
- 能分析误报漏报。
- 能优化推理速度。

练习：

- 摄像头实时检测。
- ONNX 导出。
- TensorRT 加速。
- 写一个可视化检测工具。

## 5. CV 项目基本流程

```text
需求分析 -> 图像采集 -> 数据标注 -> 数据清洗 -> 基线算法 -> 模型训练 -> 指标评估 -> 误差分析 -> 部署优化 -> 现场迭代
```

关键原则：

- 先明确业务指标，再选择算法。
- 数据质量优先于模型复杂度。
- 先做简单基线，再上复杂模型。
- 评估指标要和实际场景一致。
- 现场光照、相机、安装位置会决定很多算法成败。

## 讲义级补充：如何真正学懂《01-CV 总览与学习路线》

> 适用位置：计算机视觉CV学习笔记\01-CV总览与学习路线.md  
> 说明：本补充用于把原始提纲扩展成课堂讲义式学习材料。阅读时建议先看原文，再用本节建立知识框架、例子、实践和自测闭环。

### 1. 这一讲要解决什么问题

这类内容的核心不是公式本身，而是“数据如何变成可学习的表示，模型如何产生预测，误差如何反馈改进”。学习时要把数据、假设、损失、优化、评估和部署放在一起理解。

学习本讲时，可以用三个问题检查自己是否真的理解：

1. 它解决的真实问题是什么？
2. 如果没有它，系统会出现什么具体麻烦？
3. 在真实项目中，应该用什么现象或指标判断它做得好不好？

### 2. 核心知识拆解

可以把本讲拆成几块来学：

- 数据：样本、标签、分布、增强、清洗和划分。
- 模型：假设空间、结构、参数和表达能力。
- 训练：损失函数、优化器、正则化、学习率和收敛。
- 评估：指标、误差分析、泛化、鲁棒性和部署表现。

拆解的好处是防止“整章都懂一点，但哪块都说不清”。复习时可以逐块追问：它的输入是什么、输出是什么、依赖什么、失败时有什么表现。

### 3. 通俗类比

可以把模型类比成一个需要反复练习的学生：数据是练习题，标签是参考答案，损失函数是扣分规则，优化器是改错方法，验证集像模拟考试，测试集才像正式考试。只在训练集上表现好，就像只会背做过的题。

类比不是严格定义，但能帮助初学者先建立直觉。真正使用时，还要回到术语、公式、接口、数据结构、时序图或工程规范上，把“感觉理解”变成“可验证理解”。

### 4. 具体例子

学习《01-CV 总览与学习路线》时，可以准备一个很小的数据集，先手工看 20 个样本和标签，再训练一个基线模型，最后把预测错误的样本逐张分析。很多模型问题不是公式错了，而是数据、标注或评价方式一开始就没有定义清楚。

讲义级学习不能只停留在“概念解释”。至少要有一个能跑、能算、能画或能检查的例子。例子越小，越容易看清关键机制；等机制清楚后，再逐步扩展到复杂项目。

### 5. 学习路径

- 先明确任务类型：分类、回归、检测、分割、生成、序列建模或决策。
- 再梳理数据、特征、模型、损失函数、优化方法和评价指标。
- 最后通过误差分析定位问题来自数据、标注、模型容量、训练策略还是部署环境。

建议每学完一小节都做一次“复述练习”：不用看笔记，用自己的话讲清楚概念、输入、输出、关键步骤和常见错误。如果讲不清，通常说明还没有真正掌握。

### 6. 课堂讲解框架

可以按下面顺序讲解或复习本主题：

1. 背景：先讲这个知识为什么出现，它试图降低什么成本、解决什么风险或提升什么能力。
2. 基本概念：给出核心名词的准确定义，说明它们之间的关系。
3. 工作流程：按时间顺序描述一次完整过程，必要时画出流程图、状态机或数据流图。
4. 关键细节：解释最容易误解的机制，例如边界条件、异常处理、性能限制、资源生命周期或安全约束。
5. 实战例子：用一个足够小但完整的例子，把概念落到命令、代码、图纸、配置、数据或操作步骤上。
6. 反例与排错：展示错误做法会导致什么现象，再说明如何定位和修复。
7. 总结迁移：最后说明它和相邻知识点的区别、联系以及后续该学什么。

### 7. 最小实践任务

为了避免“看懂了但不会用”，建议为本讲配一个最小实践：

- 选一个可以在 30 到 90 分钟内完成的小任务。
- 明确输入、预期输出和验收标准。
- 记录遇到的第一个错误、定位过程和最终修复方法。
- 完成后写 5 行复盘：我原来以为是什么，实际是什么，下次会如何更快处理。

如果本主题偏理论，实践可以是手算一个小例子、画一张流程图、推导一个简化公式或解释一段真实日志；如果偏工程，实践应该尽量落到可运行命令、可测试代码、可检查配置或可测量硬件现象上。

### 8. 常见误区

- 只追求模型结构新，不检查数据质量和标注一致性。
- 只看平均指标，不看错误样本、长尾类别和业务代价。
- 把训练集效果当成真实效果，忽略验证集、测试集和线上分布变化。

遇到这些问题时，不要急着背更多资料。更有效的办法是回到一个最小例子，把输入、状态变化、输出和验证方式重新走一遍。

### 9. 自测题

1. 用一句话说明本讲主题解决的核心问题。
2. 列出本讲最重要的 3 个概念，并说明它们的关系。
3. 举一个生活类比，再指出这个类比在哪些地方不严谨。
4. 写出一个最小实践任务的验收标准。
5. 如果结果不符合预期，你会优先检查哪 3 个环节？为什么？
6. 本讲和相邻章节的知识边界是什么？哪些问题应该交给其他章节解决？

### 10. 复习口诀

先问场景，再看输入；先拆结构，再走流程；先做小例，再谈优化；先会排错，再做规模化。
