## 讲义导读：从像素到可用视觉系统

这一章讲的是 **03-图像预处理与增强精讲**。阅读时不要只记 OpenCV 函数、模型名字或论文缩写，而要把它放进完整视觉链路：图像如何采集，像素和颜色空间是否正确，光照和噪声如何影响结果，相机几何是否成立，模型输入和标注是否一致，评价指标是否对应漏检误检成本，部署时前后处理、延迟和现场漂移是否可控。CV 的目标不是让一张样例图看起来成功，而是在真实图像分布中稳定输出可用信息。

### 一句话先懂

预处理和增强是在模型或算法前整理视觉信号：减少噪声、修正光照、突出目标，同时避免破坏真实特征。

### 通俗类比

可以把 CV 系统想成流水线上的质检员：相机和光源负责把产品拍清楚，预处理像擦干净镜头和统一灯光，传统特征或神经网络负责看重点，检测和分割负责指出位置，后处理负责合并和过滤结果，评估指标负责统计漏检、误检和定位偏差，部署监控负责发现现场条件变化。

类比只是第一层直觉。回到技术上，要把每一步写成明确对象：输入图像的尺寸、通道、数据类型、坐标系、标注格式、模型输出、阈值、指标和运行环境。视觉问题很容易被表面效果迷惑，必须用可复现样本和量化指标来判断。

1. **图像来源**：确认采集设备、光照、分辨率、帧率、压缩、畸变和场景覆盖。

## 1. 预处理的目的

图像预处理是在正式算法或模型推理前，对图像进行清洗、规范化、增强和变换。它的目标不是让图片“看起来漂亮”，而是让后续算法更稳定、更准确、更容易处理。

常见目的：

- 降低噪声。
- 提高对比度。
- 统一尺寸。
- 统一亮度和颜色。
- 去除背景干扰。
- 增强目标边缘。
- 修正几何畸变。
- 扩充训练数据。
- 减少模型输入差异。

## 2. 尺寸处理

### 2.1 Resize

模型通常要求固定输入尺寸，例如 224x224、640x640。

```python
resized = cv2.resize(img, (640, 640))
```

问题：

- 直接拉伸会改变目标比例。
- 小目标可能被压缩丢失。
- 大图缩小会丢失细节。

### 2.2 Letterbox

目标检测常用 letterbox，保持比例缩放，不足部分填充。

流程：

```text
原图 -> 按比例缩放 -> 边缘填充 -> 固定尺寸输入
```

优点：

- 保持目标比例。
- 适合 YOLO 等检测模型。

注意：

- 推理后坐标需要还原到原图。
- 填充值通常用灰色或均值。

### 2.3 裁剪

裁剪可去掉无关区域。

常见方式：

- 中心裁剪。
- 随机裁剪。
- ROI 裁剪。
- 滑窗裁剪。

工业视觉中，ROI 裁剪非常重要。相机位置固定时，可以只处理目标所在区域，减少背景干扰。

## 3. 亮度和对比度处理

### 3.1 线性亮度对比度调整

公式：

```text
output = alpha * input + beta
```

- alpha 控制对比度。
- beta 控制亮度。

```python
adjusted = cv2.convertScaleAbs(img, alpha=1.2, beta=20)
```

适用：

- 图像整体偏暗或偏灰。

不适用：

- 局部光照不均。
- 过曝区域已经丢失细节。

### 3.2 Gamma 校正

Gamma 校正用于非线性调整亮度。

```text
output = input ^ gamma
```

- gamma < 1：提亮暗部。
- gamma > 1：压暗图像。

适合处理暗部细节，但不要过度，否则噪声也会被放大。

### 3.3 直方图均衡化

灰度直方图均衡化增强整体对比度。

```python
eq = cv2.equalizeHist(gray)
```

缺点：

- 可能过度增强噪声。
- 对彩色图直接逐通道均衡可能产生颜色偏移。

### 3.4 CLAHE

CLAHE 是自适应直方图均衡化，限制局部对比度增强。

```python
clahe = cv2.createCLAHE(clipLimit=2.0, tileGridSize=(8, 8))
out = clahe.apply(gray)
```

适用：

- 医学图像。
- 工业缺陷。
- 光照不均图像。

## 4. 去噪

### 4.1 均值滤波

用邻域平均值替换当前像素。

```python
blur = cv2.blur(img, (3, 3))
```

优点：

- 简单快速。

缺点：

- 模糊边缘。

### 4.2 高斯滤波

用高斯权重平滑图像。

```python
blur = cv2.GaussianBlur(img, (5, 5), 0)
```

适合：

- 高斯噪声。
- Canny 边缘检测前预处理。

### 4.3 中值滤波

用邻域中位数替换当前像素。

```python
median = cv2.medianBlur(img, 5)
```

适合：

- 椒盐噪声。

优点：

- 保边效果比均值滤波好。

### 4.4 双边滤波

双边滤波同时考虑空间距离和像素差异，能平滑噪声并保留边缘。

```python
bilateral = cv2.bilateralFilter(img, 9, 75, 75)
```

缺点：

- 计算慢。

## 5. 锐化和边缘增强

### 5.1 拉普拉斯锐化

通过增强高频信息突出边缘。

```python
kernel = np.array([[0, -1, 0],
                   [-1, 5, -1],
                   [0, -1, 0]])
sharp = cv2.filter2D(img, -1, kernel)
```

注意：

- 锐化会同时增强噪声。
- 先去噪再锐化通常更稳定。

### 5.2 Unsharp Mask

流程：

```text
模糊图像 -> 原图 - 模糊图 -> 高频细节 -> 原图 + 高频细节
```

适合增强边缘和纹理。

## 6. 颜色预处理

### 6.1 白平衡

白平衡用于修正光源色偏。

常见方法：

- 灰度世界假设。
- 白点法。
- 学习型白平衡。

工业视觉中，稳定光源比复杂白平衡更重要。

### 6.2 颜色归一化

用于减少光照和相机差异。

常见处理：

- RGB 归一化。
- 通道均值方差归一化。
- 转 HSV 后只用 H/S。
- 转 Lab 后处理 L 通道。

## 7. 二值化预处理

### 7.1 固定阈值

```python
_, binary = cv2.threshold(gray, 127, 255, cv2.THRESH_BINARY)
```

适合光照稳定、目标和背景灰度差明显的情况。

### 7.2 Otsu 阈值

自动寻找类间方差最大的阈值。

```python
_, binary = cv2.threshold(gray, 0, 255, cv2.THRESH_BINARY + cv2.THRESH_OTSU)
```

适合双峰直方图。

### 7.3 自适应阈值

根据局部区域计算阈值。

```python
binary = cv2.adaptiveThreshold(gray, 255, cv2.ADAPTIVE_THRESH_GAUSSIAN_C,
                               cv2.THRESH_BINARY, 11, 2)
```

适合光照不均。

## 8. 形态学预处理

常见操作：

- 腐蚀：去小白点、缩小白色区域。
- 膨胀：填补小孔、扩大白色区域。
- 开运算：先腐蚀后膨胀，去噪点。
- 闭运算：先膨胀后腐蚀，填小洞。
- 形态学梯度：提取边界。
- 顶帽：提取亮小目标。
- 黑帽：提取暗小目标。

```python
kernel = cv2.getStructuringElement(cv2.MORPH_RECT, (3, 3))
opened = cv2.morphologyEx(binary, cv2.MORPH_OPEN, kernel)
```

## 9. 数据增强

数据增强用于训练深度学习模型，提高泛化能力。

### 9.1 几何增强

- 翻转。
- 旋转。
- 平移。
- 缩放。
- 裁剪。
- 透视变换。
- 仿射变换。

注意：

- 检测和分割任务必须同步变换标注。
- 不合理增强会破坏标签。例如数字 6 旋转后可能变成 9。

### 9.2 颜色增强

- 亮度。
- 对比度。
- 饱和度。
- 色相。
- Gamma。
- 模糊。
- 噪声。

### 9.3 遮挡增强

- Cutout。
- Random Erasing。
- MixUp。
- CutMix。
- Mosaic。

适合提升遮挡和复杂背景鲁棒性。

## 10. 预处理选择原则

- 传统算法：预处理通常决定成败。
- 深度学习：预处理要和训练时保持一致。
- 工业视觉：优先控制光源、相机、背景，再考虑算法。
- 不要盲目堆预处理，每一步都要验证是否提升指标。

## 11. 常见错误

- 训练和推理预处理不一致。
- Resize 后标注框没有同步变换。
- 过度锐化导致误检。
- 去噪太强导致小缺陷消失。
- 阈值方法在光照变化时失效。
- 只在样例图上调参数，没有覆盖真实场景。

#### 典型场景拆解

在真实场景中，$topic 通常会经历“需求出现、方案选择、实现落地、问题暴露、持续优化”几个阶段。需求出现时，要先判断这个需求属于基础能力、性能优化、体验改进、可靠性建设还是长期架构演进。不同类型的需求对方案的评价标准不同：基础能力看正确性，性能优化看指标，体验改进看路径是否顺滑，可靠性建设看故障时能否降级和恢复，架构演进看未来变化是否容易吸收。
