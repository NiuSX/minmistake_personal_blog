# 04-传统图像处理算法精讲

## 1. 传统图像处理的价值

深度学习很强，但传统图像处理仍然重要，尤其在以下场景：

- 目标形状规则。
- 光照和背景可控。
- 数据量少。
- 需要高可解释性。
- 需要精确测量。
- 算力有限。
- 工业视觉项目快速交付。

传统算法常用流程：

```text
读图 -> 预处理 -> 阈值/边缘 -> 形态学 -> 轮廓/连通域 -> 特征计算 -> 规则判断
```

## 2. 阈值分割

阈值分割根据像素灰度或颜色把目标和背景分开。

适用条件：

- 目标和背景差异明显。
- 光照稳定。
- 图像噪声较少。

常见方法：

- 固定阈值。
- Otsu。
- 自适应阈值。
- HSV 颜色阈值。

阈值分割优点：

- 简单。
- 快速。
- 可解释。

缺点：

- 对光照敏感。
- 对阴影、反光敏感。
- 对背景复杂场景效果差。

## 3. 边缘检测

边缘是图像灰度变化剧烈的位置。

### 3.1 Sobel

Sobel 计算水平和垂直方向梯度。

```python
gx = cv2.Sobel(gray, cv2.CV_64F, 1, 0, ksize=3)
gy = cv2.Sobel(gray, cv2.CV_64F, 0, 1, ksize=3)
```

适合检测方向边缘。

### 3.2 Laplacian

二阶导数边缘检测，对噪声敏感。

```python
edge = cv2.Laplacian(gray, cv2.CV_64F)
```

### 3.3 Canny

Canny 是经典边缘检测算法。

流程：

1. 高斯滤波去噪。
2. 计算梯度幅值和方向。
3. 非极大值抑制。
4. 双阈值筛选。
5. 边缘连接。

```python
edges = cv2.Canny(gray, 50, 150)
```

参数：

- 低阈值。
- 高阈值。

经验：

- 高阈值控制强边缘。
- 低阈值控制边缘连接。
- 阈值太低会有很多噪声。
- 阈值太高会断边。

## 4. 轮廓检测

轮廓是二值图中连续目标边界。

```python
contours, hierarchy = cv2.findContours(binary, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)
```

常用特征：

- 面积。
- 周长。
- 外接矩形。
- 最小外接矩形。
- 最小外接圆。
- 质心。
- 方向角。
- 凸包。
- 轮廓近似。

示例：

```python
area = cv2.contourArea(cnt)
x, y, w, h = cv2.boundingRect(cnt)
M = cv2.moments(cnt)
cx = int(M["m10"] / M["m00"])
cy = int(M["m01"] / M["m00"])
```

应用：

- 计数。
- 定位。
- 尺寸测量。
- 形状判断。
- 缺陷区域提取。

## 5. 连通域分析

连通域分析用于找二值图中的独立区域。

```python
num_labels, labels, stats, centroids = cv2.connectedComponentsWithStats(binary)
```

stats 包含：

- x。
- y。
- width。
- height。
- area。

连通域适合做目标计数和小区域过滤。

## 6. 形态学算法

形态学基于结构元素处理二值图或灰度图。

### 6.1 腐蚀

腐蚀会缩小白色区域。

用途：

- 去除小白噪声。
- 分离细连接。
- 缩小目标。

### 6.2 膨胀

膨胀会扩大白色区域。

用途：

- 填补小缺口。
- 连接断裂区域。
- 扩大目标。

### 6.3 开运算

先腐蚀后膨胀。

用途：

- 去除孤立噪点。

### 6.4 闭运算

先膨胀后腐蚀。

用途：

- 填补小孔洞。
- 连接近邻区域。

## 7. 霍夫变换

### 7.1 直线检测

霍夫直线变换把图像空间的点映射到参数空间，寻找共线点。

```python
lines = cv2.HoughLinesP(edges, 1, np.pi / 180, threshold=80,
                        minLineLength=50, maxLineGap=10)
```

参数：

- threshold：投票阈值。
- minLineLength：最短直线长度。
- maxLineGap：允许断裂间隔。

应用：

- 车道线。
- 边框检测。
- 工件直线边缘。
- 表格线检测。

### 7.2 圆检测

```python
circles = cv2.HoughCircles(gray, cv2.HOUGH_GRADIENT, dp=1.2,
                           minDist=30, param1=100, param2=30,
                           minRadius=10, maxRadius=100)
```

应用：

- 圆孔。
- 螺丝。
- 瓶盖。
- 圆形零件。

## 8. 模板匹配

模板匹配在大图中寻找与模板相似的区域。

```python
result = cv2.matchTemplate(img, template, cv2.TM_CCOEFF_NORMED)
_, max_val, _, max_loc = cv2.minMaxLoc(result)
```

适用：

- 目标外观固定。
- 尺度和旋转变化小。
- 背景可控。

缺点：

- 对尺度变化敏感。
- 对旋转敏感。
- 对光照变化敏感。

改进：

- 多尺度模板匹配。
- 旋转模板。
- 边缘模板。
- 特征匹配。

## 9. 传统算法项目案例：检测圆形零件

流程：

```text
灰度化 -> 高斯滤波 -> Canny -> 霍夫圆 -> 圆心和半径输出 -> 规则过滤
```

需要考虑：

- 光照是否均匀。
- 圆边缘是否清晰。
- 是否存在反光。
- 是否有遮挡。
- 圆半径范围是否已知。

## 10. 传统算法失效原因

- 光照变化。
- 背景复杂。
- 目标姿态变化大。
- 目标外观不一致。
- 噪声严重。
- 反光和阴影。
- 参数对场景过拟合。

工程建议：

- 能控制硬件就先控制硬件。
- 固定光源、背景、相机位置。
- 使用 ROI 降低干扰。
- 给参数留调节入口。
- 用可视化工具显示每一步中间结果。

## 讲义级补充：如何真正学懂《04-传统图像处理算法精讲》

### 1. 这一讲要解决什么问题

这类内容的核心不是公式本身，而是“数据如何变成可学习的表示，模型如何产生预测，误差如何反馈改进”。学习时要把数据、假设、损失、优化、评估和部署放在一起理解。

学习本讲时，可以用三个问题检查自己是否真的理解：

1. 它解决的真实问题是什么？
2. 如果没有它，系统会出现什么具体麻烦？
3. 在真实项目中，应该用什么现象或指标判断它做得好不好？

### 2. 核心知识拆解

可以把本讲拆成几块来学：

- 数据：样本、标签、分布、增强、清洗和划分。
- 模型：假设空间、结构、参数和表达能力。
- 训练：损失函数、优化器、正则化、学习率和收敛。
- 评估：指标、误差分析、泛化、鲁棒性和部署表现。

拆解的好处是防止“整章都懂一点，但哪块都说不清”。复习时可以逐块追问：它的输入是什么、输出是什么、依赖什么、失败时有什么表现。

### 3. 通俗类比

可以把模型类比成一个需要反复练习的学生：数据是练习题，标签是参考答案，损失函数是扣分规则，优化器是改错方法，验证集像模拟考试，测试集才像正式考试。只在训练集上表现好，就像只会背做过的题。

类比不是严格定义，但能帮助初学者先建立直觉。真正使用时，还要回到术语、公式、接口、数据结构、时序图或工程规范上，把“感觉理解”变成“可验证理解”。

### 4. 具体例子

学习《04-传统图像处理算法精讲》时，可以准备一个很小的数据集，先手工看 20 个样本和标签，再训练一个基线模型，最后把预测错误的样本逐张分析。很多模型问题不是公式错了，而是数据、标注或评价方式一开始就没有定义清楚。

讲义级学习不能只停留在“概念解释”。至少要有一个能跑、能算、能画或能检查的例子。例子越小，越容易看清关键机制；等机制清楚后，再逐步扩展到复杂项目。

### 5. 学习路径

- 先明确任务类型：分类、回归、检测、分割、生成、序列建模或决策。
- 再梳理数据、特征、模型、损失函数、优化方法和评价指标。
- 最后通过误差分析定位问题来自数据、标注、模型容量、训练策略还是部署环境。

建议每学完一小节都做一次“复述练习”：不用看笔记，用自己的话讲清楚概念、输入、输出、关键步骤和常见错误。如果讲不清，通常说明还没有真正掌握。

### 8. 常见误区

- 只追求模型结构新，不检查数据质量和标注一致性。
- 只看平均指标，不看错误样本、长尾类别和业务代价。
- 把训练集效果当成真实效果，忽略验证集、测试集和线上分布变化。

遇到这些问题时，不要急着背更多资料。更有效的办法是回到一个最小例子，把输入、状态变化、输出和验证方式重新走一遍。
