## 讲义导读：从像素到可用视觉系统

这一章讲的是 **04-传统图像处理算法精讲**。阅读时不要只记 OpenCV 函数、模型名字或论文缩写，而要把它放进完整视觉链路：图像如何采集，像素和颜色空间是否正确，光照和噪声如何影响结果，相机几何是否成立，模型输入和标注是否一致，评价指标是否对应漏检误检成本，部署时前后处理、延迟和现场漂移是否可控。CV 的目标不是让一张样例图看起来成功，而是在真实图像分布中稳定输出可用信息。

### 一句话先懂

传统 CV 算法用明确规则处理图像，例如边缘、阈值、形态、霍夫、模板和分水岭，适合可解释且条件稳定的场景。

### 通俗类比

可以把 CV 系统想成流水线上的质检员：相机和光源负责把产品拍清楚，预处理像擦干净镜头和统一灯光，传统特征或神经网络负责看重点，检测和分割负责指出位置，后处理负责合并和过滤结果，评估指标负责统计漏检、误检和定位偏差，部署监控负责发现现场条件变化。

类比只是第一层直觉。回到技术上，要把每一步写成明确对象：输入图像的尺寸、通道、数据类型、坐标系、标注格式、模型输出、阈值、指标和运行环境。视觉问题很容易被表面效果迷惑，必须用可复现样本和量化指标来判断。

### 本章学习主线

1. **图像来源**：确认采集设备、光照、分辨率、帧率、压缩、畸变和场景覆盖。
2. **表示方式**：弄清像素数组、通道顺序、颜色空间、坐标系、数据类型和归一化。
3. **算法假设**：说明方法依赖什么条件，例如边缘明显、光照稳定、尺度不变、训练分布一致或标注可靠。
4. **输出解释**：区分类别、框、mask、关键点、文字、轨迹、三维点和置信度的含义。
5. **评估诊断**：用 IoU、mAP、Dice、OCR 准确率、MOT 指标、延迟和错误样本定位问题。
6. **工程闭环**：把训练、推理、后处理、可视化、日志、监控和现场迭代连起来。

### 概念怎么学才不容易忘

遇到视觉算法时，建议按 白话作用 -> 输入输出 -> 图像假设 -> 参数含义 -> 可视化结果 -> 失败样本 六步学习。比如 Canny 不只是一个函数，要看梯度、阈值和噪声；YOLO 不只是一个模型，要看输入尺寸、置信度、NMS、mAP 和小目标漏检；相机标定不只是拿到内参，要看重投影误差和畸变是否影响实际测量。

### 最小实践任务

用一张工业零件图依次做边缘检测、阈值分割、形态学处理和轮廓筛选，记录每个参数的影响。

实践时要保留失败样本：反光、模糊、遮挡、过曝、欠曝、背景相似、标注不一致、小目标、长尾类别、视频抖动、部署前后处理不一致。这些样本比单张成功截图更能说明系统边界。

## 1. 传统图像处理的价值

深度学习很强，但传统图像处理仍然重要，尤其在以下场景：

- 目标形状规则。
- 光照和背景可控。
- 数据量少。
- 需要高可解释性。
- 需要精确测量。
- 算力有限。
- 工业视觉项目快速交付。

传统算法常用流程：

```text
读图 -> 预处理 -> 阈值/边缘 -> 形态学 -> 轮廓/连通域 -> 特征计算 -> 规则判断
```

## 2. 阈值分割

阈值分割根据像素灰度或颜色把目标和背景分开。

适用条件：

- 目标和背景差异明显。
- 光照稳定。
- 图像噪声较少。

常见方法：

- 固定阈值。
- Otsu。
- 自适应阈值。
- HSV 颜色阈值。

阈值分割优点：

- 简单。
- 快速。
- 可解释。

缺点：

- 对光照敏感。
- 对阴影、反光敏感。
- 对背景复杂场景效果差。

## 3. 边缘检测

边缘是图像灰度变化剧烈的位置。

### 3.1 Sobel

Sobel 计算水平和垂直方向梯度。

```python
gx = cv2.Sobel(gray, cv2.CV_64F, 1, 0, ksize=3)
gy = cv2.Sobel(gray, cv2.CV_64F, 0, 1, ksize=3)
```

适合检测方向边缘。

### 3.2 Laplacian

二阶导数边缘检测，对噪声敏感。

```python
edge = cv2.Laplacian(gray, cv2.CV_64F)
```

### 3.3 Canny

Canny 是经典边缘检测算法。

流程：

1. 高斯滤波去噪。
2. 计算梯度幅值和方向。
3. 非极大值抑制。
4. 双阈值筛选。
5. 边缘连接。

```python
edges = cv2.Canny(gray, 50, 150)
```

参数：

- 低阈值。
- 高阈值。

经验：

- 高阈值控制强边缘。
- 低阈值控制边缘连接。
- 阈值太低会有很多噪声。
- 阈值太高会断边。

## 4. 轮廓检测

轮廓是二值图中连续目标边界。

```python
contours, hierarchy = cv2.findContours(binary, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)
```

常用特征：

- 面积。
- 周长。
- 外接矩形。
- 最小外接矩形。
- 最小外接圆。
- 质心。
- 方向角。
- 凸包。
- 轮廓近似。

示例：

```python
area = cv2.contourArea(cnt)
x, y, w, h = cv2.boundingRect(cnt)
M = cv2.moments(cnt)
cx = int(M["m10"] / M["m00"])
cy = int(M["m01"] / M["m00"])
```

应用：

- 计数。
- 定位。
- 尺寸测量。
- 形状判断。
- 缺陷区域提取。

## 5. 连通域分析

连通域分析用于找二值图中的独立区域。

```python
num_labels, labels, stats, centroids = cv2.connectedComponentsWithStats(binary)
```

stats 包含：

- x。
- y。
- width。
- height。
- area。

连通域适合做目标计数和小区域过滤。

## 6. 形态学算法

形态学基于结构元素处理二值图或灰度图。

### 6.1 腐蚀

腐蚀会缩小白色区域。

用途：

- 去除小白噪声。
- 分离细连接。
- 缩小目标。

### 6.2 膨胀

膨胀会扩大白色区域。

用途：

- 填补小缺口。
- 连接断裂区域。
- 扩大目标。

### 6.3 开运算

先腐蚀后膨胀。

用途：

- 去除孤立噪点。

### 6.4 闭运算

先膨胀后腐蚀。

用途：

- 填补小孔洞。
- 连接近邻区域。

## 7. 霍夫变换

### 7.1 直线检测

霍夫直线变换把图像空间的点映射到参数空间，寻找共线点。

```python
lines = cv2.HoughLinesP(edges, 1, np.pi / 180, threshold=80,
                        minLineLength=50, maxLineGap=10)
```

参数：

- threshold：投票阈值。
- minLineLength：最短直线长度。
- maxLineGap：允许断裂间隔。

应用：

- 车道线。
- 边框检测。
- 工件直线边缘。
- 表格线检测。

### 7.2 圆检测

```python
circles = cv2.HoughCircles(gray, cv2.HOUGH_GRADIENT, dp=1.2,
                           minDist=30, param1=100, param2=30,
                           minRadius=10, maxRadius=100)
```

应用：

- 圆孔。
- 螺丝。
- 瓶盖。
- 圆形零件。

## 8. 模板匹配

模板匹配在大图中寻找与模板相似的区域。

```python
result = cv2.matchTemplate(img, template, cv2.TM_CCOEFF_NORMED)
_, max_val, _, max_loc = cv2.minMaxLoc(result)
```

适用：

- 目标外观固定。
- 尺度和旋转变化小。
- 背景可控。

缺点：

- 对尺度变化敏感。
- 对旋转敏感。
- 对光照变化敏感。

改进：

- 多尺度模板匹配。
- 旋转模板。
- 边缘模板。
- 特征匹配。

## 9. 传统算法项目案例：检测圆形零件

流程：

```text
灰度化 -> 高斯滤波 -> Canny -> 霍夫圆 -> 圆心和半径输出 -> 规则过滤
```

需要考虑：

- 光照是否均匀。
- 圆边缘是否清晰。
- 是否存在反光。
- 是否有遮挡。
- 圆半径范围是否已知。

## 10. 传统算法失效原因

- 光照变化。
- 背景复杂。
- 目标姿态变化大。
- 目标外观不一致。
- 噪声严重。
- 反光和阴影。
- 参数对场景过拟合。

工程建议：

- 能控制硬件就先控制硬件。
- 固定光源、背景、相机位置。
- 使用 ROI 降低干扰。
- 给参数留调节入口。
- 用可视化工具显示每一步中间结果。
