## 讲义导读：从像素到可用视觉系统

这一章讲的是 **05-特征提取、匹配与图像检索**。阅读时不要只记 OpenCV 函数、模型名字或论文缩写，而要把它放进完整视觉链路：图像如何采集，像素和颜色空间是否正确，光照和噪声如何影响结果，相机几何是否成立，模型输入和标注是否一致，评价指标是否对应漏检误检成本，部署时前后处理、延迟和现场漂移是否可控。CV 的目标不是让一张样例图看起来成功，而是在真实图像分布中稳定输出可用信息。

### 一句话先懂

特征提取和匹配是在图像中找可重复识别的局部线索，用来做配准、检索、拼接、定位和三维重建。

### 通俗类比

可以把 CV 系统想成流水线上的质检员：相机和光源负责把产品拍清楚，预处理像擦干净镜头和统一灯光，传统特征或神经网络负责看重点，检测和分割负责指出位置，后处理负责合并和过滤结果，评估指标负责统计漏检、误检和定位偏差，部署监控负责发现现场条件变化。

类比只是第一层直觉。回到技术上，要把每一步写成明确对象：输入图像的尺寸、通道、数据类型、坐标系、标注格式、模型输出、阈值、指标和运行环境。视觉问题很容易被表面效果迷惑，必须用可复现样本和量化指标来判断。

### 本章学习主线

1. **图像来源**：确认采集设备、光照、分辨率、帧率、压缩、畸变和场景覆盖。
2. **表示方式**：弄清像素数组、通道顺序、颜色空间、坐标系、数据类型和归一化。
3. **算法假设**：说明方法依赖什么条件，例如边缘明显、光照稳定、尺度不变、训练分布一致或标注可靠。
4. **输出解释**：区分类别、框、mask、关键点、文字、轨迹、三维点和置信度的含义。
5. **评估诊断**：用 IoU、mAP、Dice、OCR 准确率、MOT 指标、延迟和错误样本定位问题。
6. **工程闭环**：把训练、推理、后处理、可视化、日志、监控和现场迭代连起来。

### 概念怎么学才不容易忘

遇到视觉算法时，建议按 白话作用 -> 输入输出 -> 图像假设 -> 参数含义 -> 可视化结果 -> 失败样本 六步学习。比如 Canny 不只是一个函数，要看梯度、阈值和噪声；YOLO 不只是一个模型，要看输入尺寸、置信度、NMS、mAP 和小目标漏检；相机标定不只是拿到内参，要看重投影误差和畸变是否影响实际测量。

## 1. 什么是图像特征

图像特征是从图像中提取的可区分信息，用来描述目标、局部区域或整张图像。

好的特征应具备：

- 可重复：同一目标在不同图像中能被检测到。
- 可区分：不同目标特征差异明显。
- 鲁棒：对光照、尺度、旋转、噪声有一定稳定性。
- 高效：计算和匹配速度可接受。

## 2. 角点

角点是图像中两个方向灰度变化都很明显的位置。

常见角点：

- 物体边角。
- 棋盘格交点。
- 纹理交叉处。

### 2.1 Harris 角点

Harris 根据局部窗口内梯度变化判断角点。

适用：

- 相机标定棋盘格。
- 图像配准。
- 运动跟踪。

### 2.2 Shi-Tomasi

Shi-Tomasi 改进 Harris，常用于光流跟踪。

```python
corners = cv2.goodFeaturesToTrack(gray, maxCorners=100, qualityLevel=0.01, minDistance=10)
```

## 3. SIFT

SIFT 是经典局部特征算法。

特点：

- 尺度不变。
- 旋转不变。
- 区分性强。
- 对光照变化有一定鲁棒性。

流程：

1. 构建尺度空间。
2. DoG 检测关键点。
3. 关键点定位。
4. 分配主方向。
5. 构建描述子。

适用：

- 图像拼接。
- 目标匹配。
- 三维重建。
- 视觉定位。

缺点：

- 计算较慢。

## 4. ORB

ORB 是快速特征算法，结合 FAST 关键点和 BRIEF 描述子，并支持旋转。

```python
orb = cv2.ORB_create()
kp, des = orb.detectAndCompute(gray, None)
```

优点：

- 快。
- 免费可用。
- 适合实时场景。

缺点：

- 区分性通常弱于 SIFT。

## 5. 特征匹配

### 5.1 暴力匹配 BFMatcher

```python
bf = cv2.BFMatcher(cv2.NORM_HAMMING, crossCheck=True)
matches = bf.match(des1, des2)
matches = sorted(matches, key=lambda x: x.distance)
```

ORB 常用 Hamming 距离。

### 5.2 FLANN

适合大量特征快速近似匹配。

### 5.3 Lowe Ratio Test

保留最佳匹配明显优于次佳匹配的点。

```text
如果 best_distance < ratio * second_distance，则认为匹配可靠
```

常用 ratio：0.7 到 0.8。

## 6. RANSAC

匹配点中会有误匹配。RANSAC 用随机采样估计模型并剔除外点。

常见用途：

- 单应性矩阵估计。
- 基础矩阵估计。
- 图像配准。
- 目标定位。

```python
H, mask = cv2.findHomography(src_pts, dst_pts, cv2.RANSAC, 5.0)
```

## 7. 图像拼接

基本流程：

```text
特征提取 -> 特征匹配 -> RANSAC 求单应性 -> 图像变换 -> 融合
```

关键问题：

- 重叠区域足够。
- 特征数量足够。
- 视角变化不能太大。
- 曝光差异需要融合处理。

## 8. 图像检索

图像检索是从图库中找到相似图像。

传统方法：

- 颜色直方图。
- SIFT/ORB + BoW。
- HOG。

深度学习方法：

- CNN 特征。
- CLIP 向量。
- 度量学习。
- 向量数据库。

流程：

```text
图片 -> 特征向量 -> 向量索引 -> 相似度搜索 -> 返回 TopK
```

常用相似度：

- 欧氏距离。
- 余弦相似度。
- 内积。

## 9. 特征方法适用场景

适合：

- 平面目标匹配。
- 图像配准。
- 拼接。
- 视觉定位。
- 少样本识别。

不适合：

- 纹理很少的物体。
- 大面积重复纹理。
- 光照剧烈变化。
- 非刚体变化大。

## 10. 实战建议

- 先检查图像是否清晰。
- 特征点要足够多且分布均匀。
- 匹配后必须做误匹配剔除。
- 用可视化画出匹配线。
- 对实时场景优先 ORB。
- 对准确率优先 SIFT 或深度特征。
