# 05-特征提取、匹配与图像检索

## 1. 什么是图像特征

图像特征是从图像中提取的可区分信息，用来描述目标、局部区域或整张图像。

好的特征应具备：

- 可重复：同一目标在不同图像中能被检测到。
- 可区分：不同目标特征差异明显。
- 鲁棒：对光照、尺度、旋转、噪声有一定稳定性。
- 高效：计算和匹配速度可接受。

## 2. 角点

角点是图像中两个方向灰度变化都很明显的位置。

常见角点：

- 物体边角。
- 棋盘格交点。
- 纹理交叉处。

### 2.1 Harris 角点

Harris 根据局部窗口内梯度变化判断角点。

适用：

- 相机标定棋盘格。
- 图像配准。
- 运动跟踪。

### 2.2 Shi-Tomasi

Shi-Tomasi 改进 Harris，常用于光流跟踪。

```python
corners = cv2.goodFeaturesToTrack(gray, maxCorners=100, qualityLevel=0.01, minDistance=10)
```

## 3. SIFT

SIFT 是经典局部特征算法。

特点：

- 尺度不变。
- 旋转不变。
- 区分性强。
- 对光照变化有一定鲁棒性。

流程：

1. 构建尺度空间。
2. DoG 检测关键点。
3. 关键点定位。
4. 分配主方向。
5. 构建描述子。

适用：

- 图像拼接。
- 目标匹配。
- 三维重建。
- 视觉定位。

缺点：

- 计算较慢。

## 4. ORB

ORB 是快速特征算法，结合 FAST 关键点和 BRIEF 描述子，并支持旋转。

```python
orb = cv2.ORB_create()
kp, des = orb.detectAndCompute(gray, None)
```

优点：

- 快。
- 免费可用。
- 适合实时场景。

缺点：

- 区分性通常弱于 SIFT。

## 5. 特征匹配

### 5.1 暴力匹配 BFMatcher

```python
bf = cv2.BFMatcher(cv2.NORM_HAMMING, crossCheck=True)
matches = bf.match(des1, des2)
matches = sorted(matches, key=lambda x: x.distance)
```

ORB 常用 Hamming 距离。

### 5.2 FLANN

适合大量特征快速近似匹配。

### 5.3 Lowe Ratio Test

保留最佳匹配明显优于次佳匹配的点。

```text
如果 best_distance < ratio * second_distance，则认为匹配可靠
```

常用 ratio：0.7 到 0.8。

## 6. RANSAC

匹配点中会有误匹配。RANSAC 用随机采样估计模型并剔除外点。

常见用途：

- 单应性矩阵估计。
- 基础矩阵估计。
- 图像配准。
- 目标定位。

```python
H, mask = cv2.findHomography(src_pts, dst_pts, cv2.RANSAC, 5.0)
```

## 7. 图像拼接

基本流程：

```text
特征提取 -> 特征匹配 -> RANSAC 求单应性 -> 图像变换 -> 融合
```

关键问题：

- 重叠区域足够。
- 特征数量足够。
- 视角变化不能太大。
- 曝光差异需要融合处理。

## 8. 图像检索

图像检索是从图库中找到相似图像。

传统方法：

- 颜色直方图。
- SIFT/ORB + BoW。
- HOG。

深度学习方法：

- CNN 特征。
- CLIP 向量。
- 度量学习。
- 向量数据库。

流程：

```text
图片 -> 特征向量 -> 向量索引 -> 相似度搜索 -> 返回 TopK
```

常用相似度：

- 欧氏距离。
- 余弦相似度。
- 内积。

## 9. 特征方法适用场景

适合：

- 平面目标匹配。
- 图像配准。
- 拼接。
- 视觉定位。
- 少样本识别。

不适合：

- 纹理很少的物体。
- 大面积重复纹理。
- 光照剧烈变化。
- 非刚体变化大。

## 10. 实战建议

- 先检查图像是否清晰。
- 特征点要足够多且分布均匀。
- 匹配后必须做误匹配剔除。
- 用可视化画出匹配线。
- 对实时场景优先 ORB。
- 对准确率优先 SIFT 或深度特征。

## 讲义级补充：如何真正学懂《05-特征提取、匹配与图像检索》

> 适用位置：计算机视觉CV学习笔记\05-特征提取匹配与图像检索.md  
> 说明：本补充用于把原始提纲扩展成课堂讲义式学习材料。阅读时建议先看原文，再用本节建立知识框架、例子、实践和自测闭环。

### 1. 这一讲要解决什么问题

这类内容的核心不是公式本身，而是“数据如何变成可学习的表示，模型如何产生预测，误差如何反馈改进”。学习时要把数据、假设、损失、优化、评估和部署放在一起理解。

学习本讲时，可以用三个问题检查自己是否真的理解：

1. 它解决的真实问题是什么？
2. 如果没有它，系统会出现什么具体麻烦？
3. 在真实项目中，应该用什么现象或指标判断它做得好不好？

### 2. 核心知识拆解

可以把本讲拆成几块来学：

- 数据：样本、标签、分布、增强、清洗和划分。
- 模型：假设空间、结构、参数和表达能力。
- 训练：损失函数、优化器、正则化、学习率和收敛。
- 评估：指标、误差分析、泛化、鲁棒性和部署表现。

拆解的好处是防止“整章都懂一点，但哪块都说不清”。复习时可以逐块追问：它的输入是什么、输出是什么、依赖什么、失败时有什么表现。

### 3. 通俗类比

可以把模型类比成一个需要反复练习的学生：数据是练习题，标签是参考答案，损失函数是扣分规则，优化器是改错方法，验证集像模拟考试，测试集才像正式考试。只在训练集上表现好，就像只会背做过的题。

类比不是严格定义，但能帮助初学者先建立直觉。真正使用时，还要回到术语、公式、接口、数据结构、时序图或工程规范上，把“感觉理解”变成“可验证理解”。

### 4. 具体例子

学习《05-特征提取、匹配与图像检索》时，可以准备一个很小的数据集，先手工看 20 个样本和标签，再训练一个基线模型，最后把预测错误的样本逐张分析。很多模型问题不是公式错了，而是数据、标注或评价方式一开始就没有定义清楚。

讲义级学习不能只停留在“概念解释”。至少要有一个能跑、能算、能画或能检查的例子。例子越小，越容易看清关键机制；等机制清楚后，再逐步扩展到复杂项目。

### 5. 学习路径

- 先明确任务类型：分类、回归、检测、分割、生成、序列建模或决策。
- 再梳理数据、特征、模型、损失函数、优化方法和评价指标。
- 最后通过误差分析定位问题来自数据、标注、模型容量、训练策略还是部署环境。

建议每学完一小节都做一次“复述练习”：不用看笔记，用自己的话讲清楚概念、输入、输出、关键步骤和常见错误。如果讲不清，通常说明还没有真正掌握。

可以按下面顺序讲解或复习本主题：

1. 背景：先讲这个知识为什么出现，它试图降低什么成本、解决什么风险或提升什么能力。

### 7. 最小实践任务

为了避免“看懂了但不会用”，建议为本讲配一个最小实践：

- 选一个可以在 30 到 90 分钟内完成的小任务。
- 明确输入、预期输出和验收标准。
- 记录遇到的第一个错误、定位过程和最终修复方法。
- 完成后写 5 行复盘：我原来以为是什么，实际是什么，下次会如何更快处理。

如果本主题偏理论，实践可以是手算一个小例子、画一张流程图、推导一个简化公式或解释一段真实日志；如果偏工程，实践应该尽量落到可运行命令、可测试代码、可检查配置或可测量硬件现象上。

### 8. 常见误区

- 只追求模型结构新，不检查数据质量和标注一致性。
- 只看平均指标，不看错误样本、长尾类别和业务代价。
- 把训练集效果当成真实效果，忽略验证集、测试集和线上分布变化。

遇到这些问题时，不要急着背更多资料。更有效的办法是回到一个最小例子，把输入、状态变化、输出和验证方式重新走一遍。
