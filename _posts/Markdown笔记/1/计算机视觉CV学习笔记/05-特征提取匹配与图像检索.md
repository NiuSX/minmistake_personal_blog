# 05-特征提取、匹配与图像检索

## 1. 什么是图像特征

图像特征是从图像中提取的可区分信息，用来描述目标、局部区域或整张图像。

好的特征应具备：

- 可重复：同一目标在不同图像中能被检测到。
- 可区分：不同目标特征差异明显。
- 鲁棒：对光照、尺度、旋转、噪声有一定稳定性。
- 高效：计算和匹配速度可接受。

## 2. 角点

角点是图像中两个方向灰度变化都很明显的位置。

常见角点：

- 物体边角。
- 棋盘格交点。
- 纹理交叉处。

### 2.1 Harris 角点

Harris 根据局部窗口内梯度变化判断角点。

适用：

- 相机标定棋盘格。
- 图像配准。
- 运动跟踪。

### 2.2 Shi-Tomasi

Shi-Tomasi 改进 Harris，常用于光流跟踪。

```python
corners = cv2.goodFeaturesToTrack(gray, maxCorners=100, qualityLevel=0.01, minDistance=10)
```

## 3. SIFT

SIFT 是经典局部特征算法。

特点：

- 尺度不变。
- 旋转不变。
- 区分性强。
- 对光照变化有一定鲁棒性。

流程：

1. 构建尺度空间。
2. DoG 检测关键点。
3. 关键点定位。
4. 分配主方向。
5. 构建描述子。

适用：

- 图像拼接。
- 目标匹配。
- 三维重建。
- 视觉定位。

缺点：

- 计算较慢。

## 4. ORB

ORB 是快速特征算法，结合 FAST 关键点和 BRIEF 描述子，并支持旋转。

```python
orb = cv2.ORB_create()
kp, des = orb.detectAndCompute(gray, None)
```

优点：

- 快。
- 免费可用。
- 适合实时场景。

缺点：

- 区分性通常弱于 SIFT。

## 5. 特征匹配

### 5.1 暴力匹配 BFMatcher

```python
bf = cv2.BFMatcher(cv2.NORM_HAMMING, crossCheck=True)
matches = bf.match(des1, des2)
matches = sorted(matches, key=lambda x: x.distance)
```

ORB 常用 Hamming 距离。

### 5.2 FLANN

适合大量特征快速近似匹配。

### 5.3 Lowe Ratio Test

保留最佳匹配明显优于次佳匹配的点。

```text
如果 best_distance < ratio * second_distance，则认为匹配可靠
```

常用 ratio：0.7 到 0.8。

## 6. RANSAC

匹配点中会有误匹配。RANSAC 用随机采样估计模型并剔除外点。

常见用途：

- 单应性矩阵估计。
- 基础矩阵估计。
- 图像配准。
- 目标定位。

```python
H, mask = cv2.findHomography(src_pts, dst_pts, cv2.RANSAC, 5.0)
```

## 7. 图像拼接

基本流程：

```text
特征提取 -> 特征匹配 -> RANSAC 求单应性 -> 图像变换 -> 融合
```

关键问题：

- 重叠区域足够。
- 特征数量足够。
- 视角变化不能太大。
- 曝光差异需要融合处理。

## 8. 图像检索

图像检索是从图库中找到相似图像。

传统方法：

- 颜色直方图。
- SIFT/ORB + BoW。
- HOG。

深度学习方法：

- CNN 特征。
- CLIP 向量。
- 度量学习。
- 向量数据库。

流程：

```text
图片 -> 特征向量 -> 向量索引 -> 相似度搜索 -> 返回 TopK
```

常用相似度：

- 欧氏距离。
- 余弦相似度。
- 内积。

## 9. 特征方法适用场景

适合：

- 平面目标匹配。
- 图像配准。
- 拼接。
- 视觉定位。
- 少样本识别。

不适合：

- 纹理很少的物体。
- 大面积重复纹理。
- 光照剧烈变化。
- 非刚体变化大。

## 10. 实战建议

- 先检查图像是否清晰。
- 特征点要足够多且分布均匀。
- 匹配后必须做误匹配剔除。
- 用可视化画出匹配线。
- 对实时场景优先 ORB。
- 对准确率优先 SIFT 或深度特征。
