## 讲义导读：从像素到可用视觉系统

这一章讲的是 **06-几何视觉、相机标定与三维重建**。阅读时不要只记 OpenCV 函数、模型名字或论文缩写，而要把它放进完整视觉链路：图像如何采集，像素和颜色空间是否正确，光照和噪声如何影响结果，相机几何是否成立，模型输入和标注是否一致，评价指标是否对应漏检误检成本，部署时前后处理、延迟和现场漂移是否可控。CV 的目标不是让一张样例图看起来成功，而是在真实图像分布中稳定输出可用信息。

### 一句话先懂

几何视觉关心图像和三维世界的关系：相机怎么成像，像素如何对应射线，多视角如何恢复深度。

### 通俗类比

可以把 CV 系统想成流水线上的质检员：相机和光源负责把产品拍清楚，预处理像擦干净镜头和统一灯光，传统特征或神经网络负责看重点，检测和分割负责指出位置，后处理负责合并和过滤结果，评估指标负责统计漏检、误检和定位偏差，部署监控负责发现现场条件变化。

类比只是第一层直觉。回到技术上，要把每一步写成明确对象：输入图像的尺寸、通道、数据类型、坐标系、标注格式、模型输出、阈值、指标和运行环境。视觉问题很容易被表面效果迷惑，必须用可复现样本和量化指标来判断。

### 本章学习主线

1. **图像来源**：确认采集设备、光照、分辨率、帧率、压缩、畸变和场景覆盖。
2. **表示方式**：弄清像素数组、通道顺序、颜色空间、坐标系、数据类型和归一化。
3. **算法假设**：说明方法依赖什么条件，例如边缘明显、光照稳定、尺度不变、训练分布一致或标注可靠。
4. **输出解释**：区分类别、框、mask、关键点、文字、轨迹、三维点和置信度的含义。
5. **评估诊断**：用 IoU、mAP、Dice、OCR 准确率、MOT 指标、延迟和错误样本定位问题。
6. **工程闭环**：把训练、推理、后处理、可视化、日志、监控和现场迭代连起来。

## 1. 几何视觉研究什么

几何视觉关注图像和三维世界之间的几何关系。它回答：

- 三维点如何投影到图像。
- 相机内参和外参是什么。
- 镜头畸变如何修正。
- 两张图之间如何建立对应关系。
- 如何从多张图恢复三维结构。
- 如何估计相机位姿。

## 2. 针孔相机模型

针孔模型把三维点投影到二维图像平面。

基本关系：

```text
s * [u, v, 1]^T = K * [R | t] * [X, Y, Z, 1]^T
```

其中：

- `(X, Y, Z)` 是世界坐标。
- `(u, v)` 是图像像素坐标。
- `K` 是相机内参。
- `R, t` 是相机外参。
- `s` 是尺度因子。

## 3. 相机内参

内参矩阵：

```text
K = [fx  0 cx
     0  fy cy
     0   0  1]
```

含义：

- `fx, fy`：焦距换算到像素单位。
- `cx, cy`：主点坐标。

内参由相机和镜头决定，同一焦距下通常固定。

## 4. 相机外参

外参描述相机相对世界坐标系的位置和姿态：

- `R`：旋转矩阵。
- `t`：平移向量。

工业视觉中，外参用于把图像坐标转换到机械坐标、机器人坐标或世界坐标。

## 5. 镜头畸变

常见畸变：

- 径向畸变：桶形、枕形。
- 切向畸变：镜头安装不完全平行导致。

畸变会导致直线变弯，影响测量和定位。

OpenCV 畸变参数常见为：

```text
k1, k2, p1, p2, k3
```

## 6. 相机标定

相机标定用于求内参和畸变参数。

常用标定板：

- 棋盘格。
- 圆点板。
- AprilTag。

OpenCV 棋盘格标定流程：

```text
采集多张棋盘格图 -> 检测角点 -> 准备世界坐标 -> calibrateCamera -> 得到内参和畸变 -> 重投影误差评估
```

关键注意：

- 标定板要覆盖画面不同位置。
- 姿态要多样。
- 图片要清晰。
- 棋盘格尺寸要准确。
- 重投影误差越小越好，但也要看实际测量效果。

## 7. 图像去畸变

```python
undistorted = cv2.undistort(img, camera_matrix, dist_coeffs)
```

去畸变后：

- 直线更直。
- 图像边缘可能裁剪。
- 测量更准确。

## 8. 单应性矩阵

单应性 Homography 描述同一平面在两张图之间的投影关系。

应用：

- 透视矫正。
- 文档矫正。
- 平面目标定位。
- 鸟瞰图变换。
- 图像拼接。

```python
H, mask = cv2.findHomography(src_pts, dst_pts, cv2.RANSAC)
warped = cv2.warpPerspective(img, H, (w, h))
```

条件：

- 点在同一平面上，或相机纯旋转。

## 9. PnP 位姿估计

PnP 用三维点和对应二维图像点估计相机位姿。

应用：

- AR 标记定位。
- 机器人抓取定位。
- 相机外参求解。

```python
success, rvec, tvec = cv2.solvePnP(object_points, image_points, K, dist)
```

需要：

- 已知 3D 点坐标。
- 对应图像 2D 点。
- 相机内参。

## 10. 双目视觉

双目通过左右相机视差估计深度。

流程：

```text
双目标定 -> 极线校正 -> 立体匹配 -> 视差图 -> 深度图
```

深度关系：

```text
Z = f * B / disparity
```

其中：

- `f` 是焦距。
- `B` 是基线距离。
- `disparity` 是视差。

问题：

- 纹理少区域匹配困难。
- 反光和透明物体困难。
- 远距离视差小，深度误差大。

## 11. 三维重建

常见方法：

- 双目重建。
- 多视图重建 SfM。
- 结构光。
- ToF 深度相机。
- 激光雷达融合。

三维输出：

- 点云。
- 网格。
- 深度图。
- TSDF。

## 12. 工业视觉中的手眼标定

手眼标定用于求相机坐标系和机器人坐标系之间的关系。

常见形式：

- 眼在手上：相机安装在机器人末端。
- 眼在手外：相机固定在外部。

关键：

- 标定动作要覆盖足够姿态变化。
- 机械重复精度会影响结果。
- 标定板检测精度很重要。
