## 讲义导读：从像素到可用视觉系统

这一章讲的是 **07-深度学习与 CNN 视觉基础**。阅读时不要只记 OpenCV 函数、模型名字或论文缩写，而要把它放进完整视觉链路：图像如何采集，像素和颜色空间是否正确，光照和噪声如何影响结果，相机几何是否成立，模型输入和标注是否一致，评价指标是否对应漏检误检成本，部署时前后处理、延迟和现场漂移是否可控。CV 的目标不是让一张样例图看起来成功，而是在真实图像分布中稳定输出可用信息。

### 一句话先懂

CNN 和深度视觉模型用可学习卷积、特征层级和大数据训练，从像素中自动提取更复杂的视觉表示。

### 通俗类比

可以把 CV 系统想成流水线上的质检员：相机和光源负责把产品拍清楚，预处理像擦干净镜头和统一灯光，传统特征或神经网络负责看重点，检测和分割负责指出位置，后处理负责合并和过滤结果，评估指标负责统计漏检、误检和定位偏差，部署监控负责发现现场条件变化。

类比只是第一层直觉。回到技术上，要把每一步写成明确对象：输入图像的尺寸、通道、数据类型、坐标系、标注格式、模型输出、阈值、指标和运行环境。视觉问题很容易被表面效果迷惑，必须用可复现样本和量化指标来判断。

### 本章学习主线

1. **图像来源**：确认采集设备、光照、分辨率、帧率、压缩、畸变和场景覆盖。
2. **表示方式**：弄清像素数组、通道顺序、颜色空间、坐标系、数据类型和归一化。
3. **算法假设**：说明方法依赖什么条件，例如边缘明显、光照稳定、尺度不变、训练分布一致或标注可靠。
4. **输出解释**：区分类别、框、mask、关键点、文字、轨迹、三维点和置信度的含义。
5. **评估诊断**：用 IoU、mAP、Dice、OCR 准确率、MOT 指标、延迟和错误样本定位问题。
6. **工程闭环**：把训练、推理、后处理、可视化、日志、监控和现场迭代连起来。

### 概念怎么学才不容易忘

遇到视觉算法时，建议按 白话作用 -> 输入输出 -> 图像假设 -> 参数含义 -> 可视化结果 -> 失败样本 六步学习。比如 Canny 不只是一个函数，要看梯度、阈值和噪声；YOLO 不只是一个模型，要看输入尺寸、置信度、NMS、mAP 和小目标漏检；相机标定不只是拿到内参，要看重投影误差和畸变是否影响实际测量。

### 最小实践任务

训练或运行一个小型分类模型，观察输入尺寸、归一化、卷积特征图、训练曲线和验证集错误样本。

实践时要保留失败样本：反光、模糊、遮挡、过曝、欠曝、背景相似、标注不一致、小目标、长尾类别、视频抖动、部署前后处理不一致。这些样本比单张成功截图更能说明系统边界。

## 1. 为什么深度学习适合视觉

传统视觉依赖人工设计特征，而深度学习可以从数据中自动学习特征。CNN 能从低层边缘、纹理逐步学习到高层语义。

典型层级：

```text
边缘 -> 纹理 -> 局部形状 -> 目标部件 -> 物体类别
```

## 2. 神经网络基本概念

核心组成：

- 输入。
- 参数。
- 前向传播。
- 损失函数。
- 反向传播。
- 优化器。
- 训练集、验证集、测试集。

训练目标是让模型预测结果接近真实标签。

## 3. 卷积

卷积层用卷积核在图像上滑动，提取局部特征。

参数：

- kernel size：卷积核大小。
- stride：步长。
- padding：填充。
- channels：通道数。

卷积优点：

- 局部连接。
- 参数共享。
- 平移等变。

## 4. 池化

池化用于降低特征图尺寸。

常见：

- Max Pooling。
- Average Pooling。

作用：

- 降低计算量。
- 增大感受野。
- 提供一定平移鲁棒性。

## 5. 激活函数

常见激活：

- ReLU。
- Leaky ReLU。
- GELU。
- SiLU/Swish。

激活函数提供非线性，否则多层线性变换等价于一层线性变换。

## 6. 归一化

常见：

- BatchNorm。
- LayerNorm。
- GroupNorm。

作用：

- 稳定训练。
- 加快收敛。
- 缓解梯度问题。

## 7. 经典 CNN

### 7.1 LeNet

早期 CNN，用于手写数字识别。

### 7.2 AlexNet

推动深度学习视觉爆发，引入 ReLU、Dropout、大规模 GPU 训练。

### 7.3 VGG

使用大量 3x3 卷积，结构简单，但参数量大。

### 7.4 ResNet

引入残差连接：

```text
y = F(x) + x
```

解决深层网络退化问题，是视觉模型重要基础。

### 7.5 MobileNet

使用深度可分离卷积，适合移动端和边缘设备。

## 8. Vision Transformer

ViT 把图像分成 patch，当作 token 输入 Transformer。

特点：

- 全局建模能力强。
- 大数据下效果好。
- 计算成本较高。

常见模型：

- ViT。
- Swin Transformer。
- DeiT。

## 9. 迁移学习

迁移学习使用预训练模型作为基础，在自定义数据上微调。

好处：

- 降低数据需求。
- 加快收敛。
- 提高性能。

常见做法：

- 冻结 backbone，只训练分类头。
- 解冻部分层微调。
- 全量微调。

## 10. 损失函数

分类：

- Cross Entropy。
- Focal Loss。

回归：

- L1 Loss。
- MSE。
- Smooth L1。

分割：

- Dice Loss。
- BCE。
- Cross Entropy。
- Lovasz Loss。

检测：

- 分类损失。
- 边框回归损失。
- IoU/GIoU/DIoU/CIoU。

## 11. 优化器

常见：

- SGD。
- Adam。
- AdamW。

学习率非常关键。过大可能发散，过小收敛慢。

常见学习率策略：

- StepLR。
- Cosine Annealing。
- Warmup。
- OneCycle。

## 12. 过拟合和欠拟合

过拟合：

- 训练集好，验证集差。
- 模型记住训练样本，不泛化。

解决：

- 增加数据。
- 数据增强。
- 正则化。
- Dropout。
- 降低模型复杂度。
- 早停。

欠拟合：

- 训练集和验证集都差。

解决：

- 增大模型。
- 训练更久。
- 提高输入质量。
- 调整学习率。
- 检查标签。

## 13. 深度学习视觉训练流程

```text
收集数据 -> 清洗数据 -> 划分数据集 -> 数据增强 -> 选择模型 -> 训练 -> 验证 -> 误差分析 -> 调参 -> 导出部署
```

训练前优先检查：

- 标签是否正确。
- 类别是否平衡。
- 输入尺寸是否合适。
- 训练和推理预处理是否一致。
- 指标是否符合业务目标。

## 2026 CV 资料与工程核对补充

计算机视觉的基础概念稳定，但模型生态、部署工具和数据集规范变化很快。复现实验前建议记录 Python、OpenCV、PyTorch、TorchVision、CUDA、cuDNN、ONNX Runtime、TensorRT、Ultralytics 或相关库版本，同时记录输入尺寸、颜色空间、归一化方式、类别表、标注格式、阈值、NMS 参数和硬件环境。

视觉项目尤其要防止训练和部署前后处理不一致。常见问题包括 BGR/RGB 搞反、归一化均值方差不同、resize 与 letterbox 不一致、坐标缩放错误、类别 id 对不上、置信度阈值和 NMS 阈值变化、摄像头现场光照不同。
