# 08-图像分类、目标检测与实例分割

## 讲义导读：从像素到可用视觉系统

这一章讲的是 **08-图像分类、目标检测与实例分割**。阅读时不要只记 OpenCV 函数、模型名字或论文缩写，而要把它放进完整视觉链路：图像如何采集，像素和颜色空间是否正确，光照和噪声如何影响结果，相机几何是否成立，模型输入和标注是否一致，评价指标是否对应漏检误检成本，部署时前后处理、延迟和现场漂移是否可控。CV 的目标不是让一张样例图看起来成功，而是在真实图像分布中稳定输出可用信息。

### 一句话先懂

分类回答图里有什么，检测回答目标在哪里，实例分割进一步回答每个目标覆盖哪些像素。

### 通俗类比

可以把 CV 系统想成流水线上的质检员：相机和光源负责把产品拍清楚，预处理像擦干净镜头和统一灯光，传统特征或神经网络负责看重点，检测和分割负责指出位置，后处理负责合并和过滤结果，评估指标负责统计漏检、误检和定位偏差，部署监控负责发现现场条件变化。

类比只是第一层直觉。回到技术上，要把每一步写成明确对象：输入图像的尺寸、通道、数据类型、坐标系、标注格式、模型输出、阈值、指标和运行环境。视觉问题很容易被表面效果迷惑，必须用可复现样本和量化指标来判断。

### 本章学习主线

1. **图像来源**：确认采集设备、光照、分辨率、帧率、压缩、畸变和场景覆盖。
2. **表示方式**：弄清像素数组、通道顺序、颜色空间、坐标系、数据类型和归一化。
3. **算法假设**：说明方法依赖什么条件，例如边缘明显、光照稳定、尺度不变、训练分布一致或标注可靠。
4. **输出解释**：区分类别、框、mask、关键点、文字、轨迹、三维点和置信度的含义。
5. **评估诊断**：用 IoU、mAP、Dice、OCR 准确率、MOT 指标、延迟和错误样本定位问题。
6. **工程闭环**：把训练、推理、后处理、可视化、日志、监控和现场迭代连起来。

### 概念怎么学才不容易忘

遇到视觉算法时，建议按 白话作用 -> 输入输出 -> 图像假设 -> 参数含义 -> 可视化结果 -> 失败样本 六步学习。比如 Canny 不只是一个函数，要看梯度、阈值和噪声；YOLO 不只是一个模型，要看输入尺寸、置信度、NMS、mAP 和小目标漏检；相机标定不只是拿到内参，要看重投影误差和畸变是否影响实际测量。

### 最小实践任务

用一个小数据集训练或推理 YOLO/Mask R-CNN 类模型，画出框、类别、置信度和误检漏检案例。

实践时要保留失败样本：反光、模糊、遮挡、过曝、欠曝、背景相似、标注不一致、小目标、长尾类别、视频抖动、部署前后处理不一致。这些样本比单张成功截图更能说明系统边界。

## 1. 图像分类

图像分类输出整张图的类别。

示例：

```text
输入：一张图片
输出：cat / dog / car
```

适用：

- 图片内容识别。
- 缺陷有无判断。
- 医学影像分类。
- 商品分类。

局限：

- 不给出目标位置。
- 多目标场景不适合只用分类。

## 2. 分类模型

常见：

- ResNet。
- MobileNet。
- EfficientNet。
- ConvNeXt。
- ViT。

评价指标：

- Accuracy。
- Precision。
- Recall。
- F1。
- Confusion Matrix。

类别不平衡时，不要只看 Accuracy。

## 3. 目标检测

目标检测输出：

- 类别。
- 边界框。
- 置信度。

```text
[class, score, x1, y1, x2, y2]
```

适用：

- 行人车辆检测。
- 工业缺陷定位。
- 商品检测。
- 安防目标检测。
- 机器人抓取定位。

## 4. 检测模型分类

### 4.1 Two-stage

代表：

- Faster R-CNN。
- Mask R-CNN。

特点：

- 先生成候选框。
- 再分类和回归。
- 精度高，速度相对慢。

### 4.2 One-stage

代表：

- YOLO。
- SSD。
- RetinaNet。

特点：

- 直接预测类别和框。
- 速度快。
- 工程落地常用。

## 5. YOLO 思想

YOLO 把检测看成一次前向预测问题。

核心：

- 输入固定尺寸图像。
- 网络输出多个尺度预测。
- 每个预测包含类别、置信度、边框。
- NMS 去除重复框。

YOLO 适合实时检测和工业部署。

## 6. Anchor 和 Anchor-Free

Anchor-based：

- 使用预设框。
- 模型预测相对偏移。

Anchor-free：

- 直接预测目标中心点或边界。
- 参数更少，设计更简洁。

## 7. IoU

IoU 是预测框和真实框的交并比。

```text
IoU = 交集面积 / 并集面积
```

用途：

- 判断预测是否命中。
- NMS。
- 损失函数。
- mAP 计算。

## 8. NMS

NMS 非极大值抑制用于删除重复检测框。

流程：

1. 按置信度排序。
2. 选择最高分框。
3. 删除与它 IoU 过高的框。
4. 重复。

问题：

- 密集目标可能误删。
- 可用 Soft-NMS 或 Weighted Boxes Fusion 改进。

## 9. mAP

mAP 是检测常用指标。

相关概念：

- TP：正确检测。
- FP：误检。
- FN：漏检。
- Precision：预测为正中有多少是真的。
- Recall：真实目标中有多少被找到。
- AP：单类别 PR 曲线面积。
- mAP：多类别 AP 平均。

注意：

- mAP 高不一定业务好。
- 工业场景可能更关注漏检率或误报率。

## 10. 小目标检测

小目标难点：

- 特征少。
- 缩放后容易消失。
- 定位误差影响大。

改进：

- 提高输入分辨率。
- 使用更浅层特征。
- 切图检测。
- 数据增强保留小目标。
- 调整 anchor 或检测头。

## 11. 实例分割

实例分割不仅检测目标框，还输出每个目标的像素级 mask。

代表：

- Mask R-CNN。
- YOLACT。
- SOLO。
- YOLO-seg。
- SAM。

应用：

- 精确面积计算。
- 缺陷区域提取。
- 医学病灶。
- 机器人抓取轮廓。

## 12. 检测项目流程

```text
定义类别 -> 采集数据 -> 标注框/mask -> 划分训练验证 -> 训练模型 -> 查看 PR 曲线 -> 误差分析 -> 调参 -> 导出部署
```

误差分析重点：

- 哪些类别混淆。
- 哪些场景漏检。
- 哪些背景误检。
- 小目标是否不足。
- 标注是否一致。
- 置信度阈值是否合理。

## 2026 CV 资料与工程核对补充

计算机视觉的基础概念稳定，但模型生态、部署工具和数据集规范变化很快。复现实验前建议记录 Python、OpenCV、PyTorch、TorchVision、CUDA、cuDNN、ONNX Runtime、TensorRT、Ultralytics 或相关库版本，同时记录输入尺寸、颜色空间、归一化方式、类别表、标注格式、阈值、NMS 参数和硬件环境。

视觉项目尤其要防止训练和部署前后处理不一致。常见问题包括 BGR/RGB 搞反、归一化均值方差不同、resize 与 letterbox 不一致、坐标缩放错误、类别 id 对不上、置信度阈值和 NMS 阈值变化、摄像头现场光照不同。
