# 09-语义分割、关键点与姿态估计

## 讲义导读：从像素到可用视觉系统

这一章讲的是 **09-语义分割、关键点与姿态估计**。阅读时不要只记 OpenCV 函数、模型名字或论文缩写，而要把它放进完整视觉链路：图像如何采集，像素和颜色空间是否正确，光照和噪声如何影响结果，相机几何是否成立，模型输入和标注是否一致，评价指标是否对应漏检误检成本，部署时前后处理、延迟和现场漂移是否可控。CV 的目标不是让一张样例图看起来成功，而是在真实图像分布中稳定输出可用信息。

### 一句话先懂

语义分割按像素理解区域，关键点和姿态估计按结构点描述物体或人体形状。

### 通俗类比

可以把 CV 系统想成流水线上的质检员：相机和光源负责把产品拍清楚，预处理像擦干净镜头和统一灯光，传统特征或神经网络负责看重点，检测和分割负责指出位置，后处理负责合并和过滤结果，评估指标负责统计漏检、误检和定位偏差，部署监控负责发现现场条件变化。

类比只是第一层直觉。回到技术上，要把每一步写成明确对象：输入图像的尺寸、通道、数据类型、坐标系、标注格式、模型输出、阈值、指标和运行环境。视觉问题很容易被表面效果迷惑，必须用可复现样本和量化指标来判断。

### 本章学习主线

1. **图像来源**：确认采集设备、光照、分辨率、帧率、压缩、畸变和场景覆盖。
2. **表示方式**：弄清像素数组、通道顺序、颜色空间、坐标系、数据类型和归一化。
3. **算法假设**：说明方法依赖什么条件，例如边缘明显、光照稳定、尺度不变、训练分布一致或标注可靠。
4. **输出解释**：区分类别、框、mask、关键点、文字、轨迹、三维点和置信度的含义。
5. **评估诊断**：用 IoU、mAP、Dice、OCR 准确率、MOT 指标、延迟和错误样本定位问题。
6. **工程闭环**：把训练、推理、后处理、可视化、日志、监控和现场迭代连起来。

### 概念怎么学才不容易忘

遇到视觉算法时，建议按 白话作用 -> 输入输出 -> 图像假设 -> 参数含义 -> 可视化结果 -> 失败样本 六步学习。比如 Canny 不只是一个函数，要看梯度、阈值和噪声；YOLO 不只是一个模型，要看输入尺寸、置信度、NMS、mAP 和小目标漏检；相机标定不只是拿到内参，要看重投影误差和畸变是否影响实际测量。

### 最小实践任务

对一张图可视化分割 mask 或人体关键点，比较 IoU、Dice、OKS 或 PCK 指标对错误的敏感性。

实践时要保留失败样本：反光、模糊、遮挡、过曝、欠曝、背景相似、标注不一致、小目标、长尾类别、视频抖动、部署前后处理不一致。这些样本比单张成功截图更能说明系统边界。

## 1. 语义分割

语义分割为每个像素分配类别。

示例：

- 道路、天空、车辆、行人。
- 缺陷、背景、边缘。
- 器官、病灶、背景。

输出是与原图同尺寸或可还原尺寸的类别 mask。

## 2. 语义分割和实例分割区别

语义分割：

- 只区分类别。
- 同类目标不区分个体。

实例分割：

- 区分类别。
- 同类目标也区分不同实例。

全景分割：

- 同时处理背景语义和前景实例。

## 3. 经典分割模型

### 3.1 FCN

全卷积网络，把分类网络改成像素级预测。

意义：

- 开启深度学习语义分割。

### 3.2 U-Net

U-Net 使用编码器-解码器结构和跳跃连接。

特点：

- 小数据表现好。
- 医学和工业常用。
- 能融合低层细节和高层语义。

### 3.3 DeepLab

使用空洞卷积和 ASPP 多尺度上下文。

适合自然场景语义分割。

### 3.4 SegFormer

基于 Transformer 的分割网络，结构简洁，效果好。

## 4. 分割损失函数

常见：

- Cross Entropy。
- BCE。
- Dice Loss。
- Focal Loss。
- Lovasz Loss。

Dice 适合前景区域小、类别不平衡的情况。

## 5. 分割评价指标

- Pixel Accuracy。
- mIoU。
- Dice。
- Precision。
- Recall。

IoU：

```text
IoU = 预测区域与真实区域交集 / 并集
```

Dice：

```text
Dice = 2 * 交集 / (预测区域 + 真实区域)
```

## 6. 分割后处理

常见后处理：

- 连通域过滤。
- 小区域删除。
- 孔洞填充。
- 边界平滑。
- 形态学开闭运算。
- CRF。

工业缺陷分割中，经常需要把 mask 转为面积、长度、宽度、最小外接矩形等指标。

## 7. 关键点检测

关键点检测输出目标上的特定点。

应用：

- 人体姿态。
- 人脸关键点。
- 手势识别。
- 工业零件定位。
- 机器人抓取点。

表示方式：

- 坐标回归。
- Heatmap 热力图。

Heatmap 方法通常更稳定。

## 8. 人体姿态估计

任务：

- 检测人体关键点，如肩、肘、腕、髋、膝、踝。
- 连接成骨架。

常见方法：

- Top-down：先检测人，再对每个人估计关键点。
- Bottom-up：先检测所有关键点，再分配到不同人。

模型：

- OpenPose。
- HRNet。
- MediaPipe Pose。
- AlphaPose。

## 9. 关键点评价指标

常见：

- PCK。
- OKS。
- mAP。

关键点评估要考虑目标尺度。同样 5 像素误差，对于大目标和小目标影响不同。

## 10. 分割和关键点常见问题

分割问题：

- 边界不精确。
- 小目标漏分。
- 类别混淆。
- mask 破碎。
- 标注不一致。

关键点问题：

- 遮挡。
- 姿态变化大。
- 多人重叠。
- 模糊。
- 关键点定义不清。

## 11. 实战建议

- 分割标注成本高，要制定清晰标注规范。
- 小缺陷分割要保留高分辨率。
- 不要只看 mIoU，要看业务尺寸误差。
- 关键点定义要稳定，标注人员必须一致。
- 后处理可以显著提升业务稳定性。

## 2026 CV 资料与工程核对补充

计算机视觉的基础概念稳定，但模型生态、部署工具和数据集规范变化很快。复现实验前建议记录 Python、OpenCV、PyTorch、TorchVision、CUDA、cuDNN、ONNX Runtime、TensorRT、Ultralytics 或相关库版本，同时记录输入尺寸、颜色空间、归一化方式、类别表、标注格式、阈值、NMS 参数和硬件环境。

视觉项目尤其要防止训练和部署前后处理不一致。常见问题包括 BGR/RGB 搞反、归一化均值方差不同、resize 与 letterbox 不一致、坐标缩放错误、类别 id 对不上、置信度阈值和 NMS 阈值变化、摄像头现场光照不同。
