# 10-视频分析、目标跟踪与行为识别

## 1. 视频和图像的区别

视频是连续帧序列，不仅有空间信息，还有时间信息。

视频任务需要考虑：

- 帧率。
- 时间连续性。
- 运动模糊。
- 遮挡。
- 目标进出画面。
- 跨帧身份一致性。
- 实时处理速度。

## 2. 视频处理基础

OpenCV 读取视频：

```python
cap = cv2.VideoCapture("video.mp4")
while True:
    ret, frame = cap.read()
    if not ret:
        break
```

摄像头实时读取：

```python
cap = cv2.VideoCapture(0)
```

注意：

- 摄像头延迟。
- 编码解码开销。
- 分辨率和 FPS。
- 多路视频并发。

## 3. 背景建模

背景建模用于检测运动前景。

常见：

- 帧差法。
- MOG2。
- KNN 背景减除。

适用：

- 固定摄像头。
- 背景较稳定。

问题：

- 光照变化。
- 背景晃动。
- 目标静止后融入背景。

## 4. 光流

光流描述像素在相邻帧之间的运动。

常见：

- Lucas-Kanade 稀疏光流。
- Farneback 稠密光流。

应用：

- 运动估计。
- 视频稳定。
- 行为分析。
- 目标跟踪辅助。

## 5. 单目标跟踪

给定第一帧目标位置，后续持续跟踪。

传统跟踪器：

- KCF。
- CSRT。
- MOSSE。

优点：

- 不需要每帧检测。
- 速度快。

缺点：

- 遮挡后容易漂移。
- 目标外观变化大时失效。

## 6. 多目标跟踪 MOT

多目标跟踪需要同时处理检测和身份关联。

基本流程：

```text
每帧目标检测 -> 预测轨迹 -> 关联检测框和已有轨迹 -> 更新轨迹 -> 管理新生和消失目标
```

常见算法：

- SORT。
- DeepSORT。
- ByteTrack。
- OC-SORT。

## 7. SORT

SORT 组成：

- 检测器。
- 卡尔曼滤波预测。
- 匈牙利算法匹配。
- IoU 关联。

优点：

- 简单快速。

缺点：

- 遮挡和交叉时 ID 容易切换。

## 8. DeepSORT

DeepSORT 增加外观特征 ReID，用于改善 ID 保持。

适合：

- 行人跟踪。
- 多目标交叉。

代价：

- 计算更重。
- 需要外观特征模型。

## 9. ByteTrack

ByteTrack 利用高分和低分检测框一起做关联，减少漏检导致的轨迹断裂。

常用于实时多目标跟踪。

## 10. 行为识别

行为识别判断视频中的动作或事件。

方法：

- 2D CNN + 时间池化。
- 3D CNN。
- Two-stream 网络。
- LSTM/GRU。
- Transformer。
- Skeleton-based 方法。

应用：

- 跌倒检测。
- 打架检测。
- 手势识别。
- 工业操作规范检测。
- 体育动作分析。

## 11. 视频事件检测

事件检测关注“什么时候发生了什么”。

示例：

- 人员越界。
- 车辆逆行。
- 物品遗留。
- 烟火检测。
- 设备异常动作。

常见工程方法：

```text
目标检测 -> 跟踪 -> 区域规则 -> 时间规则 -> 事件报警
```

## 12. 视频项目难点

- 实时性。
- 多路并发。
- 漏检导致跟踪断裂。
- 误检导致误报警。
- 摄像头抖动。
- 夜间和强光。
- 遮挡。
- 隐私合规。

## 13. 工程建议

- 用检测器保证基础准确率。
- 用跟踪减少闪烁和重复报警。
- 用时间窗口过滤瞬时误检。
- 用 ROI 区域减少无关检测。
- 保存报警前后视频片段，便于复查。
- 线上记录误报漏报样本，用于持续迭代。

## 讲义级补充：如何真正学懂《10-视频分析、目标跟踪与行为识别》

> 适用位置：计算机视觉CV学习笔记\10-视频分析目标跟踪与行为识别.md  
> 说明：本补充用于把原始提纲扩展成课堂讲义式学习材料。阅读时建议先看原文，再用本节建立知识框架、例子、实践和自测闭环。

### 1. 这一讲要解决什么问题

这类内容的核心不是公式本身，而是“数据如何变成可学习的表示，模型如何产生预测，误差如何反馈改进”。学习时要把数据、假设、损失、优化、评估和部署放在一起理解。

学习本讲时，可以用三个问题检查自己是否真的理解：

1. 它解决的真实问题是什么？
2. 如果没有它，系统会出现什么具体麻烦？
3. 在真实项目中，应该用什么现象或指标判断它做得好不好？

### 2. 核心知识拆解

可以把本讲拆成几块来学：

- 数据：样本、标签、分布、增强、清洗和划分。
- 模型：假设空间、结构、参数和表达能力。
- 训练：损失函数、优化器、正则化、学习率和收敛。
- 评估：指标、误差分析、泛化、鲁棒性和部署表现。

拆解的好处是防止“整章都懂一点，但哪块都说不清”。复习时可以逐块追问：它的输入是什么、输出是什么、依赖什么、失败时有什么表现。

### 3. 通俗类比

可以把模型类比成一个需要反复练习的学生：数据是练习题，标签是参考答案，损失函数是扣分规则，优化器是改错方法，验证集像模拟考试，测试集才像正式考试。只在训练集上表现好，就像只会背做过的题。

类比不是严格定义，但能帮助初学者先建立直觉。真正使用时，还要回到术语、公式、接口、数据结构、时序图或工程规范上，把“感觉理解”变成“可验证理解”。

### 4. 具体例子

学习《10-视频分析、目标跟踪与行为识别》时，可以准备一个很小的数据集，先手工看 20 个样本和标签，再训练一个基线模型，最后把预测错误的样本逐张分析。很多模型问题不是公式错了，而是数据、标注或评价方式一开始就没有定义清楚。

讲义级学习不能只停留在“概念解释”。至少要有一个能跑、能算、能画或能检查的例子。例子越小，越容易看清关键机制；等机制清楚后，再逐步扩展到复杂项目。

### 5. 学习路径

- 先明确任务类型：分类、回归、检测、分割、生成、序列建模或决策。
- 再梳理数据、特征、模型、损失函数、优化方法和评价指标。
- 最后通过误差分析定位问题来自数据、标注、模型容量、训练策略还是部署环境。

建议每学完一小节都做一次“复述练习”：不用看笔记，用自己的话讲清楚概念、输入、输出、关键步骤和常见错误。如果讲不清，通常说明还没有真正掌握。

### 6. 课堂讲解框架

可以按下面顺序讲解或复习本主题：

1. 背景：先讲这个知识为什么出现，它试图降低什么成本、解决什么风险或提升什么能力。
2. 基本概念：给出核心名词的准确定义，说明它们之间的关系。
3. 工作流程：按时间顺序描述一次完整过程，必要时画出流程图、状态机或数据流图。
4. 关键细节：解释最容易误解的机制，例如边界条件、异常处理、性能限制、资源生命周期或安全约束。
5. 实战例子：用一个足够小但完整的例子，把概念落到命令、代码、图纸、配置、数据或操作步骤上。
6. 反例与排错：展示错误做法会导致什么现象，再说明如何定位和修复。
7. 总结迁移：最后说明它和相邻知识点的区别、联系以及后续该学什么。

### 7. 最小实践任务

为了避免“看懂了但不会用”，建议为本讲配一个最小实践：

- 选一个可以在 30 到 90 分钟内完成的小任务。
- 明确输入、预期输出和验收标准。
- 记录遇到的第一个错误、定位过程和最终修复方法。
- 完成后写 5 行复盘：我原来以为是什么，实际是什么，下次会如何更快处理。

如果本主题偏理论，实践可以是手算一个小例子、画一张流程图、推导一个简化公式或解释一段真实日志；如果偏工程，实践应该尽量落到可运行命令、可测试代码、可检查配置或可测量硬件现象上。

### 8. 常见误区

- 只追求模型结构新，不检查数据质量和标注一致性。
- 只看平均指标，不看错误样本、长尾类别和业务代价。
- 把训练集效果当成真实效果，忽略验证集、测试集和线上分布变化。

遇到这些问题时，不要急着背更多资料。更有效的办法是回到一个最小例子，把输入、状态变化、输出和验证方式重新走一遍。

### 9. 自测题

1. 用一句话说明本讲主题解决的核心问题。
2. 列出本讲最重要的 3 个概念，并说明它们的关系。
3. 举一个生活类比，再指出这个类比在哪些地方不严谨。
4. 写出一个最小实践任务的验收标准。
5. 如果结果不符合预期，你会优先检查哪 3 个环节？为什么？
6. 本讲和相邻章节的知识边界是什么？哪些问题应该交给其他章节解决？

### 10. 复习口诀

先问场景，再看输入；先拆结构，再走流程；先做小例，再谈优化；先会排错，再做规模化。
