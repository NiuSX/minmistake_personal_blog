## 讲义导读：从像素到可用视觉系统

这一章讲的是 **10-视频分析、目标跟踪与行为识别**。阅读时不要只记 OpenCV 函数、模型名字或论文缩写，而要把它放进完整视觉链路：图像如何采集，像素和颜色空间是否正确，光照和噪声如何影响结果，相机几何是否成立，模型输入和标注是否一致，评价指标是否对应漏检误检成本，部署时前后处理、延迟和现场漂移是否可控。CV 的目标不是让一张样例图看起来成功，而是在真实图像分布中稳定输出可用信息。

### 一句话先懂

视频视觉比图像多了时间维度：同一个目标会运动、遮挡、消失和重新出现，系统要保持身份和时序理解。

### 通俗类比

可以把 CV 系统想成流水线上的质检员：相机和光源负责把产品拍清楚，预处理像擦干净镜头和统一灯光，传统特征或神经网络负责看重点，检测和分割负责指出位置，后处理负责合并和过滤结果，评估指标负责统计漏检、误检和定位偏差，部署监控负责发现现场条件变化。

类比只是第一层直觉。回到技术上，要把每一步写成明确对象：输入图像的尺寸、通道、数据类型、坐标系、标注格式、模型输出、阈值、指标和运行环境。视觉问题很容易被表面效果迷惑，必须用可复现样本和量化指标来判断。

### 本章学习主线

1. **图像来源**：确认采集设备、光照、分辨率、帧率、压缩、畸变和场景覆盖。
2. **表示方式**：弄清像素数组、通道顺序、颜色空间、坐标系、数据类型和归一化。
3. **算法假设**：说明方法依赖什么条件，例如边缘明显、光照稳定、尺度不变、训练分布一致或标注可靠。
4. **输出解释**：区分类别、框、mask、关键点、文字、轨迹、三维点和置信度的含义。
5. **评估诊断**：用 IoU、mAP、Dice、OCR 准确率、MOT 指标、延迟和错误样本定位问题。
6. **工程闭环**：把训练、推理、后处理、可视化、日志、监控和现场迭代连起来。

### 概念怎么学才不容易忘

遇到视觉算法时，建议按 白话作用 -> 输入输出 -> 图像假设 -> 参数含义 -> 可视化结果 -> 失败样本 六步学习。比如 Canny 不只是一个函数，要看梯度、阈值和噪声；YOLO 不只是一个模型，要看输入尺寸、置信度、NMS、mAP 和小目标漏检；相机标定不只是拿到内参，要看重投影误差和畸变是否影响实际测量。

### 最小实践任务

用一段短视频做目标检测加跟踪，记录 ID switch、漏检、遮挡和帧率变化。

实践时要保留失败样本：反光、模糊、遮挡、过曝、欠曝、背景相似、标注不一致、小目标、长尾类别、视频抖动、部署前后处理不一致。这些样本比单张成功截图更能说明系统边界。

## 1. 视频和图像的区别

视频是连续帧序列，不仅有空间信息，还有时间信息。

视频任务需要考虑：

- 帧率。
- 时间连续性。
- 运动模糊。
- 遮挡。
- 目标进出画面。
- 跨帧身份一致性。
- 实时处理速度。

## 2. 视频处理基础

OpenCV 读取视频：

```python
cap = cv2.VideoCapture("video.mp4")
while True:
    ret, frame = cap.read()
    if not ret:
        break
```

摄像头实时读取：

```python
cap = cv2.VideoCapture(0)
```

注意：

- 摄像头延迟。
- 编码解码开销。
- 分辨率和 FPS。
- 多路视频并发。

## 3. 背景建模

背景建模用于检测运动前景。

常见：

- 帧差法。
- MOG2。
- KNN 背景减除。

适用：

- 固定摄像头。
- 背景较稳定。

问题：

- 光照变化。
- 背景晃动。
- 目标静止后融入背景。

## 4. 光流

光流描述像素在相邻帧之间的运动。

常见：

- Lucas-Kanade 稀疏光流。
- Farneback 稠密光流。

应用：

- 运动估计。
- 视频稳定。
- 行为分析。
- 目标跟踪辅助。

## 5. 单目标跟踪

给定第一帧目标位置，后续持续跟踪。

传统跟踪器：

- KCF。
- CSRT。
- MOSSE。

优点：

- 不需要每帧检测。
- 速度快。

缺点：

- 遮挡后容易漂移。
- 目标外观变化大时失效。

## 6. 多目标跟踪 MOT

多目标跟踪需要同时处理检测和身份关联。

基本流程：

```text
每帧目标检测 -> 预测轨迹 -> 关联检测框和已有轨迹 -> 更新轨迹 -> 管理新生和消失目标
```

常见算法：

- SORT。
- DeepSORT。
- ByteTrack。
- OC-SORT。

## 7. SORT

SORT 组成：

- 检测器。
- 卡尔曼滤波预测。
- 匈牙利算法匹配。
- IoU 关联。

优点：

- 简单快速。

缺点：

- 遮挡和交叉时 ID 容易切换。

## 8. DeepSORT

DeepSORT 增加外观特征 ReID，用于改善 ID 保持。

适合：

- 行人跟踪。
- 多目标交叉。

代价：

- 计算更重。
- 需要外观特征模型。

## 9. ByteTrack

ByteTrack 利用高分和低分检测框一起做关联，减少漏检导致的轨迹断裂。

常用于实时多目标跟踪。

## 10. 行为识别

行为识别判断视频中的动作或事件。

方法：

- 2D CNN + 时间池化。
- 3D CNN。
- Two-stream 网络。
- LSTM/GRU。
- Transformer。
- Skeleton-based 方法。

应用：

- 跌倒检测。
- 打架检测。
- 手势识别。
- 工业操作规范检测。
- 体育动作分析。

## 11. 视频事件检测

事件检测关注“什么时候发生了什么”。

示例：

- 人员越界。
- 车辆逆行。
- 物品遗留。
- 烟火检测。
- 设备异常动作。

常见工程方法：

```text
目标检测 -> 跟踪 -> 区域规则 -> 时间规则 -> 事件报警
```

## 12. 视频项目难点

- 实时性。
- 多路并发。
- 漏检导致跟踪断裂。
- 误检导致误报警。
- 摄像头抖动。
- 夜间和强光。
- 遮挡。
- 隐私合规。

## 13. 工程建议

- 用检测器保证基础准确率。
- 用跟踪减少闪烁和重复报警。
- 用时间窗口过滤瞬时误检。
- 用 ROI 区域减少无关检测。
- 保存报警前后视频片段，便于复查。
- 线上记录误报漏报样本，用于持续迭代。

计算机视觉的基础概念稳定，但模型生态、部署工具和数据集规范变化很快。复现实验前建议记录 Python、OpenCV、PyTorch、TorchVision、CUDA、cuDNN、ONNX Runtime、TensorRT、Ultralytics 或相关库版本，同时记录输入尺寸、颜色空间、归一化方式、类别表、标注格式、阈值、NMS 参数和硬件环境。
