# 10-视频分析、目标跟踪与行为识别

## 1. 视频和图像的区别

视频是连续帧序列，不仅有空间信息，还有时间信息。

视频任务需要考虑：

- 帧率。
- 时间连续性。
- 运动模糊。
- 遮挡。
- 目标进出画面。
- 跨帧身份一致性。
- 实时处理速度。

## 2. 视频处理基础

OpenCV 读取视频：

```python
cap = cv2.VideoCapture("video.mp4")
while True:
    ret, frame = cap.read()
    if not ret:
        break
```

摄像头实时读取：

```python
cap = cv2.VideoCapture(0)
```

注意：

- 摄像头延迟。
- 编码解码开销。
- 分辨率和 FPS。
- 多路视频并发。

## 3. 背景建模

背景建模用于检测运动前景。

常见：

- 帧差法。
- MOG2。
- KNN 背景减除。

适用：

- 固定摄像头。
- 背景较稳定。

问题：

- 光照变化。
- 背景晃动。
- 目标静止后融入背景。

## 4. 光流

光流描述像素在相邻帧之间的运动。

常见：

- Lucas-Kanade 稀疏光流。
- Farneback 稠密光流。

应用：

- 运动估计。
- 视频稳定。
- 行为分析。
- 目标跟踪辅助。

## 5. 单目标跟踪

给定第一帧目标位置，后续持续跟踪。

传统跟踪器：

- KCF。
- CSRT。
- MOSSE。

优点：

- 不需要每帧检测。
- 速度快。

缺点：

- 遮挡后容易漂移。
- 目标外观变化大时失效。

## 6. 多目标跟踪 MOT

多目标跟踪需要同时处理检测和身份关联。

基本流程：

```text
每帧目标检测 -> 预测轨迹 -> 关联检测框和已有轨迹 -> 更新轨迹 -> 管理新生和消失目标
```

常见算法：

- SORT。
- DeepSORT。
- ByteTrack。
- OC-SORT。

## 7. SORT

SORT 组成：

- 检测器。
- 卡尔曼滤波预测。
- 匈牙利算法匹配。
- IoU 关联。

优点：

- 简单快速。

缺点：

- 遮挡和交叉时 ID 容易切换。

## 8. DeepSORT

DeepSORT 增加外观特征 ReID，用于改善 ID 保持。

适合：

- 行人跟踪。
- 多目标交叉。

代价：

- 计算更重。
- 需要外观特征模型。

## 9. ByteTrack

ByteTrack 利用高分和低分检测框一起做关联，减少漏检导致的轨迹断裂。

常用于实时多目标跟踪。

## 10. 行为识别

行为识别判断视频中的动作或事件。

方法：

- 2D CNN + 时间池化。
- 3D CNN。
- Two-stream 网络。
- LSTM/GRU。
- Transformer。
- Skeleton-based 方法。

应用：

- 跌倒检测。
- 打架检测。
- 手势识别。
- 工业操作规范检测。
- 体育动作分析。

## 11. 视频事件检测

事件检测关注“什么时候发生了什么”。

示例：

- 人员越界。
- 车辆逆行。
- 物品遗留。
- 烟火检测。
- 设备异常动作。

常见工程方法：

```text
目标检测 -> 跟踪 -> 区域规则 -> 时间规则 -> 事件报警
```

## 12. 视频项目难点

- 实时性。
- 多路并发。
- 漏检导致跟踪断裂。
- 误检导致误报警。
- 摄像头抖动。
- 夜间和强光。
- 遮挡。
- 隐私合规。

## 13. 工程建议

- 用检测器保证基础准确率。
- 用跟踪减少闪烁和重复报警。
- 用时间窗口过滤瞬时误检。
- 用 ROI 区域减少无关检测。
- 保存报警前后视频片段，便于复查。
- 线上记录误报漏报样本，用于持续迭代。
