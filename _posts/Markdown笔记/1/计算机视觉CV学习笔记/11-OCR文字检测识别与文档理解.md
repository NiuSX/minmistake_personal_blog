# 11-OCR 文字检测、识别与文档理解

## 讲义导读：从像素到可用视觉系统

这一章讲的是 **11-OCR 文字检测、识别与文档理解**。阅读时不要只记 OpenCV 函数、模型名字或论文缩写，而要把它放进完整视觉链路：图像如何采集，像素和颜色空间是否正确，光照和噪声如何影响结果，相机几何是否成立，模型输入和标注是否一致，评价指标是否对应漏检误检成本，部署时前后处理、延迟和现场漂移是否可控。CV 的目标不是让一张样例图看起来成功，而是在真实图像分布中稳定输出可用信息。

### 一句话先懂

OCR 不只是识别字，还包括文本检测、方向校正、版面分析、识别、后处理和文档结构理解。

### 通俗类比

可以把 CV 系统想成流水线上的质检员：相机和光源负责把产品拍清楚，预处理像擦干净镜头和统一灯光，传统特征或神经网络负责看重点，检测和分割负责指出位置，后处理负责合并和过滤结果，评估指标负责统计漏检、误检和定位偏差，部署监控负责发现现场条件变化。

类比只是第一层直觉。回到技术上，要把每一步写成明确对象：输入图像的尺寸、通道、数据类型、坐标系、标注格式、模型输出、阈值、指标和运行环境。视觉问题很容易被表面效果迷惑，必须用可复现样本和量化指标来判断。

### 本章学习主线

1. **图像来源**：确认采集设备、光照、分辨率、帧率、压缩、畸变和场景覆盖。
2. **表示方式**：弄清像素数组、通道顺序、颜色空间、坐标系、数据类型和归一化。
3. **算法假设**：说明方法依赖什么条件，例如边缘明显、光照稳定、尺度不变、训练分布一致或标注可靠。
4. **输出解释**：区分类别、框、mask、关键点、文字、轨迹、三维点和置信度的含义。
5. **评估诊断**：用 IoU、mAP、Dice、OCR 准确率、MOT 指标、延迟和错误样本定位问题。
6. **工程闭环**：把训练、推理、后处理、可视化、日志、监控和现场迭代连起来。

### 概念怎么学才不容易忘

遇到视觉算法时，建议按 白话作用 -> 输入输出 -> 图像假设 -> 参数含义 -> 可视化结果 -> 失败样本 六步学习。比如 Canny 不只是一个函数，要看梯度、阈值和噪声；YOLO 不只是一个模型，要看输入尺寸、置信度、NMS、mAP 和小目标漏检；相机标定不只是拿到内参，要看重投影误差和畸变是否影响实际测量。

### 最小实践任务

选一张票据或截图，标出文本框、行方向、识别结果、错字原因和后处理规则。

实践时要保留失败样本：反光、模糊、遮挡、过曝、欠曝、背景相似、标注不一致、小目标、长尾类别、视频抖动、部署前后处理不一致。这些样本比单张成功截图更能说明系统边界。

## 1. OCR 是什么

OCR Optical Character Recognition，用于从图像中检测和识别文字。

完整 OCR 通常包含：

```text
图像预处理 -> 文本检测 -> 文本方向分类 -> 文本识别 -> 后处理 -> 结构化输出
```

## 2. OCR 场景

- 身份证识别。
- 发票识别。
- 表格识别。
- 车牌识别。
- 快递单。
- 工业字符。
- 仪表读数。
- 文档扫描。
- 屏幕文字。

## 3. OCR 预处理

常见方法：

- 灰度化。
- 去噪。
- 二值化。
- 倾斜矫正。
- 透视矫正。
- 去阴影。
- 增强对比度。
- 超分辨率。

文档 OCR 中，透视矫正非常重要。拍摄文档时常出现梯形变形，需要检测四角并做透视变换。

## 4. 文本检测

文本检测输出文字区域，可能是矩形框、多边形或像素 mask。

常见模型：

- CTPN。
- EAST。
- DBNet。
- PSENet。
- TextSnake。

检测难点：

- 弯曲文字。
- 倾斜文字。
- 小字。
- 低对比度。
- 复杂背景。
- 多语言。

## 5. 文本识别

文本识别把裁剪出的文字图转成字符序列。

常见模型：

- CRNN。
- Attention OCR。
- SAR。
- SVTR。
- Transformer OCR。

CRNN 典型结构：

```text
CNN 特征提取 -> RNN 序列建模 -> CTC 解码
```

## 6. CTC

CTC 用于处理输入特征序列和输出字符序列长度不一致的问题。

特点：

- 不需要逐字符位置标注。
- 适合不定长文本识别。
- 引入 blank 字符。

## 7. 方向分类

文本可能旋转 0、90、180、270 度。方向分类用于纠正文本方向。

常见场景：

- 手机拍摄。
- 文档扫描方向不固定。
- 工业字符倒置。

## 8. 后处理

后处理可以大幅提升 OCR 业务效果。

常见：

- 字典纠错。
- 正则校验。
- 日期格式校验。
- 身份证号码校验。
- 金额格式校验。
- 语言模型纠错。
- 置信度过滤。

例如车牌识别，可以用车牌规则限制字符类型。

## 9. 表格识别

表格识别需要恢复结构。

任务：

- 表格区域检测。
- 单元格检测。
- 行列关系恢复。
- 单元格 OCR。
- 合并单元格处理。

难点：

- 无线表格。
- 合并单元格。
- 倾斜拍摄。
- 表格线断裂。
- 多页文档。

## 10. 文档理解

文档理解不仅识别文字，还要理解字段含义。

例如发票：

- 发票号码。
- 日期。
- 金额。
- 买方。
- 卖方。
- 税号。

方法：

- 规则模板。
- 版面分析。
- LayoutLM 类模型。
- OCR + NLP。
- 多模态模型。

## 11. OCR 评价指标

- 文本检测 Precision/Recall/F1。
- 字符准确率。
- 行准确率。
- 字段准确率。
- 编辑距离。

业务中更关注字段准确率。例如发票金额错一个字符就是失败。

## 12. OCR 工程建议

- 拍摄质量比模型更重要。
- 尽量控制光照、角度、焦距。
- 对固定版式优先使用模板和规则。
- 对开放场景使用检测识别模型。
- 业务字段必须加格式校验。
- 保存低置信度样本用于人工复核。

## 2026 CV 资料与工程核对补充

计算机视觉的基础概念稳定，但模型生态、部署工具和数据集规范变化很快。复现实验前建议记录 Python、OpenCV、PyTorch、TorchVision、CUDA、cuDNN、ONNX Runtime、TensorRT、Ultralytics 或相关库版本，同时记录输入尺寸、颜色空间、归一化方式、类别表、标注格式、阈值、NMS 参数和硬件环境。

视觉项目尤其要防止训练和部署前后处理不一致。常见问题包括 BGR/RGB 搞反、归一化均值方差不同、resize 与 letterbox 不一致、坐标缩放错误、类别 id 对不上、置信度阈值和 NMS 阈值变化、摄像头现场光照不同。
