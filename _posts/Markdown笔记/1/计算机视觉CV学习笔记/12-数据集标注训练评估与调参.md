# 12-数据集、标注、训练、评估与调参

## 1. 数据决定模型上限

CV 项目中，数据质量通常比模型选择更重要。很多模型效果差，不是因为网络不够先进，而是因为：

- 数据太少。
- 场景覆盖不足。
- 标注错误。
- 类别不平衡。
- 训练集和真实场景差异大。
- 测试集不代表真实业务。

## 2. 数据采集

采集要覆盖：

- 不同光照。
- 不同角度。
- 不同距离。
- 不同背景。
- 不同设备。
- 不同天气。
- 不同批次产品。
- 正常样本和异常样本。

工业视觉要特别关注：

- 真实产线节拍。
- 光源稳定性。
- 相机曝光。
- 产品批次差异。
- 污渍、反光、遮挡。

## 3. 数据划分

常见划分：

- 训练集。
- 验证集。
- 测试集。

比例示例：

```text
70% train / 20% val / 10% test
```

注意：

- 同一视频连续帧不要同时出现在训练和测试中。
- 同一产品或同一场景数据泄漏会导致指标虚高。
- 测试集必须接近真实部署环境。

## 4. 标注类型

分类：

- 图片级标签。

检测：

- 矩形框 bbox。

分割：

- 多边形。
- mask。

关键点：

- 点坐标。
- 可见性。

OCR：

- 文本框。
- 文本内容。

## 5. 标注规范

必须明确：

- 类别定义。
- 边界框是否贴边。
- 遮挡目标是否标。
- 小目标是否标。
- 模糊目标是否标。
- 截断目标是否标。
- 重叠目标如何标。
- 缺陷边界如何画。

没有标注规范，模型会学习到混乱规则。

## 6. 常见标注工具

- LabelImg。
- Labelme。
- CVAT。
- Roboflow。
- Label Studio。
- VGG Image Annotator。

格式：

- VOC XML。
- COCO JSON。
- YOLO TXT。
- Labelme JSON。

## 7. 数据清洗

需要检查：

- 图片是否损坏。
- 标签文件是否缺失。
- 类别名是否一致。
- 框是否越界。
- 空标注是否合理。
- 重复图片。
- 错标漏标。
- 极端异常样本。

## 8. 数据增强

训练增强：

- 翻转。
- 旋转。
- 裁剪。
- 缩放。
- 颜色扰动。
- 模糊。
- 噪声。
- Mosaic。
- MixUp。

原则：

- 增强要符合真实场景。
- 不要破坏标签语义。
- 增强不是替代真实数据。

## 9. 训练参数

重要参数：

- batch size。
- learning rate。
- epoch。
- optimizer。
- weight decay。
- input size。
- augmentation。
- pretrained weights。

调参顺序建议：

1. 确认数据和标注。
2. 跑通 baseline。
3. 看训练曲线。
4. 调输入尺寸。
5. 调学习率。
6. 调增强。
7. 调模型大小。

## 10. 评估指标

分类：

- Accuracy。
- Precision。
- Recall。
- F1。
- AUC。

检测：

- mAP。
- Precision。
- Recall。
- IoU。
- 漏检率。
- 误检率。

分割：

- mIoU。
- Dice。
- Pixel Accuracy。

OCR：

- 字符准确率。
- 行准确率。
- 字段准确率。

## 11. 误差分析

训练后要看失败样本。

问题分类：

- 漏检。
- 误检。
- 类别混淆。
- 定位不准。
- 小目标失败。
- 遮挡失败。
- 低光失败。
- 特定背景失败。

每类问题对应解决策略不同：

- 漏检：增加样本、降低阈值、提高分辨率。
- 误检：增加负样本、提高阈值、清洗标签。
- 类别混淆：增加区分类特征样本、合并相似类。
- 定位不准：检查标注一致性、提高输入分辨率。

## 12. 训练曲线怎么看

训练 loss 降，验证指标不升：

- 可能过拟合。
- 验证集分布不同。
- 指标实现有问题。

训练 loss 不降：

- 学习率不合适。
- 数据或标签错误。
- 模型结构问题。
- 输入预处理错误。

训练指标很好，现场很差：

- 测试集不代表现场。
- 数据泄漏。
- 现场光照和相机不同。
- 推理预处理不一致。

## 讲义级补充：如何真正学懂《12-数据集、标注、训练、评估与调参》

> 适用位置：计算机视觉CV学习笔记\12-数据集标注训练评估与调参.md  
> 说明：本补充用于把原始提纲扩展成课堂讲义式学习材料。阅读时建议先看原文，再用本节建立知识框架、例子、实践和自测闭环。

### 1. 这一讲要解决什么问题

这类内容的核心不是公式本身，而是“数据如何变成可学习的表示，模型如何产生预测，误差如何反馈改进”。学习时要把数据、假设、损失、优化、评估和部署放在一起理解。

学习本讲时，可以用三个问题检查自己是否真的理解：

1. 它解决的真实问题是什么？
2. 如果没有它，系统会出现什么具体麻烦？
3. 在真实项目中，应该用什么现象或指标判断它做得好不好？

### 2. 核心知识拆解

可以把本讲拆成几块来学：

- 数据：样本、标签、分布、增强、清洗和划分。
- 模型：假设空间、结构、参数和表达能力。
- 训练：损失函数、优化器、正则化、学习率和收敛。
- 评估：指标、误差分析、泛化、鲁棒性和部署表现。

拆解的好处是防止“整章都懂一点，但哪块都说不清”。复习时可以逐块追问：它的输入是什么、输出是什么、依赖什么、失败时有什么表现。

### 3. 通俗类比

可以把模型类比成一个需要反复练习的学生：数据是练习题，标签是参考答案，损失函数是扣分规则，优化器是改错方法，验证集像模拟考试，测试集才像正式考试。只在训练集上表现好，就像只会背做过的题。

类比不是严格定义，但能帮助初学者先建立直觉。真正使用时，还要回到术语、公式、接口、数据结构、时序图或工程规范上，把“感觉理解”变成“可验证理解”。

### 4. 具体例子

学习《12-数据集、标注、训练、评估与调参》时，可以准备一个很小的数据集，先手工看 20 个样本和标签，再训练一个基线模型，最后把预测错误的样本逐张分析。很多模型问题不是公式错了，而是数据、标注或评价方式一开始就没有定义清楚。

讲义级学习不能只停留在“概念解释”。至少要有一个能跑、能算、能画或能检查的例子。例子越小，越容易看清关键机制；等机制清楚后，再逐步扩展到复杂项目。

### 5. 学习路径

- 先明确任务类型：分类、回归、检测、分割、生成、序列建模或决策。
- 再梳理数据、特征、模型、损失函数、优化方法和评价指标。
- 最后通过误差分析定位问题来自数据、标注、模型容量、训练策略还是部署环境。

建议每学完一小节都做一次“复述练习”：不用看笔记，用自己的话讲清楚概念、输入、输出、关键步骤和常见错误。如果讲不清，通常说明还没有真正掌握。

### 6. 课堂讲解框架

可以按下面顺序讲解或复习本主题：

1. 背景：先讲这个知识为什么出现，它试图降低什么成本、解决什么风险或提升什么能力。
2. 基本概念：给出核心名词的准确定义，说明它们之间的关系。
3. 工作流程：按时间顺序描述一次完整过程，必要时画出流程图、状态机或数据流图。
4. 关键细节：解释最容易误解的机制，例如边界条件、异常处理、性能限制、资源生命周期或安全约束。
5. 实战例子：用一个足够小但完整的例子，把概念落到命令、代码、图纸、配置、数据或操作步骤上。
6. 反例与排错：展示错误做法会导致什么现象，再说明如何定位和修复。
7. 总结迁移：最后说明它和相邻知识点的区别、联系以及后续该学什么。

### 7. 最小实践任务

为了避免“看懂了但不会用”，建议为本讲配一个最小实践：

- 选一个可以在 30 到 90 分钟内完成的小任务。
- 明确输入、预期输出和验收标准。
- 记录遇到的第一个错误、定位过程和最终修复方法。
- 完成后写 5 行复盘：我原来以为是什么，实际是什么，下次会如何更快处理。

如果本主题偏理论，实践可以是手算一个小例子、画一张流程图、推导一个简化公式或解释一段真实日志；如果偏工程，实践应该尽量落到可运行命令、可测试代码、可检查配置或可测量硬件现象上。

### 8. 常见误区

- 只追求模型结构新，不检查数据质量和标注一致性。
- 只看平均指标，不看错误样本、长尾类别和业务代价。
- 把训练集效果当成真实效果，忽略验证集、测试集和线上分布变化。

遇到这些问题时，不要急着背更多资料。更有效的办法是回到一个最小例子，把输入、状态变化、输出和验证方式重新走一遍。

### 9. 自测题

1. 用一句话说明本讲主题解决的核心问题。
2. 列出本讲最重要的 3 个概念，并说明它们的关系。
3. 举一个生活类比，再指出这个类比在哪些地方不严谨。
4. 写出一个最小实践任务的验收标准。
5. 如果结果不符合预期，你会优先检查哪 3 个环节？为什么？
6. 本讲和相邻章节的知识边界是什么？哪些问题应该交给其他章节解决？

### 10. 复习口诀

先问场景，再看输入；先拆结构，再走流程；先做小例，再谈优化；先会排错，再做规模化。
