# 12-数据集、标注、训练、评估与调参

<!-- lecture-notes:integrated-v2 -->

## 讲义导读：从像素到可用视觉系统

这一章讲的是 **12-数据集、标注、训练、评估与调参**。阅读时不要只记 OpenCV 函数、模型名字或论文缩写，而要把它放进完整视觉链路：图像如何采集，像素和颜色空间是否正确，光照和噪声如何影响结果，相机几何是否成立，模型输入和标注是否一致，评价指标是否对应漏检误检成本，部署时前后处理、延迟和现场漂移是否可控。CV 的目标不是让一张样例图看起来成功，而是在真实图像分布中稳定输出可用信息。

### 一句话先懂

CV 训练成败很大程度取决于数据：采集是否覆盖场景，标注是否一致，指标是否匹配业务损失。

### 通俗类比

可以把 CV 系统想成流水线上的质检员：相机和光源负责把产品拍清楚，预处理像擦干净镜头和统一灯光，传统特征或神经网络负责看重点，检测和分割负责指出位置，后处理负责合并和过滤结果，评估指标负责统计漏检、误检和定位偏差，部署监控负责发现现场条件变化。

类比只是第一层直觉。回到技术上，要把每一步写成明确对象：输入图像的尺寸、通道、数据类型、坐标系、标注格式、模型输出、阈值、指标和运行环境。视觉问题很容易被表面效果迷惑，必须用可复现样本和量化指标来判断。

### 本章学习主线

1. **图像来源**：确认采集设备、光照、分辨率、帧率、压缩、畸变和场景覆盖。
2. **表示方式**：弄清像素数组、通道顺序、颜色空间、坐标系、数据类型和归一化。
3. **算法假设**：说明方法依赖什么条件，例如边缘明显、光照稳定、尺度不变、训练分布一致或标注可靠。
4. **输出解释**：区分类别、框、mask、关键点、文字、轨迹、三维点和置信度的含义。
5. **评估诊断**：用 IoU、mAP、Dice、OCR 准确率、MOT 指标、延迟和错误样本定位问题。
6. **工程闭环**：把训练、推理、后处理、可视化、日志、监控和现场迭代连起来。

### 概念怎么学才不容易忘

遇到视觉算法时，建议按 白话作用 -> 输入输出 -> 图像假设 -> 参数含义 -> 可视化结果 -> 失败样本 六步学习。比如 Canny 不只是一个函数，要看梯度、阈值和噪声；YOLO 不只是一个模型，要看输入尺寸、置信度、NMS、mAP 和小目标漏检；相机标定不只是拿到内参，要看重投影误差和畸变是否影响实际测量。

### 最小实践任务

为一个检测任务设计数据集划分、标注规范、训练配置、mAP/召回率评估和误差分析表。

实践时要保留失败样本：反光、模糊、遮挡、过曝、欠曝、背景相似、标注不一致、小目标、长尾类别、视频抖动、部署前后处理不一致。这些样本比单张成功截图更能说明系统边界。

- 画出最小处理流程，标清输入图像、关键参数、输出结果和评价指标。
- 说出至少三个常见失败场景，以及对应的可视化或量化诊断方法。

## 1. 数据决定模型上限

CV 项目中，数据质量通常比模型选择更重要。很多模型效果差，不是因为网络不够先进，而是因为：

- 数据太少。
- 场景覆盖不足。
- 标注错误。
- 类别不平衡。
- 训练集和真实场景差异大。
- 测试集不代表真实业务。

## 2. 数据采集

采集要覆盖：

- 不同光照。
- 不同角度。
- 不同距离。
- 不同背景。
- 不同设备。
- 不同天气。
- 不同批次产品。
- 正常样本和异常样本。

工业视觉要特别关注：

- 真实产线节拍。
- 光源稳定性。
- 相机曝光。
- 产品批次差异。
- 污渍、反光、遮挡。

## 3. 数据划分

常见划分：

- 训练集。
- 验证集。
- 测试集。

比例示例：

```text
70% train / 20% val / 10% test
```

注意：

- 同一视频连续帧不要同时出现在训练和测试中。
- 同一产品或同一场景数据泄漏会导致指标虚高。
- 测试集必须接近真实部署环境。

## 4. 标注类型

分类：

- 图片级标签。

检测：

- 矩形框 bbox。

分割：

- 多边形。
- mask。

关键点：

- 点坐标。
- 可见性。

OCR：

- 文本框。
- 文本内容。

## 5. 标注规范

必须明确：

- 类别定义。
- 边界框是否贴边。
- 遮挡目标是否标。
- 小目标是否标。
- 模糊目标是否标。
- 截断目标是否标。
- 重叠目标如何标。
- 缺陷边界如何画。

没有标注规范，模型会学习到混乱规则。

## 7. 数据清洗

需要检查：

- 图片是否损坏。
- 标签文件是否缺失。
- 类别名是否一致。
- 框是否越界。
- 空标注是否合理。
- 重复图片。
- 错标漏标。
- 极端异常样本。

## 8. 数据增强

训练增强：

- 翻转。
- 旋转。
- 裁剪。
- 缩放。
- 颜色扰动。
- 模糊。
- 噪声。
- Mosaic。
- MixUp。

原则：

- 增强要符合真实场景。
- 不要破坏标签语义。
- 增强不是替代真实数据。

## 9. 训练参数

重要参数：

- batch size。
- learning rate。
- epoch。
- optimizer。
- weight decay。
- input size。
- augmentation。
- pretrained weights。

调参顺序建议：

1. 确认数据和标注。
2. 跑通 baseline。
3. 看训练曲线。
4. 调输入尺寸。
5. 调学习率。
6. 调增强。
7. 调模型大小。

## 11. 误差分析

训练后要看失败样本。

问题分类：

- 漏检。
- 误检。
- 类别混淆。
- 定位不准。
- 小目标失败。
- 遮挡失败。
- 低光失败。
- 特定背景失败。

每类问题对应解决策略不同：

- 漏检：增加样本、降低阈值、提高分辨率。
- 误检：增加负样本、提高阈值、清洗标签。
- 类别混淆：增加区分类特征样本、合并相似类。
- 定位不准：检查标注一致性、提高输入分辨率。

## 12. 训练曲线怎么看

训练 loss 降，验证指标不升：

- 可能过拟合。
- 验证集分布不同。
- 指标实现有问题。

训练 loss 不降：

- 学习率不合适。
- 数据或标签错误。
- 模型结构问题。
- 输入预处理错误。

训练指标很好，现场很差：

- 测试集不代表现场。
- 数据泄漏。
- 现场光照和相机不同。
- 推理预处理不一致。

## 2026 CV 资料与工程核对补充

计算机视觉的基础概念稳定，但模型生态、部署工具和数据集规范变化很快。复现实验前建议记录 Python、OpenCV、PyTorch、TorchVision、CUDA、cuDNN、ONNX Runtime、TensorRT、Ultralytics 或相关库版本，同时记录输入尺寸、颜色空间、归一化方式、类别表、标注格式、阈值、NMS 参数和硬件环境。

视觉项目尤其要防止训练和部署前后处理不一致。常见问题包括 BGR/RGB 搞反、归一化均值方差不同、resize 与 letterbox 不一致、坐标缩放错误、类别 id 对不上、置信度阈值和 NMS 阈值变化、摄像头现场光照不同。
