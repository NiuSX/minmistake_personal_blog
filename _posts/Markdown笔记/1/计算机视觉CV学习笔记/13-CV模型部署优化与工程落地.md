# 13-CV 模型部署、优化与工程落地

## 讲义导读：从像素到可用视觉系统

这一章讲的是 **13-CV 模型部署、优化与工程落地**。阅读时不要只记 OpenCV 函数、模型名字或论文缩写，而要把它放进完整视觉链路：图像如何采集，像素和颜色空间是否正确，光照和噪声如何影响结果，相机几何是否成立，模型输入和标注是否一致，评价指标是否对应漏检误检成本，部署时前后处理、延迟和现场漂移是否可控。CV 的目标不是让一张样例图看起来成功，而是在真实图像分布中稳定输出可用信息。

### 一句话先懂

CV 部署要把模型从离线准确率变成现场稳定能力，关注延迟、吞吐、显存、后处理、输入质量和监控。

### 通俗类比

可以把 CV 系统想成流水线上的质检员：相机和光源负责把产品拍清楚，预处理像擦干净镜头和统一灯光，传统特征或神经网络负责看重点，检测和分割负责指出位置，后处理负责合并和过滤结果，评估指标负责统计漏检、误检和定位偏差，部署监控负责发现现场条件变化。

类比只是第一层直觉。回到技术上，要把每一步写成明确对象：输入图像的尺寸、通道、数据类型、坐标系、标注格式、模型输出、阈值、指标和运行环境。视觉问题很容易被表面效果迷惑，必须用可复现样本和量化指标来判断。

### 本章学习主线

1. **图像来源**：确认采集设备、光照、分辨率、帧率、压缩、畸变和场景覆盖。
2. **表示方式**：弄清像素数组、通道顺序、颜色空间、坐标系、数据类型和归一化。
3. **算法假设**：说明方法依赖什么条件，例如边缘明显、光照稳定、尺度不变、训练分布一致或标注可靠。
4. **输出解释**：区分类别、框、mask、关键点、文字、轨迹、三维点和置信度的含义。
5. **评估诊断**：用 IoU、mAP、Dice、OCR 准确率、MOT 指标、延迟和错误样本定位问题。
6. **工程闭环**：把训练、推理、后处理、可视化、日志、监控和现场迭代连起来。

### 概念怎么学才不容易忘

遇到视觉算法时，建议按 白话作用 -> 输入输出 -> 图像假设 -> 参数含义 -> 可视化结果 -> 失败样本 六步学习。比如 Canny 不只是一个函数，要看梯度、阈值和噪声；YOLO 不只是一个模型，要看输入尺寸、置信度、NMS、mAP 和小目标漏检；相机标定不只是拿到内参，要看重投影误差和畸变是否影响实际测量。

### 最小实践任务

把一个模型导出 ONNX，记录输入预处理、推理耗时、后处理、精度差异和部署环境。

实践时要保留失败样本：反光、模糊、遮挡、过曝、欠曝、背景相似、标注不一致、小目标、长尾类别、视频抖动、部署前后处理不一致。这些样本比单张成功截图更能说明系统边界。

### 读完本章应该能做到

- 用自己的话解释本章主题在 CV 流水线中的位置。
- 画出最小处理流程，标清输入图像、关键参数、输出结果和评价指标。
- 说出至少三个常见失败场景，以及对应的可视化或量化诊断方法。
- 给出一个可复现实验，能比较参数、数据或模型变化带来的影响。
- 能解释 ONNX、TensorRT、量化、batch、前后处理一致性、摄像头链路、延迟预算和现场监控。

## 1. 从模型到产品

训练出高指标模型只是第一步。真正落地需要：

- 稳定读取图像或视频。
- 正确预处理。
- 快速推理。
- 后处理。
- 结果可视化。
- 异常处理。
- 日志记录。
- 版本管理。
- 性能监控。

## 2. 推理流程

典型流程：

```text
采集图像 -> 预处理 -> 模型推理 -> 后处理 -> 业务规则 -> 输出结果
```

其中预处理和训练必须一致：

- 色彩空间。
- Resize/Letterbox。
- 归一化。
- 通道顺序。
- 数据类型。

很多部署错误来自预处理不一致。

## 3. 模型格式

常见格式：

- PyTorch `.pt/.pth`。
- ONNX。
- TensorRT engine。
- OpenVINO IR。
- TFLite。
- NCNN。

ONNX 常作为中间格式，便于跨框架部署。

## 4. ONNX

ONNX 是开放神经网络交换格式。

优点：

- 跨平台。
- 可被 TensorRT、OpenVINO、ONNX Runtime 使用。
- 便于 C++/Python 部署。

导出注意：

- opset 版本。
- 动态输入尺寸。
- 后处理是否包含在模型中。
- 算子是否被目标推理框架支持。

## 5. TensorRT

TensorRT 是 NVIDIA GPU 推理加速工具。

优化方式：

- 图优化。
- 层融合。
- FP16。
- INT8 量化。
- 内存优化。

适合：

- NVIDIA GPU。
- 实时检测。
- 多路视频。

注意：

- engine 和 GPU 型号、TensorRT 版本相关。
- INT8 需要校准数据。
- 动态 shape 需要配置 profile。

## 6. OpenVINO

OpenVINO 适合 Intel CPU、核显、VPU 部署。

适用：

- 工业 PC。
- CPU 推理。
- 边缘设备。

## 7. 移动端和边缘部署

常见方案：

- TFLite。
- NCNN。
- MNN。
- CoreML。
- ONNX Runtime Mobile。

优化：

- 模型剪枝。
- 量化。
- 蒸馏。
- 使用轻量 backbone。
- 降低输入分辨率。

## 8. 性能优化

指标：

- 延迟 latency。
- 吞吐 throughput。
- FPS。
- CPU 占用。
- GPU 占用。
- 内存。
- 显存。

优化方向：

- 减小输入尺寸。
- 使用更小模型。
- FP16/INT8。
- 批处理。
- 异步流水线。
- 多线程读流。
- ROI 推理。
- 降低后处理复杂度。

## 9. 后处理优化

检测后处理包括：

- 解码预测。
- 阈值过滤。
- NMS。
- 坐标还原。

NMS 在目标多时可能成为瓶颈。可使用：

- GPU NMS。
- batched NMS。
- 降低候选框数量。
- 提前阈值过滤。

## 10. 工业视觉落地

工业项目流程：

```text
现场调研 -> 相机光源选型 -> 采集样本 -> 算法验证 -> 软件集成 -> 产线测试 -> 误报漏报迭代
```

重点：

- 光源比算法更重要。
- 相机固定后不要随意移动。
- 产品位置要稳定。
- 需要触发拍照和 PLC 通信。
- 结果要能追溯。
- 缺陷样本要持续积累。

## 11. 视频系统落地

视频项目关注：

- 拉流稳定性。
- 解码效率。
- 多路并发。
- 帧丢弃策略。
- 报警去重。
- 事件录像。
- 断线重连。
- 时间同步。

## 12. 版本管理

需要记录：

- 模型版本。
- 训练数据版本。
- 代码版本。
- 配置版本。
- 阈值。
- 部署环境。
- 指标。

没有版本记录，现场问题很难追溯。

## 13. 线上监控

需要监控：

- 推理耗时。
- 摄像头状态。
- 报警数量。
- 低置信度样本。
- 崩溃日志。
- 误报反馈。
- 漏报反馈。

CV 系统上线后仍需要持续迭代。

## 2026 CV 资料与工程核对补充

计算机视觉的基础概念稳定，但模型生态、部署工具和数据集规范变化很快。复现实验前建议记录 Python、OpenCV、PyTorch、TorchVision、CUDA、cuDNN、ONNX Runtime、TensorRT、Ultralytics 或相关库版本，同时记录输入尺寸、颜色空间、归一化方式、类别表、标注格式、阈值、NMS 参数和硬件环境。

视觉项目尤其要防止训练和部署前后处理不一致。常见问题包括 BGR/RGB 搞反、归一化均值方差不同、resize 与 letterbox 不一致、坐标缩放错误、类别 id 对不上、置信度阈值和 NMS 阈值变化、摄像头现场光照不同。
