# 13-CV 模型部署、优化与工程落地

## 1. 从模型到产品

训练出高指标模型只是第一步。真正落地需要：

- 稳定读取图像或视频。
- 正确预处理。
- 快速推理。
- 后处理。
- 结果可视化。
- 异常处理。
- 日志记录。
- 版本管理。
- 性能监控。

## 2. 推理流程

典型流程：

```text
采集图像 -> 预处理 -> 模型推理 -> 后处理 -> 业务规则 -> 输出结果
```

其中预处理和训练必须一致：

- 色彩空间。
- Resize/Letterbox。
- 归一化。
- 通道顺序。
- 数据类型。

很多部署错误来自预处理不一致。

## 3. 模型格式

常见格式：

- PyTorch `.pt/.pth`。
- ONNX。
- TensorRT engine。
- OpenVINO IR。
- TFLite。
- NCNN。

ONNX 常作为中间格式，便于跨框架部署。

## 4. ONNX

ONNX 是开放神经网络交换格式。

优点：

- 跨平台。
- 可被 TensorRT、OpenVINO、ONNX Runtime 使用。
- 便于 C++/Python 部署。

导出注意：

- opset 版本。
- 动态输入尺寸。
- 后处理是否包含在模型中。
- 算子是否被目标推理框架支持。

## 5. TensorRT

TensorRT 是 NVIDIA GPU 推理加速工具。

优化方式：

- 图优化。
- 层融合。
- FP16。
- INT8 量化。
- 内存优化。

适合：

- NVIDIA GPU。
- 实时检测。
- 多路视频。

注意：

- engine 和 GPU 型号、TensorRT 版本相关。
- INT8 需要校准数据。
- 动态 shape 需要配置 profile。

## 6. OpenVINO

OpenVINO 适合 Intel CPU、核显、VPU 部署。

适用：

- 工业 PC。
- CPU 推理。
- 边缘设备。

## 7. 移动端和边缘部署

常见方案：

- TFLite。
- NCNN。
- MNN。
- CoreML。
- ONNX Runtime Mobile。

优化：

- 模型剪枝。
- 量化。
- 蒸馏。
- 使用轻量 backbone。
- 降低输入分辨率。

## 8. 性能优化

指标：

- 延迟 latency。
- 吞吐 throughput。
- FPS。
- CPU 占用。
- GPU 占用。
- 内存。
- 显存。

优化方向：

- 减小输入尺寸。
- 使用更小模型。
- FP16/INT8。
- 批处理。
- 异步流水线。
- 多线程读流。
- ROI 推理。
- 降低后处理复杂度。

## 9. 后处理优化

检测后处理包括：

- 解码预测。
- 阈值过滤。
- NMS。
- 坐标还原。

NMS 在目标多时可能成为瓶颈。可使用：

- GPU NMS。
- batched NMS。
- 降低候选框数量。
- 提前阈值过滤。

## 10. 工业视觉落地

工业项目流程：

```text
现场调研 -> 相机光源选型 -> 采集样本 -> 算法验证 -> 软件集成 -> 产线测试 -> 误报漏报迭代
```

重点：

- 光源比算法更重要。
- 相机固定后不要随意移动。
- 产品位置要稳定。
- 需要触发拍照和 PLC 通信。
- 结果要能追溯。
- 缺陷样本要持续积累。

## 11. 视频系统落地

视频项目关注：

- 拉流稳定性。
- 解码效率。
- 多路并发。
- 帧丢弃策略。
- 报警去重。
- 事件录像。
- 断线重连。
- 时间同步。

## 12. 版本管理

需要记录：

- 模型版本。
- 训练数据版本。
- 代码版本。
- 配置版本。
- 阈值。
- 部署环境。
- 指标。

没有版本记录，现场问题很难追溯。

## 13. 线上监控

需要监控：

- 推理耗时。
- 摄像头状态。
- 报警数量。
- 低置信度样本。
- 崩溃日志。
- 误报反馈。
- 漏报反馈。

CV 系统上线后仍需要持续迭代。

## 讲义级补充：如何真正学懂《13-CV 模型部署、优化与工程落地》

> 适用位置：计算机视觉CV学习笔记\13-CV模型部署优化与工程落地.md  
> 说明：本补充用于把原始提纲扩展成课堂讲义式学习材料。阅读时建议先看原文，再用本节建立知识框架、例子、实践和自测闭环。

### 1. 这一讲要解决什么问题

这类内容的核心不是公式本身，而是“数据如何变成可学习的表示，模型如何产生预测，误差如何反馈改进”。学习时要把数据、假设、损失、优化、评估和部署放在一起理解。

学习本讲时，可以用三个问题检查自己是否真的理解：

1. 它解决的真实问题是什么？
2. 如果没有它，系统会出现什么具体麻烦？
3. 在真实项目中，应该用什么现象或指标判断它做得好不好？

### 2. 核心知识拆解

可以把本讲拆成几块来学：

- 数据：样本、标签、分布、增强、清洗和划分。
- 模型：假设空间、结构、参数和表达能力。
- 训练：损失函数、优化器、正则化、学习率和收敛。
- 评估：指标、误差分析、泛化、鲁棒性和部署表现。

拆解的好处是防止“整章都懂一点，但哪块都说不清”。复习时可以逐块追问：它的输入是什么、输出是什么、依赖什么、失败时有什么表现。

### 3. 通俗类比

可以把模型类比成一个需要反复练习的学生：数据是练习题，标签是参考答案，损失函数是扣分规则，优化器是改错方法，验证集像模拟考试，测试集才像正式考试。只在训练集上表现好，就像只会背做过的题。

类比不是严格定义，但能帮助初学者先建立直觉。真正使用时，还要回到术语、公式、接口、数据结构、时序图或工程规范上，把“感觉理解”变成“可验证理解”。

### 4. 具体例子

学习《13-CV 模型部署、优化与工程落地》时，可以准备一个很小的数据集，先手工看 20 个样本和标签，再训练一个基线模型，最后把预测错误的样本逐张分析。很多模型问题不是公式错了，而是数据、标注或评价方式一开始就没有定义清楚。

讲义级学习不能只停留在“概念解释”。至少要有一个能跑、能算、能画或能检查的例子。例子越小，越容易看清关键机制；等机制清楚后，再逐步扩展到复杂项目。

### 5. 学习路径

- 先明确任务类型：分类、回归、检测、分割、生成、序列建模或决策。
- 再梳理数据、特征、模型、损失函数、优化方法和评价指标。
- 最后通过误差分析定位问题来自数据、标注、模型容量、训练策略还是部署环境。

建议每学完一小节都做一次“复述练习”：不用看笔记，用自己的话讲清楚概念、输入、输出、关键步骤和常见错误。如果讲不清，通常说明还没有真正掌握。

### 6. 课堂讲解框架

可以按下面顺序讲解或复习本主题：

1. 背景：先讲这个知识为什么出现，它试图降低什么成本、解决什么风险或提升什么能力。
2. 基本概念：给出核心名词的准确定义，说明它们之间的关系。
3. 工作流程：按时间顺序描述一次完整过程，必要时画出流程图、状态机或数据流图。
4. 关键细节：解释最容易误解的机制，例如边界条件、异常处理、性能限制、资源生命周期或安全约束。
5. 实战例子：用一个足够小但完整的例子，把概念落到命令、代码、图纸、配置、数据或操作步骤上。
6. 反例与排错：展示错误做法会导致什么现象，再说明如何定位和修复。
7. 总结迁移：最后说明它和相邻知识点的区别、联系以及后续该学什么。

### 7. 最小实践任务

为了避免“看懂了但不会用”，建议为本讲配一个最小实践：

- 选一个可以在 30 到 90 分钟内完成的小任务。
- 明确输入、预期输出和验收标准。
- 记录遇到的第一个错误、定位过程和最终修复方法。
- 完成后写 5 行复盘：我原来以为是什么，实际是什么，下次会如何更快处理。

如果本主题偏理论，实践可以是手算一个小例子、画一张流程图、推导一个简化公式或解释一段真实日志；如果偏工程，实践应该尽量落到可运行命令、可测试代码、可检查配置或可测量硬件现象上。

### 8. 常见误区

- 只追求模型结构新，不检查数据质量和标注一致性。
- 只看平均指标，不看错误样本、长尾类别和业务代价。
- 把训练集效果当成真实效果，忽略验证集、测试集和线上分布变化。

遇到这些问题时，不要急着背更多资料。更有效的办法是回到一个最小例子，把输入、状态变化、输出和验证方式重新走一遍。

### 9. 自测题

1. 用一句话说明本讲主题解决的核心问题。
2. 列出本讲最重要的 3 个概念，并说明它们的关系。
3. 举一个生活类比，再指出这个类比在哪些地方不严谨。
4. 写出一个最小实践任务的验收标准。
5. 如果结果不符合预期，你会优先检查哪 3 个环节？为什么？
6. 本讲和相邻章节的知识边界是什么？哪些问题应该交给其他章节解决？

### 10. 复习口诀

先问场景，再看输入；先拆结构，再走流程；先做小例，再谈优化；先会排错，再做规模化。
