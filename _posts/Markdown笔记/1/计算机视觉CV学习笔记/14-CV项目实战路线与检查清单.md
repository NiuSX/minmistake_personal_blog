# 14-CV 项目实战路线与检查清单

<!-- lecture-notes:integrated-v2 -->

## 讲义导读：从像素到可用视觉系统

这一章讲的是 **14-CV 项目实战路线与检查清单**。阅读时不要只记 OpenCV 函数、模型名字或论文缩写，而要把它放进完整视觉链路：图像如何采集，像素和颜色空间是否正确，光照和噪声如何影响结果，相机几何是否成立，模型输入和标注是否一致，评价指标是否对应漏检误检成本，部署时前后处理、延迟和现场漂移是否可控。CV 的目标不是让一张样例图看起来成功，而是在真实图像分布中稳定输出可用信息。

### 一句话先懂

CV 学习路线要从图像本质和 OpenCV 开始，再进入传统算法、几何视觉、深度模型、任务评估和工程部署。

### 通俗类比

可以把 CV 系统想成流水线上的质检员：相机和光源负责把产品拍清楚，预处理像擦干净镜头和统一灯光，传统特征或神经网络负责看重点，检测和分割负责指出位置，后处理负责合并和过滤结果，评估指标负责统计漏检、误检和定位偏差，部署监控负责发现现场条件变化。

类比只是第一层直觉。回到技术上，要把每一步写成明确对象：输入图像的尺寸、通道、数据类型、坐标系、标注格式、模型输出、阈值、指标和运行环境。视觉问题很容易被表面效果迷惑，必须用可复现样本和量化指标来判断。

### 本章学习主线

1. **图像来源**：确认采集设备、光照、分辨率、帧率、压缩、畸变和场景覆盖。
2. **表示方式**：弄清像素数组、通道顺序、颜色空间、坐标系、数据类型和归一化。
3. **算法假设**：说明方法依赖什么条件，例如边缘明显、光照稳定、尺度不变、训练分布一致或标注可靠。
4. **输出解释**：区分类别、框、mask、关键点、文字、轨迹、三维点和置信度的含义。
5. **评估诊断**：用 IoU、mAP、Dice、OCR 准确率、MOT 指标、延迟和错误样本定位问题。
6. **工程闭环**：把训练、推理、后处理、可视化、日志、监控和现场迭代连起来。

### 概念怎么学才不容易忘

遇到视觉算法时，建议按 白话作用 -> 输入输出 -> 图像假设 -> 参数含义 -> 可视化结果 -> 失败样本 六步学习。比如 Canny 不只是一个函数，要看梯度、阈值和噪声；YOLO 不只是一个模型，要看输入尺寸、置信度、NMS、mAP 和小目标漏检；相机标定不只是拿到内参，要看重投影误差和畸变是否影响实际测量。

### 最小实践任务

围绕一个缺陷检测或目标检测项目，画出采集、标注、训练、评估、部署和现场迭代流程。

实践时要保留失败样本：反光、模糊、遮挡、过曝、欠曝、背景相似、标注不一致、小目标、长尾类别、视频抖动、部署前后处理不一致。这些样本比单张成功截图更能说明系统边界。

### 读完本章应该能做到

- 用自己的话解释本章主题在 CV 流水线中的位置。
- 画出最小处理流程，标清输入图像、关键参数、输出结果和评价指标。
- 说出至少三个常见失败场景，以及对应的可视化或量化诊断方法。
- 给出一个可复现实验，能比较参数、数据或模型变化带来的影响。
- 能说明每个 CV 方向解决的问题、输入输出、常见指标和学习顺序。

## 1. 入门项目路线

### 项目 1：颜色目标提取

目标：

- 学会 HSV。
- 学会阈值分割。
- 学会形态学。

任务：

- 提取红色物体。
- 计算中心点。
- 输出面积。

### 项目 2：边缘和轮廓检测

目标：

- 学会 Canny。
- 学会轮廓。
- 学会几何特征。

任务：

- 检测工件轮廓。
- 计算外接矩形。
- 判断面积是否合格。

### 项目 3：模板匹配

目标：

- 学会模板匹配。
- 理解尺度和旋转限制。

任务：

- 在图中找到固定 logo 或零件。

### 项目 4：图像分类

目标：

- 学会训练分类模型。
- 学会数据划分和评估。

任务：

- 猫狗分类。
- 缺陷有无分类。

### 项目 5：目标检测

目标：

- 学会标注 bbox。
- 学会训练 YOLO。
- 学会 mAP 和误差分析。

任务：

- 检测安全帽。
- 检测零件。
- 检测车辆。

### 项目 6：语义分割

目标：

- 学会 mask 标注。
- 学会 U-Net。
- 学会 IoU 和 Dice。

任务：

- 缺陷区域分割。
- 道路分割。
- 医学区域分割。

### 项目 7：OCR

目标：

- 学会文本检测和识别。
- 学会后处理规则。

任务：

- 识别仪表数字。
- 识别发票字段。
- 识别工业喷码。

### 项目 8：视频跟踪

目标：

- 学会检测 + 跟踪。
- 学会事件规则。

任务：

- 行人计数。
- 区域入侵检测。
- 车辆跟踪。

## 2. CV 项目需求检查

- 要解决什么业务问题。
- 输入是图片、视频、相机还是文件。
- 输出是什么：类别、框、mask、坐标、尺寸、报警。
- 精度要求是什么。
- 速度要求是什么。
- 误报和漏报哪个更严重。
- 部署在哪里：服务器、工控机、边缘设备、移动端。
- 是否有样本数据。
- 是否能控制光源和相机。
- 是否需要和 PLC、机器人、MES、后端系统对接。

## 3. 数据检查

- 数据量是否足够。
- 类别是否平衡。
- 场景是否覆盖真实环境。
- 是否有难例。
- 是否有负样本。
- 标注是否一致。
- 训练集和测试集是否泄漏。
- 是否保留独立测试集。

## 4. 传统算法检查

- 是否灰度化。
- 是否需要去噪。
- 阈值是否稳定。
- 是否有光照变化。
- 轮廓是否完整。
- 参数是否过拟合样例图。
- 是否可视化每一步。
- 是否有异常输入处理。

## 5. 深度学习检查

- 模型是否适合任务。
- 输入尺寸是否合适。
- 预训练权重是否使用。
- 增强是否合理。
- loss 是否正常下降。
- 验证指标是否可信。
- 是否查看失败样本。
- 是否测试推理速度。

## 6. 部署检查

- 训练和推理预处理是否一致。
- 模型版本是否记录。
- 阈值是否可配置。
- 是否支持异常图像。
- 是否记录日志。
- 是否支持批量测试。
- 是否有可视化结果。
- 是否能保存误检漏检样本。
- 是否有性能监控。

## 7. 常见学习误区

- 只学模型，不学图像基础。
- 只看 mAP，不看真实误报漏报。
- 不清洗数据就训练。
- 标注规范不统一。
- 训练集和测试集混在一起。
- 部署时预处理写错。
- 现场相机光源不稳定，却只调模型。
- 不保存失败样本，无法迭代。

## 8. 长期能力建设

## 2026 CV 资料与工程核对补充

计算机视觉的基础概念稳定，但模型生态、部署工具和数据集规范变化很快。复现实验前建议记录 Python、OpenCV、PyTorch、TorchVision、CUDA、cuDNN、ONNX Runtime、TensorRT、Ultralytics 或相关库版本，同时记录输入尺寸、颜色空间、归一化方式、类别表、标注格式、阈值、NMS 参数和硬件环境。

视觉项目尤其要防止训练和部署前后处理不一致。常见问题包括 BGR/RGB 搞反、归一化均值方差不同、resize 与 letterbox 不一致、坐标缩放错误、类别 id 对不上、置信度阈值和 NMS 阈值变化、摄像头现场光照不同。
