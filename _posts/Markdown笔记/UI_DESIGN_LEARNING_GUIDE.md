# UI 设计零基础学习文档

这份文档面向完全没有 UI 设计经验的初学者。它不是只讲“怎么把界面做漂亮”，而是从用户、信息、视觉、交互、组件、流程、工具和练习方法几个角度，系统解释 UI 设计的基础知识。

UI 设计的核心目标不是装饰页面，而是让用户清楚地理解内容、顺利完成任务，并在使用过程中感到稳定、可信、舒适。

## 1. UI 设计是什么

UI 是 User Interface 的缩写，中文通常叫“用户界面”。UI 设计就是设计用户和产品之间的界面。

常见 UI 包括：

- 手机 App 的首页、详情页、登录页
- 网站页面
- 后台管理系统
- 小程序
- 智能设备屏幕
- 车机界面
- 软件工具界面
- 游戏菜单和操作界面

UI 设计师要解决的问题包括：

- 用户进入页面后能不能马上看懂这是哪里
- 用户能不能快速找到自己要做的事情
- 页面信息有没有主次
- 操作按钮是否明显
- 点击之后有没有反馈
- 表单是否容易填写
- 错误提示是否清楚
- 页面视觉是否统一
- 不同尺寸屏幕上是否可用

一个好的 UI 通常具有这些特点：

- 清晰：用户看得懂
- 高效：用户不用绕路
- 一致：同类操作表现相同
- 稳定：布局、状态、反馈可预期
- 美观：视觉让人愿意使用
- 可访问：不同能力、设备和环境下的人都尽量能用

## 2. UI、UX、视觉设计、产品设计的区别

很多新手容易混淆几个概念。

### UI 设计

UI 设计关注界面本身，包括布局、颜色、字体、按钮、图标、组件、交互状态等。

例子：设计一个登录页面时，UI 设计会考虑输入框样式、按钮位置、错误提示样式、页面留白、品牌色使用。

### UX 设计

UX 是 User Experience，中文叫“用户体验”。UX 更关注用户完成任务的整体过程。

例子：用户为什么要登录？什么时候提示登录？忘记密码流程是否顺畅？登录失败后如何帮助用户？

### 视觉设计

视觉设计更强调画面表现，包括品牌感、视觉风格、图片、图形、动效、氛围、审美。

例子：同样是一个音乐 App，视觉设计会决定它是年轻潮流、安静高级，还是科技感强。

### 产品设计

产品设计关注产品功能、业务逻辑、用户需求和整体方案。

例子：是否需要会员系统？用户收藏内容后在哪里查看？订单状态有哪些？这些通常属于产品设计范畴。

实际工作中，这些能力经常交叉。优秀的 UI 设计师不能只会画界面，也要理解用户、流程和业务。

## 3. 学 UI 设计前需要建立的思维

### 设计不是凭感觉

新手常说“这个好看”“那个不好看”，但专业设计需要能解释原因。

例如：

- 这个按钮不明显，是因为颜色对比度不够
- 这个页面很乱，是因为信息层级不清
- 这个表单难填，是因为字段太多且没有分组
- 这个页面显得廉价，是因为字体、间距、颜色不统一

设计需要有理由，而不是只靠主观喜好。

### UI 是为任务服务的

页面不是海报。海报可以主要追求视觉冲击，但 UI 必须帮助用户完成任务。

例如外卖 App 首页最重要的是：

- 搜索餐厅或商品
- 选择地址
- 浏览分类
- 查看优惠
- 下单

如果首页做得很炫，但用户找不到搜索框，就是失败的设计。

### 先清晰，再美观

新手容易一开始就追求渐变、阴影、插画、动效。实际上 UI 的基础顺序应该是：

1. 信息是否完整
2. 层级是否清楚
3. 布局是否稳定
4. 操作是否顺畅
5. 视觉是否统一
6. 风格是否有吸引力

清晰是基础，美观是加分项。

### 一致性非常重要

同一个产品里，类似的元素应该保持一致。

例如：

- 所有主按钮使用同一种颜色
- 所有输入框高度一致
- 所有页面标题位置一致
- 所有错误提示样式一致
- 所有图标风格一致

一致性会降低用户的理解成本，也会让产品显得专业。

## 4. 用户与场景

UI 设计不是给自己看的，而是给特定用户在特定场景中使用的。

设计前要问：

- 用户是谁？
- 用户有什么目标？
- 用户在什么环境下使用？
- 用户熟不熟悉这个产品？
- 用户最关心什么？
- 用户最害怕什么？
- 用户可能犯什么错误？

### 用户类型示例

假设你设计一个记账 App：

- 学生：关心每月花了多少钱，操作要简单
- 上班族：关心预算、分类、报表
- 家庭用户：可能需要多人共同记账
- 财务敏感用户：关心隐私和数据安全

不同用户会影响页面重点。

### 使用场景示例

同样是记账：

- 用户刚花完钱，想快速记录
- 用户月底复盘，想看图表
- 用户发现余额不对，想查历史记录

快速记录场景需要少步骤；复盘场景需要清晰的数据展示。

## 5. 信息架构

信息架构是组织信息的方式。它决定用户能不能找到内容。

常见的信息组织方式：

- 按功能分类：订单、商品、用户、设置
- 按流程分类：选择商品、确认订单、支付、完成
- 按重要性分类：核心信息在前，次要信息在后
- 按频率分类：常用功能更容易触达

### 信息层级

信息层级就是页面里哪些内容最重要，哪些次要。

常用层级手段：

- 字号：重要内容更大
- 字重：重要内容更粗
- 颜色：重要内容对比更强
- 位置：重要内容更靠上或更靠近操作区
- 留白：重要内容周围更有呼吸感
- 分组：相关内容放在一起

### 新手练习

拿一个电商商品详情页，列出页面信息：

- 商品图片
- 商品名称
- 价格
- 优惠
- 评价
- 规格选择
- 配送信息
- 店铺信息
- 商品详情
- 推荐商品
- 加入购物车
- 立即购买

然后给它们排序。你会发现 UI 设计首先是信息排序，而不是画按钮。

## 6. 版式与布局

布局是 UI 的骨架。一个页面好不好用，首先看布局是否合理。

### 常见布局类型

#### 单列布局

适合移动端、文章、设置页、表单页。

特点：

- 阅读路径简单
- 易于适配小屏幕
- 信息按纵向排列

#### 双列布局

适合详情页、后台页面、编辑器。

常见形式：

- 左侧导航，右侧内容
- 左侧列表，右侧详情
- 左侧编辑区，右侧预览区

#### 卡片网格布局

适合商品、课程、图片、模板等重复内容。

注意：

- 卡片内容结构要一致
- 不要让卡片内部信息太乱
- 卡片之间间距要统一

#### 仪表盘布局

适合数据产品、后台系统。

特点：

- 展示关键指标
- 需要清楚的数据层级
- 避免过度装饰

### 页面布局基本原则

#### 先确定主任务

每个页面最好有一个最核心的任务。

例如：

- 登录页：让用户登录
- 支付页：让用户确认并付款
- 商品页：让用户了解商品并购买
- 后台列表页：让用户查找、筛选和处理数据

主任务决定布局重心。

#### 视觉流要自然

用户通常从上到下、从左到右浏览页面。重要信息应该顺着这个路径出现。

#### 内容要分组

相关内容放在一起，不相关内容分开。分组可以通过标题、间距、背景、分割线实现。

## 7. 间距系统

间距是 UI 设计中最容易被新手忽视、但影响极大的部分。

### 为什么间距重要

间距决定：

- 页面是否透气
- 信息是否清晰
- 元素之间关系是否明确
- 页面是否显得专业

### 常见间距单位

很多设计系统使用 4 或 8 的倍数作为基础单位。

常见间距：

- 4px：极小间距，例如图标和文字之间
- 8px：小间距，例如标签之间
- 12px：紧凑内容间距
- 16px：常规模块内边距
- 24px：模块之间间距
- 32px：大模块间距
- 48px：页面大区块间距

### 间距表达关系

距离越近，关系越强；距离越远，关系越弱。

例如：

- 标题和它下面的描述应该比较近
- 两个不同模块之间应该更远
- 按钮和它对应的内容应该接近

### 新手常见问题

- 所有间距都一样，导致分组不清
- 间距太小，页面拥挤
- 间距太大，页面松散
- 随意使用 13px、19px、27px，缺乏系统

建议从 8px 间距系统开始练习。

## 8. 对齐与栅格

对齐是让页面有秩序的基础。

### 常见对齐方式

- 左对齐：最常见，适合文本和表单
- 居中对齐：适合少量内容、标题、空状态
- 右对齐：适合数字、金额、表格操作
- 顶部对齐：适合卡片、列表项

### 为什么不要乱居中

新手喜欢把很多内容居中，但大量文本居中会降低阅读效率。

适合居中的情况：

- 登录页标题
- 空状态提示
- 成功结果页
- 少量说明文字

不适合居中的情况：

- 长段落
- 表单字段
- 列表内容
- 后台数据

### 栅格系统

栅格是页面布局的参考线。Web 设计常见 12 栏栅格，移动端常用 4 栏或基于边距的布局。

栅格帮助你：

- 保持页面整齐
- 控制内容宽度
- 适配不同屏幕
- 让多个页面布局统一

## 9. 字体与文字排版

文字是 UI 中最重要的信息载体。

### 字体选择

中文界面常见字体：

- 系统默认字体
- 苹方
- 思源黑体
- 微软雅黑
- HarmonyOS Sans

新手建议先使用系统字体，不要急着追求特殊字体。

### 字号层级

常见字号：

- 12px：辅助信息、标签
- 14px：正文、小型控件
- 16px：常规正文、移动端主要文本
- 18px：小标题
- 20px：模块标题
- 24px：页面标题
- 32px 以上：营销页或大标题

后台和工具型产品通常字号更克制，营销页可以更大。

### 字重

常见字重：

- Regular：普通正文
- Medium：强调信息
- Semibold：标题或重要按钮
- Bold：强标题，不宜滥用

不要整页全是粗体。粗体用多了，所有内容都不突出。

### 行高

行高影响阅读舒适度。

建议：

- 单行按钮文字：行高接近按钮高度即可
- 正文：字号的 1.4 到 1.7 倍
- 标题：字号的 1.2 到 1.4 倍

例如 16px 正文可以使用 24px 行高。

### 文案长度

UI 文案要短、清楚、具体。

不推荐：

> 您的操作发生了一些问题，请稍后再次进行相关尝试。

推荐：

> 保存失败，请稍后重试。

按钮文案要表达动作：

- 保存
- 提交
- 删除
- 继续
- 创建项目
- 发送验证码

## 10. 色彩基础

色彩不仅影响美观，也影响信息传达。

### 常见颜色角色

一个 UI 产品通常包含：

- 主色：品牌或主要操作颜色
- 辅助色：配合主色使用
- 背景色：页面底色
- 表面色：卡片、弹窗、输入框背景
- 文本色：标题、正文、辅助文字
- 边框色：分割和轮廓
- 成功色：成功状态
- 警告色：风险提醒
- 错误色：失败、危险操作
- 信息色：普通提示

### 不要只凭喜欢选颜色

颜色需要服务功能。

例如：

- 红色通常表示错误、危险、删除
- 绿色通常表示成功、可用、通过
- 黄色或橙色通常表示警告
- 蓝色通常表示链接、信息、主操作

不要把删除按钮设计成绿色，也不要把成功提示设计成红色，除非有特殊品牌原因。

### 对比度

文字和背景需要有足够对比度。浅灰文字放在白底上会很难读。

常见文本层级：

- 主要文字：接近黑色
- 次要文字：中深灰
- 辅助文字：中灰
- 禁用文字：浅灰

不要使用过浅的灰色承载重要信息。

### 新手配色建议

对于初学者，先用简单配色：

- 1 个主色
- 1 到 2 个辅助色
- 一组中性色
- 固定的成功、警告、错误颜色

不要一个页面使用太多高饱和颜色。

## 11. 图标与插画

图标用于快速表达功能或状态。

### 好图标的特点

- 含义清楚
- 风格统一
- 粗细一致
- 尺寸一致
- 与文字对齐

### 图标使用建议

- 常见操作可以只用图标，例如搜索、关闭、返回
- 不常见操作建议图标加文字
- 危险操作不要只依赖图标，最好配文字确认
- 图标不要为了装饰而乱放

### 图标尺寸

常见尺寸：

- 16px：小控件、表格
- 20px：普通按钮、导航
- 24px：移动端常用图标
- 32px 以上：功能入口或空状态

## 12. 图片与视觉素材

图片可以增强真实感和情绪，但图片质量很影响界面品质。

### 图片使用原则

- 图片要和内容相关
- 不要使用过度模糊、过暗、过度裁剪的图片
- 商品图片要能看清商品
- 人物图片要注意授权和隐私
- 背景图片上放文字时要保证可读性

### 图片比例

常见比例：

- 1:1：头像、商品缩略图
- 4:3：内容卡片
- 16:9：视频、横向封面
- 3:4：人物、海报

同一列表中的图片比例应保持一致，否则页面会显得杂乱。

## 13. 组件基础

组件是 UI 中可复用的界面单元。

常见组件：

- 按钮
- 输入框
- 复选框
- 单选框
- 开关
- 下拉菜单
- 标签
- 卡片
- 表格
- 弹窗
- 抽屉
- 导航栏
- 面包屑
- 分页
- 提示框

组件设计要考虑：

- 默认状态
- 悬停状态
- 按下状态
- 聚焦状态
- 禁用状态
- 加载状态
- 成功状态
- 错误状态

很多新手只画了默认状态，真正开发和使用时就会出问题。

## 14. 导航设计

导航帮助用户知道自己在哪里、还能去哪里。

### 常见导航类型

#### 顶部导航

适合网站、管理系统、内容平台。

特点：

- 横向展示
- 适合一级栏目不多的产品
- 视觉上比较稳定

#### 底部标签栏

适合移动 App。

特点：

- 通常 3 到 5 个入口
- 放最重要的一级功能
- 图标加文字更清楚

#### 侧边栏

适合后台系统、复杂工具。

特点：

- 可容纳较多菜单
- 适合长期操作
- 当前选中状态要明显

#### 面包屑

适合层级较深的网站和后台系统。

例如：

> 首页 / 商品管理 / 商品详情

面包屑帮助用户理解当前位置。

### 导航设计原则

- 当前页面要有明显选中状态
- 导航名称要清楚
- 不要把太多入口都放一级
- 高频功能优先
- 同一层级的导航样式要一致

## 15. 表单设计

表单是 UI 中非常重要也很容易出错的部分。

### 表单基本元素

- 标签
- 输入框
- 占位提示
- 帮助说明
- 错误提示
- 必填标记
- 提交按钮

### 标签

标签告诉用户这个字段要填什么。

建议：

- 标签要清楚，例如“手机号”“收货地址”
- 不要只依赖占位符当标签
- 长表单最好使用左对齐或顶部标签

### 占位符

占位符用于提供示例，不应该替代标签。

例如：

- 标签：手机号
- 占位符：请输入 11 位手机号

### 错误提示

错误提示要告诉用户：

- 哪里错了
- 为什么错
- 怎么改

不推荐：

> 输入错误

推荐：

> 手机号格式不正确，请输入 11 位手机号。

### 表单布局建议

- 字段按逻辑分组
- 必填项明确标记
- 提交按钮靠近表单底部
- 错误信息出现在对应字段附近
- 长表单可以分步骤

## 16. 按钮设计

按钮代表可执行动作。

### 按钮层级

常见按钮类型：

- 主按钮：页面最重要动作
- 次按钮：辅助动作
- 文本按钮：低强调动作
- 危险按钮：删除、撤销、清空等高风险动作

一个页面不要出现太多主按钮。主按钮太多等于没有重点。

### 按钮尺寸

按钮要足够容易点击。

常见高度：

- 32px：桌面端小按钮
- 36px：桌面端常规按钮
- 40px：通用按钮
- 44px 或更高：移动端按钮

### 按钮状态

按钮至少要考虑：

- 默认
- 悬停
- 按下
- 禁用
- 加载

加载状态很重要，尤其是提交表单、支付、上传文件时。

### 按钮文案

按钮文案应该是动作，而不是模糊描述。

推荐：

- 保存
- 发布
- 创建订单
- 下载文件

不推荐：

- 好的
- 确定一下
- 点我

## 17. 列表、卡片与表格

### 列表

列表适合展示同类内容。

列表项通常包含：

- 标题
- 描述
- 状态
- 时间
- 操作
- 缩略图

列表设计重点：

- 每一项结构一致
- 重要信息靠前
- 点击区域足够大
- 分割线不要过重

### 卡片

卡片适合把一组相关信息放在一个容器里。

适合使用卡片的场景：

- 商品
- 课程
- 文章
- 项目
- 数据概览

卡片注意事项：

- 不要把卡片当成万能装饰
- 卡片内部层级要清楚
- 卡片之间间距要一致
- 卡片内操作不要太多

### 表格

表格适合展示结构化数据。

表格设计重点：

- 表头清晰
- 数字右对齐
- 文本左对齐
- 状态用文字加颜色，不只靠颜色
- 操作列保持一致
- 支持筛选、排序、分页时要清楚

后台产品中，表格通常是核心组件，应该重视可读性和效率。

## 18. 状态设计

真实产品不是只有“正常页面”。还需要设计各种状态。

常见状态：

- 加载中
- 空数据
- 成功
- 失败
- 网络错误
- 权限不足
- 禁用
- 已选中
- 未选中
- 处理中

### 空状态

空状态不是简单写“暂无数据”。好的空状态可以告诉用户下一步做什么。

例如：

> 还没有项目，创建第一个项目开始管理任务。

旁边可以提供“创建项目”按钮。

### 加载状态

加载状态可以使用：

- 旋转加载图标
- 骨架屏
- 进度条
- 按钮内 loading

长时间操作最好显示进度或说明。

### 错误状态

错误状态要避免只说“失败”。应该说明问题和解决方式。

例如：

> 文件上传失败，文件不能超过 20MB。

## 19. 交互反馈

用户操作后需要知道系统是否收到操作。

常见反馈：

- 按钮按下效果
- 输入框聚焦边框
- 保存成功提示
- 删除确认弹窗
- 加载中状态
- 错误提示
- 选中状态

### 反馈要及时

用户点击按钮后，如果界面毫无变化，会怀疑是否点到了。

例如提交按钮点击后：

- 按钮变成 loading
- 禁用重复点击
- 成功后显示结果
- 失败后显示错误原因

### 反馈要适度

不是所有操作都需要弹窗。频繁弹窗会打断用户。

可以根据重要程度选择：

- 轻量操作：Toast 提示
- 可撤销操作：提示条加撤销
- 高风险操作：确认弹窗
- 复杂错误：页面内说明

## 20. 可用性与可访问性

可用性关注产品是否容易使用。可访问性关注更多人是否能使用，包括视力较弱、使用键盘、使用读屏软件的人。

### 基础可用性原则

- 用户知道当前在哪里
- 用户知道下一步能做什么
- 操作结果清楚
- 错误可以被理解和修正
- 常用功能容易找到
- 不让用户记太多东西

### 可访问性基础

需要注意：

- 文字对比度足够
- 字号不要太小
- 点击区域足够大
- 不只靠颜色传达信息
- 表单有清晰标签
- 键盘可以操作关键流程
- 图片有替代说明

例如错误状态不能只把输入框变红，还应该有文字提示。

## 21. 设计规范与设计系统

设计规范是为了让产品保持一致。

### 设计规范包含什么

- 色彩规范
- 字体规范
- 间距规范
- 圆角规范
- 阴影规范
- 图标规范
- 组件规范
- 页面模板
- 交互状态
- 文案规则

### 设计系统是什么

设计系统是更完整的规范体系，通常包含设计文件、组件库、代码组件、使用说明和团队协作规则。

常见设计系统：

- Material Design
- Apple Human Interface Guidelines
- Ant Design
- Fluent Design

新手不需要一开始就做完整设计系统，但要养成规范意识。

## 22. 移动端 UI 设计

移动端特点：

- 屏幕小
- 手指触控
- 使用场景碎片化
- 网络环境可能不稳定
- 用户耐心有限

### 移动端设计重点

- 主要操作要容易点击
- 底部区域适合放高频操作
- 减少输入
- 页面层级不要太深
- 内容优先级要更明确
- 注意安全区和系统导航栏

### 触控尺寸

移动端点击区域建议不小于 44px。图标可以是 24px，但实际可点击区域要更大。

### 单手操作

很多用户单手拿手机，屏幕底部和中下区域更容易点击。重要操作可以考虑放在底部。

## 23. Web UI 设计

Web 设计要考虑不同屏幕尺寸。

### 响应式设计

响应式设计是让页面适配不同设备。

常见断点：

- 手机
- 平板
- 小桌面
- 大桌面

响应式不是简单缩小页面，而是重新组织布局。

例如桌面端三列卡片，在手机端可能变成单列。

### Web 常见页面

- 首页
- 登录注册页
- 列表页
- 详情页
- 个人中心
- 设置页
- 管理后台

### Web 设计注意事项

- 鼠标悬停状态
- 键盘焦点状态
- 页面最大宽度
- 浏览器窗口缩放
- 不同分辨率
- 加载速度

## 24. 后台与工具型产品设计

后台系统和工具型产品强调效率、稳定和信息密度。

### 设计风格

后台不需要过度营销化。它应该：

- 清楚
- 克制
- 高效
- 易扫描
- 操作稳定

### 常见模块

- 数据看板
- 表格列表
- 筛选器
- 搜索
- 批量操作
- 详情页
- 编辑表单
- 权限管理
- 日志记录

### 后台设计重点

- 表格可读性
- 筛选逻辑清楚
- 批量操作安全
- 危险操作要确认
- 状态标签清晰
- 数据为空时有合理提示
- 长任务有进度反馈

## 25. 设计流程

一个完整的 UI 设计流程通常包括：

1. 理解需求
2. 分析用户和场景
3. 梳理信息架构
4. 画流程图
5. 画低保真线框图
6. 确定视觉风格
7. 设计高保真界面
8. 补充交互状态
9. 做原型演示
10. 评审和修改
11. 标注与交付
12. 跟进开发还原

### 低保真线框图

低保真线框图关注结构，不关注精细视觉。

它用于快速确认：

- 页面有什么内容
- 内容放在哪里
- 用户如何操作
- 页面之间如何跳转

### 高保真设计

高保真设计接近最终效果，需要包含：

- 准确文字
- 真实数据或接近真实的数据
- 完整视觉样式
- 组件状态
- 响应式考虑

### 交付

交付给开发时，不只是给几张图，还应该说明：

- 页面尺寸
- 间距
- 字号
- 颜色
- 组件状态
- 交互逻辑
- 异常状态
- 空状态
- 加载状态

## 27. 新手练习方法

### 第一阶段：临摹

临摹不是抄袭发布，而是学习结构和细节。

建议临摹：

- 登录页
- App 首页
- 商品详情页
- 设置页
- 后台列表页

临摹时不要只看效果，要分析：

- 页面用了几种字号
- 间距是多少
- 主按钮在哪里
- 信息如何分组
- 色彩如何分层
- 图标尺寸是否统一

### 第二阶段：拆解

找一个优秀页面，写出它的设计分析：

- 用户目标是什么
- 页面主任务是什么
- 信息层级如何安排
- 使用了哪些组件
- 哪些地方提升了效率
- 哪些地方可能有问题

### 第三阶段：改版

找一个你觉得不好用的页面，尝试改版。

步骤：

1. 找出问题
2. 写出改版目标
3. 重新整理信息
4. 画线框图
5. 做高保真设计
6. 对比改版前后

### 第四阶段：完整项目

自己设计一个小产品，例如：

- 记账 App
- 待办事项 App
- 博客后台
- 课程学习平台
- 个人作品集网站

每个项目至少包含：

- 首页
- 列表页
- 详情页
- 表单页
- 空状态
- 错误状态
- 加载状态

## 28. 常见错误

### 只追求好看

页面看起来漂亮，但用户不知道怎么操作，这是 UI 失败。

### 字号太多

一个页面出现 10 种字号，会显得混乱。新手建议控制在 3 到 5 种。

### 颜色太多

过多颜色会破坏层级。颜色应该有角色，而不是随意装饰。

### 间距不统一

间距混乱会让页面显得不专业。

### 没有状态设计

只设计正常页面，忽略加载、空、错、禁用状态，会影响真实产品体验。

### 图标风格不统一

线性图标、面性图标、不同粗细混用，会显得粗糙。

### 不考虑真实数据

设计时只放短文本，开发后遇到长标题、长用户名、空数据就会崩。

设计时要测试：

- 最短内容
- 最长内容
- 空内容
- 错误内容
- 多语言或特殊字符

## 29. 审美提升方法

### 多看优秀作品

可以关注：

- Apple 产品界面
- Google Material Design
- Airbnb
- Notion
- Linear
- Stripe
- Figma
- Ant Design
- 站酷、Dribbble、Behance 上的优秀作品

注意不要只收藏，要分析。

### 建立素材库

可以分类保存：

- 登录页
- 表单
- 空状态
- 导航
- 卡片
- 表格
- 数据看板
- 营销页
- 图标
- 配色

### 学会描述

不要只说“高级”“好看”。尝试用专业语言描述：

- 留白充足
- 对比明确
- 层级清楚
- 色彩克制
- 信息密度高
- 组件统一
- 品牌感强
- 阅读路径顺畅

会描述，才会理解。

## 30. 推荐学习路线

### 第 1 周：认识 UI

目标：

- 理解 UI、UX、视觉设计区别
- 熟悉常见页面和组件
- 学会观察页面信息层级

练习：

- 每天分析 3 个 App 页面
- 标出标题、正文、按钮、导航、状态

### 第 2 周：版式、间距、字体

目标：

- 掌握对齐、间距、字号、行高
- 学会用 8px 系统

练习：

- 临摹 3 个登录页
- 临摹 3 个设置页
- 尝试只用黑白灰做页面

### 第 3 周：色彩与组件

目标：

- 理解主色、辅助色、中性色
- 设计按钮、输入框、标签、卡片

练习：

- 做一套基础组件
- 给组件补齐默认、悬停、禁用、错误状态

### 第 4 周：页面设计

目标：

- 能独立设计简单页面
- 理解页面主任务

练习：

- 设计一个记账 App 的 5 个页面
- 包含首页、记录页、统计页、设置页、空状态

### 第 5 周：完整项目

目标：

- 形成一个可展示作品

练习：

- 做一个完整小项目
- 写清楚设计目标、用户场景、设计过程
- 输出高保真页面和原型

## UI 设计检查清单

完成一个页面后，可以用下面的问题检查：

- 这个页面的主任务是什么？
- 用户第一眼能看懂页面吗？
- 最重要的信息是否最明显？
- 按钮层级是否清楚？
- 文本是否容易阅读？
- 间距是否统一？
- 对齐是否整齐？
- 颜色是否有明确角色？
- 图标风格是否统一？
- 表单是否有清楚标签？
- 错误提示是否具体？
- 是否设计了加载状态？
- 是否设计了空状态？
- 是否设计了禁用状态？
- 长文本会不会撑破布局？
- 小屏幕是否还能使用？
- 危险操作是否有确认？
- 用户操作后是否有反馈？

## 最后建议

UI 设计入门最重要的不是记住所有概念，而是建立观察和判断能力。

你可以按照这个顺序训练：

1. 看懂页面结构
2. 分析信息层级
3. 模仿优秀作品
4. 总结设计规律
5. 独立完成小页面
6. 给页面补齐状态
7. 做完整项目
8. 反复修改和复盘

真正的 UI 能力来自大量观察、练习、对比和修改。每次做完一个页面，都问自己：用户能不能更快看懂？操作能不能更简单？视觉能不能更统一？这三个问题会一直帮助你进步。
