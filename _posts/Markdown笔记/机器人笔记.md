## 1. 机器人是什么

机器人是一种能够感知环境、做出决策并执行动作的机器系统。

可以用一句话理解：

```text
机器人 = 感知 + 决策 + 执行
```

例如扫地机器人：

- 感知：用碰撞传感器、激光雷达、摄像头感知环境
- 决策：判断哪里需要清扫、怎么避障、怎么回充
- 执行：控制轮子、电机、刷子运动

例如机械臂：

- 感知：识别物体位置
- 决策：规划抓取路径
- 执行：控制关节运动到目标位置

### 1.1 机器人不一定长得像人

很多人一听机器人，会想到人形机器人。但实际机器人种类非常多：

- 工业机械臂
- AGV / AMR 移动机器人
- 无人车
- 无人机
- 扫地机器人
- 医疗机器人
- 水下机器人
- 仓储分拣机器人
- 服务机器人
- 四足机器人
- 人形机器人

只要具备一定感知、决策和执行能力，都可以属于机器人范畴。

### 1.2 机器人和普通机器的区别

普通机器通常只按固定流程运行。

机器人更强调：

- 感知环境
- 根据环境变化调整行为
- 自主或半自主完成任务
- 与外部世界交互

例如：

- 普通电风扇：只按挡位转
- 智能移动机器人：能感知障碍物并绕开

## 2. 机器人系统由哪些部分组成

一个机器人系统通常包括：

| 模块 | 作用 | 例子 |
| :--- | :--- | :--- |
| 机械结构 | 支撑和运动载体 | 车架、机械臂、底盘 |
| 执行器 | 产生动作 | 电机、舵机、气缸 |
| 传感器 | 感知环境和自身状态 | 摄像头、雷达、IMU、编码器 |
| 控制器 | 运行程序和控制逻辑 | 单片机、工控机、嵌入式板 |
| 电源系统 | 提供能量 | 电池、电源管理 |
| 通信系统 | 模块之间交换数据 | CAN、UART、Ethernet、Wi-Fi |
| 软件系统 | 算法、控制、调度 | ROS 2、导航、视觉 |
| 算法模块 | 处理感知和决策 | SLAM、路径规划、目标检测 |

### 2.1 机器人数据流

一个典型流程：

```text
传感器采集数据
  -> 计算机处理数据
    -> 判断当前状态
      -> 规划下一步动作
        -> 控制电机或执行器
          -> 机器人运动
            -> 传感器再次反馈
```

这是一个闭环系统。

### 2.2 闭环的重要性

机器人不是发一个命令就结束，而是不断感知、修正、执行。

例如让机器人走直线：

- 给两个轮子相同速度
- 编码器反馈实际轮速
- 如果左轮慢，就增加左轮控制量
- 不断调整，保持直线

这就是反馈控制。

## 4. 机器人常见类型

### 4.1 移动机器人

可以在环境中移动。

例子：

- 扫地机器人
- 仓储 AMR
- 巡检机器人
- 送餐机器人

核心问题：

- 我在哪？
- 地图是什么？
- 目标在哪？
- 怎么过去？
- 怎么避障？

### 4.2 机械臂

通过多个关节移动末端执行器。

例子：

- 工业机械臂
- 协作机械臂
- 抓取机器人

核心问题：

- 末端在哪？
- 每个关节该转多少？
- 路径是否会碰撞？
- 如何平稳运动？

### 4.3 无人机

空中机器人。

核心问题：

- 姿态控制
- 定位
- 路径规划
- 避障
- 飞行安全

### 4.4 四足机器人

通过腿运动。

核心问题：

- 步态规划
- 平衡控制
- 地形适应
- 动力学控制

### 4.5 人形机器人

模仿人类结构。

核心问题：

- 双足平衡
- 全身控制
- 视觉理解
- 操作能力
- 人机交互

### 5.3 上位机和下位机

上位机：

- 运行高级算法
- 通常是 Linux 计算机
- 运行 ROS 2

下位机：

- 控制硬件
- 通常是 MCU
- 负责实时性更强的任务

例子：

```text
ROS 2 上位机发送 /cmd_vel
  -> MCU 接收速度命令
    -> 控制左右轮电机
      -> 读取编码器
        -> 返回 odom
```

## 6. 机械结构基础

机器人机械结构决定它能怎么动。

### 6.1 底盘

常见移动底盘：

- 差速底盘
- 阿克曼底盘
- 全向轮底盘
- 麦克纳姆轮底盘
- 履带底盘

### 6.2 差速底盘

两个主动轮分别控制。

特点：

- 结构简单
- 能原地旋转
- 常见于教学和服务机器人

控制：

- 左右轮同速：直行
- 左右轮反向：原地旋转
- 左右轮不同速：转弯

### 6.3 阿克曼底盘

类似汽车转向。

特点：

- 不能原地旋转
- 转弯半径有限
- 适合无人车

### 6.4 麦克纳姆轮

可以横向移动。

特点：

- 运动灵活
- 控制复杂
- 对地面要求较高

### 6.5 机械臂结构

由多个连杆和关节组成。

常见关节：

- 旋转关节
- 直线关节

核心参数：

- 自由度 DOF
- 臂展
- 负载
- 重复定位精度

## 7. 电机与执行器

执行器负责产生动作。

### 7.1 直流电机

特点：

- 简单
- 需要驱动器
- 速度可通过 PWM 控制

常用于：

- 小车轮子
- 风扇
- 简单机构

### 7.2 编码器电机

带编码器反馈。

编码器可以测：

- 位置
- 速度

用于闭环控制。

### 7.3 舵机

输入控制信号，转到指定角度。

常用于：

- 小机械臂
- 云台
- 机器人关节

### 7.4 步进电机

按步进方式转动。

特点：

- 位置控制方便
- 低速力矩大
- 可能丢步

常用于：

- 3D 打印机
- CNC
- 精密移动平台

### 7.5 无刷电机

效率高，常用于无人机和高性能机器人。

需要：

- 电调 ESC
- 复杂控制算法

## 11. 控制系统基础

控制系统让机器人按目标运动。

### 11.1 开环控制

没有反馈。

例子：

```text
给电机 50% PWM，让它转
```

问题：

- 负载变了，速度会变
- 地面摩擦变了，运动会变

### 11.2 闭环控制

有反馈。

例子：

```text
目标速度 1m/s
编码器测实际速度 0.8m/s
控制器增加输出
```

闭环更稳定。

### 11.3 PID

PID 是最常见控制算法。

包含：

- P：比例
- I：积分
- D：微分

直观理解：

- P：现在差多少
- I：过去累计差多少
- D：变化趋势有多快

常用于：

- 电机速度控制
- 位置控制
- 温度控制
- 姿态控制

### 11.4 控制频率

控制频率越稳定越好。

例如：

- 电机速度环：50Hz - 1000Hz
- 底盘控制：20Hz - 100Hz
- 导航规划：1Hz - 20Hz

具体取决于系统。

## 12. 坐标系与空间变换

机器人必须理解“位置”和“方向”。

### 12.1 坐标系

常见坐标系：

- 世界坐标系
- 地图坐标系
- 机器人本体坐标系
- 传感器坐标系
- 机械臂基座坐标系
- 末端执行器坐标系

### 12.2 2D 位姿

移动机器人常用：

```text
x, y, theta
```

其中：

- x：横向位置
- y：纵向位置
- theta：朝向角

### 12.3 3D 位姿

三维空间需要：

- 位置：x, y, z
- 姿态：roll, pitch, yaw 或四元数

### 12.4 坐标变换

传感器看到的物体，需要转换到机器人或地图坐标系。

例如：

```text
camera_link 中的物体坐标
  -> base_link
    -> map
```

ROS 2 中使用 TF2 管理坐标变换。

## 13. 运动学基础

运动学研究运动关系，不考虑力。

### 13.1 移动机器人运动学

差速底盘：

输入：

- 左轮速度
- 右轮速度

输出：

- 机器人线速度
- 机器人角速度

### 13.2 机械臂正运动学

已知每个关节角度，求末端位置。

```text
关节角度 -> 末端位姿
```

### 13.3 机械臂逆运动学

已知末端目标位置，求关节角度。

```text
末端目标位姿 -> 关节角度
```

逆运动学通常更难，可能：

- 无解
- 多解
- 接近奇异点

## 14. 动力学基础

动力学研究力和运动的关系。

涉及：

- 质量
- 惯性
- 力
- 力矩
- 摩擦
- 加速度

### 14.1 为什么需要动力学

对于简单小车，可以先不深入动力学。

对于：

- 四足机器人
- 高速机械臂
- 无人机
- 人形机器人

动力学非常重要。

### 14.2 力矩

机械臂选电机时要考虑力矩。

力矩不足会导致：

- 抬不动负载
- 控制不稳定
- 电机过热

## 17. 机器人感知

感知是机器人理解环境的能力。

### 17.1 感知任务

- 识别障碍物
- 检测目标
- 估计距离
- 识别语义
- 建立地图
- 判断自身姿态

### 17.2 感知数据类型

- 图像
- 深度图
- 点云
- 激光扫描
- IMU 数据
- 编码器数据
- 声音

### 17.3 感知难点

- 噪声
- 光照变化
- 遮挡
- 延迟
- 标定误差

## 20. 定位与建图 SLAM

SLAM 是 Simultaneous Localization and Mapping。

中文：

```text
同步定位与建图
```

机器人在未知环境中同时：

- 建立地图
- 估计自己位置

### 20.1 为什么 SLAM 难

因为：

- 传感器有噪声
- 里程计会漂移
- 环境可能重复
- 数据关联困难
- 回环检测复杂

### 20.2 常见 SLAM 类型

- 激光 SLAM
- 视觉 SLAM
- 激光视觉融合
- RGB-D SLAM
- 视觉惯性 SLAM

### 20.3 建图和定位区别

建图：

- 生成地图

定位：

- 在已有地图中确定位置

Nav2 常使用已有地图 + AMCL 定位。

## 21. 路径规划

路径规划回答：

```text
从当前位置到目标位置怎么走？
```

### 21.1 全局规划

在地图上规划整体路线。

常见算法：

- Dijkstra
- A*
- D*
- Hybrid A*

### 21.2 局部规划

根据实时障碍物和机器人状态生成短期运动。

常见：

- DWA
- TEB
- Pure Pursuit
- MPC

### 21.3 路径和平滑

规划出来的路径可能有折线，需要平滑才能更适合机器人运动。

## 22. 导航与避障

导航是移动机器人从当前位置到目标位置的完整过程。

包括：

### 22.1 避障

避障分为：

- 静态障碍物：地图中已有
- 动态障碍物：实时传感器检测

### 22.2 代价地图

代价地图把环境表示成网格，每个格子有代价。

代价高表示：

- 有障碍物

## 29. 机器人安全

机器人会运动，安全非常重要。

### 29.1 安全风险

- 撞人
- 撞物体
- 夹手
- 电池起火
- 电机失控
- 软件死机
- 通信中断

### 29.2 安全措施

- 急停按钮
- 限速
- 碰撞检测
- 保险丝
- 电池保护
- 看门狗
- 超时停止
- 安全距离
- 机械限位

### 29.3 软件安全

例如：

如果一段时间没有收到速度命令，底盘应自动停止。

```text
cmd_vel timeout -> stop motors
```

不要让机器人在通信断开后继续运动。

## 33. 常见误区

### 误区 1：一开始就学复杂算法

很多学生一开始就看 SLAM、强化学习、人形机器人论文，但基础硬件和坐标系都不懂。

正确：

先理解传感器、控制、坐标系、ROS，再深入算法。

### 误区 2：只会仿真，不碰硬件

仿真很重要，但真实机器人会遇到：

- 延迟
- 噪声
- 打滑
- 接线问题
- 电源问题
- 安全问题

必须做实物项目。

### 误区 3：只会调包

会运行 ROS 包不等于理解机器人。

要知道：

- 输入是什么
- 输出是什么
- 坐标系是什么
- 参数影响什么
- 出错怎么排查

### 误区 4：忽略安全

机器人会运动，必须考虑：

- 急停
- 限速
- 超时停止
- 电池保护

### 误区 5：只学一个方向

机器人是系统工程。即使你主攻算法，也要懂一些硬件和控制；即使你主攻硬件，也要懂软件和通信。

## 35. 总结

机器人学习可以用这条主线理解：

```text
机械结构提供身体
电子硬件提供神经和肌肉
传感器提供感知
控制算法提供稳定运动
软件系统组织模块
规划算法决定行动
AI 增强理解和决策
安全机制保证可靠运行
```

对于完全零基础学生，最稳的路线是：

1. 先学编程、Linux、电路和单片机
2. 做一个能动的小车
3. 加编码器做闭环速度控制
4. 学 ROS 2，把小车接入话题和 TF
5. 加激光雷达做建图
6. 用 Nav2 实现导航
7. 再选择机械臂、视觉、SLAM、无人机或人形机器人方向深入

机器人是典型的长期积累型学科。不要追求几天学完，也不要只看视频。真正的理解来自一个个项目：让电机转起来，让传感器读出来，让机器人知道自己在哪，让它安全地走到目标点。
