# 第 2 章 模型评估与选择

## 1. 本章目标

本章讲机器学习最重要的问题之一：如何判断模型好不好。

核心结论：

```text
训练集表现好不代表模型好，真正重要的是泛化性能。
```

## 2. 经验误差与泛化误差

### 经验误差

模型在训练集上的误差叫经验误差，也叫训练误差。

### 泛化误差

模型在新样本上的期望误差叫泛化误差。

机器学习真正关心的是泛化误差，但泛化误差不可直接看到，只能用测试集估计。

### 过拟合

过拟合指模型把训练集中的偶然性、噪声也学进去了。

表现：

- 训练误差很低。
- 测试误差较高。

常见原因：

- 模型太复杂。
- 训练数据太少。
- 特征太多。
- 噪声标签较多。

### 欠拟合

欠拟合指模型能力不足，连训练集规律都没学好。

表现：

- 训练误差高。
- 测试误差也高。

## 3. 评估方法

### 留出法

把数据集划分为训练集和测试集。

优点：简单。

缺点：

- 划分方式影响结果。
- 小数据集上不稳定。

建议：多次随机划分取平均。

### 交叉验证

把数据分成 k 份，每次用 k-1 份训练，1 份测试，重复 k 次。

常见 k 值是 5 或 10。

优点：

- 数据利用更充分。
- 评估更稳定。

缺点：

- 训练成本更高。

### 留一法

k 等于样本数，每次只留一个样本测试。

优点：训练集最接近原始数据集。

缺点：计算成本很高，结果方差也可能较大。

### 自助法

通过有放回采样构造训练集，未被采到的样本作为测试。

适合数据集较小、希望保留训练样本规模的情况。

## 4. 性能度量

不同任务需要不同评价指标。

### 回归指标

常用均方误差：

```text
MSE = 平均 (预测值 - 真实值)^2
```

它对大误差更敏感。

### 分类准确率

准确率是预测正确样本比例。

问题：类别不平衡时可能误导。

例如 99% 样本是负类，模型全部预测负类也有 99% 准确率，但没有实际价值。

### 查准率与查全率

以二分类为例：

- TP：真正例。
- FP：假正例。
- FN：假反例。
- TN：真反例。

查准率：

```text
Precision = TP / (TP + FP)
```

含义：预测为正的样本中，有多少是真的正。

查全率：

```text
Recall = TP / (TP + FN)
```

含义：真实正样本中，有多少被找出来。

### F1

F1 是查准率和查全率的调和平均：

```text
F1 = 2PR / (P + R)
```

适合需要平衡 precision 和 recall 的场景。

### ROC 与 AUC

ROC 曲线描述分类阈值变化时真正例率和假正例率的关系。

AUC 是 ROC 曲线下面积。

直觉：AUC 越大，模型把正样本排在负样本前面的能力越强。

## 5. 比较检验

模型评估结果会受数据划分和随机性影响。两个模型分数不同，不一定说明一个真的更好。

需要统计检验判断差异是否显著。

常见思想：

- 多次评估。
- 比较均值和方差。
- 判断差异是否可能由随机波动造成。

新手先记住：不要只看一次实验结果就下结论。

## 6. 偏差与方差

泛化误差可以从偏差、方差和噪声角度理解。

### 偏差

偏差表示模型预测的期望与真实规律之间的差距。

偏差高通常说明模型太简单，欠拟合。

### 方差

方差表示模型对训练数据变化的敏感程度。

方差高通常说明模型太复杂，过拟合。

### 噪声

噪声是数据本身不可避免的不确定性，例如标签错误、测量误差。

### 偏差-方差权衡

模型太简单：偏差高，方差低。

模型太复杂：偏差低，方差高。

好模型需要平衡。

## 7. 实战建议

1. 数据少时优先考虑交叉验证。
2. 类别不平衡时不要只看准确率。
3. 调参时使用验证集，不要污染测试集。
4. 对模型比较要多次实验。
5. 关注业务目标选择指标。例如医疗筛查更重视召回率，垃圾邮件过滤可能更重视查准率。

## 8. 本章复习问题

1. 训练误差和泛化误差有什么区别？
2. 过拟合和欠拟合分别是什么？
3. 为什么要划分训练集、验证集、测试集？
4. 交叉验证相比留出法有什么优势？
5. 准确率为什么可能误导？
6. Precision 和 Recall 各自关注什么？
7. 偏差和方差如何解释欠拟合与过拟合？

## 9. 精讲：为什么评估比训练还重要

很多新手会把注意力放在“如何训练模型”上，却忽略“如何评估模型”。这很危险，因为如果评估方法错了，后面所有结论都可能错。

例如你训练了一个模型，在训练集上准确率 99%。这听起来很好，但如果测试集准确率只有 60%，说明模型只是记住了训练集，并没有学到可推广的规律。

机器学习的目标不是“在已知题库上拿高分”，而是“在未来新题上也能做对”。所以评估必须模拟未来数据。

## 10. 精讲：训练集、验证集、测试集的分工

### 训练集

训练集用于拟合参数。

例如：

- 线性模型用训练集学习权重。
- 决策树用训练集选择划分特征。
- 神经网络用训练集反向传播。

### 验证集

验证集用于选择模型和调超参数。

例如：

- 决策树最大深度选多少？
- SVM 的 C 和核函数参数怎么设？
- 神经网络学习率多大？

这些选择不能直接看测试集，否则测试集就不再是“未知数据”。

### 测试集

测试集只在最后使用，用来估计最终模型的泛化性能。

正确流程是：

```text
训练集训练多个候选模型 -> 验证集选择模型和参数 -> 测试集最终评估一次
```

## 11. 精讲：过拟合的具体表现

过拟合可以用考试类比：

- 真正学会：理解知识点，换题也会。
- 过拟合：背下题库，原题全对，换题就错。

模型过拟合时通常出现：

```text
训练误差持续下降，验证误差先下降后上升
```

这说明模型开始学习训练集中的噪声。

常见解决方法：

- 增加数据。
- 降低模型复杂度。
- 加正则化。
- 提前停止。
- 数据增强。
- 特征选择。
- 交叉验证调参。

## 12. 精讲：混淆矩阵

二分类中，预测结果可以分为四种：

| 情况 | 含义 |
| --- | --- |
| TP | 真实为正，预测为正 |
| FP | 真实为负，预测为正 |
| FN | 真实为正，预测为负 |
| TN | 真实为负，预测为负 |

举例：癌症筛查。

- TP：病人有癌症，模型判断有癌症。
- FP：病人没癌症，模型误判有癌症。
- FN：病人有癌症，模型误判没癌症。
- TN：病人没癌症，模型判断没癌症。

在这个场景中，FN 代价非常高，因为漏诊可能危及生命。所以模型应更重视召回率。

## 13. 精讲：Precision 与 Recall 的取舍

Precision 高表示“报出来的正例比较可靠”。

Recall 高表示“真实正例尽量不漏掉”。

不同业务目标不同：

### 垃圾邮件过滤

如果把正常邮件误判成垃圾邮件，用户可能错过重要信息。因此需要较高 Precision，避免误杀。

### 癌症筛查

漏诊比误报更严重，因此需要较高 Recall，尽量把疑似病例找出来。

### 搜索引擎

用户更希望第一页结果准确，因此 Precision 很重要。

### 安全风控

可能更关注 Recall，先把高风险对象捞出来，再人工复核。

## 14. 精讲：ROC 和 AUC 怎么理解

很多模型输出的不是直接类别，而是一个分数。例如逻辑回归输出属于正类的概率。

不同阈值会得到不同分类结果。

阈值高：

- 预测为正更谨慎。
- Precision 可能提高。
- Recall 可能下降。

阈值低：

- 更容易预测为正。
- Recall 可能提高。
- FP 也可能增加。

ROC 曲线考察模型在不同阈值下的表现。AUC 可以理解为：随机抽一个正样本和一个负样本，模型把正样本排在负样本前面的概率。

AUC 越接近 1，排序能力越强；接近 0.5，则接近随机猜测。

## 15. 精讲：偏差-方差如何指导调参

如果模型欠拟合：

- 训练误差高。
- 验证误差也高。
- 偏差高。

可以尝试：

- 增加模型复杂度。
- 增加特征。
- 减小正则化强度。
- 训练更久。

如果模型过拟合：

- 训练误差低。
- 验证误差高。
- 方差高。

可以尝试：

- 增加数据。
- 降低模型复杂度。
- 增强正则化。
- 删除噪声特征。
- 使用集成方法。

## 16. 本章实践清单

做任何机器学习实验时，都应记录：

1. 数据如何划分。
2. 是否有数据泄漏。
3. 使用了哪些指标。
4. 是否多次随机实验。
5. 训练集、验证集、测试集分别表现如何。
6. 是否存在类别不平衡。
7. 模型选择是否使用了测试集。

只要这张清单没做好，模型分数再高也不可信。
