# 第 4 章 决策树

## 1. 本章目标

决策树是一种用树形结构进行判断的模型。

它的形式非常直观：

```text
先看特征 A，再看特征 B，最后得到类别。
```

例如判断西瓜：

- 色泽是否青绿？
- 根蒂是否蜷缩？
- 敲声是否浊响？
- 最后判断好瓜或坏瓜。

## 2. 决策树基本流程

构造决策树通常是递归过程：

1. 从当前样本集合中选择一个最优划分特征。
2. 根据该特征的不同取值划分子集。
3. 对每个子集继续构造子树。
4. 满足停止条件时生成叶节点。

叶节点给出最终类别或预测值。

## 3. 什么是好划分

好的划分应该让子节点更“纯”。

纯的意思是：一个节点中的样本尽量属于同一类。

如果划分前节点里好瓜坏瓜混在一起，划分后每个子节点几乎都是同类，那么这个划分就好。

## 4. 信息熵与信息增益

信息熵衡量混乱程度。

若一个集合中类别越混杂，熵越大；类别越单一，熵越小。

信息增益衡量一次划分让混乱程度减少了多少。

```text
信息增益 = 划分前熵 - 划分后加权平均熵
```

ID3 算法倾向选择信息增益最大的特征。

## 5. 增益率

信息增益偏好取值较多的特征。

例如“编号”这种特征可能把每个样本单独分出去，信息增益很大，但没有泛化价值。

增益率用来缓解这个问题。

C4.5 使用增益率选择划分。

## 6. 基尼指数

基尼指数衡量从样本集中随机抽两个样本，类别不一致的概率。

基尼指数越小，集合越纯。

CART 决策树常用基尼指数选择划分。

## 7. 剪枝

决策树很容易过拟合。如果树长得太深，可能把训练集细节甚至噪声都记住。

剪枝用于控制复杂度。

### 预剪枝

在树生成过程中提前停止。

例如：

- 节点样本数太少就不再划分。
- 信息增益太小就不再划分。
- 树深达到上限就停止。

优点：训练快，防止过拟合。

缺点：可能过早停止，导致欠拟合。

### 后剪枝

先生成完整树，再自底向上删除不必要分支。

优点：通常比预剪枝更稳。

缺点：训练成本更高。

## 8. 连续值处理

连续特征不能直接像离散特征那样按取值分支。

常见做法是选择一个划分点 t：

```text
特征 <= t
特征 > t
```

算法会尝试候选划分点，选择效果最好的。

## 9. 缺失值处理

现实数据常有缺失。

处理思路：

- 忽略缺失样本。
- 用均值、众数填补。
- 按概率把样本分配到多个分支。
- 学习时对缺失样本降低权重。

核心是不要让缺失值破坏整体划分逻辑。

## 10. 决策树优缺点

优点：

- 易理解。
- 可解释性强。
- 能处理离散和连续特征。
- 对特征缩放不敏感。
- 能表达非线性边界。

缺点：

- 容易过拟合。
- 对数据扰动敏感。
- 贪心生成不保证全局最优。
- 单棵树泛化能力有限。

## 11. 与集成学习的关系

随机森林、GBDT、XGBoost 等强大模型都以决策树为基础。

单棵树可能不稳定，但很多树组合起来可以非常强。

## 13. 精讲：决策树为什么直观

决策树像人做判断时的一连串问题。

例如判断一个西瓜是否好：

```text
根蒂是否蜷缩？
  是 -> 敲声是否浊响？
       是 -> 好瓜
       否 -> 坏瓜
  否 -> 色泽是否青绿？
       是 -> 继续判断
       否 -> 坏瓜
```

这种模型的好处是非常容易解释。你不仅知道模型给出的结果，还能看到它为什么这样判断。

这也是决策树在金融风控、医疗辅助、业务规则分析中长期受欢迎的原因。

## 14. 精讲：递归划分的过程

构造决策树时，每一步都在问：

```text
当前应该用哪个特征划分样本，才能让结果更清楚？
```

假设当前节点里有 10 个好瓜、10 个坏瓜，很混乱。如果按“敲声”划分后：

- 浊响分支：9 个好瓜、1 个坏瓜。
- 清脆分支：1 个好瓜、9 个坏瓜。

划分后每个分支都更纯，这就是好划分。

如果按“编号”划分，每个编号只有一个样本，训练集纯度最高，但对新样本没有意义。这就是为什么要防止过拟合。

## 15. 精讲：信息熵怎么理解

信息熵衡量不确定性。

如果一个节点里全是好瓜，你完全确定类别，熵为 0。

如果一半好瓜一半坏瓜，你最不确定，熵最大。

决策树选择划分时，希望划分后不确定性下降最多。

这就像猜谜游戏：一个好问题应该大幅缩小可能答案范围。

## 16. 精讲：信息增益、增益率、基尼指数的区别

### 信息增益

看一次划分让熵减少多少。

优点：直观。

缺点：偏好取值很多的特征。

### 增益率

在信息增益基础上惩罚取值过多的特征。

它试图避免“编号”这类无意义但划分很细的特征。

### 基尼指数

基尼指数衡量集合不纯度。CART 常用基尼指数。

基尼指数计算相对简单，工程中常用。

三者都在追求同一个目标：让划分后的节点更纯。

## 17. 精讲：剪枝为什么必要

如果不剪枝，决策树可以一直划分到每个叶节点只有极少样本，甚至一个样本。

这样训练误差可能为 0，但模型只是记住了训练集。

例如某个叶节点规则是：

```text
色泽=青绿 且 根蒂=蜷缩 且 编号=17 -> 好瓜
```

这条规则对训练样本有效，但对未来西瓜没有价值。

剪枝就是让树不要过度追求训练集纯度，而要保留真正可泛化的结构。

## 18. 精讲：连续值划分举例

假设特征是密度，取值：

```text
0.24, 0.31, 0.43, 0.57, 0.62
```

决策树会尝试候选阈值，例如：

```text
密度 <= 0.275
密度 <= 0.37
密度 <= 0.50
密度 <= 0.595
```

每个阈值都能把样本分成两部分。算法选择让纯度提升最大的阈值。

所以连续特征在树中通常变成二分判断。

## 19. 精讲：缺失值不是简单删除

现实数据常有缺失，例如某个西瓜没有记录“敲声”。

简单删除会浪费数据，尤其数据少时影响很大。

更合理的思路包括：

- 用最常见取值填补。
- 用同类样本均值填补。
- 根据其他特征预测缺失值。
- 在决策树中按概率分配到多个分支。

核心原则是：缺失值本身也可能包含信息。例如某些检查没做，可能与病人状况或业务流程有关。

## 20. 精讲：决策树和 if-else 规则的关系

训练好的决策树可以转成一组规则。

每条从根到叶的路径就是一条规则：

```text
如果 条件1 且 条件2 且 条件3，则 类别 = C
```

这让决策树具有强可解释性。

但也要注意：可解释不等于一定正确。树如果过拟合，解释也可能只是对训练集噪声的解释。

## 21. 实战建议

1. 单棵树适合解释，不一定适合追求最高精度。
2. 控制最大深度、叶节点最小样本数可以防止过拟合。
3. 类别特征多时注意高基数特征。
4. 表格数据上可以尝试随机森林、GBDT、XGBoost 等树集成模型。
5. 决策树对特征缩放不敏感，不像 SVM、kNN、神经网络那么依赖标准化。
