# 第 7 章 贝叶斯分类器

## 1. 本章目标

本章从概率角度理解分类。

核心思想：

```text
选择后验概率最大的类别。
```

也就是看到样本 x 后，判断它最可能属于哪个类别。

## 2. 贝叶斯决策论

贝叶斯决策论提供了概率意义下的最优决策框架。

对于分类问题，我们关心：

```text
P(类别 | 样本特征)
```

这个概率叫后验概率。

如果损失相同，选择后验概率最大的类别即可。

## 3. 贝叶斯公式

贝叶斯公式：

```text
P(c | x) = P(x | c) P(c) / P(x)
```

含义：

- P(c)：先验概率，类别本身出现的概率。
- P(x | c)：似然，类别 c 下观察到 x 的概率。
- P(x)：证据，样本 x 出现的概率。
- P(c | x)：后验概率，看到 x 后属于 c 的概率。

分类时 P(x) 对所有类别相同，可忽略，只比较：

```text
P(x | c) P(c)
```

## 4. 极大似然估计

极大似然估计用于估计概率模型参数。

直觉：

```text
选择一组参数，使已经观察到的数据出现概率最大。
```

例如估计硬币正面概率，如果 10 次投掷有 7 次正面，极大似然估计通常给出 0.7。

## 5. 朴素贝叶斯

朴素贝叶斯做了一个强假设：

```text
给定类别后，各个特征条件独立。
```

于是：

```text
P(x | c) = P(x1 | c) * P(x2 | c) * ... * P(xd | c)
```

这个假设通常不完全真实，但模型简单、效率高，实践中常有效。

### 应用场景

- 文本分类。
- 垃圾邮件识别。
- 情感分析。
- 简单医疗辅助判断。

## 6. 拉普拉斯平滑

如果某个特征取值在训练集中从未出现，朴素贝叶斯会给出 0 概率，导致整个连乘为 0。

拉普拉斯平滑通过给每种情况加一个小计数，避免零概率。

直觉：没见过不代表不可能。

## 7. 半朴素贝叶斯

朴素贝叶斯的独立性假设太强。半朴素贝叶斯放松这个假设，允许特征之间有一定依赖。

它在模型复杂度和表达能力之间折中。

## 8. 贝叶斯网

贝叶斯网用有向无环图表示变量之间的概率依赖关系。

节点表示变量，边表示依赖。

优点：

- 能表达复杂条件依赖。
- 结构直观。
- 可进行概率推断。

难点：

- 结构学习困难。
- 精确推断可能计算复杂。

## 9. EM 算法

EM 用于含有隐变量的参数估计。

两个步骤：

- E 步：根据当前参数估计隐变量分布。
- M 步：根据估计的隐变量更新参数。

不断迭代，直到收敛。

典型应用：

- 高斯混合模型。
- 缺失数据估计。
- 聚类。

## 11. 精讲：贝叶斯思维是什么

贝叶斯方法的核心是：看到新证据后，更新我们对事件的相信程度。

例如判断一个西瓜是不是好瓜：

- 先验：在所有西瓜中，好瓜比例可能是 60%。
- 证据：这个西瓜色泽青绿、敲声浊响。
- 后验：看到这些特征后，它是好瓜的概率可能上升到 85%。

这就是：

```text
原来的相信程度 + 新证据 -> 更新后的相信程度
```

## 12. 精讲：贝叶斯公式逐项解释

公式：

```text
P(c | x) = P(x | c) P(c) / P(x)
```

逐项解释：

- P(c)：先验。还没看样本特征前，类别 c 本身有多常见。
- P(x | c)：似然。如果类别是 c，出现这些特征的可能性有多大。
- P(x)：证据。出现这些特征的总体概率。
- P(c | x)：后验。看到特征 x 后，属于 c 的概率。

分类时，因为 P(x) 对所有类别一样，所以比较：

```text
P(x | c) P(c)
```

哪个类别大，就选哪个。

## 13. 精讲：朴素贝叶斯为什么“朴素但有效”

朴素贝叶斯假设：给定类别后，各特征相互独立。

这个假设明显很强。例如在文本中，“机器”和“学习”并不独立，它们经常一起出现。

但朴素贝叶斯仍然常有效，原因是：

1. 它估计参数少，小数据也能训练。
2. 分类只需要比较大小，不一定需要概率完全准确。
3. 在文本分类中，词频特征虽然相关，但整体统计仍能提供强信号。

所以它是一个“假设简单但实用”的模型。

## 14. 精讲：文本分类中的朴素贝叶斯

假设要判断邮件是否垃圾邮件。

模型会估计：

```text
P(垃圾邮件)
P(正常邮件)
P(免费 | 垃圾邮件)
P(中奖 | 垃圾邮件)
P(会议 | 正常邮件)
...
```

如果一封邮件包含“免费、中奖、点击、领取”，这些词在垃圾邮件中出现概率高，则后验概率会偏向垃圾邮件。

这就是朴素贝叶斯在垃圾邮件过滤中有效的原因。

## 15. 精讲：拉普拉斯平滑为什么必要

假设训练集中从未在垃圾邮件里见过某个词“区块链”，那么：

```text
P(区块链 | 垃圾邮件) = 0
```

如果一封邮件出现这个词，整个连乘概率变成 0。这样太极端，因为没见过不代表不可能。

拉普拉斯平滑相当于给每个词都加一个很小的默认计数，让概率不会变成 0。

## 16. 精讲：生成式模型与判别式模型

朴素贝叶斯是生成式模型。

它试图建模：

```text
P(x | c) 和 P(c)
```

再用贝叶斯公式得到分类结果。

逻辑回归是判别式模型。

它直接建模：

```text
P(c | x)
```

生成式模型关注“这个类别如何生成样本”；判别式模型关注“给定样本如何划分类别”。

## 17. 精讲：EM 算法用一个例子理解

假设有两枚硬币 A 和 B，但每次投掷记录中没有告诉你用的是哪枚硬币，只记录了正反面结果。你想估计两枚硬币各自正面概率。

困难在于：硬币身份是隐变量。

EM 思路：

- E 步：根据当前参数猜每组数据更可能来自哪枚硬币。
- M 步：根据这个猜测重新估计硬币正面概率。

不断重复，直到参数稳定。

EM 的本质是：在隐变量未知时，交替估计隐变量和模型参数。

## 18. 本章实践建议

1. 文本分类可以先尝试朴素贝叶斯作为基线。
2. 连续特征要假设分布，例如高斯朴素贝叶斯。
3. 注意零概率问题，使用平滑。
4. 特征强相关时，朴素贝叶斯概率解释不一定可靠，但分类仍可能有效。
5. 如果需要表达复杂依赖，可考虑贝叶斯网或其他模型。
