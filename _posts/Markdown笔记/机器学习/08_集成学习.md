# 第 8 章 集成学习

## 1. 本章目标

集成学习研究如何把多个学习器组合成更强模型。

核心思想：

```text
多个弱模型合理组合，可能胜过单个强模型。
```

## 2. 个体学习器与集成

个体学习器是参与组合的单个模型。

集成模型把多个个体学习器的输出结合起来。

要让集成有效，个体学习器通常需要：

- 准确性不太差。
- 彼此有差异。

如果所有模型都犯同样错误，组合没有意义。

## 3. Boosting

Boosting 是串行集成方法。

核心思想：

```text
后一个模型重点学习前一个模型没学好的样本。
```

### AdaBoost

AdaBoost 会提高被错误分类样本的权重，让后续分类器更关注难样本。

最终预测时，表现更好的分类器有更大投票权。

直觉：像一轮轮补课，哪里错得多，下一轮重点补哪里。

### Boosting 特点

优点：

- 能显著提升弱学习器性能。
- 偏差降低明显。

缺点：

- 对噪声和异常值敏感。
- 串行训练，不易并行。

## 4. Bagging

Bagging 是并行集成方法。

核心思想：

1. 从训练集中有放回采样，得到多个子训练集。
2. 每个子训练集训练一个模型。
3. 多个模型投票或平均。

Bagging 主要降低方差。

## 5. 随机森林

随机森林是 Bagging 加决策树，再引入特征随机性。

每棵树训练时：

- 样本随机。
- 特征也随机选择子集。

这样树之间差异更大，集成效果更好。

优点：

- 效果强。
- 抗过拟合能力较好。
- 能处理高维数据。
- 可估计特征重要性。

缺点：

- 模型较大。
- 可解释性弱于单棵树。

## 6. 结合策略

### 平均法

回归任务常用。

多个模型输出取平均。

### 投票法

分类任务常用。

- 绝对多数投票。
- 相对多数投票。
- 加权投票。

### 学习法

用另一个模型学习如何组合多个模型输出。这类方法常称为 stacking。

## 7. 多样性

集成学习的关键不只是模型数量，而是多样性。

制造多样性的方法：

- 不同训练样本。
- 不同特征子集。
- 不同算法。
- 不同参数。
- 不同初始化。

## 8. Boosting 与 Bagging 对比

| 维度 | Boosting | Bagging |
| --- | --- | --- |
| 训练方式 | 串行 | 并行 |
| 关注点 | 纠正前面错误 | 降低模型波动 |
| 主要降低 | 偏差 | 方差 |
| 对噪声 | 较敏感 | 较稳健 |
| 代表 | AdaBoost、GBDT | 随机森林 |

## 9. 本章复习问题

1. 集成学习为什么可能提升性能？
2. 个体学习器为什么需要多样性？
3. Boosting 的核心思想是什么？
4. Bagging 如何构造多个训练集？
5. 随机森林比普通 Bagging 树多了什么随机性？
6. Boosting 和 Bagging 分别主要降低偏差还是方差？

## 10. 精讲：为什么多个模型组合会更强

一个模型可能因为训练数据、算法偏好或噪声而犯错。多个模型如果错误不完全相同，组合后就可能互相纠正。

可以类比考试投票：

- 如果 10 个学生都独立做题，每个人都有 70% 正确率。
- 如果他们错误不完全一样，多数投票可能超过 70%。

集成学习成功的关键不是“模型越多越好”，而是：

```text
个体模型要有一定准确性，并且错误要有差异。
```

## 11. 精讲：Bagging 为什么降低方差

决策树很容易受训练数据扰动影响。训练集稍微变一点，树结构可能完全不同。这叫高方差。

Bagging 通过有放回采样构造多个不同训练集，每个训练集训练一棵树。最后投票或平均。

直觉：

```text
单棵树很容易摇摆，多棵树平均后更稳定。
```

这就是 Bagging 主要降低方差的原因。

## 12. 精讲：随机森林的两个随机性

随机森林比普通 Bagging 更进一步。

### 样本随机

每棵树使用 bootstrap 采样得到的数据。

### 特征随机

每个节点划分时，不看全部特征，只随机选择一部分特征候选。

这样可以让树之间差异更大，减少它们犯相同错误的概率。

## 13. 精讲：Boosting 为什么降低偏差

Boosting 是一轮一轮训练模型。

第一轮模型可能很粗糙。第二轮会更关注第一轮分错的样本。第三轮继续关注前面没学好的样本。

这像老师批改作业后安排补课：

- 哪类题错得多，下次重点练。
- 难样本权重越来越高。
- 多轮后整体能力提升。

因此 Boosting 能把多个弱学习器组合成强学习器，主要降低偏差。

## 14. 精讲：AdaBoost 的工作直觉

AdaBoost 每轮做两件事：

1. 调整样本权重：错分样本权重提高。
2. 调整模型权重：表现好的学习器投票权更大。

最终预测时，不是所有模型一票相同，而是更可靠的模型话语权更大。

## 15. 精讲：GBDT 与 Boosting 的关系

虽然本章重点可能是 AdaBoost，但实际工程中 GBDT 更常见。

GBDT 的思想是：每一棵新树去拟合前面模型的残差或负梯度。

直觉：

```text
当前模型还差什么，下一棵树就补什么。
```

XGBoost、LightGBM、CatBoost 都是强大的梯度提升树系统，在表格数据上非常常用。

## 16. 精讲：多样性从哪里来

集成模型需要多样性。多样性来源包括：

- 训练样本不同。
- 特征子集不同。
- 模型类型不同。
- 参数不同。
- 数据扰动不同。
- 随机初始化不同。

随机森林主要靠样本随机和特征随机制造多样性。

Boosting 主要靠每轮关注样本不同制造差异。

## 17. 精讲：什么时候用 Bagging，什么时候用 Boosting

如果基础模型方差高，例如决策树容易受数据影响，可以用 Bagging 或随机森林稳定它。

如果基础模型偏差高，单个弱模型能力不足，可以用 Boosting 逐步增强。

简单经验：

- 随机森林：稳健、调参少、并行方便。
- GBDT 类：精度强、调参更重要、对表格数据非常强。
- AdaBoost：对噪声较敏感。

## 18. 本章实践建议

1. 表格分类回归可优先尝试随机森林和 GBDT。
2. 数据噪声大时，Boosting 可能追着噪声学。
3. 随机森林适合作为强基线。
4. 集成不是越大越好，要考虑训练和推理成本。
5. 用特征重要性时要注意相关特征会分摊重要性。
