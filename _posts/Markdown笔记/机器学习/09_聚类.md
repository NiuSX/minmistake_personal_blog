# 第 9 章 聚类

## 1. 本章目标

聚类是无监督学习任务，目标是在没有标签的情况下把相似样本分到同一组。

一句话：

```text
类内相似，类间不同。
```

## 2. 聚类与分类的区别

分类有标签，模型学习已知类别。

聚类无标签，算法自己发现数据中的组。

例子：

- 分类：判断邮件是否垃圾邮件。
- 聚类：把用户按行为自动分群。

## 3. 聚类性能度量

因为没有真实标签，聚类评估更难。

常见评估：

### 外部指标

如果有参考标签，可比较聚类结果和参考划分。

例如 Rand 指数、调整 Rand 指数等。

### 内部指标

没有标签时，根据簇内紧密度和簇间分离度评估。

直觉：

- 同一簇样本距离要小。
- 不同簇之间距离要大。

## 4. 距离计算

聚类高度依赖距离度量。

常见距离：

- 欧氏距离。
- 曼哈顿距离。
- 闵可夫斯基距离。
- 余弦相似度。

选择距离要符合数据含义。

例如文本向量常用余弦相似度，地理位置可用球面距离。

## 5. 原型聚类

原型聚类用一个原型代表每个簇。

### k-means

k-means 是最经典聚类算法。

流程：

1. 选择 k 个初始中心。
2. 把每个样本分到最近中心。
3. 重新计算每个簇的中心。
4. 重复 2、3，直到稳定。

优点：

- 简单。
- 快速。
- 易实现。

缺点：

- 需要预先指定 k。
- 对初始中心敏感。
- 偏好球形簇。
- 对异常值敏感。

## 6. 密度聚类

密度聚类认为簇是高密度区域。

### DBSCAN

DBSCAN 根据邻域密度发现簇。

核心概念：

- 核心对象：邻域内样本数足够多。
- 密度直达。
- 密度可达。
- 噪声点。

优点：

- 不需要预设簇数。
- 能发现任意形状簇。
- 能识别噪声。

缺点：

- 参数选择敏感。
- 不适合密度差异很大的数据。

## 7. 层次聚类

层次聚类生成树状聚类结构。

### 自底向上

一开始每个样本是一个簇，然后逐步合并最近的簇。

### 自顶向下

一开始所有样本属于一个簇，然后逐步拆分。

优点：

- 可得到不同层级结构。
- 不一定预先确定簇数。

缺点：

- 计算成本较高。
- 合并或拆分一旦完成通常不可撤销。

## 8. 聚类实战建议

1. 先标准化特征，否则尺度大的特征会支配距离。
2. 根据业务解释选择距离。
3. k-means 适合球形、规模较大数据。
4. DBSCAN 适合有噪声和非球形簇的数据。
5. 聚类结果要结合可视化和业务解释。

## 10. 精讲：聚类到底在“学”什么

聚类没有标签，因此它不是学习“标准答案”，而是在数据中发现结构。

例如一个电商平台有很多用户行为数据：

- 浏览次数。
- 下单频率。
- 客单价。
- 优惠券使用率。
- 品类偏好。

我们不知道每个用户属于什么类型，但希望算法自动分群，例如：

- 高价值稳定用户。
- 价格敏感用户。
- 新用户。
- 流失风险用户。

这就是聚类的价值：让隐藏在数据中的群体结构浮现出来。

## 11. 精讲：距离决定了“相似”的含义

聚类的核心是相似性，而相似性通常由距离度量决定。

如果距离定义错了，聚类一定会错。

例如：

- 对地理位置，应该考虑经纬度距离。
- 对文本，常用余弦相似度。
- 对用户消费行为，要先处理不同特征尺度。

假设一个用户特征有“年龄”和“年收入”：

```text
年龄：20 到 60
年收入：30000 到 1000000
```

如果不标准化，收入的数值范围远大于年龄，距离几乎完全由收入决定，年龄被淹没。

所以聚类前通常必须标准化。

## 12. 精讲：k-means 为什么简单又常用

k-means 的思想非常朴素：

```text
每个簇用一个中心代表，样本归到最近中心。
```

它反复做两件事：

1. 分配样本：每个样本找最近中心。
2. 更新中心：每个簇重新计算平均点。

这两个步骤交替进行，直到中心基本不变。

### k 的选择

k-means 必须预先给出簇数 k。

常见方法：

- 肘部法：观察误差随 k 增加的下降曲线。
- 轮廓系数：衡量样本与本簇和其他簇的距离关系。
- 业务解释：聚类结果是否可用、可解释。

### 初始中心问题

k-means 对初始中心敏感。初始点不同，结果可能不同。

常用改进是 k-means++，让初始中心尽量分散。

## 13. 精讲：k-means 的适用边界

k-means 偏好：

- 球形簇。
- 大小相近的簇。
- 密度相近的簇。

它不擅长：

- 弯月形簇。
- 环形簇。
- 密度差异大的簇。
- 噪声很多的数据。

如果数据形状复杂，DBSCAN 或谱聚类可能更合适。

## 14. 精讲：DBSCAN 如何理解

DBSCAN 不用中心点代表簇，而是看密度。

它认为：

```text
样本密集连成片的地方就是簇，稀疏地方是边界或噪声。
```

两个关键参数：

- eps：邻域半径。
- MinPts：邻域内最少样本数。

如果一个点周围 eps 范围内至少有 MinPts 个点，它就是核心点。

从核心点出发，把密度相连的区域扩展成簇。

### DBSCAN 优点

- 不需要指定簇数。
- 能发现任意形状簇。
- 能识别噪声点。

### DBSCAN 缺点

- 参数 eps 和 MinPts 敏感。
- 如果不同簇密度差异很大，效果会变差。

## 15. 精讲：层次聚类的价值

层次聚类不仅给出一个分组，还给出分组层级。

例如动物可以层次化：

```text
动物
  哺乳动物
    猫
    狗
  鸟类
    麻雀
    鹰
```

这种结构适合需要层级解释的场景，例如：

- 文档主题层级。
- 生物分类。
- 客户群体分层。

## 16. 精讲：聚类结果如何解释

聚类完成后，不要只看算法输出的簇编号。簇编号本身没有含义。

需要做画像：

1. 每个簇有多少样本？
2. 每个簇的特征均值/分布是什么？
3. 每个簇与其他簇最大差异在哪里？
4. 业务上能否命名？
5. 聚类是否稳定？

例如一个用户簇可以命名为“高频低价用户”，另一个叫“低频高客单用户”。

## 17. 本章实践清单

1. 聚类前检查缺失值和异常值。
2. 对数值特征标准化。
3. 根据数据类型选择距离。
4. 尝试可视化降维后查看结构。
5. 不要迷信聚类指标，业务解释很重要。
6. 多次运行检查结果稳定性。
