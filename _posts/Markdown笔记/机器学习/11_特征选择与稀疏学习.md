# 第 11 章 特征选择与稀疏学习

## 1. 本章目标

本章研究如何从大量特征中选择有用特征，以及如何学习稀疏表示。

核心问题：

```text
不是所有特征都有用，太多无关特征会损害学习。
```

## 2. 为什么需要特征选择

特征过多会带来：

- 训练成本高。
- 模型复杂。
- 过拟合风险大。
- 可解释性差。
- 噪声特征干扰判断。

特征选择可以提高泛化能力和可解释性。

## 3. 子集搜索与评价

理想情况下，可以枚举所有特征子集，选择最优子集。

但如果有 d 个特征，子集数量是 2^d，无法穷举。

因此需要启发式搜索：

- 前向搜索：从空集开始逐步加入特征。
- 后向搜索：从全集开始逐步删除特征。
- 双向搜索：加入和删除结合。

## 4. 过滤式选择

过滤式方法先根据统计指标选择特征，再训练模型。

常见指标：

- 方差。
- 相关系数。
- 卡方检验。
- 信息增益。
- 互信息。

优点：

- 快。
- 与具体模型无关。

缺点：

- 没有直接针对最终模型优化。
- 特征之间组合效应可能被忽略。

## 5. 包裹式选择

包裹式方法把学习器性能作为特征子集评价标准。

流程：

1. 选一个特征子集。
2. 训练模型。
3. 用验证性能评价子集。
4. 搜索更好的子集。

优点：更贴近最终模型效果。

缺点：计算成本高。

## 6. 嵌入式选择

嵌入式方法在模型训练过程中自动完成特征选择。

典型方法是 L1 正则化。

### L1 正则化

L1 会促使部分参数变成 0。

参数为 0 的特征相当于被删除。

### L2 正则化

L2 会让参数变小，但通常不直接变成 0。

它更像控制复杂度，而不是明确选择特征。

## 7. 稀疏表示

稀疏表示希望用少量基向量表示样本。

直觉：复杂数据可能由少数基本模式组合而成。

例如图像可以由少量边缘、纹理、局部结构组合。

## 8. 字典学习

字典学习同时学习：

- 一组基向量，也叫字典。
- 每个样本在字典上的稀疏系数。

目标是用少量字典元素重构数据。

## 9. 压缩感知

压缩感知说明：如果信号本身是稀疏的，可以用远少于传统采样要求的观测恢复信号。

核心前提：

- 信号可稀疏表示。
- 测量矩阵满足一定条件。

## 10. 本章复习问题

1. 为什么特征太多可能伤害模型？
2. 过滤式、包裹式、嵌入式特征选择有什么区别？
3. L1 为什么能产生稀疏参数？
4. L1 和 L2 的作用有什么不同？
5. 什么是稀疏表示？
6. 字典学习要学习什么？

## 11. 精讲：为什么不是特征越多越好

很多新手以为特征越多，模型信息越充分，效果一定越好。实际不一定。

无关特征会带来三个问题：

1. 增加噪声。
2. 增加模型复杂度。
3. 增加过拟合风险。

例如预测房价时，“房屋面积、地段、楼层”可能有用，但“房主身份证尾号”通常没有用。如果把无意义特征放进模型，模型可能在训练集中碰巧发现某些虚假规律。

这叫伪相关。

## 12. 精讲：特征选择的三种思路

### 过滤式

先不管模型，直接用统计指标筛特征。

例子：

- 方差太小的特征删除。
- 与标签相关性低的特征删除。
- 卡方检验筛选文本特征。

优点：快。

缺点：可能忽略特征组合效果。

### 包裹式

把模型训练效果作为特征子集评分。

例如尝试不同特征组合，看验证集表现。

优点：更贴近最终任务。

缺点：计算成本高。

### 嵌入式

训练模型时自动完成选择。

典型例子是 L1 正则化和决策树特征重要性。

## 13. 精讲：L1 为什么能让参数变成 0

L1 正则化在损失函数中加入参数绝对值之和：

```text
原损失 + λ * |w| 的总和
```

它会鼓励模型使用更少特征。很多特征对应的权重会被压到 0。

直觉：

```text
每使用一个特征都要付费，模型会只保留最值得付费的特征。
```

L2 正则化也惩罚参数，但更倾向于让参数变小，而不是变成 0。

## 14. 精讲：稀疏性的价值

稀疏表示指大量元素为 0，只有少数元素非零。

稀疏有几个好处：

- 可解释。
- 存储节省。
- 计算高效。
- 降低过拟合。
- 突出关键因素。

例如文本分类中，词表可能有十万个词，但一篇文章只包含其中几百个词。文本天然就是稀疏的。

## 15. 精讲：字典学习用积木理解

字典学习可以理解为学习一组“积木”，再用少量积木组合出样本。

例如图像中的小块可能由：

- 横线。
- 竖线。
- 斜线。
- 角点。
- 纹理。

组合而成。

字典学习要同时学习：

1. 哪些积木最有用。
2. 每个样本用哪些积木组合。

如果每个样本只用少量积木，就是稀疏表示。

## 16. 精讲：压缩感知为什么反直觉

传统采样认为，要恢复一个信号，需要足够高的采样率。

压缩感知说：如果信号在某个空间中是稀疏的，就可以用更少测量恢复。

这听起来反直觉，但关键在于“稀疏”。

如果一个信号虽然维度很高，但真正非零成分很少，那么我们不需要完整观察所有维度，也可能恢复它。

## 17. 实战建议

1. 建模前先删除明显无意义 ID 特征。
2. 对高维稀疏文本可使用 L1 或树模型筛特征。
3. 特征选择必须只在训练集内完成，避免数据泄漏。
4. 选择特征后要在验证集上确认效果。
5. 可解释性要求高时，稀疏模型通常更合适。
