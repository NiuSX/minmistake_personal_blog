# 特征工程实战清单

特征工程是把原始数据变成模型更容易学习的表示。它不是随意造列，而是根据任务目标提取有预测价值的信息。

## 一、数值特征

检查内容：

1. 是否有缺失值。
2. 是否有异常值。
3. 是否偏态严重。
4. 是否量纲差异很大。
5. 是否需要标准化。

常见处理：

```python
from sklearn.preprocessing import StandardScaler, MinMaxScaler
from sklearn.impute import SimpleImputer
```

对偏态特征可以尝试：

```python
import numpy as np
X["amount_log"] = np.log1p(X["amount"])
```

## 二、类别特征

低基数类别可以 one-hot：

```python
from sklearn.preprocessing import OneHotEncoder
```

高基数类别要谨慎。例如用户 ID、商品 ID 直接 one-hot 可能造成维度爆炸和过拟合。

类别特征要处理训练集未出现、测试集出现的新类别。`OneHotEncoder(handle_unknown="ignore")` 可以避免报错。

## 三、时间特征

时间字段通常不能直接丢给模型。可以拆出：

1. 年、月、日。
2. 星期几。
3. 是否周末。
4. 小时。
5. 是否节假日。
6. 距离某个事件的时间差。

时间序列任务还可以构造滞后特征和滚动统计：

```text
过去 7 天平均值
过去 30 天最大值
上一次行为距今天数
```

## 四、文本特征

传统方法：

1. 词袋模型。
2. TF-IDF。
3. n-gram。

示例：

```python
from sklearn.feature_extraction.text import TfidfVectorizer

vectorizer = TfidfVectorizer(max_features=5000, ngram_range=(1, 2))
X_text = vectorizer.fit_transform(texts)
```

## 五、交互特征

有些规律不是单个特征决定，而是组合决定。

例如：

```text
客单价 = 总消费金额 / 订单数
点击转化率 = 购买次数 / 点击次数
距离上次登录天数 = 当前日期 - 最近登录日期
```

交互特征要有业务含义，不要盲目组合所有列。

## 六、防止数据泄漏

特征工程中最危险的是泄漏。

常见泄漏：

1. 使用预测时不可获得的未来信息。
2. 在全数据上做标准化再划分。
3. 用测试集参与特征选择。
4. 用标签直接或间接构造特征。
5. 时间任务中随机划分导致未来信息进入训练。

防泄漏原则：任何会从数据中学习统计量的步骤，都只能在训练集上拟合。

## 七、特征工程复盘问题

每个新特征都问：

1. 预测时能不能获得？
2. 是否有业务含义？
3. 是否可能泄漏标签？
4. 是否在验证集上真的提升？
5. 是否增加过多复杂度？
