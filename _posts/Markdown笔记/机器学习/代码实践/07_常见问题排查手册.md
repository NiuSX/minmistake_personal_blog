# 常见问题排查手册

机器学习实践中，模型效果不好并不一定是算法不够高级。多数问题来自数据、评估、特征、泄漏和流程。

## 一、训练分数高，测试分数低

可能原因：

1. 过拟合。
2. 模型太复杂。
3. 数据量太少。
4. 特征泄漏导致验证方式不真实。
5. 训练集和测试集分布不同。

处理方法：

1. 降低模型复杂度。
2. 加强正则化。
3. 增加数据。
4. 使用交叉验证。
5. 检查数据划分方式。
6. 做特征选择或降维。

## 二、训练分数和测试分数都低

可能原因：

1. 欠拟合。
2. 特征信息不足。
3. 模型太简单。
4. 标签噪声大。
5. 任务本身定义不清。

处理方法：

1. 换更强模型。
2. 增加有效特征。
3. 调整目标定义。
4. 检查标签质量。
5. 做误差分析。

## 三、交叉验证分数波动很大

可能原因：

1. 样本量太少。
2. 数据分布不均。
3. 类别不平衡。
4. 存在异常样本。
5. 模型对数据划分敏感。

处理方法：

1. 使用分层抽样。
2. 增加折数或重复交叉验证。
3. 检查异常值。
4. 使用更稳定模型。
5. 收集更多数据。

## 四、准确率很高但实际没用

常见于类别不平衡。

例如 99% 样本是负类，模型全预测负类也有 99% 准确率，但正类完全找不到。

处理方法：

1. 看混淆矩阵。
2. 看查准率、查全率、F1。
3. 调整阈值。
4. 使用类别权重。
5. 关注业务代价。

## 五、模型上线后效果下降

可能原因：

1. 训练数据和线上数据分布不同。
2. 用户行为变化。
3. 数据采集逻辑变化。
4. 特征缺失或延迟。
5. 标签反馈延迟。

处理方法：

1. 监控输入特征分布。
2. 监控预测分布。
3. 定期重新训练。
4. 建立数据质量告警。
5. 保留人工抽检流程。

## 六、模型无法解释

处理方法：

1. 先训练一个简单可解释基线。
2. 查看线性模型系数。
3. 查看树模型特征重要性。
4. 做部分依赖分析。
5. 用错误样本解释模型行为。

复杂模型并不是不能用，但在高风险场景中必须能回答“为什么这样预测”。

## 七、排查顺序

推荐顺序：

1. 检查数据和标签。
2. 检查划分方式。
3. 检查是否泄漏。
4. 建立简单基线。
5. 看训练和验证差距。
6. 做误差分析。
7. 再考虑换模型或调参。
