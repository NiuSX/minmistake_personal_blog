# 从公式到代码的理解方法

很多新手觉得公式和代码是两套东西：书上是数学，项目里是调用库。其实代码就是公式的工程实现。学会把公式翻译成代码，机器学习才会真正连起来。

## 一、先分清数据、参数和目标

看到公式先标三类东西：

1. 数据：训练集中已知的 `X` 和 `y`。
2. 参数：模型要学习的 `w`、`b`、树结构、中心点等。
3. 目标：要最小化或最大化的函数。

例如线性回归：

```text
y_hat = w^T x + b
L = sum_i (y_i - y_hat_i)^2
```

数据是 `x_i` 和 `y_i`，参数是 `w` 和 `b`，目标是让平方误差最小。

## 二、公式到 numpy 的对应

线性模型预测：

```python
y_pred = X @ w + b
```

平方误差：

```python
loss = ((y - y_pred) ** 2).mean()
```

这两行代码就对应了线性回归的核心公式。

## 三、梯度下降的代码直觉

梯度下降做的事情：

```text
参数 = 参数 - 学习率 * 梯度
```

伪代码：

```python
for step in range(num_steps):
    y_pred = X @ w + b
    loss = ((y - y_pred) ** 2).mean()

    grad_w = ...
    grad_b = ...

    w = w - lr * grad_w
    b = b - lr * grad_b
```

重点不是一开始背梯度公式，而是理解：梯度告诉参数往哪个方向会让损失上升，所以减去梯度就是往损失下降的方向走。

## 四、逻辑回归公式到代码

sigmoid：

```python
import numpy as np

def sigmoid(z):
    return 1 / (1 + np.exp(-z))
```

预测概率：

```python
prob = sigmoid(X @ w + b)
```

分类：

```python
y_pred = (prob >= 0.5).astype(int)
```

这对应“线性打分 -> sigmoid 转概率 -> 阈值分类”。

## 五、K-means 公式到代码

K-means 的目标是让样本靠近所属中心。

核心步骤：

```python
for step in range(max_iter):
    # 计算每个样本到每个中心的距离
    # 把样本分配给最近中心
    # 重新计算每个簇的中心
```

这对应公式：

```text
min sum_i ||x_i - mu_{c_i}||^2
```

算法不是直接一次求最优，而是在分配和更新中心之间交替下降目标函数。

## 六、正则化到代码

如果原始损失是：

```python
loss = data_loss
```

加入 L2 正则后：

```python
loss = data_loss + lambda_ * (w ** 2).sum()
```

直觉：模型如果想用很大的权重降低训练误差，就要付出额外代价。

## 七、为什么库函数看不到公式

调用 sklearn 时，很多公式被封装起来了：

```python
model.fit(X_train, y_train)
```

`fit` 内部完成了参数学习。不同模型的 `fit` 方法内部算法不同：线性模型可能用优化算法，决策树用贪心划分，K-means 用交替迭代，随机森林训练多棵树。

学习时不要停在 `fit`，要知道 `fit` 背后在优化什么。

## 八、公式学习的最终目标

你不需要把每个算法都从零手写一遍，但要能做到：

1. 看到损失函数知道模型在惩罚什么。
2. 看到正则项知道它在限制什么。
3. 看到超参数知道它影响什么。
4. 看到代码能对应回公式。
5. 模型出问题时知道从目标、数据、优化还是评估排查。
