# 论文阅读入门路线

机器学习论文不适合从第一行读到最后一行。新手应按结构阅读，先抓问题和贡献，再看方法细节。

## 一、先读标题

标题通常告诉你三个信息：

1. 任务是什么。
2. 方法是什么。
3. 主要卖点是什么。

看到标题时先猜：这篇论文解决什么问题，可能和你学过的哪一章有关。

## 二、读摘要

摘要通常包含：

1. 背景和问题。
2. 现有方法不足。
3. 作者提出的方法。
4. 实验结果。
5. 主要贡献。

读摘要时，不懂细节没关系，先用一句话概括论文。

## 三、读引言

引言最重要。它解释为什么这个问题值得研究。

重点标记：

1. Problem：问题是什么？
2. Challenge：难点是什么？
3. Contribution：贡献是什么？
4. Difference：和已有方法差别是什么？

如果引言读不懂，方法部分通常更难懂。

## 四、看方法总图

很多论文有模型结构图或流程图。先看图，不要急着看公式。

问：

1. 输入是什么？
2. 输出是什么？
3. 中间模块有哪些？
4. 每个模块解决什么问题？
5. 损失函数是什么？

## 五、读公式

公式不要逐符号硬啃。按以下顺序：

1. 找优化目标。
2. 找模型输出。
3. 找损失函数。
4. 找正则项或约束。
5. 找训练流程。

如果公式太复杂，先理解它是分类损失、重构损失、对比损失、概率似然，还是价值函数。

## 六、读实验

实验部分要看：

1. 用了哪些数据集。
2. 和哪些 baseline 比较。
3. 使用了哪些指标。
4. 是否有消融实验。
5. 是否报告显著性或稳定性。
6. 是否有失败案例。

不要只看最高分。要看实验设计是否公平。

## 七、读结论和局限

结论通常总结贡献。局限部分说明方法不适合什么情况。很多论文不会充分强调局限，需要你自己判断：

1. 数据集是否太小？
2. baseline 是否弱？
3. 是否只在特定场景有效？
4. 计算成本是否过高？
5. 是否缺少解释性或鲁棒性分析？

## 八、新手论文笔记模板

```text
论文标题：
年份/会议：
研究任务：
核心问题：
主要方法：
关键公式：
使用数据集：
评价指标：
对比方法：
主要结果：
创新点：
局限：
和我学过的章节关系：
我还不懂的问题：
```

## 九、推荐阅读顺序

1. 先读综述、教程和博客。
2. 再读经典算法论文。
3. 再读改进型论文。
4. 最后读最新论文。

基础不牢时直接读最新论文，容易只记住模型名字，看不懂真正贡献。
