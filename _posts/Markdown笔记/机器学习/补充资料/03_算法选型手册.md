# 算法选型手册

算法选型不是背“哪个算法最好”，而是根据任务、数据、样本量、特征类型、可解释性要求和计算资源做折中。

## 一、先按任务类型筛选

如果有标签，并且标签是离散类别，这是分类问题。常用算法包括逻辑回归、决策树、随机森林、朴素贝叶斯、SVM、神经网络、梯度提升树。

如果有标签，并且标签是连续值，这是回归问题。常用算法包括线性回归、岭回归、Lasso、回归树、随机森林回归、梯度提升树、神经网络。

如果没有标签，希望发现结构，这是聚类问题。常用算法包括 K-means、层次聚类、DBSCAN、高斯混合模型。

如果样本维度太高，希望压缩或可视化，这是降维问题。常用方法包括 PCA、核 PCA、流形学习、t-SNE、UMAP。

如果任务是连续决策，并且动作会影响未来状态，这是强化学习问题。

## 二、小样本优先考虑什么

小样本场景最怕过拟合。优先考虑简单模型、强正则、可解释模型。

推荐起点：

1. 逻辑回归或线性回归。
2. 朴素贝叶斯，尤其文本分类。
3. 小深度决策树。
4. SVM，尤其特征维度高但样本不太多时。

不建议一开始就用大型神经网络。样本少时，复杂模型很容易记住噪声。

## 三、大样本优先考虑什么

大样本允许使用更复杂模型，但计算成本也更重要。

推荐起点：

1. 线性模型作为强基线。
2. 随机森林或梯度提升树处理表格数据。
3. 神经网络处理图像、语音、文本等非结构化数据。
4. mini-batch 梯度下降类方法处理超大规模数据。

## 四、表格数据常用选择

表格数据中，树模型通常很强，尤其是随机森林和梯度提升树。原因是它们能处理非线性、特征交互、不同尺度特征，对异常值也相对稳健。

如果要求解释性强，可以从逻辑回归、线性模型、浅层决策树、规则模型开始。

如果类别特征很多，需要注意编码方式。线性模型通常依赖 one-hot 或目标编码，树模型对编码方式也敏感。

## 五、文本数据常用选择

传统文本分类可以从朴素贝叶斯和线性模型开始。文本特征通常高维稀疏，线性模型在这种场景很有效。

如果有足够数据和计算资源，可以使用深度学习或预训练语言模型。但从学习机器学习原理的角度，先理解词袋模型、TF-IDF、朴素贝叶斯和逻辑回归非常重要。

## 六、图像数据常用选择

传统方法可以使用手工特征加 SVM，但现代图像任务通常使用卷积神经网络或视觉 Transformer。对于学习基础，重点理解：图像的局部结构、参数共享、层级特征提取。

## 七、可解释性要求高时

优先考虑：

1. 线性模型。
2. 决策树。
3. 规则学习。
4. 朴素贝叶斯。
5. 加上特征重要性分析的树模型。

神经网络和复杂集成模型可以通过解释工具辅助理解，但解释成本更高。

## 八、模型选择常见路线

一个务实路线：

1. 先做数据清洗和合理划分。
2. 建立简单基线，例如逻辑回归、线性回归、朴素贝叶斯。
3. 尝试更强模型，例如随机森林、梯度提升、SVM。
4. 根据验证集指标选择模型。
5. 做误差分析，看错在哪类样本。
6. 再决定是改特征、调参数、换模型，还是补数据。

## 九、不要这样选算法

不要因为某算法“高级”就优先使用。

不要只看训练集表现。

不要在测试集上反复调参。

不要忽视业务指标。

不要把模型选择和数据泄漏混在一起。

## 十、速查表

| 场景 | 推荐起点 | 注意事项 |
|---|---|---|
| 二分类表格数据 | 逻辑回归、随机森林 | 看类别不平衡和解释性要求 |
| 多分类文本 | 朴素贝叶斯、线性模型 | 高维稀疏特征通常适合线性模型 |
| 小样本高维 | SVM、L1 正则线性模型 | 防止过拟合 |
| 非线性表格 | 树模型、集成学习 | 注意调参和验证 |
| 无标签分群 | K-means、层次聚类、DBSCAN | 先明确相似度定义 |
| 可视化 | PCA、t-SNE、UMAP | 可视化不能当严格证明 |
| 序列决策 | 强化学习 | 先定义状态、动作、奖励 |
