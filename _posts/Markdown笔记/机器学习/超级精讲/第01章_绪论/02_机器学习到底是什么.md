# 机器学习到底是什么：万字级精讲入门版

## 1. 先从一个生活问题开始

假设你站在水果摊前，要挑一个好西瓜。

你可能会看：

- 色泽是不是青绿。
- 根蒂是不是蜷缩。
- 敲起来声音是不是浊响。
- 纹理是不是清晰。
- 瓜脐是不是凹陷。

你为什么会根据这些信息判断西瓜好坏？

因为你过去见过很多西瓜，吃过好瓜，也吃过坏瓜。你逐渐形成了一套经验：某些外观和声音更可能对应好瓜。

这个过程就是人类的经验学习。

机器学习想做的事情类似：

```text
把过去的经验数据交给计算机，让计算机总结规律，再用规律判断新情况。
```

在西瓜问题中：

- 过去见过的西瓜记录就是数据。
- 色泽、根蒂、敲声等是特征。
- 好瓜/坏瓜是标签。
- 总结出的判断规律是模型。
- 让计算机总结规律的方法是学习算法。

机器学习并不是让计算机像人一样“有感觉”，而是让计算机通过数据找到某种可计算的映射关系。

## 2. Mitchell 定义怎么理解

机器学习中常引用一个形式化定义：

```text
如果一个程序在任务 T 上，随着经验 E 的增加，性能 P 得到改善，
就说该程序从经验 E 中学习。
```

这句话有三个关键词：

- T：任务。
- E：经验。
- P：性能。

理解机器学习时，最好永远问这三个问题。

### 2.1 T：任务是什么

任务是你希望机器完成的事情。

例如：

- 判断邮件是不是垃圾邮件。
- 识别图片里是不是猫。
- 预测明天股票涨跌。
- 给用户推荐商品。
- 判断一个病人是否高风险。
- 预测房价。
- 自动驾驶中决定下一步方向盘怎么打。

不同任务决定了模型输出形式。

分类任务输出类别；回归任务输出数值；排序任务输出顺序；强化学习任务输出动作。

### 2.2 E：经验是什么

经验在机器学习中通常表现为数据。

例如垃圾邮件识别中，经验是过去大量邮件及其标签：

```text
邮件内容 -> 垃圾/正常
```

房价预测中，经验是过去房屋成交记录：

```text
面积、位置、楼层、年代、学区 -> 成交价
```

推荐系统中，经验是用户历史行为：

```text
用户点击、收藏、购买、停留时长、评分
```

经验不是越多越好，而是要与任务相关、质量可靠、分布匹配。

### 2.3 P：性能是什么

性能是评价模型做得好不好的标准。

例如：

- 分类任务可以用准确率。
- 医疗筛查可能更重视召回率。
- 搜索排序可能看点击率或 NDCG。
- 回归任务可能看均方误差。
- 推荐系统可能看转化率。

性能指标非常重要，因为模型会朝着指标优化。

如果指标选错，模型可能“看起来变好”，但实际业务变差。

例如癌症筛查中，如果只看准确率，模型可能把所有人预测为健康，因为大多数人确实没病，准确率很高。但这个模型毫无价值，因为它漏掉了真正病人。

## 3. 机器学习的基本工作流

一个典型机器学习项目可以拆成以下流程：

```text
明确任务 -> 收集数据 -> 构造特征 -> 选择模型 -> 训练模型 -> 评估模型 -> 部署使用 -> 持续监控
```

### 3.1 明确任务

首先要知道问题是什么。

很多失败项目不是算法不行，而是任务定义不清。

例如“提升用户体验”不是一个清晰的机器学习任务。

更清晰的任务可能是：

```text
预测用户未来 7 天是否会流失。
```

这里：

- 输入：用户过去行为。
- 输出：是否流失。
- 标签：未来 7 天是否未登录。
- 指标：AUC、召回率、干预转化率。

### 3.2 收集数据

数据要覆盖模型未来会遇到的情况。

如果训练数据只来自夏天，模型可能不适合冬天。

如果训练数据只来自一线城市，模型可能不适合三四线城市。

这叫训练分布和应用分布不一致。

### 3.3 构造特征

特征是模型能看到的信息。

传统机器学习中，特征工程非常关键。

例如预测用户流失，原始数据是日志，不能直接喂给普通模型。需要构造：

- 最近 1 天登录次数。
- 最近 7 天购买次数。
- 最近 30 天消费金额。
- 距离上次登录天数。
- 优惠券使用次数。

模型只能从特征中学习。如果关键特征没有提供，再强的模型也难以学好。

### 3.4 选择模型

模型选择取决于：

- 数据规模。
- 特征类型。
- 是否需要解释。
- 训练和预测成本。
- 任务复杂度。

例如：

- 小数据、强解释：逻辑回归、决策树。
- 表格数据、追求效果：随机森林、GBDT。
- 高维文本：线性模型、朴素贝叶斯、深度模型。
- 图像语音：深度神经网络。

### 3.5 训练模型

训练就是让模型参数适应数据。

线性模型学习权重；决策树学习划分结构；神经网络学习大量连接权重。

训练的核心通常是最小化损失函数。

### 3.6 评估模型

评估必须使用模型没见过的数据。

否则无法判断泛化能力。

常见划分：

- 训练集。
- 验证集。
- 测试集。

### 3.7 部署和监控

模型上线后，数据分布可能变化。

例如用户行为改变、市场环境变化、攻击方式变化，都会让模型效果下降。

所以机器学习不是一次性工作，而是持续迭代。

## 4. 机器学习到底学到了什么

很多人会问：模型到底学到了什么？

答案取决于模型类型。

### 4.1 线性模型学权重

线性模型学习每个特征的重要性。

例如房价预测中：

- 面积权重大，说明面积影响大。
- 距离地铁权重为负，说明越远价格越低。

### 4.2 决策树学规则

决策树学习一系列 if-else 判断。

例如：

```text
如果 纹理=清晰 且 敲声=浊响，则 好瓜
```

### 4.3 贝叶斯模型学概率

贝叶斯模型学习：

```text
某类别出现概率是多少？
某类别下某特征出现概率是多少？
```

### 4.4 神经网络学表示

神经网络学习多层特征表示。

图像中，低层可能学边缘，高层学部件，更高层学物体。

文本中，模型可能学习词义、句法、上下文关系。

## 5. 机器学习和传统编程的区别

传统编程是：

```text
人写规则 + 输入数据 -> 输出结果
```

机器学习是：

```text
输入数据 + 输出答案 -> 学出规则
```

然后使用时：

```text
新输入 + 学出的规则 -> 新输出
```

例如垃圾邮件识别。

传统编程会写规则：

```text
如果邮件包含“中奖”“免费”“点击领取”，则可能是垃圾邮件。
```

但规则永远写不完。垃圾邮件会变化，用词会伪装。

机器学习做法是给模型大量邮件和标签，让模型自己学习哪些词、结构、发送行为与垃圾邮件有关。

## 6. 机器学习不是万能的

机器学习适合有数据规律的问题。

如果问题没有稳定规律，模型很难学。

例如：

- 标签随机。
- 数据质量极差。
- 未来分布和过去完全不同。
- 关键因素没有被观测。
- 样本太少但问题复杂。

机器学习也不能自动理解业务目标。目标定义错了，模型会优化错误方向。

## 7. 学习算法、模型、参数、超参数

### 7.1 学习算法

学习算法是训练模型的方法。

例如：

- 梯度下降。
- 决策树生成算法。
- SVM 优化算法。
- EM 算法。

### 7.2 模型

模型是学到的规律。

例如：

- 一棵具体决策树。
- 一条具体回归直线。
- 一个训练好的神经网络。

### 7.3 参数

参数是在训练中学出来的。

例如线性模型中的 w 和 b。

### 7.4 超参数

超参数是训练前人为设置的。

例如：

- 决策树最大深度。
- kNN 中的 k。
- SVM 中的 C。
- 神经网络学习率。

超参数通常通过验证集调节。

## 8. 监督学习、无监督学习、强化学习

### 8.1 监督学习

数据带标签。

模型学输入到标签的映射。

例子：

```text
图片 -> 猫/狗
房屋信息 -> 房价
邮件 -> 垃圾/正常
```

### 8.2 无监督学习

数据没有标签。

模型发现数据结构。

例子：

```text
用户行为 -> 用户分群
高维数据 -> 低维表示
文档集合 -> 主题结构
```

### 8.3 强化学习

智能体与环境互动，通过奖赏学习策略。

例子：

```text
游戏 AI
机器人控制
自动驾驶决策
```

强化学习和监督学习最大区别是：没有每一步标准答案，只有行动后的奖赏。

## 9. 判断一个问题能否用机器学习

可以问 6 个问题：

1. 是否有明确任务？
2. 是否有可获得的数据？
3. 数据中是否包含与任务相关的信息？
4. 是否有评价指标？
5. 未来数据是否与历史数据相似？
6. 模型错误是否可接受或可控制？

如果这些问题答不清，不要急着建模。

## 10. 本知识点总结

机器学习可以用一句话概括：

```text
机器学习是让计算机从经验数据中学习模型，并用模型在新情况上做预测或决策。
```

真正理解这句话，要理解：

- 经验通常是数据。
- 模型是从数据中学到的规律。
- 学习算法负责从数据中产生模型。
- 性能指标定义什么叫好。
- 泛化能力才是最终目标。

如果你能把一个现实问题拆成：

```text
任务 T
经验 E
性能 P
输入 x
输出 y
模型 f
评估指标
```

说明你已经开始用机器学习的方式思考问题。
