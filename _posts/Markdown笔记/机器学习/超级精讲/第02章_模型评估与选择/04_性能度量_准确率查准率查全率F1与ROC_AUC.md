# 性能度量：准确率、查准率、查全率、F1、ROC 与 AUC

## 1. 为什么指标选择非常关键

性能度量定义了什么叫“模型好”。

如果指标选错，模型可能朝错误方向优化。

例如医疗筛查中，漏掉病人很严重。如果只追求准确率，模型可能倾向于把大多数人预测为健康，因为健康人占多数。准确率很高，但临床价值很差。

所以评价模型前必须先问：

```text
业务真正关心什么错误？
哪种错误代价更高？
```

## 2. 错误率与准确率

错误率：

```text
错误率 = 分类错误样本数 / 总样本数
```

准确率：

```text
准确率 = 分类正确样本数 / 总样本数
```

二者关系：

```text
准确率 = 1 - 错误率
```

准确率适合类别比较均衡、错误代价差不多的分类任务。

## 3. 准确率为什么会失效

假设欺诈交易检测：

```text
正常交易：9900 笔
欺诈交易：100 笔
```

一个模型永远预测“正常交易”。

它的准确率是：

```text
9900 / 10000 = 99%
```

但它抓到的欺诈交易数是 0。

这个模型没有实际价值。

所以类别不平衡时，准确率可能非常误导。

## 4. 混淆矩阵

二分类结果可以分成四类。

| 名称 | 含义 |
| --- | --- |
| TP | 真实为正，预测为正 |
| FP | 真实为负，预测为正 |
| FN | 真实为正，预测为负 |
| TN | 真实为负，预测为负 |

以疾病检测为例：

- TP：病人有病，模型判断有病。
- FP：病人没病，模型误判有病。
- FN：病人有病，模型误判没病。
- TN：病人没病，模型判断没病。

不同任务对 FP 和 FN 的容忍度不同。

## 5. 查准率 Precision

查准率公式：

```text
Precision = TP / (TP + FP)
```

含义：

```text
模型预测为正的样本中，有多少真的为正？
```

查准率高表示模型“报出来的正例比较靠谱”。

适合重视误报成本的场景。

例子：

- 垃圾邮件过滤。
- 搜索结果返回。
- 推荐高价值客户名单。

如果把正常邮件误判成垃圾邮件，用户可能错过重要邮件，所以垃圾邮件过滤不能只追求召回，还要保证查准率。

## 6. 查全率 Recall

查全率公式：

```text
Recall = TP / (TP + FN)
```

含义：

```text
真实正样本中，有多少被模型找出来？
```

查全率高表示模型“不容易漏掉正例”。

适合重视漏报成本的场景。

例子：

- 癌症筛查。
- 欺诈交易初筛。
- 安全风险预警。

医疗筛查中，漏诊代价高，所以常希望召回率高。

## 7. Precision 和 Recall 的矛盾

通常 Precision 和 Recall 存在取舍。

如果提高预测为正的门槛：

- 模型更谨慎。
- FP 减少。
- Precision 可能上升。
- 但 FN 增加，Recall 可能下降。

如果降低门槛：

- 模型更宽松。
- 找出更多正例。
- Recall 可能上升。
- 但 FP 增加，Precision 可能下降。

这就像安检：

- 查得严：危险物品不容易漏，但误报多。
- 查得松：误报少，但可能漏掉危险物品。

## 8. P-R 曲线

模型通常输出一个分数，而不是固定类别。

通过改变阈值，可以得到不同 Precision 和 Recall。

P-R 曲线展示 Precision 与 Recall 的关系。

在类别不平衡问题中，P-R 曲线通常比 ROC 更能反映模型对正类的识别能力。

## 9. F1 值

F1 是 Precision 和 Recall 的调和平均：

```text
F1 = 2 * Precision * Recall / (Precision + Recall)
```

为什么不用普通平均？

调和平均会惩罚极端不平衡。

例如：

```text
Precision = 1.0
Recall = 0.1
```

普通平均是 0.55，看起来还行。

但 F1 会较低，提醒你召回太差。

F1 适合希望同时兼顾查准和查全的场景。

## 10. F-beta

如果更重视 Recall，可用 F-beta。

beta > 1 时更重视 Recall。

beta < 1 时更重视 Precision。

例如医疗筛查可用 F2，更重视召回。

## 11. ROC 曲线

ROC 曲线横轴是假正例率 FPR，纵轴是真正例率 TPR。

真正例率：

```text
TPR = TP / (TP + FN)
```

它就是 Recall。

假正例率：

```text
FPR = FP / (FP + TN)
```

表示真实负例中有多少被误判为正。

ROC 曲线描述不同阈值下 TPR 和 FPR 的变化。

## 12. AUC

AUC 是 ROC 曲线下的面积。

一个常用直觉解释：

```text
AUC 等于随机抽一个正样本和一个负样本，
模型把正样本排在负样本前面的概率。
```

AUC 越大，模型排序能力越强。

- AUC = 0.5：接近随机。
- AUC = 1.0：完美排序。

## 13. ROC/AUC 的优点

AUC 不依赖某个固定阈值，衡量整体排序能力。

适合：

- 模型输出风险分数。
- 需要后续根据业务调阈值。
- 类别比例可能变化。

## 14. ROC/AUC 的局限

在极度类别不平衡时，ROC 可能看起来很好，但正类识别仍不理想。

因为 FPR 的分母是所有负例，当负例极多时，即使 FP 数量不少，FPR 也可能很小。

这时 P-R 曲线更敏感。

## 15. 回归任务指标

### 15.1 均方误差 MSE

```text
MSE = 平均 (预测值 - 真实值)^2
```

特点：

- 对大误差非常敏感。
- 数学性质好。

### 15.2 均方根误差 RMSE

```text
RMSE = sqrt(MSE)
```

单位与原目标一致，更容易解释。

### 15.3 平均绝对误差 MAE

```text
MAE = 平均 |预测值 - 真实值|
```

相比 MSE，对异常值不那么敏感。

## 16. 指标选择案例

### 16.1 垃圾邮件过滤

重点：不能误杀正常邮件。

关注：

- Precision。
- 用户投诉率。

### 16.2 癌症初筛

重点：尽量不漏诊。

关注：

- Recall。
- F2。

### 16.3 欺诈交易检测

重点：抓到欺诈，同时控制人工审核量。

关注：

- Recall。
- Precision。
- PR 曲线。
- 审核通过率。

### 16.4 广告点击率预估

重点：排序和概率校准。

关注：

- AUC。
- Logloss。
- 校准误差。

## 17. 阈值不是固定的

很多分类模型输出概率或分数。

阈值可以根据业务目标调整。

例如：

- 高风险医疗筛查可降低阈值，提高召回。
- 高成本人工审核可提高阈值，提高查准。

模型训练和阈值选择是两件事。

## 18. 本知识点总结

性能度量不是数学装饰，而是模型优化方向。

最重要原则：

```text
根据任务代价选择指标，而不是默认使用准确率。
```

类别均衡且错误代价相似时，准确率可用。

类别不平衡或错误代价不同，必须关注 Precision、Recall、F1、P-R 曲线、ROC/AUC 等指标。
