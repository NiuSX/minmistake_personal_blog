## 2. 判断题

### 2.1 测试集可以反复用来调参。

错误。

测试集应只用于最终评估。反复用测试集调参会导致测试集信息泄漏。

### 2.2 训练误差低一定说明模型泛化好。

错误。

训练误差低可能是过拟合。

### 2.3 类别不平衡时准确率可能误导。

正确。

多数类占比很高时，模型全预测多数类也能获得高准确率。

### 2.4 AUC 衡量的是模型排序能力。

正确。

AUC 可理解为随机正样本排在随机负样本前面的概率。

### 2.5 高偏差通常对应欠拟合。

正确。

高偏差说明模型系统性偏离真实规律。

## 3. 计算题：混淆矩阵指标

某二分类模型在测试集上结果如下：

```text
TP = 80
FP = 20
FN = 40
TN = 860
```

请计算：

1. Accuracy。
2. Precision。
3. Recall。
4. F1。

参考：

```text
Accuracy = (TP + TN) / 总数 = (80 + 860) / 1000 = 0.94
Precision = TP / (TP + FP) = 80 / 100 = 0.80
Recall = TP / (TP + FN) = 80 / 120 ≈ 0.667
F1 = 2PR/(P+R) ≈ 0.727
```

注意：准确率 94% 看起来很高，但 Recall 只有约 66.7%。如果这是疾病筛查，可能仍然不够。

## 4. 场景题：指标选择

### 4.1 癌症筛查

更重视 Recall。

原因：漏诊代价高。

### 4.2 垃圾邮件过滤

需要较高 Precision。

原因：误删正常邮件代价高。

### 4.3 欺诈交易初筛

通常同时关注 Recall 和 Precision。

原因：要抓到欺诈，也要控制人工审核成本。

### 4.4 广告点击率预估

常关注 AUC、Logloss、校准。

原因：排序和概率质量都重要。

## 5. 诊断题：过拟合还是欠拟合

### 情况 A

```text
训练准确率：60%
验证准确率：58%
```

判断：欠拟合或特征不足。

改进：

- 增加模型复杂度。
- 增加特征。
- 减弱正则化。
- 检查训练是否充分。

### 情况 B

```text
训练准确率：99%
验证准确率：75%
```

判断：过拟合。

改进：

- 增加数据。
- 增强正则化。
- 降低模型复杂度。
- 早停。

### 情况 C

```text
训练准确率：93%
验证准确率：92%
测试准确率：91.8%
```

判断：模型较稳定，泛化差距小。

## 6. 简答题

### 6.1 为什么交叉验证比单次留出法更稳定？

因为交叉验证让每个样本都有机会作为测试样本，减少单次划分偶然性。

### 6.2 为什么时间序列不能随机划分？

因为随机划分可能让未来数据进入训练集，导致未来信息泄漏。应使用过去训练、未来测试。

### 6.3 为什么模型比较要考虑显著性？

因为有限测试集上的分数有随机波动，观察到的差异可能只是偶然。
