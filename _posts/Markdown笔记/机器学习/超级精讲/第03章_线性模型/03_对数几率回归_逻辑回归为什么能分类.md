# 对数几率回归：逻辑回归为什么能分类

## 1. 逻辑回归的名字为什么容易误导

逻辑回归虽然叫“回归”，但主要用于分类，尤其是二分类。

它解决的问题是：

```text
给定样本特征，预测它属于正类的概率。
```

例如：

- 邮件是垃圾邮件的概率。
- 用户会流失的概率。
- 交易是欺诈的概率。
- 病人患病的概率。

## 2. 为什么不能直接用线性回归分类

线性回归输出是任意实数：

```text
f(x) = w^T x + b
```

但概率必须在 0 到 1 之间。

如果直接用线性回归预测概率，可能输出：

```text
-0.3
1.7
```

这没有概率意义。

逻辑回归的做法是：先算线性分数，再把分数压到 0-1。

## 3. Sigmoid 函数

Sigmoid 函数：

```text
σ(z) = 1 / (1 + e^-z)
```

它把任意实数 z 映射到 0 到 1。

特点：

- z 很大，输出接近 1。
- z 很小，输出接近 0。
- z = 0，输出 0.5。

逻辑回归：

```text
p = σ(w^T x + b)
```

p 就是正类概率。

## 4. 分类规则

常用阈值 0.5：

```text
p >= 0.5 -> 正类
p < 0.5 -> 负类
```

但阈值不是固定的。

在风险控制、医疗筛查等场景中，可以根据业务目标调整阈值。

## 5. 决策边界

当阈值为 0.5 时：

```text
σ(z) >= 0.5 等价于 z >= 0
```

所以决策边界是：

```text
w^T x + b = 0
```

这说明逻辑回归的决策边界是线性的。

## 6. 对数几率是什么意思

几率 odds：

```text
odds = p / (1 - p)
```

表示正类概率和负类概率的比值。

对数几率：

```text
log(p / (1 - p))
```

逻辑回归假设：

```text
log(p / (1 - p)) = w^T x + b
```

也就是说，特征的线性组合决定对数几率。

## 7. 损失函数直觉

逻辑回归常用交叉熵损失。

如果真实标签是 1，模型预测 p 越接近 1，损失越小；预测 p 越接近 0，损失越大。

如果真实标签是 0，模型预测 p 越接近 0，损失越小；预测 p 越接近 1，损失越大。

交叉熵会强烈惩罚“非常自信但错误”的预测。

## 8. 逻辑回归的优点

- 输出概率。
- 训练高效。
- 可解释性强。
- 可加正则化。
- 适合高维稀疏特征。
- 常作为分类基线。

## 9. 逻辑回归的局限

- 决策边界线性。
- 需要特征工程表达非线性。
- 对异常点和共线性可能敏感。
- 复杂任务上表达能力有限。

## 10. 逻辑回归和线性回归的区别

| 维度 | 线性回归 | 逻辑回归 |
| --- | --- | --- |
| 任务 | 回归 | 分类 |
| 输出 | 连续实数 | 正类概率 |
| 函数 | 线性函数 | 线性函数 + sigmoid |
| 损失 | 平方误差 | 交叉熵 |
| 边界 | 不适用 | 线性边界 |

## 11. 实践建议

1. 特征尺度差异大时做标准化。
2. 类别不平衡时调整阈值或类别权重。
3. 用 L1 可以做特征选择。
4. 用 L2 可以增强稳定性。
5. 不要只看准确率，结合 AUC、F1、Recall。

## 12. 本知识点总结

逻辑回归可以理解为：

```text
线性模型负责打分，sigmoid 负责把分数变成概率。
```

它是分类模型中的基础工具，也是理解概率分类、交叉熵、阈值决策的重要入口。
