# 剪枝处理：预剪枝与后剪枝

## 1. 为什么需要剪枝

决策树如果一直生长，会越来越复杂。

复杂树可能在训练集上表现很好，但泛化差。

剪枝的目标是：

```text
删除不必要分支，降低过拟合风险。
```

## 2. 树为什么会长歪

决策树每一步都贪心选择最优划分。

如果训练数据有噪声，树可能为了适应噪声继续划分。

例如某个坏瓜由于标签错误被标成好瓜，树可能专门长出一条分支去解释它。

这就是过拟合。

## 3. 预剪枝

预剪枝是在树生长过程中提前停止。

常见条件：

- 树深达到上限。
- 节点样本数太少。
- 划分收益低于阈值。
- 验证集性能没有提升。

优点：

- 训练快。
- 树较小。
- 防止过拟合。

缺点：

- 可能过早停止。
- 容易欠拟合。

## 4. 后剪枝

后剪枝是先生成完整树，再自底向上删除分支。

流程：

1. 先让树充分生长。
2. 从叶节点往上检查。
3. 如果剪掉某个子树后验证集表现不下降，就剪掉。

优点：

- 通常比预剪枝更稳。
- 不容易过早停止。

缺点：

- 训练成本更高。

## 5. 用验证集判断剪枝

剪枝不能只看训练集。

因为训练集总是喜欢更复杂的树。

应该看验证集：

```text
剪枝后验证集性能更好或不变 -> 可以剪
剪枝后验证集性能变差 -> 不剪
```

## 6. 剪枝和正则化的关系

剪枝本质上是一种正则化。

它限制树复杂度，减少模型对训练集噪声的适应。

其他限制复杂度的方法：

- 最大深度。
- 叶节点最小样本数。
- 最小信息增益。
- 最小基尼下降。

## 7. 实践建议

实际使用树模型时，常调：

- max_depth。
- min_samples_split。
- min_samples_leaf。
- min_impurity_decrease。

这些参数都在控制树复杂度。

## 8. 本知识点总结

剪枝解决决策树过拟合。

预剪枝是在生长前控制，后剪枝是长完后修剪。

核心判断标准不是训练集，而是验证集泛化表现。
