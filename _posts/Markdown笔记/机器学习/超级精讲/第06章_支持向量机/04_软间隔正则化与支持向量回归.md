# 软间隔、正则化与支持向量回归

## 1. 硬间隔的问题

硬间隔 SVM 要求所有训练样本都被正确分类。

现实中数据常有：

- 噪声。
- 异常值。
- 类别重叠。
- 标注错误。

强行全部分对，可能导致边界非常扭曲。

## 2. 软间隔

软间隔允许少数样本违反间隔要求。

它在两个目标之间权衡：

```text
间隔尽量大 + 错误尽量少
```

这比硬间隔更适合真实数据。

## 3. C 参数

C 控制对错误的惩罚。

### C 大

模型更不愿犯错。

结果：

- 训练集拟合更强。
- 间隔可能变窄。
- 可能过拟合。

### C 小

模型更能容忍错误。

结果：

- 间隔更宽。
- 模型更平滑。
- 可能欠拟合。

## 4. 正则化视角

软间隔 SVM 可以看成：

```text
控制模型复杂度 + 惩罚训练错误
```

这和正则化思想一致。

最大间隔本身就是一种控制复杂度的方式。

## 5. 合页损失

SVM 常对应合页损失。

直觉：

- 分类正确且离边界足够远：损失为 0。
- 分类正确但离边界太近：仍有损失。
- 分类错误：损失更大。

这说明 SVM 不只是追求分对，还追求分得有余量。

## 6. 支持向量回归 SVR

SVR 用于回归。

它不是要求预测值完全等于真实值，而是允许误差在一个 ε 管道内。

如果预测误差小于 ε，不惩罚。

超过 ε，才惩罚。

## 7. SVR 的直觉

普通回归可能希望每个点都尽量贴合。

SVR 认为：

```text
小误差可以接受，不必为它付出模型复杂度。
```

这让模型更稳健。

## 8. 本知识点总结

软间隔让 SVM 适应真实噪声数据。

C 参数控制拟合训练集和保持大间隔之间的平衡。

SVR 把间隔思想扩展到回归任务。
