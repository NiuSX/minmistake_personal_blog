# Bagging 与随机森林：通过平均降低方差

## 1. Bagging 的出发点

有些模型对训练数据非常敏感。

例如决策树，训练集稍微变化，树结构可能变化很大。

这种模型方差高。

Bagging 的思想是：

```text
训练多个有差异的模型，再把它们平均或投票。
```

平均可以降低波动。

## 2. Bootstrap 采样

Bagging 使用有放回采样。

从原始训练集中随机抽样，抽出与原训练集同样大小的新训练集。

因为有放回：

- 有些样本会重复出现。
- 有些样本不会被抽到。

每个采样集训练一个模型。

## 3. Bagging 流程

1. 进行多次 bootstrap 采样。
2. 每个采样集训练一个基学习器。
3. 分类任务用投票。
4. 回归任务用平均。

## 4. 为什么 Bagging 降低方差

多个模型如果误差不完全相同，平均后波动会变小。

就像多个测量仪器测同一个量，每个都有随机误差，取平均通常更稳定。

Bagging 对高方差模型特别有效。

## 5. 随机森林

随机森林是 Bagging 和决策树的结合。

它有两层随机：

1. 样本随机：每棵树用 bootstrap 样本。
2. 特征随机：每个节点只从随机特征子集中选划分。

## 6. 为什么还要特征随机

如果只做样本随机，树之间仍可能很相似，尤其是有强特征时。

特征随机让不同树看到不同候选特征，从而增加多样性。

多样性越高，集成越可能有效。

## 7. 随机森林优点

- 稳定。
- 不易过拟合。
- 可处理高维数据。
- 可估计特征重要性。
- 调参相对简单。
- 可并行训练。

## 8. 随机森林缺点

- 模型较大。
- 可解释性低于单棵树。
- 对极高维稀疏数据未必最佳。
- 预测速度比单树慢。

## 9. 本知识点总结

Bagging 通过训练多个不同模型并平均，主要降低方差。

随机森林在 Bagging 基础上加入特征随机，是非常实用的树集成模型。
