# 分类项目报告模板

## 一、项目背景

说明这个分类任务来自什么场景，要解决什么问题。

示例：

```text
本项目希望根据用户历史行为预测未来 30 天是否流失，用于提前识别高风险用户并进行运营干预。
```

## 二、问题定义

任务类型：二分类或多分类。

样本单位：一个用户、一封邮件、一张图片、一次交易等。

标签定义：正类和负类分别是什么。

预测时间点：预测时能使用哪些信息。

错误代价：误报和漏报哪个更严重。

## 三、数据说明

记录：

1. 数据来源。
2. 样本数量。
3. 特征数量。
4. 正负样本比例。
5. 缺失值情况。
6. 时间范围。

## 四、数据探索

需要回答：

1. 类别是否不平衡？
2. 重要特征分布如何？
3. 是否存在异常值？
4. 是否有明显泄漏特征？
5. 不同类别在关键特征上是否有差异？

## 五、特征工程

说明做了哪些处理：

1. 缺失值填充。
2. 数值标准化。
3. 类别编码。
4. 时间特征拆分。
5. 文本向量化。
6. 特征选择。

每一步都要说明原因。

## 六、数据划分

说明训练集、验证集、测试集比例。

如果有时间顺序，要说明是否按时间划分。

如果类别不平衡，要说明是否使用分层抽样。

## 七、基线模型

至少建立一个简单基线：

1. 多数类预测。
2. 逻辑回归。
3. 朴素贝叶斯。
4. 简单决策树。

记录基线指标，作为后续模型对照。

## 八、候选模型

列出尝试过的模型：

| 模型 | 主要参数 | 验证集指标 | 备注 |
|---|---|---|---|
| 逻辑回归 | C=1.0 |  |  |
| 随机森林 | n_estimators=200 |  |  |
| SVM | C=1, kernel=rbf |  |  |

## 九、评估指标

说明为什么选择这些指标。

分类常用：

1. Accuracy。
2. Precision。
3. Recall。
4. F1。
5. ROC-AUC。
6. PR-AUC。

类别不平衡时，重点说明为什么不能只看准确率。

## 十一、误差分析

分析：

1. 哪类样本最容易错？
2. 假阳性有什么共同点？
3. 假阴性有什么共同点？
4. 高置信错误样本是否存在标注问题？
5. 是否需要调整阈值？

## 十二、结论与改进

总结模型是否达到目标。

后续可改进：

1. 增加数据。
2. 优化标签定义。
3. 增加关键特征。
4. 调整阈值。
5. 尝试更合适模型。
