# 回归项目报告模板

## 一、项目背景

说明要预测的连续值是什么，以及预测结果如何使用。

示例：

```text
本项目希望根据房屋位置、面积、楼层、房龄等信息预测房屋成交价格，为定价和市场分析提供参考。
```

## 二、问题定义

样本单位是什么。

标签是什么。

预测时能获得哪些特征。

误差单位是什么。

业务更关注平均误差还是大误差。

## 三、数据说明

记录：

1. 样本数量。
2. 特征数量。
3. 标签范围。
4. 标签均值、中位数、分位数。
5. 缺失值情况。
6. 异常值情况。

## 四、数据探索

重点检查：

1. 标签分布是否偏态。
2. 是否有极端大值。
3. 关键特征与标签的关系。
4. 类别特征是否存在稀有类别。
5. 是否存在时间或区域差异。

## 五、特征工程

可能处理：

1. 缺失值填充。
2. 异常值处理。
3. 数值标准化。
4. 类别编码。
5. 标签对数变换。
6. 构造比例特征。
7. 构造时间差特征。

## 六、数据划分

说明划分方式。

如果数据有时间属性，优先按时间划分。

如果同一对象有多条记录，要避免同一对象同时出现在训练集和测试集造成泄漏。

## 七、基线模型

基线可以是：

1. 预测训练集均值。
2. 预测训练集中位数。
3. 线性回归。
4. 岭回归。

## 九、评估指标

MAE：平均绝对误差，直观易解释。

RMSE：对大误差更敏感。

R2：解释方差比例，但不能单独使用。

根据业务说明主要指标。

## 十、残差分析

必须检查：

1. 残差是否围绕 0 分布。
2. 高值样本是否系统性低估。
3. 低值样本是否系统性高估。
4. 大误差样本有什么共同点。
5. 残差是否随预测值变化。

## 十一、最终结论

说明最终模型、核心指标和业务可接受性。

## 十二、后续改进

可能方向：

1. 增加更有解释力的特征。
2. 分群建模。
3. 处理异常标签。
4. 使用更强非线性模型。
5. 增加时间、区域、交互特征。
