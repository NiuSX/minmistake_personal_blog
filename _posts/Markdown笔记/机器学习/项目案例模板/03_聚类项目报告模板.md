# 聚类项目报告模板

## 一、项目背景

说明为什么要聚类。

示例：

```text
本项目希望根据用户消费、访问、品类偏好等行为特征进行客户分群，为精细化运营提供依据。
```

## 二、聚类目标

明确聚类是为了：

1. 用户分群。
2. 商品分组。
3. 异常检测。
4. 数据探索。
5. 辅助后续监督学习。

不同目标会影响特征和算法选择。

## 三、数据说明

记录：

1. 样本数量。
2. 特征数量。
3. 特征含义。
4. 缺失值情况。
5. 异常值情况。
6. 是否有明显无关特征。

## 四、特征处理

聚类非常依赖距离，因此要说明：

1. 是否标准化。
2. 是否对偏态特征做变换。
3. 是否删除异常值。
4. 是否做降维。
5. 类别特征如何处理。

## 五、算法选择

| 算法 | 参数 | 选择理由 |
|---|---|---|
| K-means | K= | 数据是否近似团状 |
| 层次聚类 | linkage= | 是否需要层次结构 |
| DBSCAN | eps=, min_samples= | 是否关注噪声和不规则簇 |

## 六、簇数量选择

说明如何选择 K：

1. 肘部法。
2. 轮廓系数。
3. 业务解释性。
4. 稳定性。

不要只根据一个指标确定簇数。

## 七、聚类结果画像

每个簇都要写画像：

```text
簇 0：
样本占比：
关键特征：
可能命名：
业务解释：
建议动作：
```

## 八、可视化

可以使用 PCA、t-SNE 或 UMAP 做二维可视化，但要说明可视化只是辅助理解，不是严格证明。

## 九、稳定性检查

检查：

1. 换随机种子后结果是否稳定。
2. 抽样后聚类结构是否类似。
3. 参数轻微变化后簇是否大体一致。
4. 异常值是否强烈影响结果。

## 十、结论

说明最终采用几类，每类含义是什么，是否能指导实际动作。

## 十一、局限

聚类没有真实标签，结论依赖特征选择、距离度量、算法假设和参数。必须明确这些局限。
