# 01 硬件学习到底学什么

## 学习目标

学完本章，你应该能回答：

- 学硬件是不是只学画 PCB 软件？
- 做一块能工作的电路板需要哪些能力？
- 原理图、PCB、焊接、调试之间是什么关系？
- 新手应该先学什么，哪些内容可以后学？

硬件学习的核心不是“把元件连起来”，而是把一个功能需求转化成真实世界中稳定、可制造、可焊接、可测试、可维护的电路系统。

## 1. 硬件设计的完整链路

一个硬件项目通常经历：

```text
需求分析 -> 方案设计 -> 器件选型 -> 原理图 -> PCB -> 打样 -> 焊接 -> 调试 -> 测试 -> 改版
```

每一步都可能出错：

- 需求没想清楚，后面设计方向会错。
- 器件选型不合适，板子可能功能达不到。
- 原理图接错，PCB 再漂亮也没用。
- PCB 布局差，电源和信号可能不稳定。
- 封装错，元件焊不上。
- 没留测试点，调试困难。
- 没做保护，真实使用容易烧。

所以硬件学习是一套工程能力，不是单点技能。

## 2. 硬件学习的知识分层

| 层级 | 内容 | 你需要达到的能力 |
| :--- | :--- | :--- |
| 基础电学 | 电压、电流、电阻、电容、电感、功率 | 能做基本计算，知道器件为什么会发热或烧毁 |
| 元器件 | 电阻、电容、二极管、MOSFET、IC、连接器 | 会读参数，会选型，会判断能不能替代 |
| 电路模块 | 电源、复位、时钟、接口、驱动、传感器 | 能把复杂电路拆成模块理解 |
| 原理图 | 符号、网络、位号、ERC、分层图 | 能清楚表达电路连接 |
| PCB | 封装、布局、布线、铺铜、DRC、Gerber | 能把原理图变成可制造电路板 |
| 工艺 | 板厚、铜厚、线宽线距、过孔、阻焊、丝印 | 知道板厂能不能生产 |
| 装配 | 手焊、贴片、钢网、BOM、CPL | 知道元件能不能焊、怎么装 |
| 调试 | 万用表、示波器、逻辑分析仪、电源 | 能定位板子为什么不工作 |
| 可靠性 | ESD、过压、过流、热、EMI | 能让电路更接近真实产品 |
| 文档 | 原理图 PDF、BOM、版本记录、测试记录 | 能让自己和别人维护项目 |

## 3. 软件工具只是入口

常见 EDA 软件：

- KiCad
- Altium Designer
- 立创 EDA
- EasyEDA
- Eagle

它们能帮助你：

- 画原理图
- 分配封装
- 画 PCB
- 检查规则
- 导出生产文件

但软件不会自动告诉你：

- 这个芯片供电是否正确
- 这个电容是否应该靠近芯片
- 这根线是否能跑这么大电流
- 这个接口是否需要 ESD
- 这个 DC-DC 布局是否稳定
- 这个封装是否和实物一致

因此学习顺序应该是：

```text
先理解电路和器件 -> 再学习 EDA 工具 -> 再做真实 PCB 项目
```

## 4. 原理图和 PCB 的关系

原理图表达“电气连接关系”。

PCB 表达“物理实现方式”。

同一个原理图，可以画出质量差异很大的 PCB：

- 元件摆放是否合理
- 电源路径是否短
- 地回流是否连续
- 去耦是否靠近芯片
- 接口是否靠近板边
- 大电流是否走得够宽
- 信号是否远离噪声源

所以原理图正确只是第一步。PCB 质量决定电路在真实世界里是否稳定。

## 5. 为什么调试能力很重要

硬件不可能第一次永远正确。即使资深工程师，也需要调试。

调试能力包括：

- 会安全上电
- 会测电源
- 会判断短路
- 会看波形
- 会查通信
- 会区分硬件问题和软件问题
- 会记录现象和复现步骤

新手常犯的错误是：板子不工作后只凭感觉改。正确方式是按链路排查：

```text
电源 -> 复位 -> 时钟 -> 下载 -> 基础 IO -> 通信接口 -> 外设
```

## 6. 新手应该先避开的方向

这些方向门槛较高：

- 市电 AC-DC
- 高压电路
- 大功率电机驱动
- 锂电池快充和电池保护
- 射频电路
- 高速 DDR / PCIe
- 多层 HDI
- 医疗、汽车、航空硬件

不是不能学，而是不适合作为第一阶段。

初学建议从：

- 3.3V / 5V 低压电路
- LED、按键、蜂鸣器
- MCU 最小系统
- UART、I2C、SPI
- 简单传感器
- 小电流 MOSFET 控制

## 7. 学习成果应该怎么衡量

不要只用“看了多少教程”衡量学习。

更好的指标：

- 能独立看懂一个简单原理图
- 能根据数据手册画推荐外围电路
- 能画出可打样的双层 PCB
- 能导出 Gerber 并下单
- 能手焊并安全上电
- 能用万用表和示波器定位问题
- 能写出改版记录

## 实操练习

1. 找一块 Arduino、ESP32、STM32 或传感器模块，观察上面的元件。
2. 尝试按功能分区：电源、主控、接口、晶振、按键、LED、传感器。
3. 找对应芯片数据手册，看推荐电路是否和板子相似。
4. 写一页分析笔记：这块板为什么这样布局？

## 检查清单

- 我是否知道硬件学习不只是画 PCB？
- 我是否理解原理图和 PCB 的区别？
- 我是否知道打样后还需要焊接和调试？
- 我是否知道新手应先做低压低速项目？
- 我是否知道每个芯片都要看数据手册？

## 常见误区

- 误区：会用 EDA 软件就会硬件。
  纠正：EDA 是工具，核心是电路和工程判断。

- 误区：原理图连通就一定能工作。
  纠正：PCB 布局、供电、地、噪声和封装都会影响结果。

- 误区：第一次板子必须完美。
  纠正：硬件需要迭代，关键是记录问题并改版。

## 本章总结

硬件学习是一条从“理论”到“实物”的路线。你要学的不只是软件操作，而是需求、器件、电路、PCB、制造、焊接、调试和可靠性。真正的能力来自闭环：设计一块板，打出来，焊上，测它，修它，再改版。
