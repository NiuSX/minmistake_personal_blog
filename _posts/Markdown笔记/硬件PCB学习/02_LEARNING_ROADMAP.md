# 02 新手学习路线总览

## 学习目标

学完本章，你应该能制定自己的硬件学习计划，知道每个阶段学什么、练什么、达到什么程度再进入下一阶段。

硬件学习最怕两种情况：

- 一开始就学太难的高速、电源、射频，导致挫败。
- 只看教程不做实物，导致知识无法落地。

合理路线应该是由低压、低速、小功能开始，逐步增加复杂度。

## 1. 总体路线

推荐路线：

```text
安全意识
  -> 电路基础
  -> 元器件
  -> 仪器工具
  -> 面包板实验
  -> 数据手册
  -> 原理图
  -> PCB
  -> 打样焊接
  -> 调试
  -> MCU 项目
  -> 电源和接口
  -> 可靠性与工程化
```

每个阶段都要有可验证成果。

## 2. 阶段 1：安全和基础电学

建议时间：1-2 周。

学习内容：

- 电压、电流、电阻、功率
- 欧姆定律
- 串联、并联
- 电容、电感基本概念
- LED 限流
- 二极管方向
- 万用表基本使用

推荐练习：

- 用电池、电阻、LED 搭限流电路。
- 用万用表测电池电压。
- 用万用表测电阻。
- 用通断档检查导线。
- 计算不同限流电阻下 LED 电流。

通过标准：

- 能解释为什么 LED 需要限流电阻。
- 能计算一个简单电阻的功耗。
- 能判断串联和并联电路的电压电流关系。
- 会用万用表测电压和通断。

## 3. 阶段 2：常见元器件

建议时间：2-3 周。

学习内容：

- 电阻参数：阻值、精度、功率、封装
- 电容参数：容值、耐压、材质、ESR
- 二极管、LED、肖特基、TVS
- 三极管、MOSFET
- LDO、DC-DC
- 按键、开关、连接器
- 常见封装

推荐练习：

- 买一套常用电阻电容包。
- 识别 0603、0805、1206 封装。
- 用 MOSFET 控制一个小负载。
- 查一个 LDO 数据手册，画出推荐电路。

通过标准：

- 能看懂 `10k 1% 0603` 的含义。
- 能知道电容耐压为什么要留余量。
- 能区分 NPN、PNP、NMOS、PMOS 的基础用途。
- 能判断连接器引脚顺序。

## 4. 阶段 3：仪器和焊接

建议时间：1-2 周。

学习内容：

- 万用表
- 可调限流电源
- 烙铁
- 助焊剂
- 吸锡带
- 示波器基础
- 逻辑分析仪基础

推荐练习：

- 焊接排针。
- 焊接 0805 电阻电容。
- 焊接一个简单 LED 板。
- 用限流电源给电路上电。
- 用示波器看 PWM 或方波。

通过标准：

- 能安全设置限流电源。
- 能焊接常见贴片元件。
- 能识别虚焊和连锡。
- 能用示波器看基本波形。

## 5. 阶段 4：数据手册和原理图

建议时间：2-4 周。

学习内容：

- 数据手册结构
- 引脚功能
- 绝对最大额定值
- 推荐工作条件
- 典型应用电路
- 原理图符号
- 网络标签
- 电源网络
- ERC

推荐练习：

- 查一个 MCU 数据手册。
- 查一个 LDO 数据手册。
- 查一个传感器数据手册。
- 按典型应用画原理图。
- 画一个 LED + 按键 + MCU 最小系统。

通过标准：

- 能从数据手册找到供电范围。
- 能找到推荐外围电容。
- 能看懂引脚表。
- 能画清楚模块化原理图。

## 6. 阶段 5：PCB 入门

建议时间：2-4 周。

学习内容：

- PCB 层
- 封装
- 板框
- 走线
- 过孔
- 铺铜
- 设计规则
- DRC
- Gerber

推荐练习：

- 画一块 LED 控制板。
- 画一块 USB 转串口小板。
- 画一块 MCU 最小系统板。

通过标准：

- 能从原理图更新 PCB。
- 能分配封装并核对。
- 能完成双层板布线。
- 能 DRC 通过。
- 能导出 Gerber。

## 7. 阶段 6：打样、焊接和调试

建议时间：2-4 周。

学习内容：

- PCB 下单参数
- BOM
- CPL
- Gerber 检查
- 焊接顺序
- 上电流程
- 短路排查
- 接口调试

推荐练习：

- 实际下单一块简单板。
- 先只焊电源部分。
- 测 3.3V 或 5V。
- 再焊 MCU 和接口。
- 下载程序点亮 LED。
- 写调试记录。

通过标准：

- 知道第一次上电必须限流。
- 能排查电源短路。
- 能确认电源、复位、时钟。
- 能把板子调到基本工作。

## 8. 阶段 7：系统设计进阶

建议时间：长期。

学习内容：

- 电源设计
- 去耦和接地
- 通信接口
- 信号完整性
- 电源完整性
- EMI / EMC
- 热设计
- DFM / DFA / DFT
- 版本管理

推荐项目：

- 带传感器的 MCU 板
- 带 USB 的数据采集板
- 小功率电机控制板
- 小型 Buck 电源板
- 带无线模块的物联网板

通过标准：

- 能按模块规划电源树。
- 能解释关键回流路径。
- 能为接口增加保护。
- 能留测试点。
- 能写改版说明。

## 9. 每个阶段都要形成笔记

建议每个项目记录：

- 项目目标
- 原理图截图
- PCB 截图
- BOM
- 打样参数
- 焊接照片
- 上电电流
- 测试结果
- 遇到的问题
- 修改方案

这比只收藏教程更有价值。

## 实操练习

制定一个 4 周计划：

- 第 1 周：电路基础 + LED 面包板实验
- 第 2 周：元器件 + 数据手册
- 第 3 周：KiCad 原理图 + PCB
- 第 4 周：打样准备 + 焊接练习

每周至少产出一份笔记和一个可验证结果。

## 检查清单

- 我是否有阶段目标？
- 我是否从低压低速项目开始？
- 我是否每个阶段都有实物练习？
- 我是否会记录问题？
- 我是否知道什么时候该进入下一阶段？

## 常见误区

- 误区：先把所有理论学完再做板。
  纠正：理论和项目要交替推进。

- 误区：直接复制开源原理图就行。
  纠正：必须理解每个元件为什么存在。

- 误区：只画板不焊不测。
  纠正：调试是硬件学习的一半。

## 本章总结

硬件学习应按阶段推进。先安全和基础，再器件和工具，再原理图和 PCB，最后通过真实打样、焊接、调试建立闭环。不要急于做复杂板，先把简单板做稳定。
