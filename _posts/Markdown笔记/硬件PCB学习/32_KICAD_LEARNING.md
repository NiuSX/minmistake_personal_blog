# 32 KiCad 学习重点

## 学习目标

学完本章，你应该能：

- 理解 KiCad 的主要模块。
- 掌握从新建项目到导出 Gerber 的完整流程。
- 知道符号、封装、网络、规则、DRC/ERC 的作用。
- 能用 KiCad 完成一个简单 PCB 项目。

KiCad 是开源 EDA 工具，适合学习和真实项目。它的逻辑和其他 EDA 类似，学会 KiCad 后再学其他工具也会容易。

## 1. KiCad 主要模块

| 模块 | 用途 |
| :--- | :--- |
| Project Manager | 项目管理 |
| Schematic Editor | 原理图编辑 |
| Symbol Editor | 符号编辑 |
| PCB Editor | PCB 编辑 |
| Footprint Editor | 封装编辑 |
| Gerber Viewer | Gerber 查看 |
| Calculator Tools | 计算工具 |

## 2. KiCad 项目结构

一个项目通常包含：

- `.kicad_pro` 项目文件
- `.kicad_sch` 原理图
- `.kicad_pcb` PCB 文件
- 自定义符号库
- 自定义封装库
- 备份文件
- 导出文件

建议每个硬件项目单独一个文件夹。

## 3. KiCad 基本流程

```text
新建项目
  -> 画原理图
  -> 注释位号
  -> ERC
  -> 分配封装
  -> 更新 PCB
  -> 画板框
  -> 设置设计规则
  -> 摆放元件
  -> 布线
  -> 铺铜
  -> DRC
  -> 3D 查看
  -> 导出 Gerber 和 Drill
  -> Gerber Viewer 检查
```

## 4. 原理图编辑重点

必须掌握：

- 放置元件符号。
- 连接导线。
- 使用网络标签。
- 使用电源符号。
- 添加 No Connect。
- 设置位号。
- 填写元件值。
- ERC 检查。

建议：

- 少用长线跨页面。
- 多用清楚网络名。
- 模块化排版。

## 5. 符号库

KiCad 自带大量符号，但不可能覆盖所有器件。

自定义符号时要：

- 引脚编号和数据手册一致。
- 引脚名称清楚。
- 电源脚不能漏。
- 隐藏电源脚要谨慎。
- 未用引脚也要表示。

## 6. 封装分配

符号只是逻辑，封装决定 PCB 焊盘。

分配封装要核对：

- 封装尺寸。
- 引脚编号。
- Pin 1。
- 焊盘大小。
- 孔径。
- 实物方向。

连接器封装必须特别小心。

## 7. PCB 编辑重点

必须掌握：

- 画板框。
- 移动和旋转元件。
- 切换层。
- 走线。
- 放过孔。
- 设置网类。
- 铺铜。
- 添加丝印。
- DRC。
- 3D Viewer。

## 8. Board Setup

在 Board Setup 中设置：

- 层数。
- 设计规则。
- 线宽。
- 过孔。
- 网类。
- 阻焊间距。
- 差分线规则。

先按板厂常规能力设置，不要默认乱用。

## 9. Net Classes

不同网络可以使用不同规则。

例子：

| 网类 | 用途 | 线宽 |
| :--- | :--- | :--- |
| Signal | 普通信号 | 0.20mm |
| Power | 电源 | 0.50mm |
| HighCurrent | 大电流 | 1.00mm+ |
| USB | 差分线 | 按阻抗 |

这样布线时更不容易把电源线画得太细。

## 10. 铺铜 Zone

常用于 GND。

步骤：

1. Add Filled Zone。
2. 选择 GND 网络。
3. 画区域。
4. 填充。
5. 检查孤岛。

修改走线后要重新填充区域。

## 11. ERC 和 DRC

ERC 检查原理图电气连接。

DRC 检查 PCB 设计规则。

必须处理：

- 真实错误。
- 未连接。
- 间距不足。
- 规则不满足。

对于误报，应理解原因，不要盲目忽略。

## 12. 3D Viewer

3D 查看用于检查：

- 连接器方向。
- 元件高度。
- 按键位置。
- 安装孔。
- 外壳干涉。

如果没有 3D 模型，也要至少检查封装和丝印。

## 13. 导出 Gerber

导出：

- Copper layers。
- Solder mask。
- Silkscreen。
- Board outline。
- Drill files。

导出后用 Gerber Viewer 打开检查。

## 14. KiCad 新手常见问题

- 符号和封装没有关联。
- 封装分配错。
- 原理图改了但 PCB 没更新。
- 铺铜没重新填充。
- DRC 规则没按板厂设置。
- Gerber 没检查。
- 连接器方向错。

## 实操练习

用 KiCad 做 LED 小板：

1. 新建项目。
2. 画原理图。
3. 放电源接口、LED、电阻。
4. 分配封装。
5. ERC。
6. 更新 PCB。
7. 画 30mm x 20mm 板框。
8. 布线和铺 GND。
9. DRC。
10. 导出 Gerber。

## 检查清单

- 项目文件是否单独管理？
- 原理图是否 ERC 通过？
- 每个元件是否有封装？
- 封装是否核对数据手册？
- PCB 是否 DRC 通过？
- Gerber 是否查看？
- 板上是否有版本号？

## 常见误区

- 误区：KiCad 自带库都可信。
  纠正：关键器件必须核对封装。

- 误区：PCB 会自动跟随原理图更新。
  纠正：需要主动 Update PCB from Schematic。

- 误区：DRC 警告可以全忽略。
  纠正：每个警告都要知道原因。

## 本章总结

KiCad 学习重点是理解 EDA 流程：原理图、封装、PCB、规则检查和制造文件。把一个简单项目完整走通，比零散学按钮更有效。
