# 技术文章写作模板

本文档用于新建 `_posts/` 下的中文技术文章，保证标题、元信息、结构和发布入口保持一致。

## 文件命名

```text
_posts/YYYY-MM-DD-主题名称.md
```

文件名使用发布日期和简短主题，避免使用无法判断内容的名称。

## Front matter

```yaml
---
title: "准确描述文章主题的标题"
date: 2026-07-19
categories:
  - 技术方向
tags:
  - 具体技术
excerpt: "帮助读者判断文章内容的简短摘要。"
---
```

### 字段规则

- `title`：必须准确描述主题，必要时包含版本、场景或问题类型。
- `date`：使用可排序的有效日期。
- `categories`：使用稳定的内容领域，一篇文章通常不超过两个分类。
- `tags`：使用具体技术、工具、概念或场景，保持命名一致。
- `excerpt`：说明文章解决什么问题或包含什么内容。

## 正文结构

推荐按以下顺序组织文章：

1. 问题背景或学习目标。
2. 必要概念和上下文。
3. 可验证的步骤、代码或示例。
4. 常见错误、限制和边界情况。
5. 结论或可复用的检查清单。
6. 参考资料。

## 内容检查

- 标题、日期、分类和摘要完整。
- 代码块标注正确语言，并说明关键代码的作用。
- 图片有替代文本或图注，资源路径兼容 `/blog`。
- 宽表格、长代码和长链接在移动端不会造成页面级横向滚动。
- 站内链接使用相对部署安全的路径。
- 不把未经验证的经验写成绝对结论。
- 新增文章后只需要提交文章文件，不需要手动修改首页、归档、分类、标签或搜索模板。
