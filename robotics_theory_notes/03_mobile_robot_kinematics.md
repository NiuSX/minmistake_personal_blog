# 03. 移动机器人运动学

## 学习目标

学完本章，你应该能：

- 理解差速、阿克曼、全向底盘的运动方式。
- 从速度指令推导轮速。
- 理解里程计如何由轮速积分得到。
- 知道为什么里程计会漂移。
- 能把移动机器人运动学和 Nav2、TF、控制联系起来。

## 1. 移动机器人运动学解决什么问题

运动学只关心“怎么动”，暂时不关心“为什么会这样动”。

它回答：

- 给机器人一个速度指令，它的位姿如何变化？
- 给左右轮速度，机器人会往哪里走？
- 给目标位姿，机器人应该产生什么速度？

它不重点回答：

- 电机力矩够不够？
- 轮胎会不会打滑？
- 地面摩擦如何变化？

这些属于动力学和控制问题。

## 2. 位姿和速度

二维移动机器人位姿通常写成：

```text
q = [x, y, theta]^T
```

其中：

- `x, y` 是平面位置。
- `theta` 是朝向。

速度常写成：

```text
v = 线速度
w = 角速度
```

ROS 2 中 `/cmd_vel` 常用 `geometry_msgs/Twist`：

- `linear.x`：前进速度。
- `angular.z`：绕 Z 轴旋转速度。

## 3. 差速底盘

### 3.1 结构

差速机器人有左右两个独立驱动轮。

参数：

- `r`：轮子半径。
- `L`：左右轮间距。
- `w_l`：左轮角速度。
- `w_r`：右轮角速度。

### 3.2 轮速到机器人速度

左右轮线速度：

```text
v_l = r * w_l
v_r = r * w_r
```

机器人线速度：

```text
v = (v_r + v_l) / 2
```

机器人角速度：

```text
omega = (v_r - v_l) / L
```

直觉：

- 两轮同速：直行。
- 右轮更快：向左转。
- 左右轮等速反向：原地旋转。

### 3.3 从 `/cmd_vel` 到轮速

给定 `v` 和 `omega`：

```text
v_r = v + omega * L / 2
v_l = v - omega * L / 2
```

再转成轮子角速度：

```text
w_r = v_r / r
w_l = v_l / r
```

### 3.4 位姿积分

短时间 `dt` 内：

```text
x_next = x + v * cos(theta) * dt
y_next = y + v * sin(theta) * dt
theta_next = theta + omega * dt
```

这就是简单里程计积分。

## 4. 阿克曼转向

阿克曼转向类似汽车。

特点：

- 不能原地旋转。
- 有最小转弯半径。
- 适合高速、室外、车式平台。

简化模型：

```text
omega = v / L * tan(delta)
```

其中：

- `L` 是轴距。
- `delta` 是前轮转角。

工程影响：

- 路径规划必须考虑曲率约束。
- 不能直接用差速机器人的原地旋转假设。
- 倒车和泊车会更复杂。

## 5. 全向和麦克纳姆底盘

全向底盘可以在平面上同时控制：

- 前后速度 `vx`
- 左右速度 `vy`
- 旋转速度 `omega`

优点：

- 机动性强。
- 适合狭窄空间和对接任务。

缺点：

- 轮子打滑更明显。
- 里程计误差可能更大。
- 控制和标定更复杂。

## 6. 非完整约束

差速和阿克曼机器人通常不能直接横向移动。

这叫非完整约束。

通俗理解：

- 小车可以前进、转弯。
- 但不能像平移一样横着走。

规划器必须尊重这个约束，否则路径看起来可行，真实机器人执行不了。

## 7. 里程计误差

里程计常见误差来源：

- 轮子打滑。
- 轮径标定不准。
- 左右轮距标定不准。
- 地面不平。
- 编码器分辨率有限。
- 时间戳不准。

误差特点：

- 短期连续。
- 长期累计漂移。

这就是为什么需要 AMCL、SLAM、GPS、视觉定位等外部修正。

## 8. 和 ROS 2 的关系

移动机器人常见数据流：

```text
/cmd_vel -> 底盘控制器 -> 编码器 -> odom -> TF: odom -> base_link
```

Nav2 会根据当前位姿和目标路径输出 `/cmd_vel`。底盘驱动执行速度，里程计再反馈机器人运动结果。

## 9. 常见工程问题

| 现象 | 可能理论原因 |
|---|---|
| 机器人直行偏一边 | 左右轮半径或电机响应不同 |
| 原地旋转角度不准 | 轮距 L 标定不准 |
| Nav2 路径执行时摇摆 | 控制器参数、延迟、速度限制不合理 |
| 里程计越来越偏 | 轮滑、积分误差、缺少外部定位修正 |
| 阿克曼车规划路线执行不了 | 规划器没有考虑最小转弯半径 |

## 10. 练习

1. 用 Python 模拟差速机器人，输入左右轮速度，画出轨迹。
2. 修改轮距 `L`，观察原地旋转误差。
3. 给里程计加入随机噪声，观察轨迹漂移。
4. 用 ROS 2 写一个节点，把 `/cmd_vel` 转成左右轮速度。
5. 在 Gazebo 中比较差速和阿克曼模型的运动差异。
