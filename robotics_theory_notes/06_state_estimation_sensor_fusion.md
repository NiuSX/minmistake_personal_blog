# 06. 状态估计与传感器融合

## 学习目标

学完本章，你应该能：

- 理解机器人为什么需要状态估计。
- 理解预测、观测、更新、协方差。
- 区分 KF、EKF、UKF、粒子滤波。
- 理解轮速计、IMU、GPS、LiDAR、视觉定位如何融合。
- 能排查融合后轨迹漂移、跳变、延迟和发散问题。

## 1. 什么是状态

状态是描述机器人当前情况的一组变量。

移动机器人常见状态：

```text
x = [px, py, yaw, vx, vy, yaw_rate]^T
```

无人机或 VIO 系统可能还包含：

- 3D 位置。
- 3D 速度。
- 四元数姿态。
- IMU bias。
- 重力方向。

## 2. 为什么不能只相信一个传感器

| 传感器 | 优点 | 缺点 |
|---|---|---|
| 轮速计 | 平滑、频率高、便宜 | 打滑会漂移 |
| IMU | 高频、能测角速度和加速度 | 积分会漂移，有 bias |
| GPS | 全局定位 | 室内不可用，城市峡谷误差大 |
| LiDAR 定位 | 室内可靠，适合地图匹配 | 依赖环境结构 |
| 视觉定位 | 信息丰富 | 光照、纹理、运动模糊影响大 |

状态估计的目标是利用各自优点，降低单一传感器缺陷。

## 3. 贝叶斯滤波思想

滤波一般有两步：

```text
预测：根据运动模型估计当前状态
更新：根据传感器观测修正状态
```

直觉：

- 预测告诉机器人“我大概到了哪里”。
- 观测告诉机器人“传感器看到我应该在哪里”。
- 融合后得到更可信的状态估计。

## 4. 卡尔曼滤波 KF

KF 适用于线性系统和高斯噪声。

状态模型：

```text
x_k = A * x_{k-1} + B * u + noise
```

观测模型：

```text
z_k = H * x_k + noise
```

核心变量：

- `x`：状态估计。
- `P`：状态协方差。
- `Q`：过程噪声。
- `R`：观测噪声。

工程直觉：

- `Q` 大：不太相信运动模型。
- `R` 大：不太相信传感器观测。
- `P` 大：系统认为自己不确定。

## 5. EKF 和 UKF

真实机器人通常是非线性的，所以常用 EKF 或 UKF。

### 5.1 EKF

EKF 用雅可比把非线性模型局部线性化。

优点：

- 应用广。
- 计算量相对可控。

缺点：

- 线性化误差可能导致不稳定。
- 需要正确的模型和雅可比。

### 5.2 UKF

UKF 用 sigma points 传播非线性分布。

优点：

- 对非线性系统通常比 EKF 更稳。

缺点：

- 计算量更大。
- 参数理解成本更高。

## 6. 粒子滤波

粒子滤波用很多粒子表示可能状态。

AMCL 就是移动机器人中常见的粒子滤波定位方法。

直觉：

- 每个粒子是假设的机器人位姿。
- 根据运动模型移动粒子。
- 根据激光雷达和地图匹配程度给粒子打分。
- 重新采样，保留更可信的粒子。

优点：

- 能处理多峰分布。
- 适合全局定位。

缺点：

- 粒子数多时计算量大。
- 参数不合适会粒子退化或定位跳变。

## 7. 传感器融合关键问题

### 7.1 时间同步

不同传感器时间戳不一致会导致融合错误。

现象：

- 轨迹抖动。
- TF extrapolation error。
- 高速运动时定位偏移。

### 7.2 坐标外参

传感器相对机器人本体的位置和姿态必须准确。

外参错误会导致：

- 雷达匹配偏移。
- 视觉定位方向错误。
- IMU 加速度方向不对。

### 7.3 噪声参数

滤波效果高度依赖噪声设置。

常见误区：

- 把所有传感器都设置得过于可信。
- 协方差填 0。
- 不区分位置、速度、姿态的噪声大小。

## 8. 和 ROS 2 的关系

常见融合节点：

- `robot_localization`：EKF/UKF 融合 odom、IMU、GPS 等。
- AMCL：基于粒子滤波的 2D 地图定位。
- SLAM 系统：同时做建图和定位。

移动机器人常见输出：

```text
odom -> base_link
map -> odom
```

## 9. 常见工程问题

| 现象 | 可能原因 |
|---|---|
| 融合后轨迹比单传感器更差 | 坐标系、时间戳或协方差错误 |
| 定位突然跳变 | 观测异常、粒子重采样、地图匹配错误 |
| EKF 发散 | 模型错误、噪声过小、异常观测未剔除 |
| IMU 融合后方向不对 | IMU 坐标系或重力方向错误 |
| GPS 融合后局部轨迹变差 | 坐标转换、GPS 噪声、更新频率问题 |
