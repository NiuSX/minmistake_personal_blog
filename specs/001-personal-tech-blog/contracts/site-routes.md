# 公共接口契约：站点路由与内容入口

**项目**：个人技术博客基础站点

**部署前缀**：`/blog`

本项目没有运行时 API。以下公共路由、页面行为和文章 front matter 构成站点对读者、搜索引擎和发布流程的静态接口契约。

## 1. 页面路由

| 页面 | 生产路径 | 必须行为 |
|---|---|---|
| 首页 | `/blog/` | 说明博客定位，提供最新/精选文章和主要导航入口 |
| 文章详情 | `/blog/<post-url>/` | 展示标题、日期、分类、标签、摘要、正文和适用的相关文章/参考资料 |
| 文章归档 | `/blog/year-archive/` | 按稳定时间顺序展示文章 |
| 分类总览 | `/blog/categories/` | 展示分类入口 |
| 分类详情 | `/blog/categories/<category>/` | 展示指定分类下的文章 |
| 标签总览 | `/blog/tags/` | 展示标签入口（启用时） |
| 标签详情 | `/blog/tags/<tag>/` | 展示指定标签下的文章（启用时） |
| 搜索 | `/blog/search/` | 提供搜索输入和结果展示 |
| 关于 | `/blog/about/` | 介绍作者、博客方向和联系方式/代码入口 |
| 404 | `/blog/404.html` | 说明页面不存在，并提供首页、归档和搜索入口 |
| 订阅源 | `/blog/feed.xml` | 输出可订阅的文章更新信息 |
| 站点地图 | `/blog/sitemap.xml` | 输出允许索引的页面和文章 URL |

## 2. URL 规则

- 所有站内链接必须兼容 `url: https://NiuSX.github.io` 和 `baseurl: /blog`。
- Liquid 模板中的链接使用 `relative_url` 或 `absolute_url`。
- 页面 permalink 必须稳定；已发布文章不得无理由修改 URL。
- 外部链接必须明确是外部资源，并且不能成为核心阅读的唯一入口。
- 路由不存在时返回 404 页面，而不是空白页面或未经解释的模板错误。

## 3. 文章内容契约

文章源文件位于 `_posts/`，至少包含：

```yaml
---
title: "文章标题"
date: 2026-07-19
categories:
  - 技术方向
tags:
  - 具体技术
excerpt: "简短摘要"
---
```

### 必须规则

- `title` 必须存在且准确描述文章主题。
- `date` 必须是可排序的有效日期。
- `categories` 至少包含一个稳定分类。
- `tags` 可为空，但存在时必须使用统一命名。
- `excerpt` 建议存在，用于列表和搜索摘要。
- 正文必须能够独立阅读，不依赖页面外部的隐藏说明。

## 4. 聚合页面契约

- 首页、归档、分类、标签和搜索都必须能够链接回文章详情页。
- 新增符合 front matter 约定的文章后，作者不需要手动修改这些聚合页面。
- 聚合页面必须展示足以判断相关性的标题、日期和摘要信息。
- 无结果或无内容时必须展示明确空状态和下一步入口。

## 5. 资源失败契约

- 图片失败时保留替代文本、图注或合理的占位区域。
- 第三方脚本、统计、评论、字体或嵌入失败时，文章正文和本地导航继续可用。
- 样式资源失败时，内容仍应以可阅读的语义 HTML 输出。
- 资源路径错误应在发布前通过生产 `baseurl` 构建和检查发现。

## 6. 构建与发布契约

- 本地构建命令：`bundle exec jekyll build`。
- 生产构建命令：`bundle exec jekyll build --trace --baseurl "/blog"`。
- 生成目录为 `_site/`，不得手动编辑或作为源内容提交。
- GitHub Actions 从 `master` 分支构建，并将 `_site/` 同步到发布仓库的 `/blog` 目录。
- 构建失败时不得发布不完整的站点产物。
