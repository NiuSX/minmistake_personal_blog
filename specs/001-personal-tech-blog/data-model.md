# 数据模型：个人技术博客基础站点

## 模型范围

本项目没有运行时数据库。以下模型描述 Jekyll 构建阶段读取的 Markdown、YAML、配置和静态资源，
用于统一内容格式、页面聚合和发布验证。

## 1. 文章（Article）

**来源**：`_posts/**/*.md`

**用途**：表达一篇可独立阅读、可归档、可搜索和可关联的技术内容。

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `title` | 字符串 | 是 | 页面主标题，必须准确、可检索 |
| `date` | 日期/日期时间 | 是 | 发布排序和页面日期 |
| `categories` | 字符串数组 | 是 | 一至两个稳定内容领域 |
| `tags` | 字符串数组 | 否 | 具体技术、工具、概念或场景 |
| `excerpt` | 字符串 | 建议 | 列表、搜索和社交摘要 |
| `layout` | 枚举 | 默认 | 默认使用 `single` |
| `author_profile` | 布尔值 | 默认 | 是否展示作者信息 |
| `toc` | 布尔值 | 默认 | 长文是否展示文章目录 |
| `comments` | 布尔值 | 默认 | 是否启用评论；外部评论失败不能阻断正文 |
| 正文 | Markdown | 是 | 文章标题、段落、代码、表格、图片和参考资料 |

**派生属性**：文章 URL、纯文本摘要、阅读时间、相关分类、相关标签、上一篇/下一篇和相关文章。

**验证规则**：

- `title` 不能为空，且不能只使用“学习笔记”“记录”等无法识别主题的泛化标题。
- `date` 必须可排序，不能使用无法解析的日期格式。
- `categories` 必须使用已有分类或经过维护者确认的新分类。
- `tags` 不得将整句描述作为标签；同一技术名词应保持命名一致。
- 正文中的图片、站内链接和代码文件路径必须在构建或发布前检查。
- 文章不应通过修改首页、归档或分类模板来完成常规发布。

## 2. 分类（Category）

**来源**：文章 `categories` 字段和分类归档。

**用途**：表达稳定的内容领域，支持分类聚合和导航。

**约束**：分类数量应保持可理解；同义分类必须统一，避免同时出现中英文或大小写混用的重复分类。

**关系**：一个文章可以属于一个或多个分类；一个分类可以聚合多篇文章。

## 3. 标签（Tag）

**来源**：文章 `tags` 字段和标签归档。

**用途**：表达具体技术、工具、概念或使用场景，支持更细粒度的内容发现。

**约束**：标签应短而具体；同一概念只保留一个规范写法；标签为空时文章仍必须可正常阅读。

**关系**：一个文章可以有零个或多个标签；一个标签可以关联多篇文章。

## 4. 站点页面（Site Page）

**来源**：`_pages/` 和根目录页面。

| 页面类型 | 路径 | 作用 |
|---|---|---|
| 首页 | `/blog/` | 说明博客定位，提供最新/精选文章入口 |
| 关于页 | `/blog/about/` | 介绍作者、方向和联系方式 |
| 文章归档 | `/blog/year-archive/` | 按时间浏览所有文章 |
| 分类页 | `/blog/categories/` | 按内容领域浏览文章 |
| 标签页 | `/blog/tags/` 或具体标签路径 | 按技术关键词浏览文章 |
| 搜索页 | `/blog/search/` | 搜索文章标题和内容 |
| 404 页 | `/blog/404.html` | 处理不存在的页面并提供返回路径 |

**约束**：页面必须有唯一标题、稳定 permalink 和适当的布局；页面级资源必须遵守 `baseurl`。

## 5. 导航入口（Navigation Item）

**来源**：`_data/navigation.yml`。

**字段**：`title`、`url`，以及可选的分类或标签语义标识。

**约束**：导航项名称简短明确，主要导航控制在四至六项；当前页面必须有可感知状态；站内 URL
必须使用相对部署安全的方式生成。

## 6. 静态资源（Static Asset）

**来源**：`assets/`、`imgs/`、文章图片引用和主题资源。

**类型**：样式表、脚本、图片、字体、订阅源、站点地图和代码相关资源。

**约束**：有信息价值的图片必须有替代文本或图注；资源路径必须在 `/blog` 下可解析；第三方资源
失败不得阻断正文；新增资源必须考虑体积、缓存和移动端加载。

## 7. 站点配置（Site Configuration）

**来源**：`_config.yml`。

**关键配置**：站点标题、描述、语言、`url`、`baseurl`、主题、搜索、分页、分类归档、标签归档、
插件、默认 front matter 和评论/统计开关。

**约束**：站点级规则集中维护；页面不得复制一份与全局默认冲突的配置；涉及生产配置的变更必须验证
本地构建和 GitHub Actions 构建行为。

## 生命周期与状态

文章和页面不使用运行时状态机，发布生命周期为：

1. **源文件**：Markdown/YAML/资源存在于版本控制中。
2. **构建中**：Jekyll 读取源文件并生成静态输出。
3. **构建通过**：生成页面、资源、订阅源和站点地图可检查。
4. **已发布**：工作流将生成内容同步到发布仓库的 `/blog` 目录。
5. **更新或撤回**：通过修改或删除源文件，再次构建并发布；不得直接手动修改发布产物。

## 数据关系

```text
文章 ── 多对一/多 ── 分类
文章 ── 多对多 ──── 标签
文章 ── 生成 ────── 文章详情页
文章 ── 聚合 ────── 首页、归档、分类页、标签页、搜索结果
站点配置 ── 影响 ── 所有页面、URL、默认 front matter 和发布行为
导航数据 ── 生成 ── 顶部主导航
静态资源 ── 被引用 ─ 文章、布局、include 和页面
```
