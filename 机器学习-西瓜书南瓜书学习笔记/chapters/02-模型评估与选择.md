# 第 2 章 模型评估与选择

## 学习目标

本章回答如何判断模型好坏。核心不是“算一个分数”，而是设计可信的评估流程，估计模型在未知数据上的泛化性能，并比较多个模型之间的差异是否可靠。

## 先用人话理解

模型评估就像考试制度设计。只看平时作业成绩不可靠，因为学生可能见过原题；只考一次也不一定公平，因为题目可能刚好适合某个学生。机器学习也一样：我们要设计一套尽量公平、稳定、不泄题的评估方式。

这一章最重要的意识是：评估不是附属步骤，而是机器学习实验可信度的基础。很多模型“看起来很好”，其实只是因为测试集被间接用来调参，或者数据划分泄漏了未来信息。

## 一个小例子

假设有 1000 个病人样本，其中 990 个健康，10 个患病。一个模型永远预测“健康”，准确率是 99%。这个数字看起来很高，但它一个病人都发现不了。

这时应该看：

- 查全率：患病的人里找出来多少。
- 查准率：预测患病的人里有多少真患病。
- F1：在查准率和查全率之间做折中。

所以指标没有绝对好坏，只有是否匹配任务代价。

## 经验误差与泛化误差

训练集上的平均损失称为经验误差：

```text
E_emp(f; D) = (1/m) sum_i L(f(x_i), y_i)
```

真实分布上的期望损失称为泛化误差：

```text
E_gen(f) = E_{(x,y)~P} [ L(f(x), y) ]
```

真实分布未知，所以泛化误差不能直接计算，只能通过测试集或验证策略估计。

## 过拟合与欠拟合

欠拟合通常表现为训练误差和测试误差都高，说明模型表达能力不足、特征不足或优化不充分。

过拟合通常表现为训练误差很低、测试误差明显升高，说明模型吸收了训练集噪声或偶然模式。

常见缓解过拟合的方法：

- 增加数据。
- 降低模型复杂度。
- 正则化。
- 早停。
- 数据增强。
- 集成学习。
- 使用严格验证流程。

## 评估方法

### 留出法

将数据划分为训练集和测试集。优点是简单，缺点是评估结果受划分方式影响较大。划分时应保持类别比例，分类任务常用分层采样。

类比：从题库中抽一部分做练习，另一部分当模拟考试。若抽题方式偏了，例如简单题都进了测试集，评估就会偏乐观。

### 交叉验证

K 折交叉验证将数据分成 K 份，每次用 K-1 份训练、1 份验证，最终取平均结果。

类比：把题库轮流分成多套模拟考试，每套都考一次，最后看平均成绩。这样比只考一套题更稳定。

优点：

- 比单次留出更稳定。
- 数据利用率更高。

缺点：

- 计算成本增加。
- 对时间序列、用户泄漏、重复样本等场景要额外处理。

### 自助法

从 `m` 个样本中有放回采样 `m` 次形成训练集。某个样本一次都未被抽中的概率为：

```text
(1 - 1/m)^m -> e^(-1) ≈ 0.368
```

因此约 36.8% 的样本可作为包外测试样本。

推导要点：每次采样不选中某样本的概率是 `1 - 1/m`，连续 `m` 次都不选中即为 `(1 - 1/m)^m`，当 `m` 很大时趋近 `e^-1`。

直觉：有放回抽样就像从袋子里摸球，摸完再放回。摸很多次后，有些球会被摸到多次，也会有一部分球始终没被摸到。这些没摸到的样本可以临时充当测试样本。

## 性能度量

### 分类指标

混淆矩阵：

```text
TP: 真实为正，预测为正
FP: 真实为负，预测为正
TN: 真实为负，预测为负
FN: 真实为正，预测为负
```

常用指标：

```text
准确率 Accuracy = (TP + TN) / (TP + FP + TN + FN)
查准率 Precision = TP / (TP + FP)
查全率 Recall = TP / (TP + FN)
F1 = 2 * Precision * Recall / (Precision + Recall)
```

准确率在类别不平衡时可能误导。例如 99% 样本为负类，模型全部预测负类也有 99% 准确率，但对正类毫无识别能力。

### ROC 与 AUC

ROC 横轴是假正例率 `FPR = FP/(FP+TN)`，纵轴是真正例率 `TPR = TP/(TP+FN)`。阈值从高到低移动会得到一条曲线。

AUC 可理解为：随机抽取一个正样本和一个负样本，模型给正样本打分高于负样本的概率。

### 回归指标

常用指标：

```text
MAE = (1/m) sum_i |y_i - yhat_i|
MSE = (1/m) sum_i (y_i - yhat_i)^2
RMSE = sqrt(MSE)
R^2 = 1 - sum_i (y_i-yhat_i)^2 / sum_i (y_i-ybar)^2
```

MSE 对大误差更敏感，MAE 更稳健。

## 比较检验

单次评估结果存在随机性。比较模型时应考虑统计显著性，例如：

- 交叉验证 t 检验。
- McNemar 检验。
- Friedman 检验与 Nemenyi 后续检验。

直觉：如果两个模型分数接近，而数据划分扰动即可改变胜负，就不能轻率认为某模型更优。

## 偏差方差分解

对平方损失，期望泛化误差可分解为：

```text
期望误差 = 偏差^2 + 方差 + 噪声
```

偏差衡量模型平均预测与真实函数的偏离，方差衡量模型对训练集扰动的敏感程度，噪声是任务本身不可约的不确定性。

一般规律：

- 简单模型：偏差高、方差低。
- 复杂模型：偏差低、方差高。
- 正则化：通常提高一点偏差，换取更低方差。

## 类比：射箭理解偏差和方差

把模型预测看成射箭：

- 偏差高：箭都集中在靶心旁边某个错误区域，说明瞄准方向有系统性错误。
- 方差高：箭分散得很开，说明对环境变化太敏感。
- 噪声高：靶子本身在晃，任何人都很难稳定射中。

理想模型既要平均方向准，也要对训练数据扰动不那么敏感。

## 易错点

- 测试集只能用于最终评估，不能用于反复调参。
- 交叉验证的预处理步骤必须在每一折内部拟合，不能先在全数据上标准化再切分。
- 类别不平衡任务不能只看准确率。
- 指标选择必须匹配业务代价，例如漏诊和误诊成本不同。

## 练习

1. 推导自助法中包外样本比例约为 36.8%。
2. 手算一个混淆矩阵的 Precision、Recall、F1。
3. 解释 AUC 的排序含义。
4. 用偏差方差解释 KNN 中 K 值过小和过大时的表现。

## 长讲义扩展：模型评估是一套实验制度

很多初学者把模型评估理解成“跑完模型之后算一个准确率”。这种理解太浅。准确率只是一个数字，真正重要的是这个数字是否可信。可信的分数来自可信的实验制度：数据如何划分、预处理是否泄漏、指标是否匹配任务、比较是否具有统计意义。

评估流程的设计直接影响结论。例如同一个模型，在随机划分一次时可能得到 91%，换个随机种子可能变成 87%。如果你只报告最好的一次，就相当于挑了一套最适合自己的考卷。交叉验证、自助法、显著性检验等方法的作用，就是降低偶然划分带来的误判。

## 数据划分中的常见泄漏

数据泄漏是模型评估中最隐蔽也最严重的问题。它指模型在训练过程中看到了本不该看到的信息。常见泄漏包括：

- 先在全数据上标准化，再划分训练集和测试集。
- 先用全数据做特征选择，再评估模型。
- 同一个用户的多条记录同时出现在训练集和测试集。
- 时间序列任务中，用未来数据预测过去。
- 文本去重不彻底，测试样本与训练样本高度重复。

以标准化为例，正确做法是只在训练集上计算均值和方差，再用这组统计量变换验证集和测试集。如果你在全数据上计算均值，测试集分布信息已经泄漏到训练流程中。这个泄漏可能看起来很小，但在严谨实验中是不允许的。

## 指标必须服务任务

一个模型的好坏不能脱离任务代价。垃圾邮件过滤中，把正常邮件误判为垃圾邮件可能很严重；疾病筛查中，漏掉患者可能更严重；推荐系统中，排序前几位的质量比整体分类准确率更重要。

因此，选择指标时应先问：错误的代价是什么？

如果正类很少，准确率常常不够。假设欺诈交易只有 0.1%，一个永远预测“非欺诈”的模型准确率 99.9%，但毫无业务价值。这时更应该看查全率、查准率、PR 曲线，甚至根据人工审核成本设置收益函数。

如果模型输出分数而不是固定类别，AUC、PR-AUC、排序指标会更合适。AUC 关注随机正样本是否排在随机负样本前面，适合评价排序能力；PR-AUC 在正类稀少时通常更敏感。

## 一个手算混淆矩阵示例

假设测试集中有 100 个样本，其中 20 个正类、80 个负类。模型预测结果如下：

```text
TP = 12
FP = 8
TN = 72
FN = 8
```

准确率：

```text
(12 + 72) / 100 = 0.84
```

查准率：

```text
12 / (12 + 8) = 0.60
```

查全率：

```text
12 / (12 + 8) = 0.60
```

F1：

```text
2 * 0.60 * 0.60 / (0.60 + 0.60) = 0.60
```

这个例子说明，84% 的准确率并不代表正类识别很好。若业务目标是尽量找出正类，60% 查全率可能远远不够。

## 阈值移动：同一个模型可以有不同取舍

很多分类模型输出的是概率或分数。默认阈值 0.5 只是一个习惯，并不一定最优。降低阈值会让模型更容易预测正类，通常提高查全率、降低查准率；提高阈值则相反。

疾病筛查中，为了少漏诊，可以降低阈值，让更多人进入复检。广告投放中，为了节省预算，可以提高阈值，只投给最可能转化的人。阈值不是纯数学问题，而是业务代价问题。

## 偏差方差分解的实际意义

偏差方差分解不是只为考试推导。它提供了诊断模型的语言。

如果训练误差很高，模型连训练数据都拟合不好，优先考虑降低偏差：换更复杂模型、增加有效特征、减少过强正则、训练更久。

如果训练误差低但验证误差高，模型对训练数据过于敏感，优先考虑降低方差：增加数据、加强正则、做数据增强、降低模型复杂度、使用集成。

KNN 是理解偏差方差的好例子。`K=1` 时，每个训练点都强烈影响局部边界，模型很灵活，偏差低但方差高；`K` 很大时，边界非常平滑，方差低但可能忽略局部结构，偏差高。

## 交叉验证不是万能的

交叉验证可以提高评估稳定性，但不是任何场景都能直接使用。时间序列数据不能随机打乱，否则未来信息会进入训练集。用户行为数据要按用户划分，否则同一用户的行为模式会同时出现在训练和测试。推荐系统中还要注意交互时间，不能用用户未来点击预测过去。

因此，数据划分方式必须模拟真实部署场景。模型上线后会遇到什么样的新数据，测试集就应尽量扮演那种新数据。

## 模型比较要避免“调参不公平”

比较两个模型时，若一个模型仔细调了几十组参数，另一个模型只用默认参数，结论并不公平。调参次数越多，越可能在验证集上碰到好运气。因此严格实验需要固定搜索空间、调参预算和评估协议。

在论文或项目报告中，应该说明：数据划分方式、随机种子、预处理流程、超参数搜索范围、最终指标和置信区间。没有这些信息，单个分数很难复现，也很难判断是否可信。

## 本章学习重点回收

学完本章后，应形成三个核心观念：

第一，泛化误差无法直接看到，只能通过合理评估流程估计。第二，指标必须匹配任务，不能迷信准确率。第三，模型选择本身也会过拟合验证集，所以测试集必须保持独立。

如果把机器学习项目比作造桥，训练算法是在设计桥梁结构，模型评估就是压力测试。没有严谨压力测试，桥看起来再漂亮也不能放心使用。

## 万字版补充：评估流程的完整闭环

一个严谨的模型评估闭环通常包含六步：

1. 明确业务目标和错误代价。
2. 定义训练、验证、测试数据的时间和来源。
3. 确定预处理和特征工程只能在训练数据中拟合。
4. 选择与目标一致的指标。
5. 使用验证集或交叉验证选择模型和超参数。
6. 在最终测试集上只评估一次，并记录完整实验配置。

这套流程看起来繁琐，但它能避免很多“虚高分数”。机器学习项目中最危险的不是模型分数低，而是分数看起来很高但上线失效。

## 时间切分与随机切分

如果数据有明显时间顺序，随机切分可能不合理。比如预测用户下月是否流失，训练时不能使用未来月份的数据。正确做法通常是用较早时间训练，用较晚时间验证和测试。

随机切分适合同分布静态数据，例如许多图像分类数据集。时间切分适合金融、推荐、风控、运营、销量预测等会随时间变化的任务。

时间切分更接近真实上线，因为模型永远是用过去预测未来。它也更容易暴露分布漂移问题。

## 分层采样的意义

分类任务中，如果类别比例不均衡，随机切分可能导致验证集或测试集中少数类太少，评估不稳定。分层采样会尽量保持各划分中的类别比例一致。

例如总数据中正类占 5%，若测试集太小，随机抽样可能只抽到极少正类，导致召回率波动巨大。分层采样不能解决类别不平衡本身，但能让评估更稳定。

## 嵌套交叉验证

普通交叉验证常用于调参和估计性能，但如果同一套交叉验证既用于选择超参数，又用于报告最终性能，仍可能偏乐观。更严格的方法是嵌套交叉验证。

外层交叉验证用于估计泛化性能，内层交叉验证用于选择超参数。每个外层测试折都完全不参与内层调参。这种方法计算成本高，但在小数据和模型比较研究中更严谨。

## 置信区间比单个分数更有信息

报告模型性能时，单个数字不够。比如准确率 90%，如果测试集只有 100 个样本，和测试集有 100000 个样本，可信程度完全不同。

置信区间能表达估计不确定性。交叉验证的均值和标准差也能说明模型对数据划分的敏感程度。若两个模型平均分差 0.2%，但标准差 1%，就不能说前者一定更好。

## PR 曲线和 ROC 曲线怎么选

ROC 曲线看 TPR 和 FPR。它适合类别比较均衡，或需要整体排序能力评价的场景。

PR 曲线看 Precision 和 Recall。正类稀少时，PR 曲线往往更有意义。因为在极度不平衡任务中，即使 FPR 很低，误报数量也可能很大，Precision 会更直接反映实际使用成本。

欺诈检测、疾病筛查、故障检测、垃圾内容识别等任务通常应重点看 PR 曲线、召回率、精准率和人工审核容量。

## 校准：概率是否可信

有些模型输出概率，但这个概率未必校准。若模型对 1000 个样本都预测 0.8 概率为正，而实际约 800 个为正，则校准较好。若实际只有 500 个为正，模型过度自信。

概率校准在决策系统中很重要。例如风控模型的分数可能用于定价，医疗模型概率可能用于风险分层。常见校准方法包括 Platt scaling、isotonic regression、温度缩放等。

准确率高的模型不一定概率校准好，排序能力强的模型也不一定概率可解释。

## 评估中的样本权重

有些样本比其他样本更重要。例如高价值客户流失损失更大，严重疾病漏诊代价更高。此时可以使用加权指标或自定义收益函数。

业务中常见做法是构建成本矩阵：

```text
预测正且真实正：收益
预测正且真实负：人工或资源成本
预测负且真实正：漏判损失
预测负且真实负：无成本或小收益
```

然后根据阈值计算总收益，选择收益最大的阈值。这比盲目最大化 F1 更贴近业务。

## 模型选择中的验证集过拟合

如果你试了上百种模型、上千组超参数，最后选择验证集表现最好的一个，它可能只是碰巧适合验证集。验证集也会被过拟合。

缓解方法包括：保留最终测试集、减少无意义搜索、使用交叉验证、报告调参预算、在新时间段数据上复验。工业系统还可以通过 A/B 测试验证线上效果。

## 线上评估与离线评估

离线测试集只能模拟真实环境，不能完全替代线上评估。推荐系统中，离线点击率预测好，不一定线上用户满意度好；广告模型离线 AUC 高，不一定带来更高收入；搜索排序离线相关性高，也可能影响多样性。

线上 A/B 测试能观察真实用户反馈，但成本高、周期长，也要注意实验分流、样本量、统计显著性和长期影响。成熟团队通常结合离线评估、灰度发布和线上实验。

## 模型监控

模型上线后，评估没有结束。需要监控：

- 输入特征分布是否漂移。
- 预测分数分布是否变化。
- 关键业务指标是否下降。
- 标签回流后真实效果如何。
- 不同人群或场景下是否存在性能差异。

模型会随环境变化而老化。持续评估和再训练机制是生产系统的一部分。

## 本章自测问答

问：为什么不能在测试集上调参？

答：因为调参会让测试集信息进入模型选择过程，使测试集不再代表未知数据，最终评估偏乐观。

问：类别不平衡时为什么准确率危险？

答：多数类占比极高时，模型只预测多数类也能得到高准确率，但对少数类完全无用。

问：验证集和测试集有什么区别？

答：验证集用于模型选择和调参，测试集用于最终无偏评估。验证集会被反复使用，测试集应尽量只用一次。

问：AUC 高是否等于概率准确？

答：不等于。AUC 衡量排序能力，概率校准衡量预测概率是否对应真实频率。

## 继续扩展：一次完整评估报告应写什么

一份负责任的模型评估报告至少应包含：

- 数据来源：样本来自哪里，时间范围是什么。
- 标签定义：正负类如何定义，是否存在延迟标签。
- 划分方式：随机切分、分层切分、按时间切分还是按用户切分。
- 预处理流程：缺失值、标准化、编码、特征选择是否只在训练集拟合。
- 指标选择：为什么使用这些指标，它们对应什么业务目标。
- 基线模型：简单规则或简单模型的表现。
- 候选模型：模型类型、主要超参数、调参范围。
- 结果表：平均值、标准差、置信区间。
- 错误分析：典型 FP、FN、不同分组下表现。
- 结论：是否值得上线，风险是什么。

没有这些上下文，只给一个“准确率 95%”几乎没有意义。

## 错误分析比平均指标更有用

平均指标告诉你模型总体如何，错误分析告诉你模型为什么错。常见做法是分别抽样查看：

- 假正例：模型预测为正但真实为负。
- 假负例：模型预测为负但真实为正。
- 高置信错误：模型非常自信但错了。
- 边界样本：模型分数接近阈值的样本。

高置信错误特别值得看，因为它们可能暴露标签错误、特征泄漏、分布偏移或模型盲区。边界样本则有助于判断阈值是否合理。

## 分组评估

整体指标好，不代表所有群体都好。模型可能在某些城市、设备、人群、时间段、类别上表现很差。

例如推荐模型整体点击率预测不错，但对新用户很差；风控模型整体 AUC 高，但对小微企业样本不稳定；医学模型整体召回高，但对某年龄段患者漏诊多。

因此要做分组评估：

```text
按地区
按时间
按用户新老
按类别
按设备
按样本难度
```

分组评估能发现平均指标掩盖的问题。

## 阈值选择示例

假设一个风控模型输出违约概率。不同阈值下：

```text
阈值 0.2：召回高，但误拒很多好用户
阈值 0.5：误拒少，但漏掉部分高风险用户
阈值 0.8：只拒绝极高风险用户，但召回很低
```

最终阈值应由业务成本决定。如果坏用户造成损失很大，可以降低阈值；如果误拒好用户损失更大，可以提高阈值。阈值选择通常比模型本身更贴近业务决策。

## 模型上线前 checklist

上线前至少确认：

- 测试集没有参与训练和调参。
- 训练和线上特征计算一致。
- 所有特征在预测时可获得。
- 缺失值和异常值有兜底处理。
- 指标在关键分组上没有明显崩溃。
- 模型输出分数分布符合预期。
- 有监控、回滚和再训练方案。

机器学习不是离线 notebook 结束就完成。评估必须覆盖上线后的稳定性。

## 万字级补充：评估指标选择案例

假设你在做三个任务：垃圾邮件识别、癌症筛查、商品推荐。它们都是预测问题，但指标选择完全不同。

垃圾邮件识别中，误把正常邮件放进垃圾箱会严重影响用户，因此查准率很重要；同时也不能漏掉太多垃圾邮件，所以需要在查准率和查全率之间折中。可以用 PR 曲线选择阈值。

癌症筛查中，漏诊代价极高，因此查全率通常优先。即使误报多一些，也可以通过进一步检查确认。此时不能只看准确率，也不能简单追求查准率。

商品推荐中，用户只会看到前几个结果，因此 Top-K 排序指标更重要。即使整体分类准确率一般，只要前几位推荐质量高，系统仍可能有效。

这说明指标不是数学装饰，而是任务价值观的体现。

## 错误样本分析模板

每次模型评估后，可以建立一个错误分析表：

```text
样本 ID：
真实标签：
预测结果：
预测分数：
错误类型：FP 或 FN
关键特征：
可能原因：
是否标签错误：
是否特征缺失：
是否分布外样本：
改进建议：
```

分析几十个高置信错误，往往比盲目调参更有价值。你可能发现标签有问题、某个特征泄漏、某类样本缺失、或者业务定义本身不清楚。

## A/B 测试的基本直觉

离线评估再好，上线仍可能不符合预期。A/B 测试把用户随机分成实验组和对照组，实验组使用新模型，对照组使用旧策略，比较真实业务指标。

做 A/B 测试要注意样本量、实验周期、随机分流、互相干扰、长期指标和短期指标差异。推荐系统中，新模型可能短期点击率高，但长期留存下降，这就需要更全面的指标。
