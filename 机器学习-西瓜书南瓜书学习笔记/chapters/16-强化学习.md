# 第 16 章 强化学习

## 学习目标

强化学习研究智能体如何通过与环境交互学习策略。重点是 MDP、价值函数、Bellman 方程、动态规划、时序差分和 Q-learning。

## 先用人话理解

强化学习和监督学习最大的不同是：没有人直接告诉模型每一步的正确答案。智能体只能通过尝试动作、观察奖励，慢慢学会什么策略长期收益更高。

这像训练一个游戏玩家。你不会告诉它每个画面下唯一正确动作，而是让它玩游戏：得分增加是奖励，失败是惩罚。它要学会哪些行为虽然眼前收益小，但长期更好。

## 一个小例子：走迷宫

智能体在迷宫中移动：

- 状态：当前位置。
- 动作：上、下、左、右。
- 奖励：到达终点 +10，撞墙 -1，每走一步 -0.1。
- 策略：在每个位置选择哪个方向。

如果只看眼前奖励，智能体可能不愿走路，因为每步都有小惩罚。但如果考虑未来奖励，它会为了到达终点接受短期成本。

## 马尔可夫决策过程

MDP 通常由五元组表示：

```text
(S, A, P, R, gamma)
```

- `S`：状态空间。
- `A`：动作空间。
- `P(s'|s,a)`：状态转移概率。
- `R(s,a,s')`：奖励函数。
- `gamma`：折扣因子。

马尔可夫性表示：给定当前状态和动作，未来状态与更早历史无关。

## 策略

策略 `pi(a|s)` 表示在状态 `s` 下选择动作 `a` 的概率。

- 确定性策略：每个状态选择一个动作。
- 随机策略：每个状态给出动作分布。

强化学习目标是找到最大化期望累计回报的策略。

## 价值函数

状态价值函数：

```text
V^pi(s) = E_pi [ sum_{t=0}^infty gamma^t r_{t+1} | s_0=s ]
```

动作价值函数：

```text
Q^pi(s,a) = E_pi [ sum_{t=0}^infty gamma^t r_{t+1} | s_0=s, a_0=a ]
```

二者关系：

```text
V^pi(s) = sum_a pi(a|s) Q^pi(s,a)
```

## Bellman 方程

Bellman 期望方程：

```text
V^pi(s) = sum_a pi(a|s) sum_{s'} P(s'|s,a) [R(s,a,s') + gamma V^pi(s')]
```

最优价值函数：

```text
V*(s) = max_a sum_{s'} P(s'|s,a) [R(s,a,s') + gamma V*(s')]
```

最优动作价值函数：

```text
Q*(s,a) = sum_{s'} P(s'|s,a) [R(s,a,s') + gamma max_{a'} Q*(s',a')]
```

Bellman 方程体现了最优子结构：当前最优价值等于即时奖励加未来最优价值。

## 类比：Bellman 方程像规划路线

从一个城市去目的地，当前城市的价值取决于：

1. 从这里走下一步能得到什么即时收益或成本。
2. 到达下一个城市后，那里继续走的最佳价值。

所以长期价值可以拆成“眼前一步 + 未来价值”。这就是 Bellman 方程的核心。

## 动态规划

若环境模型 `P` 和 `R` 已知，可用动态规划。

策略迭代：

1. 策略评估：计算当前策略的价值函数。
2. 策略改进：根据价值函数贪心更新策略。
3. 重复直到策略稳定。

价值迭代：

```text
V_{k+1}(s) = max_a sum_{s'} P(s'|s,a) [R(s,a,s') + gamma V_k(s')]
```

价值迭代把策略评估和改进合并。

## 蒙特卡洛方法

蒙特卡洛方法通过完整回合采样估计回报，不需要环境模型。适合有终止状态的情景任务。

优点：简单，无模型。

缺点：必须等回合结束，方差较大。

## 时序差分学习

TD 学习结合蒙特卡洛采样和动态规划自举：

```text
V(s_t) <- V(s_t) + alpha [ r_{t+1} + gamma V(s_{t+1}) - V(s_t) ]
```

TD 误差：

```text
delta_t = r_{t+1} + gamma V(s_{t+1}) - V(s_t)
```

它不需要等回合结束，可在线更新。

## 类比：TD 像边走边修正预期

你原本以为某条路很好走，但走了一段发现堵车，就立刻下调对这条路的评价；不必等完整趟旅程结束才总结。TD 学习就是用下一步观察到的奖励和新状态价值，及时修正当前状态价值。

## Q-learning

Q-learning 是离策略 TD 控制方法：

```text
Q(s_t,a_t) <- Q(s_t,a_t) + alpha [ r_{t+1} + gamma max_a Q(s_{t+1},a) - Q(s_t,a_t) ]
```

行为策略可用于探索，更新目标使用贪心最优动作。

## 探索与利用

智能体需要在探索未知动作和利用当前最优动作之间权衡。

常用策略：

- epsilon-greedy：以 `epsilon` 概率随机探索，否则选择当前最优动作。
- softmax：按动作价值的指数权重采样。
- UCB：优先选择不确定性高的动作。

## 探索和利用的生活类比

去餐馆吃饭时，利用是继续点你最喜欢的菜，探索是尝试新菜。只利用可能错过更好的选择，只探索又可能总吃不到稳定好菜。强化学习也必须在这两者之间平衡。

## 易错点

- 奖励不是标签，强化学习从延迟反馈中学习。
- 有模型指已知转移和奖励模型，不是指使用了神经网络。
- Q-learning 是离策略，SARSA 是同策略。
- 折扣因子越大，智能体越重视远期回报。

## 长讲义扩展：强化学习和监督学习的根本不同

监督学习中，每个样本通常有明确标签。模型预测错了，马上知道正确答案。强化学习中，智能体只看到奖励，而且奖励可能延迟。走迷宫时，很多步都没有明显好坏，直到最后到达终点才知道前面选择是否有效。

这带来三个核心难题：

- 延迟反馈：当前动作的好坏可能很久以后才体现。
- 探索利用：既要尝试新动作，又要利用已知好动作。
- 序列依赖：当前动作会改变未来能看到的状态。

因此强化学习不是简单分类，而是序列决策。

## MDP 是强化学习的基本建模语言

MDP 把环境抽象为状态、动作、转移、奖励和折扣因子。它的关键假设是马尔可夫性：未来只依赖当前状态和动作，不依赖更早历史。

这个假设让问题可处理。如果当前状态已经包含所有决策所需信息，就不需要记住完整历史。例如棋盘当前局面足以决定下一步，不需要知道棋子如何走到这里。

若状态不完整，例如只看到迷宫局部视野，就变成部分可观测问题，需要记忆或信念状态。

## 价值函数的意义

价值函数不是眼前奖励，而是长期回报。一个状态当前可能没有奖励，但如果它离终点很近，价值就高。一个动作眼前奖励高，但会导致未来失败，长期价值可能低。

状态价值 `V(s)` 评估“在这个状态有多好”；动作价值 `Q(s,a)` 评估“在这个状态做这个动作有多好”。有了 `Q`，选择动作就很直接：选 `Q` 最大的动作。

## Bellman 方程的核心递归

Bellman 方程把长期价值拆成一步奖励加未来价值：

```text
当前价值 = 即时奖励 + 折扣后的下一状态价值
```

这和动态规划思想一致：大问题可以分解为一步决策和子问题。强化学习中大量算法都围绕 Bellman 方程展开。

若已知环境模型，可以用动态规划直接更新价值；若不知道模型，就通过采样经验估计价值。

## 策略迭代与价值迭代

策略迭代包含两个阶段：

1. 策略评估：当前策略下，每个状态价值是多少。
2. 策略改进：根据价值选择更好的动作。

它像先评估当前路线方案，再根据评估结果调整路线。

价值迭代则直接用最优 Bellman 更新逼近最优价值，省略完整策略评估。它通常更简洁，但每次更新都隐含一次贪心改进。

## 蒙特卡洛与 TD 的差别

蒙特卡洛方法等一整局结束后，用实际总回报更新价值。它不需要环境模型，也不需要自举，但方差较大，且必须等回合结束。

TD 学习不等结束，走一步就更新：

```text
目标 = r + gamma V(s')
```

它用下一状态的当前估计来修正当前状态，这叫自举。TD 方差较低、能在线学习，但会引入估计偏差。

## SARSA 与 Q-learning

SARSA 更新用的是实际执行的下一个动作：

```text
Q(s,a) <- Q(s,a) + alpha [r + gamma Q(s',a') - Q(s,a)]
```

因此它是同策略方法，学到的是包含探索行为的策略价值。

Q-learning 更新用的是下一状态的最大动作价值：

```text
Q(s,a) <- Q(s,a) + alpha [r + gamma max_a Q(s',a) - Q(s,a)]
```

它是离策略方法，行为可以探索，但学习目标是贪心最优策略。

在有风险路径的环境中，SARSA 可能更保守，因为它考虑探索可能带来的风险；Q-learning 更倾向于学习理想贪心路径。

## 探索为什么不可避免

如果智能体从不探索，它可能永远不知道更好的动作。比如去餐馆永远点同一道菜，就无法发现其他菜更好。若总是探索，又无法稳定利用好选择。

epsilon-greedy 是最简单方法：大多数时候选当前最优动作，少数时候随机探索。训练早期可以设较大 epsilon，后期逐渐减小。

探索是强化学习和监督学习的重要差异。监督学习的数据通常已经给定；强化学习的数据由智能体行为产生。行为策略会影响它看到什么经验。

## 奖励设计的风险

强化学习智能体会优化你给的奖励，而不一定优化你真正想要的目标。奖励设计不当，智能体可能钻空子。

例如希望机器人快速到达终点，如果只奖励速度、不惩罚碰撞，它可能学会危险路径。希望推荐系统提高点击率，如果不考虑长期满意度，模型可能推荐标题党内容。

奖励函数是任务价值观的编码，必须谨慎设计。

## 深度强化学习的联系

当状态空间很大时，表格形式的 `Q(s,a)` 不可行。深度强化学习用神经网络近似价值函数或策略。DQN 用网络近似 Q 函数，策略梯度方法直接优化策略，Actor-Critic 同时学习策略和值函数。

深度强化学习强大但训练不稳定，常需要经验回放、目标网络、归一化、奖励裁剪等技巧。

## 学习本章的主线

强化学习的核心不是某个单独公式，而是一条链路：

```text
智能体观察状态 -> 选择动作 -> 环境转移 -> 获得奖励 -> 更新对长期价值的估计
```

MDP 提供问题定义，价值函数评估长期好坏，Bellman 方程提供递归结构，TD 和 Q-learning 提供无模型学习方法。掌握这条线，就能理解大多数强化学习算法的基本骨架。

## 万字版补充：手算一次 Q-learning

假设当前状态 `s`，执行动作 `a` 后得到奖励 `r=1`，进入状态 `s'`。当前：

```text
Q(s,a) = 2.0
max_a Q(s',a) = 4.0
alpha = 0.5
gamma = 0.9
```

TD 目标：

```text
target = r + gamma * max Q(s',a) = 1 + 0.9*4 = 4.6
```

TD 误差：

```text
delta = target - Q(s,a) = 4.6 - 2.0 = 2.6
```

更新：

```text
Q(s,a) <- 2.0 + 0.5 * 2.6 = 3.3
```

这表示实际观察到的前景比原先估计更好，所以提高 `Q(s,a)`。

## 折扣因子的意义

`gamma` 控制未来奖励的重要性。`gamma=0` 时，智能体只看眼前奖励；`gamma` 接近 1 时，长期奖励影响很大。

迷宫任务中，如果 `gamma` 太低，智能体可能不愿走向远处终点，因为终点奖励被折扣得太厉害。若 `gamma` 太高，远期回报重要，但学习可能更难、价值估计更不稳定。

## 奖励稀疏问题

很多任务只有成功时才有奖励，如走到终点 +1，其余时间 0。这样智能体早期很难知道哪些动作有用，因为大多数探索都得不到反馈。

缓解方法包括奖励塑形、课程学习、模仿学习、好奇心探索、经验回放等。但奖励塑形要小心，不能改变最优策略或引导模型钻空子。

## 经验回放

DQN 使用经验回放，把智能体经历的 `(s,a,r,s')` 存入缓冲区，训练时随机采样。这样做有两个好处：

- 打破连续样本强相关。
- 提高样本利用率。

没有经验回放时，神经网络可能被最近经验强烈影响，训练不稳定。

## 目标网络

DQN 还使用目标网络计算 TD 目标。主网络频繁更新，目标网络隔一段时间同步一次。这样目标值更稳定，减少追逐移动目标带来的震荡。

这说明深度强化学习的难点不仅是函数近似，还包括训练稳定性。

## 策略梯度的直觉

价值方法先学习动作价值，再选择价值最高动作。策略梯度直接学习策略参数，让带来高回报的动作概率上升，低回报动作概率下降。

它适合连续动作空间，因为连续动作中无法枚举所有动作求最大 Q。机器人控制等任务常使用策略梯度或 Actor-Critic 方法。

## Actor-Critic

Actor-Critic 包含两个部分：

- Actor：负责选择动作，即策略。
- Critic：负责评价动作好坏，即价值函数。

Critic 提供学习信号，Actor 根据这个信号调整策略。它结合了策略方法和价值方法的优点。

## 离线强化学习

传统强化学习允许智能体在线探索，但很多现实场景不能随便试错。例如医疗治疗、金融交易、自动驾驶都不能让未成熟策略直接探索。

离线强化学习只使用历史数据学习策略。难点是模型可能选择历史数据中很少出现的动作，价值估计不可靠。需要保守约束，避免策略偏离数据分布太远。

## 强化学习适用性判断

不是所有问题都适合强化学习。如果可以构造清晰标签，监督学习通常更简单稳定。强化学习适合：

- 决策会影响未来状态。
- 奖励延迟。
- 需要长期策略优化。
- 可交互模拟或有安全探索机制。

如果只是一次性预测，不要强行使用强化学习。

## 本章自测问答

问：强化学习为什么难？

答：反馈延迟、探索利用冲突、数据由策略产生、训练分布不断变化。

问：Q-learning 为什么是离策略？

答：它更新时使用下一状态最大 Q 值，学习贪心目标策略，而行为策略可以包含探索。

问：奖励设计为什么重要？

答：智能体会优化奖励函数，奖励若不能代表真实目标，策略可能钻空子。

问：什么时候不该用强化学习？

答：如果问题是静态预测且有明确标签，监督学习通常更简单、稳定、数据效率更高。

## 继续扩展：强化学习 checklist

建模前检查：

- 是否真的是序列决策问题。
- 当前动作是否影响未来状态。
- 奖励是否能代表真实目标。
- 是否允许在线探索。
- 是否有模拟环境。
- 状态是否满足马尔可夫性。
- 动作空间是离散还是连续。

训练后检查：

- 策略是否利用了奖励漏洞。
- 是否在不同随机种子下稳定。
- 是否只在训练环境有效。
- 探索是否带来安全风险。
- 长期指标是否优于短期贪心策略。

## 常见面试题

问：价值函数和策略有什么区别？

答：价值函数评价状态或动作好坏，策略决定在状态下采取什么动作。

问：TD 和蒙特卡洛有什么区别？

答：蒙特卡洛等完整回合结束，用真实回报更新；TD 用一步奖励加估计价值在线更新。

问：为什么强化学习样本效率低？

答：需要通过交互探索获得经验，奖励可能稀疏且延迟，许多尝试对学习贡献有限。

## 深入补充：多臂老虎机作为入门模型

多臂老虎机是强化学习最简单形式。没有复杂状态，只有多个动作，每个动作有未知奖励分布。智能体要在探索和利用之间权衡。

epsilon-greedy 会以小概率随机尝试其他动作。UCB 会选择“当前估计高或不确定性高”的动作。 Thompson sampling 则从每个动作的后验分布采样，用概率方式平衡探索和利用。

多臂老虎机虽然简单，却体现了强化学习的核心矛盾：不探索就不知道更好选择，探索又会牺牲短期收益。

## 状态设计的重要性

强化学习中的状态必须包含决策所需信息。如果状态缺失关键历史，马尔可夫性不成立，学习会困难。

例如推荐系统中，只用用户年龄作为状态远远不够；还需要近期行为、长期兴趣、曝光历史、上下文等。状态设计不充分，智能体会在不完整信息下决策。

## 模拟环境

强化学习通常需要大量试错，真实环境试错成本高。因此模拟环境很重要。游戏、机器人仿真、交通仿真都可以提供训练环境。

但模拟和现实有差距，称为 sim-to-real gap。在模拟中学到的策略，现实中可能因为物理误差、传感器噪声、用户行为差异而失效。

## 离线评估困难

监督学习可以在测试集上直接比较预测和标签。强化学习策略会改变数据分布，离线评估新策略很难。历史数据来自旧策略，新策略可能选择历史中很少出现的动作。

常见方法包括重要性采样、模型化环境、保守离线 RL、线上小流量试验。评估困难是强化学习落地的重要障碍。

## 强化学习应用边界

成功场景通常有可模拟环境、明确奖励和大量交互，如游戏、机器人仿真、资源调度、广告竞价某些环节。

困难场景包括医疗、金融、自动驾驶开放道路等，因为探索风险高、奖励延迟、环境复杂且安全要求高。

## 更多自测

问：多臂老虎机和 MDP 区别是什么？

答：多臂老虎机没有状态转移或状态很简单；MDP 中动作会影响未来状态。

问：为什么奖励稀疏会难学？

答：智能体很久才得到反馈，不知道哪些早期动作贡献了最终结果。

问：离线强化学习为什么难？

答：新策略可能选择历史数据覆盖不足的动作，价值估计容易外推错误。

## 继续补充：悬崖漫步理解 SARSA 与 Q-learning

悬崖漫步是经典例子。智能体从起点走到终点，中间有一排悬崖，掉下去会得到巨大负奖励并回到起点。

Q-learning 学习的是贪心最优策略。即使行为策略探索，它更新时假设下一步会选择最优动作，因此可能学到贴着悬崖边走的最短路径。

SARSA 学习的是实际行为策略。如果 epsilon-greedy 仍会随机探索，那么贴着悬崖走有掉下去的风险，SARSA 会倾向学到离悬崖远一点的保守路径。

这个例子说明，同策略和离策略的差异不只是公式不同，还会影响策略风格。

## 强化学习章节总结

强化学习的核心是长期决策。它比监督学习更难，因为数据由策略产生，反馈延迟，探索有成本，奖励可能被钻空子。只有当动作会影响未来、且目标需要长期优化时，强化学习才真正必要。
