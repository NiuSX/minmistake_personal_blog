# 11-OCR 文字检测、识别与文档理解

## 1. OCR 是什么

OCR Optical Character Recognition，用于从图像中检测和识别文字。

完整 OCR 通常包含：

```text
图像预处理 -> 文本检测 -> 文本方向分类 -> 文本识别 -> 后处理 -> 结构化输出
```

## 2. OCR 场景

- 身份证识别。
- 发票识别。
- 表格识别。
- 车牌识别。
- 快递单。
- 工业字符。
- 仪表读数。
- 文档扫描。
- 屏幕文字。

## 3. OCR 预处理

常见方法：

- 灰度化。
- 去噪。
- 二值化。
- 倾斜矫正。
- 透视矫正。
- 去阴影。
- 增强对比度。
- 超分辨率。

文档 OCR 中，透视矫正非常重要。拍摄文档时常出现梯形变形，需要检测四角并做透视变换。

## 4. 文本检测

文本检测输出文字区域，可能是矩形框、多边形或像素 mask。

常见模型：

- CTPN。
- EAST。
- DBNet。
- PSENet。
- TextSnake。

检测难点：

- 弯曲文字。
- 倾斜文字。
- 小字。
- 低对比度。
- 复杂背景。
- 多语言。

## 5. 文本识别

文本识别把裁剪出的文字图转成字符序列。

常见模型：

- CRNN。
- Attention OCR。
- SAR。
- SVTR。
- Transformer OCR。

CRNN 典型结构：

```text
CNN 特征提取 -> RNN 序列建模 -> CTC 解码
```

## 6. CTC

CTC 用于处理输入特征序列和输出字符序列长度不一致的问题。

特点：

- 不需要逐字符位置标注。
- 适合不定长文本识别。
- 引入 blank 字符。

## 7. 方向分类

文本可能旋转 0、90、180、270 度。方向分类用于纠正文本方向。

常见场景：

- 手机拍摄。
- 文档扫描方向不固定。
- 工业字符倒置。

## 8. 后处理

后处理可以大幅提升 OCR 业务效果。

常见：

- 字典纠错。
- 正则校验。
- 日期格式校验。
- 身份证号码校验。
- 金额格式校验。
- 语言模型纠错。
- 置信度过滤。

例如车牌识别，可以用车牌规则限制字符类型。

## 9. 表格识别

表格识别需要恢复结构。

任务：

- 表格区域检测。
- 单元格检测。
- 行列关系恢复。
- 单元格 OCR。
- 合并单元格处理。

难点：

- 无线表格。
- 合并单元格。
- 倾斜拍摄。
- 表格线断裂。
- 多页文档。

## 10. 文档理解

文档理解不仅识别文字，还要理解字段含义。

例如发票：

- 发票号码。
- 日期。
- 金额。
- 买方。
- 卖方。
- 税号。

方法：

- 规则模板。
- 版面分析。
- LayoutLM 类模型。
- OCR + NLP。
- 多模态模型。

## 11. OCR 评价指标

- 文本检测 Precision/Recall/F1。
- 字符准确率。
- 行准确率。
- 字段准确率。
- 编辑距离。

业务中更关注字段准确率。例如发票金额错一个字符就是失败。

## 12. OCR 工程建议

- 拍摄质量比模型更重要。
- 尽量控制光照、角度、焦距。
- 对固定版式优先使用模板和规则。
- 对开放场景使用检测识别模型。
- 业务字段必须加格式校验。
- 保存低置信度样本用于人工复核。
