# 12-数据集、标注、训练、评估与调参

## 1. 数据决定模型上限

CV 项目中，数据质量通常比模型选择更重要。很多模型效果差，不是因为网络不够先进，而是因为：

- 数据太少。
- 场景覆盖不足。
- 标注错误。
- 类别不平衡。
- 训练集和真实场景差异大。
- 测试集不代表真实业务。

## 2. 数据采集

采集要覆盖：

- 不同光照。
- 不同角度。
- 不同距离。
- 不同背景。
- 不同设备。
- 不同天气。
- 不同批次产品。
- 正常样本和异常样本。

工业视觉要特别关注：

- 真实产线节拍。
- 光源稳定性。
- 相机曝光。
- 产品批次差异。
- 污渍、反光、遮挡。

## 3. 数据划分

常见划分：

- 训练集。
- 验证集。
- 测试集。

比例示例：

```text
70% train / 20% val / 10% test
```

注意：

- 同一视频连续帧不要同时出现在训练和测试中。
- 同一产品或同一场景数据泄漏会导致指标虚高。
- 测试集必须接近真实部署环境。

## 4. 标注类型

分类：

- 图片级标签。

检测：

- 矩形框 bbox。

分割：

- 多边形。
- mask。

关键点：

- 点坐标。
- 可见性。

OCR：

- 文本框。
- 文本内容。

## 5. 标注规范

必须明确：

- 类别定义。
- 边界框是否贴边。
- 遮挡目标是否标。
- 小目标是否标。
- 模糊目标是否标。
- 截断目标是否标。
- 重叠目标如何标。
- 缺陷边界如何画。

没有标注规范，模型会学习到混乱规则。

## 6. 常见标注工具

- LabelImg。
- Labelme。
- CVAT。
- Roboflow。
- Label Studio。
- VGG Image Annotator。

格式：

- VOC XML。
- COCO JSON。
- YOLO TXT。
- Labelme JSON。

## 7. 数据清洗

需要检查：

- 图片是否损坏。
- 标签文件是否缺失。
- 类别名是否一致。
- 框是否越界。
- 空标注是否合理。
- 重复图片。
- 错标漏标。
- 极端异常样本。

## 8. 数据增强

训练增强：

- 翻转。
- 旋转。
- 裁剪。
- 缩放。
- 颜色扰动。
- 模糊。
- 噪声。
- Mosaic。
- MixUp。

原则：

- 增强要符合真实场景。
- 不要破坏标签语义。
- 增强不是替代真实数据。

## 9. 训练参数

重要参数：

- batch size。
- learning rate。
- epoch。
- optimizer。
- weight decay。
- input size。
- augmentation。
- pretrained weights。

调参顺序建议：

1. 确认数据和标注。
2. 跑通 baseline。
3. 看训练曲线。
4. 调输入尺寸。
5. 调学习率。
6. 调增强。
7. 调模型大小。

## 11. 误差分析

训练后要看失败样本。

问题分类：

- 漏检。
- 误检。
- 类别混淆。
- 定位不准。
- 小目标失败。
- 遮挡失败。
- 低光失败。
- 特定背景失败。

每类问题对应解决策略不同：

- 漏检：增加样本、降低阈值、提高分辨率。
- 误检：增加负样本、提高阈值、清洗标签。
- 类别混淆：增加区分类特征样本、合并相似类。
- 定位不准：检查标注一致性、提高输入分辨率。

## 12. 训练曲线怎么看

训练 loss 降，验证指标不升：

- 可能过拟合。
- 验证集分布不同。
- 指标实现有问题。

训练 loss 不降：

- 学习率不合适。
- 数据或标签错误。
- 模型结构问题。
- 输入预处理错误。

训练指标很好，现场很差：

- 测试集不代表现场。
- 数据泄漏。
- 现场光照和相机不同。
- 推理预处理不一致。
