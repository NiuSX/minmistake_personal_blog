# 13-CV 模型部署、优化与工程落地

## 1. 从模型到产品

训练出高指标模型只是第一步。真正落地需要：

- 稳定读取图像或视频。
- 正确预处理。
- 快速推理。
- 后处理。
- 结果可视化。
- 异常处理。
- 日志记录。
- 版本管理。
- 性能监控。

## 2. 推理流程

典型流程：

```text
采集图像 -> 预处理 -> 模型推理 -> 后处理 -> 业务规则 -> 输出结果
```

其中预处理和训练必须一致：

- 色彩空间。
- Resize/Letterbox。
- 归一化。
- 通道顺序。
- 数据类型。

很多部署错误来自预处理不一致。

## 3. 模型格式

常见格式：

- PyTorch `.pt/.pth`。
- ONNX。
- TensorRT engine。
- OpenVINO IR。
- TFLite。
- NCNN。

ONNX 常作为中间格式，便于跨框架部署。

## 4. ONNX

ONNX 是开放神经网络交换格式。

优点：

- 跨平台。
- 可被 TensorRT、OpenVINO、ONNX Runtime 使用。
- 便于 C++/Python 部署。

导出注意：

- opset 版本。
- 动态输入尺寸。
- 后处理是否包含在模型中。
- 算子是否被目标推理框架支持。

## 5. TensorRT

TensorRT 是 NVIDIA GPU 推理加速工具。

优化方式：

- 图优化。
- 层融合。
- FP16。
- INT8 量化。
- 内存优化。

适合：

- NVIDIA GPU。
- 实时检测。
- 多路视频。

注意：

- engine 和 GPU 型号、TensorRT 版本相关。
- INT8 需要校准数据。
- 动态 shape 需要配置 profile。

## 6. OpenVINO

OpenVINO 适合 Intel CPU、核显、VPU 部署。

适用：

- 工业 PC。
- CPU 推理。
- 边缘设备。

## 7. 移动端和边缘部署

常见方案：

- TFLite。
- NCNN。
- MNN。
- CoreML。
- ONNX Runtime Mobile。

优化：

- 模型剪枝。
- 量化。
- 蒸馏。
- 使用轻量 backbone。
- 降低输入分辨率。

## 8. 性能优化

指标：

- 延迟 latency。
- 吞吐 throughput。
- FPS。
- CPU 占用。
- GPU 占用。
- 内存。
- 显存。

优化方向：

- 减小输入尺寸。
- 使用更小模型。
- FP16/INT8。
- 批处理。
- 异步流水线。
- 多线程读流。
- ROI 推理。
- 降低后处理复杂度。

## 9. 后处理优化

检测后处理包括：

- 解码预测。
- 阈值过滤。
- NMS。
- 坐标还原。

NMS 在目标多时可能成为瓶颈。可使用：

- GPU NMS。
- batched NMS。
- 降低候选框数量。
- 提前阈值过滤。

## 10. 工业视觉落地

工业项目流程：

```text
现场调研 -> 相机光源选型 -> 采集样本 -> 算法验证 -> 软件集成 -> 产线测试 -> 误报漏报迭代
```

重点：

- 光源比算法更重要。
- 相机固定后不要随意移动。
- 产品位置要稳定。
- 需要触发拍照和 PLC 通信。
- 结果要能追溯。
- 缺陷样本要持续积累。

## 11. 视频系统落地

视频项目关注：

- 拉流稳定性。
- 解码效率。
- 多路并发。
- 帧丢弃策略。
- 报警去重。
- 事件录像。
- 断线重连。
- 时间同步。

## 12. 版本管理

需要记录：

- 模型版本。
- 训练数据版本。
- 代码版本。
- 配置版本。
- 阈值。
- 部署环境。
- 指标。

没有版本记录，现场问题很难追溯。

## 13. 线上监控

需要监控：

- 推理耗时。
- 摄像头状态。
- 报警数量。
- 低置信度样本。
- 崩溃日志。
- 误报反馈。
- 漏报反馈。

CV 系统上线后仍需要持续迭代。
